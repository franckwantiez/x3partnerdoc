Propriedades Oracle

Esta função permite visualizar as principais propriedades na base de dados quando a base de dados utilizada é Oracle™.

Os dados aqui apresentados não são modificáveis.
A sua modificação deve se fazer pelos utilitários de administração próprias à base de dados utilizados.

Advertência

Esta função permite, por pedidos sobre tabelas e vistas do sistema, de dar informações técnicas sobre a base de dados. A interpretação dos resultados dados por esta função é da competência de um administrador de bases de dados, e pode depender da versão de base de dados utilizada.

Esta função permite de visualizar as propriedades da base de dados.
Estes elementos são apresentados sobre três separadores: O primeiro dá as características gerais, o segundo dá a lista dos parâmetros e opções posicionadas ao arranque da base, o terceiro dá as informações sobre a memória utilizada para a base de dados.

Cabeçalho

Permite conhecer as características da base de dados utilizada.

Nome da base de dados
  • Versão ([field] VER)

Número de versão da base de dados
  • [field] VERDES

Descrição da versão da base de dados

[Tab] Geral

O primeiro separador apresenta as propriedades gerais da base de dados.

Este ecrã é constituido de dois blocos.

Informações sobre a base de dados

Este primeiro bloco contém as principais características da base de dados.

Parâmetros NLS

Este bloco permite de visualizar as informações sobre os jogos de caracteres para a base de dados, a instância e a sessão correntes.

Nome da instância utilizada para identificar de maneira única uma instância específica quando várias instâncias partilham nomes de serviço comuns.

  • Data criação ([field] CREDAT)

  • Data de arranque ([field] STRDAT)

Data e hora de arranque da base de dados.
  • [field] STRHOU
  • Modo ARCHIVELOG ([field] LOGMOD)

Indica se o modo ARCHIVELOG está ativado ou desativado.

  • NOARCHIVELOG :
    Em modo, NOARCHIVELOG, os ficheiros de "redo log online" estão reescritos em cada vez que um ficheiro de redo log online está preenchido e que um log switch está lançado. LGWR não rescreve sobre um grupo de ficheiro de redo log tanto que o checkpoint para este grupo não está terminado.
  • ARCHIVELOG :
    Se a base de dados está configurada no modo ARCHIVELOG, então os grupos de redo preenchidos devem ser arquivados. Como todas as modificações feitas sobre a base de dados são registados nos ficheiros de redo log online, o DBA pode utilizar a salvaguarda presente sobre o disco duro e os ficheiros de redo log arquivados para restaurar a base de dados sem perder nenhum dado comité.

    Pode-se arquivar os ficheiros de redo log :
    - Manualmente.
    - Automaticamente : Metodo recomendado.

    O parâmetro de inicialização LOG_ARCHIVE_START, quando é verdade TRUE indica que a arquivação se faz automaticamente. A FALSE, indica que o DBA o faz manualmente. Em modo automático, a arquivação se faz graças ao processo ARCn, e manualmente com pedidos SQL.

 

  • Num. utilizadores ([field] NBRUSR)

Número de utilizadores definidos na base de dados.

Número de conexões activas sobre a base de dados.

[Tab] Memória

Sobre o terceiro separador, as informações afectam a utilização de menória da base de dados são apresentados.

Este ecrã dá as informações sobre as duas zonas memórias de uma base de dados Oracle.

Memória SGA (System Global Area)

É a zona de memória que contém as estruturas de dados acessíveis para todos os procesos. Ela gera a partilha dos dados dos diferentes utilizadores.

Esta zona memória está composta de vários elementos :

  • Pool partilhada : esta parametrização indica o tamanho da pool partilhada. A pool partilhada contém nomeadamente os cursores partilhados, procedimentos guardados, estruturas de controlo e tampões de mensagens de execução paralela. Os valores elevados amelhoram as performances nos sistemas multi-utilizadores. Os valores fracos utilizam menos memória. Este valor é atribuido ao parâmetro de ficheiro de inicialização SHARED_POOL_SIZE. Observação : Pool partilhada é um parâmetro dinâmico.
  • "Cache" de tampão : indica o número de tampões contidos na "cache" de tampão da base de dados. O tamanho de cada tampão situado na "cache" corresponde aquela do bloco de dados Oracle (que é especificado pelo parâmetro de inicialização DB_BLOCK_SIZE) ; por consequência, cada tampão de base de dados situada na "cache" pode conter um único bloco de dados lidos depois de um ficheiro de dados O tamanho da "cache" é limitado, então os dados do disco não podem ter a "cache". Este valor é atribuido ao parâmetro de ficheiro de inicialização DB_CACHE_SIZE(10g) ou DB_BLOCK_BUFFERS.
  • Zona de memória LARGE POOL : permite de indicar a porção de memória alocada na zona de memória LARGE POOL. Esta porção de memória é utilizada pelo servidor partilhado em tanto que memória de sessão, pela execução paralela para os tampões de mensagem e pela salvaguarda para os tampões de E/S sobre o disco. Este valor é atribuido ao parâmetro de ficheiro de inicialização LARGE_POOL_SIZE.
  • Pool Java : indica o tamanho da pool Java. Este valor é atribuido ao parâmetro de ficheiro de inicialização JAVA_POOL_SIZE.

Um gráfico apresenta a alocação de memória que cada componente SGA utiliza.

Memória PGA (Program Global Area)

É a zona memória que contem todos os dados não necessitando de ser partilhados. Ela gera o funcionamento dos diferentes processos.


Eis um esquema mostrando a utilização de cada uma das zonas memória :

Memória SGA

  • [field] TXTAUT

Gestão automática da memória partilhada (10g)

Este valor é atribuido ao parâmetro de ficheiro de inicialização SGA_TARGET.

Se SGA_TARGET está indicado, então as zonas seguintes de memória são automaticamente talhadas.

  • Buffer cache (DB_CACHE_SIZE)

  • Shared pool (SHARED_POOL_SIZE)

  • Large pool (LARGE_POOL_SIZE)

  • Java pool (SHARED_POOL_SIZE)

  • Streams pool (STREAMS_POOL_SIZE)
  • [field] FLGAUT
  • [field] TXTTOT

Tamanho total da memória SGA.

O tamanho fixo da SGA corresponde à memória alocada à zona SGA que contém informações gerais sobre o estado da base de dados e este sobre aquele da instância.

Tamanho total da SGA = pool partilhada + "cache" de tampão + zona de memória LARGE POOL + pool java + tampão de diário.
