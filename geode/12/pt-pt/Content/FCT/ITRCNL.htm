Consulta

Esta função permite consultar os dados armazenados na tabela conversão (INTERLOT) classificados estabelecimento, recipiente, parametrização conversão e lista.

De maneira detalhada, é possível consultar as conversões agrupadas por lotes e seguidamente por linhas de conversão. Um lote é um conjunto coerente de linhas conversão (Ex: Rubrica + linhas OE).

A partir desta consulta, é igualmente possível consultar as mensagens de anomalias geradas pelo tratamento (tabela INTERMES).

Um momento t, os lotes (uma rubrica e linhas correspondentes) podem tomar 4 estados:
A Tratar:
Em importação, o lote tem sido carregado correctamente tecnicamente mas ainda não foi integrado (o tratamento INTIMPORT actualizá-lo-á).
Em exportação, o lote ainda não dado lugar a um ficheiro ou não deve dado lugar à geração de um ficheiro (o tratamento INTERFIC actualizá-lo-á).

Tratado:
Em importação o lote foi integrado correctamente funcionalmente e o objecto foi criado em Warehousing.
Em exportação, o lote deu lugar a um ficheiro ou foi tratado pelo N3.

Em erro:
Em importação, o lote não foi integrado funcionalmente em Warehousing.
Em exportação, o lote não pode dar lugar um ficheiro.

Em erro tratado:
Em importação, os lotes que estavam em erro foram actualizados manualmente por esta função, de maneira mais a não ter conta durante integração ulterior

Os dados existentes são apresentados sob a forma de quadro separadores dos quais os 3 detalham os elementos.

O separador Conversões afixa uma linha por código Conversão e status, o que permite visualizar rapidamente se há erros (em especial sobre as importações).
O separador Lotes lista para a conversão seleccionada previamente os lotes correspondentes.
O separador Linhas lista os campos de um lote previamente seleccionado.
O separador Recenseamento afixa um sumário dos fluxos trocados.

Cabeçalho

Aquando da selecção de um registo no separador Conversões, os dados são apresentados na rubrica. Trata-se das informações do primeiro registo do lote da linha, as outras informações são então visualizáveis no separador Lotes.

Interface

Estabelecimento, recipiente, código de interface, nome da parametrização são afixados. A lista de estado de interface está igualmente afixada.

Lote

A referência é o identificador do lote; geralmente trata-se da referência entrada esperada, referência entrada directa, referência ordem de entrega, código artigo.

O número faz referência à identificação de cada registo. O ficheiro que faz referência contará de ficheiro gerado ou integrado. Se não foi registado, isso significa que o registo está num estado “A tratar”.

A data e hora de criação são relativas à data de integração dos dados na tabela conversão. A data e hora de modificação são relativas à integração dos dados nas tabelas vivas.

Outras condições

O processo de funcionamento das conversões é assegurado por um tratamento único chamado: CONVERSÃO. Isto é válido desde a patch n°5 que corresponde à mistura das conversões. Anteriormente, 2 tratamentos separados: INTERFIC e INTIMPORT preenchiam respectivamente as funções de carregamento ou de geração de ficheiro e integração dos dados.

O algoritmo que segue dá as principais acções e controlos efectuados em função do modo da conversão:

Importação

Para cada ficheiro detectado no directório de importação (IMPREP):

ü       Verificação se o nome do ficheiro encontra-se entre parametrizações conversão activos

Se Sim,

ü       Deslocamento do ficheiro no directório de tratamento de importação (IMPTRTREP)

ü       Controlo técnico de cada zona de registo em função do modelo de importação associado à conversão

Se controlo OK,

ü       Carregamento das zonas de cada registo do ficheiro em tabelas conversões (WI*).

ü       Deslocação do ficheiro no directório de destino final (IMBAKPREP)

ü       Adição de uma linha hora e data no ficheiro rasto ("Importação do ficheiro...") bem como uma linha para cada lote identificado ("Tratamento do lote" …").

ü       Adição de uma linha na tabela dos lotes conversão (INTERLOT) com o status
"A tratar".

Senão

ü       Adição de uma linha na tabela dos lotes conversão (INTERLOT) com o status "Erro importação".

Senão

ü       Deslocação do ficheiro no directório de erro (IMPERRREP)

Para cada lote de conversão em importação com status "A tratar":

Para cada linha do lote nas tabelas conversões

ü       Carregamento dos valores em máscaras de registo para cada zona registada

ü       Se modo criação, afectação dos valores por defeito

ü       Controlo funcional de cada zona registada

Se controlo OK,

ü       Validação da transacção

ü       Majoração do lote interface com o status "Tratado"

ü       Adição de uma linha hora e data no ficheiro rasto ("Tratamento do lote" …")

Senão,

ü       Abandono da transacção

ü       Majoração do lote interface com o status do lote interface com o status "Erro integração".

ü       Adição de uma linha na tabela das mensagens de erro (INTERMES) para cada zona em anomalia.

Exportação

 

Após cada validação do tratamento (Entrada, Saída, Transferência, Aprovisionamento,…):

 

ü       Adição de uma linha nas tabelas conversões (WI *) para cada registo em causa.

ü       Adição de uma linha na tabela dos lotes de interface (INTERLOT) com o status
"A tratar".


Se a gestão por ficheiros pratos,

ü       Incrementação do contador definido a nível da conversão ou na falta disso aquele estado unido à tabela dos lotes conversão

 

Para cada lote conversão em exportação com status "A tratar":

 

Para cada linha do lote nas tabelas conversões:

 

ü       Adição de um registo num ficheiro conversão situado no directório de tratamento exportação (EXPREP2)

Se registo OK,

ü       Deslocação do ficheiro conversão no directório exportação (EXPREP)

ü       Cópia do ficheiro no directório de salvaguarda (EXPBAKREP)

ü       Majoração do lote conversão com o status "Tratado".

Se não, Majoração do lote conversão com o status " Erro exportação".
