Ligs. de documentação

Esta função permite criar as relações documentárias utilizadas no gerador de documentação. Estas relações completam as informações do dicionário para automatizar nomeadamente dos anexos e dos pré-requisitos da documentação. Uma documentação anexo : indica, para cada tipo de documentação, as quais são as ligações possíveis, e que são os parágrafos de documentação que alimentam.

Gestão do ecrã

O bloco de cabeçalho contém um tipo de documentação e um código correspondente. Anexa-se a este tipo de um quadro de relações.

[Tab] Ecrã de registo

Cada linha do quadro representa uma ligação explorada na geração das documentações.

[Block number] 1

  • Tipo de doc. ([field] TYP)

Este tipo, definido na tabela 910, define o tipo de documentação para a qual se definem as relações.

  • Módulo ([field] MODULE)

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.
  • Cód. ([field] COD)

Este código define o código de documentação na qual são reafixados as relações.

  • Cód. activ. ([field] CODACT)

Um código actividade permite :

  • de tornar opcional um elemento do dicionário se o valor associado ao código actividade é nulo.
  • de assinar os elementos específicos desde logo são marcados por um código começando por X,Y ou Z.
  • de dimensionar um número de linhas máximas quando o código de actividade maerca os elementos de um quadro.

Assim, se o código actividade está não activo, o elemento marcado não será utilizável, e o código associado (se existe um) não será gerado nem activável.

[Grid] Ligações

  • Numero da linha ([field] NUM)
  • Tipo ([field] TYPLNK)

Este tipo de ligações faz referência à tabela diversa 913. Segundo o tipo de documentação das relações são autorizadas.

  • Chave ([field] CLELNK)

A chave de relação é ligada ao tipo previamente escolhido. Por exemplo, quando se utiliza o tipo ADO (parâmetro), regista-se neste campo o código de um parâmetro, ou o nome de uma variável local. Se se utiliza o tipo ATB (tabela), regista-se neste campo o código de uma tabela da base. Na maior parte dos casos, uma janela de selecção associada ao contexto se abre para permitir de escolher por entre os códigos possíveis. Encontrar-se-á a lista das ligações possíveis no anexo da documentação.
  • Descr. ([field] INTLNK)

Título associado ao código precedente

Botões específicos

[Block number] 1

  • [field] OBJET
  • [field] CLES

[Block number] 2

  • A partir do dossier ([field] DOSORG)

﻿Este campo permite de definir o dossier a partir do qual a ficha vai ser copiada. As sintaxes possíveis são descritas no anexo dedicado
  • Todos dossiers ([field] TOUDOS)

Esta opção permite de copiar a ficha versus todos os dossiers definidos no dicionário (tabela ADOSSIER da solução corrente).
  • Para o dossier ([field] DOSDES)

﻿Este campo permite de definir o dossier no qual a ficha vai ser copiada. As sintaxes possíveis são descritas no anexo dedicado

Este botão permite de recopiar a definição da transação depois ou versus um outro dossier.

Barra de menu

Opção / Verificação ligs.

A opção de verificação das relações percorre as relações existentes e adverte se :

  • o objecto é inexistente,
  • o módulo é diferente do objecto,
  • a documentação está inexistente,
  • o código actividade é diferente do objecto,
Opção / Expurgo ligações

A opção de expurgo das relações permite suprimir as relações religadas a:

  • uma documentação inexistente (código da documentação de cabeçalho não existe),
  • um objecto (relações pertencento ao quadro das relações).
