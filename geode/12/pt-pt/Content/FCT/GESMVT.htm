Códs. Movimento

Este ecrã permite criar, visualizar, alterar e suprimir um código movimento. É associado cada um dos fluxos e permite identificá-lo.

Exemplo: Entrada negócio, aprovisionamento fábricas, compra externa, …

Pré-requisitos

[Refer to documentation [Implementation]]

Gestão do ecrã

O registo dos códigos movimentos é realizado a partir de um quadro.

[Tab] Ecrã de registo

Para criar um código movimento, atribui-se um código que o identifica de maneira única sobre 3 caracteres alfanuméricos e um nome. Estas informações encontram-se rubrica do ecrã de gestão; reencontra-se seguidamente os elementos seguintes:

Família de tratamento

O Código Movimento é uma informação obrigatoriamente unida aos fluxos de tipo :

Entradas: registada sobre a rubrica de Entrada.

Saídas: registada sobre a rubrica de OE (Ordem de Expedição).

Ajustamentos manuais ou automáticos.

Transferências.

Por último, a geração de uma acta (conversão) é forçada por parametrização de Transmissões N3. As transferências de stocks não são objecto de uma linha.

Tipo de movimento ajustamento

Esta opção está disponível se a check box Ajustamentos manuais foi seleccionada.

O sistema permite definir vários tipos de ajustamentos:

Adição sobre recipiente (Disponível a contar do G2) Ajustamento necessário aquando da realização de um ajustamento de tipo compactação sem acumulação de fluxos.

Criação. Ajustamento quantitativo que permite criar dos stocks sem estar a passar pelo menu das Entradas, e de associar manualmente este objecto de stocks um lugar.

Qualitativo. Ajustamento qualitativo que permite alterar as informações características de um objecto, ou seja: Número de Reserva, Número de Lote, o Número de Apoio, Origem, Código Bloqueio, e Datas (entrada, fabrico, DLV, DLC, DLUO, retenção, FIFO).

Supressão. Ajustamento quantitativo que permite suprimir a totalidade dos stocks se encontram num lugar.

Aumento de quantidade. Ajustamento quantitativo que permite acrescentar dos stocks sobre um objecto já existente.

Diminuição de quantidade. Ajustamento quantitativo que permite diminuir uma parte do objecto de stocks num lugar.

[Block number] 1

  • Cod.mov. ([field] MVT)

O código movimento permite de diferenciar os movimentos de mesmo tipo.

É requerido no momento da criação de uma entrada esperada, de uma entrada (Entrada fornecedor, Entrada fábrica, Devolução cliente,...), de uma O.E., de um ajustamento, de uma transferência, de uma entrada / saída de consumíveis.... O código do movimento está verificado na tabela dos códigos de movimentos.
  • Descr. ([field] MVTDESAXX)
  • Desc. curta ([field] MVTSHOAXX)
  • Activo ([field] ENAFLG)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.

[Block number] 2

Tipo de movimento.

Tipo de ajustamento. Preenchimento 2 para os movimentos de tipo ajustamento,

o tipo indica se trata de um ajustamento em criação de stocks,

supressão de um objecto de stocks, qualitativo, em diminuição ou

aumentação de um objecto de stock. Enfim , o tipo “acréscimo sobre

conteúdo” permite de compactar um objecto sem acumulado pelo desvio de um

movimento de ajustamento.

  • Família de tratamento ([field] PROFAM)
  • Tipo ajustamento ([field] ADJTYP)
  • Tipo consumível ([field] CSBTYP)

Tipo de movimento de consumível:
Entrada: invrementação do stock de consumíveis
Saída: decrementação do stock de cosumível

Interfaces

Transmissão de uma acta para os movimentos de tipo ajustamento.

  • Transmissão N3 ([field] TRMPRO)
