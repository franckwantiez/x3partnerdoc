Afectação códigos acesso

Esta função permite afetar aos diferentes campos dos ecrãs de registo dos códigos de acesso utilizados para restringir o acesso em registo ou na afixação a certos utilizadores.

Um código de acesso é um código alfanumérico qualquer, que pode estar ou não anexado a um utilizador (no separador Acesso, uma check box a assinalar Todos os códigos de acesso permitindo um acesso global). Se não está anexado a um utilizador, todo o campo controlado por este código de acesso está inacessível ao utilizador (em consulta como registado). Se este código está presente no perfil utilizador, dispõe-se de indicadores Consulta, Modificação, Execução :

  • o direito de Consulta está ligado à possibilidade de visualizar o conteúdo de um campo de ecrã.
  • o direito de Modificação está ligado à possibilidade de modificar o conteúdo do campo de um ecrã (por pouco que a lógica aplicativa o autoriza por outro lado).
  • o direito deExecução não está utilizado neste quadro.

De referir que o direito de Modificação implica que o direito de Consulta nestes ecrãs (os códigos de acesso servindo por outro lado, este não é forçosamente o caso de outras utilizações deste código de acesso).

Todo o campo desprovido de código de acesso (o que está em princípio geral, os códigos de acesso não sendo utilizáveis que por exceção) são consultáveis e modificáveis nos limites afixados pela lógica aplicativa e as regras de acesso às funções corrrespondentes.

[Tab] Ecrã de registo

Após ter selecionado o ecrã a personalizar, vê-se aparecer, no quadro, a lista das zonas existentes neste ecrã. Pode-se então associar a cada um dentre deles um código de acesso.

[Block number] 1

  • Cód. ecrã ([field] MASQUE)

Regista-se aqui o código ecrã que se pretenda afectar os campos. Os campos registáveis aparecem no momento no quadro que segue.
  • Descr. ([field] COMMENT)

Título associado ao código precedente

[Grid]

  • Nr. ([field] NUMLIG)
  • Zona ([field] ZONE)

Esta coluna permite de fazer aparecer as zonas registáveis do ecrã, afim de modificar uma característica dada.
  • Descr. ([field] INTIT)

Título associado ao código precedente

  • Código acesso ([field] CODACC)

Regista-se aqui se necessário um código de acesso ao campo corrente do ecrã.

Observação

Em padrão, nenhum código de acesso é definido sobre os ecrãs.

Além disso, aquando de uma actualização, estes elementos, embora presentes no dicionário de dados, são respeitados.

Se deseja conhecer o código de um ecrã software, mais simples é posicionar-se em registo sobre um dos campos do ecrã, e de utilizar a tecla : O nome do campo corrente e do écran corrente então são afixados em faixa superior da caixa que permite de conhecer além disso todas as características do campo.

Observações importantes

Esta função age sobre a estrutura de ecrã que se modifica, e o registo das modificações provoca a revalidação automática de ecrã, da janela, e do tratamento associado em todas as línguas do dossier (uma caixa assinalando que estas compilações são em curso aparecem).

Quando se deseja utilizar estas funcionalidades :

  • A primeira coisa a saber é o código do ecrã que se deseja reparametrizar. Para este facto, o mais simples é de se posicionar em registo sobre os campos do ecrã em questão, depois de utilizar a função Infos Campos, acessível seja depois o menu geral de ajuda, seja pelo botão. . A janela que aparece leva o título Campo XXX / Ecrã YYY, o que permite então de dispor destas informações.
  • falta tomar guarda a não utilizar esta função sobre um ecrã gerado automaticamente pelo meio de transações (uma revalidação das transações correspondentes vai perder a modificação). Falta neste caso modificar os ecrãs modelos utilizados pela transação, e revalidar em seguida as transações que herdarão então as características assim parametrizadas.

Esta função é uma parametrização. Isto significa que as modificações feitas no quadro restam perenes mesmo em caso de evolução de versão. Com efeito, em standard, nenhuma tabela de controlo, nenhum código de acesso, nem nenhum estilo condicional está associado aos campos dos ecrãs, mesmo se estas informações fazem parte da descrição dos ecrãs. Elas não são mais afetadas por um patch qualquer que seja.
