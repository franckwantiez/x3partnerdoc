Procedim. de migração

Esta função permite descrever e sequenciar as diferentes operações unitárias (nomeadas na seguida "procedimentos") do processo de migração optimizada cilocadaas em obra na plataforma SAFE a partir da versão 6.4. Um processo de migração standard está fornecido a partir da versão 6.4.

Cada ficha criado nesta tabela representa um processo de migração unitária. As regras de sequênciamento destes processos são descritos num anexo dedicado.

É possível aqui de reacrescentar os procedimentos de migração específicos, ou de alterar (com as restrições) certas regras de sequênciamento.

É todavia recomendado de modificar precausionamento o conteúdo standard da tabela, e este para várias razões :

  • esta tabela é comum a todos os dossiers, a sua modificação impactará todos os dossiers a migrar.
  • modificar de maneira inconsequente esta tabela pode terminar a tornar o procedimento de migração incompleta ou pior a tornar inoperante.

Gestão do ecrã

O ecrã permite descrever os diferentes procedimentos que são afectados a uma etapa, uma fase, um módulo, e uma fila. Os procedimentos standard são identificados por uma check box a assinalar. As informações dadas na parte baixa do ecrã são puramente informativas. Elas permitem de saber, a título documentário, quais são as tabelas de dados impactados pelo procedimento de migração.

[Tab] Ecrã de registo

Identificação

  • Cód. ([field] CODE)

Este código identifica de maneira única a ficha em curso
  • [field] AINTIT
  • Activo ([field] ACTIF)

Esta check box a assinalar permite activar ou desactivar o procedimento de migração.

Um procedimento desactivado não será utilizado no momento do tratamento de migração.
  • Standard ([field] STANDARD)

Quando esta check box está assinalada, o procedimento é um procedimento standard : nem a etapa, nem a fase, nem o módulo não podem estar alterados.
Por outro lado, a fila é modificável

Ordem de execução

  • Etapa ([field] RANGMOD)

A etapa é o nível mais alto do sequênciamento dos procedimentos de migração. As etapas são nomeadas pelo intermédio de um menu local. Estas são as seguintes :

  • a etapa de inicialização não é utilizada em standard hoje, mas permite de desencadear os procedimentos de controlo e de tratamento preliminares diversos.
  • a etapa tronco comum se efectua em seguida. Ela está suposta de tratar todas as tabelas comuns a vários módulos, cuja migração está previamente a todo o procedimento de dados de um módulo.
  • a etapa módulo permite lançar os procedimentos de migração funcionais (na ordem das fases, depois os módulos). Um módulo está associado a cada procedimento desta etapa.
  • a etapa pós-migração é uma etapa indispensável para um funcionamento normal de um dossier migrado, mas cujos procedimentos podem ser diferentes ao preço de certos limites na utilização do dossier migrado. Definir de tais etapas e diferir o seu lançamento no momento da migração permite aos utilizadores de retomar a exploração ao mais depressa num modo degradado, em acabando a migração mais tarde, por exemplo na próxima janela de inatividade, afim de encontrar o conjunto das funcionalidade do software migrado.
  • Fase ([field] PHASE)

A fase é um número susceptível de variar de 1 a 9, que permite de sequenciar de maneira estrita os procedimentos de migração. Tanto que todos os procedimentos de uma fase são terminados, as fases seguintes não podem ser lançadas.
A fase não pode ser modificada por um procedimento standard.
  • Módulo ([field] MODULE)

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.
  • Rang ([field] RANG)

A fila é um número susceptível de variar de 1 a 99, que permite dar uma ordem preferencial de lançamento de um procedimento de migração numa fase e para um modelo dado. Esta fila não pode ser alterada para os procedimentos standard.
  • Índice global ([field] INDICEM)

Este campo unicamente afixado e um índice global calculado em função da etapa, da fase, do módulo, e da fila (nesta ordem). Este índice define a ordem na qual os procedimentos de migração são lançados, sabendo que a paralelização está somente possível sobre os procedimentos situados na mesma etapa e a última fase, e que outro lançamento de uma fase numa etapa não é possível que se as fases e etapas precedentes são todas correctamente terminadas.

Dados informativos

  • Tab. principal ([field] TABLEP)

A tabela descrita aqui é a tabela principal migrada pelo procedimento.
Esta informação está indicativa, e fornecida a título de documentação.

[Grid]

  • Tabelas anexos ([field] TABLEM)

As tabelas descritas nesta lista são as tabelas anexas colocadas em obra pelo procedimento de migração. Esta lista é indicativa, e fornecdida a título de documentação.
  • Actualização ([field] TABTYP)

Quando esta check box está assinalada, a tabela anexa correspondente está suposta de atualizar pelo procedimento de migração. Esta informação é indicativa, e fornecdida a título de documentação.
