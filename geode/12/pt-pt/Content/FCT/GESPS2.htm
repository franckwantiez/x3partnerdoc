Parâmetros estatísticas

Um utilitário geral de parametrização de estatísticas é proposto no software. Este utilitário permite criar uma base estatística num ficheiro de acumulação. Este ficheiro contém os registos sob a forma seguinte :

Código MAJ

Sociedade

Estabelecimento

Data

Critério 1

Critério 2

Critério 3

...

Montantes
(1 a 5 ou 10)

O código MAJ é um código caracteristicas utilizadas pelas listas e programas de interrogação. É possível de criar tantos códigos que necessário (cada um correspondente a um tipo de informações estatísticas)

Os códigos sociedade e estabelecimento permitem ter se necessário estatísticas detalhadas (não é obrigatório, e se este nível de detalhe não foi desejado, as zonas correspondentes restam vazias na constituição da base).

A zona data está registada com o primeiro dia do período correspondente à data de evento desencadeador (segundo o caso, será o primeiro dia da semana, da quinzena, do mês, do trimestre...)

As zonas critérios são definidas livremente em utilizando os campos das tabelas em linha (ou com partes deste campo, por exemplo os dois primeiros caracteres do código postal). Isto permite de ter as estatísticas com os eixos de análise mais ou menos detalhados. Nas versões a partir da 140, pode-se ter de 1 a 8 critérios (em versão 130, estava-se limitado a 4).

As zonas montantes contém os acumulados dos montantes que se deseja analizar em estatisticas.

A parametrização se faz por esta função, em criando as fichas desencadeando a constituição da base estatística. Uma ficha pelo menos está criada por código MAJ utilizado Cada ficha está identificada por um código único, que se confundem frequentemente com o código MAJ, excepto que se pretende duas (ou mais) fichas atualizando em dia o mesmo código MAJ (neste caso, os códigos serão diferentes).

É possível de outra maneira definir, para os parâmetros estatísticos deste tipo, se as previsões devem poder ser registadas (ou calculadas a partir do real) em face dos acumulados estatísticos.

A consulta das estatísticas pode-se fazer sobre 1 a 10 expressões calculadas a partir dos montantes acumulados sobre o período de interrogação, sobre período precedente, assim que os orçamentos registados. Se por exemplo não se deseja ver no ecrã que valores médios de um acumulado, ter-se-á tomado de acumular a expressão desejada sobre uma primeira coluna de agregação, de contar o número de agregações feitas em acumulando 1 sobre outras colunas de agregação, e de apresentar no ecrã de interrogação a relação destas duas colunas. Para consultar as estatísticas, entrar-se-á na função de consulta em indicando o código MAJ a consultar. Poder-se-á fazer zoom sobre os diferentes níveis de detalhe induzidos pelos critérios sucessivos. Os zooms finais se farão sobre o detalhe dos movimentos agregados (se estão ainda em linha).

Uma consulta contextual das estatísticas é possível depois a função Ficheiro / Estatísticas que pode ser parametrizada sobre cada objeto (existe uma check box a assinalar que a define). Quando esta função é utilizada, uma lista de códigos estatísticos acessíveis aparece, e o zoom se faz então ao nível do mais fino possível cômputo do contexto requerente. Por exemplo, no quadro de uma gestão comercial, se uma estatística foi definda por departamento (os 2 primeiros caracteres do endereço por defeito associado ao terceiro), depois por família estatística cliente, depois por cliente, depois por artigo, uma chamada às estatísticas depois uma ficha cliente autorizarão o acesso a esta estatística e se posicionará automaticamnte sobre o nível de detalhe do cliente (depois o contexto permite dar um valor por defeito ao departamento, à família cliente, e ao código cliente). Ver-se-á então diretamente aparecer o acumulado das estatística do cliente por familia artigo

Gestão do ecrã

  • Cabeçalho
  • [Tab] Geral
  • [Tab] Campos
  • [Tab] Avançado

Cabeçalho

Em cabeçalho de ficha, encontra-se um código (identificador unico para a parametrização) e um título.

  • Cód. ([field] COD)

Identifica o código estatístico corrente. A um código estatístico são reafectados um conjunto de acumulados baseados sobre um a oito caracteres cruzados.
  • Descr. ([field] INTIT)

Permite de definir um titulo associado a cada ficha.
  • Desc. curta ([field] INTSHO)

Este intitulado permite dar uma designação à ficha.
é utilizado em certos ecrãs ou listas quando não se teve suficiente local para visualizar o intitulado longo.

[Tab] Geral

Este separador define as informações principais de agregado estatístico (evento desencadeador, periodicidade, nível de definição...).

Acessibilidade

  • Activo ([field] ENAFLG)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.
  • Cód. activ. ([field] CODACT)

Um código actividade permite :

  • de tornar opcional um elemento do dicionário se o valor associado ao código actividade é nulo.
  • de assinar os elementos específicos desde logo são marcados por um código começando por X,Y ou Z.
  • de dimensionar um número de linhas máximas quando o código de actividade maerca os elementos de um quadro.

Assim, se o código actividade está não activo, o elemento marcado não será utilizável, e o código associado (se existe um) não será gerado nem activável.
  • Código acesso ([field] ACS)

Este código de acesso permite restringir o acesso à ficha corrente a certos utilizadores.
Se a zona está registada, apenas os utilizadores dispondo deste código de acesso no seu perfil podem visualizar e modificar esta ficha.

No caso das estatísticas, o direito à consulta controlada igualmente a consulta dos dados estatísticos resultantes da parametrização.
  • Módulo ([field] MODULE)

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.

Períodicidade

  • Script ([field] PERPRG)

Quando se deseja estabelecer os acumulados sobre uma periodicidade não pré-definida pelo software, pode-se definir aqui o nome de um tratamento cujos sub-programas seguintes devem ser definidos :

  • # Este sub-programa permite de rechamar uma data (definida por DAT1)
    # para que ela corresponda a um fim de período.
    # O resultado está DAT2.
    # LIBCLE corresponde ao título do período sob forma alfanumérica.
    Subprog BORNES(DAT1,DAT2,LIBCLE)
    Value    Date    DAT1
    Variable Date    DAT2
    Variable Char    LIBCLE
  • # Este sub-programa reenquadra uma data (definida por DA1) para que ela corresponda a um início de período.
    Subprog CADRE(DAT1)
    Variable Date DAT1
  • # Este sub-programa reenvia em DAT2 o fim do período definido por DAT1
    Subprog DATFIN(DAT1,DAT2)
    Value Date DAT1
    Variable Date DAT2
  • # Este sub-programa reenvia o número de períodos (assinado) separando as duas datas
    Subprog DECAL(DAT1,DAT2,DECAL)
    Value Date DAT1
    Value Date DAT2
    Variable Integer DECAL
  • # Este sub-programa acrescenta um número de períodos (assinado) a uma data de fim
    Subprog DECAL(DAT1,DAT2,DECAL)
    Value Date DAT1
    Value Date DAT2
    Variable Integer DECAL

Um exemplo de tais sub-programas pode ser consultado em face o tratamento SUBSTADEC, que define os sub-programas correspondentes à gestão por década.
  • Períodicidade ([field] PERTYP)

Periodicidade para a qual se deseja efetuar a estatística. Um menu local permite definir diferentes periodicidades (diária, semanal, por quinzena, mensal...)

Os registos de acumulado criados pela agregação estatística são datadas do primeiro dia do período.

Gestão

As datas de início e de fim de validade definidas sobre certas fichas permitem de limitar a validade de uma ficha entre duas datas.

  • Arranque ([field] RLG)

O código desencadeador da estatística permite identificar o elemento que desencadeie a estatística.

Deve estar definido na tabela dos eventos desencadeadores.

  • Validade inicio ([field] VLYSTR)
  • Validade fim ([field] VLYEND)
  • Cód. actualização ([field] UPDCOD)

Este código está por defeito o código da estatística/ Mas pode ser diferente, nomeadamente se deseja constituir um único agregado a partir de vários eventos desencadeadores diferentes (o que impõe vários códigos estatísticos). Neste caso, o código de atualização será comum a todos os códigos estatísticos.
  • Tempo conservação ([field] TIM)

Tempo de perenidade da estatística exprimida em número de meses. Este tempo está utilixado para a gestão dos expurgos, desde logo que se decide de expurgar as estatísticas.

Os dados estatísticos sendo em geral muito fortemente agregados, elas permitem um local limitado, é frequente interessante de guardar estas estatísticas durante uma longa duração.
  • Data actualização ([field] LSTDAT)

Esta data afixada permite conhecer a ultima data na qual um acumulado foi realizado por recálculo da estatística.

Consulta

  • Ecrã por defeito ([field] SCRDEF)

Permite definir o código ecrã utilizado por defeito para a consulta de estatística correspondente.

Este código ecrã, de tipo STA, está definido na função de parametrização dos ecrãs de consulta.

Na ausência de valor, o ecrã STD está utilizado.
  • Objecto para zoom ([field] OBJ)

Este campo permite dar acesso a uma ficha depois o nível o mais baixo da consulta estatística, em definindo o código do objeto correspondente. A chave desta ficha deve estar em linha a este nível. Habitualmente, trata-se do código objeto anexado à tabela cujo acumulado dos valores na estatística.
  • Lista ([field] RPT)

Este campo permite registar a lista Crystal Reports associado à consulta deste código de estatística (lançado via Ficheiro / Lista).

Uma lista genérica nomeada STAT existe, mas esta lista está presente todas as colunas de maneira unifirme, então que é possível, em conhecendo o contexto, de criar uma lista personalizada por cópia e modificação desta lista.
  • Função ([field] FONCTION)

Características

  • Nível definição ([field] UPDLEV)

Permite definir o nível organizacional do acumulado por períood (um acumulado global, por sociedade, ou por estabeleicmento).

O nível de detalhe estabelecimento não pode ser obtido que se o evento desencadeador define um código estabelecimento; o nível de detalhe sociedade ou uma sociedade são definidos no evento desencadeador.

Na restituição, poder-se-á pedir seja o detalhe o mais fino parametrizado aqui, seja um nível agregado (por sociedade ou global se acumulado foi gerido por estabelecimento).
  • Tipo ([field] TYP)

Permite definir se o acumulado nos registos estatísticos está colocado em tempo real ou em batch.

É importante notar que os únicos eventos desencadeadores standards podem dar lugar a agregações em tempo real (em cada vez que uma atualização está feita, o acumulado é realizado). Exceto se um específico está realizado, estatísticas baseadas sobre eventos desencadeadores não standards poderão unicamente ser atualizado em batch.

Para razões de performance, a colocação em obra de númerosos acumulados standards baseados sobre um evento desencadeador dá, pode atrasar a transação correspondente. É então prudente de privilijiar as estatísticas em modo batch e de prever uma tarefa batch regular carregada de as atualizar.

[Block number] 6

[Tab] Campos

Este separador permite definir, nos dois quadros sucessíveis.

  • os critérios de regrupamemto sobre os quais os acumulados estatísticos são geridos. A restituição, será possível ver os acumulados de estatísticas aos diferentes níveis de agregação sobre estes valores de critérios. Estes critérios de reagrupamento são campos (ou das sub-cadeias emitidas dos campos) extraidos nas tabelas em linha.

    Se os campos são tipificados (ligados a uma outra tabela, por exemplo), um título associado será igualmente guardado no registo de agregação. Por exemplo, se utiliza o código utilizador (de tipo AUS), o nome de utilizador será guardado e poderá ser restituido.
  • As expressões a acumular, que poderão ser restituidas seja diretamente, seja pelo meio de fórmulas (rácios, comparativos, percentagens) definidos no separador Avançado. Estas expressões são definidas num quadro de 10 linhas máximo.

[Grid] Campos

  • Tabela ([field] TBL)

Define a tabela na qual se encontra o critério de reagrupamento da linha de estatística.
  • Abreviatura ([field] TBLABR)

Quando a tabela precedente está várias vezes em linha no evento desencadeador, indicar-se-á aqui a abreviação utilizada neste caso particular.
  • Campo ([field] FLDNAM)

Define o código do campo tal que ele é conhecido na tabela.
  • Índice ([field] FLDIND)

Quando o critério escolhido está em multi-ocorrências, o valor deste campo permite de dar um índice (indo de 0 a N-1, se N é o número de ocorrências do campo).
  • Descr. ([field] FLDINTIT)

Define o título do critério tal que ele deverá aparecer no momento de interrogação da estatística. Por defeito, o título do campo está retomado no dicionário, o que permite afixar o intitulado na língua de conexão do utilizador.

Mas é igualmente possível de a modificar para lhe dar um título diferente, que falta poder definir nas diferentes línguas.

A definição de um intitulado nas outras línguas que a língua de conexão corrente é possível sobre os campos traduzíveis tais que este graças à função Traduções em linha, acessível por clique direito sobre o campo.

Uma janela permitindo este registo abre-se então.

  • Grupo ([field] FLDGRP)
  • Posição ([field] FLDPOS)

Quando o campo critério é uma cadeia de caracteres, é possível de definir a agregação sobre uma sub-cadeia de caracteres em definindo a posição do primeiro caracteres retido (1=início de cadeia) e o comprimento.

Por exemplo :

  • se deseja estruturar o acumulado sobre as duas primeiras cifras de um código postal de 5 caracteres, dar-se-á a posição 1 e o comprimento 2;
  • se deseja estruturar o acumulado sobre as três ultimas cifras de um código postal de 8 caracteres, dar-se-á a posição 6 e o comprimento 3;
  • Comprim. ([field] FLDLON)
  • Tipo ([field] FLDTYP)

Campo unicamente afixado que indica o tipo de dados (caracter, numérico,menu local, data...) correspondente ao critério registado sobre a linha.

[Grid] Valores

  • Variável ([field] VARNAM)

Define o intitulado proposto por defeito em face ao acumulado estatístico.
  • Descr. ([field] AMTDES)

Define o intitulado do acumulado guardado no registo estatístico.
  • Formulas ([field] AMTFOR)

Esta fórmula calculada, que pode fazer chamada a todas as variáveis e tabelas no momento de agregação, está avaliada, e o seu resultado está acumulado num agregado estatistico correspondente ao período e ao nível organizacional de agregação definida.

  • Previsão ([field] FRTFLG)

Se este indicador vale Sim, é possível de definir as previsões para o montante acumulado correspondente.

Estas previsões poderão ser registados, calculados a partir do real ou de previsões de um outro período, e que a uma malha de agregação livremente definido a partir dos critérios definidos para a estatística.

A restituição da estatística, poder-se-á apresentar à vez os acumulados calculados, as previsões correspondentes, e toda a expressão fazendo intervir os dois (isto se parametrizando o separador Avançado da estatística).

  • Conversão ([field] CNV)

Este indicador permite indicar se a expressão precedente está exprimida numa divisa, cujo se poderá em seguida dar a divisa por uma expressão calculada, assim que a divisa comum na qual ela deve ser convertida.

A conversão se faz à taxa histórica definida na data de referência do evento desencadeador, o tipo de câmbio utilizado para a conversão sendo definida pelo parãmetro CHGTYP (o primeiro tipo se nada está definido).

Assim, pode-se obter os montantes acumulados exprimidos de maneira homogénea.
  • Divisa origem ([field] DEVORG)

Quando o montante estatístico está exprimido em divisa, este campo permite conhecer o código da divisa de destino, afim de poder converter os montantes numa divisa comum antes de os acumular.

Trata-se de uma expressão calculada seja de tipo constante, seja fazendo intervir uma variável comum como por exemplo GSYSCUR.

  • Divisa destino ([field] DEVDES)

Quando o montante estatístico está exprimido em divisa, este campo permite conhecer o código da divisa de origem, afim de poder converter os montantes numa divisa comum antes de os acumular. Trata-se de uma expressão calculada seja de tipo constante, seja fazendo intervir os campos em linha no momento da agregação.

[Tab] Avançado

Quando a restituição estatística não se resume simplesmente a uma apresentação dos acumulados realizados sobre o separador precedente, vem-se aqui preencher este separador com as informações suplementares seguintes :

  • um quadro das colunas a afixar Por defeito, na criação de uma estatística, os intitulados dos acumulados são retomados.
  • eventuais critérios se deseja filtrar os dados que se acumula por expressões lógicas que deverão ser verificadas para que o acumulado se faça.

[Grid] Colunas

  • Descr. ([field] INTITFRM)

Este campo define o título da coluna tal que será apresentada no momento da consulta. Se deseja que a consulta possa ser feita nas outras línguas que a lingua de conexão original, falta traduzir o título.

A definição de um intitulado nas outras línguas que a língua de conexão corrente é possível sobre os campos traduzíveis tais que este graças à função Traduções em linha, acessível por clique direito sobre o campo.

Uma janela permitindo este registo abre-se então.

  • Formulas ([field] AMTFRM)

Registar-se-à aqui a fórmula que permite calcular o montante da coluna. Esta fórmula pode fazer intervir constantes, funções, e as variáveis seguintes (que podem ser selecionados via clique direito) :

  • C1, C2... Cn representam os acumulados definidos no quadro dos valores da estatística, exprimidas sobre o período corrente.
  • CC1, CC2, ... CCn representam os acumulados sobre o período precedente.
  • P1, P2... Pn representam as previsões sobre o período corrente (se elas existem).
  • PP1, PP2... PPn representam as previsões sobre o período precedente (se elas existem).

No momento da consulta, registar-se-á o período de consulta da estatística e igualmente o período precedente se pelo meos uma das fórmulas o faz aparecer dos dados relativos ao período precedente.

Fórmulas complexas, podendo fazer intervir os operadores, funções, e de variáveis, podem ser utilizados : Toda a expressão aritmética aqui está admitida, e pode ser parametrizada com ajuda de assistente de fórmulas, por exemplo.

  • C1/CC1 que permite calcular o rácio entre uma coluna a uma outra data e seu valor no período precedente.
  • C1/(CC1+(CC1=0)) que é idêntico à precedente, mas se CC1 é nulo, dividir-se-á por 1 em vez de 0...
  • max(P1,PP1)/1000 que exprime o máximo entre a previsão do período e o período precedente dividido por 1000
  • Tipo coluna ([field] CUMFRM)

Este tipo de coluna permite definir os acumulados sobre os montantes afixados. As condições possíveis são as seguintes :

  • Normal (afiza-se o montante calculado)
  • Acumulado progressivo (a linha N é igual à linha N-1 mais o resultado do cálculo sobre a linha).
  • % do total (afixa-se o rácio em percentagem entre o montante da linha e o montante acumulado de todas as linhas do quadro).
  • % acumulado (afixa-se o rácio em percentagem entre o montante acumulado das N primeiras linhas e o montante de todas as linhas do quadro).

Atenção, se o quadro não apresenta pela totalidade das linhas (é o caso se o botão Seguinte não está assinalado, sobre o primeiro separador), não se trata de uma percentagem por relação ao total geral, mas unicamente por relação às linhas presentes no quadro : Isto se produz se o número de linhas parametrizadas para o ecrã de consulta é inferior ao número de critérios do nível consultado).

Por exemplo, se temos um quadro de consulta limitado a 10 linhas, 20 critérios CR01 a CR20, para os quais o montante acumulado está alternativamente igual a 50 e 100, e 4 colunas parametrizadas C1 (Normal, Acumulado progressivo, % do total, % acumulado), vamos ver aparecer sobre o primeiro separador os montantes seguintes :

  • Formato ([field] FMTFRM)

Permite definir, sob a forma decimal, o número de números antes e após a vírgula utilizadas. Se este campo não está entregue, utiliza-se o valor do parâmetro FMTSTA.

[Grid] Condições

  • Critérios ([field] CRI)

Define as condições lóigicas levando sobre constantes, das funções, e dos campos em linha no momento do prrocesso de acumulação. Se as condições não são realizadas, o acumulado não está feito (se existem várias linhas, os critérios são ligados por um elógico).

Botões específicos

Validação

Este botão permite gerar os tratamentos automáticos ligados ao código estatístico Esta validação é normalmente encadeada na modificação ou na criação de um código. Ela não é então necessária que se os tratamentos foram suprimidos, ou se a parametrização foi criada por cópia a partir de um outro dossier (neste caso, a validação não está feita automaticamente).

  • A partir do dossier ([field] DOSORG)

  • Todos dossiers ([field] TOUDOS)
