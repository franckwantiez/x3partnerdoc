Componentes ecrã

Esta função permite de gerar dos URL e das páginas html que poderão em seguida utilizados nos ecrãs contêm um bloco browser, afim de afixar :

  • seja uma página web, em indicando um código de tipo URL
  • seja uma página HTML descrita neste dicionário, em indicando um código de tipo Html

Esta função permite igualmente de estocar os tratamentos XSL Associados aos blocos de tipo quadro, um XSL permite :

  • de os apresentar de maneira gráfica (com gráficos não previstos em flash)
  • de gerar as interacções (do tipo deslizante - desposar um outro) permitindo de actualizar a dia o quadro de maneira gráfica.

Esta maneira de fazer oferta uma abertura muito importante e uma vasta de possibilidades de parametrização dos softwares desenvolvidas a partir da ligação tecnológica X3.

Cada gráfica XSL está associado a um quadro, cuja classe [M] que será afixado ou não seguinte a parametrização do ecrã.

Os gráficos plannings e nomenclaturas são directamente modificáveis graças ao traatmento de interacção indicada deste dicionário.

Em standard, nós temos previsto de entregar XSL que são :

  • APLANM : O planning mensal representa uma ou várias actividades sobre um calendário mensal em montante de 1 a 4 meses à vez.
  • APLANY : O planning anual representa uma ou várias actividades sobre um calendário de 12 meses deslizantes consecutivas.
  • ANOMH : A nomenclatura horizontal representa uma nomenclatura ou um organigrama.
  • ANOMV : A nomenclatura vertical representa uma arborescência de tipo explorador de ficheiros de winwows.
  • ARADAR : O radar permite de afixar um conjunto de valores numéricos.

Um anexo detalhado estes cinco XSL entradas em standard.

Ecrã de registo

O cabeçalho de identificar de maneira única de cada elemento desta tabela. Ela permite igualmente de registar todas as informações necessárias sobre os componentes de tipo URL.

  • Cód. ([field] CODURL)

Código identificando a ficha corrente.

  • Tipo de lig. ([field] TYPURL)

Campo permite de determinar as três interfaces gráficas :

  • URL : permite de afixar num ecrã, por um bloco browser, uma página web, em indicando um URL.
  • XSL : permite de afixar num ecrã, por bloco quadro, um gráfico, uma nomneclatura, ou um radar, estes elementos sendo baseados sobre um quadro.
  • Html : permite de afixar num ecrã, por um bloco browser, uma página web descrita neste dicionário.
  • Descr. ([field] ZINTIT)

Titulo do componente. É nomeadamente afixado ao ecrã, quando este último propõe uma escolha de URL, ou páginas HTML.

  • Descr. avaliada ([field] INTEVAL)

Permite de ter um título para "URL", página html ou elemento XSL, a variável em função de um contexto. Se ele está preenchido, o intitulado está afectado no lugar do intitulado, no momento de afixação do ecrã no qual está colocada este componente.
  • Tipo de XSL ([field] TYPXSL)

Permite de afinar um tipo de relação XSL.

Em standard, três tipos de tratamento XSL :

  • Planning
  • Nomenclatura
  • Radar

O tipo "Diversos" está reservado aos programadores específicos.
  • Cód. parâmetro ([field] PARXSL)
  • Módulo ([field] MODULE)

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.
  • Cód. activ. ([field] CODACT)

Um código actividade permite :

  • de tornar opcional um elemento do dicionário se o valor associado ao código actividade é nulo.
  • de assinar os elementos específicos desde logo são marcados por um código começando por X,Y ou Z.
  • de dimensionar um número de linhas máximas quando o código de actividade maerca os elementos de um quadro.

Assim, se o código actividade está não activo, o elemento marcado não será utilizável, e o código associado (se existe um) não será gerado nem activável.
  • Endereço ([field] URL)

Endereço URL Ele pode ser avaliado. Exemplos:

  • http://www.sage.com/
  • ="http://www.sage.com"
  • =ADR1   ADR1 sendo uma variável a declarar e a alimentar no tratamento ligado à função na qual este será utilizado.
  • Localização ([field] PLEURL)
  • Nivel de segurança ([field] SECLEV)

  • [field] XSLCLOB

Permite escrever o código XSL, ou o código de uma página HTML.

[Tab] Parametrização

Este separador permite de registar parametrização utilizada pelo tratamento

[Grid] Menus locais

  • Cód. ([field] UCOD)

A associação de um menu local a um parâmetro permite de fornecer ao componente XSL um conjunto de literais, na língua do utilizador, que poderá utilizar no seu funcionamento. Estes menus são referênciados à geração da janela, numa estrutura que está explorada pelo componente Web. O código associado aos menus não é modificável, mas é possível de alterar os números de menus locais para fazer aparecer outros literais. É possível de definir assim até 5 menus locais.

Exemplo: Para o XSL standard de planeamento mensal, o tratamento utiliza os três menus locais seguintes :

  • 9001 : mês do ano
  • 9833 : dias da semana
  • 9834 : abreviação dos dias da semana

Cada menu local está transmitida ao tratamento XSL para um códigoque é fixo.

Exemplo: Para o XSL standard de planeamento mensal, os códigos são : MONTH, DAY, ABRDAY.
  • Menu local ([field] UMEN)
  • Descr. ([field] ULIB)

Interacções

  • Script ([field] UPRG)

As interações permitem ao utilizador de intervir sobre o gráfico. Segundo os XSL, eles podem tomar diferentes formas - deslisar-depositar, suprimir, deslocar etc. O tratamento XSL pode se fazer sucessivamente apelo a um ou três sub-programas de supervisor escrito em linguagem Sage.

Exemplo: Para o XSL standard de planeamento mensal, o tratamento utiliza os sub-programas MAJPLAN de AMAJPLAN. Permite afetar uma actividade a uma selecção do planeamento. Uma jornada pode ser cortada em quatro seleções.
  • Sub-progr. ([field] USPRG)
  • Descr. ([field] UINTIT)

[Grid] Parâmetros

  • Cód. ([field] UPARCOD)

Os parâmetros correspondentes tem valores que caracterizam quaisquer opções permitindo de reutilizar o XSL em certos contextos diferentes. Em cada parâmetro, associa-se uma designação e um valor por defeito. Este valor por defeito será proposto e poderá ser modificado, nos ecrãs que utilizam o XSL.

Exemplo: Para o XSL standard de planeamento mensal, o tratamento utiliza os quatro parâmetros seguintes :

  • NBMONT : Número de mês afixado (1 a 12)
  • MENACT : Menu local tipo de atividade (9837 em standard. Possibilidade de 8 escolhas neste menu)
  • SDATA : Atualização gráfica ()
  • PINTER : Programa de interacção (AMAJPLAN em standard)
  • Descr. ([field] UPARLIB)
  • Valor por defeito ([field] UPARDEF)
