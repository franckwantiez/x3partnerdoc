Recepções administrativas

Uma recepção corresponde ao tratamento das mercadorias de uma entidade homogénea de entrega (classicamente um aprovisionamento). Ela pode corresponder em todo ou parte de uma entrada esperada relativa a um ou vários depositantes.

Uma recepção reagrupa igualmemnte um certo número de características:

  • Um transportador/modo de transporte que pode ser diferente daquele esperado.
  • Uma referência transporte (quer dizer um identificador comum entre o estabelecimento e o fornecedor)
  • Uma carga de recepção
  • Um peso bruto

A recepção administrativa se inscreve na Cadeis das entradas.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Geral
  • [Tab] Lista EE

Este ecrã permite de gerar as recepções administrativas.

Para criar uma recepção, é necessário de afectar a recepção as entradas esperadas recepcionadas.

Para tal, 3 possibilidades:

  • Ir na lista de esquerda, na gaveta "EE a recepcionar" e seleccionar na ajuda o rato sobre os registos propostos. Estes são agrupados por data e hora previsionais de recepção, e por transportador.
  • Registar directamente nas zonas Depositante e Número, se separador 2 os nº EE a recepcionar
  • Utilizar as opções de Selecções standard para seleccionar um código memo.

Segundo o parâmetro do modo de recepção de cada EE (entrada esperada), uma lista de controlo pode ser editada automaticamente. Esta lista é utilizadas em tanto que suporte para verificar fisicamente os artigos recepcionados e os retranscrir em Warehousing.

Funções / Movimentos consumíveis

Permite de declarar 1 ou n movimentos de entrada em stock de cosumíveis ao título da recepção administrativa.

Este registo pode-se fazer em todo o momento, qualquer que seja a lista da RA (recepção administrativa).

Para realizar um movimento de entrada de consumíveis, falta registar diversas informações:

    • o código consumível: define na ficha artigo dos consumíveis
    • o código movimento: deve pertencer à família dos tratamentos "CONSOMMABLE ' e ser de tipo "ENTREE"
    • A quantidade: quantidade do movimento de entrada

Como o movimento de entrada do consumido está declarado a partir de uma entidade conhecida, as informações, transportador, modo de transporte, fornecedor e descrição fornecedor, são, se for caso disso, preenchidos com aqueles da entidade.

Para um movimento de entrada, as informações cliente e intitulado cliente não estão acessíveis assim que o tipo de entidade e o número desta última.

Segundo a parametrização, o registo de um código operador está pedido para validar o movimento de entrada. A validação, o stock está incrementado.

Após validação, o movimento não está mais modificável. Por consequente, não é actualizado no caso onde as actualizações são realizadas sobre a entidade na qual este movimento está ligado.

É possível de gerar 3 listas:

    • No Ficheiro / Imprimir / Ficha:
        • Stock dos consumíveis triados por consumível ou por quantidade.
        • Impressão ecrã para registo seleccionado
    • No Ficheiro / Imprimir / Lista:
        • Movimentos dos consumíveis triados por consumíveis ou por data.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

 Supressão impossível

A lista da entrada esperada não permite a modificação da recepção.

As mensagens de erro ligadas à declaração de movimento de consumíveis são:

Erro zona "consumível". Zona obrigatória.

O código do consumível deve obrigatóriamente ser registado.

Erro zona "Código movimento". Zona obrigatória.

O código movimento deve obrigatóriamente ser registado.

Erro zona "Código movimento". Código movimento de tipo não consumível.

O código movimento preenchido deve pertencer à família dos tratamentos 'CONSOMMABLE'.

Erro zona "Código movimento". Código movimento de tipo saída.

O código movimento deve obrigatóriamente ser de tipo 'ENTREE'.

Erro zona "Quantidade". Zona obrigatória

A quantidade do movimento de entrada deve ser obrigatoriamente ser preenchido.

Erro da aplicação. O depositante deve ser registado para esta origem.

Quando um movimento de entrada está ligado a uma entrada esperada ou uma entrada directa, o depositante deve obrigatóriamente ser preenchido.

Erro zona "Nº origem mvt". Referência origem não encontrada.

O número da entidade Warehousing deve obrigatoriamente ser registada e existente para o estabelecimento.

Erro zona "consumível". Consumível bloqueado em entrada.

O consumível deve obrigatoriamente ser autorizada em entrada para realizar um movimento de entrada.
