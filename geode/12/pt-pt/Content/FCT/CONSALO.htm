Conexões

Desde logo que o código atividade AUDIT está ativada, os softwares Sage X3 são suscetíveis de traçar a integralidade das conexões e desconexões sobre o dossier corrente. Estas informações, guardadas na tabela ALOGIN, são consultáveis pelo meio desta função.

[Refer to documentation [Implementation]]

Gestão do ecrã

  • Cabeçalho
  • [Tab] Auditoria

Cabeçalho

Na entrada na função, define-se os critérios de consulta (que podem levar sobre a data de conexão e a identidade, tanto X3 que o sistema de utilizadores).

Pode-se igualmente se limitar a traçar as conexões infrutuosas.

[Block number] 1

  • Período ([field] STRDAT)

Se esta data está preenchida, apenas os dados cuja data de última modificação ou da data de criação é inferior ou igual são visualizadas na consulta.
  • [field] ENDDAT

Se esta data está preenchida, apenas os dados cuja data de última modificação ou da data de criação é inferior ou igual são visualizadas na consulta.
  • Utilizador ([field] USER)

Se este campo foi preenchido, apenas as conexões feitas sobre o código utilizador sage correspondente (tal que é definido na tabela AUTILIS) aparecem.

  • Login adonix ([field] LOGIN)

Se este campo está preenchido, apenas as conexões feitas sobre a conta utilizador correspondente (ao nível do sistema) aparecerão.

[Block number] 2

  • Conexões infrutíferas ([field] ERR)

Se esta check box está assinalada, vê-se aparecer na consulta que as tentativas infructuosas de conexão (a razão do erro sendo detetado pelo rasto por outro lado).
  • Todos os motivos ([field] TOUS)

Se esta check box está assinalada, as conexões infrutiferas aparecerão na consulta quaisquer que sejam os motivos.

  • Motivo ([field] STAT)

Permite filtrar a afixação das conexões infructuosas correspondentes a um único motivo (se a check box Todos os motivos não foi modificada).

[Tab] Auditoria

O quadro que se afixa uma vez os critérios escolhidos contêm o conjunto das linhas de conexões escolhidas, ordenadas na ordem cronológica crescente. De referir que esta ordem é a ordem das conexões. Com efeito, no momento da conexão, uma linha está criada na tabela ALOGIN, um número cronológico lhe estando então atribuida. Na desconexão, esta linha está atualizada.

[Grid]

  • Crono ([field] SEQ)

Número sequêncial atribuido pelo sistema no momento de registo na tabela dos logs. Utiliza-se isto para fazer a sequência associada à tabela ALOGIN (pelo atraso da função sage uniquid).
  • Login adonix ([field] ADOLOG)

Este código corresponde ao código utilizador registado na caixa de conexão (identificado como código login na ficha utilizador). O valor registado está afixado tal que ele foi registado na caixa de conexão (ele pode estar em maiúsculas ou em minúsculas, mesmo se por outro lado o código login da ficha está em maiúsculas).

Deve-se notar que, no caso de uma conexão Web via ligação directa, que restabelece o utilizador sobre uma caixa de conexão, o código que aparece aqui é ASK_ (o código utilizador final sendo identificado na coluna Utilizador).
  • Utilizador ([field] ADOUSR)

Código utilizador (chave da ficha utilizador) identificando o utilizador conectado no momento da sessão. Este código corresponde à variável GUSER.
  • Login sistema ([field] SYSUSR)

Corresponde ao código utilizador permitindo a conexão ao sistema.
  • Cliente ([field] ADRCLI)

Identifica o endereço rede do posto cliente, ou do servidor Web de conexão intermediária tendo aberto a sessão.
  • Tipo de conexão ([field] ADOTYP)

Identifica o tipo de conexão sob o qual a sessão foi aberta (primária, secundária, batch...). O menu local 924 identifica todos os tipos de conexão possíveis.
  • Data conexão ([field] DATCNX)

Corresponde à data sistema e à hora sistema do servidor de tratamento no momento da conexão de utilizador.

  • Hora ([field] HOUCNX)
  • Data desconexão ([field] DATDCX)

Corresponde à data sistema e à hora sistema do servidor de tratamento no momento da desconexão de utilizador. Tanto que este campo está vazio, o utilizador é suposto de estar conectado.

O único caso onde o campo não foi preenchido então que o utilizador não foi mais conectado e o caso de uma interrupção brutal do processo (paragem do sistema ou da base de dados, interrupção da tarefa por um administrador via a vigilância das tarefas, por exemplo).
  • Hora ([field] HOUDCX)
  • Id adonix ([field] ADOID)

Este valor numerico identifica de maneira única uma sessão adonix sobre um servidor de aplicação dada. Ele serve então para o identificador da conexão. Sobre a conexão corrente, o seu valor foi obtido pela formula adxuid(1).
  • Id BDD ([field] BDDID)

Este código identifica a sessão aberta na base de dados. Segundo as bases de dados, é obtido de maneira diferente :

  • é o valor de ambiente SESSIONID sob oracle, obtido por um pedido de tipo :

select userenv('SESSIONID') From dual

  • é a variável @@spid sob Sql server.

Identifica de maneira única uma conexão a uma base de dados, e é nomeadamente utilizada para fazer a relação entre uma atualização e o código utilizador nos "gatilhos" criados via separador de auditoria em o dicionário das tabelas.
  • Mot. prazo ([field] STA)

Define o código motivo quando a conexão não terminou (utilizador incorrecto, excesso de sessões abertas, por exemplo). Os valores emitidos do menu local 948 podem ser afixados.
  • Mensagem ([field] MSG)

Corresponde à mensagem de erro reenviado ao utilizador quando a conexão foi recusada.
