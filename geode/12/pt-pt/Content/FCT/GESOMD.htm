Modos de saída

Esta tabela permite a definição das regras aplicáveis aos artigos no momento da fase de pesquisa de acondicionamentos em saída.

Gestão do ecrã

Para criar um modo de saída, atribui-se um código que o identifica de maneira única sobre 3 caracteres e um intitulado. Estas duas informações se encontram em cabeçalho de gestão.

[Tab] Ecrã de registo

As informações registáveis neste ecrã respeitam-.

  • A gestão dos números de série em entrada / saída.
  • A gestão dos pickings
  • A gestão dos pickings pivots.
  • A gestão dos pickings multiplos.
  • A prioridade dos critérios de saída.
  • A gestão dos pickings multiplos.
  • A gestão das saídas reservadas autorizadas.
  • A gestão das saídas quebradas autorizadas.

  • Modo de saída ([field] OUTMOD)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.

Picking

O sistema considera que as antecipações se fazem em prioridade nos acondicionamentos de tipo« picking «

O sistema desencadeia uma oredem de aprovisionamento se a quantidade a antecipar é superior ao stock disponível de acondicionamento picking.

Uma ordem de aprovisionamento se desencadeia, em fim de ciclo, se o stock disponível do picking é inferior ao limite de aprovisionamento de’ac ondicionamento e se o’acondicionamento foi feito’objecto de’um aprovisionamento de necessidade ao curso do ciclo.

  • Stock picking ([field] PICSTKMGT)
  • Esc. autom. picking ([field] AUTPIC)

O sistema escolhido o ou os pickings os mais apropriados para honorar

o pedido.

[Grid] Prioridade saída

  • Critérios ([field] RSVOUTPIO)

Quando o sistema deve operar um movimento de saída (ou de aprovisionamento) a partir da reserva, ele carrega o conjunto dos objectos de stock do código artigo considerado em memória, segundo um critério de ordenação definido pela combinação dos 4 elementos seguintes:

Data FIFO. Saída do objecto tendo a mais antiga data FIFO.

% ocupação. Saída de acondicionamento cuja taxa de ocupação é a mais fraca.

Quantidade UA. Saída de objecto cuja quantidade UA é a mais fraca (a mais quebrada).

Armazém Saída do armazém o mais prioritário.

Tranche FIFO.Saída de objecto tendo a data mais antiga da tranche FIFO. A tranche FIFO é calculada a partir da data FIFO do mais antigo objecto ou a partir da data FIFO do acondicionamento no caso de uma estrutura de armazenagem "Fila". Neste valor está acrescentado a tolerância FIFO definida na ficha artigo principal do artigo afectado.

Número de lote. Saída de objecto tendo o mais pequeno nº de lote.

Picking pivot

  • Picking pivot ([field] DYNPIC)

Se a check box está assinalada, o acondicionamento é de tipo "picking pivot", quer dizer que não é afectado a um conjunto depositante / artigo ; neste caso, a afectação ao lugar do lançamento em vaga. A utilização de picking reservado ao artigo é prioritário à utilização dos picking pivots qualquewr que seja a parametrização do modo de saída do artigo.

  • Reab. mono OE ([field] DYNSNGDLO)

Quando esta check box está assinalada, As Ordens de Aprovisionamento geradas pelo aprovisionamento de um acondicionamento picking não tratarão a necessidade que de única OE ; senão podem tratar a necessidade de OE do mesmo ciclo.

  • Reab. mono ciclo ([field] DYNSNGWAV)

Quando a check box está assinalada, um acondicionamento picking pivot não foi aprovisionado que para um ciclo; senão, o mesmo picking pode ser aprovisionado ao título de vários ciclos.

[Grid] Priorid. picking múltiplo

  • Critérios ([field] PICOUTPIO)

Quando o sistema deve operar um movimento de saída (ou de aprovisionamento) a partir de picking múltiplo, ele carrega dos objectos de stock do código artigo considerado em memória, segundo um critério definido pela combinação dos 4 elementos seguintes:

Objecto: A gestão dos critérios de saída para o acondicionamento picking afectados aos artigos gerados em picking múltiplo. Os critérios descritos a seguir são a classificar por ordem de prioridade.

Prioridade picking. Os acondicionamentos com a prioridade picking o mais forte, que corresponde à cifra amais baixa afectada a um acondicionamento, são priviligiados.

% ocupação. Os acondicionamentos os menos ocupados são priviligiados.

Quantidade UA. O acondicionamento com a quantidade UA disponível o maior é priviligiado.

Data FIFO. Saída de objecto tendo a mais antiga data FIFO. 

Tranche FIFO. Saída de objecto tendo a data mais antiga da tranche FIFO.

Picking múltiplo

  • Picking múltiplo ([field] MLTPIC)

Se a check box está assinalada, o artigo gerado em multipicking. Será então possível de criar vários acondicionamentos picking com um mesmo conteúdo de referência para este artigo.

  • OE mono picking ([field] SNGPIC)

Se a check box está assinalada, as antecipações relativas a uma OE para uma referência artigo serão realizados num único acondicionamento picking.

  • Saída prioritária no reabast. ([field] OUTRPN)

Se esta check box está assinalada, o aprovisionamento se fará a partir de um acondicionamento picking para o conjunto das antecipações ligadas ao ciclo.

  • Reab. mono endereço ([field] SNGSLO)

Se esta check box está associada, o reaprovisionamento se fará a partir de um acondicionamento único para o conjunto das antecipações ligadas ao ciclo.

Saída reserva autoriz.

Esta percentagem não está a definir que no caso de utilização da’opção "pedido >= % capacidade picking" para o campo’ Saída reserva autorizada».

É ’ a percentagem que está utilizada no cálculo do limite para alem do qaul a pesquisa em reserva foi desencadeada

  • Saída reserva autoriz. ([field] RSVOUTAUZ)

Indica ao sistema se a saída da reserva está autorizado, e se sim em que casos pode ser :

Não. Todas as saídas são feitas a partir do Picking.

Procura >=objecto. As antecipações se efectuam em reserva se a quantidade pedida é igual ou excede a quantidade standard do conteúdo por defeito do artigo. Após antecipações sucessivas de objectos em reserva, a quantidade residual (i.e. inferior a um objecto completo) será antecipado em Picking.

Procura >= % capacité picking. Permite antecipar num acondicionamento picking segundo a quantidade pedida na OE.

Sempre. Quelquer que seja a quantidade a registar, as antecipações se efectuam na zona reserva (sem Picking).
  • Reserva interdita ([field] RSVNEVER)
  • % capac. picking ([field] PICCAPPCT)
  • [Mouvement en contenant homogène] ([field] HMGMVT)

Saída incomp. autoriz.

  • [field] IPAOUTAUZ

Indica se a saída de palete quebrada está autorizada ou não.

Não.  Os objectos quebrados servirão unicamente aos movimentos de aprovisionamento e poderão fazer objecto de transferências com fins de compactagem.

Pedido <quantidade standard.  Os objectos quebrados poderão sair ao título de uma OE, se e somente se, o pedido ou o saldo a servir está estrictamente inferior à quantidade standard do conteúdo palete do artigo.

O sistema ordenará em prioridade os objectos completos. A saída de objecto quabrado da reserva não poderá se fazer que se nenhuma palete standard não está disponível, e que o saldo a servir é inferior ao standard de paletização do artigo.

Sempre. Os objectos de stock são tratados na ordem preferêncial definida no Mode de Saída (ver após o parâmetro intitulado "Ordem de prioridade dos critérios"). Sem prioridade para os objectos completos.

Uma escolha dos dois primeiros valores deste parâmetro (valor "Não" ou "Pedido < standard") pode provocar um curto circuito de ordem FIFO.
  • Criação incompl. autoriz. ([field] IPACREAUZ)

Indica se o sistema aceita de criar uma palete quebrada para responder a um pedido de saída. Este parâmetro não tem nenhuma influência sobre o algoritmo de aprovisionamento do picking. Este parâmetro está inibido pelo sistema quando as saídas reserva são sempre autorizados.
