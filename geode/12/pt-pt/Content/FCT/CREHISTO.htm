Criação dossier histórico

Os softwares em tecnologia SAGE X3 são susceptíveis de gerar, pelo meio de funções de expurgo / arquivo, a transferência versus os artigos ou o expurgo de movimentos que não evoluem mais, e que são considerados como suficientemente antigos para não ser consultsados frequentemente.

É importante notar que a função de arquivo permite a transferância versus uma base arquivo de dados considerados como saldados (i.é não susceptíveis de continuar a evoluir). Estes dados não podem ser consultados que separadamente (elas não podem estar vistas simultaneamente com os dados não arquivados : escolhe.se de ver seja os dados do arquibo, seja os dados de exporação). Pelo contrário, se uma tabela arquivo faz referência a dados de base (não susceptíveis de serem arquivados por definição, como os clientes, as contas...) onde os dados para os quais nenhuma tabela arquivo foi definida, a consulta poderá se fazer por zoom sobre estes dados não arquivados.

O parametrização das regras de expurgo permite definir as durações de vida a partir das quais os movimentos podem ser expurgados ou arquivados, em seguindo as regras de coerência por grupo de tabelas definidas por outro lado.

O arquivo faz-se por transferência dos dados de uma tabela do dossier versus uma tabela de mesma estrutura (com dois campos perto, CREHISDAT e CREHISUSR que permitem de conhecer as datas e operadores tendo historizado o dado). Esta tabela está definida num dossier dedicado, que utiliza por outro lado o dossier corrente como dossier mãe, e cuja criação está feita para esta função.

O interesse desta maneira de fazer reside no facto que as funções de consulta standard sobre os dados do histórico funcionando então por simples conexão sobre este dossier, da mesma maneira que eles funcionam sobre o dossier em curso.

De referir um limite na gestão do dossier historizado : o portal personalizado não está acessível num dossier historizado.

Gestão do ecrã

[Tab] Ecrã de registo

Se o dossier historizado já existe, esta função não está utilizável, uma mensagem de erro assinalam o facto. Se não é o caso, um ecrã de registo dos parâmetros do dossier historizado se afixa, que se prenche.

Uma vez estes parâmetros registados, a confirmação desta escolha provoca a criação do dossier.

[Block number] 1

  • Dossier ([field] DOSSIER)

Define o nome do dossier a criar (por defeito, o nome do dossier corrente, pré-fixado por H, está proposto).
  • Tamanho de dados ([field] SIZDAT)

Estes valores permitem dimensionar os ficheiros dados e indice da base. O tamanho está exprimido em Mb.

Estes valores não têm interesse que para a criação de um dosssier, e eles serão colocados em geral registados em fim de definição do dossier, justamente antes a criação efetiva de um dossier. Com efeito, para dispor de um valor estimado correcto, importa de ter previamente dimensionado as tabelas da base para o atalho de valores de dimensionamento (separador Tabelas), e de ter definido mais precisamente a estrutura das tabelas da base para o atalho de códigos atividade (separadores Opções, Ecrãs, Específicos).
  • Tam. índice ([field] SIZIDX)
  • Grupo ficheiro ([field] GRPFIL)

Quando a base de dados é oracle, deve-se dar um identificador sobre 4 caracteres máximo, utilizando nomeadamente para nomear os datafile associados ao utilizador de base de dados. Quando a base é SQL server, precisa-se simplesmente se se pretende gerar os grupos de ficheiros (Sim / Não).
  • Coef. aumento das tabelas ([field] COEFF)

O coeficiente de aumento das tabelas permite definir o tamanho utilizado para dimensionar a tabela arquivo, por referência ao tamanho utilizado para dimensionar a tabela no dossier de exploração. Este coeficiente pode ser inferior ou superior a 1. Se por exemplo se imagina de ter um ano de dados de exploração em linha a um momento dado, e que se pretende guardar 5 anos de arquivos, o coeficiente deverá ser igual a 5.

[Block number] 2

  • Perfil menu consultvo ([field] HISMEN)

Define o perfil menu utilizado para o acesso ao dossier arquivo. Poder-se-á referenciar este perfil em todas as funções de tipo consulta, mas igualmente funções objecto que não poderão funcionar que em modo consultativo quando será conectado sobre o dossier arquivo.

observações

A criação de um dossier historizado cria o dossier e um mínimo de tabelas necessáris ao seu funcionamento. As tabelas arquivos não criados que no momento da operação de arquivo.

Uma vez criado, o nome do dossier histórico, o coeficiente de aumento, e o código do menu são guardados nos parâmetros do dossier de exploração. Estes parâmetros, definidos no grupo SUP, se chamam HISDOS, HISCOE, HISMEN, e não são modificáveis.
