Submissão de pedidos

Esta função permite pedir a execução de um pedido a uma data e uma hora dadas. Ela pode ser chamada via um botão directamente depois a gestão dos pedidos, ou para o menu

É de referir que a opção Todos os utilizadores definido aqui a seguir deve ser acordado com discernimento; mesmo o lançamento de uma tarefa sob o código de um outro utilizador supõe o registo da sua palavra chave, é mais seguro de impor, para a maioria dos utilizadores, o lançamento de tarefas sob a sua conta.

Gestão do ecrã

[Tab] Ecrã de registo

Este ecrã permite identificar :

  • o dossier e utilizador de lançamento do pedido (se o código utilizar está modificado, é necessário de registar a palavra de passe do utilizador registado).
  • o pedido ( tarefa ou grupo de tarefas a lançar), assim que a data e hora do lançamento pedido.

[Block number] 1

  • Dossier ([field] DOSSIER)

Define o código do dossier sobre o qual o pedido está lançado.

  • Código utilizador ([field] USER)

Define o código ( e eventualmente a password) do utilizador tendo lançado o pedido.

  • Palavra-chave ([field] PASSE)

[Block number] 2

  • Grupo ([field] GRP)

Define um grupo de tarefas a lançar.
  • [field] ZGRP

[Block number] 3

  • Cód. tarefa ([field] TACHE)

Define o código da tarefa que será lançada automaticamente pelo servidor batch.

  • [field] ZTACHE

[Block number] 4

  • Língua ([field] LAN)
  • Data ([field] DAT)

Define o dia e a hora de desencadeamento pedido para o pedido.

  • Hora ([field] HEURE)
  • Modelo ([field] JOBMOD)

Se este campo foi assinalado, um ficheiro contendo os parâmetros registados para o pedido será gerado no directório dos ficheiros modelos (parâmetros servidor batch) e o pedido não será submetido ao servidor. O nome deste ficheiro é composto do código da tarefa seguimento de .mod Este ficheiro serve de modelo para criar ficheiros .job que permitem de criar os pedidos bath.

Botões específicos

Validação

valida as escolhas feitas e pode levar a abertura de uma janela permitindo o registo dos parâmetros de lançamento da tarefa (o conteúdo desta janela depende bem entendido da tarefa).

Se a check box Modelo não foi assinalado, a validação dos parâmetros leva a então a afixação de uma mensagem de confirmação. Esta mensagem é : Pedido transmitido ao servidor. É seguido de um número entre parênteses que é o número do pedido, onde o número do primeiro pedido do grupo (é então em número de ordem que se encontra sobre os outros pedidos do grupo). O número assim dado identifica de maneira única a ou as tarefas, e permitira´de os reencontrar na gestão dos pedidos. Se a check box Modelo está assinalada, uma ou várias mensagens Criação do ficheiro XXXX se afixam para assinalar que os ficheiros modelos foram bem criados.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

Nível de acesso insuficiente"

A tarefa ou o grupo de tarefas a lançar é de um nível superior ao nível de tarefas batch que o utilizador tem o direito de lançar.

"Função não autorizada"

A função que o utilizador pretende lançar não está autorizada ao utilizador (seu perfil função não lhe permite de a lançar, o que seja em directo ou em batch).

Pedido inativo

A tarefa a lançar não está ativa : o pedido não pode então ser lançado

Grupo inativo

O grupo de tarefas a lançar não está ativo : o pedido não pode então ser lançado

Expressão de pase incorrecta

Registou-se o código de um outro utilizador, e a palavra chave assoicada não está corrigida corectamente.
