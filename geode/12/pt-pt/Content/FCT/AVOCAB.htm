Vocabulário

Os softwares Sage permitem de trabalhar em várias línguas (francês, inglês, espenhol, italiano, português, chinês... nomeadamente). O utilizador, no momento da conexão ao software, escolhe a língua na qual os ecrãs de diálogo devem estar afixados.

Esta funcionalidade está baeada sobre a prsença de duas tabelas na base, gurdando todos os elementos susceptíveis de serem traduzidos : a tabela ATEXTE, que contém os textos identificados por um número e a tabela APLSTD, que contém as mesagens organizadas em capítulos. Nestas duas tabelas, o código língua está presente, para permitir de guardar o texto num conjunto de línguas.

A tabela ATEXTE corresponde aos intitulados dos campos situados sobre os ecrãs. Os textos standards são guardados com um número inferior a 100.000. Os textos específicos são guardados para além. A chave de acesso desta tabela está composta de dois elementos : a língua e o número de mensagem

A tabela APLSTD contém as mensagens de erro ou de advertência enviadas pelo software, assim que os menus locais, os intitulados dos botões radio Cada capítulo, identificado por um número, reagrupa um conjunto de elementos identificados por um número no capítulo. A chave de acesso inclui o código língua, o capítulo, e o número.

Estes elementos são entregues de maneira standard a cada nova versão e podem ser atualizados a cada "realease" para todas as línguas standard (com excepção dos textos específicos, e de certos menus locais reputados parametrizáveis) A sua tomada em conta nos ecrãs se faz no momento da validação dos ecrãs e janelas do software (para razões de performance, os textos são então diretamente inseridos em função da língua nas descrições de interface ao formato XML).

Pode ser interessante de querer personalizar certos elementos do vocabulário. Por exemplo, pode-se querer substituir o termo genérico Representantes pelo termo Agentes comerciais, ou ainda o termo Gama pelo termo Receitas. Realizar isto em modificando diretamente o conteúdo das tabelas APLSTD e ATEXTE não têm sentido, poisque isso não pode ser perene. É por tal que foi criado uma tabela particular, gerada por esta função.

Esta tabela, nomeada AVOCAB na base de dados, guarda uma lista de excepções. Está assim possível dizer, para uma dada língua, um capítulo e/ou um número de texto dados, que um texto particular substitui o texto standard.

Regista-se uma lista de textos a personalizar para uma língua dada e para uma versão dada.

Uma versão sendo um vocabulário. Isto permite entregar vários vocabulários num dossier de referência e de determinar o qual deve efetivamente ser utilizado no dossier de produção

O registo da relação do vocabulário não altera vocabulário imediatamente. É o apoio sobre o botão Validação que permite tomar em conta o novo vocabulário.

Um ecrã se abre Deve-se escolher a versão :

Versão Corrente

Versão standard 

É a versão registada no ecrã de partida que está tomada em conta. Neste caso, as tabelas ATEXTE e APLSTD do dossier corrente são atualizados com os dados da tabela AVOCAB.

Os campos CUR e VER são iguais ao número de versão.

Neste caso, as tabelas ATEXTE e APLSTD do dossier corrente são atualizados com as tabelas do mesmo nome do dossier de referência.

O campo CUR está colocado a 0. O campo VER contém sempre a versão corrente.

Em seguida ou assinala ou não "Revalidação dos ecrãs".

A revalidação dos ecrãs permite regerar os ecrãs e janelas de aplicação. Unicamente os elementos que têm um vocabulário registado. De facto isso que dizer que os ficheiros XML representam o interface utilizador contém o novo vocabulário. Apoiamo-nos sobre as tabelas ATEXTE e APLSTD.

De referir que apenas as mensagens standards (ie. as mensagens de ATEXTE cujo número é inferior a 100.000) podem ser personalizados nos casos onde o dossier corrente não foi revalidado.

Se o dossier corrente foi revalidado, os textos de ATEXTE superiores a 100 000 são remontados na tabela ATEXTE do dossier de referência. E neste caso pode-se personalizar os textos específicos.

Em standard, a tabela AVOCAB está entregue vazio,ela não é nunca modificado por uma atualização.

Gestão do ecrã

[Tab] Ecrã de registo

Escolhe-se a língua e a versão As diferentes versões são vocabulários diferentes.

[Block number] 1

  • Língua ([field] LAN)

Este campo define a língua na qual se deseja modificar as mensagens standards.

  • Versão ([field] VER)
  • Versão corrente ([field] LIBCUR)
  • Língua principal ([field] LANMAIN)

[Grid]

  • Tipo de texto ([field] TYP)

Este tipo pode corresponder :

  • seja a tabela das mensagens (APLSTD) que armazena nomeadamente os menus locais. Nesta tabela, os textos são estruturados sob a forma de um número de capítulo e de um número de mensagem.
  • seja a tabela dos textos (ATEXTE) que armazena nomeadamente as mensagens emitidas dos ecrãs. A chave de acesso é um número de mensagem inferior a 100.000 para os textos standards, que são os únicos afetados pela função de personalização do vocabulário.
  • Capítulo ([field] CHP)

Quando o texto a traduzir provém da tabela das mensagens APLSTD, indica-se aqui o número de capítulo correspondente.

  • Número ([field] NUM)

Este campo permite registar o número correspondente ao texto a traduzir De notar que uma pesquisa textual pode ser realizada quando se pesquisa uma mensagem ou um texto contendo uma cadeia de caracteres dada. Neste caso, o clique direito Pesquisa mensagem ou Pesquisa texto permite registar a cadeia a pesquisar; mas a janela de seleção não permite que registar uma mensagem ou um texto, o que torna maneira de fazer fastidioso se pesquisa um conjunto de textos. Passar-se-á então pelo clique direito sobre a linha, descrito na documentação da função.
  • Texto standard ([field] STD)

Este campo permite afixar o texto standard tal que existe no dicionário entregue com o software.

  • Texto ([field] TXT)

Regista-se aqui o texto destinado a substituir o texto standard nos ecrãs do dossier.

Acresc. textos

Este botão permite carregar num quadro o conjunto dos textos contendo uma cadeia de caracteres dados. Uma vez estes textos carregados, pode-se registar as traduções na coluna Texto.

Cópia

permite transferir o conteúdo da tabela completa versus um outro dossier.

Validação

O botão [Validação] permite de alimentar as tabelas ATEXTE e APLSTD do dossier corrente a partir do vocabulário registado na tabela AVOCAB.
Permite unicamente a validação dos ecrãs e janelas cujos textos foram modificados.
A revalidação do dicionário via a função de validação do dicionário é necessário se o texto toca por exemplo o intitulado de um menu ou de um objeto.
