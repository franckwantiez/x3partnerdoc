Habilitação funcional

Esta função permite anexar a, um código perfil função definido por outro lado, um conjunto de funções autorizadas com privilégios complementares se for caso disso.

Pré-requisitos

[Refer to documentation [Implementation]]

Gestão do ecrã

[Tab] Ecrã de registo

A definição das habilitações funcionais se faz em registo dos conjuntos (códigos perfis, códigos função) numa ficha.

O quadro apresenta sobre a ficha não é útil que quando as habilitaões particulares existem para a função. pode-se tratar :

  • Seja as restrições por estabelecimentos (neste caso, cada linha definida das habilitações para um ou vários estabelecimentos)
  • Seja as opções complementares (uma ou várias linhas podem ser então registadas segundo que as habilitações são ou não moduladas por estabelecimento.

Um controlo de coerência está feito sobre os estabelecimentos listados no quadro das habilitações. Assim, se dois estabelecimentos pertencem a reagrupamentos diferentes aos quais acordou-se as autorizações distintas, uma mensagem de erro será afixada, e o registo não poderá ser feito.

Para aceder o registo nos casos complexos, dispõe-se de botões para acrecentar globalmente um conjunto de funções ou para fusionar os perfis.

No registo, atualiza-se a tabela guardando a definição das habilitações, mas não atualiza a tabela dos cruzamentos de habilitação. Esta atualização será feita no momento da saída da função, se responde Sim à questão Atualização da tabela AFCTFCY. 

Perfil

  • Código perfil ([field] PRFCOD)

Este código identifica de maneira única a ficha em curso

Função

  • Função ([field] FNC)

Código identificando uma função do software.

  • Módulo ([field] MODULE)

Módulo ao qual a função corrente está anexada.

[Grid] Lista dos valores

  • Tipo ([field] FCYGRUCOD)

Define se as habilitações são registados para os dados ligados a um estabelecimento ou a um reagrupamento de estabelecimentos.
  • Reagrupam./estab. ([field] FCYGRU)

Permite definir o estabelecimento ou o reagrupamento de estabelecimentos afectado pelas habilitações.

Um grupo de estaeblecimentos vazio permite dar um valor por defeito para todos os estabelecimentos não explicitamente definidos para as linhas sucessivas. Se a função não possui sem habilitação por estabelecimento, este campo resta vazio, e uma só linha pode ser registada no quadro para definir globalmente os direitos de acesso.
  • Acesso ([field] ACS)

Permite autorizar ou recusar o acesso para o ou os estabelecimentos definidos sobre a linha pela função corrente e o perfil utilizador afectado. O acesso associado ao reagrupamento vazio corresponde aos direitos acordados sobre todos os estabeelcimentos não explicitamente listados no quadro.
  • Opções ([field] OPT)

Este campo permite listar, por um conjunto de códigos (1 caracter) seleccionáveis e dependendo da função, dos direitos de acesso complementares. Não pode estar registado que se autorizações complementares são anexadas à função.

Estas autorizações dependem da função, e são documentadas na secção Pré-requisitos de cada documentação funcional. Por exemplo, para a gestão dos utilizadores, existem as autorizações seguintes :

  • P permite autorizar uma alteração de palavra passe.
  • A permite autorizar uma modificação dos códigos de acesso.

Certos direitos de acesso genéricos existem, nomeadamente para as funções de tipo objecto. As opções correspondentes são as seguintes :

Opção :

Significado

C

Criação da ficha

M

Modificação da ficha

S

Supressão de ficha

N

Autorização de tornar a ficha activa (se a check box a assinalar - campo ENAFLG - existe).

Existe por outro lado outras habilitações normalizadas, e nomeadamente :

  • Quando um botão "validação" existe para a função, uma opção nomeada V permite acordar o direito a este botão.
  • desde logo que um tratamento pode ser lançado com um filtro sobre os dados associados a utilizadores, uma opção nomeada U permite assinalar uma check box. Todos utilizadores : ou de registar um código utilizador outro que o código utilizador corrente.

Observação :

Um ponto importante está a notar. a tabela dos perfis funções não gera que o código perfil, o detalhe das habilitações por grupo ou estabelecimento gerido pela tabela das habilitações funcionais. Mas, o mais importante, para a gestão das habilitações, é a tabela perfil função estabelecimento,que é colocada automaticamente a dia para esta função, mas igualmente, se necessário para a gestão dos reagrupamentos, das sociedades, e dos estabelecimentos.

Com efeito, esta tabela gera em detalhe as autorizações para cada cruzamento estabelecimento/perfil/função Conta tida do número de funções importante no software (várias centenas), um dossier com muito de estabelecimentos pode ter uma tabela AFCTFCY integrando um grande número de linhas (ela tem todavia registos de pequeno tamanho). Esta tabela permite obter as boas performances no momento da gestão das habilitações, em particular nas listas. A sua atualização se faz à saída das funções de atualização das habilitações Cômputo do grande número de informações a atualizar, esta função pode ser longa na execução. É igualmente possível de lançar esta operação diretamente em utilizando o utilitário associado (Validação das funções).

Botões específicos

  • Tipo de supressão ([field] DELTYP)

Define o tipo de supressão desejado :

  • Função não suprime que a habilitaação sobre a função corrente (para o perfil corrente).
  • Perfil função suprime as habilitações para todo o perfil função.
  • Módulo suprime as habilitações para todas as funções do módulo registo no campo que segue (para o perfil função corrente).
  • [field] MODULE

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.

Permite suprimir os dados de habilitação ligadas ao perfil.

  • Módulo ([field] MODULE1)

Módulo ao qual a função corrente está anexada.
  • Menu ([field] MENU1)

Quando se cria uma função, é indispensável da referenciar num menu, afim que esta função possa ser chamada.. Os menus são parametrizáveis pelo utilizador, mas existe um utilizador de referência, chamado aqui ADMIN, mas cujo nome está modificável pelo parâmetro ADMUSR. Os seus menus contêm todas as funções, e podem servir de modelo no momento da criação dos menus. Define-se aqui um menu de referência onde se vai encontrar a função.

O menu de partida da arborescência standard se chama GENE ; toda a função deve ser referenciada, seja no menu GENE, seja num menu ele mesmo acessível a partir do menu GENE Os constrangimentos são os seguintes :

não é possível de ter mais de 30 escolhas no menu.

não é possível de ir para além de 4 níveis de sub-menus.

O intitulado menu é o texto (traduzível) que aparecerá efetivamete no menu de utilizador para designar a função.
  • Função ([field] FNC1)

Código identificando uma função do software.

  • Descrição ([field] LIBFNC1)

Título associado ao código precedente

Permite abrir um ecrã de carregamento em massa de funções

Trata-se de definir, para um estabelecimento ou um grupo de estabelecimentos, o direito de utilização (em incluindo globalmente todas as opções listadas ou nenhuma).

As funções afetadas podem ser carregadas num quadro por picking a partir da lista arborescência se encontrando à esquerda. Esta lista retoma as funções tais que aquelas são organizadas nos menus standards de utilizador ADMIN.

Este botão permite alterar diretamente todas as funções ligadas a um módulo, outro o picking direto de uma função ou de um sub-menu.

O registo deste quadro provoca a criação das habilitações correspondentes.

Modificação de um perfil função

Este botão permite de alterar em lista esquerda todos os direitos e acesso do perfil-função afetado.

O acréscimo, a modificaação ou a anulação de direitos e acesso do perfil se faz em lista esquerda.
O quadro está alimentado automaticamente.

Para validar as modificações trazidas ao perfil, clique sobre [Registar].
