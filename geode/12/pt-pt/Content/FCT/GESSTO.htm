Armazéns

Um armazém é um conjunto de endereços que permite um corte representativo (físico ou lógico) de entreposto. Um armazém está no interior de um estabelecimento.

O numero de armazéns assim que a sua codificação são totalmente livres.

[Refer to documentation [Implementation]]

Gestão do ecrã

  • [Tab] Geral
  • [Tab] Endereços
  • [Tab] Proximidade Picking

Para criar um armazém, atribui-se um código que identifica de maneira sobre 3 caracteres e um intitulado. Estas duas informações aparecem no cabeçalho do ecrã de gestão, 2 separadores de definição permitindo dar as características logísticas.

Falta no mínimo ter criado a zona de preparação na qual vai se anexar o armazém. Certos outras tabelas são necessárias, mas elas podem ser criadas por túnel directamente depois a gestão dos armazéns.

Cabeçalho

O cabeçalho contém o código armazém identificando de maneira única.

  • Armazém ([field] STO)

Código permitindo de identificar o armazém. Verificado na tabela dos armazéns.

  • Descr. ([field] STODESAXX)
  • Desc. curta ([field] STOSHOAXX)

[Tab] Geral

Este separador permite definir as características gerais do armazém que se dividem em 5 blocos :

  • Tipo de armazém
  • Remanescentes aprovisionamento
  • Validação dos movimentos
  • Saida
  • Acumulado de fluxo

Tipo armazém

  • Tipo armazém ([field] TYP)

O tipo de armazém definido a função logística de um armazém:

  • armazém principal: "picking", reserva, massa, local de arrumação
  • armazém mono endereço: armazém lógico no qual os stocks são localizados; um únici endereço sem controlo de preenchimento pode ser definido para este armazém.
  • armazém de separação: tipo de armazém necessário para utilizar a funcionalidade "Acumulado de fluxo" (preparação em 2 fases); os acondicionamentos "zona separação" podem ser consideradas como acondicionamentos reservados utilizado como acondicionamentos pickings temporários.
  • armazém cais (de recepção, de expedição): utilizado para as funcionalidades de "Reuniões transportador" e "Stock no cais das unidades logísticas"
  • armazém Cross-docking: utilizado pela funcionalidade de "Cross-docking".
  • armazém Tampão: armazém provisório
  • armazém Transferência em 2 fases utilizado pela funcionalidade de "Trasnferência em 2 fases".

Partida

O sistema autoriza as antecipações automáticas (emitidos da pesquisa em saída) depois este armazém.

  • Zona de preparação ([field] PKZ)

Código da zona de preparação que é verificado na tabela das zonas de preparação. A noção de Zona de Preparação é utilizado pelo corte das Ordens de Antecipação.
  • Priorid. na saída ([field] OUTPIO)

A noção de prioridade armazém em saída é um dos 6 critérios de modo de saída, ela permite de indicar a ordem de exploração dos armazéns. Se vários armazéns têm o mesmo nível de prioridade é o critério segguinte que está tido em conta. Se não há critério hierarquicamente inferior, é o primeiro objecto encontrado que está desarmazenado. O valor A, por defeito, é a prioridade a mais importante.

  • Autoriz. na saída ([field] OUTAUZ)
  • Transfer. objecto completo ([field] FSOTRF)

As saidas do armazém devem obrigatoriamente se efectuar em objectos completos. No caso contrário, uma saida parcial limitada à necessidade está efectuada. Este parâmetro não foi considerado que para os movimentos não directamente ligados a uma OE (transferência 1ª fase do acumulado de fluxo ou transferência).
  • Inventário preventivo ([field] SKPFLG)

Se a check box está assinalada, o armazém os inventários preventivos sobre estes acondicionamentos picking das etapas de antecipação de OP.

Observação : apenas os acondicionamentos picking com gestão dos objectos de stock indiferenciados são afectados.

Validação dos movimentos

  • Reg. cod. controlo ([field] CTLCODDAC)

O sistema pede o registo de uma chave de controlo no momento da validação dos movimentos se:
O artigo, o armazém, o acondicionamento e o fluxo em questão (configuração das entradas, das saidas, tabelas dos estabelecimentos) têm uma parametrização "registo dos códigos igual a "sim".
  • Chv. de cálculo ([field] CTLCODKEY)

Zona numérica podendo tomar os valores 1, 2, 3.

No momneto do registo manual ou em caso de modificação manual dos códigos de controlos acondicionamente, não será jamais possível que 2 acondicionamentos cintiguos tenham o mesmo código de controlo. Se o parâmetro está a:  

  • 1, o sistema verifica que exsite pelo menos 1 endereço de desvio entre 2 códigos controlos idênticos.
  • 2, o sistema verifica que existe pelo menos 2 endereço de desvio entre 2 códigos controlos idênticos.
  • 3, o sistema verifica que existe pelo menos 3 endereço de desvio entre 2 códigos controlos idênticos.

Este parâmetro está igualmente utilizado pelo algoritmo de criação dos códigos de controlos em automático.

Excedentes reab.

Este parâmetro é utilizado para pedir o reendereçamento das paletes quebradas pelos aprovisionamentos de um ciclo (caso dos acondicionamentos picking na ca pacidade de acolho é inferior àquela de uma palete standard).

Código do armazém para as reintegrações. O armazém deve ser de tipo« principal« ou « mono-endereço «.

  • Reendereçam. ([field] RPNSHRRAL)
  • Armazém de reintegração ([field] RSTSTO)

Estrutura endereço

Todos os acondicionamentos definidos no interior de um armazém têm um endereço idêntico representa por um código genérico sobre 8 caracteres máximo. Este código aparece sobre certos ecrãs (para guiar o registo) ou listas. Um elemento deste código está constituido por um ou vários caracteres idênticos na seguida e deve ser definido como alfabético ou numérico.

Exemplo : ARRN Corredor (posição 1), Arrumação (posição 2-3), Nivel (posição 4).

5 Níveis sob cortes do endereço são possíveis. O comprimento total dos componentes não deve exeder 8 caracteres. Por defeito as 3 primeiras linhas da estrutura são designados Corredor, Arrumação, Nível mas nenhum significado lógico não está anexado.

Exemplos:

Type N A RR

Tipo A RR N

Tipo corredor

  • Tipo corredor ([field] AISTYP)

O tipo de entrada está seja em linha (os números de arrumação pares e impares de um corredor são alinhados), seja em pente (os números de arrumação pares e impares são alternados, o corredor é formado de duas linhas paralelas).
  • Nr. arrum. face a uma arrum. igual ([field] ODDAISNUM)

Esta opção está disponível unicamente para os corrredores de tipo "pente". Ela define se a classificação em face de uma classificação par é a classificação impar de número imediatamente inferior (-1) ou superior (+1).

[Grid] Estrutura endereço

Código que representa o componente de estrutura do’endereço ao qual está associado.

Esta zona permite de conservar para cada acondicionamento um segundo endereço que será utilizado sistematicamente no momento do acondicionamento em entrada para obter um modo de preenchimento diferente da ordem alfanumérica dos endereços. O valor está compreendido entre 1 e 5 correspondente a 5 linhas da estrutura de’endereço (Corredor 1, nível 2, arrumação 3, outro 4, outro 5).

  • Nível ([field] LVL)
  • Cód. ([field] ADDCOD)
  • Descrição ([field] ADDDESAXX)

Literal que pode explicitar o código ao qual ele está associado.
  • Tipo ([field] ADDTYP)

Permite de indicar ao sistema se o caracter identificando o elemento de endereço é alfabético ou numérico.
  • Comprim. ([field] ADDLEN)

Ela indica o número de características identificando o elemento de endereço.
  • [Tri rangement] ([field] SRGORN)
  • Número ([field] MAXCPN)

[Tab] Proximidade Picking

Este separador permite definir os parâmetros de proximidade picking do armazém que se divide em 3 blocos :

  • Corredor corrente
  • Corredor em face
  • Proximidade estendida

A proximidade "picking" é um modo de pesquisa de acondicionamentos em entrada permitindo localizar as paletes reservadas ao mais do acondicionamento picking para uma dado artigo.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

Erro zona de preparação , zona obrigatória.

Esta mensagem está afixada se nenhuma zona de preparação não foi preenchida no momento da criação do armazém.

Erro zona Chave de cálculo, Número incorrecto (0..3).

O valor registado na chave de cálculo não foi compreendido entre 0 e 3.

Mensagem de informação, Sem ficha seleccionada.

Não existe ficha criada correspondendo ao campo seleccionado.

Erro Zona, Armazém de reintegração.

O armazém de reintegração deve obrigatoriamente ser de tipo "principal" ou "mono endereço".

Erro zona Comprimento, zona obrigatória.

Esta mensagem está afixada se nenhuma comprimento não foi preenchida no momento da criação do endereço.

Erro zona Arrumação, zona obrigatória.

Esta mensagem está afixada se nenhuma nível de arrumação foi registado no momento da criação do endereço.

Erro zona intitulado, zona obrigatória.

Esta mensagem está afixada se nenhuma comprimento não foi preenchida no momento da criação do endereço.

Erro zona A partir do nível, formato incorrecto (xA ou (x#))

Esta mensagem está afixada se o formato registado não foi idêntico à estrutura previamente criada nos endereços.

Erro zona Arrumação, Número incorrecto (1..5).

O valor registado no campo classificação deve estar compreendida entre 1 e 5.
