Leitura dos rastos

As funções do software susceptíveis de durar um certo tempo, e em particular todas aquelas que são susceptíveis de funcionar em batch, criando um ficheiro rasto com um cômputo detalhado incluindo os erros se existir um.

Estes ficheiros rasto são criados no sub-directório TRA do diretorio do dossier, sobre o servidor. Têm um nome composto sob a forma F#.tra, onde # é um número sequencial atribuido com ajuda do contador [C]NUMIMP.

A primeira linha de um ficheiro rasto está estruturada como segue :

 

#<AdxVJ>@(#)%I% DD-MM-AA hh:mm:ss (código utilizador) Comentário explicativo

A primeira parte do rasto (apresentado em cinzento) é um cabeçalho standard ; seguem em seguida a data e hora, o código do utilizador tendo lançado a função, e enfim um comentário explicativo.

Encontra-se em seguida no rasto das linhas criadas pela função executada ; estas linhas são prefixadas pelos caracterees > ou <, seguidas de um número de erro, se a linha faz referência a um erro constatado na função.

Gestão do ecrã

Quando se entra na função, regista-se o nome do ficheiro rasto ; uma janela de seleção está acessível para selecionar os ficheiros existentes.
Esta escolha faz, o ecrã de leitura de rasto se abre.

[Tab] Ecrã de registo

Este ecrã permite afixar até 1000 linhas de rasto Se está mais no ficheiro, os botões radio não estão carregados, o ficheiro sendo cortado em páginas lógicas de 1000 linhas. É então possível de utilizar os botões redondos para ir à página seguinte, ou precedente, ou de ir em primeira ou em última página. O número de página corrente e o número total de páginas do ficheiro são afixados no título do quadro das linhas.

[Grid]

  • [field] LIGNE

Cada linha do quadro correspondente a uma linha no ficheiro de rasto.

Botões específicos

Erro seguinte

Este botão permite navegar no rasto, em se deslocando sucesivamente sobre todas as linhas onde de encontra um erro. Se o erro não se encontra na página de 1000 linhas correntes, este botão provoca a afixação da primeira página de 1000 linhas seguinte ou se um erro existe.

É de referir que uma série de linhas de erro consecutivos está considerado como uma linha de erro (salta-se efetivamente no primeiro erro não consecutiva seguinte).
Aviso seguinte

Este botão permite navegar no rasto, em se deslocando sucesivamente sobre todas as linhas onde de encontra uma advertência. Se a advertência não se encontra na página de 1000 linhas correntes, este botão provoca a afixação da primeira página de 1000 linhas seguintes ou se um erro existe.

É de referir que uma série de linhas de advertência consecutivas está considerado como uma linha de erro (salta-se efetivamente no primeiro erro não consecutivo seguinte).

Barre de menus

Opções / Página

Permite diretamente a uma página dada. Regista-se este número numa janela que se abre.
Opções / Pesquisa

Permite de pesquisar um texro num ficheiro de rasto. O texto está registado numa caixa de diálogo que se abre, e se o texto existe no rasto, uma caixa afixa o número de ocorrências encontradas. O ecrã não se afixa então que as linhas afectadas.
Opções / Erros

Permite não afixar, no ecrã de rasto, que as linhas de erro e de advertência. Esta afixação está precedida da afixação, numa caixa de informação, do número total de erros e advertência contido no ficheiro de rasto.

A linha de erro está seguida de um número de linhas entre parenteses por exemplo, (5304). Isto significa que em se posicionando sobre a página 5, encontrar-se-á o erro em linha 304.
Opções / Detalhe

Permite, quando está passado em afixação parcial do rasto (pela função de pesquisa ou em não afixando que os erros), de voltar a afixação completa do ficheiro de rasto.
Opções / Cópia

Permite copiar o ficheiro de rasto num outro ficheiro, cujo caminho está indicado de maneira absoluta. Este caminho está situado por defeito sobre o servidor de aplicação. Mais, para a sintaxe serveur@chemin, é possível de o declarar sobre um outro servidor. A sintaxe #@chemin permite nomeadamente de declarar sobre o posto cliente, na condição que o posto cliente seja de tipo Windows (em Web, os constrangiemntos de segurança ligadas ao navegador Web impedindo esta função de operar).
Opções / Envio

Permite de enviar o ficheiro rasto por mail a um conjunto de utilizadores quaisquer. Regista-se a lista dos utilizadores que o envio deve estar sido feito, assim que o texto de acompanhamento (um texto por defeito está proposto). O endereço mail associado ao utilizador conectado por proposto por defeito.

Colocação em obra técnica dos rastos

O programador gera um ficheiro de rasto para as funções seguintes :

 

Call OUVRE_TRACE("COMMENTAIRE") From LECFIC

: # Abertura de um ficheiro rasto

 

Call ECR_TRACE("TEXTE",ERREUR) From GESECRAN

: # Registo de um texto no rasto
: # si ERREUR=0 : Informação
# si ERREUR>0 : Erro afixado em vermelho
:# si ERREUR<0 : Informação numa cor particular

 

Call FERME_TRACE From LECFIC

: # Fecho do rasto

 

Call LEC_TRACE From LECFIC

: # Leitura do rasto
