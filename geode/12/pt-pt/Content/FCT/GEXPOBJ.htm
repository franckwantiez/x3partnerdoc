Exportações

Esta função permite de extrair os dados da base versus um ficheiro de tipo ascii em comprimento fixo ou variável com separadores, cuja estrutura está descrita por um modelo de importação/exportação. Esta exportação pode ser total ou cronológica, quer dizer em não exportando que os dados modificados depois a última exportação deste tipo.

 Os dados exportados, se eles são traduzidos, o serão na língua de conexão do dossier.

Gestão do ecrã

[Tab] Ecrã de registo

Ao lançamento da função, um ecrã permiite de escolher os parâmetros relativos à exportação a lançar.

[Block number] 1

  • Modelo a utilizar ([field] MODELE)

corresponde ao modelo de importação/exportação colocado em obra.

  • Gestão cronologia ([field] CHRONO)

Se esta check box está activada, a exportação faz-se unicamente sobre os registos modificados depois a última exportação. O princípio de funcionamento deste modo de exportação é o seguinte:

  • um contador (variável numérica comum a um momento dado para todos os postos de um dossier, guardado na base de dados, cujo nome é [C]EXPORT), constitui uma base de tempos (ele se incrementa em cada vez que uma exportação com gestão de tempo é lançada).
  • O valor corrente do contador [C]EXPORT é reportado nos registos da base de dados quando uma criação ou uma actualização é feita. Este mecanismo é válido para todas as tabelas permitem de realizar uma exportação cronológica (todas as tabelas de base e as entidades de movimento incluindo este campo: isto representa perto de metade das tabelas da base de dados). O campo correspondente se chama EXPNUM.
  • No momento da exportação cronológica, guarda-se o valor corrente do contador EXPORT no modelo de importação/exportação, depois se incrementa, e não se exporta que os registos da base para a qual o valor do campo EXPNUM é inferior ou igual ao valor do campo EXPORT antes de incrementação, e superior estrictamente no último valor previamente guardado no modelo.

Quando se faz uma exportação com uma gestão de tempo, é possivel de utilizar o metacaracter # no nome do ficheiro a exportar ; este caracter está então preenchido pelo valor corrente do contador EXPORT formatado sobre 4 caracteres. Assim, o nome de ficheiro dados#.asc será substituido por dados00001.asc, dados00023.asc, etc. … segundo o valor do contador.

Bem entendido, para que este tipo de exportação exista, a zona EXPNUM deve existir na tabela ligada ao objecto exportado. Se ela existe, ela é automaticamente preenchida pela gestão do objecto, e pode ser então utilizado para as exportações deste tipo.

O valor corrente do contador [C]EXPORT é modificável nos parâmetros de importação-exportação

Geração do ficheiro

  • [field] TYPEXP
  • Ficheiro de dados ([field] VOLFIL)

[Grid] Limites

  • Zona ([field] BNOM)

Define o nome do campo para o qual um limite pode ser registado
  • Início ([field] BDEB)

Permite preencher os limites inferiores e superiores sobre um conjunto de campos do modelo, afim de limitar os dados a exportar. Se estes limites não estão preenchidas, nenhuma filtragem é feita sobre o campo respeitante.
  • Fim ([field] BFIN)

[Grid] Critérios

  • Tabelas ([field] FICHIER)

As tabelas colocadas em obra no modelo de importação/exportação são listadas aqui, afim de permitir uma filtragem eventual dos dados exportados.
  • Critério selecção ([field] CRITERE)

Permite definir critérios suplementares de filtragem dos dados a exportar para a tabela afectada, sob a forma deexpressões lógicas que devem ser verificadas e que podem utilizar os campos da tabela.

Tarefa batch

[This function can be run in batch mode]. [The standard task] EXPORT [is provided for that purpose].

Botões específicos

[Grid]

  • [field] LIGNE

Contém as linhas do tratamento.

Provoca a afixação do tratamemnto de descodificação dos dados, emitido do modelo, que vai ser executado (este tratamento está nomeado WWE#####, onde ##### é um número atribuido automaticamente)

Uma janela se abre então, permitindo visualizar ver de modificar este tratamento, antes de lançar a execução pelo botão correspondente, ou ao contrário de abandonar o lançamento da importação.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

Ficheiro destino existente, apagamento (Sim/Não)

O ficheiro a exportar já existe : se confirma, o precedente ficheiro será destruído pelo resultado da exportação.

Directório inexistente

O directório no qual se pretende ver escrever o ficheiro a exportar não existe mais.

Exportação impossível sobre este modelo

O modelo não está utilizável em exportação.
