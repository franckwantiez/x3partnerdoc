Escolha personaliz.

Esta função permite utilizador de alterar certos parâmetros que afectam as suas preferências ou dos valores por defeito. Apenas os parâmetros cuja modificação está autorizada ao utilizador ele mesmo aparecem neste ecrã, contrariamente à ficha utilizador, pode-se ver potencialmente todos os parâmetros utilizadores.

Pré-requisitos

[Refer to documentation [Implementation]]

Gestão do ecrã

  • Cabeçalho
  • [Tab] Opções
  • [Tab] Favoritos funções

Cabeçalho

O código do utilizador corrente está afixado em cabeçalho, e encontra-se em seguida as informações modificáveis sobre 2 separadores.

  • Utilizador ([field] USR)

Corresponde ao código utilizador de conexão.

[Tab] Opções

Este ecrã permite definir à vez um conjunto de parâmetros utilizadores modificáveis pelo utilizador ele-mesmo, e os destinos propostos por defeito para as edições para os quais nenhuma impressora por defeito não existe, em função do tipo de destino pedido.

[Grid] Parâmetros

  • Capítulo ([field] DESCHP)

Este campo unicamente afixado define o capítulo no qual se encontra o parâmetro afectado.
  • Grupo ([field] GRPPAR)

Este campo unicamente afixado define o grupo no qual se encontra o parâmetro afectado.
  • Parâmetro ([field] PARAM)

Este código identifica o parâmetro a preencher.
  • Descr. ([field] NAME)

Título associado ao código precedente

  • Valor ([field] VALEUR)

Regista-se aqui o valor do parâmetro. Esta é registada segundo os casos como um número, uma data, ou um código alfanumérico. Quando o parâmetro é de tipo menu local, pode-se à escolha registar o início do título, ou o valor numérico correspondente à fila do intitulado na lista. Quando o código está controlado por uma tarefa anexa, um toque de seleção está utilizável para o escolher.
  • Nível ([field] NIVEAU)

Este campo indica o nível de definição sobre o qual o parâmetro está preenchido. Os valores possíveis são definidos pelo menu local 987. Quando o nível está indicado em face e um grupo de parâmetros, é o nível o mais fino reencontrado sobre os parâmetros do grupo que está afixado.

[Grid] Destinos por defeito

  • Tipo ([field] TYPIMP)

Este campo afixado define o tipo de destino a partir dos valores do menu local parametrizável número 22.
  • Destino ([field] PRTDEF)

Este código identifica, por cada tipo de impressora definida pelo menu local parametrizável 22, um destino de impressão que será proposto por defeito no lançamento de uma lista na ausência de regra mais precisa.

Diversos

  • Barra de título ([field] TIT)

Este campo permite de definir, pelo intermédio de uma expressão calculada, o conteúdo da barra detítulo da janela do software. Por defeito, encontramos um título que é o seguinte :

DOSSIER         NOME DO SOFTWARE     -  [Intitulado da função corrente]

Os campos DOSSIER e NOME DO SOFTWARE são substituidos pelo resultado da avaliação da fórmula, se ela existe. Pode-se nomeadamente fazer aaprecer nesta fórmula das constantes, mas também as variáveis globais e as funções seguintes :

nomap

Nome do dossier corrente

GUSER

Código do utilizador corrente

GNOMUSER

Nome do utilizador corrente

data$

data corrente

tempo$

hora corrente (à conexão)

É importante de notar que este título não é avaliado que apenas uma vez na entrada do software.
Ajuda

Esta opção permite de afixar a ajuda relativa ao parâmetro da linha corrente.

[Tab] Favoritos funções

Este separador, que não está acessível via a gestão dos utilizadores, permite definir os atalhos versus funções via a pressão de uma tecla (as teclas  a   estão reservadas a este uso, mas é possível de atingir as funções pelo menu Utilitários situado na barra superior de menus). Toda a função pode assim potencialmente ser chamado, a partir do momento onde o seu código está registado no quadro (as teclas sendo atribuidos na ordem do quadro). O lançamento de uma função desta maneira se faz em abrindo uma segunda sessão.

Por outro lado da função, regista-se o código da transacção (ou do ecrã, ou do parâmetro complementar), se a função admite um tal parâmetro.

Deve-se notar que ests funções não aparecem ao menu que quando está já numa função. Por outro lado, o refrescamemto do menu não se faz que à conexão : é entõa necessário de se reconectar após modificação destas escolhas.

[Grid]

  • Funções ([field] FNCCOD)

Permite definir um código função do software afim de poder o utilizar. Os códigos função, referenciados numa tabela dedicada do dicionário, podem também ser reencontrados via as info-bolhas sobre um menu utilizador, ou ainda em olhando o valor da variável GFONCTION pela calculadora, quando se encontra conectado na função.
