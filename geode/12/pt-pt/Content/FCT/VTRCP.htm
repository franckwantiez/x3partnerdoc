Recepção

Esta função permite recepcionar a mercadoria de uma entrada esperada (EE) via os procedimentos radio-frequência, quer dizer:
- de registar a quantidade dos artigos recebidos para cada linha de recepção esperada
- de declarar o fim de uma recepção administrativa e de gerar uma entrada directa (ED).

Esta recepção pode se fazer em modo normal (modo por defeito) ou "Drip & Drop":. De mais 3 procedimentos de recepção são possíveis:
- Recepção por Código de Barras fornecedor:
 - recepção por Código EAN fornecedor.
- recepção por número de suporte.

"Drip & Drop":

Método :

[Tab] Fornecedor

 Código de barras fornecedor (Recepção por Código de barras fornacedor)
- zona obrigatória
- o código de barras fornecedor, cujo formato está psarametrizado na tabela dos estabelecimentos, incluindo a referência de cabelçalho EE seguida da referência EA; deve identificar de maneira não equivoca uma única linha EE da recepção
-esta linha EE não deve ser totalmente recepcionada (lista < 3 & quantidade restante a recepcionar>0 se o modo de recepção controla a ultrapassagem de quantidade) e ela deve corresponder a uma entidade EE em curso de recepção (lista >2 e <8)
proposta a lista dos Códigos de barras dos fornecedores das linhas EE não totalmente recepcionados da recepção.

Código EAN fornecedor (Recepção por código EAN fornecedor)
- zona obrigatória
- O Código EAN fornecedor permite de identificar um trio (Fornecedor, Artigo, recipiente) permitindo seleccioanr de maneira não quivova uma única linha EE da recepção.
-esta linha EE não deve ser totalmente recepcionada (lista < 3 & quantidade restante a recepcionar>0 se o modo de recepção controla a ultrapassagem de quantidade) e ela deve corresponder a uma entidade EE em curso de recepção (lista >2 e <8)
propõe a lista dos Códigos EAN fornecedores conhecidos do sistema (i.é. declarado sobre os conteúdos artigo).

Número de suporte (Recepção por número de suporte)
- zona obrigatória
- o número de suporte vai permitir seleccionar uma linha EE da recepção.
-esta linha EE não deve ser totalmente recepcionada (lista < 3 & quantidade restante a recepcionar>0 se o modo de recepção controla a ultrapassagem de quantidade) e ela deve corresponder a uma entidade EE em curso de recepção (lista >2 e <8)
propõe a lista dos númeors de suporte conhecido da recepção.

Código de barras artigo
- zona obrigatório e registável unicamente em recepção Código de barras fornecedor.
- segundo a parametrização da tabela dos estabelecimentos, o código de barras registado ou feito por scan pode ser :
  . seja o código EAN do recipiente do artigo da linha EE, O recipiente de entrada não está então modificável.
  . seja o código artigo da linha EE, o recipiente entrado está então modificável.
É possível seleccionar o código EAN ou do código artigo com a tecla 

Quantidade
- zona positiva não nula
- o nº UC recebido (Quantidade * nº UA std do Recipiente) não deve ultrapassar nem a quant. máxima E/S do Cnt eventualmente parametrizado, nem a quantidade a recepcionar se o modo de recepção o controla.

Recipiente entrada
- não registável se recepção por código EAN fornecedor
- não registável se código de barras = Código EAN, zona obrigatória senão
propõe a lista dos recipientes para o artigo recepcionado.

[Tab] Stock/Lote

 Natureza de stock
- zona facultativa
- O operador pode registar uma natureza de stock particular em função do estado da mercadoria recepcionada (mercadoria destruida por exemplo)
propõe a lista das naturezas de stock.

Nº lote
- zona obrigatória
- registável ou não segundo a configuração das entradas e o artigo recepcionado.

Nº suporte
- zona facultativa
- registável ou não segundo a configuração das entradas

a tecla regista a quantidade recebida com as suas características (linha EE, recipiente, natureza de stock, nº suporte, nº lote) :
- seja uma linha de recepção existe já com as mesmas características, a quantidade recebida está então adicionada.
- no caso contrario, uma nova linha de recepção foi criada para a linha EE tratada.

Botões específicos

ED

Este botão está activo tanto de um código de barras Fornecedor/EAN Fornecedor/Nº suporte não registado
Este botão permite declarar o fim de uma recepção administrativa, o que :
-> Em modo normal
- saldo da recepção administrativa
- saldo eventualmente da entrada EE se é a última linha a recepcionar de EE
- gera uma entrada directa.
-> Em modo "Drip & Drop":
- gera uma entrada directa.

Segundo a parametrização da configuração das entradas, esta operação pode necessitar o registo de um código operador.
