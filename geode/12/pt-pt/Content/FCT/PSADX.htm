Utilizadores

Esta função permite visualizar as diferentes tarefas sage em curso sobre um servidor. Por defeito, as tarefas do servidor de tratamento corrente são afixadas, mas todo o servidor acessível via a rede sobre o qual torna um processo de conexão ao software com o mesmo número de serviço pode ser acedido.

Gestão do ecrã

Esta função se apresenta sob a forma de um único ecrã sem lista esquerda.

[Tab] Ecrã de registo

Quando entra nesta função, as características de conexão sobre o servidor por defeito são afixadas. Se se deseja visualizar as conexões sobre um outro servidor, é suficiente dar o seu nome. O escrutínio será possível desde logo que o número de serviço é o mesmo sobre o servidor distante.

Neste ecrã, três blocos de dados são afixados :

  • o primeiro, nomeado Sessões abertas, dá a lista das conexões estabelecidads sobre o servidor de um quadro;
  • O segundo, nomeadoProcessos ativos,  permite listar os processos relativos a uma sessão (a sessão definida por linha corrente sobre a qual o cursor está posicionado no primeiro bloco);
  • o último, nomeado Total sessões, dá as informações acumuladas.

[Block number] 1

  • Máquina ([field] SERVEUR)

Define o servidor de tratamento cujo se visualiza os processos. Por defeito, o servidor que está proposto quando se entra na função é o servidor de aplicação corrente.
  • Licença ([field] LICENCE)

Define o número de licença concedidda por Sage para a instalação. Este número de licença é único. Está associado a um conjunto de autorizações (número de sessões, opções...) assim que há duas series de textos identificando o cliente utilizador da licença e o parceiro tendo colocado a solução

Uma função dedicada permite visualizar o conjunto destas informações.

  • Total sessões ([field] NBTOTSES)
  • Tempo CPU ([field] CPUFLG)

[Grid] Sessões abertas

  • Cliente ([field] CLIENT)

Define o nome de rede do posto utilizado para estabeecer uma conexão da sessão. De referir que este nome rede pode ser :

  • seja o posto cliente que se conecta em cliente-servidor.
  • seja o servidor Web que faz a ponte entre um navegador e o servidor.
  • seja a palavra-chave BATCH, nos caso onde o processo é uma tarefa batch que não tem do cliente associado.
  • seja o servidor Web sobre o qual torna o servidor de terminais ou de web services.
  • Tipo ([field] TYP)

Define o tipo de conexão da sessão corrente. Corresponde a uma das escolhas definidas no menu local número 924.

  • Cliente web ([field] IPCLI)
  • Login ([field] USER)

Define o código utilizador (no sentido do software) sob o qual a sessão foi aberta.

  • Dossier ([field] DOSSIER)

Os campos Ident1 e Ident2 são indicadores únicos geridos pelo motor sage na conexão do utilizador. O primeiro é único para um dossier dado, corresponde ao resultado de avaliação da função adxuid(1) e é utilizada no ficheiro APLLCK (é o campo LCKPID). O segundo é único para um servidor e um número de serviços dados, corresponde ao resultado da avaliação da função adxuid(2).
  • Módulo ([field] MODULE)

Define o módulo ao qual está anexado em função em curso de execução. A lista dos módulos varia segundo o software. Ela corresponde ao menu local número14.

  • Função ([field] FONCTION)

Define o código da função executada pelo utilizador conectado. Este código corresponde ao valor da variável GFONCTION sobre o posto ele mesmo.

Se o utilizador está conectado sem estar numa função (ie. se encontra posicionados sobre um menu), esta zona está vazia

De referir que o servidor batch ele mesmo está presente como uma tarefa chamada SERVEUR, funcionando em modo BATCH sobre o dossier cujo nome depende do software, mas que começa em perincípio pelas letras "SERV".
  • Login sistema ([field] LOGIN)

Define o código utilizador (no sentido do login do sistema) sub utilizador que se está conectado está conhecido.
  • Serviço ([field] PORT)

Define o número do serviço de conexão (características da tarefa adxd na qual o posto está afetado ao pedido de conexão)
  • Data ([field] DATCON)

Define a data e hora do início de conexão do posto.
  • Hora UTC ([field] HRECON)
  • Data + hora UTC ([field] DATTIME)
  • CPU ([field] CPU)

Afixa o tempo CPU utilizado pelo processo sage depois o seu arranque. Este tempo está exprimido em minutos:segundos
  • Id Sessão ([field] ID)

Os campos Ident1 e Ident2 são indicadores únicos geridos pelo motor sage na conexão do utilizador. O primeiro é único para um dossier dado, corresponde ao resultado de avaliação da função adxuid(1) e é utilizada no ficheiro APLLCK (é o campo LCKPID). O segundo é único para um servidor e um número de serviços dados, corresponde ao resultado da avaliação da função adxuid(2).
  • Natureza ([field] NATURE)

[Grid] Processos activos

  • Nr. processo ([field] NUMERO)

Para cada processo associado à sessão da linha corrente sobre o quadro superior, encontramos aqui um número de processo único ligado ao servidor sobre o qual se executa este processo (é um processo id no sentido do sistema de exploração, quer seja UNIX, Linux ou Windows).
  • Processo ([field] PROCESS)

Define o nome do processo em curso de execução sobre o servidor de tratameto correspondente à sessão corrente no quadro superior. Por entre os nomes de processos frequentemente apresentado sobre a linha, encontrar-se-á :

  • sage, que corresponde ao motor de execução dos processos.
  • sadora, que corresponde ao piloto da sessão da base de dados oracle.
  • sadoss, que corresponde ao piloto da sessão da base de dados SQL server.
  • sadfsq, que corresponde ao processo gerando os acessos a ficheiros sequêciais.
  • sadsys, que corresponde ao processo executante das ordens sistema sobre o servidor.
  • Máquina ([field] MACHINE)

Identifica os computadores sobre os quais giram cada processo associado à sessão corrente do quadro superior. Pode-se agir de um dos servidores de rede (nomeadamente o servidor de tratamento, o servidor de aplicação, mas também o posto do cliente, no caso por exemplo onde se escreve sobre o posto local : um processo sadfsq está então aberto sobre o posto).

Paragem de processo

Esta função permite de parar o processo correspondente (por kill sob UNIX, pala função killadx sob NT). A paragem de um processo significa, do ponto de vista da base de dados, a paragem própria da transação em curso, mas igualmente a perda dos dados em curso de registo. Importa então de ser muito prodente na utilizaação desta função reservada à resolução de incidentes de exploração.

Ao residente, os direitos de utilizador a realizar este género de função dependem das características de conexão do utilizador. Se um utilizador deseja parar as tarefas que não podem ser lançadas por ele, deve ser conectado como super-utilizador (root no caso de UNIX).

Botões específicos

Atualizar

Este botão, igualmente acessível pelo atalho, permite de refrescar a afixação das sessões abertas.
Pesquisa

Este botão permite pesquisar a sessão utilizando um porcesso cujo número está registado. Se o processo pode ser anexado a uma sessão, encontra-se posicionado sobre a linha em questão no fim da pesquisa

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

xxxx : servidor inacessível

Esta mesagem significa que o servidor cujo nome está dado é inexistente, seja que não está conetado à rede, seja ainda que nenhum serviço de conexão adxd não está à espera sobre o número de serviço corrente. Atenção, esta mensagem pode ser afixada após um longo tempo de espera (o tempo que os atrasos de espera atribuir às funções rede estejam vertidas).
