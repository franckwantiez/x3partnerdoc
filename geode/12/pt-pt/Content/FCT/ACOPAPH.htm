Cópia parametrização

Esta função permite copiar parametrização de um dossier SAGE X3 versus um outro dossier SAGE X3

Dois casos são possíveis :

  • Se os dossiers estão na mesma solução SAGE X3, a cópia é directa.
  • Se os dossiers estão nas soluções SAGE X3 diferentes, esta função gera unicamente um ficheiro de tipo patch que deverá ser reintegrado pela seguinte no dossier de destino com a ajuda da função Integração de patch.

 No caso de cópia directa, você não pode indicar um caminho de ficheiro, mas indicar os dossiers de origem e de destino.
Estes dois dossiers devem se encontrar na mesma solução.

Para facilitar a cópia de parametrização, a noção de modelo de parametrização foi introduzido. A criação de modelo evita preencher em cada cópia o conjunto da parametrização.
Este campo é obrigatório e permite de pré-preencher o quadro dos dados a copiar. Portanto, este quadro pode ser modificado e completado antes do lançamento da cópia.

Gestão do ecrã

O ecrã de registo permite definir, outro o tipo de tratamento desejado (cópia directa ou criação de um ficheiro de patch), um código modelo de parametrização. Uma vez este modelo preenchido, pode-se modificar ou completar as linhas afixadas : cada linha define, pelo meio de um modelo de dados e de critérios de selecção complementares, as linhas a extrair ou copiar das tabelas de parametrização afectadas.

[Tab] Ecrã de registo

Uma vez o registo validado, a cópia é executada se foi esta operação que foi pedida. Se se pediu a criação de um "patch", entra-se automaticamente na função de criação de patch, o conjunto das linhas correspondentes ao registo feito aqui sendo apresentados. Pode-se então completar o ficheiro "patch" cuja criação é proposta.

De referir que o "patch" proposto inclui à vez as linhas de modelos de dados (tipo =AAA), mas igualmente o "patch" do modelo de parametrização ele mesmo (tipo=APH), que não é então obrigatório de criar no patch para ver os parâmetros transferidos.

Tipo de cópia

  • Tipo de cópia ([field] TYPTRS)

Selecione o tipo de cópia. Dois valores são possíveis :

  • Cópia : A parametrização está lida no dossier de origem e diretamente atualizada no dossier de destino.
    Você deve preencher os campos Dossier origem e Dossier destino.

  • Patch : Um patch está gerado unicamente. Para recuperar a parametrização no dossier de destino, o patch deve ser integrado neste dossier.
    Você deve registar o caminho do ficheiro de destino.

Cópia

  • Dossier origem ([field] DOSORG)
  • Dossier destino ([field] DOSDES)

Patch

  • Tipo de destino ([field] TYPEXP)

Selecione o tipo de destino no caso de integração de patch :

  • Cliente
  • Servidor: você deve preencher o caminho do ficheiro de destino.
  • Nome do ficheiro ([field] VOLFIL)

Modelo de parametrização

  • Código modelo ([field] MODPAT)

Identifica o código do modelo de parametrização.
  • Legislação ([field] CODLEG)

Este campo permite seleccionar os dados a tratar aqueles relativos a uma única Legislação :.

[Grid] Modelos de dados

  • Código modelo ([field] MODDON)

O modelo de dados selecionado aqui permite definir os dados que serão extraidos para as escritas no ficheiro de patch ou copiados.

 Pode existir vários modelos de dados idênticos num mesmo modelo de parametrização.
  • Seleção ([field] EXPSEL)

Regista-se aqui uma fórmula de seleção que se aplicará sobre a tabela principal definida no modelo de dados.

Esta fórmula pode nomeadamente incluir a variável [L]CODLEG, que é o código legislação registado no momento do lançamento da cópia.
  • Validação transacções ([field] TRANSAC)

Quando os dados de parametrização descrevem as transações de registo, a simples cópia de dados de parametrização não é suficiente : Falta igualmente que as transações correspondentes sejam revalidadas (i.é que do código - tratamento, ecrãs, janela segundo os casos -seja regerado. Isto se deencadeia em preenchendo aqui o código de objeto de parametrizaação correspondente.
