Perfis função

Os códigos perfis funções permitem definir as famílias de utilizadores, do ponto de vista de habilitações às funções (encontra na ficha utilizadores um código perfil função associado). Este código está definido nesta função.

[Refer to documentation [Implementation]]

Gestão do ecrã

  • Cabeçalho
  • [Tab] Acesso
  • [Tab] Estab.

Cabeçalho

A definição se faz sobre três separadores, em dando o código perfil a criar ou a modificar. A remarcar que em versão 140, reconduz-se na ficha perfil as informações que foram anteriormente na ficha cliente, em particular os dois separadores que seguem :

  • Código perfil ([field] PRFCOD)

Este código identifica de maneira única a ficha em curso
  • Descr. ([field] INTPRF)

Destinado a figurar as listas e os ecrãs nos quais o código da ficha pode ser registado ou selecionado. Este texto permite de dar uma descrição da ficha afetada.

[Tab] Acesso

Neste separador, define-se tudo então uma check box a assinalar, que permite dar acesso a todas as funções sem restrições por código perfil (em geral, um só código perfil, ligado ao administrador, terá tais privilégios).

Define-se em seguida os valores por defeito utilizados unicamente no momento de criação de um novo utilizador anexado ao perfil. Estes valores por defeito são a lista dos códigos de acesso autorizados, e uma check box Todos códigos de acesso permitindo de evitar o seu registo exaustivo.

Estes valores por defeito são justificados pelo facto que um utilizador não administrador pode ser autorizado a criar a um utilizador sem ter o direito de modificar certos direitos de acesso. Isto supõe que os valores por defeito satisfatórios possam ser propostos, anexados ao perfil função (limitar-se-á por outro lado para os perfis função utilizáveis pelo utilizador, pelo meio do parâmetro AUZFCT).

Se a check box Todas as funções autorizadas não está assinalada, poder-se-á definir de maneira detalhada a lista das funções autorizadas com se necessárias as autorizações complementares associadas. Isto é feito pela função Habilitação funcional, que está acessível seja pelo menu, seja diretamente por um botão.

Geral

  • Todas funções autorizadas ([field] ALLFCT)

Se esta check box está assinalada, o perfil dá acesso a todas as funções do software. Se ela não o é, passar-se-á pela função deHabitação funcional para definir quais funções são autorizadas.
  • Portal bloq. ([field] FLGPOR)

Defeito fixa utilizador

  • Todos cód. acesso ([field] ALLACS)

Se esta check box está assinalada, ao utilizador tem todos os direitos sobre todos os códigos de acesso (tanto em leitura, que em registo, que em execução). Isto permite evitar de ter a dar em extenso a lista dos códigos de acesso para os utilizadores priviligiados.

[Grid] Defeito fixa utilizador

  • Código acesso ([field] CODACC)

Os códigos de acesso são códigos permitem de proteger as informações ou as funções e em leitura, em registo, e em execução (quando o contexto o permite). Na ocorrência, estes códigos agem sobre pouco como uma fechadura colocada sobre um campo num ecrã, sobre a transacção, sobre uma lista, ou sobre uma ficha de base.

Define-se em seguida sobre o campo, para cada utilizador, se tem a possibilidade de aceder as informações protegidas assim (em qualquer lado se se dispõe da chave permitindo de abrir a fechadura).

O número de códigos de acesso e sua codificação está totalmente livre, pois que apenas a conta de afectação que está feita nas diferentes fichas que podem ser assim controlados.

Uma check box Todos os códigos de acesso permite dar acesso a todos os códigos presentes ou a virem. Se esta check box não foi assinalada, o registo pode-se fazer no quadro do separador.
  • Consulta ([field] CONSUL)

Se este campo está a Sim, o utilizador tem o direito de consultar as fichas (ou os campos) protegidos pelo código de acesso.

  • Modificação ([field] MODIF)

Se este campo está a Sim, o utilizador tem o direito de modificar as fichas (os os campos) protegidos pelo código de acesso.

  • Execução ([field] EXEC)

Se este campo está a Sim, o utilizador tem o direito de execução (se existe) definido no contexto das fichas protegidas pelo código de acesso. Encontrar-se-á numa documentação anexa, os casos de utilização dos direitos de execução para os códigos de acesso.
Atualização códigos acesso

Esta função permite atualizar o conjunto dos utilizadores que utilizam este perfil função para o código de acesso da linha (os débitos sobre este código de acesso serão atualizados) Por defeito de realizar esta operação, apenas os utilizadorew criados no futuro beneficiarão desta atualização.

[Tab] Estab.

Neste separador, define-se, por módulo funcional, um estabelecimento por defeito ao qual o utilizador tem este perfil está anexado. Este estabelecimento está utilizado por todo o lado onde o registo de um estabeleicmento é necessário. Apenas os módulos utilizados para o dossier aparecem no quadro em questão.

[Grid]

  • Nr. ([field] NUMLIG)
  • Módulo ([field] MODULE)
  • Estab. p/defeito ([field] FCYDEF)

Este quadro permite definir os estabelecimentos por defeito por módulo.

Esta parametrização está utilizada cada vez que um estabelecimento a registar.

[Tab] Listas

Este separador contém os campos seguintes :

Um indicador Impressões diferidas que pemite interdir a possibilidade de lançar as listas em batch para os utilizadores anexados a este perfil.

Um filtro de acesso às listas, definidas sob a forma de um modelo (em utilizando os meta caracteres ? ou *) Isto permite interdir ao utilizador o acesso a certas listas.

Um filtro de acesso por grupo de listas que permitem interdir o acesso a certas listas. Os grupos de lista são definidos no menu local 97, os 15 primeiros sendo definido pelo standard.

De referir que as informações de separador Listas não são modificáveis pelo perfil menu do administrador, definido pelo parâmetro ADMUSR (com efeito, este utilizador não tem restrição de acesso às listas)

Autorizações

  • Impressões diferidas ([field] DIFIMP)

Esta zona autoriza a impressão diferida
  • Filtro listagens ([field] FILTRE)

Esta zona permite limitar o acesso às listas a códigos que correspondem ao filtro indicado.

Por exemplo, com um filtro tal que AB??*, o utilizador não terá os direitos completos que sobre as listas que começam por AB e têm pelo menos 4 caracteres.

[Grid] Grupos

  • Nr. ([field] NUMLIG)
  • Grupo de listagens ([field] GRPETX)
  • Acesso ([field] AUTETA)

Este campo define um primeiro nível de filtro de acesso às listas. Apenas as listas pertencentes aos grupos autorizados para o perfil poderão ser imprimidos pelo utilizador (em sabendo que o código de acesso da lista permite restringir de vantagem ainda esta lista).

Privilégios

Este botão permite passar diretamente em gestão das habilitações funcionais.
 

[Block number] 1

  • [field] OBJET
  • [field] CLES

[Block number] 2

  • A partir do dossier ([field] DOSORG)

﻿Este campo permite de definir o dossier a partir do qual a ficha vai ser copiada. As sintaxes possíveis são descritas no anexo dedicado
  • Todos dossiers ([field] TOUDOS)

Esta opção permite de copiar a ficha versus todos os dossiers definidos no dicionário (tabela ADOSSIER da solução corrente).

﻿Este campo permite de definir o dossier no qual a ficha vai ser copiada. As sintaxes possíveis são descritas no anexo dedicado

Este botão permite de recopiar a definição da transação depois ou versus um outro dossier.

Opções / Atualização códigos acesso
