X3 Warehousing

Mvts. de montagem

Este ecrã permite de visualizar todos os movimentos de assemblagens.

Pré-requisitos

[Refer to documentation [Implementation]]

Gestão do ecrã

  • Cabeçalho
  • [Tab] Geral
  • [Tab] Gestão

Selecciona-se um nº de movimento e o ssitema afixa as suas características:

Cabeçalho

  • Dados OA
  • Dados Linhas de OA
  • Dados de movimento

Separadores

  • Geral para o detalhe do movimento
  • Gestão para as informações de rastreabilidade do movimento

Observação : um movimento de assemblagem não está obrigatóriamente associado a uma OA/linha de OA. Com efeito para uma vaga de assemblagem sem gestão das quebras (opção "Transferência quebrada a Não), transfere-se mais de kits/ componentes que necessário (ou ao mínimo igual). Assim só a quantidade necessário está associada à OA.

Cabeçalho

OM

  • Nr./Ref. ([field] BOMNUM)
  • [field] REF
  • Status ([field] BMHSTA)

Linha

  • Nr./Ref. ([field] BOMLIN)
  • [field] LINREF
  • Status ([field] BMLSTA)

Movimento

Sequência numérica atribuida automaticamente no movimento pelo sistema. Este número é uma informação própria ao estabelecimento, todos os depositantes simu ltaneamente. Esta informação constitui uma chave única de acesso a um movimento.

  • Número ([field] MVTNUM)
  • Artigo ([field] ITM)

Código artigo definido na tabela dos artigos Warehousing
  • Descr. ([field] ITMDESAXX)
  • Status ([field] MVTSTA)

Status do movimento.

Quantidades

Conteúdo do ’ objecto de stock.

Quantidade do ’ objecto de stock.

O conteúdo homogéneo de um objecto de stock está definido como o primeiro conteúdo de nível diferente de 1 (palete) cuja quantidade UA do stock seja um múl tiplo.

Número de componentes necessários para constituir um composto.

  • Nr UC conjunto ([field] CUQKIT)
  • [field] TYP2
  • Plano paletização ([field] PLAN)

Datas

Data correspondente na entrada de’objecto de stock no estabelecimento.

Data de fabrico correspondente no fim do ciclo de produção dos artigos

Data de Limite de consumo (DLC): A DLC pode servir a limitar a distribuição de’um produto nos tempos:

- por exemplo, no quadro de’uma operação comercial (promoção),

- ou bem do facto da sua perempção.

  • Entrada ([field] INPDAT)
  • Fabrico ([field] MND)
  • Retenção ([field] RTD)

Data de retenção. Esta data corresponde ao momento antes do qual.

O objecto de stock está indisponivel para um movimento de saída.

(quarentena por exemplo).
  • DLV ([field] SLD)

Data Limite de Venda (DLV): A DLV é geralmente utilizada

como Data FIFO para os produtos alimentares.
  • DLUO ([field] USD)

Data de Limite de conservação (DLC): Esta data é também chamada Data

Limite de Utilização Óptima. Ela figura geralmente sobre

o embalamento dos produtos alimentares ou farmacêuticos.
  • DLC ([field] SND)
  • FIFO ([field] FIFDAT)

A Data FIFO pode determinar a ordem de destocagem dos produtos (en

função da parametrização) no momento do lançamento de uma preparação de

encomenda. Ela permite igualmente de verificar a possibilidade de efectuar.

certas transacções, como a compactação.

Características

A’Origem permite uma classificação dos artigos segundo um critério escolhido por ’utilizador.

Ela está classicamente utilizado para definir ou indicar a proveniência (geográfica ou outra) dos artigos gerados pelo sistema.

Ela permite de gerar as diferenças qualitativas ao nível de um objecto de stock : perempção, reacondicionamento, ...

  • Cod.mov. ([field] MVT)

O código movimento permite de diferenciar os movimentos de mesmo tipo.

É requerido no momento da criação de uma entrada esperada, de uma entrada (Entrada fornecedor, Entrada fábrica, Devolução cliente,...), de uma O.E., de um ajustamento, de uma transferência, de uma entrada / saída de consumíveis.... O código do movimento está verificado na tabela dos códigos de movimentos.
  • Origem ([field] ITMORI)
  • Natureza de stock ([field] SKN)
