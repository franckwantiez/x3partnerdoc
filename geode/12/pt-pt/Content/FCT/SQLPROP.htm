Propriedades SQL Server

Esta função permite visualizar as principais propriedades na base de dados quando a base de dados utilizada é SQL Server™.

Os dados aqui apresentados não são modificáveis.
A sua modificação deve se fazer pelos utilitários de administração próprias à base de dados utilizados.

Advertência

Esta função permite, por pedidos sobre tabelas e vistas do sistema, de dar informações técnicas sobre a base de dados. A interpretação dos resultados dados por esta função é da competência de um administrador de bases de dados, e pode depender da versão de base de dados utilizada.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Servidor
  • [Tab] Base

Esta função permite de visualizar as propriedades da base de dados.
Estes elementos são apresentados sobre dois separadores: O primeiro dá as características gerais do servidor, o segundo dá as características da base de dados assim que a lista dos parâmetros e opções posicionados pela base de dados.

Cabeçalho

Permite conhecer as características da base de dados utilizada.

  • Base de dados ([field] BDDNAM)

Nome da base de dados
  • Versão ([field] VER)

Número de versão da base de dados
  • [field] VERDES

Descrição da versão da base de dados

[Tab] Servidor

O primeiro separador apresenta as propriedades gerais do servidor albergando a base de dados.

Este primeiro bloco dá as informações sobre a instância de SQL Server.

  • Máquina ([field] SRVMAC)

Nome de servidor sobre o qual foi arrancada a base de dados.
  • Instância ([field] SRVINS)

Nome de instância de Microsoft® SQL Server™.
  • Classificação p/defeito ([field] SRVTRI)

Tipo de classificação utilizada pela instância de SQL Server.
  • Versão ([field] VER)

Descrição da versão da base de dados

[Tab] Base

Sobre o segundo separador, vê-se as propriedades da base de dados

Este ecrã é constituido de três blocos.

Base de dados

Este bloco contém as principais características da base de dados.

Última salvaguarda

Este bloco permite de visualizar as datas das últimas salvaguarda efectuadas sobre a base de dados.

Opções

Este bloco lista as opções de base de dados e seus valores.

Existem cinco categorias de opções de base de dados :

    • opções automáticas ;
    • opções de cursor ;
    • opções de recuperação ;
    • opções SQL ;
    • opções de estado.

Base de dados

  • Proprietário ([field] OWN)

Utilizador proprietário da base de dados.

  • Data criação ([field] CREDAT)

Data e hora de criação da base de dados.
  • [field] CREHOU
  • Num. utilizadores ([field] NBRUSR)

Número de utilizadores definidos na base de dados.
  • Nr. de conexões activas ([field] NBRPRO)

Número de conexões activas sobre a base de dados.
  • Dimensão da base ([field] TAILBASE)

Tamanho da base de dados correntes, todos os dossiers simultaneos. Este tamanho inclui todos os ficheiros de dados e os ficheiros diários.
  • [field] MB1
  • Espaço n/atribuído ([field] ESPACE)

Espaço da base de dados que não foram reservados por objectos de base de dados.
  • [field] MB2
  • Classific. ([field] BDDTRI)

Tipo de classificação utilizada pela base de dados.
  • Modelo de recuperação ([field] RPCTPL)

Especifica o tipo de modelo de recuperação da base de dados

Os modelos de recuperação seguintes estão disponíveis :

    • Recuperação simples
      A recuperação simples permite a recuperação da base de dados na ajuda da salvaguarda a mais recente.
    • Recuperação completa
      A recuperação completa permite a recuperação da base de dados a partir do momento do prejuízo.
    • Recuperação diária no bloco
      A recuperação diária em bloco permite das operações de diário em bloco.
  • Nível de compatibilidade ([field] LEV)

Indique o nível de compatibilidade da base de dados.

[Grid] Opções

  • Opção ([field] OPT)

Opções de base de dados

Eis uma lista das opções que se podem encontrar :

Opção :

Descrição

auto create statistics

Se verdadeiro (verdadeiro), as estatísticas em falta, requeridas por um pedido para otimização, são criados automaticamente durante a otimização.

auto update statistics

Se verdadeiro (verdadeiro), as estatísticas não atualizadas, requeridas por um pedido para otimização, são criadas automaticamente durante a otimização.

autoclose

Se verdadeiro (verdadeiro), a base de dados está parada propriamente e seus recursos são liberados desde que o último utilizador esta desconectado.

autoshrink

Se verdadeiro (verdadeiro), os ficheiros de base de dados são candidatos a uma compactação automática periódica.

ANSI null default

Se verdadeiro (verdadeiro), CREATE TABLE segue as regras SQL-92 afim de determinar se uma coluna autoriza os valores NULL.

valores ANSI NULL

Se verdadeiro (verdadeiro), todas as comparações com valor NULL são avaliadas a desconhecido (UNKNOWN). Se falso (falso), as comparações de valores não UNICODE a um valor NULL são avaliados a verdadeiro (TRUE) se os dois valores são NULL.

ANSI warnings

Se verdadeiro (verdadeiro), erros ou advertências são emitidos se as condições tais que "divisão por zero" são verificadas.

arithabort

Se verdadeiro (verdadeiro), uma ultrapassagem de capacidade ou uma divisão por zero provocam a paragem do tratamento do pedido ou do lote de instrução. Se o erro se produz numa transação, esta última está anulada (roll back). Se falso (falso), uma mensagem de advertência se afixa, mas o tratamento do pedido, do lote de instruções ou da transação se prossegue, como se não há erro.

concat null yields null

Se verdadeiro (verdadeiro), o resultado está NULL se um dos dois operadores de concatenação está NULL.

c

Se verdadeiro (verdadeiro), todo o cursor aberto no momento onde uma transacção está comprometida ou restaurada e fechada. Se falso (falso), os cursores restam abertos quando uma transacção está comprometida, e a restauração de uma transação fecha todo o curso à excepção de aqueles que são definidos com a palavra-chave INSENSITIVE ou STATIC.

dbo use only

Se verdadeiro (verdadeiro), apenas o proprietário da base de dados pode utilizar este.

default to local cursor

Se verdadeiro (verdadeiro), as declarações de cursor podem ter o valor por defeito LOCAL.

publicação de fusão

Se verdadeiro (verdadeiro), a base de dados pode ser publicada para uma replicação de fusão.

numeric roundabort

Se verdadeiro (verdadeiro), um erro está gerado quando perto de precisão surgem numa expressão. Se falso (falso), as perdas de precisão não gerem mnesagens de erro e o resultado é arredondado em função da precisão da coluna ou da variável contendo o resultado.

offline

Se verdadeiro (verdadeiro), a base de dados está fora linha.

published

Se verdadeiro (verdadeiro), a base de dados pode ser publicada para uma replicação de fusão.

identicador entre parâmetros

Se verdadeiro (verdadeiro), você pode enquadrar aspas duplas para os identificadores delimitados.

readonly

Se verdadeiro (verdadeiro), os utilizadores são unicamente autorizados a ler os dados da base de dados, e não os modificar. A base de dados não pode estar em curso de utilização quando você especificou um novo valor para a opção read only A base de dados master faz excepções a esta regra, e apenas o administrador sistema pode utilizar durante a parametrização sa opção read only.

recursive triggers

Se verdadeiro (verdadeiro) permite a activação recursiva de desencadeadores. Se falso (falso), impede unicamente a recorrência directa. Para desactivar a recorrência indirecta, afecta o valor 0 na opção do servidor nested triggers com a ajuda de sp_configure

select into /bulkcopy

Se verdadeiro (verdadeiro), a instrução SELECT INTO e as cópias em bloco rapidos são autorizados.

single user

Se verdadeiro (verdadeiro), a base de dados não pode ser utilizada por um utilizador à vez.

subscribed

Se o valor é verdadeiro, a base de dados pode ser abandonadas a uma publicação.

torn page detection

Se verdadeiro (verdadeiro), as páginas podem ser detetadas.

trunc. log on chkpt

Se verdadeiro (verdadeiro), um ponto de controlo vazio a parte inativa quando a base de dados está em modo troncatura de diário Trata-se da única opção autorizada para base de dados master


  • Valor ([field] OPTVAL)

Última salvaguarda

  • Base de dados - completa ([field] SAVBDDDAT)

Afixa a data e hora da última salvaguarda da base de dados.

  • [field] SAVBDDHOU
  • Base de dados - diferencial ([field] SAVDBDDDAT)

Afixa a data e hora da última salvaguarda diferencial da base de dados.

  • [field] SAVDBDDHOU
  • Diário das transacções ([field] SAVLOGDAT)

Afixa a data e hora da última salvaguarda do diário das transações.

  • [field] SAVLOGHOU
