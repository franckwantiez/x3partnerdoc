Activação de ligações

Em todo o software, é possível de criar as ligações entras as fichas dos objetos de gestão. Estas ligações são definidas por uma ficha de partida, uma ficha de chegada, um grupo de ligação ligada aos utilizadores, e um significado (definido por um código numa tabela). Cada ligação pode ser registada manualmente ou criada automaticamente segundo uma parametrização ligada ao objeto. Enfim, cada ligação está datada, e dispõe de um indicador ativo (Sim/Não) : o campo ENAFLG está utilizado a este fim.

As ligações anexadas a uma ficha são apresentadas numa porta de janela da lista esquerda : o explorador de ligações, cuja presença é opcional e depende da parametrização do objeto. Apenas as ligações ativas são apresentadas no explorador de ligação. Por defeito, quando uma ligação foi criada, ela está ativa, e a única função susceptível de fazer passar as ligações de lista Activa na lista Inactiva e vice-versa é a função Activação das ligações apresentadas aqui.

Gestão do ecrã

[Tab] Ecrã de registo

Regista-se neste ecrã os critérios de seleção para saber quais são as ligações a tratar, depois se define o tratamento a realizar (activação ou desactivação).

Estes parâmetros registados, o lançamento da função provoca a atualização das ligações. Nenhum rasto foi criado.

Selecções

  • Todos os utilizad. ([field] TOUUSR)

Quando esta check box está assinalada, os dados associados a todos os utilizadores são afetados. Senão, apenas o utilizador cujo código está registado está afetado (por defeito o utilizador corrente)
  • Grupo ([field] USRCOD)

Quando as únicas ligações criadas por um utilizador são apenas afectadas, o campo o define.

[Block number] 3

  • Todos obj's. ([field] TOUOBJ)

Se esta check box está assinalada, as ligações associadas a todos os tipos de objeto são afetados pelo tratamento.

  • Assunto ([field] OBJ)

Quando as únicas ligações afectam um objecto são afectadas, este campo o define.

[Block number] 4

  • Todas ligações ([field] TOULIA)

Se esta check box assinalada, todos os tipos de ligações são afetados pelo tratamento.

  • Ligação ([field] LIA)

Quando um único tipo de relação está afectado, este campo o define.

[Block number] 5

  • Ligações automáticas ([field] AUTFLG)

Se esta check box está assinalada, as relações criadas automaticamente (a partir de regras definidas em parametrização dos objetos) são afectados.

  • Ligações manuais ([field] MANFLG)

Define se as ligações manuais são afectadas pelo tratamento.

As ligações manuais são aqueles registadas pelo utilizador sobre um objecto via a função Ligações do menu Ficheiro.

[Block number] 6

  • Limites de datas ([field] DATDEB)

Se esta data está preenchida, apenas os dados cuja data de última modificação ou da data de criação é superior ou igual são tomados em conta no tratamento.
  • [field] DATFIN

Se esta data está preenchida, apenas os dados cuja data de última modificação ou da data de criação é inferior ou igual são tomadas em conta no tratamento.

Função

  • Função ([field] FNC)

A escolha activação / desactivação define o tratamento realizado. O que seja, as relações não são suprimidas se elas estão desactivadas.
