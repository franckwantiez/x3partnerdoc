Objectos

A gestão dos objectos está na base da maior parte das funções do software. Um objecto corresponde à gestão completa das fichas de uma tabela ou de um grupo de tabelas (criação, consultas, modificação, anulação se for caso disso...). Por exemplo, a gestão dos clientes, a gestão das facturas, o registo dos registos...são implementados no software sob a forma de objectos.

Um objecto se apresenta sob uma das duas formas principais seguintes :

de uma janela composta de um cabeçalho e de um ou vários separadores situados sobre a parte direita e de listas de selecção situadas à esquerda "browser". Um registo está gerado à vez.

de uma janela composta de um ecrã único contendo a lista dos registos da tabela.

Este objecto permite gerir um registo numa tabela (uma ficha terceiros, por exemplo) ou nas várias tabelas (o cabeçalho de encomenda e das linhas correspondentes).

Em mais da lista esquerda principal do objecto, é possível de definir de mais Isto se faz. pela declaração de um objecto de tipo browser para cada lista esquerda suplemeentar.

Cabeçalho

O cabeçalho permite identificar o objecto

  • Cód. objecto ([field] ABREV)

O código de um objecto está constituido de 1 a 3 caracteres alfanuméricos. As palavras reservadas de Sage estão interditas.
  • Descr. ([field] ZLIBEL)

Destinado a figurar as listas e os ecrãs nos quais o código da ficha pode ser registado ou selecionado. Este texto permite de dar uma descrição da ficha afetada.
  • Desc. curta ([field] ZLIBSHO)
  • Tabela ligada ([field] NOMFIC)

Código da tabela principal gerada pelo objeto.

  • Modelo de dados ([field] MODELE)

[Tab] Geral

Geral

  • Módulo ([field] MODULE)

Indica o módulo aplicativo ao qual está afixado a ficha (e o código reafixado se for caso disso). As fichas utilizadas por vários módulos são classificados num dos módulos Tronco comum ou supervisor.

Um desenvolvimento associado ao módulo (e o código correspondente) não pode ser utilizado que se o módulo está activo no dossier corrente.
  • Desig. parâmetro ([field] ZLIBPAR)

Neste caso onde o objeto está identificado por uma chave a 2 componentes (por exemplo uma tabela diversa onde o primeiro componente representa o número da tabela), o primeiro componente de chave torna-se um parâmetro para a gestão de objeto. Você deve então indicar o literal deste parâmetro (Número da tabela, no exemplo).

 
  • Cód. activ. ([field] CODACT)

Se está definido, um código atividade anexado a um objeto permite de tornar este aqui operacional.
  • Zona estab. ([field] ZSITE)

Define-se aqui o campo da tabela principal onde se encontra o código de acesso aoqual está reafixado o registo.
O facto de definir este estabelecimento permite gerir automaticamente as restrições de acesso aos dados de um estabeleciemento.
  • Zona cód. acesso ([field] ZACC)

Define-se aqui o campo da tabela principal onde se encontra o código de acesso ao qual está reafixado o registo.

O facto de definir este campo permite de gerar automaticamente as restrições de acesso aos códigos de acesso.

[Block number] 2

  • Tipo de gestão ([field] TYPGES)

Esta característica define a maneira cujo objeto gera os registos da tabela. As escolhas possíveis são Simples, Em quadro, Combinado. Um quarto valor igual a Browser vai permitir criar de maneira independente, browsers que integrará em seguida numa janela.

  Simples. é o tipo mais frequente. Corresponde ao caso onde se gera uma ficha um registo da tabela principal, uma lista de registos estando afixada na lista de esquerda. O ecrã de gestão dos objetos obedece ele mesmo a esta lógica (é um objeto ele mesmo, seu código é AOB), mas encontrará numerosos outros exemplos : clientes, faturas, escritas, encomendas...são todos objetos simples.

  Quadro. Isso corresponde ao caso onde a tabela, suficientemente pequena, pode ser diretamente carregada nas linhas de um único quadro afixado ao ecrã. Neste caso, a lista de esquerda não existe mais (ela não é útil). A gestão da codificação das rubricas é ela mesmo feita assim.

  Combinado. Isso corresponde ao caso onde se carrega num quadro um conjunto de registos de uma tabela, registos cuja chave é feita de N partes (N>1), cujas N-1 partes são fixas e cuja primeira parte varia. Atenção, não é o caso para as faturas (que têm uma chave de documento / número de linha), porque neste caso, faz uma gestão de objeto simples sobre o cabeçalho e gera o carregamento das linhas no código assoicado ao objeto. Um exemplo de tal objeto é a gestão das fichas técnicas : para um código de controlo, tem-se N linhas presentes num quadro.

  Browser. Este tipo permite definir as listas de esquerda que se anexará em seguida a uma janela. As informações a registar sobre este tipo de objeto são reduzidas às informações para a constituição de uma lista de esquerda.

Ligações

  • Menu ([field] MENU)

Esta zona define o código do menu no qual a função de gestão do objeto será apresentada pelo utilizador ADMIN.
  • Posição no menu ([field] RANG)

Indique a fila onde aparecerá a função no menu acima.
Num menu as funções são triadas na ordem crescente das filas.
  • Imprimir ([field] RPT1)

Este código de impressão define a ou as listas lançadas por apoio sobre a escolha de menu Ficheiro / Imprimir. Deve-se referir que os limites da lista podem ser preenchidas a partir da ficha corrente : em efeito, a escolha Ficheiro / Imprimir não está acessível que se uma ficha corrente está afixada ao ecrã.

Por defeito, no momento de utilização destes botões, o software tentará de editar as listas cujos nomes são dados aqui. Mas existe uma tabela de indirecção que permite redefinir, a partir do código interno dado aqui, uma ou várias listas (por exemplo a impressão dos bons de preparação e guia de remessa no objecto das entregas).
  • Lista ([field] RPT2)

Este código de impressão define a ou as listas lançadas por apoio sobre a escolha de menu Ficheiro / Lista. A escolha Ficheiro / Lista está sempre acessível, e está suposto permitir de imprimir as listas gerais ligadas às tabelas em linha.

Por defeito, no momento de utilização destes botões, o software tentará de editar as listas cujos nomes são dados aqui. Mas existe uma tabela de indirecção que permite redefinir, a partir do código interno dado aqui, uma ou várias listas (por exemplo a impressão dos bons de preparação e guia de remessa no objecto das entregas).
  • Pedido ([field] RQT)

Permite propor os pedidos no menu Ficheiro/Lista da função quando nenhuma lista foi prevista. Se este campo está preenchido, o supervisor propõe todos os pedidos e pedidos sql tendo o campo GROUPE igual a este valor registado. Se não preenchido, o valor tomado em conta é aquele do código objecto. Se nenhum pedido foi parametrizado, o supervisor propõe a criação automática de um pedido, estruturado pelas colunas da lista de esquerda.

Opções

  • Estatísticas ([field] STA)

Esta check box a assinalar permite de activar o botão "Estatísticas" no objecto. Este botão permite registar as interrogações estatísticas cujo o ou os primeiros critérios são definidos por dados ligados à tabela principal ou a das tabelas ligadas ao objecto.
  • Supressão diferida ([field] DELDEF)

O supervisor autorizará ou não as supressões diferidas sobre a tabela principal assim que as tabelas secundárias geridas pelo objecto. Esta possibilidade está oferecida para os utilizadores parametrizados em confirmação de supressão. Quando este utilizador suprime uma ficha de objecto, uma janela de confirmação se afixa. E, sobre esta janela, utilizador pode assinalar a check box "supressão diferida". A ficha será então sempre presente até o que o utilitário de supressão diferido. seja lançado.
  • Importação ([field] IMPORT)

Esta zona permite indicar que o objeto é importável pela função import-export.
  • bloqueio modificação ([field] MLOCK)

Permite desencadear o "lock" lógico quando se entra em modificação de um registo, então que em standard, é posicionado desde a visualização do registo. Para entrar em modificação, falta neste caso clicar sobre o ícone representando um livro azul situado em baixo de ecrã.

 
  • Parametro afixado ([field] DSYPAR)

Sobre um objeto gerando dois elementos de chave para além do estabelecimento e do depositando, indica-se que estes elemetos aparecem em caso de erro sobre um registo.

 
  • Gestão estab./depositante ([field] FCYDEP)

Indica-se se existe uma noção de estabelecimento ou depositante sobre esta função. A noção de depositante será determinado a presença do campo DEP na chave primária da tabela ligada ao objeto

Os automatismos do supervisor, para uma função, sobre o sujeiro do estabelecimento ou depositante são os seguintes :

  • entrada na função não autorizada, se o estabelecimento e armazem corrente não serão posicionadas
  • filtro automático da lista esquerda e lista os últimos lidos em função do estabelecimento e armazém corrente.
  • alimentação do estabelecimento de criação de registo
  • alteração de ambiente possivel pelo menu : Afixação/ Alteração de ambiente

[Tab] Selecção

Encontramos todas as informações permitindo de construir a lista esquerda principal de objecto assim que a janela de selecção. Assim, temos a possibilidade de se definir as opções de selecção.

As opções de selecção

Este quadro permite definir os filtros de selecção levando sobre os registos da tabela gerada. Estes filtros, exprimidos sob forma de condições lógicas levando sobre os campos da tabela e de eventuais constantes, são identificados por uma letra (maiúscula pelo standard, minúscula para o especifico). São em seguida posicionados ou não sobre os campos tendo um tipo de dado ligado a este objecto.

Tomemos o exemplo seguinte :

Nos registos dos documentos de venda, não se pretende autorizar que os clientes activos. Para este facto, um filtro de selecção está definido num objecto Cliente, com a condição BPCSTA=2, e intitulado "Cliente activo" No ecrã das encomendas SOH0, temos um campo cliente BPCORD de tipo de dados BPC (tipo está ligado ao objecto BPC). Sobre o menu contextual do tipo de dados BPC, o menu "Opções" permite posicionar ou não cada um dos filtros declarados no objecto dos clientes.

Definição dos ecrãs de selecção
(lista esquerda, janela de selecção e de selecção avançada)

A lista das colunas devem aparecer na lista de esquerda está registada no quadro do baixo do 2º separador Estas colunas podem ser dos campos da tabela principal ou de uma tabela secundária identificada sovre o 3º separador se expressão de ligação foi registada. Elas podem ser também uma expressão ; neste caso, indicar-se-á o tipo de dados para guardar o resultado da expressão.

[Grid] Opções de selecção

  • Desig. opção ([field] ZOPTLIB)

Este quadro permite definir as opções de gestão do objeto. Uma opção permite de caracterizar os sub-conjuntos de fichas identificadas por uma condição (clientes ativos, contas letras,...).

Uma opção está definida por:

  • Uma letra chave
  • Um intitulado
  • Uma condição lógica
  • Uma mensagem de erro a afixar se o objeto não responde à opção.
  • Cód. ([field] OPTCOD)

Este quadro permite definir as opções de gestão do objeto. Uma opção permite de caracterizar os sub-conjuntos de fichas identificadas por uma condição (clientes ativos, contas letras,...).

Uma opção está definida por:

  • Uma letra chave
  • Um intitulado
  • Uma condição lógica
  • Uma mensagem de erro a afixar se o objeto não responde à opção.
  • Condição opção ([field] OPTCND)
  • Mensagem erro ([field] ZOPTERR)

Ecran de selecção

  • Index ([field] SELCLE)

Esta zona permite definir um índice de percurso do ficheiro principal para a afixação na lista de esquerda e das janelas de selecção. Se a zona está vazia, é o primeiro índice da tabela que foi utilizada.
  • Sentido ([field] SELORD)

Esta zona permite definir um índice de percurso da tabela principal para a afixação na lista de esquerda e das janelas de selecção do objecto.
  • Opções de selecção ([field] SELOPT)

Esta zona permite indicar uma ou várias opções para as fichas presentes na janela de seleção.
  • Lista hierarquizada ([field] SELTREE)

Quando a chave de percurso escolhido para a lista esquerda está composta de várias partes, a lista se faz então em apresentando então unicamente a primeira parte da chave. Em fazendo duplo-clique sobre um dos valores de chaves, explora-se os sub-valores de chave, e sobre o número de níveis de sub-chaves existentes. Quando todos os níveis de chaves são explorados, pode-se seleccionar um registo da tabela.

Um exemplo deste funcionamento pode ser constatado sobre o objecto de gestão dos estabelecimentos : cada estabelecimento anexado a uma sociedade, visualização então a lista das sociedades, e tem-se em seguida a possibilidade de desdobrar em clicando acima : vê-se então a lista dos estabelecimentos da sociedade, e pode-se seleccionar um.

se a check box não assinalado, trata-se de uma lista simples.
  • Nr. de caracteres ([field] SELCAR)

Todo o campo de um ecrã cujo tipo está ligado a um objecto dispõe de uma janela de selecção acessível por clique direito ou por .

Este valor permite definir o número de caracteres utilizados para indicar onde se deve começar a selecção, em função deste que foi registado na zona antes da selecção. Se este valor vale N, a selecção inicia-se a partir da primeira chave cujos N primeiros caracteres são superiores ou iguais à cadeia escolhida. Assim :

  • Se se parametriza o valor 2, e se se pressiona a cadeia DEF antes de abrir a selecção, a janela de selecção vai começar a pesquisa a partir do primeiro valor da chave superior ou igual a DE (se existem chaves começando por DEA, estes que pertencem aparecerão em primeiro na janela). Se se pressionou H, a selecção terá começado à primeira chave superior ou igual a H.
  • Se se dá a esta zona o valor 0, e que se digita a cadeia UVW antes de abrir a selecção, a selecção começa a A e não foi limitada.

[Table number] 2

  • Tabela ([field] SELFIC)

Regista-se a tabela na qual se encontra o campo a afixar. Esta tabela é a tabela principal, seja uma das tabelas definidas como ligada à tabela principal na definição do objeto.

Esta zona pode estar vazia se afixa uma expressão calculada.
  • Zona ([field] SELZON)

Registam-se aqui os campos presentes na lista esquerda. São emitidos na tabela definida precedentemente.
  • Expressão ([field] SELEXP)

Quando uma coluna da lista esquerda foi avaliada, define-se aqui a 'expressão calculada cujo resultado preenche a coluna. Um tipo de dados está então necessário para permitir a formatação do campo.

  • Descr. ([field] ZSELINT)

Este intitulado é utilizado como título da coluna, assim bem na lista esquerda que nas janelas de seleção do objeto.

Por defeito, o título do campo está utilizado. Falta notar que este intitulado está traduzível. Assim, se um intitulado não conhecido, a janela de escolha permitindo criar um novo texto no dicionário se abre automaticamente.
  • Tipo ([field] SELTYP)

Este tipo de dados está utilizado para formatar a expressão precedente na lista esquerda.
  • Opções ([field] SELSAI)

Esta informação permite de formatar os dados pela esolha de opção. As opções dependem do tipo de dados (utilizar a janela de pesquisa).

  • Comprim. ([field] SELLNG)

Possibilidade de indocar um comprimento a uma coluna alfanumérica construida por uma expressão Se não preenchido, o comprimento de 30 caracteres está tomada por defeito.
  • Nível ([field] SELPCK)

Indica o nível hierarquico. Isso permite de dissociar os níveis hierárquicos da lista de esquerda, da estrutura da chave. Atenção, esta paramerrização não é suficiente ela mesmo, necessita do desenvolvimento.

  • Tabelas a abrir ([field] TABFIC)

Tabelas suplementares na tabela de base gerada pelo objeto. Estas tabelas estão abertas depois fechadas automaticamente na execução de objeto, sob abreviação indicada ou senão sob abreviação do dicionário. Os campos destas tabelas estão acessíveis no objeto.

Mais, os campos destas tabelas são utilizáveis para definir uma coluna de lista esquerda e dos ecrãs de seleção a partir do momento onde a expressão de ligação está registado. É possível de definir as ligações hierarquizadas, na condição de preencher estas tabelas, no quadro, na ordem da hierarquia.

A chave de ligação deve estar sem homónimo Se ela não está preenchida, o supervisor toma a chave primária.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

Falta uma tabela ligada para este tipo de objecto

A tabela ligada é obrigatória para os objectos excepto pelos browsers.

Campos de selecção incompatíveis com o índice

Sobre uma lista hierarquizada (simples ou picking), o numero de elementos declarados para a lista de esquerda superior ou igual ao número de elementos de índice utilizado.

Zona inexistente

Se a zona "estabelecimento" está preenchida, o valor registado é um nome de campo que deve existir na tabela principal.

" caracter interdito

Por razões técnicas, o caracter "«" é interdito em todos os campos de texto : O intitulado de objecto, o intitulado parãmetro, o intitulado das opções, expressão de condição, a mensagem de erro, a expressão de ligação.

Expressão incorrecta mensagem

A expressão registada não respeita a sintaxe X3. Uma mensagem explicativa detalha o erro.

X : Opção incorrecta

Tenta-se de registar uma opção de selecção, que não existe no quadro das opções de selecção.

Tabela não definida

A tabela deve estar decclarada na lista das tabelas a abrir do separador "Ambiente".

Ligação incorrecta

A tabela deve estar decclarada na lista das tabelas a abrir do separador "Ambiente", e com uma expressão de ligação correcta.

Zona inexistente

O campo deve existir na tabela registada no campo precedente.

Zona correspondendo a um texto traduzível, utilizar uma ligação

O campo registado é de tipo "AXX". Isso significa que não existe na vase mas permite justo a ligação versus a tabela dos textos ATEXTRA.

Tipo de dados incorrecto

O campo registado é de um tipo de dados que não autoriza a estar na lista de esquerda : Tipo "AXX", Clob ou Blob.

Objecto declarado no modelo importação

tenta-se levar a flag "importação" então que os modelos de importação existem já.

Abreviação já existente ...

A abreviação de uma tabela ou de uma vista única sobre um conjunto das tabelas e das vistas definidas sobre o objecto.
