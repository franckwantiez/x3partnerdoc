Ações

Esta função permite definir as ações utilizáveis nos ecrãs, nos tipos de dados para as ações sobre campos, nas janelas para os botões de baixo de ecrã ou os menus, a nas funções.

Uma ação desencadeia geralmente um tratamento supervisor chamado "modelo" sobre o qual se pode implantar um tratamento funcional

Um certo número de açõe podem ser encadeados (até 5 no máximo)

Cabeçalho

O cabeçalho permite identificar a ação

Indica o código acção a definir.

  • Cód. acção ([field] ACTION)
  • Descr. ([field] ZINTITA)

Destinado a figurar as listas e os ecrãs nos quais o código da ficha pode ser registado ou selecionado. Este texto permite de dar uma descrição da ficha afetada.

[Tab] Geral

Encontram-se as informações gerais relativas à ação.

Características

A validação do dossier Adonix não actualizará as acções cujo código de actividade começa por X,Y ou Z.

  • Cód. activ. ([field] CODACT)
  • Módulo ([field] MODULE)

Indicar o módulo para o qual é utilizado esta acção. Se esta acção é utilizada por vários módulos, indicar "supervisor".
  • Modelo ([field] TYP)

Este campo é capital porque se deduz o tratamento a lançar, na execução desta ação. Ver o detalhe desenvolvimento. A este nível, os valores possíveis são os seguintes :

  Gestão de objecto
  Consulta
  Tratamento standard ( pode ser lançado em batch )
  Registo janela
  Seleção na lista ( janela de seleção )
  Seleção na tabela ( janela de seleção)
  Afixação diversa ( chamada de um sub-programa funcional por Gosub )

Fora modelo (chamada de um sub-programa funcional por Call)

O "Fora modelo" permite lançar diretamente um tratamento funcional sem passar por um modelo: este tipo de acção não poderá ser chamado depois de uma função.

[Block number] 2

  • Descr. botão ([field] ZINTITC)

O intitulado afixa-se no menu contextual do campo sobre o qual está colocada esta acção.
  • Cód. consulta ([field] PARAM1)

Identificador da consulta a preencher sobre uma acção de módulo "Consulta".
  • Tipo ([field] TYPUTI)

Parâmetros

  • Janela pricipal ([field] PARAM2)

Este campo serve a identificar a janela principal de registo para os modelos "registo janela" ou "tratamento standard"

[Block number] 6

  • Reg. inícial ([field] PARAM3)

A preencher sobre um modelo "Tratamento standard". Existe um registo inicial ? e de que tipo ? As condições possíveis são as seguintes :

 Sem registo inicial

  Confirmação Sim / Não ( mensagem permitindo a confirmação de execução desta acção)
  Caixa de diálogo ( registo janela restrito: botões limitados a "OK", e "FIM", sem menus, um único ecrã).
  Registo janela ( registo janela pelo modelo "Registo de janela").
  Selecção da lista ( janela de selecção )
  Selecção da tabela ( janela de selecção ).
  • Janela critério ([field] PARAM4)

A preencher se o registo inicial é :
  Caixa de diálogo
  Registo janela

[Block number] 7

  • Tratamento standard ([field] CODTRT)

Tratamento standard no qual o programador escreve os complementos do modelo supervisor ou o sub-programa. Ver o detalhe desenvolvimento.
  • Sub-programa ([field] SUBPRG)

Este campo é registável sobre uma acção de tipo "Tratamento diverso". Não há tratamento supervisor associado a esta acção, mas apenas este sub-programa. É definido seja no tratamento standard, seja no tratamento específico.

Um único sub-programa foi executado : Este do específico se o tratamento existe, senão aquele do standard. O acréscimo de específico sobre uma acção standard de tipo "Tratamento diverso" vem então em anular e substitui.

[Block number] 8

  • Tratamento especifico ([field] SPETRT)

Tratamento específico no qual o específico escreve os complementos do modelo supervisor ou o sub-programa. A actualização do tratamento específico não necessita de protecção por código actividade.  

Ver o detalhe desenvolvimento.

  • Expressão ([field] INSTRU)

Este campo é registável sobre uma acção de tipo "Fora modelo". Ele pode vir em acréscimo do sub-programa. Neste caso, ela se executará em seguida ao sub-programa.
  • identific. acção ([field] PARAM5)

Isto permite de identificar uma ou várias acções depois um tratamento, pelas variáveis seguintes :

  • TABLE para uma acção de tipo "Selecção tabela"
  • LISTE para uma acção de tipo "Selecção lista"
  • TRAIT para uma acção de tipo "Afixação diversa"

[Block number] 9

  • Acção seg. ([field] ACTSUI)

Este campo permite de indicar uma outra acção do dicionário. Temos ainda a possibilidade de encadear até 5 acções. Neste caso,o valor dos parâmetros de todas estas acções são a registar ao lançamento. As acções assim encadeadas podem ser de modelo diferente.

Se a acção principal está colocada sobre um campo, temos a possibilidade de não executar as acções encadeados em se posicionando a variável mkstat.

Se a acção principal está colocada sobre um botão de baixo de ecrã ou um menu, tem a possibilidade de não executar a acção ou as acções encadeadas em posicionando a variável OK=0 na acção AVANT_SUITE.

Acessibilidade

  • Workflow ([field] AMSFLG)

Flag permitindo de autorizar ou não esta acção em workflow. Esta autorização não permite que sobre uma acção de tipo "Tratamento diverso". Esta flag está nomeadamente controlada em criação de uma regra workflow.
  • Batch ([field] ABTFLG)

Flag permitindo de autorizar ou não esta acção em batch. Esta autorização não permite que sobre uma acção de tipo "Tratamento diverso". Esta flag está nomeadamente controlada em criação de uma tarefa batch.
  • Public ([field] PUBFLG)

Esta flag indica que este elemento está publicado. É então reutilizável pelo específico. Uma documentação associada está livre.
  • Batch obrigatório em WEB ([field] NOWEB)

Se esta flag está posicionada, esta acção não poderá se executar em interactivo, que em modo cliente/servidor e efectivamente as versões web (web nativo, web services). Isso permite evitar o tratamento pesado em interactivo em versão web.

Descritivo

  • [field] TEXTE

Texto alimentando o parágrafo PRE "apresentação", de ajuda sobre a ficha. Na gestão da documentação, identifica-se a ficha de ajuda por :

  • tipo de documentação = ACT
  • código de documentação = nome da acção

[Tab] Parâmetros

Encontram-se as informações relativas ao conjunto dos parâmetros de ação.

[Block number] 1

  • Zona corrente ([field] TYPACT)

Quando esta zona está assinalada, o primeiro parâmetro é considerado como sendo a zona corrente. Se esta acção está colocada sobre um campo, o primeiro parâmetro está preenchido automaticamente pelo valor do campo corrrente; Não aparece então, na lista dos parâmetros a preencher sobre o campo, no dicionário dos écrãs.
  • Acção antes/depois zona ([field] PARAM6)

Possibilidade de acrescentar do tratamento antes e / ou após um modelo Objecto, Consulta, Tratamento standard, Registo janela, quando esta acção está colocada sobre o menu contextual do campo, Isto permite de preparar um contexto (por exemplo afectar a variável GCONSULT) antes de executar a acção e / ou desencadear por exemplo uma afixação após a acção.

O código de acção "antes" é o nome da etiqueta precedida de A. O código da acção "após" é o nome da etiqueta precedida de B.

Exemplo : se a acção está imputada ao botão 2 do campo CHP, as etiquetas antes e após serão AB2_CHP e BB2_CHP. Elas devem ser programadas na etiqueta $ACTION do tratamento standard ou específico.

[Grid]

  • Nr. ([field] NUMLIG)
  • Cód. ([field] CODPAR)

códigos parâmetros da acção.

A ordem na qual os códigos parâmetros devem ser registados no presente quadro é muito importante : Deve corresponder à ordem na qual os parâmetros foram registados no sub-programa que permite a execução da acção.

Cada parâmetro deve ter sido declarado, no dicionário dos parâmetros. Antes de criar um código parâmetro, verificar que não existe já sob um outro código, afim de evitar as redundâncias. A tecla de selecção vos permite de obter a lista.

  • Tipo parâmetro ([field] TYPPAR)

Indica a natureza do parâmetro : char, integer, decimal etc. Esta informação não está registável que em criação do parâmetro, no dicionário dos parâmetros.

Atenção! : Não se pode utilizar um parâmetro de tipo clob/blob, numa acção, que nos dois casos seguintes:

  • Sobre uma acção de tipo "Tratamento diverso", sobre os parâmetros,
  • Sobre toda outra acção, e somente se a check box "Zona corrente" está assinalada, sobre o primeiro parâmetro.
  • Tipo argumento ([field] ADRVAL)

Indica como é passado o parâmetro

por endereço (uma variável é necessária para passar o parâmetro. Em saída do sub-programa, o valor do parâmetro eventualmente modificado retornou ao tratamento que o chama).

por valor (o parâmetro está passado por uma variável ou por constante. Em saída do sub-programa, o valor do parâmetro eventualmente modificado retornou ao tratamento que o chama).
