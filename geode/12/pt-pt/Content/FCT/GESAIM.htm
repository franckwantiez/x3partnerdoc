Destinos

O software permite enviar uma lista versus um conjunto de destinos, em particular versus toda a impressora Windows (TM) visível sobre a rede do posto cliente, mas também versus os ficheiros ou de outros destinos. Afim de facilitar a escolha e afetação dos destinos por defeito, é possível - e recomendado - de centralizar a descrição destes destinos acessíveis globalmente nesta tabela.

Encontra-se nesta tabela, os destinos, identificados por um código, com as características permitindo de sofisticar o processo da escolha e de afetação por defeito, e também de definir como está realizada a impressão (em função do tipo de saída) :

  • via Crystal Reports e um servidor de impressão que se endereçará uma impressora rede.
  • via Crystal Reports em local sobre uma impressora acessível depois o posto cliente.
  • via Crystal Reports, a lista sendo enviada por mail em passando pelo motor de workflow
  • via uma geração de ficheiro ZPL enviada seja diretamente num ficheiro, seja sobre a porta das impressoras deste tipo acessível depois o servidor de tratamento, seja em passando por um servidor de impressão que reenviará o fluxo versus uma impressora rede.
  • via Crystal Reports, em gerando um ficheiro PDF que será pré-visualizado numa janelas que se abre ao ecrã.
  • via Crystal Reports, em gerando um ficheiro PDF que será enviado a um software de GED.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Ecrã de registo

Cabeçalho

  • Cód. ([field] COD)

Este código identifica de maneira única a ficha em curso
  • Descr. ([field] DES)

Título associado ao código precedente

  • Desc. curta ([field] SHO)

Intitulado curto literal sobre 10 caracteres alfanuméricos.

[Tab] Ecrã de registo

O registo dos destinos se faz sobre um único separador. Regista-se então o código do destino, depois se define as principais características (designação, seu intitulado curto, uma flag Activo nomeadamente).

De referir que o código de acesso permite filtrar os utilizadores tendo o direito de utilizar o destino (é a autorização de execução que corresponde a este direito).

Encontra-se enfim um tipo de saida, associada à identificação da impressora, e um tipo que permite filtrar as impressoras utilizáveis por lista.

  • Activo ([field] ENAFLG)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.
  • Código acesso ([field] ACS)

Este código de acesso permite restringir o acesso à ficha corrente a certos utilizadores.
Se a zona está registada, apenas os utilizadores dispondo deste código de acesso no seu perfil podem visualizar e modificar esta ficha.

O direito de execução associado ao destino controla o direito de utilizar este destino para a edição de uma lista.
  • Tipo de saída ([field] PRT)

Este campo define o destino da lista calculada. Os tipos de destino são os seguintes :

Pré-visualização

o fluxo gerado num formato compatível com uma afixação (texto, PDF, RTF, Word...) está afixado numa janela que se abre sobre o posto do utilizador. Esta impressão está formatada por Crystal Reports seja sobre o posto de utilizador, seja um servidor de impressão.

Impressora

Crystal Reports utiliza uma impressora que pode ser local, seja ser acedida depois um servidor de impressão.

Mensagem:

Crystal Reports criado um ficheiro contendo a mensagem num formato tal que PDF, texto, RTF, Word... Esta mensagem é utilizada como documento junto de um mail enviado via o motor de workflow.

Ficheiro

Crystal Reports criado um ficheiro contendo a mensagem num formato tal que PDF, texto, RTF, Word que está guardado sobre disco.

Impressora ZPL

o fluxo está tomado em carga por um programa de formatação particular associado à lista e definida por uma parametrização dedicada. Este fluxo pode ser enviado diretamente sobre uma impressora local, sobre uma impressora acessível depois o servidor de impressão, ou guardado num ficheiro.

Arquivo

Crystal Reports cria um ficheiro contendo a mensagem num formato tal que PDF, texto, RTF, Word. Este ficheio está endereçado a um sistema de GED para arquivo, via o conector standard GED.
  • Ficheiro destino ([field] PRTZPL)

Este campo permite indicar o nome do ficheiro gerado por uma impressora de tipo ZPL, ou o nome do directório no qual o ficheiro está criado. Deve se indicado com o caminho completo em "dur" ou numa fórmula (sob a forma = expressão).

Se o ficheiro se termina por um dos dois caracteres "/" ou "\" , é considerado como um directório de depósito dos ficheiros, e neste caso, o ficheiro gerado no sub-directório PAT do dossier está copiado no directório. O nome do ficheiro utilizado está composto da concatenação das três informações seguintes : o código do modelo ZPL, o número do processo corrente (adxuid(1)), e a data e hora corrente do servidor de tratamento, exprimidos sob o formato YYMMDDhhmmss.

Se o campo "Servidor de impressão" está registado, falta indicar um caminho acessível depois o servidor de impressão (exemplo : file://machine/répertoire/fichier). Se o campo "Servidor de impressão" não foi registado, pode-se indicar um caminho sob a forma máquina: service@chemin ou /chemin ou c:\chemin. Se o nome de ficheiro contém um "@", o ficheiro é copiado por uma ordem sistema depois o servidor de tratamento, senão o cliente.

Em todos os casos, é importante de bem parametrizar os direitos de acesso às máquinas.

Na função impressão, este nome de ficheiro está proposta por defeito após ter escolhido o destino da lista.
  • Servidor ([field] PRTSRV)

Pode-se definir o servidor de impressão a partir do qual será imprimindo a lista Se esta zona está vazia, a impressão será realizada localmente sobre o posto cliente.
  • Impressora ([field] PRTNAM)

O nome de impressora definida será proposta a impressão da lista. Deve corresponder ao caminho rede de impressora, tal e qual está definida no painel de configuração Windows.
  • Formato exp. ([field] PRTFMT)

Quando a saida se faz sobre o ficheiro, existem diferentes formatos possíveis que se podem seleccionar aqui. Esta lista de formatos depende da versão corrrente de Crystal Reports, ela é susceptível de evoluir no tempo.
  • Tipo ([field] PRTNAT)

Este campo permite afetar um tipo ao destino Este tipo, definido pelo menu local 22, possui as características seguintes :

  • o primerio é um tipo chave-mestra (toda a lista está suscetível de ser enviado a destinos deste tipo).
  • os outros tipos permitem de estabelecer as categorias particulares (impressoras código-barra, fax, etc.) que não podem ser utilizados que por certas listas, Com efeito, no dicionários das listas, é possível de definir um tipo de saida dedicada a uma lista em utilizando o mesmo menu local. Ainda, se este tipos não é o primeiro, deve ter correspondência entre o tipo afetado na lista e o tipo do destino utilizado.
  • Numero de cópias ([field] NBRCPY)

Permite, quando a saida se faz sobre impressora, de dar um valor por defeito ao número de cópias a realizar.
  • Cópias montadas ([field] ASSCPY)

Se várias cópias são pedias e se a chack box está assinalada, as páginas antes de editar o exemplar seguinte são assemblados. Senão, cada página está repetida. Por exemplo, no caso de uma impressão de 3 páginas em dois exemplares :

  • se a check box está assinalada, a ordem das páginas é 1, 2, 3, 1, 2, 3,
  • se a check box está assinalada, a ordem das páginas é 1, 1, 2, 2, 3, 3.

[Block number] 2

  • A partir do dossier ([field] DOSORG)

﻿Este campo permite de definir o dossier a partir do qual a ficha vai ser copiada. As sintaxes possíveis são descritas no anexo dedicado

Limites conhecidos

A utiização de um destino passando pelo servidor de impressão inibe a impressão dos documentos juntos. Estes não podem ser editados que sobre uma impressora em direto.
