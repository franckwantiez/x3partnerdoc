Pedidos análise

Esta função permite de criar os pedidos de análise (PA) para proceder a um controlo qualidade.

Um pedido de análise foi criado para um conjunto (nº ED, Artigo) ou um triplo (nº ED, Artigo, Nº de lote) sobre um armazém de "controlo qualidade" (de tipo mono-endereço).Cada conjunto/trio foi constituido a partir dos objectos de stocks (validados e de natureza de stock "qualidade") presente neste armazém.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Geral
  • [Tab] Objectos

O procedimento de criação de um pedido de análise (PA) é o seguinte:

  • registo de um armazém (de tipo mono-endereço) dedicado às operações de controlo qualidade.
  • Selecção de um conjunto (Nº de ED, Artigo) ou de um triplo (nº ED, Artigo,nº lote) a controlar para a lista esquerda "Entradas a controlar" (presentes sobre o armazém registado).
  • No registo do PA, se o artigo foi configurado para fazer de amostragem, o sistema determina o tamanho de amostra (em UA) a tomar para o controlo.

No final, o pedido de análise foi criado com as próprias características (flag endereçamento automático, tipo de amostra, ficha qualidade...) e os Objectos de Stock do conjunto ou triplo seleccionado são associados.

Cabeçalho

 

  • Número ([field] QLYNUM)

Sequência numérica atribuida automaticamente pelo sistema no momento da criação do pedido de análise. Este número é uma informação própria ao estabelecimento, todos os depositantes simultaneamente. Esta informação constitui uma chave única de acesso a um pedido de análise.
  • Referência ([field] REF)
  • Data ([field] DAT)
  • Status ([field] QLYSTA)

as diferentes listas de um pedido de análise intervindo no procedimento de controlo qualidade são:

  • 1 - Espera controlo qualidade
  • "7-Espera de validação":
  • 9 - validado/saldado
  • Armazém ([field] STO)

Código permitindo de identificar o armazém. Verificado na tabela dos armazéns.

[Tab] Geral

Este separador recapitula as características principais do pedido de análise.

[Block number] 1

Sequência numérica atribuida automaticamente pelo sistema no momento da validação do’ Cabeçalho de’Entrada Este número é uma informação própria ao estabelecimento, todos os depositantes simultaneamente. Esta informação constitui uma chave única de acesso a uma Entrada Directa.

  • Artigo ([field] ITM)

Código artigo definido na tabela dos artigos Warehousing
  • Nr. entrada ([field] INPNUM)
  • Endereç. auto ([field] ALLQLYAUT)

Quando a flag está activada, o endereçamento automatico dos stocks controlado está realizado na validação/encerramento de um Pedido de Análise: As transferências do armazém qualidade versus são criados em automatico.

Inform. gerais

  • Nr. lote ([field] LOT)

Lote sobre o qual porta a devolução.
  • Ficha qualid. ([field] QLYCRD)

A ficha qualidade permite de consignar dois tipos de informações:

  • texto descrevendo o processo de controlo ou o modo operatório.
  • jogo de questões/respostas afim de introduzir um registo de parâmetros no momento do controlo qualidade.
    As respostas fornecidas podem ser controlados por valores predefinidos.
  • Data fecho ([field] CLODAT)

Data de encerramento do pedido de análise.

  • Tipo de amostragem ([field] SMPTYP)

Permite de activar o controlo qualidade com amostragem: Se configurado a "simples" uma amostra a controlar será automaticamente calculado segundo os parâmetros seguintes do plano de amostragem; senão, o controlo qualidade levará sobre o conjunto da quantidade recepcionado.

  • Quantid. UA/US total ([field] TOTQTY)
  • [field] UOMSYM
  • Nível geral de inspecção ([field] GENLEVINS)
  • NQA ([field] NQA)

Nível de Qualidade Aceitável : permite o cálculo do número de documentos não conforme a partir do qual a amostra controlada está considerada regeitada.

Amostragem

  • Dimensão amostra em UA/US ([field] SMP)

Este campo indica o tamanho de amostra em UA para um pedido de análise com amostragem. Este tamanho é determinada segundo a norma ISO 2859 dependente do tamanho do lote completo e do parâmetro artigo de amostragem NQA (Nível de Qualidade Aceitável).

  • [field] UOMSYM1
  • Status ([field] WSMPSTA)

indica a lista da amostra no seguimento do controlo qualidade:

  • Não qaulificado: nenhum controlo foi feito.
  • Aceite: A amostra está considerada como conforme
  • Rejeitada: A amostra está considerada como não conforme

[Grid] Objectos

  • [field] NBLIG
  • Nr cnts ([field] CTRSMP)
  • Conteúdo ([field] CTR)
  • Nr. UA/US por cnt ([field] CTRCOE)
  • US ([field] UOMSYM2)

Quantidades

  • Números peças em conformidade ([field] ALLQTY)

  • para um pedido de análise sem amostragem, é o número de peça conforme do lote completo.
  • para um pedido de análise com amostragem, é o número de peça conforme da amostra.
  • Número de peças não conforme ([field] DILQTY)

  • para um pedido de análise sem amostragem, é o número de peça não conforme do lote completo.
  • para um pedido de análise com amostragem, é o número de peça não conforme da amostra.
  • Número de peças consumidas ([field] USEQTY)

  • para um pedido de análise sem amostragem, é o número de peça consumível do lote completo.
  • para um pedido de análise com amostragem, é o número de peça condumida por entre as peças não conformes da amostra.

As diferentes listas de uma linha de pedido de análise (linha de OS) intervindo no procedimento de controlo qualidade são:

  • +R: ficha qualidade registada (se existe) e OS rejeitado.
  • *: OS completamente qualificado.
