Declaração de kit

Este ecrã permite proceder a:

  • declaração de assemblagem de kits no procedimento de assemblagem.
  • a declaração de desasssemblagem de kits no procedimento de desassemblagem.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Movimentos agrupados
  • [Tab] Kit/Componentes

O ecrã se decompõe em :

  • uma lista de esquerda permitindo seleccionar o número de declaração que se deseja tratar.
  • uma parte direita dando o detalhe da declaração:
      • Um cabeçalho
      • um separador "A consumir" listando todos os movimentos componentes/kits à linha de OA.
      • um separador "A gerar" dando as características das OS Kit/componente que serão criados à validação da declaração.
      • um separador "Kit / Componentes" afixando a linha kit de OA afecta as quantidades componentes seleccionados durante o procedimento de declaração. Este separador permite também verificar que as relações quantitativas estão respeitadas ou não (a check box "Total UA" se afixa em vermelho se incoerência)

Procedimento de declaração de assemblagem

O procedimento de declaração de assemblagem de kit se desenrola em 3 etapas:

  • Etapa 1 : Registo da Qtd de kit que se deseja criar no separador "A gerar".
  • Etapa 2 : selecção dos movimentos componentes e das quantidades a assemblat no separador "A consumir".
  • Etapa 3 : Validação da declaração.
Etapa 1 : Registo da Qtd de kit que se deseja criar no separador "A gerar".

O operador regista as informações do bloco "quantidade" (nº recipiente, recipiente ...). A quantidade do kit a fabricar está indicada no campo "Nº de UA".

Etapa 2 : selecção dos movimentos componentes e das quantidades a assemblar no separador "A consumir".

Esta etapa consiste em assinalar as linhas componentes e de registar a quantidade a assemblar (campo "Nº de UA"). Esta operação pode ser manual ou bem automática graças ao botão "Auto".

Em cada vez que uma linha está assinalada, o sistema repercute automaticamente a qtd registada no "Total UA" da linha componente correspondente de separador "Kit/Componentes".

Segundo as quantidades componentes seleccionados e a quantidade de kit pedida, a declaração toma os status seguintes :

  • "5-Não conforme": Isso indica que existe uma incoerência sobre os pontos seguintes :
      • sobre as quantidades componentes não respeitando as relações quantitativas do Kit
      • a quantidade de kit a fabricar é diferente daquela podendo ser realmente declarada com os componentes seleccionados.
  • "6 - Conforme": Isso indica que não existe nenhuma incoerência tanto do ponto de vista de relações quantitativas do kit a fabricar. Apenas as declarações neste estado poderão ser validadas na etapa 3.

Observação : O botão "Registar" permite salvaguardar a lista de avanço da declaração.

Etapa 3 : Validação da declaração.

Quando todos os componentes foram seleccionados e que a declaração está em status "6 - Conforme", o operador pode então o validar.

A validação da declaração e após registo possível de um código operador (segundo configuração ao nível da config. dos stocks), o sistema:

  • cria as OS kit por ajustamento automático (o código movimento utilizado é aquele definido na configuração dos stocks) segundo as características de separador "Objecto de stock".
  • valida os movimentos de saida dos componentes.
  • passa a declaração kit em status "9 - Validado" e cria uma nova declaração kit para a quantidade kit restante a criar (caso onde a linha Kit de OA não esteja terminada).
  • Actualiza a linha de OA afectada.

Procedimento de declaração de desassemblagem

O procedimento de declaração de desassemblagem de kit se desenrola em 3 etapas:

  • Etapa 1 : Registo da Qtd de kit que se deseja desassemblar no separador "A consumir".
  • Etapa 2 : Registo dos movimentos componentes a criar no separador "A gerar".
  • Etapa 3 : Validação da declaração.
Etapa 1 : Registo da Qtd de kit que se deseja desassemblar no separador "A consumir".

O operador regista as informações do bloco "quantidade" (nº recipiente, recipiente ...). A quantidade do kit está indicada no campo "Nº de UA".

Etapa 2 : Registo dos movimentos componentes a criar no separador "A gerar".

Esta etapa consiste em assinalar as linhas componentes e de registar a quantidade a criar (campo "Nº de UA").

Em cada vez que uma linha está assinalada, o sistema repercute automaticamente a qtd registada no "Total UA" da linha componente correspondente de separador "Kit/Componentes".

Observação : O botão "Registar" permite salvaguardar o estado de avanço da declaração.

Etapa 3 : Validação da declaração.

Quando todos os componentes foram seleccionados e que a declaração está em status "6 - Conforme", o operador pode então o validar.

A validação da declaração e após registo possível de um código operador (segundo configuração ao nível da config. dos stocks), o sistema:

  • cria as OS kit por ajustamento automático (o código movimento utilizado é aquele definido na configuração dos stocks) segundo as características de separador "A gerar".
  • valida os movimentos de saida dos componentes.
  • passa a declaração em status "9 - Validado" e cria uma nova declaração do mesmo tipo para a quantidade restante a desassemblar (caso onde a linha Kit de OA não esteja terminada).
  • Actualiza a linha de OA afectada.

Botões específicos

Validar

Permite validar uma declaração. Este botão está activo unicamente se a declaração está num status "6 - Conforme".
Suprimir

Permite suprimir uma declaração. Este botão está activo unicamente se nenhum movimento componente foi seleccionado.

O facto de suprimir esta declaração vai arrastar o fecho da linha de OA.

Auto

Permite seleccionar automaticamente os movmentos "A consumir" segundo a quantidade de kit a declarar. Este botão está activo e utilizável unicamente para o procedimento de assemblagem.

Barre de menus

Zoom / Movimentos

Permite de ir consultar os movimentos de assemblagens relativo à declaração.
