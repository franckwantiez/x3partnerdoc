Famílias vinhetas

Esta função permite de definir :

  • A relação entre as vinhetas que serão apresentados no portal e a função carregada de asssegurar a afixação. Esta relação é estabelecida pelo intermédio do módulo técnico.
  • Os parâmetros que permitem caracterizar as vinhetas utilizando esta família, em oferecendo a possibilidade de definir os valores por defeito que serão propostos no momento da criação de uma vinheta.

As diferentes famílias disponíveis são as seguintes :

  • Favoritos (FAV) : esta família serve exactamente a apresentar a lista das funções "favoritas" escolhidas pelo utilizador. O único parâmetro associado é a altura da vinheta inicializada por defeito ao valor 450
  • Leitor de fluxo de video (FLV) : esta família permite de afixar os videos definidos em formato flv. Os parâmetros associados a esta vinheta são:
      • A altura (inicializada por defeito a 300)
      • A fonte dos dados : indica o acondicionamento do ficheiro flv a ler (sob a forma de ligação http)
      • A mensagem de espera : que afixará sobre a vinheta quando o leitor video não está activado.
  • Leitor componente "flash" (FSH) : esta família permite afixar todo o tipo de componnete "flash" disponível na aplicação sob a forma de vista. Os parâmetros associados :
      • A altura (inicializada por defeito a 300)
      • O acesso aos dados : dá o nome do programa carregado de entregar o fluxo XML de dados que serão interpretados pelo componente "flash" (por defeito é o valor SUBAPSMON:CLOBEXP que é preenchido correspondente ao subprograma CLOBEXP do programa SUBAPSMON)
      • A vista portal : O código da vista portal a afixar (a vista do portal deve ser parametrizada com um componente visual de tipo "flash" - opção B do objecto APV)
  • Menu (MENU) : permite definir um menu aoqual serão afixados as vinhetas. Este menu pode ser desdobrado ou não pelo utilizador e conter ele próprio de sub-menus.
      • Menu desdobrado : O menu aparece desdobrado ou não segundo o valor (inicializado por defeito a True=desdobrado)
  • Navegador (NAVIG) : esta família serve exeactamente a apresentar a lista das funções definidas no perfil menu do utilizador afectado. O único parâmetro associado é a altura da vinheta inicializada por defeito ao valor 450
  • Processos (PROC) : esta família é utilizada para apresentar os processos desenhados com o editor de processo. O módulo de afixação (aparelho de leitura) apresente estes processos num modo interactivo permitindo de desencadear as acções e de abrir as funções do software. Os parâmetros a definir são os seguintes :
      • Altura : definido na altura da vinheta (por defeito 300, é possível de ir sem scrolling vertical até 450)
      • Acesso aos dados : Nome do programa de acesso fornecendo os dados do processos ao aparelho de leitura (por defeito SUBAPR:CLOBEXP)
      • Código processo: O processo deve ser presente na tabela APROCESSUS
      • Menu processo : É possível de associar a cada vinheta processos um menu que aparecerá ao alto na parte esquerda sob a forma de "treeview" e que permitirá navegar entre os processos de um mesmo "objecto". Se o parâmetro não foi preenchido, não existe navegação inter processos de proposta sobre a vinheta. Este código menu processo deve fazer referência a um elemento da tabela APROMEN.
  • Afixador de texto (READ) : Esta família permite de afixar todo o tipo de fluxo texto, html ou xml numa vinheta. Os parâmetros a preencher são os seguintes
      • Altura : definindo a altura da vinheta (por defeito 300)
      • Acesso aos dados : definido o programa de acesso aos dados que dará o fluxo a afixar. Este programa está livre mas deve obrigatóriamente ser definido com dois parâmetros
          • Uma zona parâmetro de tipo Car passado em valor que tornará o conteúdo de "parâmetros"
          • Um "clob" passado em variável que conterá na devolução o fluxo a afixar na vinheta
      • Parâmetros define uma cadeia alfanumérica que permitirá passar os parâmetros ao programa a executar.
  • Leitor de fluxo RSS (RSS) : Esta família permite visualizar numa vinheta dos fluxos rss publicados sob um formato rss 2.0 Os parâmetros a preencher são os seguintes
      • Fonte de dados : URL da linha rss a afixar (por exemplo eis o endereço da linha rss do diário "o mundo" http://www.lemonde.fr/rss/sequence/0,2-3208,1-0,0.xm)
      • Afixação detalhada: afixa o texto associado à informação dos fluxos rss (por defeito verdade= detalhe afixado)
      • Número de elementos : a afixar na linha o fio rss da vinheta (por defeito 3)
  • Separador (SEP) : Esta família não foi utilizada que para criar uma linha de separação na afixação das vinhetas a seleccionar. Ela é de tipo "separador" e possui um unico parâmetro que dá o caracter a utilizar como separador (por defeito-)
  • Afixação de tabelas (TAB) :
      • Número de linhas : Número de linhas a afixar (por defeito 20)
      • Fonte de dados : Programa a chamar para obter o fluxo de dados (por defeito SUBAPSMON:CLOBEXP)
      • Parâmetros O código da vista portal a afixar (a vista do portal deve ser parametrizado com um componente visual de tipo tabela - opção C do objecto APV)
  • Página Web (WEB) : Esta família permite de afixar as páginas web
      • Altura : definindo a altura da vinheta (por defeito 300)
      • Scrolling : Autoriza o "scrolling" sobre a vinheta ou não (por defeito verdade=scrolling autorizado)
      • Fonte de dados : URL a afixar
  • Notas (WNOTE) : Esta família permite de dispor vinhetas de tipo bloco notas.
      • Texto : preencha aqui o texto de convite que aparecerá na vinheta. Será preenchido pelo texto que o utilizador registará uma vez que terá utilizado a vinheta.
