Arranques estatísticos

Um utilitário geral de parametrização de estatísticas é proposto no software. Este utilitário permite criar uma base de estatística cujas características são as seguintes :

  • Lista de eventos desencadeadores gerados pelo software e permitindo lançar a atualização de base estatística.
  • Definição de códigos estatísticos permitindo acumular os montantes cifrados, se apoiando sobre um evento desencadeador e dos critérios (de 1 a 8 critérios banalizados ao máximo, mais a sociedade e o estabelecimento).
  • Geração para cada código de registos de acumulados segundo as combinações dos critérios definidos.

Estes registos de acumulados podem então ser utilizados por listas ou de interrogações acessíveis diretamente depois uma ficha de base, para obter os quadros estatísticos.

A parametrização dos eventos desencadeadores permite indicar, para cada evento suscetível de ser chamado pelo software, os dados em linha e os critérios de seleção correspondentes.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Geral
  • [Tab] Avançado

Cabeçalho

Permite de identificar o código desencadeador.

  • Cód. ([field] COD)

O desencadeador estatístico permite identificar o elemento que desencadeie a estatística.
  • Descr. ([field] INTIT)

Permite de definir um titulo associado a cada ficha.

[Tab] Geral

Define-se no primeiro separador as informações gerais ligadas ao desencadeador estatística : a tabela principal sobre a qual está gerada a agregação, e das zonas permitindo determinar o registo de acumulado a movimentar.

Estas zonas importantes, na medida onde a tabela STAT que contém os dados está estruturada por sociedade, estabelecimento, e data (segundo a periodicidade), e que o sub-programa de acumulado deve saber nas quais zonas da tabela desencadeadora se encontram os valores a utilizar.

Características

  • Desc. curta ([field] INTSHO)

Intitulado curto literal sobre 10 caracteres alfanuméricos.

  • Módulo ([field] MODULE)

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.
  • Cód. activ. ([field] CODACT)

Um código actividade permite :

  • de tornar opcional um elemento do dicionário se o valor associado ao código actividade é nulo.
  • de assinar os elementos específicos desde logo são marcados por um código começando por X,Y ou Z.
  • de dimensionar um número de linhas máximas quando o código de actividade maerca os elementos de um quadro.

Assim, se o código actividade está não activo, o elemento marcado não será utilizável, e o código associado (se existe um) não será gerado nem activável.

[Block number] 2

  • Tabela de arranque ([field] RLGTBL)

Define a tabela principal a partir da qual se define os campos que vão servir a desencadear a atualização das estatísticas. Esta tabela deve estar em linha no momento de chamada com um registo corrente característica dos dados a acumular.

Opções

  • Data zona ([field] DATFLD)

Define o campo da tabela de desencadeamento onde se encontra o código da sociedade àqual está anexada o registo. Esta data permite determinar no qual período o valor estatístico relativo ao registos será acumulado, cômputo da periodicidade de cálculo escolhido.
  • Zona socied. ([field] CPYFLD)

Define o campo da tabela de desencadeamento onde se encontra o código da sociedade àqual está anexada o registo. Se este campo, ou se o campo estabelecimento são preenchidos, as estatísticas se apoiam sobre este código poderão se declinados por sociedade
  • Zona estab. ([field] FCYFLD)

Define o campo da tabela de desencadeamento onde se encontra o código da sociedade àqual está anexado o registo. Se este campo está registado, as estatisticas se apoiando sobre este código poderão ser declinados pelo estabelecimento.

[Block number] 4

[Tab] Avançado

O segundo separador permite completar a descrição do contexto de desencadeamento, quando este está mais complexo, nomeadamente:

  • em descrevendo, num primeiro quadro, junções permitindo provocar a leitura de outras tabelas ligadas durante o processo de acumulado. A ordem de registo das tabelas no quadro é importante, pois que uma tabela pode ser ligada em cascata à tabela principal e a todas as tabelas que a precedem.
  • em dando as condições lógicas que devem ser realizadas para que o acumulado seja feito.
  • em definindo as variáveis (via as fórmulas de cálculo) utilizáveis diretamente em definição de estatísticas.

[Table number] 1

  • Nr. ([field] NUMLIG3)
  • Tabelas ligadas ([field] TBL)

Define uma tabela que será atualizada em linha no momento do processo de acumulado, cujo registo está determinado por um valor de chave da ligação. Isto permite igualmente de basear os critérios de estatísticas e os valores acumulados sobre os campos desta tabela.
  • Abreviatura ([field] ABRLNK)

Corresponde à abreviação sob a qual a tabela ligada está aberta.

Se este campo não está registado, a abreviação é a abreviação por defeito da tabela. Pode ser útil de registar uma abreviação para não entrar em conflito com o contexto chamador
  • Chv ligação ([field] CLELNK)

Define o código da chave da tabela utilizada para realizar a leitura das linhas ligadas. Por defeito, a primerira chave da tabela é utilizada.

Uma vez este campo alimentado, o quadro seguinte Mapa chaves - propriedades está alimentado segundo o código dos diferentes segmentos de chave.
  • Expressão de ligação ([field] EXPLNK)

Este campo está definido como uma ou várias expressões calculadas separadas por um ponto e vírgula. Cada expressão está avaliada, e o resultado permite de conhecer o valor da chave utilizada para realizar a junção. Quando as junções múltiplas são autorizadas, pode-se não dar que os primeiros elementos da chave.

Nas expressões, podem-se utilizar constantes, e campos emitidos das tabelas previamente definidas na lista das relações.

[Table number] 2

  • Nr. ([field] NUMLIG1)
  • Critérios de arranque ([field] CRI)

Permite de definir um critério sob forma de uma expressão lógica podendo fazer intervir todas as tabelas em linha, constantes e das funções.

Apenas as linhas para as quais a avaliação das condições reenvia um resultado não nulo (ie. verdadeiro) são afetados pela extração de dados.

[Table number] 3

  • Nr. ([field] NUMLIG2)
  • Variável ([field] VARNAM)

Define o nome de uma variável diretamente utilizável nos acumulados estatísticos baseados sobre o evento desencadeador, para evitar de ter a definir as expressões calculadas baseadas sobre os campos das tabelas em linha.
  • Descr. ([field] VARINTIT)

Título associado ao código precedente

  • Expressão ([field] VARFOR)

Define a fórmula de cálculo correspondnete à variável.

Esta fórmula é uma expressão calculada podendo fazer intervir as constantes, os operadores, funções, e todas as variáveis em linha no contexto de agregação

  • Divisa ([field] VARCUR)

Afim de permitir de criar os acumulados sobre os montantes exprimidos em divisas diferentes convertidos automaticamente, pode-se registar aqui uma constante ou um campo definindo um código divisa. Registar-se-á em seguida, na definição do acumulado estatístico, o código da divisa na qual deve ser gerada cada acumulado.

Lista dos diferentes eventos desencadeadores standard

Os eventos desencadeadores standard são documentados nas documentações, organizados por módulo, e acessível pela ligação seguinte.

De referir que, nos casos de documentos complexos (de tipo cabeçalho e linha), pode existir dois eventos desencadeadores, uma chamada sobre uma vez por cabeçalho, a outra chamada um vez por linha Segundo o nível de detalhe desejado pela agregação, escolher-se-á um ou outro.
