Clientes

A base de dados dos clientes permite registar regras de gestão limpas a cada cliente. Estas informações serão exploradas seguidamente para os fluxos de saída (Endereço de entrega, de facturação, de preparação, transportador, …).

As informações clientes entregues são informação obrigatória em Ordens de Entrega  (OE). Podem ser transmitidas por conversão ou registos directamente aquando da criação da OE.

Se forem transmitidas por conversão, só o nível 3 será mestre das informações clientes e nesta base clientes em Warehousing não será preenchido.

Para criar um cliente, 4 separadores necessários.

Cabeçalho

Atribui-se um código que o identifica de maneira única sobre 10 caracteres, bem como um título. Estas duas informações encontram-se em cabeçalho do ecrã de gestão; encontram-se seguidamente 3 separadores de definição.

Código de cliente.

  • Cliente ([field] CLT)
  • Descr. ([field] NAM)
  • Desc. curta ([field] DESSHO)
  • Activo ([field] ENAFLG)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.

[Tab] Geral

Este separador permite registar a informação geral do cliente nomeadamente o seu endereço de entrega.

No momento do registo do código postal, um controlo automático pode ser realizado e a cidade ser preenchida (isto depende dos parâmetros CTY, POSCOD).

Nome do país. Se o código país está preenchido, então este campo está registado automaticamente por relação à tabela dos países. Se o código país não’ ;está preenchido, então o campo pode ser registado livremente pelo’utilizador.

Indicação precisa do domicílio.

Campo de observação. Registo livre.

  • País ([field] CRY)

Código permitindo de identificar o país. Este código está controlado na tabela dos países. O nome correspondente ao código país está então afixado automaticamente. Se o código país não foi ultrapasssado pelo utilizador, então o nome do país poderá ser registar livremente.
  • [field] CRYNAM
  • [field] NAM
  • [field] ADD
  • Código Postal ([field] POS)

Código postal cujo formato de registo está definido na tabela País.
O código postal permite de determinar a cidade. Quando o código postal está modificado, a cidade correspondente ao novo código está automaticamente afixado.
Casos particulares :

  • Para Andorra, indique 99130
  • Para Mónaco, indique 99138
  • Para os outros países estrangeitos, indique o código INSEE do país ou 99999 se ele é desconhecido.
  • Cidade ([field] CTY)

Nome da cidade. A cidade proposta automaticamente é aquela associada código postal indicado previamente.
É possível igualmente de registar directamente o nome da cidade:

    se esta está definida, o código postal associado está automaticamente afixada

    se esta não existe, uma mensagem de advertência aparece e a prossegue do registo está autorizado.

  • Região ([field] SAT)

Este campo contém a subdivisão geográfica entrando na definição de endereço postal.
Por exemplo, pode-se tratar do Departamento para a França, do Estado para os Estados Unidos. O título da subdivisão assim que o controlo ou não sobre o registo desta informação são definidos ao nível da parametrização do País.
Se está controlado, este código está dependente da sub-divisão geográfica correspondente.

Esta subdivisão está alimentada autromaticamente após o registo do código postal e da cidade, a partir da parametrização efetuada ao nível dos Códigos postais.
  • Telefone ([field] TEL)

Número de telefone cujo formato de registo depende do país.
  • Fax ([field] FAX)

Número de telecópia cujo formato de registo é idêntico aquele do telefone e depende do país.
  • E-mail ([field] ADDEML)

Sem ajuda ligada a este campo

  • Observações ([field] REM)

[Tab] Entrega

Tratam-se de opções que permitem conhecer os actos de gestão que cliente é um susceptível de realizar. Estas informações permitem inicializar as da O. E. quando elas criam uma OE que utiliza o código com parâmetros na tabela dos clientes:

Preparação :
Entrega parcial: Este valor condiciona a possibilidade de preparar e expedir OE incompletas.
Sim. Todos as parciais são autorizadas.
Por linha. Os parciais de linha são proibidos. Uma linha de OE que seja preparada parcialmente, não será lançada em preparação.
Não. Os parciais são proibidos. Uma OE que é preparada parcialmente, não será lançada em preparação.

Prioridade de serviço. Permite impor uma ordem de tratamento das OE na cadeia das saídas e por conseguinte arbitrar entre as rupturas de stocks.
Observação: O valor "0” é menos prioritário que "9”. O valor "branco” vale para "0”.
A configuração das saídas do recipiente permite indicar se a prioridade de serviço é mais importante que o parâmetro Entrega Parcial autorizado para o depositante corrente.

Transporte

Volta genérica. Selecção da volta genérica afectada por defeito aquando da criação da OE.
Transportador. Selecção do transportador afectado por defeito no momento da criação da OE.
Modo Transporte. Selecção do modo de transporte afectado por defeito no momento da criação da OE.
Posição encontrada. Permite o arranjo das preparações no camião em função da posição de entrega do cliente na volta.
Zona de entrega. A zona de entrega corresponde a um código a definir utilizável na afectação do transportador/modo.

Facturação

Tipo de porte. Selecção do tipo de porte afectado por defeito aquando da criação da OE.
Cliente facturado. Selecção do código cliente afectado por defeito aquando da criação da OE

Bom de entrega

Língua. Indica a língua de edição do Cupão de Entrega
Edição. Indicado se o B.E deve ser imprimido
Divisa. Divisa a utilizar aquando da pré-facturação transporte

Cálculo ascendente porte BL (Guia de entrega). Este parâmetro permite activar o cálculo do montante de porte associado a cada BL (Guia de entrega) expedido.
Código tarifa. Indica o código tarifa a utilizar para calcular o montante do porte BE (Bom de Entrega).

Preparação

Indica se o cliente aceita ’ser entregue parcialmente ; Existem várias possíbilidades, seja sim (aceita qualquer que seja a quantidade pedido), seja por linha (aceita na condição que’a integralidade da linha seja entregue), seja não (não’é aceite).

A prioridade de serviço permite definir uma ordem de prioridade no quadro do tratamento das ordens de entrega.

  • Gestão SSCC na saída ([field] SSCFLG)

se assinalada, geram-se os códigos SSCC sobre as OE deste cliente.

  • Gestão lote ([field] LOTMGT)
  • Entrega parcial ([field] PTLPIK)
  • Prioridade serviço ([field] PIR)

Transporte

A volta genérica é uma jornada de entrega « fixo » (por relação aos dias, na «hora limite de«expedição), uma volta sendo o conteúdo de«um camião.

Ordem na qual a volta deve ser carregada. Esta informação permite atribuir às OE ( ordens de encomenda) de uma volta, um número de sequência de descarga ou de carregamento.

Zona de entrega do cliente entregue.

  • Rota genérica ([field] GDR)
  • Transportador ([field] CRR)

Código identificando o transportador. Este código deve ser definido na tabela dos transportadores.

  • Modo transpor. ([field] CRRMOD)

Geralmente, o modo de transporte corresponde às noções de servidor de mensagem, fretagem, expresso,... O modo de transporte está verificado por relação à tabela dos modos de transporte.
  • Posição rota ([field] DRNPST)
  • Zona entrega ([field] ARE)

Faturação

O tipo de porte pode conter por exemplo os códigos Incoterm (despesas de porte "devido" ;, "pago", …).

O código cliente de facturação pode ser diferente do código do cliente dest inatário. É verificado na tabela dos códigos clientes.

  • Tipo de porto ([field] CGT)
  • Cliente fact. ([field] INVCLT)
  • [field] ZINVCLT

Guia de expedição

Parametrização da língua desejada para editar o cupão de entrega e a lista de emba lagem.

Código estado para a “edição da Guia de Remessa”. Este código & eacute; verificado na tabela dos códigos listas.

Permite calcular o montante do porte do RT e/ou GT - Guia de transporte, &agra ve; confirmação de expedição pela funcionalidade "funcionalidade pré-facturação transporte".

Código tarifa utilizada para calcular o montante do transporte por

funcionalidade "funcion alidade pré-facturação transporte".

  • Língua ([field] EDTLAN)
  • Edição ([field] DLNEDT)
  • Cálculo mnt. de porto ([field] DLNCST)
  • Cód. tarifa ([field] DLNRAC)
  • Divisa ([field] CUR)

Este código permite identificar a divisa de um estabelecimento, de um terceiro, ou de definir um valor por defeito. Este código está controlado na tabela das divisas.
É aconselhável de utilizar a codificação ISO no momento da criação de uma nova divisa.

[Tab] Contactos

Este separador permite definir uma lista de contactos associados a este estabelecimento. Cada contacto é identificado por um nome listado na parte esquerda do ecrã. A parte direita define as coordenadas completas associadas ao nome.

Cada registo de um novo contacto neste quadro alimenta automaticamente o ficheiro dos Interlocutores. Assim, um mesmo contacto pode ser associado a vários estabelecimentos, a sociedades ou terceiros sem estar a duplicar o registo interlocutor

  • Cód. ([field] CNTCOD)

Detalhe

Este campo associa uma fórmula de polidez ao nome patronímico do contacto.
Esta informação pode ser explorada sobre os documentos com destino dos contactos.
  • Cód. ([field] XCNTCOD)

Permite de dar um código ao interlocutor.
  • Apelido ([field] XCNTLNA)

Sem ajuda ligada a este campo

  • Nome ([field] XCNTFNA)

Este campo indica o prenome do contacto.

Contatos

  • País ([field] XCNTCRY)

Este código permite de identificar o país associado à informação.

Na tabela dos países, que controla o campo, associa-se um certo número de características úteis para realizar os controlos sobre informações que dependem, e nomeadamente :

  • o formato do número de telefone
  • o formato do número identificador uma empresa ou uma atividade (Cód. comercial, NAF em França) e o carater obrigatório ou não do registo,
  • o formato do código postal/cidade e da subdivisão geográfica e o caracter obrigatórios ou não do seu registo,
  • o caracter obrigatório ou não do registo do código SIREN, do número único de identificação assim que o nº de IVA intracomunitário,
  • o formato do código NIB
  • Telefone ([field] XCNTTEL)

Número de telefone cujo formato de registo depende do país. O parâmetro geral CTLTEL - Controlo número de telefone permite indicar o nível de controlo efectuado por relação ao formato do número registado.
  • Telemóvel ([field] XCNTMOB)

Este campo indica o número de telefone portátil cujo formato de registo depende do país.
  • Fax ([field] XCNTFAX)

Esta zona indica o número de telécópia cujo formato de registo está idêntico aquele do telefone e depende do país.
  • E-mail ([field] XCNTWEB)

Este campo indica o endereço de correio electrónico do contacto.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

Razão social, zona obrigatória.

Esta mensagem é afixada se nenhum Razão social for registado no momento da criação do recipiente.

Um endereço deve existir pelo menos

Esta mensagem aparece em criação ou modificação do terceiro, quando, no momento da validação, nenhum endereço foi atribuído ao terceiro.

Um endereço deve ser designado por defeito

Esta mensagem aparece em criação ou modificação do terceiro, quando, no momento da validação, nenhum dos endereços foi declarado como endereço por defeito.

País:... Ficha inexistente

Esta mensagem aparece quando o código do país registado não existe na tabela dos países. Tem a possibilidade de criar o país na tabela dos países via o túnel (Botão contextual).

Volta genérica, Ficha inexistente.

Esta mensagem é afixada se a volta genérica preenchida não foi conhecida no ficheiro das voltas genéricas.

Transportador, Ficha inexistente.

Esta mensagem é afixada se o transportador informado não for conhecido no ficheiro dos transportadores.

Modo transporte, Ficha inexistente.

Esta mensagem é afixada se o modo de transporte informado não for conhecido no ficheiro dos modos de transporte.

Tipo de porte, Ficha inexistente.

Esta mensagem é afixada se o tipo de porte informado não for conhecido no ficheiro dos tipos de porte.

Cliente expedido, Ficha inexistente.

Esta mensagem é afixada se o cliente expedido preenchido não consta no ficheiro dos clientes.
