Gestão das assinaturas

É possível de definir graças a esta função das assinaturas permitindo um lançamento regular para o servidor batch :

  • seja de tarefas batch
  • seja deencadeamentos de tarefas batch

Estas assinaturas provocarão uma execução com uma periodicidade parametrizável.

Os parãmetros de lançamento são registados via um botão presente em baixo de ecrã, mas é possível de definir, para todo ou parte dos parâmetros de tipo datas ligadas à tarefa, de regras de cálculo fazendo nomeadamente intervir o periodo corrente e das deslocações de tempos.

Gestão do ecrã

[Tab] Ecrã de registo

Regista-se aqui à vez as condições de lançamento (identificação do dossier e do utilizador sob identificação da qual se lança a assinatura) e os horários afetados.

[Block number] 1

  • Código assinatura ([field] CODABT)

Identifica o código assinatura.

  • Descr. ([field] NOMABT)

Título associado ao código precedente

Características

  • Dossier ([field] DOSSIER)

Define o nome do dossier no qual será lançado o pedido (o código do dossier corrente está proposto por defeito).

  • Código utilizador ([field] USER)

Define o código de um utilizador sob identidade da qual a tarefa será lançada (o código do utilizador corrente está proposta por defeito).
  • Palavra-chave ([field] PASSE)

Quando o dossier ou a tarefa deve ser executado não o é pelo dossier corrente, ou se o utilizador não corresponde ao código utilizador corrrente, é necessário de registar a password correspondente afim de se identificar.
  • Grupo ([field] GRP)

Define um grupo de tarefas a lançar.
  • Cód. tarefa ([field] TACHE)

Define o código da tarefa que será lançada automaticamente pelo servidor batch.

  • Língua ([field] LAN)

[Block number] 3

  • Activo ([field] ENAFLG)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.
  • Última execução ([field] DJOUR)

Quando a assinatura já deu lugar a execuções de tarefas, este campo permite conhecer a data da última execução.

Períodicidade

  • Períodicidade ([field] PERIO)

Este campo define a periodicidade de execução da tarefa.

Se o período é o mês, pode-se indicar uma quantia (data no mês compreendido entre o 1º e o 31º) e/ou assinalar a check box fim de mês. Se o período é hebdomadário, falta assinalar vários dias da semana.
  • Dias excluídos ([field] CAL)

Este código calendário permite excluir certos dias pela execução de tarefas abonadas.

Semanal

  • [field] JOUR

Indica-se aqui os dias da semana retidos para a execução de uma tarefa hebdomadária.

Mensal

  • Dias certos ([field] QUANT)

Pode-se indicar, no caso onde a assinatura está definida como mensal, de uma a cinco quantias (mais eventualmente o fim de mês pela check box correspondente) para definir os dias do mês onde o pedido deverá ser lançado.
  • Fim de mês ([field] FDM)

Quando a assinatura é mensal, esta check box permite de desencadear a execução da tarefa o último dia de cada mês (se esta data não foi excluida por através do código de calendário).

Zona horária

  • Hora início ([field] HDEB)

Permite de limitar entre duas horas a execução de uma tarefa definida em frequência.
  • Hora fim ([field] HFIN)
  • Frequência (mn) ([field] FRQ)

Quando uma tarefa deve ser executada de maneira repetitiva, define-se uma hora de início, de fim, e uma frequência de execução em minutos. Esta frequência permite de relançar automaticamente a tarefa.
  • Um único pedido ([field] ONE)

Esta check box pode ser assinalada se a assinatura está definida em frequência.

Se ela está assinalada, um único pedido está lançado por dia para executar o tratamento pedido, e desde que o tratamento está terminado, a tarefa se coloca em observação durante o número de minutos definidos por frequência, para retomar em seguida a sua execução, isto até que a hora de fim seja ultrapassada. O pedido está então afixado no seu estado Em curso durante todo o intervalo de execução.

Garante-se assim que o pedido é sempre presente em memória uma vez que será lançado, o que pode ser no detrimento de outras tarefas se o número máximo de tarefas lançadas simultaneamente está atingido.
  • Depuração ([field] EPUR)

Esta check box não pode ser assinada que para uma assinatura em frequência. Quando é o caso, não se guarda o registo das execuções sucessivas da tarefa na função de gestão dos pedidos. Apenas o pedido em curso e o precedente pedido são guardados na tabela correspondente.
  • Continuar se erro ([field] CNTERR)

Se esta check box está assinalada, uma tarefa "frequência" ou uma tarefa de um grupo será relançada mesmo em caso de erro.
Os erros que podem interromper uma tarefa :
- variável GOK diferente de 1 : indica geralmente que existe um erro numa transacção de actualização da base.
- variável GERRBATCH inferior a 100 : esta variável pode ser posicionada no tratamento que executa a tarefa.
- variável GERREUR diferente de 0 : em certos tratamentos, esta variável está posicionada num sub-programa de gestão de erro desencadeado pela instrução "Onerrgo".

Horas fixas

  • Hora ([field] HEURE)

Quando a tarefa não foi definido em frequência, é possível de lançar a três horas diferentes numa jornada indicada aqui.
  • Execução forçada ([field] FORCE)

Este indicador não pode ser assinalado que se dão as horas fixas de execução para uma assinatura. Garante a criação do pedido de execução mesmo se a hora foi ultrapassada no momento onde o servidor batch trata as assinaturas do diário.

Por exemplo, se uma tarefa está programada a 7 horas, 10 horas e 15 horas, mas se o servidor batch arranca uma manhã às 8 horas :

  • se a check box está assinalada, 3 pedidos de execução (a 7 horas, 10 horas e 15 horas) serão criados. O pedido de execução datado de 7 horas será executado ou não segundo os parâmetros do servidor (pode-se fixar um atraso de execução máximo nos parametros do servidor) Mas em todo o estado de causa, uma linha será programada para 7 horas.
  • se a check box não foi assinalada, apenas os dois pedidos de execução posteriores à hora corrente serão criados.

[Grid] Data relativa

  • Data zona ([field] DATZON)

Permite preencher o nome da zona ecrã a inicializar.
É possível especificar a abreviação do ecrã se a caixa de registo dos parâmetros da tarefa utiliza vários ecrãs.
  • Data de base ([field] DATDEP)

Permite de preencher a data de referência para o cálculo.

  • Incremento ([field] DATNBR)

Número a reacrescentar (ou a retrair) à data de referência.
Este número depende da unidade utilizada (dias, semanas, meses)

  • Unid. de tempo ([field] DATJRS)

Unidade de tempos :

  • Formula ([field] DATFRM)

Permite preencher uma fórmula que será aplicada no ambiente do registo dos parâmetros da assinatura da tarefa batch: tabelas abertas, variáveis globais, variáveis utilizadores, etc...

Botões específicos

Parâmetros

permite registar os parâmetros de lançamento da tarefa. Quando se lança um grupo de tarefas, uma janela intermediária se abre, para permitir de escolher a tarefa cujo se quer registar os parâmetros.

Barre de menus

Opções / Relançar

permite relaçar a assinatura se os horários definidos na assinatura incluem a hora corrente, e que a tarefa não está já em vias de voltar.

Funcionamento técnico das assinaturas

De referir que no arranque do servidor batch (ou após a passagem de minuto quando o servidor batch torna incessantemente), todas as assinaturas de uma diária são criadas sob a forma de pedidos de execução de tarefas visíveis no gestor das tarefas batch.

Para as assinaturas em frequência, um único pedido de execução está criado; no fim de execução da tarefa, um novo pedido de execução está criado.

Isto significa que uma vez a primeira tarefa de uma assinatura em frequência lançada :

  • Na gestão dos pedidos, toda a interrupção (se ela volta), ou toda a supressão (se ela está em espera) da terefa correspondnete interrompe a repitição desta tarefa, que não será reavisada que na madrugada.
  • toda a modificação de frequência não será tida em conta antes a madrugada por defeito.

Se se deseja modificar as características de uma tarefa depois uma assinatura em frequência, falta começar por suprimir a próxima tarefa ou interromper se ela está em curso, depois modificar as características de assinatura. Poder-se-á em seguida a assinatura com ajuda da função acessível depois a barra de menus ou por clique direito sobre a linha na função de gestão dos pedidos.
