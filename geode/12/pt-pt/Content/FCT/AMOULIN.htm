Monitor de encadeamento

Esta função não pode ser utilizada que no dossier supervisor, e vai permitir de definir e seguir a execução do conjunto dos procedimentos de migração de um dado dossier

Para este facto, vai-se poder aqui criar um plano de migração identificada por um código. O código a utilizar está livre, mas se deseja personalizar as condições de migração de um dado dossier, dever-se-á criar um plano de migração tendo o mesmo código que aquele do dossier a migrar, e de colocar em espera.

Se não é o facto, no momento do lançamento da migração de um dossier dado (pela função de revalidação do dossier), um plano de migração chamado MIGmmdd## (mm e dd sendo os números do mês e do dia de lançando,## sendo um numero sequencial) está automaticamente criado e lançado.

A criação de um plano de migração vai duplicar todos os procedimentos definidos na tabela dos procedimentos numa tabela de sequênciamente cujas linhas são apresentados neste ecrã

Poder-se-á em seguida lançar e controlar manualmente o encadeamento das operações, as interromper ou as relançar, verificar o rasto obtido.

Gestão do ecrã

  • Cabeçalho
  • [Tab] Monitor
  • [Tab] Opções
  • [Tab] Lista dos planos

O ecrã da função apresenta um cabeçalho com as informações globais ligadas ao plano de execução corrente, um primero separador que permite ver o detalhe das operações, e um segundo separador que permite conhecer a lista dos outros planos de migração com a lista global.

Cabeçalho

O cabeçalho permite definir os parâmetro gerais que definem a maneira cujo plano de migração deve se executar, o estatuto corrente quando a migração está em curso, e um conjunto de botões de controlo.

Identificação

  • Cód. plano ([field] PLAN)

Este campo identifica o código do plano de migração (é o código do dossier por defeito)
  • Descr. ([field] INTIT)

Permite de definir um titulo associado a cada ficha.
  • Dossier ([field] DOSSIER)

Define o código do dossier sobre o qual se deseja trabalhar (tal que está definido na tabela dos dossiers).

  • Nr. de lançamentos paralelos ([field] NBPARL)

Este valor define o número máximo de tarefas de migração susceptíveis de serem lançados em paralelo. Este número não pode ultrapassar o número de tarefas batch autorizadas. Não pode mais, a um momento dado da migração, ultrapassar o número de tarefas da fase corrente da etapa em curso
  • Arranque auto das fases ([field] PHASEAUTO)

Quando esta check box está assinalada, existe encadeamento automático das fases : Quando todos os procedimentos de uma fase de migração são terminados, a fase seguinte está lançada automaticamente. Se este indicador não está assinalado, o utilizador deverá utilizar o botão correspondente para relançar a execução depois o plano de trabalho das migrações

Esta check box assinalada por defeito se o plano foi criado automaticamente por validação de dossier
  • Arranque auto pós-migração ([field] POSTAUTO)

Quando esta check box está assinalada, os procedimentos de etapa de pós-migração são automaticamente lançados uma vez que o procedimento das fases e etaps precedentes são terminadas (não é o caso, falta os lançar manualemnte depois o plano de trabalho).

Esta check box assinalada por defeito se o plano foi criado automaticamente por validação de dossier

Monitor

  • Status global ([field] STATUT)

Este indicador dá a lista global do plano de migração, que pode tomar os valores seguintes :

  • Em espera. está pronto a ser lançado.
  • Em curso : ao menos um procedimento de migração está sempre em curso.
  • Terminado: todos os procedimentos do plano de migração são terminados com sucesso.
  • Terminado com erros : pelo menos um dos procedimentos de migração venceu. Os procedimentos das fases seguintes não têm podido se lançar. Falta na véspera remediar o erro (frequente ligado aos dados tratados no procedimento tendo vencido), depois relaçar o procedimento em questão para poder continuar a migração.
  • Interrompido : O utilizador lançou a migração , depois a interrompe : falta utilizar o botão Retoma para que a migração retoma o seu curso.
  • Em espera de interrupção : O utilizador pediu a interrupção da migração, mas certos procedimentos não terminaram o tratamento do pacote de dados em curso antes de se interromper.
  • Em espera paragem. o utilizador pediu uma paragem, mas esta não está ainda efectiva
  • Lançada : a migração está lançada, mas nenhuma tarefa não foi ainda arrancada.
  • Data lançamento ([field] DATEL)

Este campo afixado permite conhecer a data na qual a migração foi lançada.
  • Hora lançamento ([field] HEUL)

Este campo afixado permite conhecer a hora na qual a migração foi lançada.
  • Tempo global (HHH:mm) ([field] ELAPSEDG)

Este campo indica o tempo total passado depois o início da migração. Este tempo corresponde à diferença entre :

  • a data e hora de fim se a execução do plano está terminado,
  • a data e hora corrente se a execução do plano não foi terminado

e a data e hora de partida.

Dito de outra forma, se não se trata de um tempo de migração acumulada tendo em conta os tempos de execução reais.

Controlo

  • Lançar/Retomar ([field] ACTIV)

Este botão permite lançar a migração, ou de a retomar se ela está interrompida previamente.
  • Interromper ([field] INTERR)

Este botão permite interromper o conjunto dos procedimentos de migração em curso de execução. A interrupção de uma tarefa não está forçosamente imediata. Com efeito, uma tarefa trata as linhas na base de dados por grupos (o número de linhas tratadas aparece no ecrã). Quando este botão está utilizado, uma ordem de interrupção foi enviada à tarefa, e aquela aqui termina o grupo de linhas em curso antes de parar.

Será assim possível retomar a execução do grupo de dados seguintes.
  • Parar os lançamentos ([field] ARRET)

Este botão permite de parar os lançamentos de procedimentos de migração. Isto significa que as tarefas batch foram já lançadas vão se terminar, mas que nenhuma tarefa ligada a um procedimento de migração do plano não será lançado.
  • Refrescar ([field] RAFR)

Este botão permite refrescar o quadro que dá a indicação dos procedimentos em curso, do seu estado,e do número de linhas tratadas.
  • Bloq. tudo ([field] BLOQUER)

Este botão permite bloquear todos os procedimentos em cursos. Um procedimento bloqueado não será lançado, mesmo se ele poderá o ser porque as condições (terminação das etapas e fases anteriores) o permitem Um procedimento bloqueado pode estar desbloquado individualmente por clique direito sobre a linha.
  • Desbloq. tudo ([field] DEBLOQUER)

Este botão permite desbloquear todos os procedimentos bloqueados Um desbloqueio individual de uma tarefa está também possível por clique direito sobre a linha.

A fase não pode ser modificada por um procedimento standard.

Retomar

Permite retomar a execução de um procedimento previamente parado. A retoma se faz a partir do primeiro dado ainda não tratado pelo procedimento.

Interromper

Permite de interromper a execução de um procedimento de migração.

Rastro

Permite visualizar o rasto de execução de um procedimento

Relançar

Permite relançar a execução de um procedimento (i. é de retomar a execução depois o início, em repartindo de uma tabela destino vazio e em retomando o conjunto dos dados a transcodificar)

Bloquear

Permite bloquear um procedimento de migração que não foi ainda lançada, quer dizer de interdir de execução, mesmo se as condições de sequênciamento permitirão o seu lançamento. Isto tem por consequência de não permitir a execução dos procedimentos se encontrando nas etapas ou fases posterores, pois que a tarefa bloqueada deva ter sido desbloqueado, depois executado, para que as condições de encadeamento permitindo continuar.

Desbloquear

Permite desbloquear um procedimento de migração previamente bloqueado.

Desligar

Permite, quando um procedimento se termina com os erros, de encadear ainda os procedimentos de fase ou etapas superiores.

Atenção! É recomendado de não utilizar esta desengatagem numa migração real sem ter bem medido os riscos em curso. Com efeito, desengatar um procedimento pode criar os efeitos de bordo sobre os procedimentos seguintes, e no final a conduzir a um dossier migrado.

Botões específicos

Rasto global

Permite visualizar o rasto global por agregação de todos os procedimentos de migração. Este rasto está unicamente vísível se a migração foi lançada em batch (normalmente pela validação de dossier).

Este rasto global está actualizado no fim de cada procedimento de migração, em acumulando as linhas de rasto do procedimento correspondente. Assim, os procedimentos aparecem na ordem de terminação. Se um procedimento está reactivado várias vezes (na seguida de uma tomada em controlo manual do plano), o rasto apresentará várias vezes o detalhe do procedimento, mas com uma linha "lançamento inicial" no momento do primeiro lançamento, e "reactivação N" no caso de um Nésimo reaviso.

Permite de fazer zoom sobre a gestão dos pedidos batch.
