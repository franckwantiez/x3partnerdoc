  • Depositante ([field] DEP)

Código que representa o recipiente Em termos logístico, a noção de recipiente permite tratar os fluxos ou tráfegos distintos no mesmo estabelecimento. Ela recupera a noção de cliente dador de ordens para os prestadores de servíços.

Após o estabelecimento, o recipiente é a chave secundária de acesso aos ficheiros Artigo,Entrada e Ordem de Entrega. Para o acondicionamento, o código recipiente está prenchido se o acondicionamento está efectuado ao recipiente ou ao artigo, e se o acondicionamento é mono recipiente artigo e que não está vazio.

Data Limite de Venda (DLV): A DLV é geralmente utilizada

como Data FIFO para os produtos alimentares.

Data de Limite de conservação (DLC): Esta data é também chamada Data

Limite de Utilização Óptima. Ela figura geralmente sobre

o embalamento dos produtos alimentares ou farmacêuticos.
  • DLC ([field] SND)
  • D.Fifo ([field] FIFDAT)

A Data FIFO pode determinar a ordem de destocagem dos produtos (en

função da parametrização) no momento do lançamento de uma preparação de

encomenda. Ela permite igualmente de verificar a possibilidade de efectuar.

certas transacções, como a compactação.

(quarentena por exemplo).
