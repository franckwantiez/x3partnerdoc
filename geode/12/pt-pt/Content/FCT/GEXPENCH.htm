Encadeamentos export

Esta função permite de extrair os dados da base depois versus os ficheiros de tipo ascii segundo as estruturas descritas por um conjunto de modelos de importação/exportação identificados por um código encadeamento. Esta exportação pode ser total ou cronológica, quer dizer em não exportando que os dados modificados depois a última exportação deste tipo.

 Os dados importados, se eles estão traduzidos, o serão na língua de conexão do dossier .

Gestão do ecrã

[Tab] Ecrã de registo

O ecrã que se abre permite de registar os parâmetros relativos à importação pedida. Este facto, o procedimento de exportação está lançada.

Quando ela está finita, uma janela se abre, afixando num ficheiro de rasto um resumo da operação (em princípio, tem uma linha por ficheiro, com uma mensagem tal que NNN linhas exportadas, e uma linha no momento da transferência do ficheiro versus o directório definitivo quando um directório temporário está utilizado). Em caso de erro, uma caixa de erro abre-se antes a afixação do ficheiro de rasto.

[Block number] 1

  • Cód. encadeamento ([field] CODE)

Trata-se do código definindo à vez na lista dos modelos a utilizar e o caminho dos ficheiros a considerar no momento da operação de importação ou de exportação. Este caminho não pode ser modificado, é definido em cada modelo de importação/exportação, seja sob forma de um caminho absoluto, seja sob a forma de um caminho relativo (neste caso, será procurado nos directórios definidos nos parâmetros importação/exportação).

[Block number] 2

  • Gestão cronologia ([field] CHRONO)

Se esta check box está activada, a exportação faz-se unicamente sobre os registos modificados depois a última exportação. O princípio de funcionamento deste modo de exportação é o seguinte:

  • um contador sage (variável numérica comum a um momento dado para todos os postos de um dossier, guardado na base de dados, cujo nome é [C]EXPORT), constitui uma base de tempos (ele se incrementa em cada vez que uma exportação com gestão de tempo.
  • O valor corrente do contador [C]EXPORT é reportado nos registos da base de dados quando uma criação ou uma atualização é feita. Este mecanismo é válido para todas as tabelas permitindo realizar uma exportação cronológica (todas as tabelas permanentes e todos os cabeçalhos de movimento incluido este campo : isto representa perto de metade das tabelas da base de dados). O campo correspondente se chama EXPNUM.
  • No momento da exportação cronológica, guarda-se o valor corrente do contador EXPORT nos modelos de importação/exportação afetados, depois se incrementa, e não se exporta que os registos da base para a qual o valor do campo EXPNUM é inferior ou igual ao valor do campo EXPORT antes de incrementação, e superior estrictamente no último valor previamente guardado em cada um dos modelos. O contador EXPORT não estando incrementado que uma vez, o conjunto dos dados exportados em todos os ficheiros definido pelo encadeamente está coerente do ponto de vista temporal.
  • Quando se faz um encadeamento de exportação com uma gestão de tempo, é possivel de utilizar o metacaracter # no nome dos ficheiros a exportar ; este caracter está então preenchido pelo valor corrente do contador EXPORT formatado sobre 4 caracteres. Assim, o nome de ficheiro transfert#.asc será substituido por transfert0001.asc, transfert0002.asc, etc. … segundo o valor do contador.
