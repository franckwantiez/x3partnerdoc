Vistas do portal

A vista é o elemento unitário componete uma página de portal. Ela se compõe de dois elementos, registados cada um sobre os separadores da funçao :

  • Uma fonte de dados que define um processo de extracção de informações a partir do referêncial Sage X3. Uma fonte de dados reenvia os valores organizados em linhas, cada linha contendo as colunas (o número de colunas depende da fonte de dados, o número de linhas depende dos dados efectivamnte extraídos quando se interroga a fonte e que se lê a base)
  • Um componente visual, que descreve a maneira cuja vai ser apresentada os dados extraidos.

A associação destes dois elementos se acompanha da definição de uma série de parâmetros que são registados nos quadros de cada separador.

Cabeçalho

No cabeçalho, regista-se o código, que identifica uma vista de portal, e um intitulado.

  • Cód. ([field] APVCOD)

Este código identifica de maneira única a ficha em curso
  • [field] APVINTIT

[Tab] Fonte dos dados

Este separador permite definir a fonte de dados, quer dizer a origem das informações que vão ser apresentadas.

[Block number] 1

  • Tipo ([field] SRCCODTYP)

Permite definir um tipo de fonte de dados em função do tipo, registar-se-á, se necessário, um código complementar identificando a fonte ela mesmo, e uma série de parãmetros complementares definidos no quadro do baixo de separador.
  • Cód. ([field] SRCCOD)

Identifica a fonte de dados. Em função do tipo registado previamente, este campo pode ser :

  • um código de multi-lista (tipo de fonte MLT).
  • o código de um pedido, ou de um pedido SQL (tipo de fonte REQ).
  • O código de uma estatística (fonte de dados STA). 
  • O código de um tratamento (fonte de dados TRT). 
  • vazio (caso dos tipos de fontes URL e AGD, que não têm necessidade de uma lista de parãmetros no quadro que segue).
  • [field] WDES

[Grid] Source dados (DATASOURCE)

  • Parâmetro ([field] SRCPARCOD)

Define o código do parãmetro necessário à fonte de dados
  • Desig. parâmetro ([field] SRCPARDES)

Título associado ao código precedente

  • Formula ([field] SRCPARFOR)

Permite registar, sob a forma de uma fórmula avaliada, o valor do parãmetro.

Deve-se referir que :

  • esta fórmula pode ser alfanumérica, numérica, ou de tipo data segundo os casos, mas que se o tipo de expressão não corresponde ao que é esperado, existe conversão.
  • em certos casos, os valores constantes podem ser dados, e selecionados pelo clique direito.
  • quando vários valores podem ser dados (por exemplo dos números de colunas), exprime-se sob a forma de uma cadeia de caracteres contendo uma lista de valores numéricos separados por pontos e vírgulas. A seleção sucessiva destas colunas pode em geral ser feito por clique direito, a lista de valores se completando então.
  • Descr. ([field] SRCPARLIB)

Título associado ao código precedente

[Tab] Componente visual

Este separador permite definir o formato de apresentação da informação (sob forma de quadro, de gráfico, de agenda...). Este formato, identificado por um componnete visual e os parâmetros que lhe estão anexados, depende da fonte de dados (certos componentes visuais não funcionam que com certas fontes de dados).

Os componentes fornecidos e constrangimentos correspondentes são definidos no anexo a seguir.

  • Componen. visual ([field] CMPCODTYP)

  • Descr. ([field] CMPPARDES)
  • Formula ([field] CMPPARFOR)

Lista das fontes de dados standard

  • AGD (Agenda)
  • MLT (Multi-lista)
  • REQ (Pedido)
  • STA (Estatística)
  • TRT (Tratamento)
  • WEB (Página Web)

Os tipos de fontes disponíveis em standard são descritos a seguir

AGD (Agenda)

Este tipo de fonte corrresponde à agenda CRM.

Nenhum código lhe está associado.

Os parâmetros que devem ser registados permitindo de definir uma janela de datas de consulta :

  • O número de dias antes (a data corrente).
  • O Número de dias após (a data corrente).

Todos os dois são de expressões numéricas.

MLT (Multi-lista)

Este tipo de fontes de dados corresponde a um conjunto de listas de selecção (tais que elas são visualizadas na lista de esquerda de um objecto)

Um código de multi-lista lhe está associado.

Não existe outro parâmetro associado.

REQ (Pedido)

Este tipo de fontes de dados correspondente a um pedido, ou a um pedido SQL, identificada pelo seu código. Dois parâmetros complementares são então definidos :

  • o primeiro, nomeado FRQ, definido na frequência de extracção. O valor, que pode ser seleccionado numa lista, vai de 1 (a cada utilização) a 6 (a primeira vez), em passando por níveis intermédios tais que 3 (todos os dias). No momento de utilização da fonte de dados, a data de último cálculo do pedido é lido para saber se existe lugar de recalcular antes a afixação, em função do valor do parâmetro.
  • O segundo, nomeado INITLEVEL, é útil para os pedidos não definidos em SQL. Permite definir o nível de detalhe ao qual se coloca para afixação os resultados do pedido, quando este utiliza as rupturas (coluna Grupo do requerente). Ao nível 1, se afixa que as rubricas cujo nível (indicado na coluna nível) está igual a 1, e as colunas acumuladas correspondentes (os outros são enviados vazios). Ao nível 2, dá-se o detalhe das rubricas de nível 1 e 2, e assim de seguida.

STA (Estatística)

Este tipo de fonte corrresponde à afixagem de uma estatística, cujo código é dado. Os parâmetros seguintes devem ser dados :

  • a sociedade (CPY) e o estabeleimento (FCY) sobre as quais a interrogação está feita (se uma cadeia está dada, tomar-se-ão todas) Uma selecção é possível sobre estes parâmetros.
  • as datas de início e de fim para os dois intervalos no tempo possível (DATDEB1 e DATDEB2, DATFIN1 e DATFIN2)
  • o valor dos critérios (de 1 a 8 máximo, nomeados CRIT1 a CRIT8). O número de critérios para os quais um valor diferentes da cadeia vazia é dada definido a profundidade na qual a estatística está afixada.

TRT (Tratamento)

Este tipo de fonte corresponde à execução de um tratamento normalizado, cujo nome é dado. Os parâmetros dependendo do tratamento podem ser pedidos (por defeito, se se trata de 4 parâmetros banalizados nomeados P1 a P4). O tratamnto SUBAPSTRT, e o tratamento SUBAPSTRTI documentam a maneira de criar uma tal fonte de dados.

WEB (Página Web)

Este tipo de fonte corresponde a afixação de uma página Web cujo endereço foi avaliado a partir do parâmetro URL que deve ser definido.

Lista dos componentes visuas disponíveis em standard

  • FAG (Agenda flash)
  • FCA (Calendário flash)
  • FGR (Gráficos flash)
  • ML1 (Multi-lista standard)
  • RE1 (Pedido standard)
  • ST1 (Pedido standard)
  • TB1 (Quadro standard)

Os componentes visuais disponíveis em standard são descritos a seguir.

FAG (Agenda flash)

Este componente visual não pode ser associado que na agenda CRM (fonte AGD). Apresenta sob a forma de um calendário os campos emitidos da agenda, com as interacções ligadas às especificidades de agenda CRM. Os parâmetros associados são :

  •  O tipo de agenda (FCCTYPE), cujos valores podem ser obtidos pela selecão. Os valores possíveis são "XMONTH" (agenda mensal), "XWEEK" (agenda semanal), "XDAY" (agenda diária).
  • o número de coluna dos campos Tipo de acção, Crono acção, Data de início, Hora de início, Data de fim, Hora de fim, Terceiros, Interlocutor, Categoria,Objecto,Flag efectuada. Estes campos estão apresentados na ordem exacta da fonte de dados AGD ; não existe então local em geral de modificar as colunas por defeito (de 1 a N) apresentados em face dos parâmetros, excepto se se deseja mencionar certas informações, ao qual se colocará o valor 0 em face da coluna. Pode ser o caso por exemplo se a data de início é igual à data de fim, ou se a noção de hora não foi utilizada.
  • O indicador que define se o selector de calendário utilizado para passar de uma e outra destas representações está presente (2=Sim                 , 1=Não).

FCA (Calendário flash)

Este componente visual permite apresentar os eventos datados sob forma de um calendário visual. Não é utilizável que com as fontes de dados seguintes : tratamento, pedido.

Os parâmetros associados são :

  • O tipo de calendário (FCCTYPE), cujos valores podem ser obtidos pela selecção. Os valores possíveis são "XMONTH" (calendário mensal), "XWEEK" (calendário semanal), "XDAY" (calendário diário).
  • O indicador que define se o selector de calendário utilizado para passar de uma e outra destas representações está presente (2=Sim                 , 1=Não).
  • o número de colunas da fonte de dados indicando respectivamente data e hora de início, depois data e hora de fim. Fora a data de início, que permite posicionar o dado sobre o calendário, estas informações podem ser omissas. A selecção directa dos campos via clique direito é possível se a fonte de dados associado é um pedido.
  • o número de coluna da fonte de dados indicando o ícone e a cor utilizadas para simbolizar o evento (campos opcionais).

FGR (Gráficos flash)

Este componente visual pode ser associado às estatísticas (fonte de dados STA), aos pedidos (fonte de dados REQ), ou aos tratamentos (fonte de dados TRT). Apresenta um dos montantes emitidos da fonte de dados sob a forma de uma representação gráfica, com como título um dos elementos caracteres emitidos da fonte de dados. Os parâmetros associados são :

  • o tipo de gráfico (FGGTYPE) que pode ser seleccionado por clique direito na lista seguinte : "XLINE" (gráficos em linhas), "XBAR" (gráfico em barras), "XAREA" (gráfico em áreas), "XPIE" (gráfico em sectores).
  • os números das colunas do campo Valor (FCGVAL) afixáveis. Se existe uma só coluna, dá-se directamente o seu número; se existem vários, devem estar numa cadeia de caracteres, separados por ponto-e-vírgula. Obter-se-á assim, por exemplo, uma sintaxe de tipo "4;7;3;8"
  • o número de coluna correspondente ao valor afixado por defeito (FGCDEF). Se este campo não está registado, o primeiro valor é utilizado.
  • O número de coluna do campo intitulado (FGCLABEL) afixado. De referir que com uma fonte STA, o primeiro montante e a coluna número 3, a coluna número 1 sendo o código guardado na tabela de agregação estatística e a coluna 2 sendo o intitulado extraído da tabela ligada a este código (apenas estas duas colunas podem ser utilizadas para registar FGCLABEL, poisque todas as outras colunas são numéricas neste caso).
  • os indicadores que definem se o selector de gráfico (FGCSELG) utilizado para passar de um a outro as representações, e o selector de valor (FGCSELC) para apresentar uma das colunas numéricas presentes na lista das colunas possíveis, são presentes (2=Sim                 , 1=Não).

ML1 (Multi-lista standard)

Este componente visual permite de apresentar os dados sob forma de listas de valores (cada uma das listas sendo identificadas por uma faixa, o facto de clicar sobre a faixa desencadeia a afixação do conteúdo da lista). Não é utilizável que para a fonte de dados de tipo MLT (multi-lista), e nenhum parâmetro.

Na utilização deste componente, notar-se-á que os campos sublinhados, que correspondem à chave principal das fichas listadas, permitindo, por simples clique, de entrar sobre a ficha correspondente em túnel.

RE1 (Pedido standard)

Este componente visual permite apresentar os dados sob forma de um quadro de valores. Não é utilizável que para a fonte de dados de tipo REQ (pedido), e nenhum parâmetro.

Contrariamente ao componente TB1 que é mais genérico, autoriza um desdobramento dos níveis sucessíveis do pedido a partir do qual está apresentado (parâmetro INITLEVEL), por apoio sobre um ícone presente sobre a faixa do título do quadro.

A utilização deste componente, notar-se-á que certos campos podem estar sublinhados. Neste caso, trata-se das chaves principais ligadas a tabelas em linha no pedido. Um simples clique sobre o campo permite então de entrar sobre a ficha correspondente em túnel.

ST1 (Pedido standard)

Este componente visual permite apresentar os dados sob forma de um quadro de valores. Não é utilizável que para a fonte de dados de tipo STA (estatística), e não tem nenhum parâmetro.

Contrariamente ao componente TB1 que é mais genérico, autoriza um zoom sobre uma das linhas do pedido a partir do nível no qual ela está apresentada (parâmetro INITLEVEL), pelo apoio sobre um ícone presente sobre a faixa do título do quadro.

A utilização deste componente, notar-se-á que certos campos podem estar sublinhados. Neste caso, tratam-se das chaves principais ligadas a tabelas em linha na estatística. Um simples clique sobre o campo permite então de entrar sobre a ficha correspondente em túnel.

TB1 (Quadro standard)

Este componente visual permite apresentar os dados sob forma de um quadro. É utilizável com as fontes de dados de tipo AGD (agenda), REQ (pedido), STA (estatistica), ou TRT (tratamento), e não tem nenhum parâmetro.

Contrariamente aos componentes ST1 e RE1, não permite nenhum zoom ou desdobrar. É bem adaptado quando se deseja apresentar os dados numa lista a um dado nível sem permitir visualizar um sub-detalhe, mesmo se ele existe.
