Listas

Esta função permite capitalizar o conjunto das listas Crystal Report utilizáveis depois um dos produtos da gama X3. Neste dicionário das listas, regista-se das listas, regista-se :

  • As características gerais
  • As características da saida
  • Os tratamentos eventualmente associados
  • Os direitos de acesso
  • Os parãmetros de lançamento.
  • As fontes de dados, quando a lista de acesso tem tabelas de um outro dossier.

Uma carta para o desenvolvimento das listas Crystal Report foi definida nos documentos juntos

Cabeçalho

O cabeçalho permite identificar a lista, e de dar as suas características gerais.

  • Cód. lista ([field] RPTCOD)

Código identificando a ficha corrente.

  • Descr. ([field] ZRPTDES)

Destinado a figurar as listas e os ecrãs nos quais o código da ficha pode ser registado ou selecionado. Este texto permite de dar uma descrição da ficha afetada.

[Tab] Geral

Encontram-se as informações ligadas à gestão global da lista.

Generalidades

  • Activo ([field] ENAFLG)

Esta check box a assinalar permite ativar ou de desativar a ficha corrente sem portanto perder o seu conteúdo.

Uma ficha desativada não pode ser utilizada (por chamada do seu código) em outras fichas (documentos, parametrizações...), ou no tratamento de massa.

As habilitações sobre uma função dada podem interdir a criação de uma ficha ativa. Neste caso, a check box está desativada por defeito, e está modificável unicamente por um utilizador autorizado, ou via um circuito de assinatura definido por Workflow.
  • Cód. activ. ([field] CODACT)

Um código actividade permite :

  • de tornar opcional um elemento do dicionário se o valor associado ao código actividade é nulo.
  • de assinar os elementos específicos desde logo são marcados por um código começando por X,Y ou Z.
  • de dimensionar um número de linhas máximas quando o código de actividade maerca os elementos de um quadro.

Assim, se o código actividade está não activo, o elemento marcado não será utilizável, e o código associado (se existe um) não será gerado nem activável.
  • Desc. curta ([field] ZRPTSHO)

Este intitulado permite dar uma designação à ficha.
é utilizado em certos ecrãs ou listas quando não se teve suficiente local para visualizar o intitulado longo.
  • Módulo ([field] MODULE)

Módulo para o qual está utilizada a lista.
  • Grupo ([field] GRP)

Trata-se do grupo de impressão à qual, se associa a lista. Graças a esta informação, poder-se-á lançar a lista depois o menu de impressão com o sub-menu RPTxx (onde xx é o nº do elemento do menu local 97 para o campo "GROUPE"). A atualização desta informação não necessita de proteção por código atividade.

Grupo sobre o qual se dá um direito de acesso na gestão dos utilizadores.

Língua

  • Multi-línguas ([field] MULLAN)

Se esta check box está assinalada, isso indica que a lista foi gerada em todas as línguas geradas pelo dossier, senão a lista está unicamente na linha de conexão.
  • Língua origem ([field] LAN)

Este campo permite reservar a lista a uma língua; língua na qual foi concebida a lista.

Características da saída

  • Tipo ([field] PRTNAT)

Ao lançamento de impressão, o destino está determinado :

  • por fórmula completamente, se este existe
  • senão, pelo destino
  • ou enfim, se os 2 primeiros elementos não estão registados, pelo tipo

O tipo serve a determinar um destino no caso ou a fórmula complemento e o destino não estão registados. No lançamento de impressão, o supervisor determina o destino para um dos quatros parãmetros associado ao tipo. Em cada valor do tipo corresponde um parãmetro, no qual se regista um destino :

  • Normal: parãmetro PRT1
  • Fax :             parâmetro PRT2
  • Termico : parãmetro PRT3
  • Cor: parãmetro PRT4

Observação : Os valores deste menu local 22 são modificáveis.
  • Impressora ZPL ([field] GESZPL)

Indica que esta lista pode ser imprimida sobre uma impressora ZPL

  • Cod. parametriz. ([field] CODZPL)

Este código permite de identificar a parametrização de uma impressora ZPL É necessário para que esta lista possa ser imprimida sobre este tipo de impressora.

  • Destino ([field] PRTDEF)

Indica uma ficha destino. Esta ficha contém as características de impressão.

  • Obrigatório ([field] PRTOBL)

Se esta check box está assinalada, não se pode modificar o destino ao lançamento de impressão de lista.
  • Impressões ligadas ([field] IMPLIE)

O servidor de edição é multi-processos, pode tratar em paralelo N pedidos. Por outro lado, uma impressora dada não pode imprimir que uma única lista à vez.

O que quer dizer que se duas listas são submetidas, no mesmo instante sobre o mesmo servidor de edição, com destino da mesma impressora, toda a fase de tratamento Crystal Report listas (pedidos SQL, colocação em forma, etc...) está paralelizada, apenas a fase de produção "papel" está serializada.

Esta opção permite respeitar a ordem de definição das listas na ficha dicionário ou em caso de utilização de um parãmetro de segmentação.
  • Fórmula complemento ([field] PRTFRM)

Preenchido ou não, estes campos servem à pesquisa de um destino por utilizador e por lista. Este destino, se está encontrado, será prioritário por relação aquele indicado a seguir.

No momento de impressão, no algoritmo de afetação de impressora, esta zona será comparada ao complemento definido na função de afetação das impressoras pelo utilizador. Exemplo: Se para a lista X, definiu-se um compleemnto com o valor "PAR" correspondente à impressora Y e que a fórmula de lista X contém a expressão"PARAM(estabelecimento)", então, no momento de impimir a lista, se a parametrização "estabelecimento" está igual a PAR,a impressora será inicializada a Y.

Scripts

  • Tratamento standard ([field] TRTINI)

Tratamentos nos quais, será possível de escrever sub-programas complementares à impressão desta lista. São executados pelo procedimento de impressão justo antes de executar a lista Crystal Reports. Permitem inicializar as variáveis definidas na parametrização da lista ou eventualmente de preparar os ficheiros antes impressão, ou mesmo de atualizar os campos na base de dados (ex flag de edição). Estes tratamentos de inicialização são executados para cada lista.

Um tratamento está reservado ao desenvolvimento standard, e o outro aos específicos. A atualização do tratamento específico não necessita de protecção por código actividade.  Ajuda desenvolvimento

Serão nomeados : fichier.ext, fichier_1.ext, fichier_2.ext, … fichier_n.ext. Para cada lista, indica-se a orientação(retrato/ paisagem) O tratamento de inicialização está executada para cada lista

Você deve definir aqui o tipo da zona. Os principais tipos são :

A : Alfanumérico
C : Inteiro curto
L : Inteiro longo
DCB : Decimal
D : Data
M : Menu local

Outros tipos pré-definidos existem. Fazem frequentemente referência a uma tabela anexa (divisa, secção,...) A tecla F12 permite de obter a lista.

  • Menu ([field] PARNOLIB)

Define o número de menu local associado ao campo definido sobre a linha

Quando um campo é de tipo menu local, é guardado sob a forma de um valor numérico (de 1 a 255) correspondente à fila de um título numa tabela chamada menu local, guardado na tabela das mensagens APLSTD.

Em registo ou em afixação, vê-se, segundo as escolhas feitas sobre interface :

  • seja um intitulado podendo ser escolhido numa lista deslizante comummente chamada combo box
  • seja uma lista de botões de escolha.

O interesse deste tipo de registo é que a lista das escolhas está apresentada na língua de conexão do utilizador.

Cada número de menu local caracteriza a lista dos intitulados possíveis. Por exemplo, o menu local 1 correspondente à escolha Não / Sim nesta ordem. Neste caso particular, o interface utilizador pode ser assim a check box a assinalar.

Indica se falta registar um intervalo do valor.

  • Ent ([field] PARSAI)

Indica se o parâmetro está registável ou não ao lançamento da lista.
  • Valor por defeito (início) ([field] PARDEF1)

É possível de registar uma expressão que será avaliada ao registo para inicializar o parâmetro.

  • Valor por defeito (fim) ([field] PARDEF2)
  • Controlo ([field] PARCTL)

É possível de registar uma fórmula para controlar o registo deste parãmetro. A variável VALEUR representa o valor registado do parãmetro.

  • Tabela de controlo ([field] PARCTLTAB)

Pode-se indicar aqui o código de uma tabela controlada para verificar o registo do parâmetro.

  • Parâmetro obj./dependência ([field] PARPAR)

Permite registar o primeiro elemento de uma chave de um objeto, se este objeto a uma chave ou a dias componentes. Exemplo: nº tabela para uma tabela diversa.

Particularidade para uma tabela diversa dependente: este campo permite registar o elemento mestre da tabela diversa "mestre"; neste caso, o número da tabela diversa está a registar na coluna "menu".
  • Opções ([field] PAROPT)

Registável se o tipo de dados está ligado a um objeto. Este campo permite definir uma ou várias opções de filtro definidos no objeto. O supervisor controla o valor registado no parãmetro em função dos filtros posicionados. Assim, a janela de seleção, tem em conta os filtros.
  • Código acesso ([field] PARACS)

Este código de acesso permite interdir o registo e a visualização do parâmetro.

Com efeito, se a zona está registada, apenas os utilizadores tendo este código de acesso com o direito de modificação modificar o parãmetro.

Segmentação

  • Parâmetro de segmentação ([field] PARSEG)

Permite separar a impressão em várias impressões, no objetivo de melhorar as performances de uma lista muito volumosa. Trata-se então de definir aqui o parãmetro sobre o qual serão a registar as tranches de valores tomados em conta para a separação de lista.

Ao lançamento de lista, o registo destas diferentes tranches de valores deve se fazer por clique direito sobre o parâmetro de segmentação.

[Tab] Dados

Se a lista se apoia sobre pelo menos uma tabela de um outro dossier, terá a obrigação de indicar a fonte de dados ligada a cada dossier suplementar sob a forma "solução;dossier". Falta que o dossier seja um dossier ligado na gestão dossier. O número de fontes de dados está limitado a 5. O dossier registado não está, em facto, que um valor por defeito, pois que a fonte é modificável no lançamento de impressão. Se a solução não está indicada, o supervisor toma a solução corrente, é o mesmo para o dossier. Para entender as tabelas do dossier mãe, pode-se utilizar a variável GDOSX3.

Em seguida, indicar-se-á para cada fonte de dados, as tabelas utilizadas na lista. Se uma tabela não está referenciada, ela será automaticamente associados à fonte de dados do dossier corrente. Deste facto, as tabelas do dossier corrente não foram referenciadas aqui. O nome da tabela deve ser esta utilizado em Crytstal ; isso pode ser o nome de pseudómino, se existe em Crystal. Um limite a 10 tabelas está a respeitar aqui, limitado devido ao servidor de impressão. O nome de uma tabela está limitada a 200 caracteres.

[Grid] Fontes de dados

  • Descr. ([field] ZSRC)

Se a lista se apoia sobre pelo menos uma tabela de um outro dossier, tem-se a obrigação de indicar a fonte de dados ligada a cada dossier suplementar sob a forma "solução;dossier". É necessário que o dossier seja um dossier na gestão do dossier. O número de fontes de dados está limitado a 5. O dossier registado não está, em facto, que um valor por defeito, poisque a fonte está modificável ao lançamento de impressão. Se a solução não foi indicadaa, o supervisor toma a solução corrente; é mesmo para o dossier. Para atingir as tabelas do dossier mãe, pode-se utilizar a variável GDOSX3, disponível por todo o produto. Para atingir as tabelas do dossier de exploração X3, depois um outro produto (Warehousing GX, Abel X3, Pagamentos), as variáveis GSOLCPT e GDOSCPT contêm respectivamente a solução e o dossier X3.
