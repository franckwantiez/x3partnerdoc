Gestão dos pedidos

Esta função permite listar e seguir os pedidos enviados ao servidor de tarefas, seja pela função de submissão de pedidos, seja por registo de ficheiros de pedido num diretório dedicado.

O software integra em efeito um servidor batch permitindo a execução de tarefas (compreendidas das edições) e de grupos de tarefas em modo diferido. Estas tarefas podem ser lançadas diretamente ou abonadas para um lançamento regular.

Este servidor batch está implementado pelo intermédio de uma aplicação SERVX3, na qual se encontra um tratamento que escruta periódicamente uma tabela dos pedidos se encontrando no dossier supervisor. Quando as tarefas devem ser desencadeadas, elas são lançadas sob a forma de processo sage separados no dossier de lançamento.

O tratamento do servidor batch é o mesmo de lançar e de parar uma tarefa, e de controlar o número de tarefas simultaneamente activas em limitando o seu número.

A função de gestão de pedidos permite em todo o momento de conhecer a lista do servidor de pedidos que pode estar na lista :

  • ativo (o número de processos está então afixado)
  • Em curso de paragem :
  • inativo

Gestão do ecrã

[Tab] Ecrã de registo

A função de gestão dos pedidos permite fazer aparecer, num ecrã único, a lista dos pedidos submetidos ao servidor (que eles tenhem sido ou não lançados) e sua lista.

Segundo a lista do pedido, a linha aparece em negro (pedido terminado sem erro), em vermelho (pedido interrompido ou terminado com erros), em verde (pedido em curso de execução), ou em azul (pedido em espera de execução)

[Grid] Queries

  • Número ([field] NUMREQ)

Este número único identifica o pedido. Está atribuido em falando de um número inicial nos parâmetros do servidor batch.

  • Dossier ([field] DOSSIER)

Define o dossier no qual a execução da tarefa foi pedida.

  • Cód. tarefa ([field] TACHE)

Código identificando a tarefa lançada.
  • Língua ([field] LAN)
  • Utilizador ([field] USER)

Define o código de utilizador tendo lançado o pedido.
  • Data ([field] DAT)

Define a data e a hora prevista de lançamento da tarefa.

  • Hora ([field] HEURE)
  • Mono ([field] MONO)

Se o campo vale Sim, a tarefa não pode ser executada que em modo mono-utilizador sobre o dossier.
  • Status ([field] FLAG)

Este indicador de estado dos pedidos em curso pode tomar os valores definidos pelo menu local 21 :

Espera : O pedido não foi então ainda lançado.

  • - Em curso : O pedido está em curso de execução.
  • Terminada : o pedido está correctamente terminado
  • Abortado: O pedido foi interrompido
  • Erro : a tarefa terminou sobre um erro
  • Bloqueado: O pedido fazendo parte de um grupo em curso de execução ou bloqueado manualmente por uma interrupção.
  • A "matar" Pediu a paragem do pedido.
  • Fora atraso : A tarefa foi submetida fora atraso (pelo intermediário de um ficheiro)
  • Warning : A tarefa terminou sobre um código de erro não bloqueante (variável GERRBATCH<100)
  • Hora de fim ([field] HFIN)

Define a data e a hora de fim de execução da tarefa. Estas informações não são prenchidas que se a tarefa foi já lançada e se ela está terminada.
  • Data de fim ([field] DFIN)
  • Tipo tarefa ([field] TYPTAC)

Definido pelo menu local 20, este campo determina se a tarefa lançada é um tratamento sage ou um script (shellscript ou ficheiro de comando) executado sobre o servidor de tratamento.
  • Script ([field] ETAT)

Define o código do tratamento ou do script, quando a tarefa lançada não corresponde a uma função de base do software.

  • Servidor ([field] SERVER)
  • Porto ([field] PORT)
  • Identificador entidade sessão ([field] SESSIONID)
  • Nr. processo ([field] PROCESS)

Cada pedido é executado por um processo adonix dado sobre o servidor de aplicação. Este campo identifica o número de processos (nomeados em geral o processo id, ou pid).
  • Time-out ([field] TIMOUT)

Quando este campo não é nulo, identifica uma duração de "time-out", quer dizer um limite de tempo para além do qual a tarefa está automaticamente parada se ela volta outra vez.
  • Grupo ([field] GRP)

Quando a tarefa faz parte de um grupo de tarefas lançadas, encontra-se no quadro um número comum a todas as tarefas do grupo, e um número de ordem no grupo (estes campos estão vazios se o pedido não faz parte de um grupo).
  • Nr. ordem ([field] NUMGRP)
  • Sequência ([field] SEQGRP)
  • Frequência (mn) ([field] FRQ)

Quando a tarefa batch foi lançada em frequência todos os N minutos, esta coluna indica o valor de N.
  • Reaviso ([field] IMPETX)

Quando uma tarefa foi abonada para ser executada em frequência, uma única linha aparece para assinalar a sua execução, mas cada vez que ela é relançada, acrescenta-se 1 a esta coluna. Assim, pode-se saber quantas vezes ela foi executada.
  • Fich. batch ([field] JOB)

Quando a tarefa foi lançada pelo intermediário de um ficheiro job (cf. anexo técnico correspondente), encontra-se aqui o nome do ficheiro que foi submetido (sem extensão).

[Block number] 2

  • [field] PAGCUR

Afixa o número de página corrrente no quadro, assim que o número total de páginas.
Registo de parâmetros

permite modificar os parâmetros de um pedido (que ele seja em espera de lançamento ou já terminado).

Interrupção

permite pedir ao servidor batch de interromper a execução de uma tarefa.

Modificação

permite modificar a data, a hora de lançamento e a prioridade de execução de um pedido ainda não lançado.

Rastro

permite afixar o rasto de execução de um pedido já executado (ela se encontra no diretório TRA do diretório SERVX3, ela mesmo situada no diretório Runtime, e seu nome está RQT seguido do numero de pedido)

Relançar

permite criar uma nova tarefa, com os mesmos parâmetros de lançamento que a precedente. Por defeito, a execução está pedida o mesmo dia, em 5 minutos (o que deixa o tempo de modificar a sua data e a sua hora de lançamento graças à ação de Modificação). A tarefa a relançar deve ter sido lançado sobre o dossier corrente. Ela pode ter sido lançada por um outro utilizador, mas o seu reaviso vai se fazer a conta de utilizador corrente, após verificação dos seus direitos.

Botões específicos

Supressão

permite suprimir o pedido sobre o qual o cursor está posicionado. Isso não vale que para os pedidos em espera de execução: se elas estão em curso, utilizar-se-á eventualmente a função de paragem, e se elas estão já terminados, utilizar-se-á o expurgo.

[Block number] 1

  • Dossier ([field] DOSSIER)

Define o código do dossier sobre o qual o pedido está lançado.

  • Código utilizador ([field] USER)

Define o código ( e eventualmente a password) do utilizador tendo lançado o pedido.

  • Palavra-chave ([field] PASSE)

[Block number] 2

  • Grupo ([field] GRP)

Define um grupo de tarefas a lançar.
  • [field] ZGRP

[Block number] 3

  • Cód. tarefa ([field] TACHE)

Define o código da tarefa que será lançada automaticamente pelo servidor batch.

  • [field] ZTACHE

[Block number] 4

  • Língua ([field] LAN)
  • Data ([field] DAT)

Define o dia e a hora de desencadeamento pedido para o pedido.

  • Hora ([field] HEURE)
  • Modelo ([field] JOBMOD)

Se este campo foi assinalado, um ficheiro contendo os parâmetros registados para o pedido será gerado no directório dos ficheiros modelos (parâmetros servidor batch) e o pedido não será submetido ao servidor. O nome deste ficheiro é composto do código da tarefa seguimento de .mod Este ficheiro serve de modelo para criar ficheiros .job que permitem de criar os pedidos bath.

permite submeter um novo pedido.

Atualizar

permite de refrescar a lista dos pedidos e das listas correspondentes. Esta função está também acessível via a tecla de atalho
Rastro

permite afixar o ficheiro de rasto geral do servidor (hora de arranque, lançamento dos pedidos, fim de pedidos...). Este ficheiro se chama serveur.tra, e ele se encontra no diretório TRA do diretório SERVX3, ele mesmo situado no diretório Runtime.

Expurgo

desencadeia o expurgo das linhas do quadro. Os pedidos terminados a uma dada data (registo num ecrã que se abre então) são suprimidos da lista das tarefas.

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

O pedido não está mais em espera de execução
Anulação impossível

Esta mensagem está afixada quando se tenta apagar um pedido cuja execução teve já lugar.

Sem pedido em curso

O quadro dos pedidos a executar ou executar está vazio (isto pode chegar após um expurgo, ou se o servidor batch não foi lançado).

"Quadro cheio"

Existem mais linhas que o gestor de pedidos não pode afixar no seu quadro : apenas as primeiras linhas serão visíveis.

Esta tarefa deve estar ativada depois o dossier XXXX

Tentou-se relançar uma tarefa sobre um dossier outro que o dossier corrente.

Em baixo de ecrã, a lista do servidor aparece (Ativo/Inativo), assim que o número de processos correspondentes, se está ativa. O facto que o servidor está inativo não impede submeter os pedidos (mas por outro lado, elas não serão executados imediatamente mesmo se a hora de desencadeamento chegou).
