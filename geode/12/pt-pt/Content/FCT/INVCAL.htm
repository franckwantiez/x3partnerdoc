Cálculo

Esta função permite de lançar o cálculo de uma factura para um dossier de facturação dado e sobre um período dado.

Pré-requisitos

[Refer to documentation [Implementation]]

Gestão do ecrã

O cálculo de uma factura logistica se desenrola em 2 fases:

  • Fase 1: Registo dos parâmetros de cálculo
  • Fase 2: o cálculo com geração de uma factura no final.

Fase 1 - Registo dos parâmetros de cálculo

Para lançar o cálculo, o operador deve obrigatoriamente registar o dossier de facturação e o período de cálculo das prestações logísticas. Para informação, afixa o número de última factura validado assim que o período afectado.

Deve-se referir que :

  • por defeito o limite inferior do período de cálculo está inicializado com o limite superior do período na última factura + 1 dia.
  • o limite superior do período de cálculo não pode ser superior à data do dia.

Fase 2 - O cálculo da factura logística

O princípio de cálculo da factura logística é a seguinte:

  • Calcular o valor de cada rubrica sobre o período de cálculo (tomada em conta do dossier corrente e histórico) com salvaguarda dos dados do cálculo.
  • Cálculo do montante líquido de cada rubrica segundo a Tarifa correspondente ao período de cálculo.
  • Cálculo do montante líquido de cada prestação (soma dos montantes líquidos das rubricas associadas).
  • Cálculo do montante líquido da factura logística para o dossier de facturação (soma dos montantes líquidos das prestações associados).
  • Geração da factura logistica na lista "7-Espera de validação".

Observação sobre a divisa do dossier de facturação e a divisa do dossier X3: todas as Tarifas estão na divisa do dossier X3. No entanto a factura pode ser gerado numa outra divisa parametrizável ao nível do dossier de facturação. Nos outros casos todos os preços convertidos automaticamente na divisa do dossier de facturação seguinte a tabela dos câmbios das divisas.

[Tab] Ecrã de registo

 

[Block number] 1

  • Código dossier ([field] IFHCOD)

Código do dossier de facturação definido na tabela dos dossiers de facturação Warehousing. É este intitulado para o qual a facturação logística será calculada. Este dossier está afixado a um Estabelecimento/Depositante.

  • Divisa ([field] IFHCUR)

Este campo indica a divisa de facturação utilizada para as pré-facturas de um dossier de facturação.

Última fatura

  • Última fatura ([field] LASINVNUM)

Este campo indica o número da última pré-factura validado sobre este dossier de facturação.

  • Data validação ([field] LASVALDAT)

Este campo dá a data de validação da última pré-factura.

  • Período ([field] LASSTRDAT)

Este campo indica a data de início do período de cálculo da primeira pré-factura.

  • [field] LASENDDAT

Este campo indica a data de fim do período de cálculo da última pré-factura.

Período de cálculo

  • Data início ([field] WSTRDAT)

Este campo indica a data de início do período de cálculo da pré-factura.
  • Data fim ([field] WENDDAT)

Este campo indica a data de fim do período de cálculo da pré-factura.

[Block number] 4

Permite de’afixar o fim do tratamento, o rasto (desenrolamento) do programa.

  • Rastro detalhado ([field] TRA)
  • Edição pré-factura ([field] WIMP)

Se assinalada, a pré-factura será editada no fim do cálculo.

Tarefa batch

[This function can be run in batch mode]. [The standard task] INVCAL [is provided for that purpose].

Mensagens de erro

  • [Generic messages]

[In addition to the generic error messages, the following messages can appear during the entry] :

Erro zona "Período de cálculo": cruzamento de periodo interdito

Nenhuma factura validada não deve existir sobre o período de cálculo.
