Códs. atividade

Os códigos atividades oferecem as possibilidades seguintes :

  • permite tornar ativos ou inativos dos elementos do dicionário (exemplo : tabelas, índices, separadores, blocos ou campos de ecrã ...) Por exemplo, o facto de tornar ativo ou não um código atividade dada permite então de desativar, nos ecrãs, certos campos opcionais.
  • Afetar um dimensionamento às zonas de tabelas.
  • tornar parametrizável o dimensionamnto dos campos, dos blocos de ecrã
  • identificar os elementos desenvolvidos para as localizações standard : estes elementos são identificados por um código atividade começando por K.
  • identificar os elementos desenvolvidos em específico e em vertical. Desde logo que o código atividade começa por X, Y ou Z, o elemento está considerado como não standard e não está afetado por um patch standard. Para ser mais preciso, uma norma foi estabelecida a partir da versão 5 dos softwares X3 :
    • Os códigos começando por X são reservados aos desenvolvimento verticais.
    • Os códigos começando por Y ou Z são reservados aos desenvolvimentos específicos.

O valor dos códigos atividade esta definido em gestão de dossier. Apenas os códigos que não são dependentes de um outro código são registados, os outros são deduzidos. Para mais detalhe, e em particular para consultar a regra de nomeação dos códigos ver a ajuda sobre o campo "dependência".

Modo de emprego para modifcar o valor de um código atividade.

O método :

Se colocar, no dossier mãe,

  • Modifiar o código atividade, sobre a ficha do dossier filho a tratar, para a função GESADS.
    ( NÃO SOBRETUDO modificar diretamente a ficha código atividade no dossier a tratar. Com efeito, a validação se apoiando sobre a diferença código actividade (entre a ficha código atividade e a ficha dossier), não terá nada a tratar.)
  • Lançar a validação do dossier a tratar A validação arrasta uma supressão ou criação dos elementos submetidos ao código atividade

Solução de exceção para programadores iniciados :

Para progradores experientes que não desejam passar pela gestão dos dossiers (validação de dossiers muito longo, ou impossível a realizar sobre os dossiers que são sempre utilizados).

Para os códigos de atividade de dimensionamento unicamente. Interdito para os códigos atividade ativo/inativo. Com efeito, a supressão ou a criação de elemento em função de ativação ou não de um código atividade está completamente gerido na validação dossier, mas não o é na validação de cada dicionário.

Se colocar no dossier a tratar,

  • Modificar o código atividade, diretamente sobre a ficha código atividade.
  • Validar unicamente os elementos submetidos ao código atividade pela função de validação do dicionário situado em : "Desenvolvimento \ Utilitários \ Dicionário \ Validação"

Gestão do ecrã

Esta parametrização se faz sobre um único separador.

[Tab] Ecrã de registo

Descreve as características detalhadas do código atividade.

Identificador

Este intitulado permite de explicar a que serve o código de actividade.

  • Cód. activ. ([field] CODACT)

O código actividade identifica uma opção que pode ser de tipo sim/não, ou que define um número de ocorrências.

Esta opção, que deve ser preenchida na criação de um dossier, permite de influenciar sobre a estrutura dos ecrãs do software e também eventualmente sobre a estrutura da base de dados.

  • Descr. ([field] ZLIBACT)

Características

  • Activo ([field] FLACT)

Para o desenvolvimento, este campo posicionado a Sim activa as tabelas, os ecrãs, ou os campos nas tabelas e os ecrãs que dependem do código de actividade. Inversamente, se este campo está a Não, os ecrãs e as tabelas, ou os campos que dependem não vão mais ser acessíveis e desaparecerão.

Atenção, em exploração, para toda a carga de posicionamento deste código actividade, é necessário de :

  • fazer a modificação do código actividade, na ficha dossier, a partir do dossier mãe.
  • depois de desencadear uma validação do dossier filha.

[Block number] 3

  • Módulo ([field] MODULE)

Módulo de pertença da parametrização. Este campo permite registar se o ecrã deve estar criado na base de dados do dossier. É o módulo ao qual ecrã está anexada está ativa para o dossier.
  • Rang ([field] RANG)

Este campo permite de indicar a ordem na qual se vão afixar os códigos actividades na gestão dos dossiers. Os códigos actividades dependendo de uma fórmula são calculados na ordem crescente deste número.

[Block number] 4

  • Tipo ([field] TYP)

Este campo permite de classificar os códigos actividades nos separadores da gestão dos dossiers.
  • Dimensão min. ([field] DIMFIL)

Certos campos nas tabelas podem ver variar o número de ocorrências em função de um código actividade. Frequentemente o número de ocorrências é o mesmo para um campo dado entre a tabela e o ecrã, mas não chega por vezes que se deseja simplesmente esconder os campos em guardando as ocorrências na tabela.

Neste caso, é possível de dar um valor diferente a esta coluna por relação ao precedente. Esta dimensão corresponde geralmente à dimensão gerado nos estados Crystal Reports.

  • Dimensão máx. ([field] DIMMAX)

Este valor corresponde à dimensão máxima das variáveis geradas nos tratamentos.

  • Dimensão ecrã ([field] DIME)

Define o número de ocorrências utilizadas nos ecrãs, e igualmente nas tabelas afectadas, sabendo que para uma tabela, um número mínimo (e um número máximo) podem existir, o que conduzirá então a utilizar, para dimensionar as tabelas, a fórmula :

min(max(MINI,ECRAN),MAXI).

[Block number] 5

  • Dependência ([field] DEP)

Desde o momento que o seu valor é diferente de Não, este campo permite de indicar que o valor de um código actividade não foi registado, mas automaticamente determinado em função do valor de outros códigos actividade. As condições possíveis são as seguintes :

  • Inverso : O código de actividade toma o valor inverso do código registado em face; será activo se o código actividade correspondente está inactivo, e inactivo se está activo.
  • Dimensionamento : O código de actividade está então composto de uma raíz, seguida de um número M (de 1 a 9), e é associado a um código de actividade que pode tomar valores numéricos de 1 a N. O código de actividade está activo se o valor do código associado é superior ou igual a M; senão, está inactivo.

Pode-se tomar o exemplo do código ANA, associados por ligações de dimensionamento aos códigos AX1 a AX9. Se ANA vale 5, os códigos AX1 a AX5 serão activos, os códigos AX6 a AX9 serão inactivos.

  • Fórmula : permite de calcular o valor de um código actividade em função de uma fórmula registada. Esta expressão calculada pode integrar as constantes, das funções, e das variáveis sob a forma de outros códigos actividade. Estes códigos de actividade pode ser dos códigos registados, ou dos códigos calculados anteriormente (a ordem de cálculo dos códigos actividade depende de códigos anteriores e definidos pela fila. Um código actividade não portador de uma dimensão vale 1 se está activo, e 0 se está inactivo.

Por exemplo, se a fórmula dada para o código XXX é max(XAA,XAB), XXX será activado se um dos dois códigos actividade XAA ou XAB está activo. Se estes códigos portam uma dimensão, a dimensão obtida será o máximo das duas dimensões.
  • Cód. activ. ([field] ACTDEP)
  • Formula ([field] ACTFOR)

Quando o código actividade depende de uma fórmula, define-se aqui a fórmula a avaliar para obter o valor do código de actividade. Esta fórmula pode ser :

  • uma fórmula lógica (se ela for verdadeira, o código de actividade será activo, senão será inactivo).
  • uma fórmula numérica reenviando um valor inteiro positivo (para os códigos actividade portadores de uma dimensão).
