Configurações

Esta tabela permite parametrizar as regras de gestão gerais ligadas aos stocks para cada recipiente de um estabelecimento.

Pré-requisitos

[Refer to documentation [Implementation]]

Para criar a configuração dos stocks, selecciona-se um recipiente, depois deve-se preencher os separadores dedicados a cada um dos fluxos.

Cabeçalho

Registo do código recipiente

  • Depositante ([field] DEP)

Código que representa o depositante. Em termos de logística, a noção de depositante permite de tratar os fluxos ou tráficos distintos no mesmo estabelecimento. Ela cobre a noção de cliente dador de ordens para os prestadores de serviços.
Após o estabelecimento, o depositante é a segunda chave de acesso aos ficheiros Artigo, Entrada e Ordem de Entrega.
Para o acondicionamento, o código depositante está preenchido se o acondicionamento está afectado ao depositante ou ao artigo, e se o acondicionamento é mono depositante ou mono artigo e que não está vazio.

 
  • Descr. ([field] NAM)

[Tab] Parametrização

Este separador contém a parametrização geral sobre o stock.

Gestão

Gestão dos números de suporte (ao nível do depositante) para cada objecto entrado.

A autorização de registo desta informação é contida ao nível do conteúdo artigo.

  • Nr. de suporte ([field] SPTMGT)
  • Número de lote ([field] LOTMGT)

O registo do número de lote para a criação de um objecto é obrigatório se a configuração dos stocks do depositante e a parametrização artigo comporta a parametrização "Número de lote" a "sim".

[Block number] 2

  • Transferência ([field] MVTADJAUT)

Código movimento que será utilizado para os ajustamentos automáticos ligados às transferências de stocks.

O código movimento registado deve existeir na tabela dos códigos movimentos e deve ser de tipo "Ajustamento automático".
  • transferencia 2 fases ([field] MVTADJAU2)
  • Inventário ([field] SKCAJM)

Código movimento de tipo ajustamento automática utilizada pelo sistema no momento da geração de ajustamento automático seguinte ao inventário.

O código movimento registado deve existeir na tabela dos códigos movimentos e deve ser de tipo "Ajustamento automático".
  • Pedido de análise ([field] MVTQLYAUT)
  • Montagem ([field] MVTDCLKIT)

indica o código movimento utilizado para os ajustamentos automáticos que surjem no procedimento de assemblagem.
  • Desmontagem ([field] MVTDCLCPN)

indica o código movimento utilizado pelos ajustamentos automaticos de desassemblagem.

[Block number] 3

  • Reab. sobre ciclo ([field] MVTWAVRPN)

Código movimento que será utilizado no momento de um aprovisionamento deû a um lançamento em ciclo

O código movimento registado deve existir na tabela dos códigos movimentos e deve ser de tipo "transferência".
  • Qtd. falta reabast. ([field] MVTRPNSHR)

Código movimento que será utilizado no momento de uma Transferência devida a remanescentes de objectos de stock aparecidos a uma ordem de aprovisionamento. O código movimento deve existir na tabela dos códigos movimentos e deve ser de tipo "Transferência".

  • Acum. de fluxos ([field] MVTTRFFLC)

Código movimento que será utilizado na primeira fase da preparação de tipo acumulado de fluxos no momento da geração das transferências versus os armazéns de separação.

O código movimento registado deve existir na tabela dos códigos movimentos e deve ser de tipo "transferência".
  • Pedido de análise ([field] MVTTRFQLY)

Código movimento a utilizar para a transferência dos stocks em controlo qualidade, seguida à validação / encerramento de um Pedido de Análise.

O código movimento registado deve existir na tabela dos códigos movimentos e deve ser de tipo "transferência".

  • Montagem ([field] MVTTRFCPN)

indica o código movimento utilizado para a transferência (1ª fase) do ciclo acumulado de fluxo de assemblagem.
  • Desmontagem ([field] MVTTRFKIT)

Indica o código movimento utilizado para as transferências ligadas às OA (Ordens de assmblagem) de tipo desassemblagem.

[Tab] Aprov.

Este separador contém os parâmetros utilizados no quadro da geração de uma ordem de aprovisionamento.

Gestão

Este parâmetro condiciona o funcionamento do algoritmo de aprovisionamento picking.

seja a check box assinalada e o algoritmo de aprovisio namento considera que os em-cursos de saída não são jamais executados nos seus cálculos de colocação disponível em picking (a regra de aprovisionamento em palete está inibida em caso de necessidade),

seja a check box’não está assinal ada e o algoritmo de aprovisionamento considera que os em-cursos de saída são sempre executados nos seus cálculos de colocação disponível em picking (a regra de aprovisionamento em palete completa utilizada nos casos de acondicionamentos maiores que uma palete standard).

O sistema autoriza a transferência automatica das paletes quebradas. Este valor está a confirmar da noção de reendereçamento dos remanescentes de ap rovisionamento no armazém reserva deùestá destocado esta palete.

  • Reender. execed. reab. ([field] IPAAUTTRF)

[Tab] Montagem

Este separador contém os parâmetros utilizados no quadro do procedimento de assemblagem.

Ordens de montagem

  • Ref. única ([field] BOMUNQREF)

Indica se uma mesma referência de ordem de assemblagem (OA) poderá figurar um número ilimitado de vezes no portfólio das OA (todas as listas simultaneamente), ou bem uma única vez (qualquer que seja o estado da OA), ou várias vezes (na condição que as outras sejam validadas ou saldadas).

Edições

  • Lista de transf. ([field] BOMLST)

Se assinalada, a edição de etiqueta de transferência está automaticamente na criação de OT (Ordem de Transferência) asssemblagem
  • Etiq. de transferência ([field] BOMLBL)

Se assinalada, a edição de etiqueta de transferência está automaticamente na criação de OT (Ordem de Transferência) asssemblagem
  • Ruptura p/artigo ([field] BOMSSHITM)

Se assinalada, a edição de Ruptura por artigo está automaticamente na criação de OT assemblagem.
  • Ruptura p/OM ([field] BOMSSHORD)

Se assinalada, a edição Ruptura por OA está automaticamente na criação de OT assemblagem.

Ciclos

  • Stock disponivel ([field] BOMSTKAVL)

Se assinalada, a pesquisa em saída da Transferência 1ª fase em fazendo prioridade sobre os acondicionamento de assemblagem.
  • Stock picking ([field] BOMSTKPIC)

Se assinalada, a pesquisa em saída da Transferência 1ª fase se fará também sobre os acondicionamentos picking.
Se não assinalada, a pesquisa em saída se fará unicamente sobre os acondicionamento reservas.
  • Transf. Incompl. ([field] BOMTRFIPA)

Se assinalado, a única quantidade necessária foi transferida mesmo se para tal se cria uma quebra em stock.
Se não assinalado, a transferência dos objectos componentes é realizada sem quebra de objectos de stock. Pode-se transferir uma quantidade superior à quantidade pedida.

Constituição de OT

  • Mono zona prep. de recolha ([field] BOMPKZPIK)

Se assinalada, as ordens de transferência serão agrupadas por zona de preparação de registo ligado ao armazém.
  • Mono zona prep. de levant. ([field] BOMPKZDPS)

Se assinalada, as ordens de transferência reagrupadas por zona de preparação de registo ligado ao armazém.
  • Corte de carga ([field] BOMWKL)

Se assinalada, isso significa que se vai cortar as OT assemblagem de maneira a respeitar os limites de carga definidos sobre a zona de preparação.
O cálculo da carga far-se-á sempre sobre a zona de preparação de tomada. No caso das OT multi-zona de preparação de tomada, falta obrigatoriamente indicar no campo "Zona de Preparação" a zona de preparação a considerar para o cálculo.
  • Zona prep. ([field] BOMWKLPKZ)

indica a zona de preparação a utilizar para o corte de carga nos casos onde uma OT multi-zona de preparação de tomada.

Montagem

  • [field] W1
  • Tipo de validação ([field] CPNVLD)

Se asinalada, a validação da transferência da 1ª fase será automática.
  • [field] W2
  • Cód. Operador ([field] DCLOPE)

Se assinalado, o registo de um código operador será pedido no momento da declaração do kit.

Desmontagem

  • [field] W3
  • Tipo de validação ([field] KITVLD)

Se asinalada, a validação da transferência da 1ª fase será automática.
  • [field] W4
  • Cód. Operador ([field] LCDOPE)

Se assinalado, o registo de um código operador será pedido no momento da declaração do kit.

[Tab] Controlo qualidade

Este separador contém os parâmetros utilizados no quadro do procedimento de controlo de qualidade.

Controlo qualidade

  • Natur. de stock Rejeitado ([field] DILSKN)

Natureza de stock a utilizar para a actualização dos objectos de stock qualificados não conformes do controlo de qualidade. O ajustamento qualitativo está realizado na validação do pedido de análise.

  • Natur. de stock Consumido ([field] USESKN)

Permite de configurar a natureza de stock a utilizar para as quantidades de stock consumíveis no momento do controlo de qualidade : esta natrureza de stock permite de identificar e de descontar os stocks consumidos no momento do controlo qualidade.

  • Cód. Operador ([field] QLYOPE)

Quando o parâmetro está a sim, o registo de um código operador foi pedido para registar o registo do controlo qualidade.

Validação pedidos de análise

  • Endereçam. automático ([field] ALLQLYAUT)

Quando a flag está activada, o endereçamento automatico dos stocks controlado está realizado na validação/encerramento de um Pedido de Análise: As transferências do armazém qualidade versus são criados em automatico.

  • Cód. Operador ([field] VQLOPE)

Quando a flag está activada, o registo de um código operador foi pedido para a validação de um Pedido de Análise.

Edição

  • Ficha qualid. ([field] PRNAUT)

Qaundo a flag está activa, a edição da ficha qualidade configurada pelo artigo está automáticamente desencadeado na criação do Pedido de Análise.

[Tab] Picking pivot

Este separador contém os parâmetros utilizados no quadro do lançamento em ciclo no momento da pesquisa de acondicionamentos picking pivots vazios a afectar (artigos com modo de saida picking pivot).

Prioridades de critérios de pesquisa de afectação picking pivot.

O valor óptimo dos critérios os mais importantes será conservado em prioridade por relação aos outros critérios para a afectação do artigo a um picking pivot vazio.

Recipiente
Optimo : Pesquisa de acondicionamentos afectados ao Recipiente do artigo a afectar.
Degradado : Pesquisa de acondicionamentos vazios não reservados.

Armazém
Optimo : Pesquisa de acondicionamentos no armazém prioritário definido mais alto na configuração dos stocks para o picking pivot.
Degradado : Utilização da Tabelas Encadeamento dos Armazéns.

Classe de Rotação.
Óptimo : Pesquisa de acondicionamentos tendo uma classe de rotação idêntica àquela afectada ao artigo.
Alocado : Utilização do parâmetro "Encadeamento" na tabela das classes de rotação.

Classe de Afectação
Optima : Pesquisa de acondicionamentos tendo uma classe de afectação idêntica àquela afectada ao artigo.
Alocado : Utilização do parâmetro "Encadeamento" na tabela das classes de afectação.

Gestão

  • Cód. encadeamento ([field] CHNMODDYN)

Os encadeamentos permitindo de encontrar o armazém de entrada prioritária e armazéns encadeados no momento dos aprovisionamentos dos acondicionamentos picking pivots. Obrigatório.
  • Encadeamento armazéns ([field] CHNDYN)

Define se encadeamento dos armazéns está utilizada para a pesquisa de acondicionamento picking pivots.

  • Encadeamento classes de rotação ([field] CHNABCDYN)

Define se encadeamento das classes de rotação está utilizada para pesquisa de acondicionamentos picking pivots.

  • Encadeam. classes de afectação ([field] CHNASGDYN)

Define se o encadeamento das classes de afectação está utilizada para a pesquisa de acondicionamentos picking pivots.

[Grid] Priorid. picking pivot

  • Critérios ([field] DYNPIO)

Definida a prioridade dos critérios a degradar para a pesquisa de acondicionamento picking pivots : Depositante, Armazém, Rotação, Afectação.
