Stock counts

This function is used when reporting a stock count for a site. Two types of stock counts can be accessed in Warehousing GX.
The stock count by product is used to count the product stock, in other words, the locations where the products are located at system level.

The stock count by address is used to count the stock for each delivery address.

A stock count by address blocks the addresses relating to the stock count on input and output. The stock count no. in process is recorded in the location record.

One or several counts are reported for a stock count and then entered in the system.

Matching operations between the entered stock and the logical stock can then be printed.

Based on the setup (flow control), a stock count by product can block the stock inputs and outputs for the products subject of the stock count.

See the documentation on stock counts.

Tab Header

By address/product
The stock count type specifies whether the stock count is by product: in this case, the product codes of the products to count must be selected. If the stock count is by address: in this case, the addresses need to be selected. These selections can be performed via the Selection tab.

Serial no. entry
In this case, on the occasion of a stock count by product managed with serial no., the serial nos. of each stock unit of the product will need to be entered.
Otherwise, even if the product is managed with serial nos., the serial nos. will not be requested when entering the stock counts by product.
Not activated for address-type stock counts.

WIP stock counts
If this option is checked, those products with WIP will be taken into account in the stock count. Otherwise they will be left aside.

Maximum container level (1...5)
When entering the stock count, the products can be counted in a container level at the maximum. For instance, in case of level 5, the stock count will be entered in packing units for all the products and locations of the stock count. In the case where a product code does not have the maximum level, the system uses the immediately lower container level.

Campaign
For stock counts by product, it is possible to assign them to the current campaign. The campaign number must be entered then.
In inquiry mode, the campaign number assigned to the stock count is displayed.

Stock count
Number of stock counts, irrespective of their statuses, created by the stock count.

This tab is used to select the criteria by product or by address.

As long as there are no counts for the stock count, it is possible to add or delete selected records.

The result of the selection is displayed in the "By address" tab if it is a stock count by address, or in the "By product" tab if it is a stock count by product.

The Sel field is used to manage multi-criteria selections: for the value "and", enter the same identifier, otherwise increment the value.

Specific Buttons

  • Simulation
  • Close
  • Close

Simulation

After entering the criteria, this button is used to display the errors along with the locations and selected products. It is then possible to modify the selection.

Close

This button is used to close the stock count and thus to normally end the stock count procedure.
The system lists all the records of the stock count and compares them with the stock counts actually carried out. If all the products or locations have not been counted, the system lists the faulty records and suggests a partial closure of the stock count.

  • Stock count by product: the date of the last stock count is updated in the product header and the product is unblocked. The campaign is also updated if the stock count has been assigned to a campaign.
  • Stock count by address: the last stock count date is updated for locations. Locations are unblocked.

Close

This button is used to close a stock count, which closes all counts under way. This closure features an abnormal end to the stock count in case of error or premature end to the stock count.
The last stock count date is thus not updated in the product record in case of a stock count by product. The product is no longer being the subject of a stock count.
In the case of a stock count by address, the last stock count date is thus not updated for locations. Locations are unblocked. Menu BarFunctions/Count This function is used to directly request a count for the stock count under progress.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

The selection does not match any record.

The stock count cannot be created because the selection set up does not match any data relating to the site.

Location/Product already in stock count XXX.

It is not possible to assign a product/location to two stock counts under progress.

Locations with WIP.

The stock count type created does not have any WIP and the selection concerns products/locations with WIP.
