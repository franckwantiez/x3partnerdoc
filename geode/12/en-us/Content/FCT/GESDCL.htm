Declarations

This screen is used for:

  • The kit assembling declaration in the assembling procedure.
  • the kit deassembling declaration in the disassembling procedure.

Prerequisites

Refer to documentation Implementation

Screen management

  • Header
  • Tab To be consumed
  • Tab Kit / Components
  • Tab To generate

The screen is sub-divided into:

  • a left list to select the declaration number that needs to be processed
  • a right part providing the detail of the declaration:
      • a header.
      • a "To consume" tab listing all the component/kit movements linked to the AO line.
      • a "To generate" tab providing the characteristics of the Kit/component WOs that will be created upon validation of the declaration.
      • a "Kit/Components" tab displaying the involved AO kit line with the component quantities selected during the declaration procedure. This tab is also used to check whether or not the kit proportions are complied with (the "Total DU" box is displayed in red in case of inconsistency).

Assembling declaration procedure

The assembling declaration procedure is sub-divided into 3 steps:

  • Step 1: Entry of the kit qty that needs to be created in the "To generate" tab.
  • Step 2: Selection of the component movements and quantities to be assembled in the "To consume" tab.
  • Step 3: Declaration validation.
 Step 1: Entry of the kit qty that needs to be created in the "To generate" tab.

The operator enters the information of the ‘quantity’ block (container no., container, etc.). The quantity for the kit to manufacture is indicated in the “Number PAC” field.

 Step 2: Selection of the component movements and quantities to be assembled in the "To be consumed" tab.

This step consists in checking the component lines and entering the quantity to be assembled ("Number PAC" field). This operation can be done either manually or automatically using the "Auto" button.

Whenever a line is checked, the system automatically transfers the entered qty to the "Total DU" field of the corresponding component line of the "Kit/Component" tab.

Based on the selected component quantities and the requested kit quantity, the declaration can take the following statuses:

  • "5-Non compliant": there are some inconsistencies regarding one of the following points:
      • the component quantities that do not comply with the kit proportions
      • the kit quantity to be produced is different from the one that can be actually declared with the selected components.
  • "6-Compliant": There are no inconsistencies regarding the kit proportions or the kit quantity to be produced. Only the declarations with this status can be validated during step 3.

Note: The "Save" button is used to save the declaration progress status.

Step 3: Declaration validation.

When all the components have been selected and the declaration has the status "6-Compliant", the operator can then validate it.

Upon declaration validation and after possibly entering an operator code (based on the configuration at stock configuration level), the system:

  • create the kit SO by automatic adjustment (the movement code used is the one defined in the stock configuration) according to the characteristics of the Stock object" tab.
  • validates the output movements of the components.
  • moves the kit declaration to status "9-Validated" and creates a new kit declaration for the kit quantity that remains to be created (case where the AO kit line is not completed).
  • updates the AO line concerned.

Disassembling declaration procedure

The kit disassembling declaration procedure is sub-divided into 3 steps:

  • Step 1: Entry of the kit qty that needs to be disassembled in the "To consume" tab.
  • Step 2: Entry of the component movements to create in the "To generate" tab.
  • Step 3: Declaration validation.
 Step 1: Entry of the kit qty that needs to be disassembled in the "To be consumed" tab.

The operator enters the information of the ‘quantity’ block (container no., container, etc.). The quantity of the kit is indicated in the “Number PAC” field.

 Step 2: entry of the component movements to create in the "To generate" tab.

This step consists in checking the component lines and entering the quantity to be created ("Number PAC" field).

Whenever a line is checked, the system automatically transfers the entered qty to the "Total DU" field of the corresponding component line of the "Kit/Component" tab.

Note: The "Save" button is used to save the declaration progress status.

 Step 3: Declaration validation.

When all the components have been selected and the declaration has the status "6-Compliant", the operator can then validate it.

Upon declaration validation and after possibly entering an operator code (based on the configuration at stock configuration level), the system:

  • create the components SO by automatic adjustment (the movement code used is the one defined in the stock configuration) according to the characteristics of the "To generate" tab.
  • validates the output movements of the kits.
  • moves the kit declaration to status "9-Validated" and creates a new declaration of the same type for the kit quantity that remains to be disassembled (case where the AO kit line is not completed).
  • updates the AO line concerned.

Tab Kit / Components

This tab displays the involved AO kit line with the component quantities selected during the declaration procedure. It is also used to check the consistency of the kit proportions as well as the kit quantity to be produced (the "Total DU" box is displayed in red in case of inconsistency).

  • Inconsistency in the component quantities that do not comply with the kit proportions

If the component quantities do not comply with the kit composition coefficients, the "Total DU" box is displayed in red for the involved component lines.

  • Inconsistency in the kit quantity to be produced, different from the one that can be actually declared with the selected components.

Based on the quantity of selected components, the system determines the number of kits that can be declared ("Total DU" field of the first Kit line). If this number is different from the kit quantity to be produced defined in the "Stock object" tab ("Number PAC" field), then the "Total DU" box is displayed in red.

Tab To generate

 This tab is used to define the characteristics of the stock objects to be produced, both from a quantitative (quantity, container, etc...) and qualitative (FIFO date, lot no., stock nature, reservation no., etc.) point of view:

  • Kit stock objects for the assembling AO.
  • Component stock objects for the disassembling AO.

Specific Buttons

  • Auto
  • Validate
  • Auto

Auto

Used to validate a declaration. This button is active only if the declaration has the status "6-Compliant".

Validate

This is used to delete a declaration. This button is active only if no component movements has been selected.

Deleting this declaration will close the AO line.

Auto

Used to automatically select movements “to be consumed” based on the kit quantity to be declared. This button is active and can only be accessed during the assembling procedure.
