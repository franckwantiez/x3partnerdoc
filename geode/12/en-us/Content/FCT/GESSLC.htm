Storing lists

This screen allows the creation, the view, the update, the delete of a Storing list (S.L).

Il permet aussi de visualiser et éditer la charge d’une liste de rangement.

The S.L has a  status t allowing the view of the state of its progress.

Prerequisite

Refer to documentation Implementation

Screen management

  • Header
  • Tab Grouped movements

The screen is made up of two tabs : an identification tab of the SL and a tab with the list of input movements of the SL.

Header

This tab presents the identification information of the storing list.  

The SL Number is assigned automatically at the creation of the entity.

During the creation of the SL, the system date is proposed by default.
The date can be changed by the user.

The  status code  gives the state of progress of the SL.  

Field descriptions

Storage list number to which the input movement is attached if it is dependent on a SL.

Creation date of the storage list.

Number (field STLNUM)
Date (field DAT)
Status (field STLSTA)

Tab Grouped movements

This tab presents the input movements of the SL.

The input movements of an SL are displayed grouped; meaning that all the movements having the following characteristics identical are accumulated (only on display) so as to facilitate their reading:
Input N° and input line n°
Store and address
Reservation code, Lot n°, Support n°, origin code, FIFO date
Quantity, Item container, Homogeneous container
Status

Example :   2 movements of 1PP1 of 100 ACU for the same consignee will be displayed grouped as 1 line of 2PP1 of 100 ACU or  200 ACU

Other conditions

In entering in this function, only the non-processed SL are displayed. To display the processed SL (status 4-validated), click on Selection then All.

Creation

 The creation of a SL is necessary when manual addressing is carried out, as and when needed or when we have to delete a SL.

 File/New then allocate the addressed input movements to the SL

 The list on the left has a drawer  « addressed movements » that displays the addressed movements (status 3) without S.L. n°. Depending on the input configuration, the display might differ :

If the SL is  mono input, and the sequence of the storing is « input/store » :
It is sorted by Input N°, Input reference, then Store, Address, Movement N°

If the SL is mono store, and the sequence of storing is « store/input » :
It is sorted by Store then Address, Input N° and Reference and Movement N°

If the SL is mono store and mono input :
It is sorted by Store Address, Input N° and Reference then by Address and Movement N°

 Two ways to assign input movements to a SL :
Select the movements from the drawer « addressed movements » on the list on the left
Select by « picking selection » of the memo already defined or by criteria not yet recorded

 The   button validates the action and create the SL

Suppression

 The S.L can be deleted when it is in status 1-Not validated.

 The   button deletes the S.L. The input movements are then unallocated from the S.L but remains in status 4-addressed, and available to be assigned to another SL, existing or to be created.

Modification

 The modification of an S.L in allocation or disallocation of movements is possible  when the SL is in status 1-Not validated and status 3-Partially validated.

 To allocate movements to a SL already created, just use the same procedure as in creation.

 To disallocate movements from a SL already created, the cursor must be position on the line of the movement to be removed and « click right on the mouse » and Delete line and click on the  button.

Reports

By default, the following reports are associated with this function :

  ETIQENT : Input labels

  RANGEMENT : Storing lists

This can be changed using a different setup.

Menu Bar

Functions / Load analysis

Allows to launch the analysis of the workload of the selected storing list

The workload analysis function is conditioned by the activity code LA2 and the status of the storing list that should be inferior to 4 (not validated storing list).

The analysis will bear on the list of pending input movements linked to the storing list

After a request of analysis of workload :
• a new analysis is created with the following characteristics :
- Indicator to be conserved = « no »
- Template (for information) :
   ▪ that associated to the triplet (function, current site, current depositor) for a single depositor function
   ▪ By default, or for a multi depositors, that associated to the triplet (function, current site, depositor « in blank »).
   ▪ By default, that associated to the triplet (function, site « in blank », depositor « in blank »)
   ▪ By default, it is left to blank.
• The  calculation is launched for the analysis
• The operator accede to the screens of display of workload from where he can also accede to the information of the analysis (description and indicator « to be conserved ») and ask for the printing of the result of the analysis.
