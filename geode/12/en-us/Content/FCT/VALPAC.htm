Mass validation

This screen is used to validate or cancel one or more parcels for DOs having the prepacking or prepalletization as their picking mode.
The serial nos. will need to be entered in the unit parcel validation option.

The parcel includes a status to view the progress.
The parcels with a status above 4 can no longer be modified.

Prerequisite

Refer to documentation Implementation

Screen management

The screen is made up of:

  • a selection filter to view the prepacked parcels (coming from DOs in prepacking picking mode) and/or the prepalletized parcels (coming from DOs in prepalletization picking mode)
  • a table displaying the list of parcels to be processed (status 4-not validated).

Entry screen

  • To validate several parcel recordings, it is necessary to check the recordings to be validated in the table, and then to trigger the.
    The parcels and output movements are then validated and the locations are updated.
  • Entry of a control code is totally prohibited when using this functionality irrespective of the set up.
  • If the parcel requires the entry of serial numbers, the unit validation function will have to be used or the serial nos. will need to be previously entered.
  • Entry of an operator code can be requested by the sytem. For that purpose, the configuration of the outputs for the current depositor must be set up accordingly. The validation will only be possible if the entered operator code authorizes parcel validation.
  • Entry of the specific information related to the parcel can be requested according to the Actual weight entry setup of the output configuration
    If a parcel range is imposed to the DO line, the latter cannot be modified.
  • In the event of the validation of a parcel of large parcel type, if the output configuration of the current despositor specifies it, and if the palletization is managed at DO level, the parcel is automatically palletized.
    A SU no. is assigned to it, the SU is closed and the SU type is picked up from the output configuration.

Specific Buttons

  • Cancel
  • Criteria

Cancel

  • It is possible to cancel one or more parcels by selecting the recordings and activating the .
  • The cancelled outout movements moves to status 5- Cancelled upon picking.
    In the event of the total cancellation of a parcel, the parcel no. is deleted.
  • A control is carried out in the event of a partial delivery not authorized for the DO or the DO line.
    It is possible to validate, then cancel a movement. On the other hand, it is not possible to cancel, then validate a movement.

Criteria

Used to select the parcels to be managed based on such criteria as parcel range, parcel type, wave no., PO no., DO no. and DO reference.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Operator code invalid.

The entered operator code does not enable the parcels to be validated.

Range/Parcel type: record non-existent

The entered range or type is not defined.

Actual weight: Mandatory field

It is mandatory to enter the actual weight..
