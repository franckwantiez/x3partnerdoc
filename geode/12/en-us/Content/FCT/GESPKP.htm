Picking order

Use this screen to view the details of a picking order (PO), regardless of its status.

The PO is associated to a status that reflects its progress.

When opening the function, only the POs to be processed are displayed (status 1- not validated or 3- partly validated). To display the already processed POs (status 4-validated), click Selection, then All.

You can also use this screen to view and print an active PO load (not validated).

The screen includes three sections: the first section identifies the PO, the second one presents general information about the PO, and the third one lists the output movements related to this PO.

Header

This tab provides the identification information of the PO: the PO number, the wave number that generated the PO, the PO creation date and time and the PO status.

The PO Number is automatically assigned when creating the entity.

The status code indicates the progress of the PO.

Tab Picking order

The PO header contains general information characterizing the PO and the preparation mode:

The PO is of the Kit type if it contains parent product and component lines.
The kit POs are generated as mixed, multi-level and for multiple preparation areas, regardless of the associated preparation mode settings.

The PO can be set as mixed, picking or reserve based to the preparation mode settings (not the actual picking).

The preparation area is entered if the Single preparation area mode is used. If not, the entered area is the preparation mode area.

The DO no. is entered if the preparation mode is single DO.

The Round no. is entered if the preparation mode is single round.

The Level is entered if the preparation mode is single-level.

It also contains the PO dimension information:
- the number of parcels in the PO, if prepacking is enabled; otherwise, the number of movements
- the number of parcels in the PO
- the PO weight and volume
- the PO preparation time
- the PO surface equivalence if prepacking is enabled

Specific actions

Functions / Load analysis

Use this function to launch the load analysis for the selected PO.

The load analysis function is conditioned by the LA2 activity code and by the non validated PO status.

The analysis will apply to:
• the list of output movements in progress for the PO
• the list of non-reported active parcels of the PO

Following a load analysis request:
• a new analysis is created with the following characteristics:
- Flag to maintain = "no"
- Template (for info.):
   ▪the template associated to the triplet (function, current site, current depositor) for a single-depositor function
   ▪ By default, or for a multi-depositor function, the template associated to the triplet (function, current site, depositor set to "blank").
   ▪ By default, the template associated to the triplet (function, site set to "blank", depositor set to "blank")
   ▪ By default, the template is set to "blank".
▪ The calculation for the analysis is launched.
▪ The operator accesses the load display screens from which it can also access the information related to the analysis (description and flag "To be kept") and request the printing of the analysis results.
