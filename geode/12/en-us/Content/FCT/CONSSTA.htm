Inquiries

This function is used to inquire the statistics that have been created by the statistics parameterization function, the latter defining for each statistical code:

  • a set of analysis criteria (from 1 to 8), to which the company (and the site, if needed) can be added
  • 1 to 10 cumulated values that will be stored in the statistics totals table, with a frequency to be set
  • calculation formulas between cumulated values or forecasts (if they have been defined and entered) over the current period and the previous periods, along with the forecasts if they have been defined and entered, with a view to displaying ratios, frequencies etc.

This inquiry is performed:

  • when directly entering the function, from the general level (the various values of the first criterion for which cumulated records already exist can be viewed), then using zooms into the lower criteria levels. At each intermediate level, it is possible to zoom into the object that manages the criteria.
  • at the lowest level, after successive zooms, when viewing the detail of those records justifying the total (if such records are still in the database, which may not be the case given the fact that the transaction details can be purged). It is then possible to view the justifying document in detail (if a zoom into the document has been parameterized). For instance, a statistic based on the invoices will enable a detailed viewing of the invoices and a zoom into the invoice itself will be possible if it has been parameterized.
  • via the File / Statistics function, from the object management, if the object parameterization enables it. Are then accessible the statistics where at least the first criterion is explicitly defined by the calling context (it must either be a field in the object main table, or one of the explicitly linked tables, or a field whose type corresponds to the object key). In inquiry, the detail level defined by the context is the one to be reached.

This inquiry can be carried out (based on the settings and the criteria entered in the header) either over a period defined by two dates, or over two periods (provided the data from the current period and the previous period are integrated into the tables), for the data of a site, a company, or all companies at the same time. The selected periods must naturally represent a multiple of the statistic inquiry frequency. This inquiry is sub-divided into two tabs:

  • the first tab presents the breakdown of the current statistic level according to the next sub-level (for instance, in the case of a statistic by country, then by customer, it is possible to view, for a customer, the list of figures of the customers in the country). The number of lines of the current level is limited by a pagination criterion (defined in the parameterization of the inquiry screens, for the STA code), that imposes the use of navigation buttons, if need be.
  • the second tab presents the total for the level (without taking account of the pagination limitation), then the breakdown into sub-periods of the inquiry period (the sub-periods corresponds to the frequency of the statistic). In case of a yearly, quarterly, monthly or bimonthly frequency, all the periods are displayed in the table, even those that present a null amount. In case of a lower frequency, only those lines for which there is a transaction will be displayed.

Next to the amount grids, instead of these grids, or using a toggle function from one representation to another, and depending on the screen parameterization, it will be possible to display the data in graphical form. For further information, refer to the corresponding annex.

Header

The header is used to select the characteristics of the requested statistic inquiry (code, periods, site and company criteria, if any).

Field descriptions

Block number 1

Code (field COD)

It identifies the current statistical code. A set of totals based on one to eight crossed criteria is linked to each statistical code.
Company (field CPY)

When this field is populated with a company code, the inquiry is filtered in order that only those data concerning the selected company are displayed.
Site (field FCY)

When this field is populated with a site code, the inquiry is filtered in order that only those data concerning the selected site are displayed.
Criterion (field LIBEL)

It is used to display the values of the criteria of the current level.

Block number 2

Period 1 (field DATDEB)

The statistical inquiry can be delineated by date, remembering that the start and end dates given here are redefined depending on the frequency period chosen for the statistic.

field DATFIN
Period 2 (field DATDEB2)

When the parameterization of the statistic displays comparisons between two periods, the previous period is given here. By default, a period of the same duration and adjacent to the start period is proposed.
field DATFIN2

Block number 3

Level (field NIVAFF)

It is used to display the current level at which the user is positioned, and the maximum possible level for the statistical code.
field NIVCOM
field NIVMAX

Tab Detail

This tab displays the amounts cumulated and calculated for each criteria combination while taking account of:

  • criteria defined in the header, given the current level and the order in which the criteria are run through.
  • the line criterion.
Field descriptions

Grid

field LIBCLE

It is used to display the values of the criteria of the current level.

field AMT1

Each column contains the cumulated or calculated amounts defined when the statistic was parameterized. These amounts are displayed on each line at the level of the corresponding total.

Action icon
Base Product Management

When the current criterion corresponds to a record key (for instance, the country code), a right-click displays a line (called Country in this instance) that enables the current record to be displayed (the record of the country whose code is present on the line of the instance).
Base Product Management

An additional function can be activated by right click, it is the tunnel function to a basic record, when the current criterion is the key of a record. The title that appears when using the right click depends on the record onto which the tunnel is possible.

For instance, for a statistic built on a set of criteria, one of them being the country code, when positioned at a level giving the detail by country code, a tunnel called Country is obtained, that enables the record of the current country to be displayed.
Filter

This button is used to filter lines, when positioned at a statistic level with many lines, by providing the requested code or a template of the requested codes.

Once the filter is set, only the corresponding lines are displayed. It should be noted that the breakdown by period of the second tab is not recalculated in totals for the lines. On the other hand, if the allocation by period of a line issued from the filter is requested, it suffices to switch to the next level using the corresponding button at the bottom of the screen. The content of the second tab will then be refreshed.

Tab Fiscal Periods

This tab presents the detail of the amounts of the previous tab by period (cumulated for all the lines of the level, and not only those on display). This detail is broken down into periods, depending on the frequency of the statistic. Thus, depending on the cases, there will be a line per week, per month, even per day.

For readability purposes, those lines corresponding to periods for which there are no transactions are not displayed in this grid, except for quarterly and yearly frequencies.

Specific Buttons

  • Criteria
  • Level -
  • Level +
Criteria

This button is used to display a criteria entry screen, that makes it possible to modify the screen being used (defined by the parameterization of the inquiry screens), the periods, the inquiry company and site, in order to perform the inquiry again and display the data again. Moreover it is possible to modify the following information in this screen:

Selection

Screen code (field ECRAN)

It is used to define the screen code (parameterized via the inquiry screen management). A screen code makes it possible, among others, to customize the graphical presentation of the columns and the number of displayed lines (this number corresponds to the N number displayed in front of the Top N check box).
Active memo (field MEMO)

The memo code makes it possible to save the criteria for an entry screen. This field is used to find out if a saved memo exists for the current values.

The following buttons are used to manage memos:

  This button makes it possible to recall a previously defined and saved memo.

  This button is used to save the current values of the screen in a memo file which code is going to be entered.

  This button makes it possible to delete a previously saved memo.

If a memo code called STD associated with the screen exists, it is loaded at the beginning of the corresponding function.
Company (field CPY)

When this field is populated with a company code, the inquiry is filtered in order that only those data concerning the selected company are displayed.
Site (field FCY)

When this field is populated with a site code, the inquiry is filtered in order that only those data concerning the selected site are displayed.
Period 1 (field DATDEB)

The statistical inquiry can be delineated by date, remembering that the start and end dates given here are redefined depending on the frequency period chosen for the statistic.

field DATFIN
Period 2 (field DATDEB2)

When the parameterization of the statistic displays comparisons between two periods, the previous period is given here. By default, a period of the same duration and adjacent to the start period is proposed.
field DATFIN2

Order of criteria

field TRI

It defines the criteria order used to display the totals and successives line breaks.

By default, the order 1, 2, 3... corresponds to the criteria declaration order in the parameterization of the statistic.
field CRIT

Title associated with each criterion.

Display

field TOP

When this check box is activated, the first N amounts of the column populated next are displayed in decreasing order.

The N value corresponds to the number of lines simultaneously displayed in the screen. This characteristic of the inquiry screen can be parameterized and it is called via the Screen code.
Column (field COLTOT)

It is used to define the number of the column for which the top N is calculated (from 1 to the number of columns defined in the statistic).
