Resources

This function is used to manage the resources on the current site. A resource represents a physical person in the warehouse. It is thus unique by site. Consequently only one resource by site can be associated to a user.
The resource management is used to:
  • manage the provisional planning of the resources working in the warehouse,
  • monitor the actual activity of the resources within the warehouse (validation of the operator codes)
  • Use the directive mission functionality (GEOMIPS optional module)

A standard Warehousing resource is characterized by:
  • their prototype planning: the definition of the public holidays, holidays and work days (including the planned activities) over a prototype year. This planning will be used upon initialization of a month in the provisional planning.
  • its validation process by operator code in the VT procedures: 3 various processes are possible:
    • Manual: in case of operator validation, entry of an operator code will always be requested (it corresponds to the default operation).
    • Semi-automatic: in case of operator validation, entry of an operator code will be requested upon first validation, then, for all the following validations, the code will no longer be requested and Warehousing will automatically use the code entered the first time.
    • Automatic: in case of operator validation, no entry of an operator code will be requested and Warehousing will automatically use the operator code defined for the resource.

New characteristics can be added (those originating from the GEOMIPS directive mission functionality):

  • the resource type
  • their mission sequencing code
  • their maximum authorized load.
  • their status for the directive missions (available or not, mission in progress, materials used and their current position within the warehouse).

 This functionality is available from the Advanced version

Screen management

The screen is comprised of a single entry screen.

Entry screen

Each created resource is identified in a unique manner on a site by means of a code equal to the user code which it is associated with.

Then, various options make it possible to define the prototype planning and their own management of the operator codes.

Field descriptions

Block number 1

Code (field RSS)

Specifies the resource code.
Name (field NOMRSS)

Block number 3

Resource type (field RSSTYP)

Code identifying the type of resource (defined by site).

A resource type identifies a set of resources likely on any given site to perform the same types of missions, access the same mission areas and use the same types of materials.
Mission sequencing (field MSNCHN)

Code identifying a mission sequencing rule (defined by site).

A mission sequencing code defines the search order of the mission types to follow each other.

Handheld procedures

Operator code management (field TYPCODOPE)

This parameter is used to specify the operator code management that should be applied in the VT procedures for a resource.

3 options are possible:

  • "Manual": entry of an operator code is systematically requested upon each validation.
  • "Semi-automatic": entry of an operator code is requested upon first validation. Then Geode will use the same code for all the other validations.
  • "Automatic": the validation by operator code will occur automatically using the value defined in the "operator" field.
