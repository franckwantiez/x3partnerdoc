Ideal classes

This function will allow :

 to gather data from the history of the movements and from the  output forecasts  table to determine the statistical data needed in the  calculation of the ideal rotation and volume classes, taking into account the different parameters filled in.

 to launch only the calculation of ideal classes with out recalculating the statistical data, after eventual change of the settings..

Prerequisite

Refer to documentation Implementation

Screen management

The screen is made up of a single tab that will allow to set the triggering process of the calculation of the ideal classes.

Entry screen

For the period requested by the user and for each item container manage in the process of picking dynamisation (depending of  output mode ), the system calculates :
 the mean daily promptings
 the volumic flow and the mean coverage of picking
from the history of the movements and the output forecasts

To set off :
 the ideal rotation class
 the ideal volume class

The system generates a picking reassignment campaign for each couple (site, container)

The screen allows the keying in of :

 the siteof the processing.
By default,  the  system proposes the current site.
The processing can be carried out for a given site or for all the sites of a folder ( by leaving the field blank).

 the picking reassignment campaign n° already launched.
This data will allow to :
- relaunch the processing for a campaign already existing.
To do that, there should be no record of the campaign reassigned.
The system then will delete the existing campaign and creates a new one.
- to recalculate only the ideal classes of the keyed in campaign n° without recalculating the statistical data as a sequel to a change of setting.
To do that, there should be no record of the campaign reassigned.
The system keeps the existing campaign and recalculate only the ideal classes.

 the container to be processed
The processing can be carried out for a given container or for all the containers of the site being processed (by leaving the field blank)

 a reference of the campaign to be launched

the operations to be carried on the campaign to be launched
The possible operations are:    : if the box is ticked, includes in the statistical calculations, the sales forecast set onthe output forecasts tablefor the processing period. ‘  and Total of forecasts
In the case of recalculation of ideal classes without recalculation of statistical data, the two boxes Total of archives and Total of forecasts should be unticked.  
-Ideal rotation class: if the box is ticked, the system defines the ideal rotation class according to the calculated statistical data.
In the case of a new campaign launching, the calculation of a new ideal rotation class is compulsory.    
In the case of a new campaign launching, the calculation of a new ideal volume class is compulsory.
The user should ask the calculation of at least one of the two ideal classes (rotation or volume).    
-Total of archives: if the box is ticked, includes in the statistical calculations, the picking output movements and the replenishments/transfers carried out during the period of processing. 
-Total of forecasts
If it is a new campaign launching, the user should at least tick one of these two boxes Total of archives’‘’ ‘’‘’
By default, the system takes into account the archives and the forecasts in the calculation of the classes.  
In the case of recalculation of classes without the recalculation of statistical data, there is the possibility or not to ask for the calculation of the ideal rotation class. 
-Ideal volume class: if the box is ticked, the system defines the ideal volume class according to the calculated statistical data.
In the case of recalculation of classes without the recalculation of statistical data, there is the possibility or not to ask for the calculation of the ideal volume class.
By default, the system always calculates the two ideal classes.

 the dates of processing.
There are two types of dates:
- the accumulation date ranges corresponding to the processing period  of the calculation of statistical data (archives + forecasts)
By default, the system proposes, for the start date of the accumulation, the date of the day – the number of calendar days of the calculation period set on the site  and for the end date, the start date of the accumulation + the number of calendar days of the calculation period. 
- the reassignment date ranges corresponding to the period wherein the completed picking reassignments will take effect.
This period of reassignment is used to calculate the seasonality coefficient.
By default, the system proposes for the start date of reassignment the date of the day and for the end date, the start date of reassignment + the number of calendar days of minimum activity before reassignment set on the sites table.

The date ranges can be keyed in only in the case of a new campaign launching.

 the cutting-of the campaign in several picking rearrangements according to physical criteria..
- By clicking on the field ‘Store’, it is possible  to create picking rearrangements for the processed item containers that have an existing picking location in the same store.
- By clicking on the field ‘Aisle’, it is possible  to create picking rearrangements for the processed item containers that have an existing picking location in the same aisle.
- By clicking on the field ‘Depositor’, it is possible  to create picking rearrangements for the  processed item that have the same depositor.
- By clicking on the field ‘Family’, it is possible  to create picking rearrangements for the  processed item that have the same family 
- By clicking on the field ‘sub-family’, it is possible  to create picking rearrangements for the  processed item that have the same sub-family 
- By clicking on the field ‘group’, it is possible  to create picking rearrangements for the  processed item that have the same group
- By clicking on the field ‘sub-group’, it is possible  to create picking rearrangements for the  processed item that have the same sub-group

After the launching of the process, the system generates :

 a log file with the number of the generated campaign or with the list of errors..

 a campaign cut into picking reassignments according to the criteria set during the launching of the process.
Each picking reassignment is consist of records corresponding to item containers to be reassigned and detailing the statistical data and the ideal classes calculated by the process.
These records can be viewed in the  Picking rearrangement  function.

Field descriptions

Block number 1

Site (field FCY)

Code of the logistics site.
PR campaign no (field MAHNUM)

Numerical sequence automatically assigned by the system

upon creation of the picking replanning campaign (upon launching of the

calculation of the ideal classes).
Container (field CTR)

The term Container encompasses the type and code of the container.
Reference (field REF)

Reference of the picking replanning campaign.

Operations

Archive total (field HIS)

If this box is checked, the output picking and replenish./transfer movements carried out during the processing period are taken into account in the statistical calculations

Forecast total (field FRT)

If this box is checked, the sales forecasts set up in the output forecast table for the processing period are taken into account in the statistical calculations.
Ideal rotation class (field ABC)

If this box is checked, the system defines the ideal ABC class based on the calculated statistical data and the set-up.
Ideal volume class (field VOL)

If this box is checked, the system defines the ideal volume class based on the calculated statistical data and the set-up.
Detailed log file (field TRA)

Sort

Single store (field SNGSTO)

If checked, GEODE generates replannings by entity upon generation.

In case of single store, then 1 RP by store, etc.
Single aisle (field SNGAIS)
Single depositor (field SNGDEP)
Single family (field SNGFAM)
Single sub-family (field SNGSFA)
Mono group (field SNGGRU)
Single sub-group (field SNGSGR)

Date range

From totals (field STRSTT)

The ideal class calculation is performed from enterable dates.

The processing takes these dates into account in order to identify the movements carried out during this period.

If it is not entered, the start date is by default the current date - the number of calendar days of the calculation period set up in the site table,

the end date being the current date.
To totals (field ENDSTT)
From reallocations (field STRRAL)

The reassignment calculation relating to the picking locations of ideal classes is carried out from enterable dates.

This reassignment period is used to calculate the seasonality coefficient.

If it is not entered, the start date is by default the current date,

the end date is the current date + the mini. number of calendar days for the activity before reassignment set up in the site table.
To reallocations (field ENDRAL)

Batch task

This function can be run in batch mode. The standard task MAPUPD is provided for that purpose.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Error field ‘Site’ : Unknown data
The keyed in value is unknown on the sites table

Error field ‘PR campaign n°’ : Picking already reassigned : X
The rearrangement picking n° keyed in already had a record reassigned and therefore cannot be relaunched.

Error field ‘PR campaign n°’ : Rearrangements picking : X, Unknown data
The value keyed in is unknown in the picking rearrangements table of the keyed in site.

Error field ‘Container’ : Containers : X, Unknown data
The value keyed in is unknown in the containers  table of the keyed in site.

Container : Y X : No movement
For the period of  ideal classes calculation process, the system cannot find any archived or forecasted movement for the container of the campaign.

Container : Y X : pending rearrangement picking campaign for this container 
There is already  a rearrangement campaign being processed for the container.

Container : Y X : Prompting without rotation class : Z
The mean prompting per day calculated for an item container of the campaign  does not allow to identify an ideal rotation class through the theoretical method because this prompting belongs to no prompting interval set for each rotation class on the rotation class table.  

Container : Y X : Volumic flow without volume class : Z
The mean picking volumefor an item container of the campaign does not allow to identify an ideal volume class through the theoretical method because this prompting belongs to no prompting interval set for each volume class on the volume class table. 

Container : Y X : ideal rotation class : No more available location for the item Z
The number of available locations is not enough in relation to the number of item containers to be reassigned and therefore the system cannot attribute the ideal rotation class to the item container specified by the method of the existing locations.

Container : Y X : ideal volume class : No more available location for the item Z
The number of available locations is not enough in relation to the number of item containers to be reassigned and therefore the system cannot attribute the ideal volume class to the item container specified by the method of the existing locations.
