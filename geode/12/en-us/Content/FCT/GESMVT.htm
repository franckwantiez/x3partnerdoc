Movement Codes

This screen is used to create, display, modify and delete a movement code. It is associated with each flow and it allows each of these flows to be identified.

The movement code is used to draw the distinction between the flows, and also to trigger an interface with respect to the level 3.
Eg.: distribution input, factory replenishment, external purchase, etc.

Some movement codes can be used by default in some processes. For instance, for a manual transfer with qualitative change, an automatic adjustment movement is generated with the setting up of the movement code used in the stock configuration.

Prerequisite

Refer to documentation Implementation

Screen management

The movement code is created in record mode, it can be modified, including if there are flows that use it. It is not possible to delete a movement code if there is a flow that uses it, irrespective of whether it is archived or not.

Entry screen

To create a movement code, a code is assigned to it that identifies it in a unique manner with 3 alphanumerical characters, and a name.
Two groups of flows are used:
the flows relating to the products: Input, Transfer, Manual adjustment, Automatic adjustment, Output, Assembling, Disassembling
the flows relating to the consumables: Consumables

  • When a movement code of the adjustment group is entered, the adjustment type can be entered.
  • When a movement code of the consumable group is entered, the consumable type can be entered.
  • Finally, the generation of a report (interface) is dependent on the N3 transmission setting-up. The stock transfers are not submitted to a report.

Field descriptions

Block number 1

Mvt code (field MVT)

The movement code is used to make the distinction between movements of the same type.
It is required when creating:

  • An expected input
  • A supplier input
  • A factory input
  • A customer return
  • A DO
  • An adjustment
  • A transfer
  • A consumable input/output

The movement code is checked against the movement code table.
Description (field MVTDESAXX)
Short description (field MVTSHOAXX)
Active (field ENAFLG)

Select this check box to activate the current record.

Disabled records keep their content and setup but cannot be used by recalling their code:

  • On other records such as documents and settings
  • On mass processes

The authorizations for a given function can prohibit the creation of an active record. In this case, the check box is disabled by default. It can only be modified by an authorized user or through a signature workflow.

Block number 2

Processing family (field PROFAM)

Movement type associated with the movement code. Input, Transfer, Manual Adjustment, Automatic Adjustment, Output, Consumable, Assembling and Disassembling.
Checked with respect to the entry context (eg1: movement code of the 'input' group entered to create an input; eg2: movement code of the 'consumable' group selected to carry out a movement relating to a consumable product.
Adjustment movement type (field ADJTYP)

Adjustment type:

Addition to container: used to create an object by linking it to another object
Creation: used to create a stock object
Qualitative: used to modify one of the characteristics of a stock object
Deletion: used to delete a stock object
Quantity increase/decrease: used to modify the quantity of a stock object
Consumable type (field CSBTYP)

Movement type of the consumable:
     Input: incrementation of the consumable stock
     Output: decrementation of the consumable stock

Interfaces

Transmission L3 (field TRMPRO)

Transmission of an interface report for the expected inputs (XEIN), the direct inputs (XINP), the adjustments (XAJM) and the delivery orders (XDLO, XSHP).
