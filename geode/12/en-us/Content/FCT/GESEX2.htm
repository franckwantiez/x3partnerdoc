Cancellation by DO

Use this function to fully or partly cancel the shipment of an unpacked delivery order (DO).

This function is useful when you want to irrevocably delete a DO shipping, either because the final customer canceled the delivery, or because the order was damaged between the preparation and shipping phases.

For intersite shipments, you can cancel a DO soon to be "pending transfer" (or "in transit"), as long as the related intersite round has not been shipped. In other words, you cannot delete the order when it is effectively "pending transfer".

If you want to cancel the shipment of a packed DO, use the Cancellation by parcel function.

Unlike the Postponements by DO function, this cancellation irrevocably deletes the DO preparation, whereas the postponement will only push it to a later date.

A movement code of automatic adjustment type must be defined in the movement code table for the current site.

The DO must have the 8-Pending shipmentstatus and must not be at the prepacking or declarative packing phases.

Screen management

The screen includes a single entry section.

Entry screen

After selecting the DO to be cancelled, the DO lines and related output movements are displayed.

The following information is displayed on each DO line:

- Prepared: The prepared CU quantity, validated on PO validation.

- Shipped: The previously shipped CU quantity.

- Canceled: The CU quantity deleted in the DO cancellation function.

- Postponed: The CU quantity to be reintegrated or postponed.

- To be cancelled: This field depends on the quantity to be deleted on the associated output movements.

- To be shipped: The quantity to be shipped during the next shipment. This field depends on the quantity to be canceled on the DO line.

- In transit: The quantity in CUs of a DO "pending transfer" (or "in transit"). This means the DO is to be shipped to another site. The goods cannot be canceled as long as they are "in transit".

For each output movement line, for the selected DO, the following information is displayed:

- PAC quantity: The PAC quantity of the output movement.

- To be cancelled quantity: Use this field to enter the quantity to be canceled on the selected output movement.

- Destination container: Use this field to enter the container where you want the stock to be reintegrated.

Follow these steps if you want to partly or fully cancel a DO:

- Enter the quantities to be canceled on each output movement. If the product manages serial numbers, enter the localized serial numbers to be canceled using the Serial number entry action on the output movement.

- Enter the address for the stock reintegration.

- Enter the movement code to be used.

- Click Save to validate.

The automatic adjustment type movement code is used to create the corresponding stock objects in the reintegration store.

A report can be printed automatically, listing all reintegrated elements.

Generally, the stocks will then be reintegrated into reserve stores / addresses by creating transfer movements.

Only "normal" lines or kits can be edited.

When entering a kit, the related component lines are selected by default.

If you're performing a partial cancellation, the output movements will then be updated and split. The status of the cancelled movements moves to status 6 - Cancelled upon shipment.

Depending on the DO balance management, the DO will move to status 2 - Pending nth preparation, or to status 9 - Shipped, Completed.

Canceled movements are not taken into account when shipping the DO.

If serial numbers are managed, the cancellation will only be possible if the serial numbers to be cancelled are specified.

To enter these numbers, select the output movement and click the Serial number entry action.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Partial delivery prohibited.

The setup defined in the DO header does not allow you to cancel and split the shipment.

‘Number’ field error, Delivery Order: 9999, Record does not exist

This DO number does not exist.

Serial no. not prepared

The entered serial number has not been prepared and therefore cannot be canceled.

"Reintegration location" field error, mandatory field

The entry of a reintegration location is mandatory.

"Reintegration location" field error, Stores: XX: Record does not exist

The entered reintegration store does not exist.

"Reintegration location" field error, RCA: Type of store <> Dock

The entered reintegration store is not set to 'Dock'.

Incorrect format

The format of the reintegration address is incorrect.

Locations: QUA Q3, Record does not exist

The entered reintegration location does not exist.

"Adjustment mvt code" field error, mandatory field

The entry of an adjustment movement code is mandatory

"Adjustment mvt code" field error, Movement code: A, Record does not exist

The entered adjustment movement code does not exist.

"Adjustment mvt code" field error, E01: Mvt code prohibited.

The entered adjustment movement code must be set to 'Automatic adjustment'.

Number of serial nos. <> Number of CUs to be cancelled line no. X (0<>1)

The number of serial numbers to be cancelled is different from the quantity to be cancelled for a DO line managing serial numbers.

'To be cancelled' field error. Maximum quantity XX CUs

The quantity to be canceled must not be larger than the quantity to be shipped minus the quantity in transit (pending transfer).
