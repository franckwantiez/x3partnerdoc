Carrier appointment

Based on the notion of "Dock assignment" common to receipt and shipment activities, this function is used to:

  • Manage the dock assignment planning.
  • Manage the scheduling of appointments with carriers for receipts and shipments.

A dock assignment corresponds to an activity type (generally, receipt or shipment) for a dock and a planned or actual period. It can also be linked to an expected input (EI) or to rounds.
The carrier appointment can manage multiple depositors.

Appointment creation is made in two steps:

  • creating the appointment general information.
  • assigning EIs to be received or Rounds to be shipped.

In order to create a carrier appointment, the following is required:

  • Number: Number assigned by the system and used to identify the carrier appointment in a unique way within a site. If this number is entered, the characteristics of the existing carrier appointment are displayed.
  • Dock: Dock related to the assignment. Must correspond to a store of the "dock" type, defined for the site. Mandatory if the "Dock assignment management" is set to "Yes" for the current site.
  • BP block: is used to specify if this appointment is exclusively dedicated to one BP (depositor, receipts supplier, customer for shipments)
  • Transport block: is used to specify the transport data (carrier, transport reference...)
  • Truck tracking block: is used to track the truck before the actual arrival at the warehouse dock. This data is freely entered and modified by the user, at any time of an appointment cycle.
  • Dock assignment block: is used to track the planned and actual dock assignment.
  • Load block: is used to define and manage loads on the appointment.

Possible actions:

  • Declaring the dock assignment for appointments at the "Planned" status. The appointment moves to the "In progress" status and the actual dock assignment start date is automatically updated.
  • Declaring the dock release for appointments at the "In progress" status. The appointment moves to the "Completed" status and the actual dock assignment end date is automatically updated.

Deleting a carrier appointment

To delete a carrier appointment, select the appointment and click Delete.
No control is applied to the EIs or rounds included in the appointment.


Modifying a carrier appointment

The information related to the appointment can only be modified for appointments with the "Planned" status (except for the Truck tracking block).
To assign or cancel the assignment of an expected input or round for a given appointment: go to the corresponding tab, click the Actions icon, and click Line deletion.

Specifies the carrier appointment type:

  • "Receipt": the appointment only manages the arrival of expected inputs on the docks
  • "Shipment": the appointment only manages the shipping of rounds on the docks.
  • "Other": the appointment manages the other activities (outside of receipt and shipment) on the docks.

Status (field RDVSTA)

Specifies the carrier appointment status:

  • "Planned": the appointment is created but the truck has not reached the dock yet.
  • "In progress": the truck has reached the appointment dock.
  • "Completed": the truck has left and has freed the dock.

Tab EI list

This tab is used to select the EIs to be assigned to the appointment. Two methods are available:

  • Enter the EI number in the EI number column
  • Use the selection panel of EIs to be received and select the EI by picking. The EI tab then updates your selection.

EIs are grouped by provisional receipt date and time, and then detailed by depositor, number and excepted input reference.

Field descriptions

Grid

Numerical sequence automatically assigned by the system upon validation of the Input header. This number features an information specific to the sys tem, all depositors included. This piece of information features a unique access key to an Expected Input.

Depositor (field DEP)

Code representing the depositor. In logistics terms, the concept of depositor makes it possible to process separate flows or traffics within the same site. It covers the notion of order giving customer for the service provisions.
After the site, the depositor is the second access key to the Product, Input and Delivery Order files.
Concerning the location, the depositor code is entered if the location is assigned to the depositor or the product, and if the location is single-depositor or single-product and it is not empty.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Store field error, mandatory field

The store of the "dock" type must be entered. Mandatory if the "Dock assignment management" is set to "Yes" for the current site.

X3 application error, mandatory field

If the carrier code is not selected, a description must be entered.

Carrier field error, record does not exist.

An attempt has been made to enter a non-existent carrier code.

Transport method error, record does not exist.

An attempt has been made to enter a non-existent transport method.

X3 application error, no line entered

At least one EI must exist in order to create a carrier appointment.

X3 application error, Record already exists

Attempt at entering a carrier appointment, with the same date, time and dock as an existing appointment.

Warning, Overlapping with appointment XXXXXX

The created carrier appointment will overlap with another carrier appointment.

Status not compatible (X-status description)

The EI status does not allow the EI receipt (status 8).
