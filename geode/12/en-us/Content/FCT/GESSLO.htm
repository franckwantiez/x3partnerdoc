Locations

Use this function to view, edit or delete locations.

Some fields are only available when the location is empty, such as: the storage type, the reference container, the product and the assigned depositor.

Note: You can only "destroy" a location if it has no associated stock and if it is not included in a non-archived movement.

Header

Field descriptions

Block number 1

Address of the location in the store.

Store (field STO)

Code used to identify the store. This code is controlled in the store table.
Type (field STOTYP)

The store type defines the logistical function of a store:

  • main store: picking, reserve, mass, junk
  • single address store: logical store in which the stock is located. A single address without fill control can be defined for this store.
  • splitting store: store type necessary to use the "merging of flows" functionality (preparation in 2 phases). The "splitting area" locations can be considered as reserve locations only used as temporary picking locations.
  • dock store (receipt, shipping): used by the "Carrier appointment" and "Stock on quay of logistical units" functionalities.
  • Cross-docking store: used by the "Cross-Docking" functionality.
  • Buffer store: Store for temporary storage.
  • 2-phase transfer store: used by the "2-phase transfer" functionality.
Active (field ENAFLG)

Select this check box to activate the current record.

Disabled records keep their content and setup but cannot be used by recalling their code:

  • On other records such as documents and settings
  • On mass processes

The authorizations for a given function can prohibit the creation of an active record. In this case, the check box is disabled by default. It can only be modified by an authorized user or through a signature workflow.
Address (field ADD)
field ADDSTC
Putaway sort (field SRGADD)

This field is used to keep a second address for each location, to be used systematically during the search for input locations, with a view to obtaining a different filling mode from the address alphanumerical order. The structure of the storage address mirrors that which is set up in the store table.
field SRGSTC

Block number 2

Storage type (field STY)

The storage type defines the management characteristics of a group of locations. These management rules are implemented during an addressing phase on input.
Type (field TYP)
Depositor (field DEP)

Code representing the depositor. In logistics terms, the concept of depositor makes it possible to process separate flows or traffics within the same site. It covers the notion of order giving customer for the service provisions.
After the site, the depositor is the second access key to the Product, Input and Delivery Order files.
Concerning the location, the depositor code is entered if the location is assigned to the depositor or the product, and if the location is single-depositor or single-product and it is not empty.

 
Product (field ITM)

A location can be assigned to a product in the following cases:

- If the location has the "Picking" type, it must necessarily be assigned to a product to be activated.

- If the location has the "Reserve" type, and the storage type forces said location to be reserved to a product.

- If the location has the "Reserve" type, this field mentions the product code specific to the object stored at this address, it the storage type of the location forces said location to be single-product.

- If several products are present at the same time in the lcoation or the latter is empty, the field is left blank.

A location can only be dedicated to a product if it is reserved to a depositor.
Reference cntr. (field REFCTR)

The reference container of a location represents the priority container dedicated to the location and in which the occupancy concepts are expressed. The location can accommodate other containers than the planned reference container, by means of the "container matchings".

 
Level (field REFCTRLVL)

Tab General

A few notes on:

  • Blocking: Picking locations do not have a depositor or assigned items, which means that they are automatically blocked on input and output (field disabled).
  • History: First input or arrival, last input or receipt, last stock count. These dates are generated automatically. They respectively correspond to the input date of the first stock object in this location, to the input date of the last object in this location, and to the closing date of the last inventory.
  • Dimensions: These are the physical dimensions of the location. They are not taken into account when storing products on locations. The reference container is used to run calculations in this context.

Control of dimensions (field DIMCTL)

Indicates the dimension control type performed during the stock introduction (on input, transfer/replenishment) into this location:


  • "No": no control
  • "Height": a control checks that the stock container height is lower than or equal to the location height
  • "Height + (Width <=> Depth): the 3 dimensions are checked, with a possible 90° rotation of the stock object (width becomes depth and depth becomes width). As long as the rotation is valid, the stock can be accepted on the location.
  • "Height + Width + Depth": the three locations are checked without rotation.
  • "Width <=> Depth": the width and depth are checked with a possible 90° rotation of the object
  • "Width + Depth": the two dimensions are checked without rotation.

Tab Picking

Additional information on:

  • Stock objects: For locations in single-lot management (set up from the storage type), the lot number is the one assigned to the stock objects stored on the location at a given time.
  • Dynamic picking: The PR no. corresponds to the current "Picking rearrangement" (PR) for this location; the Status field indicates the current status of the PR for this location.
  • Quantities:
    • In stock: Quantity in stock on this location expressed in reference containers. The available quantity includes the pending input and picking replenishment movements.
    • Occupation rate: Occupancy percentage of the location (100 * stock quantity / max. quantity).
    • Pending input, Pending output, Reorder in process: These are the quantities during the input, output and reordering processes that have been addressed but not validated yet.

Complete objects addressing threshold (field INPCTLBIN)

The goal here is to prevent incomplete objects to be placed in reserve when the picking location is full.

You use this field to limit the number of complete objects on the picking location, based on the product container setup.

This field is taken into account during the addressing on input and on preventive or stock-out reordering.


Example

Let's consider a level-5 (CUU) picking location:

- The product has a full pallet of 1 PP1 and 25 CUUs.

- The addressing threshold for complete objects = 50 CUUs

- The occupation rate of complete objects = 0%


1/ Let's address a pallet movement on input for 1 PP1 and 25 CUUs (complete), which easily fits in the picking location:

- Stock = 25 CUUs

- Addressing threshold for complete objects = 50 CUUs

- Occupation rate of complete objects = 50%


1/ Let's now address a second pallet movement on input for 1 PP1 and 25 CUUs (complete), again fitting in the picking location:

- Stock = 50 CUUs

- Addressing threshold for complete objects = 50 CUUs

- Occupation rate of complete objects = 100%


1/ Finally, let's address a third pallet movement on input for 1 PP1 and 25 CUUs (complete). This time, this does not fit in the picking location because the addressing threshold for complete object has been reached.


4/ If we now address a pallet movement on input for 1 PP1 and 10 CUUs (incomplete), this movement does fit in the picking location:

- Stock = 60 CUUs

- Addressing threshold for complete objects = 50 CUUs

- Occupation rate of complete objects = 100%

Management

Picking mvt in superior cntr. (field CTRSUP)

Used to specify in which palletization program (or container), the system will express the output movements to be picked from the picking location.
It is a maximum container, the container used in the movements will depend on the quantity to be issued. If they are not entered, movements are expressed in the reference container of the location.

Conditions of use: (to express an output movement from a CU picking location in CC1 superior container)
- the product must not have a CC1 picking location
- the CC1 container must be set up in the CU picking location of the product, in the 'mvt in superior cnt' field
- the CC1 container of the product must be "packing pattern"

Here are a few examples:let us consider a product with the following palletization program: 1PP1= 2CC1= 24cu, 1 CC1= 6BB1= 12cu, 1 BB1= 2 cu

Example 1a: cu picking, BB1 modulus
Request for 5 cus, if flag 'Picking mvt in superior cnt' is set to Yes, then creation of 1 mvt of 2 BB1 and 1 mvt of 1 cu to be picked from the uc picking location.
Request for 5 cu, if flag 'Picking mvt in superior cnt' is set to No, then creation of 1 mvt of 5 cu.

Example 1b: uc picking, CC1 and BB1 modulus
Request for 15 cu, if flag 'Picking mvt in superior cnt' is set to Yes, then creation of 1 mvt of 1 CC1 , 1 mvt of 1 BB1 and 1 mvt of 1 cu to be picked from the uc picking location.

Example 2: 3 picking: CC1, BB1 and CU
Request for 6 BB1, if flag 'Picking mvt in superior cnt' is set to Yes, then creation of 1 mvt of 1 CC1 to be picked from the BB1 picking, but since there is a CC1 location, the original mvt of 6 BB1 is kept.
Picking pivot (field DYNPIC)
Picking priority (field PIOPIC)

This field is used to display, assign, and modify, a picking priority to a container + product pair. The assigned locations are sorted by priority (numerical field). The smallest number corresponds to the highest priority.

Dynamic Picking

Temporary picking (field TPYPIC)

Stipulates if the handled picking location is temporary or permanent.

The concept of temporary picking is used in the dynamic picking module in order to determine whether or not the the stock of the former picking location of a product container needs to be transferred to the new picking location.

The stock transfer takes place if the new picking location is permanent.

Tab Reserve

Further information on:

  • Stock objects: This block contains the characteristics of the stock object on the location, according to the storage type configuration:
    • Container: Displays the container type of the stock object if the Single container check box is selected in the storage type configuration.
    • Source: Displays the origin of the stock object if the Single origin check box is selected in the storage type configuration.
    • Stock status: Displays the stock status of the stock object.
    • Lot no.: Displays the lot number of the stock object if the Single lot check box is selected in the storage type configuration.
    • Reservation no.: Displays the reservation number of the stock object if the Single-reservation check box is selected in the storage type configuration.
    • FIFO date: Displays the stock FIFO date of the stock object.
    • Cntr. coef.: Displays the homogeneous container level for the container when the location is set with homogeneous containers (see the storage type configuration).
  • The quantities:
    • In stock: Number of stock objects present in the location. The available quantity includes the pending input and picking replenishment movements.
    • Occupation rate: Occupancy percentage of the location (100 * stock quantity / max. quantity).
    • Maximum current cntr.: Maximum capacity of the location calculated by the system. This takes into account: container matches, the control on filling based on the storage type, the product and container of the location, and the stock objects stored in the location.

Tab Dock

Use this section to define the characteristics and access rules of a dock location when managing carrier appointments.


This section includes:
  • The dock characteristics (type, maximum load, current carrier appointment...).
  • A table displaying the list of authorized carriers for this dock.
  • A table displaying the list of authorized customers for this dock.
  • A table displaying the list of authorized suppliers for this dock.
Field descriptions

General

Dock type (field TYPQAY)

Specifies the dock type(receiving dock, shipping dock or General):

  • A receiving dock can only manage receiving carrier appointments.
  • A shipping dock can only manage shipping carrier appointments.
  • A general dock can manage any type of carrier appointments (receiving, shipping or other).
Dock leveler (field NIVQYM)

Indicates if the dock address includes a dock levelling system (used to fill in the space between the truck and dock or to compensate for a difference in height).

Dock height (field HEIQYM)

Indicates the dock's height. (only available for entry if no dock leveler is used).

Maximum load (field CHGMAX)

Indicates the maximum dock capacity in load unit (defined on the site). If this field is blank, the capacity is unlimited.

This parameter is only considered when the load analysis is used. It is used to avoid assigning carrier appointments to docks that cannot support the related loads.
