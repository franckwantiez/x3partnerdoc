Declarative packing

This screen is used to realize a DO declarative packing, i.e. to declare in the IS the parcels used to pack the DO.

This operation is necessary to all DOs whose preparation mode is in declarative packing mode.

A DO declarative packing can be deleted and re-created as many times as necessary without consequences.

To perform a DO packing:
 the DO preparation must be validated by options  Unitary PO validation or  Mass PO validation  
 The DO must be at status 4-Preparation under progress and have at least one validated output movement, or at status 5-Pending packing: an output movement that must be validated (status 8 or 9 depending on the context) to be packed.

The screen is sub-divided into a tab identifying the DO, another tab defining the parcel and a third tab enumerating the movements of the selected parcel.

Header

To pack a DO, it is necessary to select it via the left list or by directly entering the DO number.

The tab then presents the selected DO indentification information: the DO number, the DO reference, the wave number and the status.

The DO Number is automatically assigned to the creation of the entity.

The status code provides the status progress of the DO.

Tab Packing

The tab is made of two blocks:

  • The 1st block (parcel header) lists the DO existing parcels. If the DO has not been packed yet, the grid is empty.
  • The 2nd block (parcel details) lists all the movements of the parcel selected in the 1st block or all the DO movements to be packed.

To create a parcel, enter the routing and the parcel type used in the first block, then select the movements in the second block.
The movements can only be selected if the parcel is entered in the 1st block.

The "Ship" parameter indicates whether the realised parcel can be shipped as is or not.
If the flag is entered to "yes", the container of the selected movement can be shipped as is.

Depending on the output configuration, the actual weight will be compulsory and will have to be entered.

It is possible to add or to delete a parcel movement without deleting it. Just select the parcel to be modified, select the movement to be added or uncheck it to deledte it.

Menu Bar

  • Auto
  • Delete

Auto

The "Auto" button is used by the system to detect the DO parcels that can be shipped as they are.

A window indicating the number of DO parcel that can be shipped as they are is displayed.
This suggestion can either be accepted or amended by entering a number of parcels.

Delete

The "Delete" button is used to delete a parcel after selecting it and so restart the declarative packing.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Movement cannot be selected: container not shippable as is

The container of the selected product is not defined as being shippable as is.

Maximum quantity

Entered quantity higher than the CU quantity of the movement.

Product container: Record does not exist.

The entered container is not defined in the palletization program of the product.

Imposed range for the DO line

A parcel range has been transmitted for the DO line and the entered range is different.
