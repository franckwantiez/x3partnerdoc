Picking modes

This screen is used to create, display, modify or delete a Preparation Mode.

The Preparation Mode regroups all the management rules linked to the DO preparation phase. It is used to determine the processing mode upon preparation launch.

Prerequisite

Refer to documentation Implementation

Screen management

To create a preparation mode, a code is assigned to it that identifies it in a unique manner with 2 character and a title. Then the various characteristics are entered.

Entry screen

The information that can be entered in this screen relates to:

  • The wave management.
  • The establishment of the POs.
  • The load splitting.
  • The load preparation.
  • palletization
  • "single-lot" shipment of the DO lines.
  • The multi preparation area.
  • The print-outs.
  • The sequence.
Field descriptions

Block number 1

Code representing the preparation mode. The preparation mode regroups all the management rules linked to each DO. It makes it possible to specify th e processing mode of said DO upon preparation launching.

Picking mode (field PMD)
Description (field PMDDESAXX)
Short description (field PMDSHOAXX)
Active (field ENAFLG)

Select this check box to activate the current record.

Disabled records keep their content and setup but cannot be used by recalling their code:

  • On other records such as documents and settings
  • On mass processes

The authorizations for a given function can prohibit the creation of an active record. In this case, the check box is disabled by default. It can only be modified by an authorized user or through a signature workflow.

Wave

Concerning a DO line holding a reservation number, the system only accepts to issue quantities having the same number. In case of shortage on a reservatio n number, the balance is never taken from non-reserved stock.

When a DO line displays a FIFO date in the FIFO date field, the system looks for all the stock objects having a date strictly equal to the one mentioned. I f the parameter is not checked, the system looks for a date later than or equal to the one on the DO line.

Triggers the flow merging management at preparation mode level.

Exclusive reservation (field RSAXCL)
Exclusive FIFO date (field FIFDATXCL)
Date contract (field CONDAT)
Multiple picking (field MLTPIC)

If the check box is selected, the product will be managed in multipicking. This mode allows you to create multiple picking locations using the same reference container for the selected product.

Merging of flows (field WAVFLC)
Single lot on output (field SNGLOTMGT)

This parameter is used to trigger the "single-lot" management on output whose principle is to give the ability to address a DO line from a single lot.

Cross docking (field CDKAUZ)

This parameter specifies the cross-docking management type for this picking mode. There are three possible values:

  • "No": no cross-docking
  • "Yes if shortage": upon launching of a wave, a standard addressing is carried out and if there is a stock shortage, a cross-docking search is started.
  • "Priority yes" upon launching of a wave, a cross-docking search is carried out in priority. If no cross-docking stock is found or no cross-docking requirement has been expressed, a standard addressing is then carried out.

Creation of PO

The picking order can be mixed, reserve and picking.

Note: this setup is ignored by the prepacking function that prepares the POs from a parcel grouping.

The reserve picking order is single DO.

The picking order is single DO.

The picking order is single preparation area. At least one PO is then available for each preparation area.

If the PO is multiple preparation are a, a preparation area will need to be defined in the field located on the right. The POs will be printed on the printer dedicated to this preparation area.

The output movements at different levels (pallet, box, CU) are regrouped by container level on the same PO.

The picking order regroups the DOs belonging to the same shipping round.

PO mixed (field MPO)
PO Single DO reserve (field SNGDLORSV)
PO Single DO picking (field SNGDLOPIC)
PO Single pick area (field SNGPKZ)
PO Single output level (field SNGOUTLVL)
One round PO (field SNGDRN)
PO Single ship-to customer (field SNGCLT)

If checked, the POs will be single-recipient, i.e. they will be comprised of DO movements having the same recipient.

Definition of the recipient of a DO: it is the Customer code, Company name (1st line) and postal code combination of the "Ship-to customer" tab of the DO.

Packing

Used to specify the packing type used: no packing, declarative packing or prepacking.

Packing (field PKG)

Dividing of load

The system uses the criteria specific to the preparation areas to split the picking POs.

The system uses the criteria specific to the preparation areas to split the reserve POs.

Picking (field PIKWKLPIC)
Reserve (field PIKWKLRSV)

Prepalletization

Prepalletization (field BEFPLZ)

Used to trigger the prepalletization. This flag can only be accessed if prepacking is activated.

Picking

Handeld grouping criterion (field OUMGRUPMD)

Multi prep area

Kit (field PKZKIT)

This preparation area will be the one used in the case of kit POs (multi preparation areas). The code of the preparation area is checked against the preparation area table.
PO/Parcel (field PKZ)

This preparation area will be the one used in the case of multi preparation area POs (especially the printer used for the print-outs). The code of the preparation area is checked against the preparation area table.

Prints

Flag to mention if the PO print file is systematically generated.

The stock objects with the same characteristics are regrouped in the report to form a single line.

The system prints output labels.

PO (field PKOEDT)
Grouped movements (field PKOEDTGRU)
Parcel labels (field LBLEDT)
SU labels (field SHPEDT)

Sequence

Informs the system of the sequence according to which the POs must be printed: either by Preparation Area - DO (for an area, the sytem prints all the POs c ontaining DOs being the subject of a preparation in this area), or by DO - Preparation Area (for a DO, the sytem prints all the POs for all the areas concerned by this DO. This operation is repeated for all the DOs).

Sequence (field PKOEDTORD)
