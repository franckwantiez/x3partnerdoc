Repacking

This screen is used to repack a DO, in other words, modify the packing of a DO whose preparation mode is the prepacking.

The following actions can be declared:

  • Grouping of the parcels of a DO
  • Splitting of movements in a new parcel
  • Splitting of movements in an existing parcel
  • Change of routing and parcel type

The DO must have the status 4- Pending preparation, or status 6- Pending palletization, or status 7-Pending loading, or status 8-Pending shipment. Moreover the prepacking is only possible for parcels that are neither palletized, nor loaded, nor shipped nor postponed.

The screen is sub-divided into a tab identifying the DO, another tab defining the parcel and a third tab enumerating the movements of the selected parcel.

Header

To repack a DO, it is necessary to select it via the left list or by directly entering the DO number.

The tab then displays the information identifying the selected DO, the DO number, the DO reference, the wave number and the status.

The DO Number is automatically assigned to the creation of the entity.

The status code provides the status progress of the DO.

Tab Packing

The 1st block of this tab (parcel header) enumerates the existing parcels of the DO. Since the DO is in prepacking mode, all the DO movements are packed.

The 2nd block (parcel detail) enumerates all the movements for the DO selected in the 1st block.

Specific Buttons

  • Updated
  • Regroup
  • Split

Updated

Stage 1: Select the parcel to be modified and press the button "Modify".

Stage 2: An entry screen of the parcel information is then displayed. After entering the parcel information, the system asks the user if they want to reprint a parcel label.

Regroup

Stage 1: In the Rgp column, select the parcel no. of the DO to be regrouped and press the button "Regroup".

Stage 2: The movements (with status 3 or 8) are then regrouped in a new parcel. For this new parcel, the system requests the entry of some information, such as the routing, the parcel type and the dimensions and actual weight.

The status of the new parcel will be that of the parcel whose status is the lowest. The dimensions, weight and carrier parcel no. will be updated.

Stage 3: A new parcel label is automatically printed for the newly created parcel if the output configuration of the current depositor specifies it.

Split

Stage 1: Select the parcel to be split in the parcel header block, then select the movements in the Parcel detail block.

Stage 2: It is possible to transfer a whole movement to another parcel or to partly split a movement into another parcel.

To fully transfer this movement, it needs to be checked. Only the normal or kit movements can be selected. In the case of a kit, the associated components are also transferred.

To transfer part of this movement, the quantity to be transferred needs to be specified in the "CU nb" column or in the "Nb" and "Homog cnt" columns. The following controls are carried out on these quantities:

The entered or deduced CU number must be a multiple of the CU number of the homogeneous container of the original container, it must be a multiple of the CU number of the reference container of the picking location and it must be lower than or equal to the CU number of the movement.

Stage 3: Once the elements have been selected, press the button "Split".

Stage 4: An entry screen of the parcel information is then displayed. It is possible to perform a transfer to an already existing parcel of the DO by entering the parcel number, or by performing the transfer to a new parcel.

If the DO line is associated with a specific range of parcels, and the new range entered is different, a non-blocking message warns the user

In both cases, a parcel label is printed if the output configuration of the current depositor specifies it.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Parcel does not exist.

The parcel no. does not exist for the DO.

Grouping impossible, parcel already loaded or shipped.

The parcel status does not allow any modification to the parcel.

Maximum quantity.

Entered quantity higher than the CU quantity of the movement.

Product container: Record does not exist.

The entered container is not defined in the palletization program of the product.

DO line.

A parcel range has been transmitted for the DO line and the entered range is different.
