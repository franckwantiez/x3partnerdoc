Selective replenish

Use this function to generate replenishment orders for a picking location by selecting stock objects to be replenished.

Note: using this function, you can move a picking location stock to another picking location.

Prerequisite

Refer to documentation Implementation

Screen management

The screen includes one tab and an input criteria header as well as a table used to select stock objects.

Entry screen

The information to enter is:

Product/Container: code of the product to be replenished; when the product code is entered, the container to be replenished must be entered.
Information related to the product and product container of the picking location to be replenished is mandatory. The product must not be blocked (based on the output configuration) or in inventory process.

Location: store and address of the picking location. The location can be selected from the product/reference container combination, displayed if only one location exists, or directly entered. The location must not be blocked on input or on output and it must not be in inventory process.

Container nb./CU: based on the container level on the picking location, the CU number defined on the product record is displayed for the selected location.

Replenishment threshold: the replenishment threshold defined on the selected picking location is displayed; the quantity is expressed in the picking location container.

The stock objects available for replenishment are then displayed and must meet the following criteria:
- location <> entered picking location
- location not blocked on output
- location not under inventory process
- store authorized on output
- store of type <> location
- SBD, UBD and SbD dates are null or not set up on the product record or >= date of the day
- the detention date < current date or not set up on the product record
- reservation numbers and the stock nature are not entered
- container >= location container  
- palletization plan includes the location container
- available quantity (object) > 0

It is then possible to perform a unit or mass selection depending on the available space in a picking location.

As soon as a stock object has been selected, the Qty column is initialized to the stock object quantity if the object is available and if the available space on the picking location is sufficient. The quantity is expressed in the reference container of the location. The quantity also includes the minimum output container of the product container for the selected product.

Transfer movement code. Movement code to be used on replenishment It must always be set to "transfer".

Button is then used to create the replenishment. A replenishment label for each selected object and a replenishment list can be generated if the output configuration for the current depositor allows it.

Based on the output configuration, the generated replenishment will need to be validated in the Unit validation function or Mass validation function, or otherwise validated automatically.

Limit to products managed in STK

Replenishments applying to a product in STK, the transfer is only authorized for complete objects in picking. Objects cannot be incomplete.

For example: with a product in STK (cloth roll) with the palletization plan (STK = a meter, 1 RLX or roll = 50.6 meters, 1 PP2 of 20 RLX make 1,012 meters)

Example 1: With a bulk picking (reference container = STK = a meter)
meters, complete rolls or complete pallets can be replenished.
if the following SO are in reserve:
- 1 PP2 of 62.5m => I transfer the complete SO or nothing
- 1 RLX of 50.6m => I transfer the complete SO or nothing
- 1 RLX of 20m => I transfer the complete SO or nothing
- 1 PP2 of 10 RLX (each RLX = 30m so not homogeneous based on palletization plan) => transferring the complete PP2 SO or complete RLX rolls in the pallet.

Example 2: With a roll picking (reference container = RLX)
only standard homogeneous rolls can be replenished.
if the following SO are in reserve:
- 1 PP2 of 62.5m => I cannot transfer this SO
- 1 RLX of 50.6m => I transfer the complete SO or nothing
- 1 RLX of 20m => I cannot transfer this SO (non standard homogeneous roll)
- 1 PP2 of 10 RLX (each RLX = 30m so not homogeneous based on palletization plan) => I cannot transfer these rolls.
- 1 PP2 of 10 RLX (each RLX = 50.6m so homogeneous standard based on palletization plan) => I can transfer part or all of these 10 rolls.

Block number 4

Transfer mvt code (field MVT)

The movement code is used to make the distinction between movements of the same type.
It is required when creating:

  • An expected input
  • A supplier input
  • A factory input
  • A customer return
  • A DO
  • An adjustment
  • A transfer
  • A consumable input/output

The movement code is checked against the movement code table.
