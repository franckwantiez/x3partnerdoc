Movement by Object

This screen allows the consultation of  pending or validated movements of a stock object. The finest tracking is then put in place.

Standardly, this function is called from a more global consultation.

Prerequisite

Refer to documentation Implementation

Screen management

  • Header
  • Tab Transactions

The screen is made up of two tabs : a first tab allowing the selection of stock object to be consulted and a second proposing the list of movements of the selected stock object.

Header

The keying of stock object n° is mandatory. The original stock object n° is displayed. In most of the times, these 2 zones are identical.  
If the stock object n° keyed in is not the original, meaning that it was submitted to modification that generated a new object (transfer), then these 2 N° are different..

A box to be ticked « same container n° » allows the interrogation of the movements of an object conserving the same container n° or not. Therefore allowing to list all the stock movements of the same container n°.  

The depositor, the item code, the location of the stock objects are displayed.

Field descriptions

Block number 1

Identifier created by GEODE upon creation of the stock object.

If this box is checked, this makes it possible to query the movements of an object keeping the same container number, in other words, to list all the stock movements having the same container numbers.

Stock object (field SKONUM)
Same container no (field CTRLIKNUM)

Block number 2

Depositor (field DEP)
Product (field ITM)
Store (field STO)
Address (field ADD)
field TYP

Tab Transactions

 This screen displays all the stock movements according to the keyed in criteria.

 The records appear sorted by movement date.

 The screen has the following data according to the contextual criteria keyed in :

Date
Date of the movement 

Stock object
The stock object n°is displayed if the box « the same container n° » is ticked because it may be different from the stock object asked.

Store/Location 
Store and address of the stock object movement

Type 
The type indicates the movement nature that may take one of the following :
« E » for Entries
« S» for Outputs
« A+ » for creation Adjustments or Add to a container or increase in quantity
« A » for qualitative Adjustments
« A-» for suppression Adjustment or decrease of quantity
« T- » for the pick of an object during the carrying out of Transfer
« T+» for the deposit  of an object during the carrying out of Transfer

Stat 
Status of the movement

Qty and Cnt :
Quantity and Container type and code of the movement.

Homogeneous quantity and Homogeneous Container :
Number of containers inferior to the movement container.

+/-
Depending on the movement type, display the positive or negative meaning.
« E » for Entries => meaning +
« S» for Outputs => meaning -
« A+ » for creation Adjustments or Add to a container or increase in quantity
« A » for qualitative Adjustments => no meaning
« A-» for suppression Adjustment or decrease of quantity => meaning -
« T- » for the pick of an object during the carrying out of Transfer => meaning -
« T+» for the deposit  of an object during the carrying out of Transfer => meaning +

CU quantity : Total CU of the movement

Movement code
Movement codeused to carry out the movement

Nature/lot/reservation/support
Characteristics of the movement

Movement n°
Movement counter, peculiar to each movement type.

Header
TheEntity n° associated to the movement.
For a direct input movement, it corresponds to the direct input n° to which it depends.
For an output  movement, it corresponds to  the DO n° to which it depends.
For a transfer movement, it corresponds to the Transfer n° to which it depends.
For an adjustment movement, it corresponds to the Adjustment n° to which it depends.

User
Name of the user that created the movement.

 the default palletization plan of the stock object of the selected movement is displayed at the bottom of the screen . The total CU is displayed at the bottom of the screen,  it does not include the pending inputs not the pending outputs.

 thru the  tunnel one can accede in a contextual manner to the detail of the movement of the stock object.

Menu Bar

  • Search

Search

Allows to research and display the movements of the selected stock objects according to the criteria keyed in .

Sort

Allows to modify the sorting of the displayed movements.

Allows to view the item record of the selected stock object.

Allows to view the detail of the location of the selected stock object.
