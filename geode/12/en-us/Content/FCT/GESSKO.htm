Stock objects

This screen provides comprehensive information on a given stock object.

Prerequisites

Refer to documentation Implementation

Screen management

The stock object is selected in the drop-down list: the system displays its characteristics in the right part of the screen.

Entry screen

The available information is:

Number Stock object number ; sequence created when creating the stock object.

Type. Store type of the location of the stock object.

Depositor/Product/Description Depositor code, object product code, short description of the product record.

Store/Address/Picking or reserve. Physical storage address of the stock object; location type

Quantities

Pd Input. Total input in progress for the line in expressed in DU.

Pd Output. Total output in progress for the line in expressed in DU.

Container. Quantity and container code of the stock object. The stock container is the container used for the location occupancy.

Homogeneous cnt Number of lower containers for the object (i.e. 3 pallets of 100 DU)

Number of DU Stock total for the line expressed in DU.

Lot no./Reservation/Support/Stock nature/Source. Characteristics of the stock object

Dates

Input, Manufacturing, Detention, SBD, UBD, SbD, FIFO. Key dates of the stock object.

Management

Input number. Number of the direct entry at the origin of the stock object creation; not assigned if the stock object has been created by adjustment.

Adjustment number. Number of the adjustment at the origin of the stock object creation; not assigned if the stock object has been created by direct entry.

Container link no. Upon creation of the stock object, this field is initialized to the stock object number.
During the stock object lifetime in the warehouse, the initial stock object can undergo modifications, and among others, by generating other stock objects (for instance, partial transfer). In that case, the link number containing new stock objects will be updated by the original object number.Therefore, the overall tracking of the object is ensured.

Link movement no. When stock objects have dependency links (Merging without accrual functionality) , this field is entered. This field is updated with the number of the stock object to which it is linked. If this link number is identical to the object number, the object is a "master" object; if the number is equal to another object number, then it is a sub-object.

Allocation Seq This field is calculated by the system based on the output mode of the product. It is used to perform an optimized stock removal of stock objects.

Field descriptions

Block number 1

Identifier created by GEODE upon creation of the stock object.

Physical storage address of the stock object.

Number (field SKONUM)
Depositor (field DEP)

Code representing the depositor. In logistics terms, the concept of depositor makes it possible to process separate flows or traffics within the same site. It covers the notion of order giving customer for the service provisions.
After the site, the depositor is the second access key to the Product, Input and Delivery Order files.
Concerning the location, the depositor code is entered if the location is assigned to the depositor or the product, and if the location is single-depositor or single-product and it is not empty.

 
Product (field ITM)

Product code defined in the GEODE product table.
Description (field ITMDESAXX)
Store (field STO)

Code used to identify the store. This code is controlled in the store table.
Type (field STOTYP)

The store type defines the logistical function of a store:

  • main store: picking, reserve, mass, junk
  • single address store: logical store in which the stock is located. A single address without fill control can be defined for this store.
  • splitting store: store type necessary to use the "merging of flows" functionality (preparation in 2 phases). The "splitting area" locations can be considered as reserve locations only used as temporary picking locations.
  • dock store (receipt, shipping): used by the "Carrier appointment" and "Stock on quay of logistical units" functionalities.
  • Cross-docking store: used by the "Cross-Docking" functionality.
  • Buffer store: Store for temporary storage.
  • 2-phase transfer store: used by the "2-phase transfer" functionality.
Address (field ADD)
Type (field SLOTYP)

The location type determines whether the locations are of the reserve type (the allocation and vacating of the locations are carried out in dynamic mode) or of the picking type (the locations are assigned to a container for the Depositor/Product pair).

Total expressed in CUs of the input pending movements for the object.

Total expressed in CUs of the output pending movements for the object.

Stock object quantity.

The homogeneous container of a stock object is defined as the first container of a level other than level 1 (the pallet) whose CU stock quantity is a multi ple.

Stock total for the object expressed in CUs.

Dates

Date corresponding to the input to store of the stock object.

Production date corresponding to the end of the item production cycle.

Use-by date (SBD). The UBD makes it possible to limit the circulation of a product in time:

- for instance, within the framework of a commercial operation (discount),

- or because of its expiry date.

Input (field INPDAT)
Manufacturing (field MND)
Detention (field RTD)

Detention date. This date corresponds to the instant before which

the stock object is unavailable for an output movement.

(for instance, quarantine).
SBD (field SLD)

Sale-by Date (SbD). The SbD is generally used

as FIFO date for food products.
UBD (field USD)

Best Before Date (BBD). This date is also called

Optimum Use By Date. It is usually mentioned on

the packaging of food or pharmaceutical products.
SbD (field SND)
FIFO (field FIFDAT)

The FIFO date can determine the stock removal order of the products (depending on the setup) when an order picking is launched. This date is used to ensure that some operations like merging are possible.

Storage type (field STLDAT)

Support number(s) of the stock object(s).

It is used to manage the quality-related differences at stock object level: expiry, reconditioning, etc.
