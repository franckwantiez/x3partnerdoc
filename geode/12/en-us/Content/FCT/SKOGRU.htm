Grouped Objects

This screen allows the consultation of stock by item depositor and/or by address.  

Through this function, the records are displayed in groups to facilitate the reading. So, through this function, we can have : 

 Display of grouped stock object by depositor

 Display of grouped stock object by item

 Display of grouped stock object by store

 Display of grouped stock object by address

 Display of grouped stock object by depositor et store

 Display of grouped stock object by depositor et address

 Display of grouped stock object by item et par store

 Display of grouped stock object by item et par address

Screen management

  • Header
  • Tab Stock objects
  • Tab Locations

The screen is made up of three tabs : a first tab characterizing the consultation (by depositor, by item…), a second proposing the list of stock objects and a third displaying the allocated empty picking locations.  

Header

By default, the current depositor t is displayed in the depositor field, but it can be changed depending on the right of access of the user.

If the item field is keyed in, the depositor is mandatory and the system controls that the item really belongs to the depositor keyed in.

If the location field is keyed in, the store to which it belongs should also be keyed in. The nature of the location of the type « Picking » or « Reserve » is displayed.

If the stock is demanded for an address, the maximum quantity of the location is displayed on the right hand corner of the screen, and also the available space in this location. The calculation of available spaces depends on the  storage type  of the location (parameter filling-up control).

Through the tunnel it is possible to attain in contextual manner the item files, unitary locations, unitary stock objects, and Movements consultation.

Field descriptions

Selection

Physical storage address of the stock object.

Depositor (field DEP)

Code representing the depositor. In logistics terms, the concept of depositor makes it possible to process separate flows or traffics within the same site. It covers the notion of order giving customer for the service provisions.
After the site, the depositor is the second access key to the Product, Input and Delivery Order files.
Concerning the location, the depositor code is entered if the location is assigned to the depositor or the product, and if the location is single-depositor or single-product and it is not empty.

 
Product (field ITM)

Product code defined in the GEODE product table.
Store (field STO)

Code used to identify the store. This code is controlled in the store table.

Tab Stock objects

 This screen displays the list of addressed stock objects for the entity defined on the header tab.

 The list displays the not empty picking stocks and the reserve stocks sorted by item/store/address.

 The records appear grouped if they have the following identical characteristics : 
Location
Item
Container
Homogeneous container
Homogeneous quantity and container

The quantities of unitary objects are accumulated.

 The screen has the following information :

- Item :
Item of the accumulated stock of the line

- Store/Location :
Physical address of the storage of the line

- Picking :
Sign of the location type.
If the box is ticked the location is of the picking type, if not it is of the reserve type.

- Blocked :
Sign indicating that the stock objects are blocked.
If the box is ticked then at least one of the grouped stock object of the line is blocked.
The reason of this blockage is displayed by click right  of the mouse and can be one or several of the following :

Item blocked in output
Store not allowed in output
Location blocked in output
Item container not allowed in output
Stock object with reservation number
Stock object with stock nature 
Stock object under retention
Stock object whose SBD is overpassed 
Stock object whose SbD is overpassed
Stock object whose UBD is overpassed

- Linked :
Sign indicating that the stock objects are linked.
If the box is ticked then at least one of the grouped stock object of the line is linked.

- Qty and Cnt :
Quantity and the container type and code of the grouped stock object. The container of the stock is the container type that intervenes in the occupation of the location.

- Homogeneous quantity and container :
Number of inferior containers of each of the object of the line  (for example, 3 pallets of 100 CU)

- CU quantity : Total stock for the line expressed in CU 

- Pd input :
Total of pending inputs for the line expressed in CU

- Pd output :
Total of pending outputs for the line expressed in CU

 The default palletization plan of the line of the selected stock is displayed at the bottom of the screen. The total available CU is displayed at the bottom of the screen, it does not include the pending input nor the pending output.

Tab Locations

 This tab displays the assigned empty picking locations.
Depending on the general context, the data are displayed differently :
- if the display of stock is demanded for an item, the empty picking locations of the item keyed in are displayed.
- if the display of stock is demanded for a depositor, the assigned or not assigned empty picking locations of the keyed in depositor are displayed.
- if the display of stock is demanded for a store, the assigned or not assigned empty picking locations of the store are displayed..

 The screen has the following information :

- Store and address :
Physical store address of the picking location

- Storage :
Storage type of the picking location

- Picking :
If the box is ticked, the location is of the picking type.

- Item :
Item code assigned to the location

- Maximum quantity :
Maximum quantity, expressed in reference container of the location

- Cnt :
Reference container of the location

- Blk I, Blk O :
Blocking parameter of the location. If ticked, indicates that the location is blocked in Input and/or in Output.

- Inventory :
Parameter of a pending inventory of the location file.

Menu Bar

  • Search
  • Criteria
  • Sort

Search

Allows the display of the list of grouped stock objects by taking into account the keyed in selection criteria.

Criteria

Allows to carry out the selections on the stock objects table,  for example, consultation of stock by Lot n°, reservation n° etc…

Sort

Allows to modify the sorting of the grouped stock objects lines displayed.

Menu Bar

Drill Down / Transactions

 Allows to accede to the contextual consultation of the movements :
- if the stock is asked for a  depositor, display of movements of the keyed in depositor,
- if the stock is asked for an  item, display of movements for the item asked.
- if the stock is asked for an address, display of  movements of the address etc…
