Sites

The site is a homogeneous and independent management unit for all the management functions. It is usually equivalent to the concept of warehouse or physical site.

A Warehousing application is used to manage several sites in an independent manner. A Warehousing application that only manages a single site is referred to as "single-site" and a Warehousing application that manages several sites is referred to as "multi-site".

Note: The number of physical sites managed by a Warehousing application can be submitted to contractual limitations.

Tab Dynamic Picking

This tab provides the set-up linked to the dynamic picking module.

It stipulates, for each site:

  • the algorithm used to determine the ideal ABC and volume classes of each product container from the calculated volume flows and promptings (based on the picking availabilities or according to the brackets defined in the tables of the ABC and volume classes)
  • the rules applied during the search of a destination picking from an initial picking: authorized chaining processes (depositor, storage type, store, ABC class, assignment class, volume class) and setting priorities for the search criteria.
Field descriptions

Ideal Rotation class

field ABCCRI

Used to define the calculation mode of the ideal ABC classes. If the parameter is set to location, the possible locations for the reassignment are sorted by ABC class order, from the highest to the lowest (with respect to the maximum number of promptings defined in the ABC class table). Products are also sorted by decreasing order of number of promptings. The ideal class for the product with the most promptings will be the ABC class of the first location (highest ABC class). The next product will have the ABC class of the second location, and so on and so forth. If If the parameter is set to ABC class, the ideal ABC class of a product is determined by comparing the number of promptings for this product with the maximum number of promptings defined for each class in the ABC class table.

Calendar rules

Minimum period of time needed between two reassignments for a product/container pair.

Min delay b/w 2 reallocation (field MINTER)
field DAY2
Min activity before allocation (field MINACT)

This parameter specifies the minimum activity period of the product before the first reassignement. Below this value, the calculation prompting is not significant because the calculation period is insufficient. (this entered number corresponds to a number of calendar days).

This number is also used to calculate:

  • the default date of the reassignment period start for the statistical data calculation.
    When starting the calculation of the ideal classes, the system deduces this number of days from the current date in order to obtain the start date of the reassignment period that is displayed by default on the launch screen.
  • the default date of the reassignment period end for the statistical data calculation.
    When starting the calculation of the ideal classes, the system deduces this number of days from the start date in order to obtain the end date of the reassignment period that is displayed by default on the launch screen.

The reassignment period corresponds to the time during which the reassignment will be effective. This period is used for the weighing calculation linked to the seasonality.

 
field DAY3
Calculation period (field STTPER)

This entered number corresponds to a number of calendar days.
It is used for various purposes:

  • the calculation of the default date of the cumulation period start for the statistical data calculation.
    When starting the calculation of the ideal classes, the system deduces this number of days from the current date in order to obtain the start date of the calcuation period that is displayed by default on the launch screen.
  • the calculation of the default date of the cumulation period end for the statistical data calculation.
    When starting the calculation of the ideal classes, the system deduces this number of days from the current date in order to obtain the end date of the calcuation period that is displayed by default on the launch screen.

During the calculation of the ideal classes, the system deduces the days not worked from the defined calculation period. The days not worked are determined from the calendar for the forecast period and from the movements for the historical record.

Location rules

During a picking reallocation (or allocation), the system looks for the new location by searching for a location with the same depositor.

During a picking reallocation (or allocation), the system looks for the new location by searching for a location with the storage type.

Single depositor (field SNGDEP)
Single storage type (field SNGSTY)
Stores chaining (field STOCHN)

No sequencing of the stores (box not checked): for the ideal class calculation (depending on the existing locations) and for the reassignments, the system will only consider picking locations belonging to the same store as the picking location of origin.
If the product containers do no have any picking locations, the system will consider the picking locations belonging to the priority store of the product container.

Sequencing of the stores (box checked): for the ideal class calculation (depending on the existing locations) and for the reassignments, the system will only consider picking locations belonging to:
- the same store as the picking location of origin.
- the priority store of the product container to be reassigned
- the stores linked to the priority store of the product container to be reassigned.
ABC classes chaining (field ABCCHN)

No sequencing of the ABC classes (box not checked): during the reassignment search, the system only considers picking locations with the same ABC class as the ideal ABC class of the product container.

Sequencing of the ABC classes (box checked): during the reassignment search, the system will consider picking locations with an ABC class:
- identical to the ideal ABC class of the product container.
- linked to the ideal ABC class in the ABC class table.
Chaining assignment classes (field ASGCHN)

No sequencing of the assignment classes (box not checked): during the ideal class calculation (depending on the existing locations) and during the reassignment search, the system only considers picking locations with the same assignment class as the assignment class of the product container considered.

Sequencing of the assignment classes (box checked): during the ideal class calculation (depending on the existing locations) and during the reassignment search, the system only considers picking locations with an assignment class:
- identical to the assignment class of the product container considered.
- linked to the assignment class of the product container considered in the assignment class table.
Chaining volume classes (field VOLCHN)

No sequencing of the volume classes (box not checked): during the reassignment search, the system only considers picking locations with the same volume class as the ideal volume class of the product container.

Sequencing of the volume classes (box checked): during the reassignment search, the system will consider picking locations with a volume class:
- identical to the ideal volume class of the product container.
- linked to the ideal volume class in the volume class table.
Products in stock picking without prompting (field FLGITMSSO)

By default the products managed by picking but without history movements or planning cannot be chosen for the dynamic picking campaign.

When this flag is checked, these product will be taken into account in the campaign and will have a default ABC class and volume class specified in this tab.
ABC classes (field ABCITMSSO)

This fields specifies the default ABC class that will be assigned to the products in picking stock and without prompting that are taken into account in the dynamic picking campaign.
Volume class (field VOLITMSSO)

This fields specifies the default volume class that will be assigned to the products in picking stock and without prompting that are taken into account in the dynamic picking campaign.

Address Priority

field ALLPIO

Upon picking reassignment (or assignment), the system uses these priority criteria to address the transfer.

The six possible downgrades correspond to the six following parameters:

  • 1 = Depositor : when the system cannot find any location during picking reassignment for a given depositor and the "Single-depositor" box is not checked (multi-depositor management), selecting this criterion makes it possible to search for a location defined for another depositor.
  • 2 = Storage type: when the system cannot find any location during picking reassignment with a specific storage type and the "Single storage-type" box is not checked (multiple storage type management), selecting this criterion makes it possible to search for a location with another storage type.
  • 3 = Store: when the system cannot find any location during picking reassignment in a given store and the "Store chainings" box is checked, selecting this criterion makes it possible to search for a location in a store different from that of the start picking (search performed in the priority store of the product container, then in the chained stores).
  • 4 = ABC class : when the system cannot find any location during reassignment of a picking whose ABC class is the same as the ideal ABC class, and the "ABC class chainings" box is checked, selecting this criterion makes it possible to search for a location with an ABC class different from the ideal ABC class (taking into account of those ABC classes that are chained to the ideal ABC class in the ABC class table).
  • 5 = Assignment class: when the system cannot find any location during reassignment of a picking whose assignment class is the same as the assignment class of the product container, and the "Assignment class chainings" box is checked, selecting this criterion makes it possible to search for a location with an assignment class different from the assignment class of the product container (taking into account of those assignemnt classes that are chained to the assignment class of the product container in the assignment class table).
  • 6=Volume class: lwhen the system cannot find any location during reassignment of a picking whose volume class is the same as the ideal volume class, and the "Volume class chainings" box is checked, selecting this criterion makes it possible to search for a location with a volume class different from the ideal volume class (taking into account of those volume classes that are chained to the ideal volume class in the volume class table).

Non-work Days

field HLY

In the calendar management function, when creating a new month, the system will define the default public holidays for this month using these parameters.
