Screen management

  • Header

    The screen is made up of:
  • A left list containing the headings existing for the current X3 folder.
  • A part on the right-hand side with:
    • The heading header describing its main characteristics (code, type, etc..)
    • 2 tabs "Links" and "Parameters" used to create the query.

    The information to enter varies according to the type of heading to define:
    Heading with the type '1-Table':
      • Calculation unit
      • the primary table defining the entity to measure
      • the date field of the primary table, which will be linked to the extracted entry
      • the secondary tables
      • the entries selection criteria
      • the formula used to calculate the value of the heading
    Heading with the type '2-Statistics':
      • Calculation unit
      • The statistical setup and the statistical value to take into account
      • The statistical selection criteria
    Heading with the type '3-Fixed price': no information to enter

    Notes on the management rules of headings:
  • Modification Mode: a heading can only be modified if all associated invoices are validated.
  • Deletion Mode: a heading can only be deleted if it is not associated to any service, price list code or price list.

Header

At header level, the main characteristics of the heading are displayed.

Field descriptions

Description

Heading (field IHHCOD)

Section code defined in GEODE section table.

This section is a service invoicing element such as "the number of DN shipped", "the number of parcel shipped", "the weight of pallets received", etc.
Description (field IHHDESAXX)

Enter the description of the relevant record.

This long description is used as a title in screens and reports.

Characteristics

Active (field ENAFLG)

Select or clear this check box to respectively enable or disable the Sort code.

  • If the check box is selected: you can apply the Sort code to a wave or preparation area.
  • If the check box is cleared: you cannot use the Sort code. If you try to use it, you will receive an error message explaining that the code cannot be used.

Type (field IHHTYP)

Specifies the section type. There are three possible values:

  • 1-Table: the section is based on the Geode tables.
  • 2-Statistics: the section is based on a statistic figure.
  • 3 -Fixed: The section is of fixed price type. It is therefore not based on logistic data from Geode tables or on statistics.
Calculation unit (field IHHUOM)

Indicates the section calculation unit: Length, Surface, Volume, Mass/Weight, Packaging, Times, Numeration, Others...

Table

Table (field IHHTAB)

In the case of a section of type '1-Table', this field indicates the Geode primary table identifying the Geode entity to track.

Stat grp

Stat grp (field IHHSTACOD)

In the case of a section of type '2-Statistics', this field indicates the statistic parameter used.

Statistic value (field IHHSTAFOR)

In the case of a section of type '2-Statistics', this field indicates the statistic value that will be taken into account for the section evaluation.

Operation (field IHHSTAOPE)

This indicates the type of operation that will be used for the calculation of the statistical heading:

  • Mini: The heading is equal to the minimum value among the statistical values for the calculation period.
  • Maxi: The heading is equal to the maximum value among the statistical values for the calculation period.
  • Sum: The heading is equal to the sum of the statistical values for the calculation period.
  • Average: The heading is equal to the average of the statistical values for the calculation period.

Tab Links

This tab is used to create the query for headings with the type "1-Table".

Field descriptions

Reference date

Date (field IHHTABDAT)

In the case of a section of type '1-Table', this field indicates the field of the primary table where the date to which the record is linked can be found.
field IHHTABDATL

Grid Links

Table (field IHLSEC)

In the case of a section of type '1-Table', this field indicates the secondary tables to take into account in the section evaluation.

Abbreviation (field WABRSEC)
Key (field IHLSECKEY)

It defines the table key code used to read the linked lines.

Link (field IHLSECLNK)

This field is defined as one or more calculated expressions separated by a semi colon. Each expression is evaluated, and the result used to identify the value of the key used to perform the join. When multiple joins are authorized, only the first elements of the key can be given.

In the expressions, constants can be used, as well as fields coming from the tables previously defined in the link list.
Priority (field IHLSECPIO)

This field indicates the order of the secondary tables in the query.

Tab Set-up

This tab is used to set up the query for headings with the type "1-Table" and "2-Statistics".

Heading with the type '1-Table':

It is not necessary to add the "site" and "depositor" filters in the selection criteria. On heading calculation Warehousing automatically adds these filter according to the invoicing folder (1 folder = 1 site/depositor).

Heading with the type '2-Statistics'

Contrary to a heading with the type "Table" a heading with the type "Statistics" will necessarily be defined for a given invoicing folder. This stems from the current structure of the statistical setups.

For this type of header the statistical setup used must contain the search criteria "site" and "depositor" otherwise the statistical result could not be filtered using a given site/depositor. There is 2 possible options to specify the statistics:

  • Option 1: the site is entered as criterion in the statistical trigger. Only the depositor field is then left to add in the statistical setup.
  • Option 2: the site is not entered as criterion in the statistical trigger. The depositor and site fields must then be added in the statistical setup.

Afterwards in the heading selection criteria:

  • Option 1: there is no need to enter the site, Warehousing automatically adds this filter according to the invoicing folder. However the depositor must be manually added.
  • Option 2: the site and the depositor must be manually added.

In addition the criteria for the statistics must be filtered, knowing that the value "$$$" means that all the values are selected or, when there is no filter, the blank value are selected by default.

Field descriptions

Criteria

field IHHSEL

In the case of a section of type '1-Table', this field is used to enter the selection criteria for the query defined by the primary/secondary tables.

Formula

field IHHFOR

In the case of a section of type '1-Table', this field is used to enter the calculation formula to use for the section. This formula relies on all fields from the primary/secondary tables.

Stat. criteria

field IHHSTASEL

In the case of a section of type '2-Statistics', this field is used to enter the selection criteria for the statistic fields named CRIx (x = 1 to 5 maximum).

The entry help (accessible by right click) gives the signification of the CRIx variables.

Grid Assignment

Site (field IHPFCY)

Code of the site for which the section can be used. Sections can thus be authorized on whatever site of the X3 folder.

By default, when no site is entered, the section can then be used on all sites of the X3 folder. But when sites are entered the section can then be used only for those entered.

Specific Buttons

  • Validate
  • Copy

Validate

This button is used to generate the automatic processes linked to this heading. This validation is normally linked to the modification or the creation of a code. It is only necessary if the processes have not been deleted or if the setups have been entered by copy from another folder (in this case, the validation is not carried out automatically).

Copy

This button is used to copy the heading to another X3 folder.

Note: the list of authorized sites is not copied. If this is the first time the header is copied (the header does not exist in the destination folder), it must be authorized for all sites. If the header had already been copied beforehand (the header exists in the destination folder), the authorizations existing in the destination folder are retained.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Error on "Date" field: Incorrect data type

This field only authorizes fields with the type Date.

Error on "Statistical value" field: Mandatory field

This message is displayed when no value has been entered but also when an attempt is made to select this value by right click. In this case, it stems from the fact that the code of the statistical setup is different from the one of the statistical trigger.
