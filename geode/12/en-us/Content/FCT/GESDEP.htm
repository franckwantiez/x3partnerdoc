Depositor

In logistical terms, the concept of depositor makes it possible to process separate flows or traffics within the same site.

It covers the concept of order-giving customer, often the stock owner, for the logistical service providers.

For industrialists, it can also represent the notion of "sales brands" insofar as they require a separate logistical management.

Each depositor belong to a single site. It is possible to manage several depositors belonging to the same physical site. The distinction is drawn between a single-depositor site and a multi-depositor site

When the same depositor is physically managed on several different sites, this depositor needs to be created for each site.

Note: The number of depositors managed by a Warehousing application can be submitted to contractual limitations.

To create a depositor, a code is assigned to it that identifies it in a unique manner with 10 character and a title. These two pieces of information are located in the header of the management screen. Then, there are 3 separate definition tabs:

Header

Entry of the depositor code.

Field descriptions
Depositor (field DEP)

Code representing the depositor. In logistics terms, the concept of depositor makes it possible to process separate flows or traffics within the same site. It covers the notion of order giving customer for the service provisions.
After the site, the depositor is the second access key to the Product, Input and Delivery Order files.
Concerning the location, the depositor code is entered if the location is assigned to the depositor or the product, and if the location is single-depositor or single-product and it is not empty.

Postal code (field POS)

Postal code where the entry format is defined in the Country table.
The postal code is used to determine the city/town. When the postal code is changed the city/town corresponding to the new code is automatically displayed.
Particular cases:

  • For ANDORRA, indicate 99130
  • For MONACO, indicate 99138
  • For the other overseas countries, indicate the postal code of the country or 99999 if not known.
City (field CTY)

Name of the town/city. The city automatically proposed is the one associated with the postal code previously indicated.
It is also possible to manually enter the city.

If the city is already defined, the associated postal code is automatically displayed.

if it does not exist, a warning message will appear and proceeding with the entry is authorized.

Region (county, state, ..) (field SAT)

This field includes the geographical subdivision which is part of the postal address definition.
For instance, it can be the department for France or the state for the United States. The subdivision title and the potential control on its entry are defined in the Country setup.
If it is controlled, this code depends on the corresponding geographical subdivision.

This subdivision is automatically fed after the postal code and town/city have been entered, based on the Postal codes setup.

Tab Contacts

This option makes it possible to know the movement code to be used by the system upon creation of automatic movements.

- Adjustment via a transfer: upon creation of a transfer, the quality-related characteristics of a stock object can be modified. A quality-related adjustment is then automatically generated, such an adjustment being carried out with this movement code.

- Replenishment on wave: upon wave launching, replenishments are automatically managed by means of this movement code.

- Adjustment on inventory: using the 'Adjustment on inventory' function creates automatic adjustments of a qualitative or quantitative nature by means of this movement code.

Details

Country (field XCNTCRY)

This code is used to identify the country associated with the information.

In the country table, which controls this field, there is a certain number of characteristics useful to perform controls on dependent information, and especially:

  • the telephone number format,
  • the format of the number that identifies a company or an activity (site tax ID number, NAF in France), and whether its entry is mandatory or not,
  • the format of the postal code/town/city and the geographical subdivision, and whether its entry is mandatory or not,
  • the mandatory or not mandatory aspect of the Site Registration number entry, of the identification number of the company as well as the EU tax no.,
  • the format of the Bank ID number code.
