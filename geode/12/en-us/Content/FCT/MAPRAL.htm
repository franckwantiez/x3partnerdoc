Reassignments

This function allows the :

 selection of the item containers having a current picking locations needing a reassignment or a removal of the picking location

 automatic research or the manual assignment of a picking address to the item containers selected

Prerequisite

Refer to documentation Implementation

Screen management

The screen is made up of a single tab.

Entry screen

The user should first select the picking rearrangement of the item containers that he wishes to allocate.
The selected Picking rearrangement n° should have the item containers to be reallocated (container record having status less than 4) with current picking location.
If the PR N° has item containers with ideal classes not yet calculated the system warns the user that then can continue or abandon the reallocation of the PR keyed in.

If the Picking Rearrangement n° is coherent, then the system displays the general information of the PR (reference of the campaign of the PR and the container processed in the campaign) and the list of item containers of the keyed in PR waiting to be allocated (status inferior to 4) without picking location.
The user can then choose only certain containers of the list through the  

For each item container to be processed and so as to facilitate its picking reassignment, display :
 permanent information : end of life date of the item and current picking locations information (blockage of the location identified by  ‘!’, store, address, rotation class, assignment class, volume class, blockage I/O, storage type, maximum quantity of the location, quantity in stock, quantity in pending input, quantity in pending output)
The blocking reason of the location can be consulted through the option ‘Blocking reason’ of the right click on the mouse.
  statistical information : ideal rotation class, ideal volume class, ideal volume of picking location and daily mean promptings.

The user chooses the item containers to be reallocated one by one from the list by ticking the box ‘Select’ or all the containers of the list through the option ‘check all’ of the right button of the mouse

The system controls for the item containers selected :
 The coherence between the last reassignment date and the minimum delay between two reassignments set-up on the sites table.
If there is an incoherence, the system warns the user that can confirm or abandon the selection of the item container.
 The coherence between the date of the first picking activity and the minimum activity before reassignment set-up on the sites table.
If there is an incoherence, the system warns the user that can confirm or abandon the selection of the item container.

Two types of operations can be carried out on the selected item containers :
 Remove the item container from the current picking location.
To do it, the user should select the value ‘Reserve’ on the zone ‘Destination’.
 Remove the item container from the current picking location and to reassign it to a new picking location.  
The system proposes this operation by default (Value ‘picking’ on the  zone ‘Destination’).

All the selected item containers selected in reassignment should be assigned to a picking location.

The address of the new picking could be :
 keyed in on the zones ‘store’ and ‘address’
 selected on a list..
The list is accessible by putting the cursor on the zone ‘address’ and by using the option ‘Selection’ of the right button of the mouse.
The system proposes the empty picking  locations of the keyed in store, blocked in Input/Output, not being inventoried and assigned to the same container as that of the PR being processed.
 Calculated by the system.
The automatic research of the new picking is triggered by the  button or by the option ‘Automatic addressing’ of the right button of the mouse by putting the cursor on the zones ‘store’ and ‘address’.

After the keying in of the address,  the system do the following controls on the chosen location   :
 the location should be of the picking type and the reference container identical to the item container to be reassigned.
 the location should not be in pending inventory and it should be empty
If the location is empty and is assigned to another item then the system warns the user that then decides to confirm or to cancel the keyed in location
 the depositor of the location should be identical to the depositor of the item to be reassigned if the setting of the picking dynamization is single depositor.
 the storage type of the location should be identical to the storage type of the initial picking of the item container  if the setting of the picking dynamization is single storage type
 the store of the location should be identical to the store of the initial picking of the item container  if the setting of the picking dynamization does not allow the chaining of the stores or if not, chained to the priority store..
 the assignment class of the location should be identical to the assignment class of the container if the setting of the picking dynamization does not allow the chaining of the assignment classes or if not, chained to the assignment class of the item container. .
 the rotation class of the location should be identical to the ideal rotation class of the container if the setting of the picking dynamization does not allow the chaining of the rotation classes or if not,  chained to the ideal rotation class of the item container.
 the volume class of the location should be identical to the ideal volume class of the container  if the setting of the picking dynamization does not allow the chaining of volume classes or if not, chained to the ideal volume class of the item container.

The type of reassignment can be temporary or permanent and is defined by the zone ‘temporary’.
By default, it is assigned to permanent (not temporary).

After the validation of the reassignment of the item containers by the  button, the following processes are carried out 
 automatic research of reassignment picking location if it is not defined
 Blockage in input and in output of the former picking
 Removal of the former picking if no stock and no pending remains and if the reassignment is permanent type.  
 Unblocking and reservation of the new picking
 If the reassignment is temporary,  activation of the indicator ‘temporary picking’ on the new picking location
 Update of the allocated address and the date of the reassignment and passage of the records of the processed PR to validated status.
Evolution of the status of the PR record
- If permanent reassignment and no stock nor any pending on the former picking, passage to status ‘validated’.
- If temporary reassignment, passage to status ‘validated’.
- If permanent reassignment and there are still stock on the former picking, passage to status ’Waiting generation of movements’.
- If permanent reassignment and there are pendings on the former picking, passage to status ‘Waiting validation of movements’.   

After the validation of the removal of the item containers by the  button, the following processes are carried out 
 Blockage in input and in output of the former picking
 Removal of the former picking if no stock and no pending remains.
 Update of the status of the PR records processed
Evolution of the status of the PR record
- If no stocks nor pendings on the former picking, passage to status ‘validated’.  
- If balance of stock on the former picking,  passage to status ’Waiting generation of movements’.
- If there are still pendings on the former picking, passage to status ‘Waiting validation of movements’

Menu Bar

Cancel

It allows to cancel the pending reassignment of the selected item containers.

After the annulation of the reassignments, the system passes the processed PR record to ‘cancelled’ status

Criteria

It allows the keying in of the multiple criteria for the display of the item containers of the PR to be processed (containers not reassigned and with out current picking locations)..

The selection criteria of the containers are :
 either the permanent information of the item container (depositor and the item of the container)
 or the permanent information of the initial picking of the item container  (store, storage type, rotation class and volume class)
 or the calculated statistical information for the container (ideal rotation class, ideal volume class,   mean daily promptings and volumic flows)
 or the comparisons between the ideal classes (rotation and  volume) calculated for the item container  and the  classes of the initial picking.

Addressing

It allows the automatic research of the new picking address of the selected item containers.

After the research, the system displays the selected picking location on the zones ‘store’ and ‘address’ or displays an error message if the system can not find an address corresponding to the criteria set-up for the research.

The Algorithm of the automatic assignment of a picking location is the following :

 the objective of this algorithm is to find an available picking location whose characteristics best fit the permanent and dynamic characteristics of the item container to be reassigned.

 the system will only include the ‘available’ locations, meaning not assigned and not blocked, whose reference container is equal to the container to be assigned.

 Moreover, the system will search in priority the available locations whose :
- depositor is identical to the depositor of the container to be assigned
- store is identical to the priority store of the item container
- the storage type is identical to the storage type of the initial picking of the item container
- the rotation class is equal to the ideal rotation class calculated for the container
- the volume class is equal to the ideal volume class calculated for the container
- the assignment  class is equal to the assignment class of the container.

 If no location fits, and if the setting in the  sites table  allows it, the system continues the research by degrading the criteria depositor, store, rotation lass, volume class and the assignment class through the chainings defined on the tables..

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Error zone ‘Rearrangement n°’ : Mandatory zone
The Rearrangement n° keyed in should be different to 0.

Error zone ‘Rearrangement n°’ : Picking rearrangement X ; Unknown record
The Rearrangement n° keyed in should exist on the picking rearrangement table of the current site

Error zone ‘Rearrangement n°’ : All were already reassigned
For the  PR n° keyed in, there is no item container  not reassigned and having a picking location.

All the ideal classes were not calculated.
For the  PR n° keyed in, there is at least an item container with an undetermined  ideal class (rotation or volume).
The user can therefore  either confirm or cancel the keyed in PR n°..

Line n°X : First activity very recent
The time used since the first activity of the picking item is less than the minimum activity before the reassignment  set-up on the sites table.
The user can therefore either confirm or cancel the selection of the container.

Line n°X : Last reassignment very recent
The time used since the last reassignment of the  item container is less than the minimum delay between two reassignments set-up on the sites table.  
The user can therefore  either confirm or cancel the selection of the container.

Line n°X : Locations in pending inventory
The current picking location of the item container to be reassigned is in pending inventory.  

Error zone ‘Sto’ : Store X ; Unknown record
The store keyed in should exist on the table of the stores of the current site

Error zone ‘Sto’ : Store not belonging to the chain
The store keyed in should correspond to the store of the initial picking, to the priority store of the container or should be chained to the priority store of the item container.

Error zone ‘Sto’ : Addressing without chaining and different store
The store of the location keyed in is different from the store of the current picking and the setting does not allow the chaining of the stores.

Error zone ‘Address’ : X ; Unknown record
The location keyed in should exist on the location tables

Error zone ‘Address’ : X ; Reserve location
The location keyed in is a reserve location..

Error zone ‘Address’ : Location being inventoried.
The location keyed in is being inventoried

Error zone ‘Address’ : Different reference container
The location keyed in is assigned to a container different from the container to be reassigned

Error zone ‘Address’ : Not empty location
The location keyed in has a stock

Location assigned to item X
The picking location keyed in is empty but is assigned to another item.
The user can therefore confirm or do not confirm the address keyed in.

Error zone ‘Address’ : Addressing without chaining and different store
The store keyed in is different is different from the store of the current picking and the setting does not allow the chaining of the stores.

Error zone ‘Address’ : Addressing without chaining and different rotation class
The location keyed in has a rotation class different from the ideal rotation class of the container and the setting does not allow the chaining of the rotation classes.

Error zone ‘Address’ : Rotation class excluded from the chaining
The location keyed in has a rotation class not chained to the ideal rotation class of the container.

Error zone ‘Address’ : Addressing without chaining and different assignment class
The location keyed in has an assignment class different from the ideal assignment class of the container and the setting does not allow the chaining of the assignment classes.

Error zone ‘Address’ : Assignment class excluded from the chaining
The location keyed in has an assignment class not chained to the ideal assignment class of the container.

Error zone ‘Address’ : Incompatible assignment class
The assignment class of the location keyed in corresponds to a forbidden class of the container to be reassigned  or the assignment class of the container to be reassigned corresponds to the forbidden class of the location keyed in..

Error zone ‘Address’ : Addressing without chaining and different volume class
The location keyed in has a volume class different from the ideal volume class of the container and the setting does not allow the chaining of the volume classes.

Error zone ‘Address’ : Volume class excluded from the chaining
The location keyed in has a volume class not chained to the ideal volume class of the container.

Error zone ‘Address’ : Single depositor addressing and depositor different
The location keyed in has a depositor different from the depositor of the container and the setting is single depositor.

Location assigned to depositor X
The picking location keyed in is assigned to a depositor other than that of the item container and the setting is multi depositors.
The user can therefore confirm or do not confirm the address keyed in.

Error zone ‘Address’ : Single storage type addressing and different storage type
The location keyed in has a storage type different from the storage type of the initial picking of the container the setting is single storage type.

No address to be reassigned
The system displays this message on a Log file if it cannot find any picking location to be allocated to an item container by the automatic research.
