Cancellation by parcel

Use this function to fully or partly cancel the shipment of a packed delivery order (DO).

This function is useful when you want to irrevocably delete a DO shipping, either because the final customer canceled the delivery, or because the order was damaged between the preparation and shipping phases.

If you want to cancel the shipment of an unpacked DO, use the Cancellation by DO function.

Unlike the Postponements by parcel function, the cancellation immediately deletes the DO preparation, whereas the postponement will only push it to a later date.

Prerequisite

Refer to documentation Implementation

Screen management

Select a DO, then select the parcels you want to cancel.

Make sure that:

- A movement code set to 'Automatic adjustment' is defined in the Movement codes table for the current site.

- The DO status is higher than status 4 - Pending preparation and the assigned mode is either prepacking or declarative packing.

A parcel can be canceled as long as it is not loaded or postponed.

Entry screen

Steps

  • Step 1: Select the DO. The parcel lines are displayed.
  • Step 2: Use the Cancelled column to specify the parcel lines to be deleted.
  • Step 3: Use the Destination container column to specify the container where you want the stock to be reintegrated.
  • Step 4: Enter the stock reintegration location. This must be a store of the 'Dock' type.
  • Step 5: Specify a movement code set to Automatic adjustment to create the corresponding stock objects in the reintegration store.
  • Step 6: Click Save to validate your parcel cancellation.


Process

A report is printed automatically, listing all the reintegrated elements.

Generally, stocks are then reintegrated into reserve stores / addresses by creating transfer movements.

The output movements are then updated. The status of the canceled movements moves to status 6 - Cancelled upon shipment.

Depending on the DO balance management, for a complete DO cancellation, the DO will move to status 2 - Pending xth preparation, or to status 9 - Shipped, Completed.

The DO can then be shipped. The shipment will only include the movements that are not canceled.

If serial numbers are managed, the cancellation will only be possible if the serial numbers to be canceled are specified.

Serial numbers

In the table of output movements, you can use the Serial number entry action to specify which serial number you want to cancel. This action is only available for products that manage serial numbers.

Note: This action is not needed when the product manages localized serial numbers. These numbers are already recorded in the system (they are associated with the output movements).

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Partial delivery prohibited.

The setup defined in the DO header does not allow you to cancel and split the shipment.

Serial numbers <> CU number to be cancelled line no. x PRODUCT (y<>z).

The parcel contains products to be canceled but their serial numbers are not defined. To enter the serial numbers, click the Actions icon on the parcel line.
