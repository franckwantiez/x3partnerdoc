Unit validation

This screen is used to:

  • partially or completely validate a parcel,
  • cancel a parcel.

The displayed parcels come from the DO wave launching, whose preparation mode is in prepacking or palletization.

The parcel includes a status to view the progress.
The parcels with a status above 4 can no longer be modified.

Refer to documentation Implementation

Screen management

  • Header

The screen is made of three tabs: a tab to identify the parcel, a tab containing the general information on the parcel, and a tab enumerating the output movements of the parcel.

Header

This tab displays:

  • identification information of the parcel: Parcel number, PO number, DO number, DO reference, wave number and status.
  • the parcel size information: Range and type of parcel, size and volume

The parcel Number is automatically assigned to the creation of the entity.

The status code provides the status progress of the parcel.

Entry of the specific information related to the parcel can be requested according to the Actual weight entry setup of the output configuration
If a parcel range is imposed to the DO line, the latter cannot be modified.

Tab Grouped movements

Parcel movements are displayed as a group: any movement showing the characteristics below are cumulated (at display level) in order to make their reading easier:
DO number and DO line number
Store and address,
Reservation code, lot no., support no., source code and stock nature,
Source product container and quantity, homogeneous container and quantity,
Status

Example:
Let us consider 2 movements of 1PP1 of 100 CUUs in the same take, displayed as a group, i.e. a single line of 2PP1s of 100 CUUs, i.e. 200 CUUs.

In order to partially or completely validate a parcel, it is necessary to tick the records to be validated in the grid and to press the button .
Output movements of the parcel are then validated and the locations are freed.
Those recordings showing a quantity in a "grayed-out" font cannot be selected because the corresponding movements have already been validated.

To modify the quantity of a recording, the 'CU no.', 'No.' and 'homogeneous container' fields can be modified, only to be decreased.
The following controls are performed:
- The entered quantity, calculated in CU, must be a multiple of the homogeneous level of the container taken from the product record
- For picking locations, the entered quantity must be a minimum multiple of the reference container of the picking location
- The entered container must belong to the palletization plan of the product container for the output movement.

In case of partial validation of a record, the source movement is split. The entered quantity is updated on the validated movement (switching to status 8-Validated) and the balance movement remains on status 3-Addressed to be later validated or canceled.

In case of partial validation of kit components, the system checks the consistency of kit component proportions.

In case of partial validation of a product with serial number management, the validation can only be performed when all serial numbers have been entered.
In order to enter serial numbers, select the serial number option in the actions menu.

Entry of a control code can be requested by the system. Entry is mandatory when the 'Control code' field of the table displays the '?' character:  for that purpose, the product, store and location setup as well as the output configuration for the current depositor must be set up accordingly.
Validation will only be possible if the entered control code matches the expected control code and can be viewed in the location record.

Entry of an operator code can be requested by the system. For that purpose, the configuration of the outputs for the current depositor must be set up accordingly.
The validation will only be possible if the entered operator code authorizes parcel validation.

In the event of the validation of a big parcel, if the output configuration of the current depositor specifies it, and if the palletization is managed at DO level, the parcel is automatically palletized.
A SU no. is assigned to it, the SU is closed and the SU type is picked up from the output configuration.

Menu Bar

Cancel

  • Canceling a parcel or parcel records is possible by selecting the recordings and clicking .
  • The canceled output movements moves to status 5-Canceled upon picking.
    In the event of the total cancellation of a parcel, the parcel no. is deleted.
  • A control is applied in case of non authorized partial delivery for the DO or DO line.
    It is possible to validate a movement and cancel it, but it is not possible to cancel and then validate.

Error messages

  • Generic messages

In addition to the generic error messages, the following messages can appear during the entry :

Error on CU no. field

The entered quantity is larger than the quantity of cumulated movements.

Container missing from the palletization program

The entered container does not belong to the palletization plan of the movement container.

Quantity not a multiple of the homogeneous quantity

The homogeneous level of the input container does not allow the validation of a quantity lower than this level.

Serial numbers < Number of validated packing units

The numbers of expected serial numbers has not been completely entered.

Serial numbers > Number of validated packing units

The number of entered serial numbers is superior to the quantity to be validated.

Serial number not entered

The product being managed via a serial number complete traceability, a control is carried out between the serial number keyed in and the already entered ones; in this case, there is no correspondence.

Operator code invalid.

The entered operator code does not enable the parcels to be validated.

Confirmation impossible. Partial delivery prohibited. At least one movement has been canceled.

The DO is in prohibited partial delivery, and part of the DO has been canceled. The closing has to be canceled, too.
