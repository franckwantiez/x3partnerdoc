Configurations

This table is used to define the management rules of the input and output flow expected for each depositor.

The Input configuration is to be defined for each depositor of the site.

Prerequisite

Refer to documentation Implementation

Screen management

  • Header
  • Tab Input config.

In order to create the Input configuration, you need to select a depositor. This information is located in the header of the management screen.

Header

Entry of the depositor code.

Field descriptions
Depositor (field DEP)

Code representing the depositor. In logistics terms, the concept of depositor makes it possible to process separate flows or traffics within the same site. It covers the notion of order giving customer for the service provisions.
After the site, the depositor is the second access key to the Product, Input and Delivery Order files.
Concerning the location, the depositor code is entered if the location is assigned to the depositor or the product, and if the location is single-depositor or single-product and it is not empty.

 
Description (field NAM)

Tab Input config.

Note on Flow control. This parameter is linked to the product record parameter "Blocked in input" and "Blocked in output".

Input and output controls are performed during the following transactions:

  • For inputs:
    • receipt validation
    • input line entry
    • search for input locations
    • adjustment on quantity creation or increase
  • For outputs:
    • DO line entry
    • addressing on output
    • adjustment on quantity deletion or decrease

Transfer movements are still possible regardless of the parameter value.

Field descriptions

Block number 1

This parameter triggers the control of the values of the Input Block and Output Block parameters on the product record upon input or output transact ions.

Flow control (field IFLCTL)

This parameter indicates if the same input reference can be displayed countless times in the input portfolio (all statuses included), or only once (irres pective of the input status), or several times (provided the others are validated or closed).

This parameter specifies for the inputs received via an interface whether or not a manual modification of the quantity can be carried out.

Unique reference (field UNQIRF)
Quantity control (field CTLQTY)

Storing list

Defines the printing sequence of the Storage Lists, either by store, then by Input Reference, or by Input Reference, then by store.

Stipulates if the storage list is a single-input list. If the addressing request on input concerns several Input references, then GEODE generates a storage list by input reference.

If the addressing request on input concerns several storage stores, then GEODE will generate a storage list by store.

Print sequence (field STLSEQ)
Single input (field SNGINPSTL)
Single store (field SNGSTOSTL)

Validation

An operator code is necessary to validate the expected input or the direct input.

A check will be carried out to ensure that the operator has performed the repositioning of the stock object in the right place with respect to the s ystem.

Stipulates whether or not is it necessary (manually if no, automatically if yes) to validate the input for the products to be available. If the vali dation is automatic, upon addressing of an input, the entities linked to said input will be validated (header, line et input movements).

Operator code (field STLOPE)
Control code (field STLCTL)
Validation type (field STLVLD)

Prints

Stipulates if the storage list must be printed.

Putaway or input labels will be systematically generated during location search. A label is printed for each addressed stock object.

Storing list (field STLEDT)
Putaway label (field STLLBL)
