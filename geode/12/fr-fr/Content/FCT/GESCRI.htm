Critères transporteur

Les critères transporteurs permettent de paramétrer l’algorithme de choix du couple (transporteur, mode) en tenant compte des critères physiques, des critères financiers ou bien des deux types de critères

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

La saisie doit être réalisée pour tous les OL dont il est nécessaire de déterminer au moins un transporteur, en caractérisant les enregistrements par le couple (mode de préparation, code mouvement).

Ecran de saisie

Remarque sur le délais de préparation : zone obligatoire, si non géré affecter la valeur 0000 (00:00).
Utilisée uniquement pour la prise en compte des tournées d’expédition sans paramétrage des tournées génériques.
A date égale, l’heure de calcul, additionnée au délais de préparation, doit être inférieur à l’heure de la tournée; sinon l’O.L est affecté à une autre tournée d’expédition.
Exemple: délais de préparation de 5 heures; le calcul est lancé à 14 heures, la tournée d’expédition est à 18 heures; l’O.L sera affectée au prochain jour de passage de la tournée.
Ce délais ne vaut que pour les tournées d’expédition non générique; pour les tournées génériques c’est le délais de préparation indiquée dans la table des tournées génériques.

Descriptions des champs

Tableau

Code qui représente le mode de préparation. Le mode de préparation regroupe l’ensemble des règles de gestion lié &agra ve; chaque O.L. Il permet d’en déterminer le mode de traitement lors du lancement en préparation.

Permet de créer ultérieurement des vacations d’O.L urgents à servir en priorité.

Permet de calculer le montant du port du RT et/ou du BT à la confirmation d’expédition par la fonctionnalité « fonctionnalit&eacut e; préfacturation transport ».

Séquence (champ ORN)

Numéro permettant d'utiliser les règles définies dans cet écran par

ordre de priorité.
Code mvt (champ MVT)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
Déposant (champ DEP)

Code qui représente le déposant. En terme logistique, la notion de déposant permet de traiter des flux ou trafics distincts dans le même site. Elle recouvre la notion de client donneur d'ordres pour les prestataires de services.

Après le site, le déposant est la deuxième clef d'accès aux fichiers Article, Entrée et Ordre de Livraison. Pour l'emplacement, le code déposant est renseigné si l'emplacement est affecté au déposant ou à l'article, et si l'emplacement est mono déposant ou mono article et qu'il n'est pas vide.
Mode préparation (champ PMD)
Urgent (champ URG)
Délai de préparation (champ PRPLTI)

Lors de l'affectation automatique d'un OL à une tournée, le traitement prend en compte ce paramétre ajouté à l'heure de lancement du traitement pour déterminer la tournée la plus optimale.

Si les tournées génériques sont gérées, c'est le paramètre de la tournée qui est pris en compte, sinon c'est celui de la table des critères transport.

Si il n'y a pas d'affectation automatique des tournées, ce paramétre n'est pas utilisé.
Priorité (champ CRO)

Indique lors du choix du transporteur par le systéme si le critère Priorité transport défini dans la table critére transporteur doit être pris en compte.
Coût (champ CST)
