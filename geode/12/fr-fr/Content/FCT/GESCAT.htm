Tarifs transporteur

Cette option permet d’associer à chaque triplet (transporteur, mode de transport, code tarif), un numéro de tarif valable pour une période donnée, auquel sont attachées les règles génériques de facturation, toute destination confondue (montant du timbre, gestion du « payant pour », définition des tranches tarifaires,...)..

Les tarifs transporteurs sont ensuite associés à des N° de tarifs précisant le coût de transport.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Les informations sont saisies sur un tableau à partir d’un triplet (transporteur, mode, code tarif)

Ecran de saisie

N° tarif : le N° de tarif permet d’identifier les règles de tarification d’un transporteur sur une période.

Transporteur/Mode : le couple transporteur/mode doit exister et être associé entre eux dans la table des modes des transporteurs.

Code tarif : le code tarif n’est pas modifiable pour un couple (transporteur, mode) si il a été défini comme code tarif a utiliser pour le calcul de la pré facturation transport dans la table des transporteurs.

Période de validité : il ne peut exister plusieurs tarifs de même période de validité. Le tarif est utilisé en fonction de la date d’expédition de l’O.L.

Payant pour à partir de la tranche :
Ce paramètre permet, compte-tenu de la forte dégressivité du tarif, de "payer pour" un poids supérieur au poids réel.
Vous pouvez indiquer un numéro de tranche à partir duquel la règle s'applique. Si cette zone est laissée à blanc la règle n'est pas appliquée.

Exemple : Si le tarif s'établit ainsi: de 0 à 100 kg = 35 € par kilo de 101 à 300 kg = 28 € par kilo. Le chargeur qui a 85 kg à transporter aura intérêt à payer pour 100 kg car 85 * 35 = 2975 € alors que 100 * 28 = 2800 €.

Masse volumique minimale : Les tarifs de transport pour lesquels la taxation est donnée au poids sont souvent assortis d'une règle de ratio poids/volume.

Dans ce cas, dès lors que le poids réel est inférieur au poids théorique obtenu par le calcul [(Volume réel en m3 / Poids réel) x poids nominal au m3], la tarification est calculée à partir de ce dernier

Unité des tranches : permet de définir l’unité de saisie des tranches.
Dès lors qu’une unité est saisie, elle est alors considérée comme étant proportionnelle à l’unité ; si elle n’est pas saisie elle est alors fixe pour la tranche.
L’unité saisie est le Kg : le prix sera donc saisi en €/kg de la tranche
L’unité n’est pas saisi : le prix sera  fixe pour la tranche exprimé dans l’unité du tarif.

Livraison contre remboursement : montant fixe et taux
Le montant fixe correspond à la taxation fixe perçue par le transporteur dans le cas d'une livraison avec Contre-Remboursement (coût fixe de la prestation).

Le taux correspond à la taxation variable perçue par le transporteur dans le cas d'une livraison avec Contre-Remboursement (coût de la prestation indexée sur le montant du C.R.). Cette valeur vient s'ajouter à la précédente.

Timbre : Montant et A partir de
Si la base (valeur servant à déterminer la tranche, c'est-à-dire le poids, le nombre de colis ou le métrage linéaire) est supérieure à la valeur indiquée dans la zone "à partir de" le montant du timbre est ajouté au coût du transport

Taux de remise : Ce paramètre permet d'enregistrer le tarif d'un transporteur "tel que" et d'appliquer une remise systématique sur les valeurs obtenues sans être obligé d'appliquer ce coefficient de réduction à chaque prix

Tranches Tarifaires : définition des tranches pour lesquelles un tarif sera défini.

Les tranches ne seront plus modifiables une fois saisies.

Unités : l’unité si elle est saisie est l’unité de calcul du prix dans la tranche.

Exemple : l’unité est KG, le prix saisi sera exprimé en €/KG. L’unité n’est pas renseigné, le prix de la tranche est fixe pour la tranche.

Lors de la duplication d’un code tarif, une fenêtre est affiché permettant de saisir le coefficient de variation du code tarif.

Automatiquement, des prix tarifs sont calculés en fonction du pourcentage indiqué, en dupliquant tous les prix tarifs associés au prix transporteur dupliqué (s’ils existent).

Cette fonction est utile pour calculer les augmentions de tarifs d’un transporteur d’une période sur l’autre.

Le bouton "Supprimer" supprime à la fois le tarif transporteur et les prix tarifs associés au tarif.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Quadruplet (transporteur, mode, code, période) ayant déjà un numéro de tarif : xxxxx

Il existe déjà un enregistrement pour le transporteur, mode de transport, et code tarif saisi pour la période.

Mode de transport non associé au Transporteur XXXX

Le mode de transport doit être défini pour le transporteur (table des transporteur)

Type d’unité incorrect (<> Masse/Poids)

Le type d’unité choisi pour définir les tranches tarifaires n’est pas conforme au type de tarification saisi pour le couple (transporteur, mode) dans la table des transporteurs.
