Configurations

Cette table permet de paramétrer les règles de gestion générales liés au stocks pour chaque déposant d’un site.

Pour créer la configuration des stocks, on sélectionne un déposant, puis on doit renseigner les onglets dédiés à chacun des flux. 

En-tête

Saisie du code déposant.

Descriptions des champs
Déposant (champ DEP)

Code qui représente le déposant. En terme logistique, la notion de déposant permet de traiter des flux ou trafics distincts dans le même site. Elle recouvre la notion de client donneur d’ordres pour les prestataires de services.
Après le site, le déposant est la deuxième clef d’accès aux fichiers Article, Entrée et Ordre de Livraison.
Pour l’emplacement, le code déposant est renseigné si l’emplacement est affecté au déposant ou à l’article, et si l’emplacement est mono déposant ou mono article et qu’il n’est pas vide.

 
Intitulé (champ NAM)

Onglet Paramétrage

Cet onglet contient le paramètrage général sur le stock.

Descriptions des champs

Gestion

Gestion des numéros de support (au niveau du déposant) pour chaque objet entré.

L'autorisation de saisie de cette information est contenue au niveau du contenant article.

Numéro de support (champ SPTMGT)
Numéro de lot (champ LOTMGT)

La saisie du numéro de lot pour la création d’un objet est obligatoire si la configuration des stocks du déposant et le paramétrage article comporte le paramétrage «Numéro de lot» à « oui ».

Bloc numéro 2

Transfert (champ MVTADJAUT)

Code mouvement qui sera utilisé pour les ajustements automatiques liés aux transferts de stocks.

Le code mouvement saisi doit exister dans la table des codes mouvements et doit être de type "Ajustement automatique".
Transfert en 2 phases (champ MVTADJAU2)
Inventaire (champ SKCAJM)

Code mouvement de type ajustement automatique utilisé par le système lors de la génération d'ajustement automatique suite à inventaire.

Le code mouvement saisi doit exister dans la table des codes mouvements et doit être de type "Ajustement automatique".
Demande d'analyse (champ MVTQLYAUT)
Assemblage (champ MVTDCLKIT)

indique le code mouvement utilisé pour les ajustements automatique survenant dans la procédure d'assemblage.
Désassemblage (champ MVTDCLCPN)

indique le code mouvement utilisé pour les ajustements automatiques de désassemblage.

Bloc numéro 3

Réappro sur vacation (champ MVTWAVRPN)

Code mouvement qui sera utilisé lors d'un réapprovisionnement dû à un lancement en vacation.

Le code mouvement saisi doit exister dans la table des codes mouvements et doit être de type "Transfert".
Reliquat de réappro (champ MVTRPNSHR)

Code mouvement qui sera utilisé lors d'un Transfert dû à un réadressage des reliquats d'objets de stock apparus suite à un ordre de réappro. Le code mouvement saisi doit exister dans la table des codes mouvements et doit être de type "Transfert".

Cumul de flux (champ MVTTRFFLC)

Code mouvement qui sera utilisé dans la première phase de la préparation de type cumul de flux lors de la génération des transferts vers les magasins d'éclatement.

Le code mouvement saisi doit exister dans la table des codes mouvements et doit être de type "transfert".
Demande d'analyse (champ MVTTRFQLY)

Code mouvement a utiliser pour le transfert des stock en contrôle qualité, suite à la validation / clôture d'une Demande d'Analyse.

Le code mouvement saisi doit exister dans la table des codes mouvements et doit être de type "Transfert".

Assemblage (champ MVTTRFCPN)

indique le code mouvement utilisé pour le transfert (1ere phase) de la vacation cumul de flux d'assemblage.
Désassemblage (champ MVTTRFKIT)

Indique le code mouvement utilisé pour les transferts liés aux OA (Ordre d'assemblage) de type désassemblage.

Onglet Réappro

Cet onglet contient les paramètres utilisés dans le cadre de la génération d’un ordre de réapprovisionnement.

Descriptions des champs

Gestion

Ce paramètre conditionne le fonctionnement de l'algorithme de réapprovisionnement picking :

soit la case est cochée et l'algo rithme de réappro considère que les encours de sortie ne sont jamais exécutés dans ses calculs de place disponible en picking (la règle de réappro en palette complète est inhibée en cas de r&eac ute;appro de nécessité),

soit la case n’est pas cochée et l'algorithme de réappro considère que les encours de sortie sont toujours exécutés dans ses calculs de place disponible en picki ng (la règle de réappro en palette complète est systématiquement utilisée dans le cas des emplacements plus grands qu'une palette standard).

Le système autorise le transfert automatique des palettes rompues. Cette valeur est à rapprocher de la notion de réadressage des reliquats de réappro dans le magasin réserve d'où est déstockée cette palette.

Réappro toujours exécutable (champ SRO)
Réadressage reliquat réappro (champ IPAAUTTRF)

Constitution des OR

Mono zone prépa de prise (champ RPNPKZPIK)

Si la case est cochée, le système créera des OR mono zone de prépa de prise.

Mono zone prépa de dépose (champ RPNPKZDPS)

Si la case est cochée, le système créera des OR mono zone de préparation de dépose.

Validation

La saisie de l'identification des opérateurs est obligatoire, lors des validations de flux de réapprovisionnement.

Un contrôle sera effectué pour s’ assurer que le système et l’opérateur ont effectué la prise de l’objet de stock au bon endroit.

Un contrôle sera effectué pour s’ assurer que le système et l’opérateur ont effectué la dépose de l’ objet de stock au bon endroit.

Indique si une opération de validation est nécessaire lorsque les mouvements physiques ont été réalisés. Si oui, la v alidation est manuelle. Sinon, la validation est automatique (la saisie du code opérateur et de la clé de contrôle de l'emplacement ne sera demandée, même si les cases sont cochées).

Code opérateur (champ RPNOPE)
Code contrôle en prise (champ PIKCTL)
Code contrôle en dépose (champ DPSCTL)
Type de validation (champ RPNVLD)

Editions

L’édition des ordres de réapprovisionnement est activée.

Des étiquettes de réapprovisionnement seront générées et imprimées pour chaque mouvement.

Liste de réappro (champ RPNLST)
Etiquette de réappro (champ RPNLBL)

Onglet Assemblage

Cet onglet contient les paramètres utilisés dans le cadre de la procédure d'assemblage.

Descriptions des champs

Ordres d'assemblage

Référence unique (champ BOMUNQREF)

indique si une même référence d’ordre d'assemblage (OA) pourra figurer un nombre illimité de fois dans le portefeuille des OA (tous états confondus), ou bien une seule fois (quel que soit l’état de l’OA), ou bien plusieurs fois (à condition que les autres soient validées ou soldées).

Editions

Liste de transfert (champ BOMLST)

Si coché, l'édition de la liste de transfert est automatique à la création de l'OT assemblage.
Etiquette de transfert (champ BOMLBL)

Si coché, l'édition de l'étiquette de transfert est automatique à la création de l'OT assemblage.
Rupture par article (champ BOMSSHITM)

Si coché, l'édition Rupture par article est automatique à la création de l'OT assemblage.
Rupture par OA (champ BOMSSHORD)

Si coché, l'édition Rupture par OA est automatique à la création de l'OT assemblage.

Vacations

Stock disponible (champ BOMSTKAVL)

Si coché, la recherche en sortie du Transfert 1ere phase se fait en priorité sur les emplacements d'assemblage.
Stock picking (champ BOMSTKPIC)

Si coché, la recherche en sortie du Transfert 1ere phase se fera aussi sur les emplacements picking.
Si non coché, la recherche en sortie se fera uniquement sur les emplacements réserves.
Transfert rompu (champ BOMTRFIPA)

Si coché, la seule qté strictement nécessaire est transférée même si pour cela on crée un rompu en stock.
Si non coché, le transfert des objets composants est réalisé sans rompre d'objets de stock. On peut donc transférer une quantité supérieure à la quantité demandée.

Constitution des OT

Mono zone prépa de prise (champ BOMPKZPIK)

Si coché, les ordres de transfert seront regroupés par zone de préparation de prise liée au magasin.
Mono zone prépa de dépose (champ BOMPKZDPS)

Si coché, les ordres de transfert seront regroupés par zone de préparation de dépose liée au magasin.
Découpage de charge (champ BOMWKL)

Si coché, cela signifie que l'on va découper les OT assemblage de façon à respecter les limites de charge définies sur la zone de préparation.
Le calcul de la charge se fera toujours sur la zone de préparation de prise. Dans le cas d'OT multi zone de préparation de prise, il faudra obligatoirement indiquer dans le champs 'Zone Prepa' la zone de préparation à considérer pour le calcul.
Zone prépa (champ BOMWKLPKZ)

indique la zone de préparation à utiliser pour le découpage de charge dans le cas d'OT multi zone de préparation de prise.

Assemblage

champ W1
Type de validation (champ CPNVLD)

Si coché, la validation du transfert 1ere phase sera automatique.
champ W2
Code opérateur (champ DCLOPE)

Si coché, la saisie d'un code opérateur sera demandée lors de la déclaration de kit.

Désassemblage

champ W3
Type de validation (champ KITVLD)

Si coché, la validation du transfert 1ere phase sera automatique.
champ W4
Code opérateur (champ LCDOPE)

Si coché, la saisie d'un code opérateur sera demandée lors de la déclaration de kit.

Onglet Contrôle qualité

Cet onglet contient les paramètres utilisés dans le cadre de la procédure de contrôle qualité.

Descriptions des champs

Contrôle qualité

Nature de stock Rebut (champ DILSKN)

Nature de stock à utiliser pour la mise à jour des objets de stock qualifiés non conformes lors du contrôle qualité. L'ajustement qualitatif est réalisé à la validation de la Demande d'Analyse.

Nature de stock Consommé (champ USESKN)

Permet de configurer la nature de stock à utiliser pour les quantités de stock consommées lors du contrôle qualité : cette nature de stock permet d'identifier et de décompter les stocks consommés lors de la validation du contrôle qualité.

Code opérateur (champ QLYOPE)

Lorsque le paramètre est à oui, la saisie d'un code opérateur est demandée pour enregistrer la saisie du contrôle qualité.

Validation demandes d'analyse

Adressage automatique (champ ALLQLYAUT)

Lorsque le flag est activé, l'adressage automatique des stocks contrôlé est réalisé à la validation / clôture d'une Demande d'Analyse : les transferts du magasin qualité vers le stock sont créés en automatique.

Code opérateur (champ VQLOPE)

Lorsque le flag est activé, la saisie d'un code opérateur est demandée pour la validation d'une Demande d'Analyse.

Edition

Fiche qualité (champ PRNAUT)

Lorsque le flag est actif, l'édition de la fiche qualité configurée pour l'article est automatiquement déclenchée à la création de la Demande d'Analyse.

Onglet Picking pivot

Cet onglet contient les paramètres utilisés dans le cadre du lancement en vacation lors de la recherche d'emplacements picking pivots vides à affecter (articles avec mode de sortie picking pivot).

Priorités de critères de recherche d'affectation picking pivot

La valeur optimum des critères les plus importants sera conservée en priorité par rapport aux autres critères pour l'affectation de l'article à un picking pivot vide.

Déposant
Optimum : Recherche d'emplacements affectés au Déposant de l'article à affecter.
Dégradé : Recherche d'emplacements vides non réservés.

Magasin
Optimum : Recherche d’emplacements dans le magasin prioritaire défini plus haut dans la configuration des stocks pour le picking pivot.
Dégradé : Utilisation de la Table Enchaînement des Magasins.

Classe de Rotation.
Optimum : Recherche d'emplacements ayant une classe de rotation identique à celle affectée à l'article.
Dégradé : Utilisation du paramètre « Enchaînement » dans la Table des Classes de Rotation.

Classe d'Affectation
Optimum : Recherche d'emplacements ayant une classe d'affectation identique à celle affectée à l'article.
Dégradé : Utilisation du paramètre « Enchaînement » dans la Table des Classes d’Affectation.

Descriptions des champs

Gestion

Code enchaînement (champ CHNMODDYN)

Enchaînement permettant de trouver le magasin d'entrée prioritaire et magasins enchaînés lors de l'adressage des réapprovisionnement des emplacements picking pivots. Obligatoire.
Enchaînement magasins (champ CHNDYN)

Défini si l'enchaînement des magasins est utilisé pour la recherche d'emplacements picking pivots.

Enchaînement classes de rotation (champ CHNABCDYN)

Défini si l'enchaînement des classes de rotation est utilisé pour la recherche d'emplacements picking pivots.

Enchaînement classes d'affectation (champ CHNASGDYN)

Défini si l'enchaînement des classes d'affectation est utilisé pour la recharche d'emplacements picking pivots.

Tableau Priorité picking pivot

Critères (champ DYNPIO)

Défini la priorité des critères à dégrader pour la recherche d'emplacements picking pivots : Déposant, Magasin, Rotation, Affectation.
