Bons de livraison

Cette table permet de visualiser un bon de livraison (BL).

Le bon de livraison est le document accompagnant la marchandise d’un OL lors de la livraison.

Un BL est généré par OL au lancement de la vacation si la gestion du BL provisoire est active pour le déposant courant et/ou un BL définitif est généré par OL par tournée d’expédition.

Onglet EDI transporteur

Il s’agit des informations EDI transport du Bon de livraison

L’onglet affiche :
 les informations du type de gestion EDI utilisé pour le BL
La zone ‘Gestion EDI’ permet de savoir si l’EDI transport est géré pour le BL et si oui, son état d’avancement
La zone ‘Niveau EDI’ permet de connaître le type d’entité géré dans les messages EDI : colis, UM, BL ou RT

 les informations du dernier message EDI reçu avec la gravité la plus forte des entités liées au BL

Descriptions des champs

Bloc numéro 2

Gestion EDI (champ EDISTA)

Permet de savoir si l’EDI transport est géré pour le BL et si oui, son état d’avancement

Les valeurs possibles sont :

'En-cours' : BL géré par l'EDI transport et n'ayant pas reçu son dernier message EDI

'Terminé' : BL géré par l'EDI transport et ayant reçu son dernier message EDI

'Non concerné' : BL non géré par l'EDI transport

champ DLNNUM2

La loupe permet de visualiser l'ensemble des messages EDI dèjà reçus pour le BL traité.
Niveau EDI (champ TYPEDI)

Indique le type d'entité gérée dans les messages EDI.

L'entité traitée peut être soit le RT, soit le BL, soit l'UM ou soit le colis.
champ NBRPRCEDI2

Message de plus forte gravité

Numéro (champ EDHNUM)

Numéro du message EDI.

Séquence numérique attribuée automatiquement par le système lors de la

création d'un message EDI soit par interface soit manuellement.. Ce numéro est une information propre au

site, tous déposants confondus. Cette information constitue une clef

unique d'accès au message EDI.
Entité (champ TYPNUM2)

Type et numéro de l'entité traitée dans le message EDI
Gravité (champ NUMERR)

Indique une hierarchie des codes raisons en fonction de la gravité

Exemple : un code raison 'conforme' a pour code gravité 0
Code statut (champ COD1)

Il décrit la situation de l'entité (Enlevé, livraison en cours...)

Les statuts sont définis pour l’ensemble des transporteurs et leur codification est définie à chaque accord d’interchange avec le transporteur.
Phase (champ ORN)

Le code phase indique l’ordre dans la réception des messages.

Il permet au système de contrôler lors de l’intégration des messages ‘retour’ que le code phase est supérieur au code phase des messages déjà reçus pour l’entité traitée.
Code raison (champ COD2)

Le code raison donne une justification ou une explication autour d’un statut (Exemple : manquant, détérioré...)

Ils sont définis lors de l’accord d’interchange avec le transporteur
Date évènement (champ CLTDAT)

Date et heure de l'évènement traité dans le message EDI.

Lors de l'intégration des messages EDI, des contrôles de cohérence sont réalisés entre les dates et heures évènement du nouveau message et des messages déjà reçus pour l'entité.

La date et heure évènement du nouveau message doivent être inférieures à celles des messages déjà reçus.
champ CLTTIM
Fin (champ LASREC)

Il permet de définir à partir des couples (code statut, code raison) si le message EDI est le dernier à recevoir pour l’entité traitée ou non.

Les valeurs possibles sont :

Oui : dernier message à recevoir pour l’entité

Non : le système accepte de recevoir d’autres messages pour l’entité
Date traitement (champ CRRDAT)

Date et heure de traitement de l'évènement décrit dans le message EDI.
champ CRRTIM
Prochain message (champ NEXDAT)

Date et heure du prochain message à recevoir pour l'entité traitée dans le message EDI
champ NEXTIM
