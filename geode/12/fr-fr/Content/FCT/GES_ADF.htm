Liens de documentation

La fonction de gestion des liens de documentation permet d'associer, à une documentation donnée, des liens vers des éléments du dictionnaire. Ces éléments sont utilisés, en complément des informations trouvées dans le dictionnaire, pour générer automatiquement des paragraphes et leur contenu, notamment dans les sections pré-requis et les sections annexes finales.

La liste des liens existants et la façon dont ils sont utilisés selon le type de documentation d'origine, est donné dans la suite. On trouve dans ces tableaux une colonne Compl.

Cette colonne définit, lorsqu'elle est à Oui, que les liens décrits par ce code viennent en complément des liens trouvés dans le dictionnaire. Ainsi, lorsqu'on génère une documentation fonctionnelle, les fenêtres, tables, objets qui lui sont associés sont analysés pour retrouver les codes activités associés, mais il est possible qu'un code activité non directement relié à la fonction influe sur son fonctionnement. On ajoutera alors ce code activité via un lien.

Lorsque la colonne Compl est à Non, aucune autre informatation du dictionnaire ne permet de définir le lien en question. Il est donc obligatoire de renseigner ce type de lien si on veut que cette information figure dans la documentation générée. Il faut noter que dans certains cas (tables diverses, menus locaux), on pourrait retrouver des liens de ce type, mais de façon peu fiable (ne nous intéressent dans ce cas que les tables diverses et les menus locaux dont le paramétrage est un pré-requis de la fonction).

Cas particulier des points d'entrée

Les points d'entrée s'appuient de façon un peu particulière sur les liens. En effet :

  • l'élément du dictionnaire qui est documenté est le traitement dans le dictionnaire des traitements
  • chaque point d'entrée est défini par un lien vers le type APE, et le code associé est le nom du point d'entrée. Les liens qui suivent (jusqu'au prochain lien APE) caractérisent ce point d'entrée. Si des caractéristiques communes existent avant le premier lien APE, il sont supposés être valables par défaut pour tous les poits d'entrée.
  • les liens ATB et AT2 sont utilisés pour définir les tables ouvertes (respectivement avec un contenu significatif ou non significatif)
  • le lien TRS permet de définir s'il y a une transaction en cours (valeur 2) ou non (valeur 1)
  • le lien TRA permet de définir s'il y a un fichier trace ouvert (valeur 2) ou pas (valeur 1)

Ainsi, si on a le tableau de liens suivant :

TRS 1
TRA 2
ATB TABLE1
AT2 TABLE2
APE POINT1
ATB TABLE3
APE POINT2
TRA 1
APE POINT3
AT2 TABLE4

Le traitement dispose de 3 points d'entrée POINT1, POINT2, POINT3 :

  • Le POINT1 n'a pas de transaction ouverte, gère une trace, la table TABLE1 est en ligne et significative, la table TABLE2 est en ligne sans être significative (valeurs par défaut), la table TABLE3 est en ligne et significative.
  • Le point POINT2 a les mêmes valeurs par défaut, sauf qu'il n'y a pas trace (la table TABLE3 n'est pas en ligne).
  • Le point POINT3 a les mêmes valeurs par défaut (une trace est ouverte). La table TABLE4 est ouverte sans être significative.
