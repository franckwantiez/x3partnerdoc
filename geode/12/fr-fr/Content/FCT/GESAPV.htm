Vues du portail

La vue est l'élément unitaire composant une page de portail. Elle se compose de deux éléments, saisis chacun sur un des onglets de la fonction :

  • Une source de données qui définit un processus d'extraction d'informations à partir du référentiel ADONIX X3. Une source de données renvoie des valeurs organisées en lignes, chaque ligne contenant des colonnes (le nombre de colonnes dépendant de la source de données, le nombre de lignes dépendant des données effectivement extraites lorsqu'on interroge la source et que l'on lit la base)
  • Un composant visuel, qui décrit la façon dont vont être présentées les données extraites.

L'association de ces deux éléments s'accompagne de la définition d'une série de paramètres qui sont saisis dans les tableaux de chaque onglet.

En-tête

Dans l'en-tête, on saisit le code, qui identifie une vue de portail, et un intitulé.

Descriptions des champs
Code (champ APVCOD)

Ce code identifie de façon unique la fiche en cours.
champ APVINTIT

Onglet Source de données

Cet onglet permet de définir la source de données, c'est-à-dire l'origine des informations qui vont être présentées.

Descriptions des champs

Bloc numéro 1

Type (champ SRCCODTYP)

Permet de définir un type de source de données En fonction du type, on saisira, si nécessaire, un code complémentaire identifiant la source elle-même, et une série de paramètres complémentaires définis dans le tableau du bas de l'onglet.
Code (champ SRCCOD)

Identifie la source de donnée. En fonction du type saisi précédemment, ce champ peut être :

  • un code de multi-liste (type de source MLT).
  • le code d'une requête, ou d'une requête SQL (type de source REQ).
  • Le code d'une statistique (source de données STA).
  • Le code d'un traitement (source de données TRT).
  • vide (cas des types de sources URL et AGD, qui n'ont besoin que d'une liste de paramètres dans le tableau qui suit).
champ WDES

Tableau Source de données

Paramètre (champ SRCPARCOD)

Définit le code du paramètre nécessaire à la source de données.
Intitulé paramètre (champ SRCPARDES)

Intitulé associé au code précédent
Formule (champ SRCPARFOR)

Permet de saisir, sous la forme d'une formule évaluée, la valeur du paramètre.

Il est à noter que :

  • cette formule peut être alphanumérique, numérique, ou de type date selon les cas, mais que si le type de l'expression ne correspond pas à ce qui est attendu, il y a conversion.
  • dans certains cas, des valeurs constantes peuvent être données, et sélectionnées par clic droit.
  • lorsque plusieurs valeurs peuvent être données (par exemple des numéros de colonnes), on les exprime sous la forme d'une chaine de caractères contenant une liste de valeurs numériques séparées par des points-virgules. La sélection successive de ces colonnes peut en général être faite par clic droit, la liste de valeurs se complétant alors.
Intitulé (champ SRCPARLIB)

Intitulé associé au code précédent

Onglet Composant visuel

Cet onglet permet de définir le format de présentation de l'information (sous forme de tableau, de graphique, d'agenda...). Ce format, identifié par un composant visuel et les paramètres qui y sont attachés, dépend de la source de données (certains composants visuels ne fonctionnent qu'avec certaines sources de données).

Les composants fournis et les contraintes correspondantes sont définies dans l'annexe ci-dessous.

Liste des sources de données standard

  • AGD (Agenda)
  • MLT (Multi-liste)
  • REQ (Requête)
  • STA (Statistique)
  • TRT (Traitement)
  • WEB (Page Web)

Les types de sources disponibles en standard sont décrites ci dessous.

AGD (Agenda)

Ce type de source correspond à l'agenda CRM.

Aucun code ne lui est associé.

Les paramètres qui doivent être saisis permettent de définir une fenêtre de dates de consultation :

  • le Nombre de jour avant (la date courante).
  • le Nombre de jours après (la date courante).

Tous deux sont des expressions numériques.

MLT (Multi-liste)

Ce type de sources de données correspond à un ensemble de listes de sélection (telles qu'elles sont visualisées dans la liste gauche d'un objet).

Un code de multi-liste lui est associé.

Il n'y a pas d'autre paramètre associé.

REQ (Requête)

Ce type de sources de données correspond à une requête, ou à une requête SQL, identifiée par son code. Deux paramètres complémentaires sont alors définis :

  • le premier, nommé FRQ, définit la fréquence d'extraction. La valeur, qui peut être sélectionnée dans une liste, va de 1 (à chaque utilisation) à 6 (la première fois), en passant par des niveaux intermédiaires tel que 3 (tous les jours). Lors de l'utilisation de la source de données, la date de dernier calcul de la requête est lue pour savoir s'il y a lieu de la recalculer avant l'affichage, en fonction de la valeur du paramètre.
  • le second, nommé INITLEVEL, est utile pour les requêtes non définies en SQL. Il permet de définir le niveau de détail auquel on se place pour l'affichage des résultats de la requête, lorsque celle-ci utilise des ruptures (colonne Groupe du requêteur). Au niveau 1, on n'affiche que les rubriques dont le niveau (indiqué dans la colonne niveau) est égal à 1, et les colonne cumulées correspondantes (les autres sont envoyées vides). Au niveau 2, on donne le détail des rubriques de niveau 1 et 2, et ainsi de suite.

STA (Statistique)

Ce type de source correspond à l'affichage d'une statistique, dont le code est donné. Les paramètres suivants doivent être donnés :

  • la société (CPY) et le site (FCY) sur lesquels l'interrogation est faite (si une chaîne vide est donnée, on les prendra tous). Une sélection est possible sur ces paramètres.
  • les dates de début et de fin pour les deux intervalles de temps possibles (DATDEB1 et DATDEB2, DATFIN1 et DATFIN2)
  • la valeur des critères (de 1 à 8 maximum, nommés CRIT1 à CRIT8). Le nombre de critères pour lesquels une valeur différente de la chaîne vide est donnée définit la profondeur à laquelle la statistique est affichée.

TRT (Traitement)

Ce type de source correspond à l'exécution d'un traitement normalisé, dont le nom est donné. Des paramètres dépendant du traitement peuvent être demandés (par défaut, il s'agit de 4 paramètres banalisés nommés P1 à P4). Le traitement SUBAPSTRT, et le traitement SUBAPSTRTI documentent la façon de créer une telle source de données.

WEB (Page Web)

Ce type de source correspond à l'affichage d'une page Web dont l'adresse est évaluée à partir du paramètre URL qui doit être défini.

Liste des composants visuels disponibles en standard

  • FAG (Agenda flash)
  • FCA (Calendrier flash)
  • FGR (Graphes flash)
  • ML1 (Multi-liste standard)
  • RE1 (Requête standard)
  • ST1 (Requête standard)
  • TB1 (Tableau standard)

Les composants visuels disponibles en standard sont décrites ci dessous.

FAG (Agenda flash)

Ce composant visuel ne peut être associé qu'à l'agenda CRM (source AGD). Il présente sous la forme d'un calendrier les champs issus de l'agenda, avec des interactions liées aux spécificités de l'agenda CRM. Les paramètres associés sont :

  •  le type d'agenda (FCCTYPE), dont les valeurs peuvent être obtenues par sélection. Les valeurs possibles sont 'XMONTH' (agenda mensuel), 'XWEEK' (agenda hebdomadaire), 'XDAY' (agenda journalier).
  • le numéro de colonne des champs Type d'action, Chrono action, Date de début, Heure de début, Date de fin, Heure de fin, Tiers, Interlocuteur, Catégorie, Objet, Flag effectué. Ces champs sont présentés dans l'ordre exact d'extraction de la source de données AGD ; il n'y a donc pas lieu en général de modifier les colonnes par défaut (de 1 à N) présentées en regard des paramètres, sauf si on désire ne pas mentionner certaines informations, auquel cas on mettra la valeur 0 en regard de la colonne. Ce peut être le cas par exemple si la date de début est égale à la date de fin, ou si la notion d'heure n'est pas utilisée.
  • l'indicateur qui définit si le sélecteur de calendrier utilisé pour passer de l'une à l'autre de ces représentations est présent (2=Oui, 1=Non).

FCA (Calendrier flash)

Ce composant visuel permet de présenter des événements datés sous forme d'un calendrier visuel. Il n'est utilisable qu'avec les sources de données suivantes : traitement, requête.

Les paramètres associés sont :

  • le type de calendrier (FCCTYPE), dont les valeurs peuvent être obtenues par sélection. Les valeurs possibles sont 'XMONTH' (calendrier mensuel), 'XWEEK' (calendrier hebdomadaire), 'XDAY' (calendrier journalier).
  • l'indicateur qui définit si le sélecteur de calendrier utilisé pour passer de l'une à l'autre de ces représentations est présent (2=Oui, 1=Non).
  • le numéro des colonnes de la source de données indiquant respectivement date et heure de début, puis date et heure de fin. Hormis la date de début, qui permet de positionner la donnée sur le calendrier, ces informations peuvent être omises. La sélection directe des champs via clic droit est possible si la source de données associée est une requête.
  • le numéro de colonne de la source de données indiquant l'icône et la couleur utilisées pour symboliser l'événement (champs optionnels).

FGR (Graphes flash)

Ce composant visuel peut être associé aux statistiques (source de données STA), aux requêtes (source de données REQ), ou aux traitements (source de données TRT). Il présente un des montants issus de la source de données sous la forme d'une représentation graphique, avec comme titre un des éléments caractères issus de la source de données. Les paramètres associés sont :

  • le type de graphe (FGGTYPE) qui peut être sélectionné par clic droit dans la liste suivante : 'XLINE' (graphe en lignes), 'XBAR' (graphe en barres), 'XAREA' (graphe en aires), 'XPIE' (graphe en secteurs).
  • les numéros des colonnes du champ Valeur (FGCVAL) affichables. S'il y a une seule colonne, on donne directement son numéro; s'il y en a plusieurs, ils doivent être dans une chaîne de caractères, séparés par un point-virgule. On obtiendra ainsi, par exemple, une syntaxe de type "4;7;3;8"
  • le numéro de colonne correspondant à la valeur affichée par défaut (FGCDEF). Si ce champ n'est pas renseigné, la première valeur est utilisée.
  • le numéro de colonne du champ Intitulé (FGCLABEL) affiché. Il est à noter qu'avec une source STA, le premier montant est la colonne numéro 3, la colonne numéro 1 étant le code stocké dans la table d'agrégation statistique et la colonne 2 étant l'intitulé extrait de la table liée à ce code (seules ces deux colonnes peuvent être utilisées pour renseigner FGCLABEL, puisque toutes les autres colonnes sont numériques dans ce cas).
  • les indicateurs qui définissent si le sélecteur de graphe (FGCSELG) utilisé pour passer de l'une à l'autre des représentations, et le sélecteur de valeur (FGCSELC) pour présenter une des colonnes numériques présentes dans la liste des colonnes possibles, sont présents (2=Oui, 1=Non).

ML1 (Multi-liste standard)

Ce composant visuel permet de présenter des données sous la forme de listes de valeurs (chacune des listes étant identifiée par un bandeau, le fait de cliquer sur le bandeau déclenchant l'affichage du contenu de la liste). Il n'est utilisable qu'avec la source de données de type MLT (multi-liste), et n'a aucun paramètre.

A l'utilisation de ce composant, on notera que les champs soulignés, qui correspondent à la clé principale des fiches listées, permettent, par simple clic, d'entrer sur la fiche correspondante en tunnel.

RE1 (Requête standard)

Ce composant visuel permet de présenter des données sous la forme d'un tableau de valeurs. Il n'est utilisable qu'avec la source de données de type REQ (requête), et n'a aucun paramètre.

Contrairement au composant TB1 qui est plus générique, il autorise un dépliage des niveaux successifs de la requête à partir du niveau à laquelle elle est présentée (paramètre INITLEVEL), par l'appui sur une icône présente sur le bandeau de titre du tableau.

A l'utilisation de ce composant, on notera que certains champs peuvent être soulignés. Dans ce cas, il s'agit de clés principales liées à des tables en ligne dans la requête. Un simple clic sur le champ permet alors d'entrer sur la fiche correspondante en tunnel.

ST1 (Requête standard)

Ce composant visuel permet de présenter des données sous la forme d'un tableau de valeurs. Il n'est utilisable qu'avec la source de données de type STA (statistique), et n'a aucun paramètre.

Contrairement au composant TB1 qui est plus générique, il autorise un zoom sur une des lignes de la requête à partir du niveau à laquelle elle est présentée (paramètre INITLEVEL), par l'appui sur une icône présente sur le bandeau de titre du tableau.

A l'utilisation de ce composant, on notera que certains champs peuvent être soulignés. Dans ce cas, il s'agit de clés principales liées à des tables en ligne dans la statistique. Un simple clic sur le champ permet alors d'entrer sur la fiche correspondante en tunnel.

TB1 (Tableau standard)

Ce composant visuel permet de présenter des données sous la forme de tableaux. Il est utilisable avec les sources de données de type AGD (agenda), REQ (requête), STA (statistique), ou TRT (traitement), et n'a aucun paramètre.

Contrairement aux composants ST1 et RE1, il ne permet aucun zoom ou dépliage. Il est donc bien adapté lorsqu'on désire présenter des données dans une liste à un niveau donné sans permettre de visualiser un sous-détail, même s'il existe.
