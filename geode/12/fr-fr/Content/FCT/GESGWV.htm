Vacations génériques

Cet écran permet de créer, visualiser, modifier, supprimer une vacation générique.

La vacation générique est une vacation pour laquelle l'utilisateur définit les critères de sélection des OL. Elle va permettre de créer en automatique des vacations en fonction de ces critères.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Pour créer une vacation générique, on attribue un code qui l’identifie de façon unique sur 10 caractères et un intitulé. Puis on saisi les différents critères de sélection.

Ecran de saisie

Descriptions des champs

Bloc numéro 1

Code (champ GWVCOD)

Code permettant d'identifier une vacation générique.

champ GWVDESAXX
Intitulé court (champ GWVSHOAXX)
Actif (champ ENAFLG)

Cochez ou décochez cette cache pour spécifier si votre Code Tri est actif ou inactif.

  • Si la case est cochée, vous pouvez appliquer le Code Tri à une vacation ou à une zone de préparation.
  • Si la case est décochée, vous ne pouvez pas appliquer le Code Tri. Si vous essayez de le faire, vous recevrez un message d’erreur qui vous indique que vous ne pouvez pas l’utiliser.   

Règles

Priorité (champ PIO)

Indique l'ordre par lequel les vacations génériques sont lancées si il n'y a un abonnement dédié.

Critères de selection (champ SELCRI)

Nom du code mémo associé à la vacation générique.

Gestion

Permet de sélectionner les OL qui sont gérés en cumul de flux.

Lancement automatique (champ AUTLNCFLG)
Vacation mono OL (champ SNGDOHFLG)

Si coché, une seule vacation par OL est créée par l'abonnement.
Si non coché, pour le lancement d'un abonnement, le nombre d'OL par vacation n'est pas limité en dehors des limites de charges paramétrées.
Ajout autorisé (champ ADDFLG)

Si coché, un abonnement peut ajouter des OL à une vacation non lancée, créée précédemment.
Si non coché, un l'abonnement crée une autre vacation de même "vacation générique".
Cumul de flux (champ WAVFLC)
Code Tri (champ CTSCOD)

Un code de 3 caractères avec lequel vous pouvez définir l’ensemble des critères de tri appliqués à une vacation ou à une zone de préparation. Ce code est alors affecté à un ordre de préparation provenant de la vacation ou de la zone de préparation. 


Le Code Tri est saisissable dans les fonctions suivantes :

  • Vacation générique
  • Vacation
  • Zones de préparation
 Le Code tri

Si le Code tri est appliqué à une vacation générique, il est alors appliqué par défaut aux vacations issues de cette vacation générique.

Limite de charge

Poste de charge (champ LOP)

Activité dont on veut mesurer la charge suivant 1 à 3 unités de mesure.
Unité (champ UOM)

Unité pour laquelle on veut calculer une charge.
Limite de charge (champ MAXLOD)

Indique que l'on veut restreindre la sélection des OL dès que la limite précisée est atteinte.

Missions directives

A bloquer (champ MSNLOK)

Ce flag est utilisé pour initialiser le flag "A bloquer" de la vacation issue d'une vacation générique.
Ce paramètre permet alors, au lancement d'une vacation, de forcer la création des missions associées en état '6-bloqué' outrepassant le paramètrage des missions au niveau site.

 Ce paramètrage ne concerne que les missions de type prélèvement (PKOMVT, PKOPRC et PKOSHP) et les missions de type transfert cumul de flux (TRFMERG). Toutes les autres types de missions suivront le paramètrage des missions.
Priorité (champ MSNPIO)

Ce flag est utilisé pour initialiser le flag "Priorité" de la vacation issue d'une vacation générique.
Ce paramètre permet alors, au lancement d'une vacation, de créer les missions associées dans la priorité définie sur la vacation au lieu de celle définie sur les types de mission (voir paramètrage missions).

 Ce paramètrage ne concerne que les missions de type prélèvement (PKOMVT, PKOPRC et PKOSHP) et les missions de type transfert cumul de flux (TRFMERG). Toutes les autres types de missions suivront le paramètrage des missions.

Messages d'erreur

  • Messages génériques

Il n'y a pas de message d'erreur autre que les messages d'erreur génériques.
