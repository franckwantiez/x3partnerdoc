Livrable

Cette fonction permet de créer et de mettre à jour un livrable.

Un livrable est un ensemble complet, identifié par un code, dans lequel on retrouve tous les paramètres et règles de gestion et toutes les données utilisées. Il permet d'extraire les données lors de la constitution du livrable.

Un livrable se caractérise par :

  • un produit (comme par exemple Sage X3).
  • une édition de diffusion, de publication (comme par exemple Premium).
  • une liste de modules
  • une liste de législations
  • une liste de codes activités
  • une liste de fonctions/badges
  • une liste de représentations/badges
  • une liste de sessions/devices
  • une liste de paramètres/licence
  • une liste de paramètres
  • une liste de kit de paramétrage
  • une liste d'éléments de paramétrage

Il est nécessaire d'avoir défini les principales options liées à la gestion du livrable :

  • les modules qui vont être utilisés (chaque progiciel SAFE X3 est découpé en modules fonctionnels qui peuvent être activés ou non sur un dossier...)
  • les principales options de gestion définies par les codes activités (fonctionnalités, localisations), qui correspondent à la gestion optionnelle de certaines données de la base.
  • des paramètres par défaut (langues utilisées par exemple...)

Les champs suivis du symbole "$" sont pris en compte dans la politique de licence (Policy).

En-tête

Un livrable est défini par un code alphanumérique, auquel est associé un ensemble de paramètres regroupés par onglet. Un livrable contenant un ~ dans son code est un livrable historisé et aucune modification n'est alors autorisée sur cet enregistrement.

Onglet Modules

Dans cet onglet, on retrouve la liste des modules liée au progiciel.

Seul un des trois choix peut être activé ou aucun.

Aucun choix activé signifie que par défaut il sera mis à la valeur 'Non' dans une fiche dossier mais pourra être activé par le client sans condition de licence.

Onglet Options

Dans cet onglet, on retrouve la liste des codes activités de type fonctionnel et de localisation.

Seul un des trois choix peut être activé ou aucun.

Aucun choix activé signifie que par défaut il sera mis à la valeur 'Non' dans une fiche dossier mais pourra être activé par le client sans condition de licence.

Possibilité de recharger les codes activités de la fiche du dossier courant : la valeur par défaut des codes activités du livrable est alors réinitialisée par la valeur des codes activités de la fiche dossier (actif ou inactif) et les notions de 'soumis à licence' ou de 'non disponible' sont réinitialisées à la valeur NON.

Possibilité de recharger les codes activités d'un livrable de même type  : la valeur par défaut , la notion de 'non disponible' et la notion de 'soumis à licence' sont repris à l'identique.

Dans le cas de l'apparition de nouveaux codes activités de type fonctionnel et/ou de localisation et pour un livrable déjà défini , ces derniers sont signalés par un message sur le livrable et sont affichés en bas du tableau. Le bouton 'Enregistrer' est alors dégrisé.

Onglet Eléments paramétrage

Dans cet onglet, on trouve les  éléments de paramétrage filtrés pour un livrable.

Ils sont associés à un modèle de données de type patch et flagué « Définition livrable ».

Les enregistrements dont la table principale porte la notion de législation sont filtrés en fonction des législations référencées pour le livrable en cours de paramétrage.

Tous les enregistrements des  éléments de paramétrage non définis pour un livrable seront par défaut pris en compte lors de la construction du dossier racine.

Pour ajouter des lignes dans le premier tableau, vous avez par click droit l'option 'Sélection d'un nouveau modèle'.

Pour ajouter des lignes dans le second tableau, vous avez par click droit l'option 'Sélection de modèle en cours'.

La colonne 'Invalide' n'est visible que sur des éléments de paramétrage qui ne sont plus cohérents, comme dans le cas d'un modèle de données qui n'existe plus. Dans ce cas, l'option 'Sélection sur le modèle en cours' n'est plus accessible et le seul choix possible est de supprimer la ligne en question. Vous avez aussi la possibilité de faire le choix fonction  'Suppression des lignes invalides', choix accessible depuis la barre d'outils. Cet option supprime ,après confirmation, toutes les lignes des éléments de paramétrage invalides.

Vérif cohérence

Ce bouton permet de lancer une vérification de cohérence du livrable.

La vérification porte sur les points suivants :

  • Contrôle sur les modules dans le cas où l'application ajoute des modules non référencés dans le livrable.
  • Contrôle de l'existence d'au moins une langue de définie dans le livrable
  • Contrôle de l'existence d'au moins une législation de définie dans le livrable
  • Contrôle de l'existence des codes activités de type fonctionnel ou localisation
  • Contrôle de l'existence de nouveaux codes activités de type fonctionnel ou localisation
  • Contrôle de l'existence des codes activités de type dimension
  • Contrôle de la définition des codes activités
  • Contrôle de l'existence des paramètres
  • Contrôle de la législation définie sur des paramètres
  • Contrôle des valeurs saisies sur les paramètres
  • Contrôle du modèle sur les éléments de paramétrage
  • Contrôle de la cohérence entre le modèle et la table définie dans le livrable
  • Contrôle unicité du couple modèle - table
  • Contrôle de la législation définie sur les éléments de paramétrage
  • Contrôle de l'existence de la fonction et/ou du badge syracuse définis dans l'onglet 'Badges'
  • Contrôle de la définition des fonctions.
  • Contrôle de l'existence de la représentation et/ou du badge syracuse définis dans l'onglet 'Badges'
  • Contrôle de la définition des représentations.
  • Contrôle de l'existence des sessions définis dans l'onglet 'Contrôle accès'
  • Contrôle de l'existence des paramètres/licence définis dans l'onglet 'Contrôle accès'
    • Une trace est générée au lancement.

 

Génération

Ce bouton permet de lancer la génération des fichiers json associée au livrable.

Avant de générer ces fichiers, la vérification de cohérence est lancée. Si le livrable est correct, un fichier est ensuite généré sous X3_PUB : fichier Policy.

Une trace est générée au lancement.

 

Historisation

Ce bouton permet d'historiser/archiver la description d'un livrable actif. Une confirmation est demandée si l'utilisateur n'est pas le responsable du livrable.

Avant d'archiver la définition du livrable, la vérification de cohérence est lancée. Si le livrable est correct, le livrable est alors dupliqué avec le code = <codedu livrable>~V<numero version>. Cet enregistrement est alors non modifiable. Example un livrable SAGEERP sera historisé en SAGEERPV72 pour la version 1.7.2.

Une trace est générée au lancement.

 

Barre de menus

Options / Vérifications / Vérification cohérence tous livrables

Cette option lance la vérification avec génération d'une trace pour tous les livrables actifs mais non historisés.
Options / Actions / Suppression lignes invalides

Cette option supprime ,après confirmation, toutes les lignes des éléments de paramétrage invalides.
Options / Actions / Génération tous les livrables

Cette option lance la génération json avec génération d'une trace pour tous les livrables actifs mais non historisés.

Chargement valeurs à partir d'un livrable

Cette sélection peut être déclenchée sur l'onglet 'Options', sur l'onglet 'Dimensions' ou sur l'onglet 'Paramètres'.

Elle affiche les livrables de même type et permet d'en sélectionner un seul (les livrables historisés n'apparaissent pas dans cette sélection).
