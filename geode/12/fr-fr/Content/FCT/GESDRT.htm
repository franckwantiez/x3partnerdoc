Expéditions par tournée

A partir de cet écran, vous pouvez :

Lancer le process d’expédition d’une tournée d’expédition.

La confirmation d’expédition peut être réalisée par OL ou bien par tournée d’expédition (fonction décrite).

La confirmation d’expédition par tournée ou par OL est nécessaire si le paramétrage de l’O.L le stipule (gestion des expéditions).

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Liste OL
  • Onglet Chargement

Pour expédier une tournée: la sélectionner puis sélectionner les OL dans l'onglet "Liste OL".

Seules les tournées non validées (non confirmées) sont affichées.

En-tête

Cette onglet récapitule les caractéristiques de la tournée sélectionnée (numéro, Date et heure d'expédition, code transport et mode de transport, la référence transport et le quai).

Inter site: permet de préciser si la tournée est de type "inter-site" ou non.

    • Si oui, la tournée est de type "inter-site" et elle a pour destination un autre site. Auquel cas, le champ suivant, "site de destination", doit être renseigné.
    • Si non, la tournée est a destination des clients finaux.

Site de destination: permet d'indiquer le site de destination d'une tournée réputée "inter-site". Ce champ est obligatoire lorsque le paramètre précédent est activé à 'oui'.

    • Le site de destination doit être différent du site courant.
    • Le site de destination doit appartenir au même dossier que le site courant.

A la confirmation d'une tournée "inter-site", plusieurs documents dédiés à ce type d'expédition sont générés (selon paramétrage):

- Bon de livraison inter-site (TNSBL) => 1 BL inter-site par OL présent dans la tournée, selon paramétrage de la configuration des sorties (paramétrage niveau déposant). 

- Récépissé de transport inter-site (TNSRT) => 1 RT inter-site par client destinataire présent dans la tournée, selon paramétrage de la fiche transporteur.

- Bordereau de transport inter-site (TNSBT) => 1 BT inter-site par tournée inter-site, selon paramétrage de la fiche transporteur.

Pour chacun de ces 3 documents, si le paramétrage n'est pas fait pour les expéditions de type "inter-site", se sont les documents 'normaux' qui sont édités.

Onglet Liste OL

Etape 1: Sélectionner les lignes des OL à expédier ou faire clic droit souris "Tout cocher" pour que le système détecte tous les OL pouvant être expédiés.

Un certain nombre de contrôle sont réalisés au moment de la sélection en particulier :

Pour les O.L sans colisage, il existe au moins un mouvement  de sortie à expédier (ni reporté, ni annulé à l’expédition, ni annulé à la préparation).

Pour les OL colisés, le système s’assure en plus de la condition ci dessus, que toutes les opérations nécessaires au traitement de l’O.L ont été réalisés. Ceci selon le contexte de l’O.L.

Par exemple, il ne sera pas possible d’expédier un O.L non chargé si la configuration de l’O.L stipule la gestion du chargement.

De même il ne sera pas possible d’expédier un OL partiellement palettisé, même si la configuration d’expédition de l’OL ne mentionne pas d’étape de contrôle de chargement. Pour réaliser cet exemple, il est nécessaire de reporter ou d’annuler par les fonctions correspondantes les colis ou les UM qu’il ne faut pas expédier.

Etape 2: L’action du bouton valide l’action.

Contrôles : les OL des UM à expédier peuvent être expédiés  (ce contrôle est réalisé pour éviter qu’on accepte d’expédier une UM en plusieurs fois).

Traitement:
Pour les OL non expédiés de la tournée car non suffisamment avancés ou reportés ou non sélectionnés : les OL sont désaffectés automatiquement de la tournée.

Pour les OL non entièrement expédiés (en cas de report): ils restent affectés à la tournée d’origine et devront être réaffectés manuellement à une autre tournée d’expédition.

Pour la tournée expédiée:
Mise à jour du statut Validé et calcul des informations poids/volume/M.L réellement expédiés.
Création d’un bordereau de transport (BT) par tournée d’expédition.
Création d’un récépissé transport (RT) par code client livré.
Création ou modification du bon de livraison (BL) définitif ou provisoire.

Pour les OL expédiés, selon la configuration des OL:
Mise à jour des états des entités expédiés à l’état 9-Expédié.
Mise à jour des quantités expédiées de lignes OL.
Calcul de l’OL reliquat si la gestion des reliquats est active au niveau de l’OL.

Le BT, les RT sont édités en fonction du paramétrage du transporteur associés à

Le BL et la liste de colisage sont édités en fonction de la configuration des sorties de l’OL expédié.

Etats

Par défaut, les états suivants sont associés à la fonction :

  EXPEDITION : Statistiques d'expédition

Mais ceci peut être modifié par paramétrage.

Barre de menus

Fonctions / Mouvements consommables

Permet de déclarer 1 ou n mouvements de sortie de stock de consommables au titre de la tournée.

Cette saisie peut se faire à tout moment, quelque soit l'état de la tournée.

Pour réaliser un mouvement de sortie de consommable, il faut saisir diverses informations:

    • Le code consommable: définit dans la fiche article des consommables
    • Le code mouvement: doit appartenir à la famille des traitements 'CONSOMMABLE ' et être de type 'SORTIE'
    • La quantité: quantité du mouvement de sortie

Comme le mouvement de sortie du consommable est déclaré à partir d'une entité connue, les informations, transporteur, mode de transport, client et intitulé client, sont, le cas échéant, renseignés avec celles de l'entité.

Pour un mouvement de sortie, les informations fournisseur et intitulé fournisseur ne sont pas accessibles ainsi que le type de l'entité et le numéro de cette dernière.

Selon le paramétrage, la saisie d'un code opérateur est demandée pour valider le mouvement d'entrée. A la validation, le stock est incrémenté.

Après validation, le mouvement n'est plus modifiable. Par conséquent, il n'est pas actualisé dans le cas où des mises à jour sont réalisées sur l'entité à laquelle ce mouvement est lié.

Il est possible de générer 3 états:

    • Dans Fichier / imprimer / fiche:
        • Stock des consommables trié par consommable ou par quantité
        • Impression écran pour l'enregistrement sélectionné 
    • Dans Fichier / imprimer / liste:
        • Mouvements des consommables trié par consommable ou par date

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

OL XXX : Aucun mouvement à valider

L’OL sélectionné est à l’état 8 mais il a fait l’objet de report d’expédition qu’il faut réintégrer pour l’expédier.

Les messages d'erreur liés à la déclaration de mouvement de consommables sont:

Erreur zone "consommable". Zone obligatoire.

Le code du consommable doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Zone obligatoire.

Le code mouvement doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Code mouvement de type non consommable.

Le code mouvement renseigné doit appartenir à la famille des traitements 'CONSOMMABNLE'.

Erreur zone "Code mouvement". Code mouvement de type entrée.

Le code mouvement renseigné doit obligatoirement être de type 'SORTIE'.

Erreur zone "Quantité". Zone obligatoire

La quantité du mouvement d'entrée doit obligatoirement être renseignée.

Erreur de l'application. Le déposant doit être renseigné pour cette origine.

Lorsqu'un mouvement d'entrée est lié à une entrée attendue ou une entrée directe, le déposant doit obligatoirement être renseigné. 

Erreur zone "N°origine mvt". Référence origine non trouvée.

Le numéro de l'entité Warehousing doit obligatoirement être renseigné et existante pour le site.

Erreur zone "Consommable". Consommable bloqué en sortie.

Le consommable doit obligatoirement être autorisé en sortie pour réaliser un mouvement de sortie.

Erreur de l'application. Forçage expé OL : Code opérateur non autorisé.

Le code opérateur doit être autorisé à faire du "forçage expédition OL".

Le site de destination doit être différent du site courant.

Une tournée dite "inter-site" ne peut pas avoir comme site de destination le site courant.

Erreur zone "Site de destination": Fiche inexistante

Le site de destination doit appartenir au même dossier que le site courant.
