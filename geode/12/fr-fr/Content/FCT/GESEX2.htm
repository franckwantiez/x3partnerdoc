Annulations par OL

Utilisez cette fonction pour annuler partiellement ou totalement l’expédition d’un OL non colisé.

Cette fonction est utile lorsque l’expédition de l’OL doit être définitivement supprimée, soit parce que le client final ne souhaite plus être livré, soit parce que la marchandise a été endommagée entre la préparation de la commande et son expédition.

Dans le cadre d'une expédition intersite, l'annulation de la marchandise d'un OL prochainement "en transit" est possible tant que la tournée intersite à laquelle il est affecté n'est pas expédiée. Dit autrement, la marchandise ne peut pas être annulée si elle est réputée "en transit".

L’annulation d’expédition d’un OL colisé est réalisée par la fonction Annulation d’expédition par colis.

A la différence de la fonction Report d’expédition, l’annulation supprime irrémédiablement la préparation de l’OL, alors que le report indique que l’expédition est seulement reportée dans le temps.

Un code mouvement de type ajustement automatique doit être défini dans la table des codes mouvements pour le site courant.

L’OL doit être à l’état8-En attente d’expédition et ne doit pas être en mode précolisage ou colisage déclaratif.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

L’écran est constitué d’une unique section de saisie.

Ecran de saisie

Suite à la sélection de l’OL à annuler, les lignes OL et leurs mouvements de sortie correspondants sont affichées.

Pour chaque ligne de l’OL sélectionné, le système affiche la quantité :

- Préparée : quantité UC préparée, validée lors de la validation de l’OP

- Expédiée : quantité UC expédié préalablement

- Annulée: quantité UC supprimée par la fonction Annulation OL

- Reportée : quantité UC à réintégrer ou à reporter

- A annuler : ce champ est mis à jour en fonction de la saisie de la quantité à annuler sur les mouvements de sortie associés

- A expédier : quantité à expédier lors de la prochaine expédition. Le champ est mis à jour en fonction de la quantité à annuler sur la ligne d'OL.

- En transit : quantité en UC d'un OL réputé "en transit", c'est-à-dire d'un OL expédié vers un autre site. Cette marchandise ne peut pas être annulée tant qu'elle est "en transit". 

Pour chaque mouvement de sortie (selon la ligne de l’OL sélectionné), le système affiche :

- la Quantité UC : la quantité UC du mouvement de sortie

- la quantité A annuler : ce champ permet de saisir la quantité à annuler sur le mouvement de sortie concerné

- le Contenant destination : ce champ permet d'indiquer le contenant dans lequel on veut réintégrer le stock.

Pour annuler partiellement ou totalement un OL, il faut :

- saisir les quantités à annuler sur chaque mouvement de sortie et si l'article gère les numéros de série localisés, il faudra aussi indiquer ces numéros de séries qu'on annule ( action Saisie numéros de série sur le mouvement de sortie),

- saisir l‘adresse de réintégration de stock,

- saisir le code mouvement à utiliser,

- valider l’action par l'action Enregistrer.

Le code mouvement de type ajustement automatique est utilisé pour créer les objets de stock correspondants dans le magasin de réintégration.

Une impression, lancée automatiquement, permet de lister les éléments réintégrés.

Procéduralement, ces stocks devront ensuite être réintégrés dans des magasins / adresses de réserve par la création de mouvements de transfert.

Seules les lignes de nature normale et kit sont saisissables.

Par défaut lors de la saisie d’un kit, les lignes composantes s’y rattachant sont sélectionnées.

Les mouvements de sortie sont alors mis à jour et découpés s’il s’agit d’une annulation partielle de ligne. Le statut des mouvements annulés passe à l’état 6 - Annulé à l’expédition.

En fonction de la gestion des reliquats de l’OL, l’OL passera à l’état 2 – En attente de xième préparation, ou à l’état 9 - Expédié, soldé.

Les mouvements annulés ne sont pas pris en compte lors de l’expédition de l’OL.

Dans le cas où les numéros de série sont gérés, l’annulation ne pourra se faire que si les numéros de série à annuler sont saisis.

Pour saisir les numéros de série, il faut se positionner sur le mouvement de sortie et utiliser l'action Saisie numéros de série.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Livraison partielle interdite.

Le paramétrage de l’entête de l’OL ne permet pas d’annuler l’expédition de manière fractionnée.

Erreur zone ‘Numéro’, Ordre de livraison : 9999, Fiche inexistante

Le numéro d'OL saisi n’existe pas.

N° de série non préparé

Le numéro de série saisi n’a pas été préparé et ne peut donc pas être annulé.

Erreur zone ‘Emplacement de réintégration’, zone obligatoire

La saisie de l’emplacement de réintégration est obligatoire.

Erreur zone ‘Emplacement de réintégration’, Magasins : XX : Fiche inexistante

Le magasin de réintégration saisi n’existe pas.

Erreur zone ‘Emplacement de réintégration’, RCA : Type de magasin <> Quai

Le magasin de réintégration saisi n’est pas de type quai.

Format incorrect

Le format de l’adresse de réintégration saisie n’est pas correct.

Emplacements : QUA Q3,  Fiche inexistante

L’emplacement de réintégration saisi n’existe pas.

Erreur zone ‘Code mvt ajustement’, zone obligatoire

La saisie du code mouvement d’ajustement est obligatoire.

Erreur zone ‘Code mvt ajustement’,  Code mouvement : A,  Fiche inexistante

Le code mouvement d’ajustement saisi n’existe pas.

Erreur zone ‘Code mvt ajustement’,  E01 : Code mvt interdit

Le code mouvement saisi doit être de type ajustement automatique.

Nombre de N° série <> Nombre d’UC à annuler ligne N°X (0<>1)

Le nombre de numéros de série saisi à annuler est différent de la quantité à annuler pour une ligne d'OL gérée avec numéro de série.

Erreur zone 'A annuler'. Quantité maximum XX UC

La quantité à annuler ne doit pas être supérieure à la quantité à expédier moins la quantité en transit.
