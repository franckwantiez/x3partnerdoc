Gestion du portail

Cette fonction uniquement interactive permet d'interagir de trois manières avec le portail interatif.

Elle permet :

  • d'initialiser, de dupliquer ou bien de supprimer un portail
    • associé à un utilisateur,
    • ou associé à un code métier (dans ce cas le code doit être saisi sous la forme  "_" + code métier).

 Une option permet de copier les favoris en cas de duplication. Lorsqu'un utilisateur ayant un portail vide se connecte à l'application, son portail est initialisé :

    • par le portail associé au code métier "_"+code métier (si un code métier est associé à cet utilisateur),
    • ou si pas de code métier ou pas de portail associé au code métier, par les vignettes "Menu" et "Favoris".

  • de vérifier l'ensemble des portails définis dans la base et de détecter d'éventuelles incohérences (une trace est produite),
  • de rechercher les cas d'utilisation d'une vignette donnée.

Gestion de l'écran

Ecran de saisie

Descriptions des champs

Initialisation

Utilisateur à initialiser (champ UT1)

Ce champ permet de saisir un utilisateur ou bien un code de la forme "_" + un code profil fonction.
Initialiser vide (champ INITVIDE)

Si cette case est cochée, il est possible de créer le portail utilisateur à vide de l'utilisateur à initialiser.

Si le portail _XXX existe, où XXX est le profil fonction de l'utilisateur à initialiser, alors ce portail est repris.
Supprimer (champ INITSUPP)

Si cette case est cochée, il est possible de supprimer le portail utilisateur ou le portail profil fonction qui est renseigné dans le champ Utilisateur à initialiser.

A partir de (champ UT2)

Permet de saisir un utilisateur ou bien un code de la forme "_" + un code profil fonction.

Dupliquer les favoris (champ DUPFAV)
Exécuter (champ ALANCER)

Cet icône permet de créer, de dupliquer ou de supprimer un portail pour un utilisateur dans le cas où le champ Utilisateur à initialiser est renseigné.

  • Si la case Initialiser vide est cochée, un portail est créé à vide pour l'utilisateur à initialiser. Si un portail existe sur le code profil fonction de cet utilisateur, alors il est repris.

Exemple :

Si l'utilisateur à initialiser a pour profil fonction VEN ( paramétré directement dans la fiche utilisateur ou bien dans le code métier associé) et si le portail "_VEN" existe, alors il sera repris.

  • Si la case Supprimer est cochée, le portail de l'utilisateur à initialiser est supprimé.
  • Si les deux cases ne sont pas cochées, le portail de l'utilisateur A partir de est dupliqué vers l'Utilisateur à initialiser.

Vérification

Exécuter (champ LANCEVERIF)

Recherche

Vignette (champ VIGNETTE)

Code d'une vignette portail.

Nombre d'utilisation (champ NBUST)

Est le nombre d'utilisations rencontrées pour une vignette donnée.

Exécuter (champ ARECH)

Cette icône permet de lister le nombre d'utilisations d'une vignette renseignée dans le champ Vignette.

Tableau

Utilisateur (champ AUT1)

Est l'utilisateur du portail dans lequel la vignette est présente.

Onglet (champ ATAB)

Est l'onglêt du portail dans lequel la vignette est présente.

Module (champ AMOD)

Titre de la vignette dans le portail qui est modifiable
