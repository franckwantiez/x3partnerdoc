Gestion en masse

Cet écran permet de créer/modifier ou supprimer en masse des adresses pour un magasin donné.

Remarque: La saisie des adresses doit être conforme à la structure d’adresse définie pour le magasin.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Principe d'utilisation de cet écran

Etape 1: la saisie obligatoire du magasin pour lequel on souhaite mettre à jour les emplacements.

Etape 2: choix du type d’action (Modification/Suppression/Création) à entreprendre sur les emplacements.
On peut, par exemple, créer des emplacements en masse, ayant les mêmes caractéristiques. On peut aussi modifier une donnée identique à plusieurs emplacements. Enfin, si l’on souhaite réorganiser la structure du magasin, on peut supprimer des emplacements.

Etape 3: choix de la MAJ codes contrôles. Si la case est cochée, les codes contrôles des emplacements seront recalculés en fonction de la clé préalablement définie dans la gestion des magasins.

Etape 4: sélection des Adresses
La structure d’adresse du magasin saisi est affiché (table des magasins).
Les données que l’on peut renseigner sont les suivantes:

  • Début: Spécifier pour chacune des composantes adresses l’adresse de début.
  • Fin: Spécifier pour chacune des composantes adresses l’adresse de fin.
  • Pas: Pas qu'il faut franchir pour passer de la valeur courante d'un élément à la valeur suivante à traiter.

Exemple: Pour créer toutes les rangées impaires d'une allée, saisissez la première rangée impaire dans la zone de début "de" et indiquez un pas de 2.

Le champ «Nb» indique le nombre maximal de chaque composantes de la structure d’adresse du magasin selon les bornes saisies.
Le champ «Nombre d’emplacements» indique le nombre d’emplacements total qui vont être créés/modifiés/supprimés par le traitement.

Etape 5: les critères de sélection
Dans le cas de la modification ou de la suppression de l’emplacement, les critères de sélection vont permettre d’afficher les caractéristiques d’un des emplacements de type picking ou réserve.

Par exemple, si on souhaite modifier tous les emplacements de type réserve ayant comme classe de rotation «A», on va saisir les critères Type d'emplacement = Reserve et Classe de rotation = A.

Etape 6: les critères de modification
Après avoir validé, le premier emplacement trouvé correspondant aux critères saisis est affiché.
On peut alors saisir les critères à modifier.

Enfin, en cliquant sur le bouton "Enregistrer", la modification va être réalisée sur tous les emplacements de la sélection (et pas seulement l’exemple affiché) et un compte rendu de mise à jour et de suppression sera affiché.

En-tête

 

Descriptions des champs

Bloc numéro 1

Adresse de l’emplacement dans le magasin.

Magasin (champ STO)

Code permettant d’identifier le magasin. Il est vérifié dans la table des magasins.
Type (champ STOTYP)

Le type de magasin définit la fonction logistique d’un magasin:

  • magasin principal: picking, réserve, masse, fourre tout
  • magasin mono adresse: magasin logique dans lequel les stocks sont localisés; une seule adresse sans contrôle de remplissage peut être définie pour ce magasin
  • magasin d’éclatement: type de magasin nécessaire pour utiliser la fonctionnalité "cumul de flux" (préparation en 2 phases); les emplacements "zone éclatement" peuvent être considérés comme des emplacements réserve utilisés comme des emplacements pickings temporaires
  • magasin quai (de réception, d'expédition): utilisé par les fonctionnalités de "Rendez vous transporteur" et "Stock à quai des unités logistiques"
  • magasin Cross-docking: utilisé par la fonctionnalité de "Cross-docking".
  • magasin Tampon: magasin de stockage provisoire.
  • magasin Transfert en 2 phases: utilisé par la fonctionnalité de "Transfert en 2 phases".
Actif (champ ENAFLG)

Sélectionnez cette case à cocher pour activer la fiche courante.

Les enregistrements non sélectionnés conservent leur contenu et paramétrage, mais ne pourront pas être utilisés en rappelant leur code dans :

  • les autres enregistrements, comme les documents et paramètres,
  • les traitements en masse.

Les habilitations sur une fonction donnée peuvent interdire la création d'une fiche active. Dans ce cas, la case est désactivée par défaut. Elle est modifiable uniquement par un utilisateur autorisé, ou via un Workflow de signature.
Adresse (champ ADD)
champ ADDSTC
Tri rangement (champ SRGADD)

Cette zone permet de conserver pour chaque emplacement une seconde adresse qui sera utilisée systématiquement lors de la recherche d'emplacements en entrée pour obtenir un mode de remplissage différent de l'ordre alphanumérique des adresses. La structure de l'adresse de rangement est reprise de celle paramétrée dans la table des magasins.  
champ SRGSTC

Bloc numéro 2

Type de stockage (champ STY)

Le type de stockage définit les caractéristiques de gestion d'un ensemble d'emplacements. Ces règles de gestion sont mises en oeuvre lors d'une étape d'adressage en entrée.
Type (champ TYP)
Déposant (champ DEP)

Code qui représente le déposant. En terme logistique, la notion de déposant permet de traiter des flux ou trafics distincts dans le même site. Elle recouvre la notion de client donneur d’ordres pour les prestataires de services.
Après le site, le déposant est la deuxième clef d’accès aux fichiers Article, Entrée et Ordre de Livraison.
Pour l’emplacement, le code déposant est renseigné si l’emplacement est affecté au déposant ou à l’article, et si l’emplacement est mono déposant ou mono article et qu’il n’est pas vide.

 
Article (champ ITM)

Un emplacement peut être affecté à un article dans les cas suivants :

- Si l’emplacement est de type «Picking», il doit obligatoirement être affecté à un article pour être actif.

- Si l’emplacement est de type «Réserve» et que le type de stockage impose qu’il soit réservé à un article.

- Si l’emplacement est de type «Réserve» banalisée, cette zone indique le code article propre à l’objet stocké à cette adresse, si le type de stockage de l’emplacement impose qu’il soit mono article.

- Si plusieurs articles sont présents au même instant dans l’emplacement ou si celui-ci est vide, la zone est à blanc.

Un emplacement ne peut être dédié à un article que s’il est réservé à un déposant.
Cnt de référence (champ REFCTR)

Le contenant de référence d’un emplacement représente le contenant prioritaire dédié à l’emplacement et dans lequel les notions d’occupation sont exprimées. L'emplacement peut recevoir d'autres contenants que le contenant de référence prévu, par le biais des "correspondances de contenants".

 
Niveau (champ REFCTRLVL)

Validation des mouvements

Saisie code contrôle (champ CTLCODDAC)

Le système demande la saisie d’une clé de contrôle lors de la validation des mouvements si :
L’article, le magasin, l’emplacement et le flux en question (configuration des entrées, des sorties, tables des sites) ont un paramétrage « saisie des codes contrôles égal à « oui ».
Code contrôle (champ CTLCOD)

Valeur du code contrôle (alphanumérique de 6 caractères). Une vérification sera réalisée lors de la saisie du code contrôle afin d’en vérifier sa véracité.

Contrôle dimensions

Poids maximal.

Contrôle dimensions (champ DIMCTL)

Indique le type de contrôle en dimension réalisé lors de l'introduction de stock (en entrée, en transfert/réapprovisionnement) dans cet emplacement:


  • "Non": pas de contrôle
  • "Hauteur": on contrôle que la hauteur du contenant du stock est inférieur ou égale à la hauteur de l'emplacement
  • "Hauteur + (Largeur <=> Profondeur): on contrôle les 3 dimensions avec pivotement possible de l'objet de stock de 90° (la largeur devenant la profondeur et la profondeur devenant la largeur). Il suffit qu'un pivotement convienne pour que l'emplacement puisse accepter le stock.
  • "Hauteur + Largeur + Profondeur": on contrôles les 3 dimensions sans pivotement.
  • "Largeur <=> Profondeur": on contrôle la largeur et la profondeur avec pivotement possible de l'objet de 90°
  • "Largeur + Profondeur": on contrôle les 2 dimensions largeur et profondeur sans pivotement.
Poids maxi (champ WEIMAX)

Seuil de réappro (champ MINRPN)

Si le stock disponible est inférieur au seuil de réapprovisionnement, le système va effectuer un réapprovisionnement préventif si le mode de sortie de l’article l’indique.
Seuil adressage objets complets (champ INPCTLBIN)

Dans ce contexte, l'objectif est d'éviter que des objets rompus se retrouve en réserve car l'emplacement de picking est plein.

Ce champ vous permet de limiter le nombre d'objets complets (non rompus selon le paramétrage du contenant article) dans l'emplacement picking.

Il est pris en compte lors de l'adressage en entrée, ainsi que lors des réapprovisionnements de nécessité et préventif. 


Exemple

Soit un emplacement picking de niveau 5 (UUC) :

- l'article a une palette complète de 1 PP1 de 25 UUC,

- le seuil d'adressage en objets complets = 50 UUC,

- le taux d'occupation en objets complets = 0 %.


1/ On adresse en entrée un mouvement palette de 1 PP1 de 25 UUC (complet) qui va sans problème dans l'emplacement de picking UUC :

- en stock = 25 UUC,

- seuil adressage objets complets = 50 UUC,

- taux d'occupation objets complets = 50 %.


2/ On adresse en entrée un deuxième mouvement palette de 1 PP1 de 25 UUC (complet) qui va sans problème dans l'emplacement de picking UUC :

- en stock = 50 UUC,

- seuil adressage objets complets = 50 UUC,

- taux d'occupation objets complets = 100 %.


3/ On adresse en entrée un troisième mouvement palette de 1 PP1 de 25 UUC (complet) qui n'ira pas dans l'emplacement de picking UUC car on a atteint le seuil d'adressage en objets complets.


4/ En revanche, si on adresse en entrée un mouvement palette de 1 PP1 de 10 UUC (rompu), il ira dans l'emplacement de picking UUC :

- en stock = 60 UUC,

- seuil adressage objets complets = 50 UUC,

- taux d'occupation objets complets = 100 %.

Gestion

Mvt picking en cnt supérieur (champ CTRSUP)

Permet de préciser dans quel plan de palettisation (ou contenant) le système va exprimer les mouvements de sortie à prélever dans l'emplacement picking.
Il s’agit d’un contenant maximum, le contenant utilisé dans les mouvements dépendra de la quantité à sortir. Si non renseignée, les mouvements sont exprimés dans le contenant de référence de l'emplacement.

Conditions d'utilisation: (pour exprimer un mouvement de sortie d'un emplacement picking UUC en contenant supérieur CC1)
- l'article ne doit pas avoir d'emplacement picking CC1
- le contenant CC1 doit être paramétré sur l'emplacement picking UUC de l'article dans la zone 'mvt en cnt supérieur'
- le contenant CC1 de l'article doit être "modulo colisage"

voyons quelques exemples, soit 1 article avec le plan de pal suivant: 1PP1= 2CC1= 24uuc, 1 CC1= 6BB1= 12uc, 1 BB1= 2 uc

Exemple 1a: picking uc, modulo BB1
Demande de 5 uc, si mvt picking en cnt supérieur à oui, alors création de 1 mvt de 2 BB1 et 1 mvt de 1 uuc à prélever dans l'emplacement picking uc.
Demande de 5 uc, si mvt picking en cnt supérieur à non, alors cération de 1 mvt de 5 uc.

Exemple 1b: picking uc, modulo CC1 & BB1
Demande de 15 uc, si mvt picking en cnt supérieur à oui, alors création de 1 mvt de 1 CC1 , 1 mvt de 1 BB1 et 1 mvt de 1 uuc à prélever dans l'emplacement picking uc.

Exemple 2: 3 picking CC1, BB1 et UUC
Demande de 6 BB1, si mvt picking en cnt supérieur à oui, alors création de 1 mvt de 1 CC1 à prélever dans le picking BB1 mais comme il existe un emplacement CC1, le mvt de 6 BB1 d'origine est conservé.
Picking pivot (champ DYNPIC)
Priorité picking multiple (champ PIOPIC)

Ce champ permet d'afficher, d'affecter et de modifier une priorité picking à un couple contenant + article. Les emplacements affectés sont classés par priorité (zone numérique). Le chiffre le plus petit correspond à la priorité la plus haute.

Réapprovisionnement

champ RPN

Ce paramètre conditionne le fonctionnement de l'algorithme de réapprovisionnement picking. Les réapprovisionnements de cet emplacement seront réalisés à partir d’emplacements du type qui est choisi : picking ou réserve.
Niveau (champ RPNLVL)

Si les réapprovisionnements sont de type picking, ce champ indique au système à partir de quel niveau de contenants il réalisera les réapprovisionnements.

Dynamisation picking

Picking temporaire (champ TPYPIC)

Indique si l'emplacement picking traité est temporaire ou permanent.

La notion de picking temporaire est utilisée dans le module de dynamisation picking pour savoir s'il faut transférer ou non le stock de l'ancien emplacement picking d'un contenant article vers le nouveau.

Le transfert de stock a lieu si le nouvel emplacement picking est permanent.

Onglet Quai

Descriptions des champs

Généralités

Type quai (champ TYPQAY)

indique le type de quai (Réception, Expédition ou Général):

  • Un quai de réception ne peut gérer que des RDV transporteur de type Réception. 
  • Un quai d'expédition ne peut gérer que des RDV transporteur de type expédition. 
  • Un quai général peut gérer tout type de RDV transporteur (réception, expédition ou autre).
Niveleur de quai (champ NIVQYM)

Indique si l'adresse quai possède un système de niveleur de quai (permettant de combler l'espace entre le camion et le quai ainsi que de compenser la différence de hauteur).

Hauteur de quai (champ HEIQYM)

Indique la hauteur du quai. (saisissable uniquement si il n'y a pas de niveleur de quai)

Charge maximale (champ CHGMAX)

indique la capacité maximale du quai en unité de charge (défini sur le site). Si non saisi, alors la capacité est infini.

Ce paramètre est pris en compte uniquement si on utilise l'analyse de charge. Il permet de prévenir d'affecter des RDV transporteurs à des quais ne pouvant pas supporter leur charges.

champ QAYTYPUNI

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur zone Magasin, zone obligatoire

Le magasin doit obligatoirement est renseigné.

Erreur Magasin, Fiche inexistante.

On a tenté de saisir un Magasin inexistant.

Erreur zone Magasin, Magasin de type Mono adresse

Le magasin sélectionné ne doit pas être de type mono adresse

Erreur zone Fin, La borne est inférieure à la borne début

La borne doit être supérieur ou égal à la borne de début. Le format est celui défini dans la fiche magasin.

Erreur zone Type de stockage, le type de stockage doit être Principal

Le type de stockage doit être de même type que le type d’emplacement.

Erreur type de stockage, zone obligatoire

Le type de stockage doit obligatoirement est renseigné.

Erreur Type de stockage, Fiche inexistante.

On a tenté de saisir un type de stockage inexistant.

Erreur zone Cnt de référence, zone obligatoire

Le contenant de référence doit obligatoirement est renseigné.

Erreur zone Cnt de référence, Fiche inexistante.

On a tenté de saisir un contenant de référence inexistant.

Erreur zone Déposant, zone obligatoire

Le déposant doit obligatoirement est renseigné.

Erreur zone Déposant, Fiche inexistante.

On a tenté de saisir un déposant de référence inexistant.

Erreur zone Rotation, zone obligatoire

La classe de rotation doit obligatoirement est renseignée.

Erreur zone Rotation, Fiche inexistante.

On a tenté de saisir une classe de rotation inexistante.

Erreur zone Affectation, zone obligatoire

La classe d’affectation doit obligatoirement est renseignée.

Erreur zone Affectation, Fiche inexistante.

On a tenté de saisir une classe d’affectation inexistante.

Erreur "Dernier emplacement pour l'allée de circulation (Inventaire): X    Veuillez supprimer l'allée de circulation avant de supprimer cet emplacement".

Il est impossible de supprimer le dernier emplacement d'une allée de circulation X d'inventaire tant que cette allée existe.

Erreur "Dernier emplacement pour l'allée de circulation (Prélèvement): X    Veuillez supprimer l'allée de circulation avant de supprimer cet emplacement".

Il est impossible de supprimer le dernier emplacement d'une allée de circulation X de prélèvement tant que cette allée existe.

Erreur "Dernier emplacement pour l'allée de circulation (Réapprovisionnement): X    Veuillez supprimer l'allée de circulation avant de supprimer cet emplacement".

Il est impossible de supprimer le dernier emplacement d'une allée de circulation X de réapprovisionnement tant que cette allée existe.
