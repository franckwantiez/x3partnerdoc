X3 Warehousing

Propriétés Oracle

Cette fonction permet de visualiser les principales propriétés de la base de données lorsque la base de données utilisée est Oracle™.

Les données ici présentées ne sont pas modifiables.
Leur modification doit se faire par les outils d'administration propres à la base de données utilisée.

Avertissement

Cette fonction permet, par des requêtes sur des tables et vues du système, de donner des informations techniques sur la base de données. L'interprétation des résultats donnés par cette fonction est du ressort d'un administrateur de bases de données, et peut dépendre de la version de base de données utilisée.

Cette fonction permet de visualiser les propriétés de la base de données.
Ces éléments sont présentés sur trois onglets : le premier donne les caractéristiques générales, le deuxième donne la liste des paramètres et options positionnés au démarrage de la base, le troisième donne des informations sur la mémoire utilisée par la base de données.

Mode ARCHIVELOG (champ LOGMOD)

Indique si le mode ARCHIVELOG est activé ou désactivé.

  • NOARCHIVELOG :
    En mode, NOARCHIVELOG, les fichiers de redo log online sont réécrits à chaque fois qu'un fichier de redo log online est rempli et qu'un log switch est lancé. LGWR ne réécrit pas sur un groupe de fichier de redo log tant que le checkpoint pour ce groupe n'est pas terminé.
  • ARCHIVELOG :
    Si la base de données est configurée dans le mode ARCHIVELOG, alors les groupes de redo remplis doivent être archive. Comme toutes les modifications faites sur la base de données sont enregistrées dans les fichiers de redo log online, le DBA peut utiliser la sauvegarde présente sur le disque dur et les fichiers de redo log archivés pour restaurer la base de données sans perdre aucune donnée comitée.

    On peut archiver les fichiers de redo log :
    - Manuellement.
    - Automatiquement : Méthode recommandée.

    Le paramètre d'initialisation LOG_ARCHIVE_START, lorsqu'il est à TRUE indique que l'archivage se fait automatiquement. A FALSE, indique que le DBA le fait manuellement. En mode automatique, l'archivage se fait grâce au process ARCn, et manuellement avec des requêtes SQL..

Onglet Mémoire

Sur le troisième onglet, les informations concernant l'utilisation de mémoire de la base de données sont présentées.

Cet écran donne des informations sur les deux zones mémoires d'une base de données Oracle.

Mémoire SGA (System Global Area)

C'est la zone mémoire qui contient les structures de données accessibles par tous les processus. Elle gère le partage des données des différents utilisateurs.

Cette zone mémoire est composée de plusieurs élements:

  • Pool partagé : ce paramètre précise la taille du pool partagé. Le pool partagé contient notamment des curseurs partagés, des procédures stockées, des structures de contrôle et des tampons de message d'exécution parallèle. Les valeurs élevées améliorent les performances dans les systèmes multi-utilisateurs. Les valeurs faibles utilisent moins de mémoire. Cette valeur est attribuée au paramètre de fichier d'initialisation SHARED_POOL_SIZE. Remarque : Pool partagé est un paramètre dynamique.
  • Cache de tampon : précise le nombre de tampons contenus dans le cache de tampon de la base de données. La taille de chaque tampon situé dans le cache correspond à celle d'un bloc de données Oracle (qui est spécifiée par le paramètre d'initialisation DB_BLOCK_SIZE) ; par conséquent, chaque tampon de base de données situé dans le cache peut contenir un seul bloc de données lu depuis un fichier de données. La taille du cache est limitée, donc toutes les données du disque ne peuvent pas tenir dans le cache. Cette valeur est attribuée au paramètre de fichier d'initialisation DB_CACHE_SIZE(10g) ou DB_BLOCK_BUFFERS.
  • Zone de mémoire LARGE POOL : permet de préciser la portion de mémoire allouée à la zone de mémoire LARGE POOL. Cette portion de mémoire est utilisée par le serveur partagé en tant que mémoire de session, par l'exécution parallèle pour les tampons de message et par la sauvegarde pour les tampons d'E/S sur disque. Cette valeur est attribuée au paramètre de fichier d'initialisation LARGE_POOL_SIZE.
  • Pool Java : précise la taille du pool Java. Cette valeur est attribuée au paramètre de fichier d'initialisation JAVA_POOL_SIZE.

Un graphique présente l'allocation de mémoire que chaque composant SGA utilise.

Mémoire PGA (Program Global Area)

C'est la zone mémoire qui contient toutes les données ne nécessitant pas d'être partagées. Elle gère le fonctionnement des différents processus.
