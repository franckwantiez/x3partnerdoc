X3 Warehousing

Mouvements d'assemblage

Cet écran permet de visualiser tous les mouvements d'assemblages.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Général
  • Onglet Gestion

On sélectionne un N° de mouvement et le système affiche ses caractéristiques:

Entête

  • Données OA
  • Données Lignes d'OA
  • Données du mouvement

Onglets

  • Général pour le détail du mouvement
  • Gestion pour les infos de tracabilité du mouvement

Remarque: un mouvement d'assemblage n'est pas obligatoirement associé à un OA/ligne d'OA. En effet pour une vacation d'assemblage avec pas de gestion des rompus (option "Transfert rompu" à Non),  on transfère plus de kits/composants que nécessaire (ou au minimum égal). Ainsi seule la quantitée nécessaire est associée à l'OA.

Caractéristiques

L’Origine permet une classification des articles selon un critère choisi par l’utilisateur.

Elle est classiquement utilisée pour d&ea cute;finir ou préciser la provenance (géographique ou autre) des articles gérés par le système.

Elle permet de gérer des différences qualitatives au niveau d'un objet de stock : péremption, reconditionnement, ….

Code mvt (champ MVT)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
