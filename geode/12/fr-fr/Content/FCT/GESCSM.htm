Entrées

Cette option permet de réaliser des mouvements d'entrée de consommable afin d'incrémenter son stock.

Ces incrémentations de stock peuvent se faire ponctuellement et indépendamment des opérations liées aux flux d'entrées. On parle alors de mouvement d'origine "directe".

A l'inverse, ces mouvements d'entrée en stock peuvent se faire, à tout moment, dans les fonctions suivantes:

    • Entrées attendues
    • Réceptions administratives
    • Validation des réceptions
    • Entrées directes

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Pour réaliser un mouvement d'entrée de consommable, il faut saisir diverses informations obligatoires:

    • Le code consommable: définit dans la fiche article des consommables
    • Le code mouvement: doit appartenir à la famille des traitements 'CONSOMMABLE ' et être de type 'ENTREE'
    • La quantité: quantité du mouvement d'entrée

Dans le cadre d'une saisie d'un mouvement d'entrée dit "Direct", toutes les autres informations sont facultatives. Sinon, lorsque le mouvement d'entrée est lié à une entité de Warehousing telle que l'entrée attendue ou l'entrée directe, le déposant devient obligatoire et le numéro de l'entité doit être précisée. 

A la saisie de l'entité d'origine, les informations, transporteur, mode de transport, fournisseur et intitulé fournisseur, sont, le cas échéant, renseignés avec celles de l'entité.

Pour un mouvement d'entrée, les informations client et intitulé client ne sont pas accessibles.

Selon le paramétrage, la saisie d'un code opérateur est demandée pour valider le mouvement d'entrée. A la validation, le stock est incrémenté.

Après validation, le mouvement n'est plus modifiable. Par conséquent, il n'est pas actualisé dans le cas où des mises à jour sont réalisées sur l'entité à laquelle ce mouvement est lié.

Il est possible de générer 3 états:

    • Dans Fichier / imprimer / fiche:
        • Stock des consommables trié par consommable ou par quantité
        • Impression écran pour l'enregistrement sélectionné 
    • Dans Fichier / imprimer / liste:
        • Mouvements des consommables trié par consommable ou par date

Ecran de saisie

Descriptions des champs
Numéro (champ CSMNUM)

Numéro du mouvement du consommable.

Consommable (champ CSB)

Code du consommable définit dans la table des consommables GEODE. 20 caractères alphanumériques. Obligatoire
Code mouvement (champ MVTCOD)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
Quantité (champ QTY)
Déposant (champ DEP)

Code qui représente le déposant. En terme logistique, la notion de déposant permet de traiter des flux ou trafics distincts dans le même site. Elle recouvre la notion de client donneur d'ordres pour les prestataires de services.

Après le site, le déposant est la deuxième clef d'accès aux fichiers Article, Entrée et Ordre de Livraison. Pour l'emplacement, le code déposant est renseigné si l'emplacement est affecté au déposant ou à l'article, et si l'emplacement est mono déposant ou mono article et qu'il n'est pas vide.
Entité d'origine (champ CSMORI)

Indique l'entité d'origine du mouvement d'entrée ou de sortie du consommable.

Pour les entrées:

    • EA: entrée attendue existante pour le couple (site; déposant).
    • Réception: réception administrative existante pour le site en-cours.
    • ED: entrée directe existante pour le couple (site; déposant)

Pour les sorties:

    • OL: ordre de livraison existant pour le couple (site; déposant)
    • Tournée: tournée existante pour le site en-cours

Direct: aucun lien et contrôle avec les entités existantes Geode. l'opérateur peut saisir ce qu'il souhaite.
No Entité (champ REFORI)

Numéro de l'entité Geode à laquelle le mouvement du consommable est lié.

Transporteur (champ CRR)

Code identifiant le transporteur. Ce code doit être défini dans la table des transporteurs.
Mode transport (champ CRRMOD)

Généralement, le mode de transport correspond aux notions de messagerie, affrètement, express, ... Le mode transport est vérifié par rapport à la table des modes de transport.
Fournisseur (champ DVR)

Code du fournisseur qui doit être définit dans la table des fournisseurs.
Intitulé fournisseur (champ DVRNAM)

Raison sociale du fournisseur (par défaut, la raison sociale est définie pour le fournisseur dans la table des fournisseurs).
Client (champ CLT)

Code du client livré qui doit être définit dans la table des Clients.
Intitulé client (champ CLTNAM)

Raison sociale du client (par défaut, la raison sociale est définie pour le client dans la table des clients).
Commentaire (champ CSMREM)
Date (champ CREDAT)
Opérateur (champ OPENAM)

Code opérateur ayant validé le mouvement.

Etats

Par défaut, les états suivants sont associés à la fonction :

  CSB : Stock des consommables

  CSBMVT : Mouvements consommables

Mais ceci peut être modifié par paramétrage.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur zone "consommable". Zone obligatoire.

Le code du consommable doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Zone obligatoire.

Le code mouvement doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Code mouvement pas de type consommable.

Le code mouvement renseigné doit appartenir à la famille des traitements 'CONSOMMABNLE'.

Erreur zone "Code mouveement". Code mouvement de type sortie.

Le code mouvement renseigné doit obligatoirement être de type 'ENTREE'.

Erreur zone "Quantité". Zone obligatoire

La quantité du mouvement d'entrée doit obligatoirement être renseignée.

Erreur de l'application. Le déposant doit être renseigné pour cette origine.

Lorsqu'un mouvement d'entrée est lié à une entrée attendue ou une entrée directe, le déposant doit obligatoirement être renseigné. 

Erreur zone "N°origine mvt". Référence origine non trouvée.

Le numéro de l'entité Warehousing doit obligatoirement être renseigné et existante pour le site.

Erreur zone "Consommable". Consommable bloqué en entrée.

Le consommable doit obligatoirement être autorisé en entrée pour réaliser un mouvement d'entrée.
