Consultation

Cet écran permet de consulter l'ensemble des mouvements ou un type de mouvement pour un consommable ou pour l'ensemble des consommables du site.

Les critères de sélection sont les suivants:

    • Code consommable
    • Déposant
    • Type de mouvement
    • Origine mouvement

L'état du stock du consommable est affiché.

Gestion de l'écran

  • En-tête
  • Onglet Mouvements

Les critères de recherche sont présentés dans la partie supérieure de l'écran. Ce sont les suivants:

  • Code consommable: si la zone est à blanc, on considère l'ensemble des codes consommable.
  • Déposant: si la zone est à blanc, on considère l'ensemble des déposant du site en-cours.
  • Type de mouvement: entrée, sortie ou bien les deux.
  • Origine mouvement: sens entité Warehousing; entrée attendue, réception administrative, entrée directe, ordre de livraison, tournée ou bien toutes entités confondues.

    • Une fois les critères de sélection précisés, la recherche peut être est lancée. 

    Le résultat de la recherche est affiché sous forme de tableau.

    Le "Total mouvements" affiché en bas à droite de l'écran, correspond à la somme de tous les mouvements d'entrée moins la somme de tous les mouvements de sortie pour le site en-cours.

    La "Quantité stock" affichée en bas à droite de l'écran, représente le stock disponible du consommable pour le site en-cours. 

    Des liens vers les entités sont accessibles par un clic droit sur les lignes du tableau lorsque le mouvement du consommable est liée à une entité Warehousing.

    Il est possible d'obtenir 2 états à partir de cet écran:

      • Stock des consommables, trié par consommable ou par quantité
      • Mouvements des consommables, trié par consommable ou par date

    Saisie des critères de sélection.

    Descriptions des champs
    Consommable (champ CSB)

    Code du consommable définit dans la table des consommables GEODE. 20 caractères alphanumériques. Obligatoire
    Type mouvement (champ TYPMVT)

    Type du mouvement. Deux types de mouvements possibles pour un consommable:

        • entrée (+)
        • sortie (-)
    Déposant (champ DEP)

    Code qui représente le déposant. En terme logistique, la notion de déposant permet de traiter des flux ou trafics distincts dans le même site. Elle recouvre la notion de client donneur d'ordres pour les prestataires de services.

    Après le site, le déposant est la deuxième clef d'accès aux fichiers Article, Entrée et Ordre de Livraison. Pour l'emplacement, le code déposant est renseigné si l'emplacement est affecté au déposant ou à l'article, et si l'emplacement est mono déposant ou mono article et qu'il n'est pas vide.
    Origine mouvement (champ CSMORI)

    Indique l'entité d'origine du mouvement d'entrée ou de sortie du consommable.

    Pour les entrées:

      • EA: entrée attendue existante pour le couple (site; déposant).
      • Réception: réception administrative existante pour le site en-cours.
      • ED: entrée directe existante pour le couple (site; déposant)

    Pour les sorties:

      • OL: ordre de livraison existant pour le couple (site; déposant)
      • Tournée: tournée existante pour le site en-cours

    Direct: aucun lien et contrôle avec les entités existantes Geode. l'opérateur peut saisir ce qu'il souhaite.

    Onglet Mouvements

    Affichage, sous forme de tableau, du résultat de la recherche trié par n° de mouvement consommable descendant.

    Etats

    Par défaut, les états suivants sont associés à la fonction :

      CSB : Stock des consommables

      CSBMVT : Mouvements consommables

    Mais ceci peut être modifié par paramétrage.
