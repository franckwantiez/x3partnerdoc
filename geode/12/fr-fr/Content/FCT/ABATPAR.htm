Paramètres serveur batch

Votre logiciel Sage X3 inclut un serveur batch intégré vous permettant de lancer des tâches en masse sur le serveur d’application.

Utilisez cette fonction pour modifier les paramètres qui exécutent le serveur batch et contrôler le lancement des tâches batch.

Veuillez consulter la documentation Cinématique de fonctionnement du serveur batch avant de définir ou de modifier les paramètres du serveur batch.

Gestion de l'écran

La fonction Paramètres serveur batch affiche un écran unique dans lequel vous devez renseigner les caractéristiques de votre serveur batch.

Onglet Écran de saisie

Utilisez cet écran pour définir les paramètres de votre serveur batch.

Descriptions des champs

Serveur batch

Nom du serveur (champ APPLI)

Ce champ affiche le nom du dossier dans lequel le serveur batch est utilisé. Le nom fait référence à l’application, par exemple SERVX3 pour Sage X3.
Utilisation des fichiers batchs (champ JOB)

Activez cette case à cocher pour lancer des requêtes à partir de fichiers créés dans un répertoire spécifique.

Vous pouvez uniquement utiliser cette fonction si votre paramètre utilisateur EXTBATCH - Gestion des fichiers batchs (chapitre SUP, groupe AUZ) a pour valeur Oui.

Se reporter à l’annexe technique pour plus d’informations sur la soumission de requête par fichier.

Bloc numéro 2

Répertoire des fichiers lancement requête (.job) (champ REPJOB)

Cet ensemble de paramètres définit les différents répertoires utilisés pour la gestion des requêtes soumis par fichier. Chaque répertoire doit être localisé sur le serveur d'application par défaut. Il est possible d'indiquer un serveur différent sur le réseau en utilisant la syntaxe server@directory.

Un répertoire ne doit pas être localisé sur un poste client.

L'extension utilisée pour les fichiers traités est indiquée entre parenthèses dans l'intitulé de la rubrique.

Se reporter à la documentation correspondante pour plus d’informations sur la soumission de requête par fichier.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Maximum : N

Trop de tâches batch ont été lancées simultanément. Se reporter au champ Nb maximum de requêtes actives (NBTACHE) pour visualiser le nombre de tâches batch pouvant être exécutées simultanément sur ce serveur d’application.
