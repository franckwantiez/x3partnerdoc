Réceptions administratives

Une réception correspond au traitement des marchandises d’une entité homogène de livraison (classiquement un approvisionnement). Elle peut correspondre à tout ou partie d’une entrée attendue relative à un ou plusieurs déposants.

Une réception regroupe également un certain nombre de caractéristiques:

  • Un transporteur/mode de transport qui peut être différent de celui attendu
  • Une référence transport (c’est à dire un identifiant commun entre le site et le fournisseur)
  • Une charge de réception
  • Un poids brut

La réception administrative s’inscrit dans la Chaîne des entrées.

Cet écran permet de gérer les réceptions administratives.

Pour créer une réception, il est nécessaire d’affecter à la réception les entrées attendues réceptionnées.

Pour cela, 3 possibilités:

  • Aller dans la liste de gauche, dans le tiroir «EA à réceptionner» et sélectionner à l’aide la souris les enregistrements proposés. Ceux ci sont regroupés par date et heure prévisionnelles de réception, et par transporteur.
  • Saisir directement dans les zones Déposant et Numéro,de l’onglet 2 les N°EA à réceptionner.
  • Utiliser les options de Sélections standard pour sélectionner une code mémo.

Selon le paramètre du mode de réception de chaque EA, une liste de contrôle peut être éditée automatiquement. Cette liste est utilisée en tant que support pour vérifier physiquement les articles réceptionnées et les retranscrire dans Warehousing.

Barre de menus

Fonctions / Analyse de charge

Cette fonction permet de lancer l'analyse de charge sur la réception administrative sélectionnée.

Fonctions / Mouvements consommables

Permet de déclarer 1 ou n mouvements d'entrée en stock de consommables au titre de la réception administrative.

Cette saisie peut se faire à tout moment, quelque soit l'état de la RA.

Pour réaliser un mouvement d'entrée de consommable, il faut saisir diverses informations:

    • Le code consommable: définit dans la fiche article des consommables
    • Le code mouvement: doit appartenir à la famille des traitements 'CONSOMMABLE ' et être de type 'ENTREE'
    • La quantité: quantité du mouvement d'entrée

Comme le mouvement d'entrée du consommable est déclaré à partir d'une entité connue, les informations, transporteur, mode de transport, fournisseur et intitulé fournisseur, sont, le cas échéant, renseignés avec celles de l'entité.

Pour un mouvement d'entrée, les informations client et intitulé client ne sont pas accessibles ainsi que le type de l'entité et le numéro de cette dernière.

Selon le paramétrage, la saisie d'un code opérateur est demandée pour valider le mouvement d'entrée. A la validation, le stock est incrémenté.

Après validation, le mouvement n'est plus modifiable. Par conséquent, il n'est pas actualisé dans le cas où des mises à jour sont réalisées sur l'entité à laquelle ce mouvement est lié.

Il est possible de générer 3 états:

    • Dans Fichier / imprimer / fiche:
        • Stock des consommables trié par consommable ou par quantité
        • Impression écran pour l'enregistrement sélectionné 
    • Dans Fichier / imprimer / liste:
        • Mouvements des consommables trié par consommable ou par date

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

 Suppression impossible

L’état de l’entrée attendue ne permet pas la modification de la réception.

Les messages d'erreur liés à la déclaration de mouvement de consommables sont:

Erreur zone "consommable". Zone obligatoire.

Le code du consommable doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Zone obligatoire.

Le code mouvement doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Code mouvement de type non consommable.

Le code mouvement renseigné doit appartenir à la famille des traitements 'CONSOMMABLE'.

Erreur zone "Code mouveement". Code mouvement de type sortie.

Le code mouvement renseigné doit obligatoirement être de type 'ENTREE'.

Erreur zone "Quantité". Zone obligatoire

La quantité du mouvement d'entrée doit obligatoirement être renseignée.

Erreur de l'application. Le déposant doit être renseigné pour cette origine.

Lorsqu'un mouvement d'entrée est lié à une entrée attendue ou une entrée directe, le déposant doit obligatoirement être renseigné. 

Erreur zone "N°origine mvt". Référence origine non trouvée.

Le numéro de l'entité Warehousing doit obligatoirement être renseigné et existante pour le site.

Erreur zone "Consommable". Consommable bloqué en entrée.

Le consommable doit obligatoirement être autorisé en entrée pour réaliser un mouvement d'entrée.
