Modes de préparation

Cet écran permet de créer, visualiser, modifier, supprimer un Mode de Préparation.

Le Mode de Préparation regroupe l’ensemble des règles de gestion lié à la phase de préparation d’OL. Il permet d’en déterminer le mode de traitement lors du lancement en préparation.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Pour créer un mode de préparation, on attribue un code qui l’identifie de façon unique sur 2 caractères et un intitulé. Puis on saisi les différentes caractéristiques.

Ecran de saisie

Les informations saisissables dans cet écran concernent:

  • La gestion des vacations.
  • La constitution des OP.
  • Le découpage de charge.
  • Le colisage.
  • la palettisation.
  • l'expédition "mono-lot" des lignes d'OL.
  • La zone de prépa multi.
  • Les éditions.
  • La séquence.
Descriptions des champs

Bloc numéro 1

Code qui représente le mode de préparation. Le mode de préparation regroupe l’ensemble des règles de gestion lié &agra ve; chaque O.L. Il permet d’en déterminer le mode de traitement lors du lancement en préparation.

Mode préparation (champ PMD)
Intitulé (champ PMDDESAXX)
Intitulé court (champ PMDSHOAXX)
Actif (champ ENAFLG)

Sélectionnez cette case à cocher pour activer la fiche courante.

Les enregistrements non sélectionnés conservent leur contenu et paramétrage, mais ne pourront pas être utilisés en rappelant leur code dans :

  • les autres enregistrements, comme les documents et paramètres,
  • les traitements en masse.

Les habilitations sur une fonction donnée peuvent interdire la création d'une fiche active. Dans ce cas, la case est désactivée par défaut. Elle est modifiable uniquement par un utilisateur autorisé, ou via un Workflow de signature.

Vacation

Le système n'accepte de sortir au titre d'une ligne d'OL titulaire d'un numéro de réservation, que des quantités portant ce même num&e acute;ro. En cas de rupture sur N° de réservation, le solde n'est jamais prélevé sur du stock non réservé.

Lorsqu’une ligne d’O.L présente une date FIFO sur le champs « Date FIFO », le système recherche l’ensemble des obj ets de stocks ayant une date strictement égale à celle mentionnée. Si le paramètre n’est pas coché, le système recherche une date supérieure ou égale à celle de ligne O.L.

Active la gestion de cumul de flux au niveau du mode de préparation.

Réservation exclusive (champ RSAXCL)
Date FIFO exclusive (champ FIFDATXCL)
Contrat date (champ CONDAT)

Indique si le mode de préparation autorise la gestion du contrat date client.

Picking multiple (champ MLTPIC)

Si la case est cochée, l'article est géré en multipicking. Vous pouvez donc créer plusieurs emplacements picking avec un même contenant de référence pour cet article.

Cumul de flux (champ WAVFLC)
Mono lot en sortie (champ SNGLOTMGT)

Ce paramètre permet d'activer la gestion du "mono-lot" en sortie dont le principe est de pouvoir servir une ligne d'OL à partir d'un seul et même lot.

Cross docking (champ CDKAUZ)

Ce paramètre indique le type de gestion cross-docking pour ce mode de préparation. 3 valeurs sont possibles:

  • "Non": on ne fait pas de cross-docking
  • "Oui si rupture": au lancement en vacation, on fait un adressage standard et si il y a rupture de stock, on lance une recherche cross-docking.
  • "Oui prioritaire": au lancement en vacation, on fait prioritairement une recherche cross-docking. Si pas de stock cross-docking trouvé ou de besoin cross-docking exprimé, on fait alors un adressage standard.

Constitution des OP

L’ordre de prélèvement est potentiellement mixte réserve et picking.

Remarque : ce paramétrage est ignoré par la fonction de précolisage qui prépare les O.P. à partir d’un regroupement de colis.

L’Ordre de Prélèvement réserve est mono OL.

L’ordre de prélèvement picking est mono OL.

L’ordre de prélèvement est mono zone de préparation. On a alors au moins un O.P. par zone de préparation.

Si l’OP est m ulti-zone de préparation, il faudra définir une zone de préparation dans le champ situé à droite. Les OP seront édités sur l’imprimante dédiée à cette zone de préparat ion.

Les mouvements de sortie de niveau différent (palette, carton, UC) sont regroupés par niveau de contenant sur un même O.P.

L’Ordre de Prélèvement regroupe les O.L. appartenant à la même tournée d’expédition.

OP mixte (champ MPO)
OP mono OL Réserve (champ SNGDLORSV)
OP mono OL Picking (champ SNGDLOPIC)
OP mono zone préparation (champ SNGPKZ)
OP mono niveau sortie (champ SNGOUTLVL)
OP mono tournée (champ SNGDRN)
OP mono destinataire (champ SNGCLT)

Si coché, les OP seront mono destinataire. I.e. les OP seront composés de mvts d'OL ayant le même destinataire.

Définition du destinataire d'un OL: c'est le trio Code client, Raison sociale (1ere ligne) et code postal de l'onglet "Client livré" de l'OL.

Colisage

Permet de préciser le type de colisage utilisé : pas de colisage, colisage déclaratif ou bien précolisage.

Colisage (champ PKG)

Découpage de charge

Le système utilise les critères propres aux zones de préparation pour découper les O.P. picking.

Le système utilise les critères propres aux zones de préparation pour découper les O.P. réserve.

Picking (champ PIKWKLPIC)
Réserve (champ PIKWKLRSV)

Prépalettisation

Prépalettisation (champ BEFPLZ)

Permet d'activer la prépalettisation.  Ce flag est accessible uniquement si le précolisage est actif.

Prélèvement

Critère de regroupement RF (champ OUMGRUPMD)

permet d'activer un critère optionnel de regroupement des mouvements en prélèvement RF:

  • "Aucun":  pas de critère
  • "Client":  le client livré de l'OL sera un critère de regroupement.
  • "OL":  le numéro d'OL sera un critère de regroupement
  • "Ligne d'OL": le numéro de la ligne d'OL sera un critère de regroupement.

Zone prépa multi

Kit (champ PKZKIT)

Cette zone de préparation sera celle utilisée dans le cas d’O.P kit (multi zones de préparation). Le code de la zone de préparation est vérifié dans la table des zones de préparation.
OP/Colis/UM (champ PKZ)

Cette zone de préparation sera celle utilisée dans le cas d’O.P multi zones de préparation (en particulier l’imprimante utilisée pour les éditons). Le code de la zone de préparation est vérifié dans la table des zones de préparation.

Editions

Flag qui indique si le fichier édition des O.P. est systématiquement généré.

Les objets de stocks de mêmes caractéristiques seront regroupés sur l’état afin d’en faire une seule ligne.

Le système édite des étiquettes de sortie.

OP (champ PKOEDT)
Mouvements regroupés (champ PKOEDTGRU)
Etiquettes colis (champ LBLEDT)
Etiquettes UM (champ SHPEDT)

Séquence

Indique au système selon quelle séquence doivent être édités les O.P. : soit par Zone préparation - OL (pour une zone , le système édite tous les O.P. qui contiennent des O.L. faisant l'objet d'une préparation dans cette zone), soit par OL - Zone préparation (pour un O.L., le système édite tous les O.P. pour toutes les zones concernées par cet O.L. Cette opération est réitérée pour tous les O.L.).

Séquence (champ PKOEDTORD)
