Ressources

Cette fonction permet de gérer les ressources sur le site courant. Une ressource représente une personne physique dans l'entrepôt. Elle est donc unique par site. Ainsi à un utilisateur, on peux lui associer une ressource par site et une seule.
La gestion des ressources permet de:
  • gérer le planning prévisionnel de ses ressources travaillant dans l'entrepôt
  • suivre l'activité réelle de ses ressources dans l'entrepôt (validation des codes opérateur)
  • Utiliser la fonctionnalité des missions directives (module optionnel connu sous le nom de GEOMIPS)

Une ressource Warehousing se caractérise en standard par:
  • son planning type: définition des jours fériés, chomés et ouvrés (incluant les activités planifiées) sur une année type.  Ce planning sera utilisée lors de l'initialisation d'un mois dans le planning prévisionnel.
  • son process de validation par code opérateur dans les procédures portables: on a 3 process possible
    • Manuel: en cas de validation opérateur, la saisie d'un code opérateur sera toujours demandée (c'est le fonctionnement par défaut)
    • Semi-automatique: en cas de validation opérateur, la saisie d'un code opérateur sera demandée à la première validation, ensuite pour toute les validations suivantes, on ne le demandera plus et Warehousing utilisera automatiquement le code saisi la première fois.
    • Automatique: en cas de validation opérateur, on ne demandera pas de saisie mais Warehousing utilisera automatiquement le code opérateur défini sur la ressource.

A cela s'ajoute de nouvelles caractéristiques issues de la fonctionnalité des missions directives (GEOMIPS):

  • son type de ressource
  • son code enchainement des missions
  • sa charge maximale autorisée.
  • son statut sur les missions directives (disponible ou non, mission en cours, matériels utilisés et sa position actuelle dans l'entrepôt).

 Cette fonctionnalité est disponible à compter de la version Advanced

Gestion de l'écran

L'écran est constitué d'un unique écran de saisie.

Ecran de saisie

Chaque ressource créée est identifiée de façon unique sur un site par un code égal au code utilisateur auquel il est associé.

Ensuite on retrouve les différents options permettant de définir le planning type et sa propre gestion des codes opérateurs.

Descriptions des champs

Bloc numéro 1

Code (champ RSS)

Indique le code de la ressource.
Nom (champ NOMRSS)

Bloc numéro 3

Type ressource (champ RSSTYP)

Code identifiant le type de ressource (défini par site).

Un type de ressource identifie sur un site un ensemble de ressources susceptibles de réaliser les même types de mission, d'accéder aux mêmes zones de mission et d'utiliser les mêmes types de matériel.
Enchaînement missions (champ MSNCHN)

code identifiant une régle d'enchainement de mission (défini par site).

Un code d'enchainement de mission défini l'ordre de recherche des types missions à enchainer.

Procédures portables

Gestion code opérateur (champ TYPCODOPE)

Ce paramètre permet d'indiquer la gestion du code opérateur que l'on souhaite suivre dans les procédures portables pour une ressource.

On a 3 options possible:

  • "Manuelle": on demande toujours la saisie d'un code opérateur à chaque validation
  • "Semi-automatique": on demande la saisie d'un code opérateur à la première validation. Ensuite Geode utilisera ce même code pour toutes les autres validations.
  • "Automatique": la validation par code opérateur sera automatique en utilisant la valeur définie dans le champ "opérateur".

Missions directives

Indisponible (champ UVYFLG)

indique si la ressource est indisponible.

Motif (champ UVYCOD)

Matériel de manutention (champ MAT1)

Position (champ UVHPLC)

Etats

Par défaut, les états suivants sont associés à la fonction :

  ETIQRSS : Etiquettes ressources

Mais ceci peut être modifié par paramétrage.

Barre de menus

Fonctions / Planning

permet d'accéder au planning prévisionnel de la ressource sélectionnée.

Fonctions / Indisponiblité

permet de rendre disponible ou non la ressource sélectionnée.

Fonctions / Suivi des opérateurs

permet d'accéder au suivi des validations opérateurs.

Fonctions / Statistiques

permet d'accéder aux statistiques de la ressource sur la réalisation des missions directives en RF (Module optionnel connu sous le nom de GEOMIPS)
