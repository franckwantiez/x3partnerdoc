Validation unitaire

Cet écran permet de valider partiellement ou totalement un transfert ou un réapprovisionnement  de type cumul de flux, ou bien de l’annuler.

Le mouvement de transfert/réapprovisionnement comprend un statut permettant la visualisation de l’état d’avancement.

Autres conditions

AFFICHAGE

En entrant dans la fonction, seuls les Transferts/Réapprovisionnement de type cumul de flux à traiter sont affichés.

Affichage du N° de la vacation de type cumul ayant généré le transfert/réappro ainsi que la date et l’heure de génération de l’ordre.

Les mouvements de transfert/réappro d’un ordre de transfert/réappro sont affichés de manière regroupés ; c’est à dire que tout mouvement ayant les caractéristiques ci après identiques sont cumulés (au niveau de l’affichage) afin d’en faciliter leur lecture :

  • N° de transfert 
  • Magasin et adresse de prise et de dépose
  • Code réservation, N° de lot, N° de support, code origine et date FIFO
  • Quantité, Contenant article, contenant homogène
  • Etat
    Exemple :
    soit 2 mouvements d’1PP1 de 100 UUC pour la même destination seront affichés regroupés soit une seule ligne de 2PP1 de 100 UUC soit 200 UUC.

Le type de transfert/réappro peut prendre les valeurs suivantes :

  • Transfert cumul de flux
  • Réapprovisionnement cumul de flux

VALIDATION

Pour valider totalement ou partiellement un ordre de transfert/réappro, il est nécessaire de cocher dans le tableau les enregistrements à valider, puis d’actionner le bouton "Enregistrer".
Les mouvements de transferts/réappro sont alors validés, les emplacements de prise et de dépose ainsi que les objets de stock sont mis à jour.
Dans le cas des ordres de transfert, les quantités transférées des lignes des OL de la vacation sont mises à jour et le stock devient disponible dans le magasin d’éclatement utilisé pour la préparation.

Les enregistrements dont la quantité apparaît en police « grisé » ne sont pas sélectionnables, car les mouvements correspondants ont déjà été validés.

Pour modifier la quantité d’un enregistrement, les zones Nb UC, Nb et contenant homogène sont modifiables, uniquement en diminution.
Les contrôles suivants sont réalisés :

  • Pour les emplacements pickings, la quantité saisie doit être un multiple du contenant de référence de l’emplacement picking
  • Le contenant saisi doit faire partie du plan de palettisation du contenant d’entrée de l’article.
  • La quantité saisie doit être un multiple du niveau homogène du contenant de l’article à transférer
  • Pas de transfert partiel autorisé si le paramétrage du magasin du stock à transférer oblige les transferts d’objet complet

En cas de validation partielle d’un enregistrement, le mouvement d’origine est découpé, la quantité saisie est mise à jour sur le mouvement validé (passage à l’état 8-Validé) et le mouvement solde reste à l’état 3-Adressé pour être ultérieurement soit validé soit annulé.

La saisie d’un code contrôle peut être demandée par le système ; la saisie est obligatoire si la zone code Ctrl du tableau comporte le caractère « ? »
Pour cela il faut que le paramétrage article, emplacement, magasin et configuration des entrées/sorties du déposant courant soient paramétrées en conséquence.
La validation ne sera possible que si le code contrôle saisi correspond au code contrôle attendu, et visualisable dans la fiche emplacement.

La saisie d’un code opérateur peut être demandée par le système ;
Pour les mouvements de réappro, il faut que le paramétrage de la configuration des sorties du déposant courant soit paramétrée en conséquence.
Pour les mouvements de transferts, il faut que le paramétrage du site courant indique la saisie d’un code opérateur.
La validation ne sera possible que si le code opérateur saisi est habilité à valider des transferts et/ou des réapprovisionnements en fonction du flux traité.

ANNULATION

L’annulation d’un transfert/réappro ou d’enregistrements d’un ordre est possible en sélectionnant les enregistrements et en actionnant le bouton "Annuler".

L’annulation d’un ordre, ou d’un enregistrement d’un ordre, fait passer les mouvements à l’état 9-Annulé.
Les encours de sortie de l’emplacement de prise, et les cours d’entrée de l’emplacement de dépose sont annulés.
Mise à jour des objets de stock et dans le cas des transferts, mise à jour des quantités non transférées sur les lignes de l’OL.

Il est impossible d’annuler un mouvement de réapprovisionnement déjà utilisé par un mouvement de sortie

L’annulation et la validation totale d’un mouvement peuvent également être obtenue par la fonction de Validation en masse.

Etats

Par défaut, les états suivants sont associés à la fonction :

  ETIQREA : Etiquette de réappros

  REAPPRO : Transferts / Réappros

Mais ceci peut être modifié par paramétrage.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur zone nombre d’UC

La quantité saisie est supérieure à la quantité des mouvements cumulés.

Pas de rompu en picking

La quantité saisie n’est pas multiple de la quantité du contenant de référence de l’emplacement picking

Quantité non multiple de la quantité homogène

La quantité saisie n’est pas multiple du niveau homogène du contenant de l’objet de stock à transférer

Transfert partiel interdit

Le magasin de l’objet de stock à transférer ne permet pas de transfert partiel.

Qté UC à annuler > Total uc – En courts de sortie

Le mouvement de réapprovisionnement à annuler est utilisé pour servir un mouvement de sortie

Contenant absent du plan de palettisation

Le contenant saisi n’appartient pas au plan de palettisation du contenant d’entrée du mouvement.

Code opérateur invalide

Le code opérateur saisi ne permet pas de valider des transferts/réapprovisionnement. Les habilitations d’un opérateur sont visualisables dans la table des opérateurs.

Erreur zone ‘Code ctrl’

Le code contrôle saisi ne correspond pas au code contrôle attendu. La valeur du code contrôle est visualisable sur l’emplacement
