Types de stockage

 

Cette table permet la définition des caractéristiques de gestion d'un ensemble d'emplacements.

Le type de stockage regroupe l'ensemble des règles de gestion (liées à l'emplacement) mises en œuvre lors d'une étape d'adressage en entrée.

Remarque : Le type de stockage détermine la nature réserve / picking de l'emplacement.

En cours d'exploitation, la modification des paramètres de type «Affectation» (Produit, Déposant), et «Mono / Multi» (Déposant, Article, Contenant, Origine, Lot, Date) est acceptée par le système; toutefois, ces changements ne seront pris en compte qu'à partir du moment où les emplacements concernés sont vides.

En cours d'exploitation, si une modification des paramètres liés à la gestion des dates FIFO est effectuée, seuls les objets de stock "entrant" suivront les nouvelles règles de gestion.

Le nombre de types de stockage et leur codification sont totalement libres.

Descriptions des champs

Type de magasin

Type de magasin (champ STOTYP)

Le type de magasin définit la fonction logistique d’un magasin:

  • magasin principal: picking, réserve, masse, fourre tout
  • magasin mono adresse: magasin logique dans lequel les stocks sont localisés; une seule adresse sans contrôle de remplissage peut être définie pour ce magasin
  • magasin d’éclatement: type de magasin nécessaire pour utiliser la fonctionnalité "cumul de flux" (préparation en 2 phases); les emplacements "zone éclatement" peuvent être considérés comme des emplacements réserve utilisés comme des emplacements pickings temporaires
  • magasin quai (de réception, d'expédition): utilisé par les fonctionnalités de "Rendez vous transporteur" et "Stock à quai des unités logistiques"
  • magasin Cross-docking: utilisé par la fonctionnalité de "Cross-docking".
  • magasin Tampon: magasin de stockage provisoire.
  • magasin Transfert en 2 phases: utilisé par la fonctionnalité de "Transfert en 2 phases".

Type d'emplacement

Type d'emplacement (champ SLOTYP)

Le type d’emplacement définit si les emplacements sont de type réserve (gérés de façon banalisée; l'attribution et la libération des emplacements s'effectuent en mode dynamique) ou picking (les emplacements sont affectés à un contenant pour le couple Déposant/Article).
Assemblage (champ BOMAUZ)

Ce paramètre est réservé aux magasins de type mono adresse.

Si coché, les emplacements sur ce type de stockage peuvent être utiliser pour la procédure d'assemblage (assignation d'un emplacement assemblage à la vacation d'assemblage).

Picking

Cela indique si le dépassement de capacité d'un emplacement picking est autorisé dans le cadre d’un adressage manuel ou de type ajustement. L ors de l’adressage automatique, la capacité maximum d’un emplacement picking sera toujours respecté quelle que soit la valeur de ce paramètre.

Si la case est cochée, il n’y aura qu’un seul objet de stock dans un emplacement picking ayant ce type de stockage. Tous les objets de stock entrant prendront les caractéristiques et le numéro d’objet du premier objet présent dans l’emplacement.

Sinon, chaque objet de stock dans un emplacement picking ayant ce type de stockage gardera ses pr opres caractéristiques et son numéro d’objet.

Picking pivot (champ DYNPIC)

Si la case est cochée, l'emplacement picking est de type "picking pivot", c'est à dire qu'il n'est pas affecté à un couple Déposant / Article ; dans ce cas, l'affectation a lieu lors du lancement en vacation. L'utilisation de picking réservé à l'article est prioritaire à l'utilisation des picking pivots quelque soit le paramétrage du mode de sortie de l'article.

Dépassement capacité maxi (champ MAXPIC)
Objet de stock indifférencié (champ SNGSKO)

Editions

Etiquettes objets (champ PALLBLEDT)

Gestion déposant

Gestion déposant (champ DEPMGT)

Indique le mode de gestion par rapport au déposant :
- Affecté : les emplacements sont réservés au déposant saisi lors de la création des emplacements ; l'affectation au déposant est obligatoire si le paramètre "affectation article" est actif.

- Mono : l'emplacement est banalisé en ce qui concerne le déposant ; dès l‘occupation de l’emplacement par un objet de stock appartenant à un déposant, l'emplacement sera réservé à ce déposant et dès que l’emplacement sera à nouveau vide, un nouveau déposant pourra utiliser l’emplacement.

- Multi déposant : à un instant t, selon la capacité maximum de l’emplacement, plusieurs objets de stocks appartenant à des déposants différents pourront être stockés dans l’emplacement.

 

Gestion article

Gestion article (champ ITMMGT)

Indique le mode de gestion par rapport aux articles :

- Affectation article : l'emplacement est réservé à un couple Produit/Déposant. La saisie du code déposant et du code articele sera obligatoire lors de la création des adresses titulaires de ce type de stockage. L'affectation au déposant est obligatoire si le paramètre "affectation article" est actif.

- Mono article : un seul code article peut exister à un moment donné dans l'emplacement. Un emplacement picing est toujours mono-produit.

- Multi articles : plusieurs codes articles peuvent exister à un moment donné dans l'emplacement.

Caractéristiques

Une seule origine peut exister à un moment donné dans l'emplacement.

Une seule nature de stock peut exister à un moment donné dans l'emplacement.

Un seul numéro de réservation peut exister à un moment donné dans l'emplacement.

Un seul numéro de lot peut exister à un moment donné dans l'emplacement.

Mono origine (champ SNGORI)
Mono nature de stock (champ SNGSKN)
Mono réservation (champ SNGRSA)
Mono lot (champ SNGLOT)

Gestion contenant

La correspondance de contenants pourra être utilisée pour les emplacements ayant ce type de stockage. L’emplacement peut donc contenir des contenants différents du contenant de référence.

Mono contenant (champ SNGCTR)

Un seul type et code contenant peut exister à un moment donné dans l'emplacement. Dès l'occupation de l'emplacement par un objet de stock, l'emplacement sera réservé au type et code contenant de cet objet ; puis lorsque l'emplacement sera à nouveau vide un nouveau contenant (le même ou un autre) pourra utiliser l'emplacement.
Correspondance contenants (champ CTRCSP)
Contrôle de remplissage (champ SLOMES)

Sert à définir la manière de calculer le taux de remplissage d’un emplacement.
- Non : il n’y a pas de règle de calcul, ce qui implique que la place disponible de l’emplacement n’est jamais calculée. Cette valeur est affectée par le sytème pour un type de stockage sans contrôle du taux de remplissage de l'emplacement (emplacement de contenance illimitée par exemple).

- En contenant de référence : la place disponible est calculée par rapport au nombre de contenant maximum de référence de l’emplacement ; ce calcul intègre donc la correspondance des contenants définis pour le magasin, en fonction des contenants déjà présents dans l’emplacement.

Exemple : emplacement multi-contenants de capacité égale à 20 palettes P1. Règle d'équivalence contenants 2 P2 = 3 P1. Entrée de 4 palettes  P1 + 2 palettes P2. Le taux d'occupation de l'emplacement sera égal à 7 /20.

- En nombre de contenant : l’occupation est calculée par rapport au nombre de contenants présents dans l’emplacement, même si ce sont des contenants différents. Cette option peut être utile pour définir des emplacements 'fourre-tout'.

Si on prend le même exemple que précédemment, le taux d'occupation sera de 6 / 20.

- En contenant de référence au sol : ce paramètre est utilisé pour les emplacement dits « de masse » ; le nombre de contenants au sol correspond au nombre de contenants maxi de l’emplacement ; en fonction de l’article entrant, la capacité maximum de l’emplacement est calculée en tenant compte du nombre de contenants au sol, de la hauteur de gerbage et du correctif défini pour le contenant article entrant (notion de palette "à cheval" sur le dernier niveau).

Exemple : capacité au sol 10 ; coeff. de gerbage = 5 (produit non gerbable =1 ) ; correctif de gerbage = 2. La capacité maxi est égale à (10 * 5) - 2 = 48 palettes de ce produit.
Objet homogène (champ HMGPAL)

Sert à définir si les emplacements pourront recevoir des objets non conformes par rapport au plan de palettisation défini pour l’article :
- Non : l’emplacement pourra recevoir des objets de stocks non homogènes, c'est-à-dire non conformes au standard de palettisation établi pour le contenant article considéré. Un palettier classique peut accueillir des palettes rompues ou sur-palettisé.

- Standard homogène : l'emplacement ne pourra accueillir que des palettes conformes au standard de palettisation en vigueur. Cette option ets bien adaptée à la gestion des travées de masse.

- Homogène : l'emplacement ne pourra accueillir que des palettes identiques à celles déjà contenues dans l'emplacement.

Gestion date FIFO

Les produits entrant dans cet emplacement perdent leur date FIFO et sont assurés de sortir les premiers (ex : Picking, Réserve Avancée affect& eacute;e, ...).

Autorisation ou non pour un emplacement de contenir des objets de dates FIFO différentes (par rapport au jour, au mois, à l’année).

Raz date FIFO en entrée (champ RESFIFDAT)
FIFO sur emplacement (champ FIFSLOMGT)

Si la case n’est pas cochée, le système ne tient pas compte de la date FIFO de l’emplacement. Cette valeur est prise par défaut lorsqu'il s'agit d'un emplacement de type «Picking». En effet, un emplacement peut abriter à un instant ‘t’ un certain nombre de dates FIFO différentes (en fonction des réapprovisionnements notamment). Dans ce cas, chaque objet entré à l'emplacement conserve sa propre date FIFO.

Si la case est cochée, le système prend en compte la date FIFO de l’emplacement (au lieu de celle de l’objet de stock) dans l’algorithme de recherche en sortie. La date FIFO de l’emplacement est égale à la date du premier objet entrant dans l’emplacement. Tous les objets de stock prennent alors la date FIFO de l’emplacement.
Entrée date FIFO antérieure autorisée (champ OLDDATINP)

Cette règle correspond à l'autorisation de remplissage d'un emplacement par des produits de date FIFO plus ancienne.

Si la case n’est pas cochée, toute entrée à l'emplacement d'un produit ayant une date FIFO antérieure à la date FIFO du plus vieil objet de stock de l'emplacement est interdite.

Si la case est cochée, l'opérateur pourra, s'il le désire, faire entrer un objet de date FIFO dépassée dans un emplacement n'étant pas censé pouvoir l'accueillir. Ce paramètre permet notamment de libérer des emplacements de forte capacité, n'étant occupés que par un ou deux objets de date ancienne.

Contrôle écart FIFO en entrée (champ FIFDEV)

Si la case n’est pas cochée, aucun contrôle n'est effectué sur la date FIFO pour permettre l'entrée dans cet emplacement.
Si la case est cochée, un contrôle portant sur la date FIFO de l'article entré est effectué pour permettre le stockage dans cet emplacement. Les contrôles sur la date FIFO s'effectuent en fonction de la durée définie au niveau de chaque article (paramètre « Ecart FIFO maxi » dans Fiche Article Annexe). Ils considèrent : la date FIFO de l'entrée (Date Entrée) et la date FIFO du plus vieil objet de l'emplacement (Date Emplacement).
Exemple : soit (date FIFO de l'objet entrant) > (date du plus vieil objet de l'emplacement + durée de l'écart FIFO maximum) => l'entrée dans l'emplacement est interdite.
Mono date (champ SNGDAT)

Ordre déstockage

Pour des objets ayant tous les autres critères égaux, cela définit quel sera le premier objet qui sera sorti : le premier objet entré ( FIFO) ou le dernier objet entré (LIFO).

Déstockage (champ TOOMGT)

Gestion du Swap

Swap du lot (champ SWAPLOT)

Indique si ce type de stockage autorise de faire du swap de lot dans les emplacements de ce type.

Split mouvement auto (champ SPLITMVT)

Indique si ce type de stockage autorise de splitter les mouvements dans le cadre de l'utilisation de la fonctionnalité "Swap de lot".


Exemple d'utilisation: lors du prélèvement partiel en RF d'un mouvement de sortie, si l'opérateur prélève un autre lot que celui attendu, le mouvement initial sera splitté en 2 mouvements...
1 mvt de sortie de 10 uc sur le lot A
L'opérateur prélève 4 uc sur le lot B (swap de lot)
on a alors 2 mouvements
1 mvt de sortie de 4 uc sur le lot B
1 mvt de sortie de 6 uc sur le lot A (en attente de prélèvement)
Remarque: si on autorise pas de splitter le mouvement, alors l'opérateur ne pourra pas prélever partiellement sur le lot B, Il devra nécessairement prélever la totalité (10 uc) sur le lot B.

Regroupement RF

Objet de stock (champ GRUMVTSKO)

permet d'indiquer que l'objet de stock est un critère de regroupement des mouvements en prélèvement RF.

Date FIFO (champ GRUMVTFIF)

permet d'indiquer que la date FIFO est un critère de regroupement des mouvements en prélèvement RF.
