Consultation

Cette fonction permet de consulter toutes les factures existantes sur le site/déposant courant.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Client facturé
  • Onglet Lignes

L'écran est constitué de:

  • une liste gauche contenant les factures existantes sur le site/déposant courant.
  • Une partie droite contenant:
        • L'entête de la facture précisant ses principales caractéristiques (dossier de facturation, période, montant HT...)
        • 1 onglet "Client facturé" donnant l'adresse de facturation (c-a-d celle du déposant)
        • 1 onglet "Lignes" donnant le détail de la facture (1 ligne par rubrique de facturation)

    En-tête

    Au niveau de l'entête de la facture, on retrouve les caractéristiques principales.

    Descriptions des champs
    Facture (champ INVNUM)

    Numéro de la pré-facture.

    Date (champ IVHVALDAT)

    Ce champ donne la date de validation de la pré-facture.

    Etat (champ IVHSTA)

      C'est l'état d'une pré-facture. 3 valeurs sont possibles:
    • '7-Attente validation': la pré-facture a été créée et en attente de validation.
    • '8-Validée': la pré-facture a été validée. c'est un état final.
    • '9-Annulée': la pré-facture a été annulée. c'est aussi un état final.
    Date début (champ IVHSTRDAT)

    Ce champ indique la date de début de la période de calcul de la pré-facture.

    Date fin (champ IVHENDDAT)

    Ce champ indique la date de fin de la période de calcul de la pré-facture.

    Code dossier (champ IFHCOD)

    Code du dossier de facturation définit dans la table des dossiers de facturation GEODE. C'est l'entité pour laquelle la facturation logistique sera calculée. Ce dossier est rattaché à un Site/Déposant.

    champ IFHDESAXX
    Montant HT (champ IVHAMT)

    Indique le montant total HT de la pré-facture.

    champ IFHCUR

    Ce champ indique la devise de facturation utilisée pour les pré-factures d'un dossier de facturation.

    Transmis (champ TRM)

    Ce champ indique si l'entité concernée a été transmise par interface.

    Onglet Client facturé

    Cet onglet donne les coordonnées du client à facturer. Pour information c'est l'adresse du déposant du dossier de facturation.

    Descriptions des champs

    Bloc numéro 1

    Cet intitulé, qui permet de donner une désignation à la fiche, est utilisé dans certains écrans ou états lorsqu'il n'y a pas assez de place pour visualiser l'intitulé long.

    Nom du pays. Si le code pays est renseigné, alors ce champ est renseigné automatiquement par rapport à la table des pays. Si le code pays n’e st pas renseigné, alors ce champ peut être renseigné librement par l’utilisateur.

    Dénomination de certaines sociétés, comportant le nom de tout ou une partie des associés suivi de « & Cie ».

    Déposant (champ DEP)
    Intitulé court (champ DESSHO)
    Pays (champ CRY)

    Code permettant d'identifier le pays. Ce code est contrôlé dans la table des pays. Le nom correspondant au code pays est alors affiché automatiquement. Si le code pays n’est pas renseigné par l’utilisateur, alors le nom du pays pourra être renseigné librement.
    champ CRYNAM
    Raison sociale (champ NAM)
    champ ADR

    Indication précise du domicile.
    Code postal (champ POS)

    ode postal dont le format de saisie est défini dans la table Pays.
    Le code postal permet de déterminer la ville. Quand le code postal est modifié, la ville correspondant au nouveau code est automatiquement affichée.
    Cas particuliers :

    • Pour Andorre, indiquez 99130
    • Pour Monaco, indiquez 99138
    • Pour les autres pays étrangers, indiquez le code INSEE du pays ou 99999 s’il est inconnu.
    Ville (champ CTY)

    Nom de la ville. La ville proposée automatiquement est celle associée au code postal indiqué précédemment.
    Il est possible également de saisir directement le nom d'une ville :

        si celle-ci est définie, le code postal associé est automatiquement affiché

        si celle-ci n’existe pas, un message d'avertissement apparaît et la poursuite de la saisie est autorisée.

    Etat (province) (champ SAT)

    Ce champ contient la subdivision géographique entrant dans la définition de l'adresse postale.
    Par exemple, il peut s'agir du Département pour la France, de l'Etat pour les Etats-Unis. L'intitulé de la subdivision ainsi que le contrôle ou non sur la saisie de cette information sont définis au niveau du paramétrage du Pays.
    S'il est contrôlé, ce code est dépendant de la subdivision géographique correspondante.

    Cette subdivision est alimentée automatiquement après saisie du code postal et de la ville, à partir du paramétrage effectué au niveau des Codes postaux.
    Téléphone (champ TEL)

    Numéro de téléphone dont le format de saisie dépend du pays.
    Fax (champ FAX)

    Numéro de télécopie dont le format de saisie est identique à celui du téléphone et dépend du pays.
    E-mail (champ ADDEML)

    Pas d'aide liée à ce champ.

    Bloc numéro 2

    Numéro d'identification nationale délivré par l'INSEE. Le format de ce numéro SIRET dépend du code pays.

    C'est le montant (essentiellement informatif) du capital social de la société.

    Registre du commerce et des sociétés.

    SIRET (champ CRN)
    Capital social (champ RGC)
    RCS (champ RCS)

    Onglet Lignes

    Cet onglet donne le détail de la facture. Chaque ligne correspond à une rubrique de facturation. De plus les lignes sont triées dans l'ordre défini au niveau des prestations et du dossier de facturation.

    Descriptions des champs

    Tableau

    Rubrique (champ IHHCOD)

    Code de la rubrique définit dans la table des rubriques GEODE.

    La rubrique est un poste de facturation d'une prestation tel que "le nombre de BL expédiés", "le nombre de colis expédiés", "le poids de palettes réceptionnées" ...
    Intitulé (champ IHHDESAXX)
    Prestation (champ ISHCOD)

    Code prestation définit dans la table des prestations GEODE.

    La prestation permet de regrouper des éléments de facturation (Rubriques) relative à un domaine donné comme par exemple la réception, le stockage, l'expédition ...
    Code tarif (champ IPCCOD)

    Code tarif utilisé pour calculer le montant de la rubrique dans la fonctionnalité GEOPLO (Module de pré-facturation des prestations logistiques).

    Tarif (champ IPHCOD)

    Indique le nom d'un tarif.

    Pour info, un tarif défini une tarification d'un code tarif sur une période de validité donnée.
    Type (champ IVLTYP)

    Indique le type de la ligne de facturation:

    • 'Forfaitaire': ligne liée à une rubrique de type forfaitaire.
    • 'Proportionnel': ligne liée à une rubrique de type table ou statistique.
    Montant forfaitaire (champ IVLFIXAMT)

    Indique le montant forfaitaire de la rubrique.

    Valeur (champ IVLUNINBR)

    Indique la valeur calculée pour la ligne rubrique. 

    Unité (champ WUOM)
    Prix unitaire (champ IVLUNIPRI)

    Indique le prix unitaire applicable sur cette ligne rubrique.

    Prix calculé (champ IVLPRIAMT)

    Indique le prix calculée pour la tranche tarifaire relative à la valeur calculée pour la ligne rubrique.

    (Prix calculée = Valeur rubrique * prix unitaire de la tranche tarifaire retenue)
    Montant HT (champ IVLAMT)

    Indique le montant total HT d'une ligne rubrique.

    (Montant HT rubrique = montant forfaitaire + prix calculé pour la tranche)
    Icône Actions
    Visu texte

    permet de visualiser les données extraites ayant servies au calcul de la rubrique.

    Pour info, ces données restent toujours liées à la ligne de facturation (même si la facture est historisée). Cependant elles sont supprimées si la facture est annulée.

    Etats

    Par défaut, les états suivants sont associés à la fonction :

      INVOICE : Préfacture logistique

    Mais ceci peut être modifié par paramétrage.

    Messages d'erreur

    • Messages génériques

    Il n'y a pas de message d'erreur autre que les messages d'erreur génériques.
