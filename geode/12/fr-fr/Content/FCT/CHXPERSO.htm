X3 Warehousing

Personnalisation

Cette fonction permet à l’utilisateur de changer certains paramètres qui concernent ses préférences ou des valeurs par défaut. Seuls les paramètres dont la modification est autorisée à l'utilisateur lui-même apparaissent dans cet écran, contrairement à la fiche utilisateur, où on peut voir potentiellement tous les paramètres utilisateurs.

En-tête

Le code de l'utilisateur courant est affiché en tête, et on retrouve ensuite les informations modifiables sur deux onglets.

Descriptions des champs
Utilisateur (champ USR)

Correspond au code utilisateur de connexion.

Onglet Options

Cet écran permet de définir à la fois un ensemble de paramètres utilisateurs modifiables par l'utilisateur lui-même, et les destinations proposées par défaut pour les éditions pour lesquelles aucune imprimante par défaut n’existe, en fonction du type de destination demandée.

Descriptions des champs

Tableau Paramètres

Chapitre (champ DESCHP)

Ce champ uniquement affiché définit le chapitre dans lequel se trouve le paramètre concerné.
Groupe (champ GRPPAR)

Ce champ uniquement affiché définit le groupe dans lequel se trouve le paramètre concerné.
Paramètre (champ PARAM)

Ce code identifie le paramètre à renseigner.
Intitulé (champ NAME)

Intitulé associé au code précédent
Valeur (champ VALEUR)

On saisit ici la valeur du paramètre. Celle-ci est saisie selon les cas comme un nombre, une date, ou un code alphanumérique. Lorsque le paramètre est de type menu local, on peut au choix saisir le début de l'intitulé, ou la valeur numérique correspondant au rang de l'intitulé dans la liste. Lorsque le code est contrôlé par une table annexe, une touche de sélection est utilisable pour le choisir.
Niveau (champ NIVEAU)

Ce champ indique le niveau de définition sur lequel le paramètre est renseigné. Les valeurs possibles sont définies par le menu local 987. Quand le niveau est indiqué en regard d'un groupe de paramètres, c'est le niveau le plus fin rencontré sur les paramètres du groupe qui est affiché.

Tableau Destinations par défaut

Type (champ TYPIMP)

Ce champ affiché définit le type de destination à partir des valeurs du menu local paramétrable numéro 22.
Destination (champ PRTDEF)

Ce code identifie, pour chaque type d'imprimante défini par le menu local paramétrable 22, une destination d'impression qui sera proposée par défaut au lancement d'un état en l'absence de règle plus précise.

Divers

Barre de titre (champ TIT)

Ce champ permet de définir, par l'intermédiaire d'une expression calculée, le contenu de la barre de titre de la fenêtre du progiciel. Par défaut, on retrouve un titre qui est le suivant :

DOSSIER         NOM DU PROGICIEL     -  [Intitulé de la fonction courante]

Les champs DOSSIER et NOM DU PROGICIEL sont remplacés par le résultat de l'évaluation de la formule, si elle existe. On peut notamment faire apparaître dans cette formule des constantes, mais aussi les variables globales et les fonctions suivantes :

nomap

Nom du dossier courant

GUSER

Code de l'utilisateur courant

GNOMUSER

Nom de l'utilisateur courant

date$

date courante

time$

heure courante (à la connexion)

Il est important de noter que ce titre n'est évaluée qu'une fois à l'entrée du progiciel.
Icône Actions
Aide

Cette option permet d'afficher l'aide relative au paramètre de la ligne courante.

Onglet Menu

Cet onglet, qui n’est pas accessible via la gestion des utilisateurs, permet de définir des raccourcis vers des fonctions via la frappe d’une touche (les touches    à     sont réservées à cet usage, mais il est aussi possible d’atteindre les fonctions par le menu Outils situé dans la barre supérieure de menus). Toute fonction peut ainsi potentiellement être appelée, à partir du moment où son code est saisi dans le tableau (les touches étant attribuées dans l’ordre du tableau). Le lancement d’une fonction de cette façon se fait en ouvrant une deuxième session.

Outre le nom de la fonction, on saisit le code de la transaction (ou de l’écran, ou du paramètre complémentaire), si la fonction admet un tel paramètre.

Il est à noter que ces fonctions n’apparaissent au menu que lorsqu’on est déjà dans une fonction. En outre, le rafraîchissement du menu ne se fait qu’à la connexion : il est donc nécessaire de se reconnecter après modification de ces choix.
