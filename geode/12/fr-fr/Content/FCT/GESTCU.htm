Devises

Cette table permet de définir les différentes devises utilisées dans le progiciel pour caractériser les montants monétaires saisis. Elle permet uniquement de définir leurs caractéristiques principales. La parité entre devises est définie dans la tables des Cours.

Onglet Généralités

Les informations générales relatives à la devise sont présentées dans cette section.

Descriptions des champs

Code

Les deux premières lettres du code à trois lettres ISO 4217 sont les mêmes que le code pays et, si possible, la troisième lettre correspond à la première lettre du no m de la devise en anglais. Exemples :

  • USD pour "United States" et "Dollar",
  • GBP pour "Great Britain" et "Pound" (livre),
  • JPY pour "Japon" et "Yen",
  • EUR pour "Euro" (qui est une excep tion à la règle ci-dessus).

Il est recommandé d'utiliser cette codification normalisée par l'ISO comme code de la devise.

Ce code est le symbole usuel d'une devise, apparaissant devant ou derrière un montant sur un document.

Code numérique à 3 chiffres représentant la devise. Ce code numérique, associé à une devise selon la norme ISO 4217, est généralement le même que le code numérique as socié au pays correspondant par la norme ISO 3166-1.

Par exemple, 840 est le code numérique ISO pour la devise USD ("United States Dollar"), mais aussi le code numérique pour le pays US ("United States)".

Code ISO (champ ISOCOD)
Symbole (champ CURSYM)
Code ISO numerique (champ ISONUMCOD)

Format

Ce champ concerne le format utilisé pour un montant normal (montant TTC de facture par exemple). Ce format est variable dans la limite de 20 caractères significatifs. Ce format inclue les éventuels espaces délimiteurs.

Exemple :

  • Format maximum limité à 13.2 chiffres significatifs : 13 entiers, 2 décimales et 5 espaces délimiteurs (#,###,###,###,###.##).
  • Format maximum limité à 15 chiffres significatifs : 15 entiers, 0 décimales et 4 espaces délimiteurs (###,###,###,###.###).
  • Format maximum limité à 12.3 chiffres significatifs : 12 entiers, 3 décimales et 4 espaces délimiteurs (###,###,###,###.###).

Ce champ concerne le format utilisé pour un montant court (prix unitaire, remise par exemple). Ce format est variable dans la limite de 10 caractères significatifs. C e format inclue les éventuels espaces délimiteurs.

Exemple :

  • Format maximum limité à 6.2 chiffres significatifs : 6 entiers, 2 décimales et 2 espaces délimiteurs (###,###.##).
  • Format maximum limité à 8 chiffres significatifs : 8 entiers, 0 décimale et 2 espaces délimiteurs (##,###,###).
  • Format maximum limité à 5.3 chiffres significatifs : 5 entiers, 3 décimales et 2 espaces délimiteurs (##,###.###).

Le nombre de décimales effectives des prix unitaires est déterminé par la variable GDECPRI.

Ce champ concerne le format utilisé pour un montant cumulé (totaux de balances ou de statistiques, par exemple). Ce format est variable dans la limite de 20 chiffres significatifs. Ce format inclue les éventuels espaces délimiteurs.

Exemple :

  • Format maximum limité à 17.2 chiffres significatifs : 17 entiers, 2 décimales et 1 espace délimiteur (#################,##).
  • Format maximum limité à 20 chiffres significatifs : 20 entiers, 0 décimale et 0 espace délimiteur (####################).
  • Format maximum limité à 16.3 chiffres significatifs : 16 entiers, 3 décimales et 1 espace délimiteur (################,###).

Ce champ indique les décimales significatives du montant, exprimé dans la devise.

Ce champ est l'arrondi associé à cette devise. Tout montant exprimé dans cette devise sera arrondi à cette précision.

Conversion

Sélectionnez cette case à cocher pour indiquer que la devise fait partie de la zone Euro. Dans ce cas, vous devez saisir la contre-valeur de cette devise en euros, da ns le champ Cours euro.

Ce champ est accessible uniquement si la devise fait partie de la zone Euro (la case à cocher Euro est sélectionnée). Dans ce cas, saisissez la contre-valeur d e cette devise, en Euros. Si la devise ne fait pas partie de la zone Euro, utilisez la table des cours de façon classique.

Dans le cas particulier de la devise Euro, le cours doit oblig atoirement être renseigné avec la valeur 1.
