Priorités transporteur

La table de priorités transporteurs permet de déterminer les critères d’affectation d’un couple (transporteur, mode de transport) aux Ordres de Livraison par la fonction Calcul Transporteur (PINCRR).

La table est consultée par rapport au N° de séquence et l’algorithme s’arrête dès lors qu’il a trouvé un enregistrement ; c’est pourquoi, il est important de saisir les enregistrements du plus restrictif au plus large.

Dans tous les cas, il est recommandé de déterminer un couple (transporteur, lot) le plus « permissif » possible

La saisie est réalisée à partir d’un couple (transporteur, mode de transport) à partir de 2 onglets.

Celle ci doit se faire dans l’ordre de priorité des transporteurs.

Il est conseillé de créer un enregistrement sans restriction.

En-tête

l'entête precise le trio (N° séquence, code transporteur, mode de transport) pour lequel les priorités transporteurs sont rattachés.

Descriptions des champs
Séquence (champ ORN)

Numéro permettant d'utiliser les règles définies dans cet écran par

ordre de priorité.
Intitulé (champ CRODESAXX)
Transporteur (champ CRR)

Code identifiant le transporteur. Ce code doit être défini dans la table des transporteurs.
Mode transport (champ CRRMOD)

Généralement, le mode de transport correspond aux notions de messagerie, affrètement, express, ... Le mode transport est vérifié par rapport à la table des modes de transport.

Onglet Général

Pays : facultatif, si renseigné la recherche est réalisé en tenant compte de ce critère ; si blanc, le transporteur peut être sélectionné quel que soit le pays de l’O.L..

Code postal/Ville : facultatif, si renseigné la recherche est réalisé en tenant compte de ce critère ; si blanc, le transporteur peut être sélectionné quel que soit le code postal/ville de l’O.L.

Zone de livraison : facultatif, si renseigné la recherche est réalisée en tenant compte de ce critère ; si blanc, le transporteur peut être sélectionné quel que soit la valeur de la zone de livraison.

Poids : facultatif, si renseigné, le poids de l’O.L doit être inférieur ou égal au poids mentionné pour le couple (transporteur/mode).

Volume : facultatif, si renseigné, le volume de l’O.L doit être inférieur ou égal au volume mentionné pour le couple (transporteur/mode).

Longueur d’un article : facultatif, si renseigné, la plus importante longueur des lignes OL doit être inférieure ou égal au volume mentionné pour le couple (transporteur/mode).

Onglet Restrictions

 Familles: Liste des codes familles interdites pour le couple (transporteur/mode).

Risques: liste des types de risques interdits pour le couple (transporteur/mode).

Descriptions des champs

Tableau Familles interdites

champ FAMNBR

Pas d'aide liée à ce champ.

Famille (champ FBDFAM)

Si renseigné, ce transporteur ne sera pas choisi pour un OL comportant un article de même famille.

Utiliser dans le cadre de l'affectation automatique d'un transporteur ou tournée.

Risque (champ FBDDGG)

Utilisé dans le cadre de l'affectation automatique d'un transporteur ou d'une tournée.
