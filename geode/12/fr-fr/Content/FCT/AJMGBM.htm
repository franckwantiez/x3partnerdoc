Ajustements en masse

Cet écran permet de réaliser des ajustements de stock qualitatif en réserve en sélectionnant au préalable plusieurs objets de stocks par la saisie de critères.
L’ajustement de stock ne permet pas de déplacer un objet de stock mais de le modifier en le laissant dans son emplacement ; pour cela l’objet ne doit ni être en cours d’entrée, ni être en cours de sortie.

Il est également possible de réaliser un ajustement qualitatif lors du transfert de stocks c’est à dire avec déplacement de l’objet de stock.

Dès sa création, le mouvement est validé et il n’est pas possible d’avoir une phase « d’encours » sur ce type de mouvement. De ce fait, un mouvement d’ajustement ne peut être ni soldé ni supprimé.

Cette fonction ne permet de réaliser que des ajustements de type Qualitatif.

Selon le code mouvement utilisé, pour réaliser ces ajustements et selon le paramétrage des interfaces, un compte rendu de mouvement peut être généré pour le niveau amont.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Objets de stock
  • Onglet Ajustement

Les écrans des ajustements en masse sont affichés suite à une saisie de critères affichant les objets de stock sélectionnés.

Viennent ensuite les phases suivantes:

- Sélection des objets de stocks

- Saisie des caractéristiques à modifier

- Lancement du traitement d’ajustement

- Affichage des anomalies s’il y a lieu.

En-tête

La saisie du code mouvement est obligatoire et il doit être de type Ajustement qualitatif.

La date et heure système est affichée par défaut.

Descriptions des champs
Code mvt (champ MVT)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
Date (champ DAT)

Date de création de la transaction.
Heure (champ TIM)

Heure de création de la transaction.

Onglet Objets de stock

Le tableau affiche les objets de stocks unitaires du site. L’icône 4 dans la barre d’outil permet de paginer pour visualiser l’ensemble des objets.

Seuls les objets de stocks sans en cours et localisés dans des emplacements de type « réserve » sont affichés.

Les objets de stock peuvent être sélectionnés un à un ou en masse par Clic Droit Souris Tout Cocher.

Onglet Ajustement

Saisir la caractéristique à modifier en sélectionnant la ligne dans la  colonne MaJ. 

Saisir ensuite la valeur à initialiser ou à remplacer dans la colonne Valeur.
Si aucune valeur n’est saisie, le traitement remettra à blanc la zone en  question.
A l’enregistrement de la transaction, l’affichage des anomalies permet  d’afficher les résultats du traitement.
Un compte rendu d’ajustement est affiché selon les erreurs rencontrées  durant le traitement.

Autres conditions

Selon le paramétrage du site, un code opérateur peut être demandé. Celui ci doit être alors autorisé à saisir des ajustements.

Barre de menu

  • Critères
  • Commentaire

Critères

Le bouton critère permet d’accéder à l’écran de sélection des objets de stocks. 3 familles de critères peuvent être utilisées :

-          critères emplacement

-          critères article

-          critères objets de stock

La saisie peut être combinée sur les  types de critères.

Commentaire

Saisie possible d’un commentaire explicatif sur l’ajustement.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Code mouvement interdit

Le code mouvement saisi n’est pas un code de type ajustement qualitatif.

Aucune modification à traiter.

Dans le cadre d’un ajustement qualitatif, aucune donnée n’a été modifiée par rapport à celle de l’objet de stock.

Magasin : fiche inexistante

Le magasin saisi n’existe pas.

Format incorrect

Le format de l’adresse saisi ne correspond pas à celle du magasin.

Emplacement : fiche inexistante

L’emplacement saisi n’existe pas dans le magasin sélectionné.

Article: fiche inexistante

Le code article saisi n’existe pas.

Pas de fiche sélectionné

Les critères saisis emplacement/article ne permettent pas de sélectionner un objet de stock.

Code mouvement : fiche inexistante

Le code mouvement saisi n’existe pas pour ce déposant.

N° de lot : zone obligatoire

L’article doit être géré avec un N° de lot, qui n’a pas été saisi.
