Ergonomie des traitements X3

Introduction

Les progiciels X3 sont organisés en fonctions, chaque fonction correspondant à une entrée de menu. Ces fonctions sont normalisées dans leur ergonomie, et obéissent à différents modèles. Le traitement est l'un de ces modèles.

Cette documentation décrit l'ergonomie liée aux traitements, que l'on utilise le poste client Windows™ ou un navigateur Web. Les deux ergonomies étant très proches, il n'existe aucune différence de comportement dans l'interface dans ce cas.

Cette ergonomie se caractérise par :

  • la saisie de critères de lancement dans une fenêtre préliminaire.
  • le lancement, par un bouton nommé selon les cas OK ou Exécution.
  • l'affichage optionnel (en fin de traitement, qui peut être long) d'une trace des opérations réalisées.

Ce type de fonction est le plus fréquemment soumis en batch (ce d'autant plus qu'en Web, un délai maximum d'exécution d'une heure est accordé pour des tâches interactives longues). Un tel lancement est réalisé par le biais d'une tâche prédéfinie.

Les différents éléments de cette gestion sont décrits dans le document qui suit.

Entrée dans un traitement

La saisie des paramètres est la première opération à réaliser.

Le plus souvent, lorsque le nombre de paramètres à saisir est important, il est possible de mémoriser les critères saisis (bouton Mémo) en leur donnant un code, de rappeler un mémo existant (bouton Rappel), ou d'effacer le mémo en question (bouton Eff. Memo). S'il existe un mémo nommé STD, il est automatiquement chargé au lancement de la fonction.

Une fois les paramètres saisis, la validation de l'écran entraîne le lancement de la fonction. Si la fonction est lancée via la soumission d'une requête, cette saisie est identique, mais le lancement provoque l'affichage du message Requête soumise au serveur.

Dans le cas où le lancement est réalisé en direct, une boîte affiche un message de type Exécution en cours, ce jusqu'à ce que l'exécution soit effectivement terminée.

Affichage d'une trace

Cet affichage se fait exactement comme décrit dans la fonction de gestion des traces (sauf que l'on ne choisit pas le fichier de trace à afficher). Le fait de quitter cette trace permet de terminer la fonction.
