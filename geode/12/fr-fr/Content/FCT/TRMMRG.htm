Compactage sur stock

Cette fonction permet de lancer de façon interactive ou en batch la fonctionnalité de compactage sur stock. Une option de simulation est également possible.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Les informations sur les objets à compacter d’origine et de destination sont à saisir dans une boîte de dialogue.

Ecran de saisie

Code mouvement : saisie obligatoire du code mouvement de type Transfert. Les transferts générés par cette fonction sont des transferts de type Compactage avec Cumul.

Nb maxi mvts/objets : exprime le nombre de mouvements à réaliser par objet de stock. Dans la fonction de compactage en entrée, la valeur est fixée à 2 c’est à dire qu’un objet à compacter ne peut pas être compacté sur plus de 2 objets rompus.

Nb mxi mvts : exprime le nombre de mouvements total de transfert à réaliser par lancement du traitement.

Simulation : case à cocher, en simulation un tableau récapitulatif des transferts réalisables est affiché

Déposant : critère de limitation des objets à compacter

Articles : critère de limitation des objets à compacter

Objets rompus seulement : si coché, le traitement prend en compte uniquement les objets considérés comme rompus. Un objet rompu est un objet dont la quantité UC est inférieure au produit % de rompu et quantité UC du contenant article de l’objet de stock. Si non coché, tous les objets de stocks non standards sont pris en compte.

% d’occupation prioritaire par rapport au % de rompu : si coché, le traitement favorise les emplacements les moins occupés pour sélectionner les objets de stocks à compacter au détriment de la quantité de l’objet de stock. Si non coché, les objets les plus rompus ou ceux dont la quantité UC est la plus petite sont favorisés par rapport au taux d’occupation des emplacements.

Origine

Magasin : critère géographique de prise en compte des objets d’origine à compacter.

Emplacements : critère de limitation des adresses des objets d’origine à compacter.

Y compris magasins interdits en sortie : prise en compte ou pas des magasins externes, en général ce paramètre doit être non coché.

Remarque : la notion d’origine est ici liée à l’objet de stock que l’on va déplacer pour être compacté avec d’autres (dits de destination).

Destination

Magasin : critère géographique de prise en compte des objets de destination à compacter

Emplacements : critère de limitation des adresses des objets de destination à compacter.

Y compris magasins interdits en sortie : prise en compte ou pas des magasins externes, en général ce paramètre doit être non coché

Sélection avancée Origine/Destination

Fenêtre de saisie de différentes formules selon les besoins.

L’action du bouton  permet de lancer le traitement interactif soit en simulation, soit en création.

Selon le paramétrage des listes et étiquettes peuvent être générées.
Selon le paramétrage, les mouvements associés peuvent être validés automatiquement, dans ce cas un code opérateur peut alors être demandé. 

Descriptions des champs

Transfert

Valeur (modifiable) qui indique le nombre maximum de mouvements de transferts que le système peut créer pour un transfert.
Exemple : si les objets de stocks sont physiquement des palettes, le nombre de mouvements pourra être égal au nombre de palettes maximum supporté par le moyen de manutention ou de transport pour aller d’un magasin à l’autre.

Permet d’afficher à la fin du traitement, la trace (déroulement) du programme.

Code mvt (champ MVTCOD)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
Nb maxi mvts / objet (champ MAXMRG)

Indique le nombre de déplacement à réaliser par objets de stocks pour constituer un objet complet.
Nb maxi mvts (champ MAXTRF)
Simulation (champ SIM)

Permet de passer d’un mode réel (par défaut) au mode de simulation où toutes les données sont supprimées à l’issue du traitement.
Trace détaillée (champ TRA)

Sélection

Déposant (champ DEP)

Code qui représente le déposant. En terme logistique, la notion de déposant permet de traiter des flux ou trafics distincts dans le même site. Elle recouvre la notion de client donneur d'ordres pour les prestataires de services.

Après le site, le déposant est la deuxième clef d'accès aux fichiers Article, Entrée et Ordre de Livraison. Pour l'emplacement, le code déposant est renseigné si l'emplacement est affecté au déposant ou à l'article, et si l'emplacement est mono déposant ou mono article et qu'il n'est pas vide.
Article début (champ ITMSTR)

Code article défini dans la table des articles GEODE.
Article fin (champ ITMEND)
Objets rompus seulement (champ IPA)

Indique si le compactage doit prendre en compte seulement les objets considérés comme rompus ou bien tous les objets de stocks.

% occupation prioritaire devant % rompu (champ OCPRAT)

Indique s'il faut prioriser les emplacements les moins occupés ou les objets de stocks les plus rompus.

Origine

Champ qui sert à spécifier l’adresse de début de la borne d’adresses emplacements.

Champ qui sert à spécifier l’adresse de fin de la borne d’adresses emplacements.

Magasin (champ PIKSTO)

Code permettant d’identifier le magasin. Il est vérifié dans la table des magasins.
Emplacement début (champ PIKSTR)
Emplacement fin (champ PIKEND)
Y compris magasins interdits en sortie (champ PIKOUTFBD)

Si coché la recherche d'objet à compacter tient compte des magasins interdits en sortie.

Si non coché, seuls les magasins autorisés en sortie sont pris en compte.

Destination

Champ qui sert à spécifier l’adresse de début de la borne d’adresses emplacements.

Champ qui sert à spécifier l’adresse de fin de la borne d’adresses emplacements.

Magasin (champ DPSSTO)

Code permettant d’identifier le magasin. Il est vérifié dans la table des magasins.
Emplacement début (champ DPSSTR)
Emplacement fin (champ DPSEND)
Y compris magasins interdits en sortie (champ DPSOUTFBD)

Si coché la recherche d'objet à compacter tient compte des magasins interdits en sortie.

Si non coché, seuls les magasins autorisés en sortie sont pris en compte.

Sélection avancée

champ PIKSKO

Fenêtre de saisie d'une formule pour sélectionner les objets de stocks "d'origine" à compacter.

Sélection avancée

champ DPSSKO

Fenêtre de saisie d'une formule pour sélectionner les objets de stocks "de destination" à compacter.

Tâche batch

Cette fonction peut être lancée en batch, mais il n'existe pas de tâche standard dédiée à son lancement.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Code mvt : Zone obligatoire

La saisie d’un code mouvement est obligatoire

Code mvt : Fiche inexistante

Le code mouvement saisi n’existe pas pour le site courant.

Code mvt : Code mvt interdit

Le code mouvement saisi n’est pas de type transfert.

Nb maxi mvts / objet : Zone obligatoire

Le nombre maxi de mouvements par objet doit être différent de 0.

Nb maxi mvts : Zone obligatoire

Le nombre maxi de mouvements doit être différent de 0.

Nb maxi mvts : Valeur incorrecte (<X)

Le nombre maxi de mouvements doit être supérieur ou égal au nombre maxi de mouvements / objet.

Déposant : Fiche inexistante

Le déposant saisi n’existe pas pour le site courant.

Articles : Fiche inexistante

L’article saisi n’existe pas pour le site courant et le déposant saisi.

Magasin : Fiche inexistante

Le magasin saisi n’existe pas pour le site courant.

Magasin : Type de magasin <> Principal

Le magasin saisi doit être de type principal.

Emplacements : Format incorrect

Le format de l’emplacement saisi est différent de celui du magasin

Emplacements : Fiche inexistante

L’emplacement saisi n’existe pas pour le magasin

Emplacements : Emplacement de picking

L’emplacement saisi doit être de type réserve

Emplacements : valeur incorrecte

L’emplacement de fin est < à l’emplacement de début
