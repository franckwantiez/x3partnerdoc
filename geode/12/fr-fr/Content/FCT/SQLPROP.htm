Propriétés Sql Server

Cette fonction permet de visualiser les principales propriétés de la base de données lorsque la base de données utilisée est SQL Server™.

Les données ici présentées ne sont pas modifiables.
Leur modification doit se faire par les outils d'administration propres à la base de données utilisée.

Avertissement

Cette fonction permet, par des requêtes sur des tables et vues du système, de donner des informations techniques sur la base de données. L'interprétation des résultats donnés par cette fonction est du ressort d'un administrateur de bases de données, et peut dépendre de la version de base de données utilisée.

Gestion de l'écran

  • En-tête
  • Onglet Serveur
  • Onglet Base

Cette fonction permet de visualiser les propriétés de la base de données.
Ces éléments sont présentés sur deux onglets : le premier donne les caractéristiques générales du serveur, le deuxième donne caractéristiques de la base de données ainsi que la liste des paramètres et options positionnés pour la base de données.

En-tête

Permet de connaître les caractéristiques de la base de données utilisée.

Descriptions des champs
Base de données (champ BDDNAM)

Nom de la base de données
Version (champ VER)

Numéro de version de la base de données
champ VERDES

Description de la version de la base de données

Onglet Serveur

Le premier onglet présente les propriétés générales du serveur hébergeant la base de données.

Ce premier bloc donne des informations sur l'instance de Sql Server. 

Descriptions des champs
Machine (champ SRVMAC)

Nom du serveur sur lequel est démarrée la base de données.
Instance (champ SRVINS)

Nom de l'instance de Microsoft® SQL Server™.
Classement par défaut (champ SRVTRI)

Type de classement utilisé par l'instance de SQL Server.
Version (champ VER)

Description de la version de la base de données

Onglet Base

Sur le deuxième onglet, on voit les propriétés de la base de données

Cet écran est constitué de trois blocs.

Base de données

Ce bloc contient les principales caractéristiques de la base de données. 

Dernière sauvegarde

Ce bloc permet de visualiser les dates des dernières sauvegardes effectuées sur la base de données.

Options

Ce bloc liste les options de base de données et leurs valeurs.

Il existe cinq catégories d'options de base de données :

    • options automatiques ;
    • options de curseur ;
    • options de récupération ;
    • options SQL ;
    • options d'état.

Descriptions des champs

Base de données

Propriétaire (champ OWN)

Utilisateur propriétaire de la base de données

Date création (champ CREDAT)

Date et heure de création de la base de données
champ CREHOU
Nombre d'utilisateurs (champ NBRUSR)

Nombre d'utilisateurs définis dans la base de données.
Nombre de connexions actives (champ NBRPRO)

Nombre de connexions actives sur la base de données.
Taille de la base (champ TAILBASE)

Taille totale de la base de données courante, tous dossiers confondus. Cette taille inclut tous les fichiers de données et les fichiers journaux.
champ MB1
Espace non alloué (champ ESPACE)

Espace de la base de données qui n'a pas été réservé pour des objets de base de données.
champ MB2
Classement (champ BDDTRI)

Type de classement utilisé par la base de données.
Modèle de récupération (champ RPCTPL)

Spécifie le type de modèle de récupération de la base de données

Les modèles de récupération suivants sont disponibles :

    • Récupération simple
      La récupération simple permet la récupération de la base de données à l'aide de la sauvegarde la plus récente.
    • Récupération complète
      La récupération complète permet la récupération de la base de données à partir du moment de l'échec.
    • Récupération journalisée en bloc
      La récupération journalisée en bloc permet des opérations de journalisation en bloc.
Niveau de compatibilité (champ LEV)

Indique le niveau de compatibilité de la base de données.

Tableau Options

Option (champ OPT)

Options de base de données.

Voici une liste des options que l'on peut trouver :

Option

Description

auto create statistics

Si vrai (true), les statistiques manquantes, requises par une requête pour l'optimisation, sont créées automatiquement durant l'optimisation.

auto update statistics

Si vrai (true), les statistiques non mises à jour, requises par une requête pour l'optimisation, sont créées automatiquement durant l'optimisation.

autoclose

Si vrai (true), la base de données est arrêtée proprement et ses ressources sont libérées dès que le dernier utilisateur s'est déconnecté.

autoshrink

Si vrai (true), les fichiers de base de données sont candidats à un compactage automatique périodique.

ANSI null default

Si vrai (true), CREATE TABLE suit les règles SQL-92 afin de déterminer si une colonne autorise les valeurs NULL.

valeurs ANSI NULL

Si vrai (true), toutes les comparaisons à une valeur NULL sont évaluées à inconnu (UNKNOWN). Si faux (false), les comparaisons de valeurs non UNICODE à une valeur NULL sont évaluées à vrai (TRUE) si les deux valeurs sont NULL.

ANSI warnings

Si vrai (true), des erreurs ou avertissements sont émis si des conditions telles que « division par zéro » sont vérifiées.

arithabort

Si vrai (true), un dépassement de capacité ou une division par zéro provoquent l'arrêt du traitement de la requête ou du lot d'instructions. Si l'erreur se produit dans une transaction, cette dernière est annulée (roll back). Si faux (false), un message d'avertissement s'affiche, mais le traitement de la requête, du lot d'instructions ou de la transaction se poursuit, comme s'il n'y avait pas d'erreur.

concat null yields null

Si vrai (true), le résultat est NULL si l'un des deux opérandes de l'opération de concaténation est NULL.

c

Si vrai (true), tout curseur ouvert au moment où une transaction est engagée ou restaurée est fermé. Si faux (false), ces curseurs restent ouverts lorsqu'une transaction est engagée, et la restauration d'une transaction ferme tout curseur à l'exception de ceux qui sont définis avec le mot-clé INSENSITIVE ou STATIC.

dbo use only

Si vrai (true), seul le propriétaire de la base de données peut utiliser celle-ci.

default to local cursor

Si vrai (true), les déclarations de curseur ont pour valeur par défaut LOCAL.

publication de fusion

Si vrai (true), la base de données peut être publiée pour une réplication de fusion.

numeric roundabort

Si vrai (true), une erreur est générée lorsqu'une perte de précision survient dans une expression. Si faux (false), les pertes de précision ne génèrent pas de messages d'erreur et le résultat est arrondi en fonction de la précision de la colonne ou de la variable contenant le résultat.

offline

Si vrai (true), la base de données est hors ligne.

published

Si vrai (true), la base de données peut être publiée pour la réplication.

identicateur entre guillemets

Si vrai (true), vous pouvez encadrer de guillemets doubles les identificateurs délimités.

readonly

Si vrai (true), les utilisateurs sont uniquement autorisés à lire les données de la base de données, et non à les modifier. La base de données ne peut pas être en cours d'utilisation lorsque vous spécifiez une nouvelle value pour l'option read only. La base de données master fait exception à cette règle, et seul l'administrateur système peut l'utiliser pendant le paramétrage de l'option read only.

recursive triggers

Si vrai (true), permet l'activation récursive de déclencheurs. Si faux (false), empêche uniquement la récurrence directe. Pour désactiver la récurrence indirecte, affectez la valeur 0 à l'option de serveur nested triggers à l'aide de sp_configure.

select into /bulkcopy

Si vrai (true), l'instruction SELECT INTO et les copies en bloc rapides sont autorisées.

single user

Si vrai (true), la base de données ne peut être utilisée que par un seul utilisateur à la fois.

subscribed

Si la valeur est true, la base de données peut être abonnée à une publication.

torn page detection

Si vrai (true), les pages incomplètes peuvent être détectées.

trunc. log on chkpt

Si vrai (true), un point de contrôle vide la partie inactive du journal lorsque la base de données est dans le mode troncature de journal. Il s'agit de la seule option autorisée pour la base de données master.


Valeur (champ OPTVAL)

Dernière sauvegarde
