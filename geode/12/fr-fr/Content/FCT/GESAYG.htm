Pages Web

 Avant de créer votre site , il préférable de lire la documentation :Avant de commencer.

Fonction

Cette fonction permet de déclarer les pages du site XTEND.

Elle fait le lien entre un code ou alias utilisé par XTEND et le fichier HTML associé.

Le répertoire racine des fichiers HTML sur  le serveur X3 est le suivant :
/X3SOLUTION/dossiers/X3_PUB/X3FOLDER/X_TEND/X_HTML/SITEXTEND/CODELANGUE/

  • CODELANGUE : Code langue utilisateur
  • SITEXTEND  : Code du site
  • X3FOLDER   : Code dossier X3
  • X3SOLUTION : Code solution X3

En-tête

Descriptions des champs
Code page (champ PAGCOD)

Code du token page Web.

Site (champ FCYLIB)

Site Web courant.

Intitulé (champ INTIT)

Permet de définir un intitulé associé à chaque fiche.

Ecran de saisie

Descriptions des champs

Page

Page par défaut (champ FICDEF)

Nom du fichier html utilisé par défaut pour toutes les langues.
Bloc principal (champ BLCMAI)

Code du bloc principal pour la page.

Le bloc principal est identifié par le code générique AMAIN. Il permet d'utiliser des tokens génériques (par exemple les bloc conditionnés) et de factoriser du code HTML (par exemple le code qui gère la pagination d'un bloc).

Sécurité

Accès protégé (champ ACSMOD)

Oui : pour n'autoriser l'accès qu'aux utilisateurs signés ou disposant du profil XTEND adéquate
Non : pour autoriser l'accès à tous les utilisateurs (anonymes)

Protocole (champ WEBPRO)

Protocole HTTP ou HTTPS utilisé pour afficher la page.

'Site web' : indique qu'il faut utiliser le protocole par défaut défini dans la fiche site.

Http ou Https permet de forcer l'utilisation d'un protocole particulier pour cette page. Par exemple la page de login force l'utilisation du protocole Https pour crypter les données.

Onglet Avancé

Descriptions des champs

Bloc

Bloc de fond (champ BLCBCK)

Si ce paramètre contient le code d'un bloc, ce bloc sera automatiquement inséré dans la page sous le tag <body> sans avoir à déclarer le token <!adx="MonBloc">..<!adx="MonBloc"> dans la page.

Suivi de l'activité (champ LOGMOD)

Oui : pour tracer les accès à cette page dans le log XTEND

Tableau Exceptions par langue

Code langue utilisateur
Page (champ FICLAN)

Nom du fichier pour la langue utilisateur

Etats

Par défaut, les états suivants sont associés à la fonction :

  PRTSCR : Impression écran

Mais ceci peut être modifié par paramétrage.

Boutons spécifiques

  • Validation
  • Copie

Validation

Ce bouton permet de mettre à jour le dictionnaire XTEND au format XML.

Le serveur X3WEB prend en compte uniquement le dictionnaire sous ce format.

Il existe un fichier xml par dictionnaire.

Par exemple le fichier ACT.xml correspondant à l'ensemble des actions web pour le site Web courant.

Copie

Ce bouton permet de copier un token page Web.

Outils / Visualisation page web

Cet outil permet de visualiser la page dans le navigateur Web par défaut.

Bloc numéro 1

  • Chemin d'accès (champ CHEMIN)

Ce champ contient l'url qui sera visualisé par le navigateur web si l'utilisateur clique sur l'icone et par le champ AC1.

  • champ CLIC

En cliquant, le navigateur par défaut est ouvert chargé avec l'url dans le champ CHEMIN.

  • champ AC1
