Valeurs paramètres

Cette fonction permet de mettre à jour les différents paramètres utilisés par le progiciel dans tous ses modules.

Un paramètre est une valeur influant le fonctionnement du progiciel, sans pour autant que la structure de la base de données en dépende. Il peut être de type :

  • alphanumérique (on saisit une série de caractères). Si la valeur est contrôlée par une table, la fenêtre de sélection est disponible par clic droit.
  • numérique
  • de type menu local, c'est-à-dire un choix dans une liste d'intitulés tels que Oui/Non par exemple. Un menu local est une table identifiée par un numéro. On peut la saisir soit en sélectionnant par clic droit dans la liste des choix possibles, soit en saisissant les premiers caractères discriminants (dans ce cas, la saisie est complétée : si on tape O dans le cas d'un choix Oui/Non, cela suffit).

On trouvera par exemple comme paramètres dans le superviseur un code administrateur par défaut, un nombre maximum de tentatives infructueuses de saisie de mot de passe avant blocage, une question relative à la trace des opérations (réponsesà choix multiples). Dans tous les modules, on trouvera ainsi des paramètres à renseigner.

Un paramètre se caractérise par un code interne (ADMUSR pour le code cité ci-dessus), un intitulé qui est plus parlant, et un niveau de localisation. En effet, certains paramètres doivent être définis pour tout le dossier, alors que d'autres peuvent être définis par législation, par société, voire par site. Lorsqu'un paramètre n'a pas été défini pour un site donné, on reprend la valeur de la société dont il dépend, et si aucune valeur n'a été définie pour la société, on reprend la valeur de la législation dont il dépend, et si aucune valeur n'est définie à ce niveau, on reprend la valeur par défaut du dossier. Ainsi même si un paramètre peut être défini par site, il est possible de donner une valeur globale pour le dossier, et de ne définir des exceptions que pour certaines sociétés, législations et/ou certains sites.

Gestion de l'écran

Ecran de saisie

On saisit tout d'abord le code du chapitre, qui correspond à un module fonctionnel (ACH pour achats, CPT pour comptabilité, etc.…).

Puis on se place au niveau de définition souhaité en saisissant au choix :

  • soit un code législation
  • soit un code société
  • soit un code site
  • soit aucun code (pour être au niveau Dossier)

On peut alors modifier les paramètres qui peuvent être définis au niveau correspondant, et aux niveaux situés en dessous. Ainsi, si on a saisi un site, seuls les paramètres définissables à ce niveau de détail peuvent être modifiés. Si on a saisi un code société, les paramètres définissables au niveau Société et au niveau Site peuvent être modifiés (on peut ainsi donner une valeur par défaut pour tous les sites de la société qui n'ont pas de valeur par ailleurs).

Au niveau Dossier, tous les paramètres peuvent être modifiés, ce qui permet de leur donner une valeur par défaut.

Certains paramètres sont définis au niveau Utilisateur : ils sont en général modifiables, puisqu'on peut leur donner une valeur par défaut au niveau de leur site. Mais si on désire changer leur valeur pour un utilisateur donné, il faudra le faire dans l'onglet correspondant de la fiche utilisateur.

Une fois le choix du niveau fait, on voit apparaître une liste de groupes de paramètres. Chaque groupe correspond à un ensemble de paramètres à renseigner de façon simultanée. La liste gauche nommée Paramètres par groupe permet d'afficher le détail des paramètres du groupe situé sur la ligne courante avec leur valeur. La dernière colonne du tableau donne le niveau de définition le plus fin d'au moins un des paramètres du groupe. Ceci permet de savoir, par exemple, qu'au niveau d'un site, pour un groupe donné, tous les paramètres sont hérités du niveau Société ou d'un niveau supérieur, et qu'aucun n'a réellement été défini au iveau du site.

La modification des paramètres peut se faire :

  • soit en passant au détail des paramètres, avec le bouton décrit ci-dessous (ou par double clic sur le paramètre correspondant dans la liste gauche Paramètres par groupe : la fenêtre correspondant aux valeurs s'ouvre alors, le curseur étant positionné sur la ligne correspondante).
  • soit en affectant un jeu de valeurs prédéfini au groupe. Ces jeux de paramètres permettent de disposer de valeurs par défaut sans avoir à redescendre au niveau de chaque paramètre.
Descriptions des champs

Bloc numéro 1

Chapitre (champ CHAPITRE)

Ce champ identifie le chapitre fonctionnel auquel appartiennent les paramètres que l'on souhaite modifier.
Législation (champ LEG)

Ce champ définit la législation pour laquelle on veut modifier les paramètres (si ce champ reste vide, on modifiera les paramètres à un autre niveau).
Société (champ CMP)

Ce champ définit la société pour laquelle on veut modifier les paramètres (si ce champ reste vide, on modifiera les paramètres à un autre niveau).
Site (champ FCY)

Ce champ définit le site pour lequel on veut modifier les paramètres (si ce champ reste vide, on modifiera les paramètres à un autre niveau).

Tableau numéro 1

Groupe (champ GRPCOD)

A l'intérieur d'un chapitre de paramètres, on définit des groupes logiques de paramètres qui sont en général modifiés simultanément, et pour lesquels on peut définir des jeux de valeurs globales.
Jeu valeurs (champ JEUCOD)

Ce champ permet d'imposer un jeu de valeur prédéfini aux paramètres du groupe. S'il est vide, cela signifie que les valeurs des paramètres ont été renseignées séparément, et ne correspondent pas à un jeu de valeur prédéfini.
Niveau (champ NIVGRP)

Ce champ affiche le niveau de détail auquel le paramètre est défini. Sur une ligne de groupe, on affiche le niveau de détail le plus fin auquel au moins un des paramètres du groupe a été défini.
Icône Actions
Détail

Cette fonction accessible par clic droit sur la zone Jeu de valeurs permet de saisir les valeurs de chaque paramètre séparément. On retrouve alors un tableau contenant les différentes valeurs, avec l'indication du niveau auquel elle a été définie. En étant positionné sur la ligne courante sans être en saisie, on peut, par clic droit, avoir accès aux fonctions suivantes :

Aide

Cette fonction permet d'accéder à une aide contextuelle, qui précise l'utilisation du paramètre de la ligne courante.

Sélection d'un jeu

Cette fonction permet de choisir un jeu de valeurs pour affecter l'ensemble des paramètres (que l'on peut ensuite modifier de façon individuelle).

Jeux de valeurs

Permet un tunnel vers cette fonction.

Quand est en saisie sur une valeur, on trouve la possibilité, via des choix accessibles par clic droit, de retrouver la valeur définie à un niveau supérieur en supprimant la valeur du niveau courant. Ainsi, sur une valeur définie au niveau Site, on retrouve par clic droit les choix Valeur Société, Valeur Législation, Valeur Dossier.

Etats

Par défaut, les états suivants sont associés à la fonction :

  ADOVAL : Valeurs paramètres

Mais ceci peut être modifié par paramétrage.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Modification en cours sur un autre poste

Ceci signifie qu'une autre personne est en train de modifier les mêmes paramètres sur un autre poste. En effet, le verrouillage de cette fonction se fait par chapitre et niveau (par société, ou site,  ou avec une valeur vide dans les deux champs). Il est donc impossible de réaliser la modification sur deux postes d'un même chapitre au même niveau pour la même société ou site.
