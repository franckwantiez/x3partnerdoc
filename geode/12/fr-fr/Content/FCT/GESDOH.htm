Ordres de livraison

A partir de cet écran, vous pouvez :

  • Créer/modifier une entête d’ordre de livraison (O.L) et ses lignes.
  • Visualiser une entête O.L et les lignes associées.
  • Solder une entête/ligne d’ordre de livraison.
  • Visualiser les mouvements de sortie, les colis, les unités de manutention (U.M), et les lignes en rupture d’un O.L.
  • Visualiser et éditer la charge d’un OL actif, c’est à dire non soldé.

Un O.L est créé à l’aide de 4 onglets pour l’entête et 2 pour les lignes.

En-tête

Numéro. Séquence numérique attribuée automatiquement par le système lors de l’enregistrement de l’O.L. Ce numéro est une information propre au site, tous déposants confondus. Cette information constitue une clef unique d'accès à un O.L.

Référence. Référence de l’O.L. Saisie facultative.
Si la zone est laissée à blanc, le système y reporte automatiquement le Numéro d’O.L qui est modifiable.
Si la zone est renseignée, le système impose l’unicité de la zone en fonction du paramétrage de la configuration des sorties.

En transit. Indique le type d'OL.

Si le flag "en transit" est activé, cela signifie que l'OL a été expédié, via une tournée "inter-site", vers un autre site. Sur ce site, l'expédition finale vers le client destinatire n'a pas encore eu lieu. Ce n'est qu'une fois l'expédition finale confirmée que l'OL, sur le site de préparation, passera à l'état '9-expédié/soldé'.

Un OL ne peut être expédié vers un autre site et par conséquent réputé "en transit", que via l'expédition par tournée "inter-site".

Sur le site de destination, un OL réputé "en transit" ne peut pas être reporté, annulé, déchargé ou fractionné. L'OL, "en transit", peut être consolidé avec d'autres OL du site courant dans une même tournée d'expédition finale ou bien re-expédié vers son site d'origine via une tournée "inter-site" dite de retour.

Lorsqu'un OL est expédié vers un autre site, dans le cadre d'une tournée "inter-site", un BL dit "inter-site" peut être édité si la configuration de niveau déposant est faite (Configuration des sorties).

Etat. Ce champ précise le code état de l’entête de l’O.L.

Date. Date de création de l’O.L. Lors de la création d'un O.L, la date système est proposée par défaut. Ce champ est obligatoire. La date de l’O.L peut être modifiée par l'utilisateur.

Préparation

informations relatives à la préparation de l'OL.

Régles de fonctionnement du flag "UM mono destinataire":
- un colis d’un OL flagé « UM mono destinataire » ne pourra être palettisé que sur un UM flagé « UM mono destinataire » ayant le même destinataire.
- un colis d’un OL non flagé « UM mono destinataire » pourra être mis:
        - soit sur un UM non flagé « UM mono destinataire »
        - soit sur un UM flagé « UM mono destinataire » ayant le même destinataire.

Régles de fonctionnement du flag "UM mono OL":
- un colis d’un OL flagé « UM mono OL » ne pourra être palettisé que sur un UM flagé « UM mono OL » sur le même OL.
- un colis d’un OL non flagé « UM mono OL » pourra être mis:
        - soit sur un UM non flagé « UM mono OL »
        - soit sur un UM flagé « UM mono OL » sur le même OL.

Livraison partielle

Pour toutes les autres fonctions pouvant amender la quantité lancée en préparation (validation OP, validation colis, report O.L etc…), le principe suivant est retenu : il est possible pour un O.L de valider partiellement puis d’annuler le solde; par contre, il n’est pas possible d’annuler puis de valider le solde. Ce principe a été retenu de façon à répondre à la problématique de casse «à la préparation» dans la mesure où une quantité a déjà été préparée et validée.

Options de gestion

Palettisation des colis (champ PRCPLZ)

Indique si la phase de palettisation des colis est active pour le déposant courant.
Si la case n’est pas cochée, la palettisation des colis n'est pas gérée.
Si la case est cochée, la palettisation des colis est gérée. Chaque colis de l’OL devra être palettisé.

Chargement des colis (champ LODCTL)

Indique si une déclaration unitaire de chargement des colis doit être effectuée.
A ce niveau, on entend par "unitaire" : le colis, si la palettisation n'est pas gérée ou l'UM, si la palettisation est gérée.
Si la case n’est pas cochée, la déclaration de chargement colis n'est pas utilisée.
Si la case est cochée, la déclaration de chargement colis est utilisée.

Gestion des expéditions (champ DSPMGT)

Renseigne le système sur l'utilisation de la fonction "confirmation d'expédition" pour un OL qu’il soit ou pas colisé.
Si la case n’est pas cochée, la confirmation d'expédition n'est pas utilisée. Le système fait évoluer le statut de l’OL à 9, expédié, soldé, à l’ultime phase précédent l’expédition.
Si la case est cochée, la confirmation d'expédition est utilisée selon deux modes possibles, par OL ou bien par tournée d’expédition.

Remarque : Il n'est pas nécessaire que tous les prélèvements (colis/O.P) d'un OL soient validés pour confirmer l'expédition d'un OL (expédition partielle) si les reports d'expédition sont utilisés.
Gestion des reliquats (champ SHRMGT)

Si la case n’est pas cochée, un OL partiellement servi ou non servi est soldé automatiquement suite au passage à l’état 9.
Si la case est cochée, la totalité des lignes OL non servies est conservée en portefeuille.

Un OL est conservé dans son intégralité (passage au statut 2, en attente de xième préparation) :
- s’il est en rupture partielle ou totale au lancement de vacation,
- si les ordres de prélèvement sont annulés ou annulés partiellement
- si l’annulation d’expédition est utilisée.

Pour le solder définitivement, il est alors nécessaire de venir sur chacun des OL pour les solder définitivement.

Solder

Il permet de solder une entête ou une ligne ; le solde est une opération de passage à l’état 9 de l’entête ou des lignes ; en effet la suppression n’est pas autorisée pour des entités pouvant être transmises par interface.
Le solde d’une ligne de type Kit solde toutes les lignes composantes associées.

Lignes

Pour accéder directement au détail des lignes de l’O.L.

En-tête

Les éléments dans cette section sont ceux issus de l’entête de l’O.L préalablement créée ou sélectionnée. On y retrouve :

Numéro, Référence, Etat et Date.

Ligne

Numéro. Numéro attribué par le système. Dans le cadre d’une consultation, il suffit de saisir le numéro de la ligne que l’on souhaite interroger et son contenu est automatiquement affiché.

Etat. Ce champ précise le code état de la ligne de l’O.L.

Nature de ligne (Disponible à compter de la version Advanced). Indique si l’article saisi est de type « normal », «Kit » (ou composé) ou « composant ».
Lors de la sélection de la valeur « Kit », le système recherche le code article saisi dans la table des kits :
s’il est trouvé, il est demandé à l’opérateur s’il souhaite que le système crée les lignes suivantes selon la nomenclature définie ; la création est alors proposée de manière unitaire à l’opérateur, qui peut valider par le boutonou interrompre la création par le bouton.
s’il n’est pas trouvé, la déclaration de la nomenclature est unitaire et laissée libre.

Une ligne de nature Kit doit avoir au moins deux lignes de nature Composant.
Les quantités de chaque ligne composantes doivent être multiples du nombre de kit.

Intitulé. Libellé du code article.

Général

Quantités

Quantité et code U.L. Quantité et type et code contenant de la ligne d'O.L.
Le contenant saisi doit être défini dans le plan de palettisation de l’article et ne doit pas être interdit en sortie.
Si le code U.L saisi est de niveau 5, le système assure lors du lancement de vacation de rechercher un ou plusieurs objets de stocks de niveaux différents dont la somme répond à la quantité demandée.
Si le code U.L demandé est de niveau différent, un objet de stock exactement de même niveau sera recherché lors du lancement de vacation.

Le total UC lié à la quantité U.L est affiché, de même que le plan de palettisation du code U.L saisi.

Commande. Saisie facultative, N° de ligne de commande ou N° de commande si l’O.L est multi commandes.

Dates

Lignes O.L. Date de création de l’entête O.L.

FIFO. Date FIFO demandée. Si renseignée, lors du lancement en vacation, selon le paramétrage de la configuration des sorties, le système prélève des objets de stocks de date FIFO strictement égale à celle demandée ou de date supérieure ou égale à celle de la ligne O.L.

Code mouvement. Code mouvement par défaut de l’entête O.L.

Numéros

N° de réservation. N° de réservation demandé. Si renseigné, lors du lancement en vacation, selon le paramétrage de la configuration des sorties, le système prélève des objets de stocks de même code réservation ou de code réservation différent s’il n’y a pas suffisamment de stock.

N°de support (Disponible à compter de la version Advanced). N° de support demandé. Si renseigné, lors du lancement en vacation, le système prélève des objets de stocks de même n° de support.

Colisage

Gamme colis. Gamme colis imposée pour la ligne ; lors du précolisage, le système cour circuite la recherche du colis optimum pour utiliser celui imposé pour la ligne O.L.

Regroupement. Si renseigné, lors du précolisage, le système court circuite la recherche du code regroupement standard pour utiliser uniquement celui de la ligne O.L.

Niveau maximum d’expédiabilité en l’état. Par défaut est égal au niveau 5. Ce paramètre permet de court circuiter le paramétrage expédiable en l’état du contenant article. Pour être expédiable en l’état, un contenant article doit en plus de son propre paramétrage, être supérieur ou égal à celui de la ligne.

Exemple : soit 1 article de 5 niveaux de contenants expédiables en l’état.

La ligne OL est égale à 1 => seuls les objets de niveaux 1 seront expédiables en l’état.
La ligne OL est égale à 2 => seuls les objets de niveaux 1 et 2 seront expédiables en l’état, les autres seront colisés en vrac.

Etc…

La ligne OL est égale à 5 => tous les objets seront expédiables en l’état.

Commentaires

Commentaires. Si renseigné, le commentaire de la ligne est imprimé sur chaque ligne de B.L.

Observations. La zone n’est pas imprimée sur le B.L.

Avancement

Préparation

Lancée. Quantité exprimée en UC mise à jour par le système lors de la création de la ligne O.L, et lors du passage à l’état 9 de l’O.L s’il y a gestion des reliquats.
Par défaut la quantité UC lancée est égale à la quantité UC de la ligne O.L, puis en cas de gestion des reliquats elle est égale au solde restant à livrer.

En rupture. Quantité exprimée en UC mise à jour par le système, lors du lancement en vacation de la ligne O.L s’il y a rupture, lors de l’annulation d’un OP ou d’un colis s’il y a un manquant, lors de l’annulation d’expédition, si l’O.L est partiellement annulé.

Transféré(Disponible à compter de la version Premium). Quantité exprimée en UC mise à jour par le système si l’O.L est en mode de préparation de type cumul de flux, et correspondant à la quantité UC transférée (validée) pour servir la ligne O.L.

Non transféré (Disponible à compter de la version Premium). Quantité exprimée en UC mise à jour par le système si l’O.L est en mode de préparation de type cumul de flux, et correspondant à la quantité UC non transférée (annulée) pour servir la ligne O.L.

Préparé. Quantité exprimée en UC mise à jour par le système, lors de la validation partielle ou totale du mouvement de sortie.

Expédition

Annulé. Quantité exprimée en UC mise à jour par le système, lors de l’annulation de l’expédition soi par O.L soit par colis.

Reporté. Quantité exprimée en UC mise à jour par le système, lors du report d’expédition.

Expédié. Quantité exprimée en UC mise à jour par le système, lors de l’expédition de l’O.L.

Dates

Lancement. La date et heure de lancement est la dernière date et heure de lancement en vacation de la ligne O.L.

Attente Expédition. La date et heure d’attente d’expédition est la dernière date et heure d’attente d’expédition de la ligne O.L

Expédition. La date et heure d’expédition est la dernière date et heure d’expédition de la ligne O.L

Fonctions / Analyse de charge

Permet de lancer l’analyse des charges de l’OL sélectionné

La fonction d’analyse des charges est conditionnée par :
- le code activité LA0 pour un OL en état inférieur à 4
- le code activité LA2 pour un OL en état inférieur à 9
- l’état de l’OL qui doit être inférieur à 9 (OL non expédié/soldé).

Le système considère la charge restant à recevoir au titre de l’OL c’est à dire la somme des charges propres à :
- l’en-tête OL
- chaque ligne de l’OL non ou partiellement expédiée, en ne considérant que les quantités restant à expédier et non reportées.
- Chaque mouvement de sortie en cours associé à l’OL
- Chaque mouvement de sortie validé non colisé de l’OL (mode de préparation avec colisage déclaratif)
- Chaque colis de l’OL non expédié
- Chaque UM de l’OL non expédié

Suite à une demande d’analyse de charge :
• une nouvelle analyse est créée avec les caractéristiques suivantes :
- Indicateur à conserver = « non »
- Modèle (pour information) :
   ▪ celui associé au triplet (fonction, site en cours, déposant en cours) pour une fonction mono déposant
   ▪ Par défaut, ou pour une fonction multi déposants, celui associé au triplet (fonction, site en cours, déposant « à blanc ».
   ▪ Par défaut, celui associé au triplet (fonction, site « à blanc », déposant « à blanc »)
   ▪ Par défaut, il est laissé à blanc.
• Le calcul est lancé pour l’analyse
• L’opérateur accède aux écrans d’affichage de la charge à partir desquels il pourra également accéder aux informations de l’analyse (description et indicateur « à conserver ») et demander l’édition des résultats de l’analyse.
Fonctions / Mouvements consommables

Permet de déclarer 1 ou n mouvements de sortie de stock de consommables au titre de l'ordre de livraison.

Cette saisie peut se faire à tout moment, quelque soit l'état de l'OL.

Pour réaliser un mouvement de sortie de consommable, il faut saisir diverses informations:

    • Le code consommable: définit dans la fiche article des consommables
    • Le code mouvement: doit appartenir à la famille des traitements 'CONSOMMABLE ' et être de type 'SORTIE'
    • La quantité: quantité du mouvement de sortie

Comme le mouvement de sortie du consommable est déclaré à partir d'une entité connue, les informations, transporteur, mode de transport, client et intitulé client, sont, le cas échéant, renseignés avec celles de l'entité.

Pour un mouvement de sortie, les informations fournisseur et intitulé fournisseur ne sont pas accessibles ainsi que le type de l'entité et le numéro de cette dernière.

Selon le paramétrage, la saisie d'un code opérateur est demandée pour valider le mouvement d'entrée. A la validation, le stock est incrémenté.

Après validation, le mouvement n'est plus modifiable. Par conséquent, il n'est pas actualisé dans le cas où des mises à jour sont réalisées sur l'entité à laquelle ce mouvement est lié.

Il est possible de générer 3 états:

    • Dans Fichier / imprimer / fiche:
        • Stock des consommables trié par consommable ou par quantité
        • Impression écran pour l'enregistrement sélectionné 
    • Dans Fichier / imprimer / liste:
        • Mouvements des consommables trié par consommable ou par date

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur Zone Code mouvement (ligne ou en-tête), Fiche inexistante.

On a tenté de saisir un code mouvement inexistant.

Erreur zone Mode préparation, zone obligatoire

Le mode de préparation doit obligatoirement être renseigné.

Erreur Zone Transporteur, Fiche inexistante.

On a tenté de saisir un code transporteur inexistant.

Erreur Zone Mode transport, Fiche inexistante.

On a tenté de saisir un mode de transport inexistant.

Erreur Zone Pays, Fiche inexistante.

On a tenté de saisir un pays inexistant.

Erreur Zone Code, Fiche inexistante.

On a tenté de saisir un code client inexistant.

Les messages d'erreur liés à la déclaration de mouvement de consommables sont:

Erreur zone "consommable". Zone obligatoire.

Le code du consommable doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Zone obligatoire.

Le code mouvement doit obligatoirement être renseigné.

Erreur zone "Code mouvement". Code mouvement de type non consommable.

Le code mouvement renseigné doit appartenir à la famille des traitements 'CONSOMMABNLE'.

Erreur zone "Code mouveement". Code mouvement de type entrée.

Le code mouvement renseigné doit obligatoirement être de type 'SORTIE'.

Erreur zone "Quantité". Zone obligatoire

La quantité du mouvement d'entrée doit obligatoirement être renseignée.

Erreur de l'application. Le déposant doit être renseigné pour cette origine.

Lorsqu'un mouvement d'entrée est lié à une entrée attendue ou une entrée directe, le déposant doit obligatoirement être renseigné. 

Erreur zone "N°origine mvt". Référence origine non trouvée.

Le numéro de l'entité Warehousing doit obligatoirement être renseigné et existante pour le site.

Erreur zone "Consommable". Consommable bloqué en sortie.

Le consommable doit obligatoirement être autorisé en sortie pour réaliser un mouvement de sortie.
