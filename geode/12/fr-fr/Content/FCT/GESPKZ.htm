Zones de préparation

Cet écran permet de créer, visualiser, modifier, supprimer une Zone de Préparation d’un site.

La notion de Zone de Préparation est utilisée pour le découpage des Ordre de Prélèvement.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Général
  • Onglet Préparation

Pour créer une zone de préparation, on attribue un code qui l’identifie de façon unique sur 4 caractères et un intitulé. Puis on saisi les différentes informations dans les 2 onglets présents.

En-tête

Les informations à saisir sont:

Zone de préparation. un code qui l’identifie de façon unique sur 4 caractères

Intitulé. Description en clair de la zone de préparation.

Descriptions des champs
Zone de préparation (champ PKZ)

Code de la zone de préparation qui est vérifié dans la table des zones de préparation. La notion de Zone de Préparation est utilisée pour le découpage des Ordre de Prélèvement.
Intitulé (champ PKZDESAXX)
Intitulé court (champ PKZSHOAXX)

Onglet Général

Dans cet onglet, on retrouve les éléments d’édition (Thermique et laser) et de transmission vers N1.

Remarque sur le paramètrage Imprimante: Cette notion permet d'affecter à chaque zone de préparation des imprimante dédiées et d'éviter aux opérateurs d'avoir à revenir à un point central d'édition.

Au lancement de la préparation:
- Si OP préco, on imprime les étiquettes colis sur l'imprimante thermique 1.
- Si OP prépal, selon le paramètre "Etiquettes groupées" de la config. des sorties (Onglet Colisage):
          - si actif, on imprime les étiquettes UM+Colis sur l'imprimante thermique 2.
          - sinon, on imprime les étiquettes UM sur l'imprimante thermique 2 et les étiquettes colis sur l'imprimante thermique 1.

Lors de la palettisation automatique:
- pour l'OPA (Ordre de PAlettisation), on imprime les étiquettes UM sur l'imprimante thermique 2.

Descriptions des champs

Bloc numéro 1

L’ensemble des préparations seront transmises au système N1.

Transmission N1 (champ TRMLVL1)

Tri

Code Tri (champ CTSCOD)

Un code de 3 caractères avec lequel vous pouvez définir l’ensemble des critères de tri appliqués à une vacation ou à une zone de préparation. Ce code est alors affecté à un ordre de préparation provenant de la vacation ou de la zone de préparation. 


Le Code Tri est saisissable dans les fonctions suivantes :

  • Vacation générique
  • Vacation
  • Zones de préparation
 Le Code tri

Le Code Tri de la zone de préparation est affecté aux ordres de préparation si aucun Code Tri n’est indiqué dans les vacations.

Editions

OP (champ PKOEDT)

C'est le code état qui définit l'édition qui sera édité en tant qu'Ordre de Préparation. Ce code état est vérifié dans la table des codes Etats.

Imprimantes

champ PRN1

Code de l'imprimante vers laquelle doivent être routés les documents de préparation et de réapprovisionnement nécessitant une imprimante Laser. Ce code est vérifié dans la table des imprimantes.
champ PRN2

Code de l'imprimante vers laquelle doivent être routés les documents de préparation et de réapprovisionnement nécessitant une imprimante thermique (Zebra). Ce code est vérifié dans la table des imprimantes.

          - sinon, on imprime les étiquettes UM sur l'imprimante thermique 2 et les étiquettes colis sur l'imprimante thermique 1.

champ PRN3

Onglet Préparation

Dans cet onglet, on retrouve les critères de la zone de préparation.

Maxima OP

Les valeurs ci-dessous sont les maxima admis pour un ordre de préparation:
Temps de préparation
Nombre de lignes
Nombre d’UC
Poids brut
Volume brut

Le mode de préparation de l’OL donne le type d’OP:

OP picking : tous les prélèvements sont à réaliser à partir d’emplacement de type picking.
OP réserve : tous les prélèvements sont à réaliser à partir d’emplacements de type réserve.
OP mixte : les prélèvements sont à réaliser à partir d’emplacement de type picking et réserve.

En cas de dépassement de l'un des critères fixés, le système découpe l'Ordre de Prélèvement de façon à respecter les limites établies.

Calcul du temps de préparation. Dans le cas d'un mouvement réserve, le système considère le temps de déstockage de l'emplacement. Dans le cas d'un emplacement picking, le système multiplie le temps de déstockage de l'emplacement par le nombre d'objets prélevés.
