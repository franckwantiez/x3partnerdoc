Destinations

Cette fonction vous permet d'envoyer un état vers un ensemble de destinations, en particulier vers toute imprimante Windows(TM) visible sur le réseau du poste client. Vous pouvez également envoyer des états vers des fichiers ou d'autres destinations. Afin de faciliter la sélection et l'affectation des destinations par défaut, il est recommandé de centraliser la description de ces destinations accessibles globalement dans cette table.

Cette table contient les destinations, identifiées par un code, et un ensemble de caractéristiques permettant d'affiner la sélection et l'affectation par défaut des destinations. Vous pouvez également indiquer si un serveur d'impression Crystal Reports(TM) est utilisé.

Descriptions des champs
Code (champ COD)

Ce code identifie de façon unique la fiche en cours.
Intitulé (champ DES)

Intitulé associé au code précédent
Intitulé court (champ SHO)

Intitulé court libellé sur 10 caractères alphanumériques.

Onglet Écran de saisie

La saisie des destinations s'effectue sur une seule section. Vous devez d'abord saisir le code de la destination, puis ses principales caractéristiques.

Vous pouvez paramétrer la désignation, l'intitulé court, mais aussi activer/désactiver l'indicateur Actif, définir un code d'accès pour filtrer les utilisateurs autorisés à utiliser la destination. Vous pouvez également définir le type de sortie, de la façon suivante :

  • la pré-visualisation (rien d'autre ne doit être saisi);
  • l'imprimante normale dont on définit les autres caractéristiques;
  • le message et le fichier, pour lequel seul le format doit être saisi (dans une liste  définie par le menu local 91, comprenant notamment Word(TM), différentes versions d'Excel(TM), de Lotus 123(TM), de HTML, le format RTF, ASCII, le format CSV, etc.);
  • l'imprimante/fichier (on définit l'imprimante pour générer un fichier d'extension .prn, qui contient l'ensemble du code à envoyer à l'imprimante). Vous pouvez également indiquer le format d'export si nécessaire, puis le serveur d'édition Crystal Reports et l'imprimante, sous forme de nom de réseau.

Vous pouvez par ailleurs affecter un type à la destination. Ce type, défini par le menu local 22, possède les caractéristiques suivantes :

  • le premier est un type passe-partout (tout état est susceptible d'être envoyé à des destinations de ce type) ;
  • les autres types permettent d'établir des catégories particulières (imprimantes code-barre, fax, etc.) qui ne peuvent être utilisées que par certains états. En effet, dans le dictionnaire des états, il est possible de définir un type de sortie dédié à un état en utilisant le même menu local. Là encore, si ce type n'est pas le premier, il doit y avoir correspondance entre le type affecté à l'état et le type de la destination utilisée.

Enfin, vous pouvez attribuer par défaut une valeur au nombre de copies à envoyer sur la destination utilisée.

Descriptions des champs
Actif (champ ENAFLG)

Sélectionnez cette case à cocher pour activer la fiche courante.

Les enregistrements non sélectionnés conservent leur contenu et paramétrage, mais ne pourront pas être utilisés en rappelant leur code dans :

  • les autres enregistrements, comme les documents et paramètres,
  • les traitements en masse.

Les habilitations sur une fonction donnée peuvent interdire la création d'une fiche active. Dans ce cas, la case est désactivée par défaut. Elle est modifiable uniquement par un utilisateur autorisé, ou via un Workflow de signature.
Code d'accès (champ ACS)

Ce code d'accès permet de restreindre l'accès à la fiche courante à certains utilisateurs.
Si la zone est renseignée, seuls les utilisateurs disposant de ce code d'accès dans leur profil peuvent visualiser et modifier cette fiche.

Le droit d'exécution associé à la destination contrôle le droit d'utiliser cette destination pour l'édition d'un état.
Type de sortie (champ PRT)

Ce champ définit la destination de l'état calculé. Les types de destination sont les suivants :

Pré-visualisation

Le flux généré dans un format compatible avec un affichage (texte, PDF, RTF, Word, etc.) est affiché dans une fenêtre qui s'ouvre sur le poste de l'utilisateur. Cette impression est formatée par Crystal Reports soit sur le poste de l'utilisateur, soit via un serveur d'impression.

Imprimante

Crystal Reports utilise une imprimante qui peut soit être locale, soit être accédée depuis un serveur d'impression.

Message

Crystal Reports crée un fichier contenant le message dans un format tel que PDF, texte, RTF, Word. Ce message est utilisé comme pièce jointe d'un mail envoyé via le moteur de Workflow.

Fichier

Crystal Reports crée un fichier contenant le message dans un format tel que PDF, texte, RTF, Word qui est stocké sur disque.

Imprimante ZPL

Le flux est pris en charge par un programme de formatage particulier associé à l'état et défini par un paramétrage dédié. Ce flux peut être envoyé directement sur une imprimante locale, sur une imprimante accessible depuis le serveur d'impression, ou stocké dans un fichier.

Archivage

Crystal Reports crée un fichier contenant le message dans un format tel que PDF, texte, RTF, Word. Ce fichier est adressé à un système de GED pour archivage, via le connecteur standard GED.
Fichier destination (champ PRTZPL)

Ce champ permet d'indiquer le nom du fichier généré pour une imprimante de type ZPL, ou le nom du répertoire dans lequel le fichier est créé. Il doit être indiqué avec le chemin complet en "dur" ou dans une formule (sous la forme =expression).

Si le fichier se termine par l'un des deux caractères '/' ou '\', il est considéré comme un répertoire de dépôt des fichiers, et dans ce cas, le fichier généré dans le sous-répertoire PAT du dossier est copié dans le répertoire. Le nom du fichier utilisé est composé de la concaténation des trois informations suivantes : le code du modèle ZPL, le numéro du processus courant (adxuid(1)), et la date et l'heure courante du serveur de traitement, exprimées sous le format YYMMDDhhmmss.

Si le champ Serveur est renseigné, vous devez indiquer un chemin accessible depuis le serveur d'impression (par exemple: file://machine/répertoire/fichier).

Si le champ Serveur n'est pas renseigné, vous pouvez indiquer un chemin sous une des formes suivante :

  • machine:service@chemin
  • /chemin 
  • c:\chemin

Si le nom de fichier contient un "@", le fichier est copié par un ordre système depuis le serveur de traitement, sinon depuis le client.

Dans tous les cas, il est important de bien paramétrer les droits d'accès aux machines. 

Dans la fonction Impression, ce nom de fichier est proposé par défaut après avoir choisi la destination de l'état.
Serveur (champ PRTSRV)

Définissez le serveur d'impression à partir duquel sera imprimé l'état. Si cette zone est vide, l'impression sera réalisée localement sur le poste client.

À partir de la mise à jour 8.0.0 :

Solution Cloud

Faites une demande d'assistance Sage en indiquant l'adresse de l'imprimante IPP pour qu'elle puisse être ajoutée à votre site Sage X3 en Cloud. Puis saisissez les emplacements de serveur et d'imprimante fournis par l'assistance technique.
Imprimante (champ PRTNAM)

Le nom d'imprimante défini ici sera proposé à l'impression de l'état. Il doit correspondre au nom de l'imprimante défini dans le panneau de configuration Windows.

 
Format export (champ PRTFMT)

Lorsque la sortie se fait sur fichier, il existe différents formats possibles. Cette liste de formats dépend de la version courante de Crystal Reports, elle est susceptible d'évoluer dans le temps.
Type (champ PRTNAT)

Ce champ permet d'affecter un type à la destination. Ce type, défini par le menu local 22, possède les caractéristiques suivantes :

  • Le premier est un type passe-partout (tout état est susceptible d'être envoyé à des destinations de ce type).
  • Les autres types permettent d'établir des catégories particulières (imprimantes code-barre, fax, etc.) qui ne peuvent être utilisées que par certains états. En effet, dans le dictionnaire des états, il est possible de définir un type de sortie dédié à un état en utilisant le même menu local. Là encore, si ce type n'est pas le premier, il doit y avoir correspondance entre le type affecté à l'état et le type de la destination utilisée.
Nombre de copies (champ NBRCPY)

Lorsque la sortie se fait sur imprimante, renseignez une valeur par défaut du nombre de copies à réaliser.
Copies assemblées (champ ASSCPY)

Si plusieurs copies sont demandées et si la case Copies assemblées est cochée, les pages sont assemblées avant d'éditer l'exemplaire. Sinon, chaque page est répétée.

Par exemple, dans le cas d'une impression de trois pages en deux exemplaires :

  • Si la case est cochée, l'ordre des pages est 1, 2, 3, 1, 2, 3.
  • Si la case n'est pas cochée, l'ordre des pages est 1, 1, 2, 2, 3, 3.

Imprimante non disponible
