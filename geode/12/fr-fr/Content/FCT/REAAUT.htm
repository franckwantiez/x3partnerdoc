Réappro automatique

Cette fonction permet de générer des ordres de réapprovisionnement d’un ensemble d’emplacements picking selon deux types de réapprovisionnement préventif : le réapprovisionnement sur seuil et le réapprovisionnement systématique.

Une simulation peut être demandée.

Gestion de l'écran

 L’écran est constitué d’un unique onglet.

Ecran de saisie

 L’écran permet de paramétrer le traitement de génération automatique des réapprovisionnements.

 Les zones à renseigner sont :

Magasin : saisie obligatoire du magasin à réapprovisionner. Seuls les emplacements picking du magasin sont réapprovisionnés.

Emplacement : saisie d’une plage d’emplacements à réapprovisionner ou d’un seul emplacement.

Type de réapprovisionnement : Réapprovisionnement sur seuil (valeur par défaut) ou réapprovisionnement systématique.
Le premier est déclenché si le stock de l’emplacement picking est strictement inférieur au seuil défini pour l’emplacement à réapprovisionner.
Le stock de l’emplacement picking est défini comme étant le total des objets de stocks de l’emplacement en prenant en compte les en cours d’entrée – en cours de sortie (en fonction du paramètre ‘Réappro toujours exécutable’ de la configuration des sorties).

Le second est systématiquement déclenché quel que soit le stock disponible si l’emplacement picking n’est pas plein.

Code mouvement transfert : il s’agit du code mouvement de type Transfert utilisé pour réaliser le réapprovisionnement.

La case à cocher Simulation indique s’il s’agit d’une simulation du réapprovisionnement ou bien si le système doit réellement générer les réapprovisionnements.

 Un emplacement est réapprovisionné si :
- l’emplacement n’est pas bloqué en entrée, ni en sortie, ni en cours d’inventaire,
- l’article associé à l’emplacement n’est pas en cours d’inventaire,
- le magasin est autorisé en sortie.

 Pour réaliser le réapprovisionnement, les objets de stock sont sélectionnés en fonction du mode de sortie de l’article.
Les contrôles suivants sont définis :
- l’emplacement de l’objet de stock <> l’emplacement picking à réapprovisionner
- l’emplacement de l’objet de stock n’est pas bloqué en sortie
- l’emplacement de l’objet de n’est pas en cours d’inventaire
- le magasin de l’objet de stock est autorisé en sortie
- le magasin de l’objet de stock n’est pas de type Eclatement
- les dates DLV, DLUO, DLC de l’objet de stock sont nulles ou non paramétrés dans la fiche article ou >= date du jour
- la date de rétention de l’objet de stock < date du jour ou non paramétré dans la fiche article
- les numéros de réservation et nature de stock ne sont pas renseignés
- le contenant est >= au contenant de l’emplacement 
- le plan de palettisation comprend le contenant de l’emplacement
- la qté disponible de l’objet de stock est > 0

 Le système va rompre l’objet de stock pour réaliser le réappro si sa quantité est supérieure à la pace disponible dans l’emplacement picking et s’il est le premier objet de stock sélectionné.

 L’action du bouton  permet de créer les réapprovisionnements.
Une étiquette de réappro par objet sélectionné ainsi qu’une liste de réapprovisionnement peuvent être générées si le paramétrage de la configuration des sorties du déposant courant le permet.

 Selon la configuration des sorties, le réappro généré sera à valider par les fonctions de Validation unitaire ou de Validation en masse , soit il sera automatiquement validé.

Descriptions des champs

Réapprovisionnement sur seuil : le réapprovisionnement est déclenché si le stock de l’emplacement picking est strictement inf&eac ute;rieur au seuil défini pour l’emplacement à réapprovisionner. Le stock de l’emplacement picking est défini comme étant le total des objets de stocks de l’emplacement avec ou sans en cours d&rsqu o;entrée – en cours de sortie.

Réapprovisionnement systématique : le réapprovisionnement est systématiquement déclenché quel que soit le stock disponible si l’emplacement picki ng n’est pas plein.

Magasin (champ STO)

Code permettant d’identifier le magasin. Il est vérifié dans la table des magasins.
Emplacement début (champ ADRDEB)

Adresse de début de la plage d'emplacements à réapprovisionner.
Emplacement fin (champ ADRFIN)

Adresse de fin de la plage d'emplacements à réapprovisionner
Type réappro (champ TYP)
Code mvt transfert (champ MVT)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
Simulation (champ SIM)

Permet de passer d’un mode réel (par défaut) au mode de simulation où toutes les données sont supprimées à l’issue du traitement.

Tâche batch

Cette fonction peut être lancée en batch. La tâche standard REAAUT est prévue à cet effet.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur Magasin, zone obligatoire

Le magasin doit obligatoirement est renseigné.

Erreur Magasin, Fiche inexistante.

On a tenté de saisir un Magasin inexistant.

Erreur Code mouvement transfert, zone obligatoire.

Un code mouvement doit être saisi.

Erreur Code mouvement transfert, Fiche inexistante.

Le code mouvement saisi n’existe pas pour le site courant.

Erreur Code mouvement transfert, Code mvt interdit.

Le code mouvement saisi doit être de type transfert.
