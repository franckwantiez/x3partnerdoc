Classes d'affectation

Cette fonction permet la définition des classes d'affectation associées aux contenantsarticle et aux emplacements.

La classe d'affectation peut être utilisée pour tenir compte de contraintes de stockage du type:
- article lourd, article encombrant, article dangereux ,ou tout autre élément caractéristique d’un article.

Pour chaque emplacement, on définit une classe d'affectation prioritaire, notion également liée aux contenants produits. Selon un processus identique à celui utilisé pour les classes de rotation, dans le cadre de la recherche d'emplacements, le système essaie de faire correspondre la classe de l'emplacement et la classe du contenant article.

Exemple : Sont définis de classe L les niveaux '0' et '1' d'un palettier, particulièrement avantageux pour le stockage de produits lourds. Les produits les plus lourds (en Kg par palette, par exemple) seront affectés à une même classe L.

La classe d'affectation est un des 5 critères communs entre articles et emplacements qui permet l'optimisation de la mise en emplacement dans Warehousing.

Les autres critères sont: Magasin,  Classe de rotation, Contenant, Déposant

Enfin, l’enchaînement des classes permet de donner au système l'ordre dans lequel les classes doivent être explorées. L'autorisation d'utilisation des règles d'enchaînement est conditionnée par le paramétrage du Mode d'entrée.

Remarque : La règle d'enchaînement ne peut être définie qu'après création des classes correspondantes, le système effectuant un contrôle d'existence des valeurs saisies.

Gestion de l'écran

La classe est définie puis les classes chaînées sont définies dans un tableau.

Ecran de saisie

Pour créer une classe d’affectation, on attribue un code qui l’identifie de façon unique sur 1 caractère et un intitulé. Ces informations se trouvent en tête de l’écran de gestion, on retrouve ensuite 1 tableau pour l’enchaînement des classes d’affectation.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Classe, La valeur maître ne peut appartenir à l’enchaînement

La classe maître ne peut figurer dans l’enchaînement.

Code déjà existant en ligne N

La même classe d’affectation a été saisie deux fois sur deux lignes différentes du tableau.
