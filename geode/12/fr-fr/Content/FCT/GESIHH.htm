Rubriques

Cette fonction permet de gérer (création, modification et suppression) les rubriques sur le dossier X3 courant.

Définition d'une rubrique: C'est un poste de facturation d'une prestation logistique tel que "le nombre de BL expédiés", "le nombre de colis expédiés", "le poids de palettes réceptionnées" ...

Il existe 3 types de rubriques:

    • 1-Table: la rubrique est une requête basée sur les tables Warehousing.
    • 2-Statistique: la rubrique est une requête basée sur une statistique existante.
    • 3-Forfait: la rubrique est de type forfaitaire. Elle n'est donc pas basée sur des données logistiques issues de tables Warehousing ou de statistiques mais applique uniquement un forfait.

Cette fonction fait partie du module optionnel de facturation des Prestations LOgistiques (GEOPLO).

Remarque: la requete est lancée sur le dossier courant mais aussi sur le dossier historique.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Liens
  • Onglet Paramétrage

    L'écran est constitué de:
  • une liste gauche contenant les rubriques existantes sur le dossier X3 courant.
  • Une partie droite contenant:
    • L'entête de la rubrique précisant ses principales caractéristiques (code, type ...)
    • 2 onglets "Liens" et "Paramétrage" permettant de construire la requête.

    Selon le type de rubrique que l'on souhaite définir, les informations à saisir sont différentes:
    Rubrique de type '1-Table':
      • Unité de calcul
      • la table primaire caractérisant l'entité à mesurer
      • le champ date de la table primaire qui sera rattaché à l'enregistrement extrait
      • les tables secondaires
      • les critères de sélection des enregistrements
      • la formule permettant de calculer la valeur de la rubrique
    Rubrique de type '2-Statistique':
      • Unité de calcul
      • le paramètre statistique et la valeur statistique à prendre en compte
      • les critères statistiques de sélection
    Rubrique de type '3-Forfait': aucune information à saisir

    Remarques sur les règles de gestion des rubriques:
  • Mode Modification: une rubrique est modifiable uniquement si toutes les factures associées sont validées.
  • Mode Suppression: une rubrique est supprimable uniquement si elle n'est associée à aucune prestation, code tarif ou tarif.

En-tête

Au niveau de l'entête, on retrouve les caractéristiques principales de la rubrique.

Descriptions des champs

Désignation

Saisissez la description de la fiche concernée.

Cet intitulé long est utilisé en titre dans les écrans et les états.

Rubrique (champ IHHCOD)

Code de la rubrique définit dans la table des rubriques GEODE.

La rubrique est un poste de facturation d'une prestation tel que "le nombre de BL expédiés", "le nombre de colis expédiés", "le poids de palettes réceptionnées" ...
Intitulé (champ IHHDESAXX)

Caractéristiques

Actif (champ ENAFLG)

Cochez ou décochez cette cache pour spécifier si votre Code Tri est actif ou inactif.

  • Si la case est cochée, vous pouvez appliquer le Code Tri à une vacation ou à une zone de préparation.
  • Si la case est décochée, vous ne pouvez pas appliquer le Code Tri. Si vous essayez de le faire, vous recevrez un message d’erreur qui vous indique que vous ne pouvez pas l’utiliser.   
Type (champ IHHTYP)

Indique le type de rubrique. 3 valeurs sont possibles:

  • 1-Table: la rubrique est basée sur les tables Geode.
  • 2-Statistique: la rubrique est basée sur une statistique.
  • 3-Forfait: la rubrique est de type forfaitaire. Elle n'est donc pas basée sur des données logistiques issues de tables Geode ou de statistiques.
Unité de calcul (champ IHHUOM)

Indique l'unité de calcul de la rubrique: Longueur, Surface, Volume, Masse/Poids, Conditionnement, Temps, Numération, Autres...

Table

Table (champ IHHTAB)

Dans le cas d'une rubrique de type '1-Table', ce champ indique la table primaire Geode identifiant l'entité Geode que l'on souhaite suivre.

Statistique

Statistique (champ IHHSTACOD)

Dans le cas d'une rubrique de type '2-Statistique', ce champ indique le paramètre statistique utilisé.

Valeur statistique (champ IHHSTAFOR)

Dans le cas d'une rubrique de type '2-Statistique', ce champ indique la valeur statistique qui sera prise en compte pour l'évaluation de la rubrique.

Opération (champ IHHSTAOPE)

Indique le type d'opération qui sera utilisé pour le calcul de la rubrique statistique:

  • Mini: la rubrique est égale à la valeur minimale des valeurs statistiques sur la période de calcul.
  • Maxi: la rubrique est égale à la valeur maximale des valeurs statistiques sur la période de calcul.
  • Somme: la rubrique est égale à la somme des valeurs statistiques sur la période de calcul.
  • Moyenne: la rubrique est égale à la moyenne des valeurs statistiques sur la période de calcul.

Onglet Liens

Cet onglet permet de construire la requête des rubriques de type "1-Table".

Descriptions des champs

Date de référence

Date (champ IHHTABDAT)

Dans le cas d'une rubrique de type '1-Table', ce champ donne le champ de la table primaire où se trouve la date à laquelle est rattaché l'enregistrement.
champ IHHTABDATL

Tableau Liens

Table (champ IHLSEC)

Dans le cas d'une rubrique de type '1-Table', ce champ indique les tables secondaires à prendre en compte dans l'évaluation de la rubrique.

Abréviation (champ WABRSEC)
Clé (champ IHLSECKEY)

Définit le code de la clé de la table utilisée pour réaliser la lecture des lignes liées.

Lien (champ IHLSECLNK)

Ce champ est défini comme une ou plusieurs expressions calculées et séparées par un point-virgule. Chaque expression est évaluée, et le résultat permet de connaître la valeur de la clé utilisée pour réaliser la jointure. Lorsque les jointures multiples sont autorisées, vous ne pouvez donner que les premiers segments de clé.

Dans les expressions, vous pouvez utiliser des constantes, et des champs issus des tables précédemment définies dans la liste des liens.
Priorité (champ IHLSECPIO)

Ce champ indique l'ordre des tables secondaires dans la requete.

Onglet Paramétrage

Cet onglet permet de paramétrer la requête des rubriques de type "1-Table" et "2-Statistique".

Rubrique de type "1-Table"

Concernant les "critères" de sélection, il n'est pas nécessaire d'ajouter les filtres "site" et "déposant". En effet lors du calcul de la rubrique, Warehousing ajoute automatiquement ces filtres en fonction du dossier de facturation (1 dossier = 1 site/déposant).

Rubrique de type "2-Statistique"

Au contraire d'une rubrique de type "Table", Il faut savoir qu'une rubrique de type "Statistique" sera obligatoirement définie pour un dossier de facturation donnée. Cela est dû à la structure actuelle des paramètres statistiques.

Dans ce type de rubrique, il est impératif que le paramètre statistique utilisé comporte les axes de recherche "site" et "déposant" sinon on ne pourra pas filtrer le résultat de la statistique pour un site/déposant donné. On a 2 options possibles pour définir la statistique:

  • Option 1: le "site" est défini comme critère dans le déclencheur statistique. Alors au niveau du paramètre statistique, il suffit juste d'ajouter le champ "déposant".
  • Option 2: le "site" n'est pas défini comme critère dans le déclencheur statistique. Alors au niveau du paramètre statistique, il faut ajouter les champs "site" et "déposant".

Aprés au niveau des "critères" de sélection de la rubrique:

  • Option 1: pas besoin de mettre le "site", Warehousing ajoute automatiquement ce filtre en fonction du dossier de facturation. Par contre il faut impérativement ajouter manuellement le "déposant".
  • Option 2: il faut impérativement ajouter manuellement le "site" et le "déposant".

De plus il faut filtrer les critères de la statistique en sachant que la valeur "$$$" signifie qu'on prend toutes les valeurs et si pas de filtre on prend par défaut les valeurs à blanc.

Prenons l'exemple suivant: soit la statistique STOCK donnant par déposant (CRI1), article (CRI2), nature de stock (CRI3), numéro de lot (CRI4), le stock UC (QTYUC). Supposons que l'on est en stock pour l'article ITEM:

Descriptions des champs

Critères

champ IHHSEL

Dans le cas d'une rubrique de type '1-Table', ce champ permet d'indiquer les critères de sélection sur la requete définie par les tables primaire/secondaires.

Formule

champ IHHFOR

Dans le cas d'une rubrique de type '1-Table', ce champ permet d'indiquer la formule de calcul applicable sur la rubrique. Cette formule s'appuie sur tout champ des tables primaire/secondaires.

Critères stat

champ IHHSTASEL

Dans le cas d'une rubrique de type '2-Statistique', ce champ permet d'indiquer les critères de sélection sur les champs statistiques nommés CRIx (x = 1 à 5 maximum).

L'aide de saisie (accessible par bouton droit) permet de donner la signification des variables CRIx.

Tableau Affectation

Site (champ IHPFCY)

Code du site pour lequel la rubrique est utilisable. Cela permet entre autre d'autoriser ou non des rubriques sur tout site du dossier X3.

Par défaut, si aucun site n'est précisé, alors la rubrique est utilisable sur tous les sites du dossier X3. Par contre si des sites sont précisés, alors la rubrique ne sera utilisable que sur ceux mentionnés.

Boutons spécifiques

  • Valider
  • Copie

Valider

Ce bouton permet de générer les traitements automatiques liés a cette rubrique. Cette validation est normalement enchaînée à la modification ou à la création d'un code. Elle n'est donc nécessaire que si les traitements ont été supprimés, ou si le paramétrage a été créé par copie à partir d'un autre dossier (dans ce cas, la validation n'est pas faite automatiquement).

Copie

Permet de copier une rubrique vers un autre dossier X3.

Remarque: on ne copie pas la liste des sites autorisés. Si première copie (la rubrique n'existe pas dans le dossier de destination), on autorise cette rubrique sur tous les sites, si Xième copie (la rubrique existe déjà dans le dossier de destination), on laisse les autorisations existantes dans le dossier destination.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur zone "Date": Type de données incorrect

Ce champ autorise uniquement des champs de type Date.

Erreur zone "Valeur statistique": Zone obligatoire

Outre le fait d'avoir ce message si aucune valeur n'a été saisie, cela peut se produire aussi lors de la tentative de sélection de cette valeur par click droit. Dans ce cas cela est dû au fait que le paramètre statistique a un code différent de celui du déclencheur statistique.
