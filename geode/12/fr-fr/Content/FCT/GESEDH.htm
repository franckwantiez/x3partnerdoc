Saisie

Cette fonction permet:

  • de créer manuellement un message EDI de type retour.
    Un message de type retour est au départ du prestataire de transport et à destination de Warehousing.
    Cette fonction permet en particulier de solder un BL avec un statut EDI ‘en-cours’.
    Elle est utilisée dans le cas où le dernier message n’est pas transmis par EDI mais par un moyen manuel : par fax ou par téléphone.
  • de visualiser le détail des messages EDI créés manuellement ou par interface
    Pour les messages EDI de type colis et BL, la loupe permet de visualiser l’ensemble des messages EDI reçus pour l’entité.
  • d’éditer l’état statistique transport qui récapitule le nombre de BL expédiés et gérés par l’EDI

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

  • En-tête
  • Onglet Message

L’écran est constitué de deux onglets:

  • un onglet entête décrivant l’entité traitée dans le message EDI
  • un onglet décrivant le détail du message EDI

En-tête

Il permet de définir le type et le numéro de l’entité traitée par le message EDI

Les messages EDI peuvent concerner quatre types d’entité différents:

  • Le Récépissé Transport (RT)  
    Il est défini dans le message à partir de son numéro de RT.
  • Le Bon de livraison (BL)
    Il est défini dans le message  à partir du déposant (information facultative) et de son numéro de BL
  • L’unité de manutention (UM)
    Elle est définie dans le message à partir du déposant et de son numéro UM
  • Le colis 
    Il est défini dans le message à partir du déposant (information facultative), du numéro de colis transporteur et/ou du numéro de colis.

A la création du message, le système contrôle que:

  • l’entité existe dans Warehousing
  • l’entité est expédiée pour les entités de type colis et UM
  • le(s) BL(s) associé(s) à l’entité est géré par EDI
  • l’entité n’a pas reçu son message de fin

Numéro du message EDI.

Séquence numérique attribuée automatiquement par le système lors de la

création d'un message EDI soit par interface soit manuellement.. Ce numéro est une information propre au

site, tous déposants confondus. Cette information constitue une clef

unique d'accès au message EDI.

Indique le type d'entité gérée dans les messages EDI.

L'entité traitée peut être soit le RT, soit le BL, soit l'UM ou soit le colis.

Onglet Message

Il permet de définir le détail du message EDI 

L’opérateur doit renseigner obligatoirement :

  • les codes statut et raison
    Les codes statut et raison doivent exister et leur correspondance également
    Le système détermine alors :
    - la phase du message à partir du code statut et de la table des codes statuts
    Le système contrôle que la phase du message est supérieure ou égale à la phase du dernier message reçu pour l’entité
    - la gravité du message à partir du code raison et de la table des codes raison
    - si le message est le dernier à recevoir pour l’entité à partir du couple (code statut, code raison) et de la table des correspondances.
    Pour que le message permette de solder le BL, le couple (statut, raison) doit être de type ‘message de fin’.
  • La date et heure de l’événement
    La date et heure de l’évènement doivent être cohérentes et supérieures à celles du dernier message reçu pour l’entité
  • Un code opérateur en fonction du paramétrage de la table des sites
    L’opérateur devra alors saisir un code opérateur habilité à gérer l’EDI transporteurs pour pouvoir valider la création manuelle du message EDI.

L’opérateur peut renseigner aussi:

  • la date et heure de traitement du message
  • la date et heure du prochain message

A la validation du message EDI, le système:

  • met à jour la date et heure de création du message
  • affecte un numéro au message EDI
  • pour un message de type RT, crée un message avec les mêmes caractéristiques pour tous les BL du RT car le BL est plus petite entité remontée en gestion sans colisage.
  • pour un message de type UM, crée un message avec les mêmes caractéristiques pour tous les colis de l’UM car le colis est plus petite entité remontée en gestion de colisage.
  • Met à jour le(s) BL(s) lié(s) à l’entité traitée dans le message EDI 
    - mise à jour du BL avec les informations du dernier message avec la gravité la plus forte des entités liées au BL
    - mise à jour à l’état ‘terminé ‘ de l’indicateur ‘gestion EDI’ du BL s’il est totalement livré : réception de tous les messages de fin des entités liées au BL.
  • envoie un workflow à destination du déposant, du transporteur et/ou du client en fonction du paramétrage du couple (code statut, code raison) de la table des correspondances.
Descriptions des champs

Bloc numéro 1

Phase (champ ORN)
Code statut (champ COD1)

Il décrit la situation de l'entité (Enlevé, livraison en cours...)

Les statuts sont définis pour l’ensemble des transporteurs et leur codification est définie à chaque accord d’interchange avec le transporteur.
Gravité (champ NUMERR)
Code raison (champ COD2)

Le code raison donne une justification ou une explication autour d’un statut (Exemple : manquant, détérioré...)

Ils sont définis lors de l’accord d’interchange avec le transporteur
Date évènement (champ CLTDAT)

Date et heure de l'évènement traité dans le message EDI.

Lors de l'intégration des messages EDI, des contrôles de cohérence sont réalisés entre les dates et heures évènement du nouveau message et des messages déjà reçus pour l'entité.

La date et heure évènement du nouveau message doivent être inférieures à celles des messages déjà reçus.
Heure (champ CLTTIM)
Fin (champ LASREC)
Date traitement (champ CRRDAT)

Date et heure de traitement de l'évènement décrit dans le message EDI.
Heure (champ CRRTIM)
Prochain message (champ NEXDAT)

Date et heure du prochain message à recevoir pour l'entité traitée dans le message EDI
Heure (champ NEXTIM)

Etats

Par défaut, les états suivants sont associés à la fonction :

  EXPEDI : Statistiques transport

Mais ceci peut être modifié par paramétrage.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur zone ‘Déposant’, Fiche inexistante
La valeur saisie doit exister dans la table des déposants pour le site courant

Erreur zone ‘N° RT’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘N° RT’, RT non déterminé
La valeur saisie doit exister dans la table des RT pour le site courant.

Erreur zone ‘N° RT’, Niveau de gestion EDI : Colis ou UM  ou BL
Le RT saisi est déjà géré en EDI dans un niveau différent : colis, UM ou BL

Erreur zone ‘N° BL’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘N° BL’, BL non déterminé
La valeur saisie doit exister dans la table des BL pour le site courant.

Erreur zone ‘N° BL’, Gestion EDI <> En cours pour le BL
Le BL saisi est totalement livré ou non géré par l’EDI.

Erreur zone ‘N° BL’, Niveau de gestion EDI : Colis ou UM  ou RT
Le BL saisi est déjà géré en EDI dans un niveau différent : colis, UM ou RT

Erreur zone ‘N° UM’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘N° UM’, UM non déterminée
La valeur saisie doit exister dans la table des UM pour le site courant.

Erreur zone ‘N° UM’, UM non expédiée
L’UM saisie n’est pas encore expédiée (UM à l’état 6, 7 ou 8)

Erreur zone ‘N° UM’, Gestion EDI <> En cours pour le BL
Le BL de l’UM saisie est totalement livré ou non géré par l’EDI.

Erreur zone ‘N° UM’, Niveau de gestion EDI : Colis ou BL ou RT
L’UM saisie est déjà gérée en EDI dans un niveau différent : colis, BL ou RT

Erreur zone ‘N° colis’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘N° colis’, colis non déterminé
La valeur saisie doit exister dans la table des colis pour le site courant.

Erreur zone ‘N° colis’, colis non expédié
Le colis saisi n’est pas encore expédié (colis à l’état 4, 6, 7 ou 8)

Erreur zone ‘N° colis’, Gestion EDI <> En cours pour le BL
Le BL du colis saisi est totalement livré ou non géré par l’EDI.

Erreur zone ‘N° colis’, Niveau de gestion EDI : UM ou BL ou RT
Le colis saisi est déjà géré en EDI dans un niveau différent : UM, BL ou RT

Erreur zone ‘Code statut’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘Code statut’, Fiche inexistante
La valeur saisie doit exister dans la table des statuts pour le site courant

Erreur zone ‘Code raison’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘Code raison’, Fiche inexistante
La valeur saisie doit exister dans la table des raisons pour le site courant

Erreur zone ‘Code raison’, couple inexistant
Le couple (code statut, code raison) saisi doit exister dans la table des correspondances pour le site courant

Erreur de l’application, zone obligatoire
La valeur saisie doit être différente de blanc
Ce message s’applique pour la zone ‘Heure évènement’

Erreur de l’application, Heure incorrecte
La valeur saisie doit être au format heure
Ce message s’applique pour les zones ‘Heure évènement’, ‘Heure de traitement’ et ‘Heure prochain message’

Erreur zone ‘Code opérateur’, zone obligatoire
La valeur saisie doit être différente de blanc

Erreur zone ‘Code opérateur’, Code opérateur invalide
La valeur saisie doit exister dans la table des codes opérateurs et doit être habilitée à réaliser l’EDI transporteurs.

Erreur de l’application, Phase incohérente avec le dernier message
La phase du message doit être supérieure ou égale à celle du dernier message reçu pour l’entité traitée

Erreur de l’application, Message antérieur au dernier message
La date et heure évènement du message doivent être supérieures à celle du dernier message reçu pour l’entité traitée

Erreur de l’application, Dernier message déjà reçu
L’entité traitée a déjà reçu son message de fin
