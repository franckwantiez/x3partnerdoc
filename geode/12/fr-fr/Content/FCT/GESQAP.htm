Critères affectation colis

Cette table permet de déterminer le quai et les attributs de mise à quai du colis lors de sa validation (création colis en déclaratif ou validation colis en précolisage).

Les critères d'affectation sont:

  • Code mouvement: (optionnel)
  • Mode de préparation: (optionnel)
  • Zone de préparation: (optionnel)
  • Gamme de colis: (optionnel)
  • Type de colis: (optionnel)
  • Tournée générique: (optionnel)
  • Code Transporteur: (optionnel) zone non saisissable si une tournée générique est renseignée
  • Mode Transporteur: (optionnel) zone non saisissable si une tournée générique est renseignée
  • Expédiable en état: 3 options possibles
    • "Oui": on considère que les colis expédiables en état
    • "Non": on considère que les colis non expédiables en état
    • "-": on considère tous les colis (expédiables en état ou pas)

les attributs déterminés sont:

  • Magasin/adresse: l'adresse où sera transféré le colis
  • Validation: le mode de validation de mise à quai du colis
  • Ctrl mise à quai: active ou non le contrôle du statut mise à quai des colis lors de toutes opérations.

Gestion de l'écran

L'écran est constitué d'un tableau simple.

Ecran de saisie

Définition des critères d’affectation de quai pour le colis.
Le N° de séquence est obligatoire et il est recalculé systématiquement en appliquant un pas de 10 pour permettre les insertions ultérieures.
Le N° de séquence est significatif dans la recherche du quai.
Algorithme de détermination du quai :
On va considérer les lignes de la table dans l’ordre des séquences. Si chaque critère saisi de la ligne a une valeur identique à celle du colis, le quai et ses attributs sont trouvés sinon on considère la ligne suivante.

Remarque: la tournée générique, le code transporteur et le mode transporteur sont déterminés de l'OL associé au colis.

Descriptions des champs

Tableau

Code qui représente le mode de préparation. Le mode de préparation regroupe l’ensemble des règles de gestion lié &agra ve; chaque O.L. Il permet d’en déterminer le mode de traitement lors du lancement en préparation.

Code de la gamme colis. La gamme constitue la clé d’accès principale à la définition des colis eux-mêmes. Elle permet de disting uer, pour des familles d’articles données ou dans le cas de flux particuliers (export, ....), le type d’emballage préférentiel qui sera retenu.

La tournée générique est une tournée de livraison « fixe » (par rapport aux jours, à l’heure limite d&rsq uo;expédition), une tournée étant le contenu d’un camion.

Indique si le colis est un contenant expédiable en l’état (le contenant est expédié tel quel).

Adresse de l’emplacement dans le magasin.

Séquence (champ ORN)

Numéro permettant d'utiliser les règles définies dans cet écran par

ordre de priorité.
Code mvt (champ MVT)

Le code mouvement permet de différencier les mouvements de même type.
Il est requis lors de la création :

  • d'une entrée attendue,
  • d'une entrée fournisseur,
  • d’une entrée usine,
  • d’un retour client,
  • d'un O.L,
  • d'un ajustement,
  • d'un transfert,
  • d'une entrée ou sortie de consommable.

Le code mouvement est vérifié par rapport à la table des codes mouvements.
Mode prépa (champ PMD)
Zone prépa (champ PKZ)

Code de la zone de préparation qui est vérifié dans la table des zones de préparation. La notion de Zone de Préparation est utilisée pour le découpage des Ordre de Prélèvement.
Gamme (champ PRA)
Type (champ PACTYP)

Code du type colis. Le type permet de définir, pour une gamme de colis considérée, 1 à n dimensions de colis.
Tournée G (champ GDR)
Transporteur (champ CRR)

Code identifiant le transporteur. Ce code doit être défini dans la table des transporteurs.
Mode transport (champ CRRMOD)

Généralement, le mode de transport correspond aux notions de messagerie, affrètement, express, ... Le mode transport est vérifié par rapport à la table des modes de transport.
Expé en état (champ DSTFLG)
--> (champ ATEXTE1)
Magasin (champ STOQAT)

Code permettant d’identifier le magasin. Il est vérifié dans la table des magasins.
Adresse (champ ADDQAT)
Validation (champ QATVLDAUT)

Indique le mode de validation de l'affectation colis/UM:

  • Validation "Immédiate": Lors de l'affectation, les colis/UM prennent directement le statut "A quai".
  • Validation "Différée": Lors de l'affectation, les colis/UM sont mis en transfert, i.e. en statut "En transfert".
Ctrl mise à quai (champ FLGCTLQAT)

Indique le type de contrôle qui sera effectué sur le colis/UM avant toute opération (palettisation, dépalettisation, ouverture et fermeture UM, Maj colis/UM...):

  • "Oui": le colis/UM devra toujours être en statut "A quai" pour pouvoir faire toute opération dessus.
  • "Non": aucun contrôle sur le statut de mise à quai ("A quai", "En transfert"). Toutes les opérations sont réalisables quelquesoit le statut de mise à quai.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Zone non saisissable si la tournée générique est saisie

la zone du tableau ne peut avoir une valeur tant que la zone "Tournée générique" est renseignée.
