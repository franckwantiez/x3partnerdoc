Correspondances rompus

Cette fonction permet de créer, visualiser, modifier et supprimer les équivalences définies pour une palette.

Les équivalences de rompus sont utilisés dans les traitements d'adressage en entrée et de transfert. Elles permettent d'adresser une palette rompue non pas dans un emplacement de même contenant mais dans un emplacement de contenant de référence plus petit.

Les contenants équivalents devraient avoir le même format de base que le contenant de départ et une hauteur inférieure.

Un mvt d'entrée est considéré comme rompu ou pas selon le paramètrage "% rompu" du contenant article.

La correspondance des rompus nécessite obligatoirement d'avoir la correspondance des contenants pour fonctionner (tout cela étant du au fait que le calcul de l'occupation d'un objet de stock dans un emplacement se fait à partir de la correspondance des contenants).

Exemples: soit les correspondances de rompus suivantes
1 PP1  80% = PP2  => 1 PP1 rempli à 80% est équivalent à une PP2
1 PP1  50% = PP3  => 1 PP1 rempli à 50% est équivalent à une PP3
Soit le plan de pal de l'article: 1 PP1 = 10 CC1 = 100 uc
Cas 1: mvt d'entrée 1 PP1 de 100uc,  c'est un mvt standard (non rompu), donc pas de "corresp. de rompus".
         => C'est le process d'adressage standard actuel (Recherche emplacement de type PP1 ...)
Cas 2: Mvt d'entrée 1 PP1 de 90uc, là le mvt est rompu, on peut donc utiliser la "corresp. de rompus". 1 mvt PP1 90 uc représente donc un % d'équivalence avec le contenant standard (1 PP1 de 100uc) de 90%. Donc pas d'équivalence (80% < 90%)
         => Recherche emplacement de type PP1... (process standard)
Cas 3: Mvt d'entrée 1 PP1 de 70uc, là le mvt est rompu, on peut donc utiliser la "corresp. de rompus". 1 mvt PP1 70 uc représente donc un % d'équivalence avec le contenant standard (1 PP1 de 100uc) de 70%. Donc équivalence possible avec PP2
         => Recherche emplacement de type PP2 (le contenant équivalent le plus proche du mvt d'entrée tout en étant supérieur). Si pas d'emplacement trouvé alors on dégrade.
         => Recherche emplacement de type PP1... (process standard)
Cas 4: Si mvt d'entrée 1 PP1 de 40uc => % d'équivalence de 40%. Donc équivalence possible avec PP3 puis PP2.
         => Recherche emplacement de type PP3. Si pas d'emplacement trouvé alors on dégrade.
         => Recherche emplacement de type PP2. Si pas d'emplacement trouvé alors on dégrade.
         => Recherche emplacement de type PP1... (process standard)

Gestion de l'écran

Pour créer une correspondance d’objets rompus, on renseigne les différents champs du tableau, avec les informations suivantes:

  • Contenant: Le contenant est un élément du plan de palettisation d'un article. Pour chaque article, la composition du ou des plans de palettisation dans lesquels l'article peut être décrit doit être connue du système. Un plan de palettisation peut être constitué de 5 niveaux de contenants (liés au type), hiérarchiquement imbriqués (principe des poupées russes).
    Un contenant unique de type 'UC' doit obligatoirement être défini pour chaque article. Tout contenant article, hormis le niveau 'UC', doit être décrit en fonction du contenant qu'il contient directement (contenant inférieur).
    Le contenant inférieur a un niveau de contenant supérieur au contenant décrit. Tous les niveaux ne sont pas obligatoirement présents dans la description d'un contenant.
  • % d’UC équivalent: Pourcentage appliqué au nombre d'UC du contenant standard en deçà duquel la règle d'équivalence peut être appliquée.
  • Equivalent: Contenant équivalent au contenant d'entrée quand celui-ci contient un nombre d'UC inférieur ou égal au pourcentage du contenant standard associé.

Messages d'erreur

  • Messages génériques

Outre les messages génériques, les messages d'erreur suivants peuvent apparaître lors de la saisie :

Erreur Contenant, Fiche inexistante.

On a tenté de saisir un contenant inexistant.

Erreur % d’UC équivalent, Zone obligatoire.

Le % d’UC équivalent doit obligatoirement être renseigné.

Erreur % d’UC équivalent, Nombre incorrect.

Le % d’UC équivalent doit entre compris entre 1 et 100 inclus.

Erreur Contenant, Zone obligatoire.

Le contenant doit obligatoirement être renseigné.

Erreur Equivalent, Correspondance entre 2 mêmes contenants.

On a tenté de saisir 2 mêmes contenants pour une même règle.

Erreur Equivalent, Correspondance entre 2 mêmes contenants.

On a tenté de saisir une règle d’équivalence avec des contenants de différents niveaux.

Même couple de contenants que la ligne N

La même règle d’équivalence a été saisie deux fois sur deux lignes différentes du tableau.
