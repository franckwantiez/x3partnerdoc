X3 Warehousing

Consultation

Cette fonction permet de consulter les données stockées dans la table interface (INTERLOT).

Dans un premier écran, on a une vision globale des interfaces (nbre de lots par interface et par état) pouvant être filtré par Site, Déposant et borne de dates.

Par le bouton "Détail", on peut acceder à nouvel écran permettant de voir ces interfaces classées par site, déposant, interface et état. De façon détaillée, il est possible de consulter les interfaces groupées par lots et ensuite par lignes d’interface. Un lot est un ensemble cohérent de lignes interface (Ex: Entête + lignes OL).

A partir de cette consultation, il est également possible de consulter les messages d’anomalies générées par le traitement (table INTERMES).

A un instant t, les lots (un entête et lignes correspondantes) peuvent prendre 4 états :
A Traiter :
En import, le lot a été chargé correctement techniquement mais n’a pas encore été intégré (le traitement INTIMPORT le mettra à jour).
En export, le lot n’ pas encore donné lieu à un fichier ou ne doit pas donné lieu à la génération d’un fichier (le traitement INTERFIC le mettra à jour).

Traité :
En import le lot a été correctement intégré fonctionnellement et l’objet a été créé dans Warehousing.
En export, le lot a donné lieu à un fichier ou a été traité par le N3.

En erreur :
En import, le lot n’a pas été intégré fonctionnellement dans Warehousing.
En export, le lot n’a pu donner lieu à un fichier.

En erreur traité :
En import, les lots qui étaient en erreur ont été mis à jour manuellement par cette fonction, de manière à ne plus en tenir compte au cours d’intégration ultérieure

Le premier écran:
aprés avoir indiqué ou non un site, un déposant et/ou une borne de dates, les données existantes sont affichées sous forme de tableau + graphique.

Le second écran:
A partir des résultats du premier écran, on peut avoir un détail où les données existantes sont affichées sous forme de tableau dont les 3 onglets détaillent les éléments:
- L’onglet Interfaces affichent une ligne par code Interface et Statut, ce qui permet de visualiser rapidement si il y a des erreurs (en particulier sur les imports).
- L’onglet Lots liste pour l’interface sélectionnée précédemment les lots correspondants.
- L’onglet Lignes liste les champs d’un lot précédemment sélectionné.
- L’onglet Dénombrement affiche un récapitulatif des flux échangés.

En-tête

Lors de la sélection d’un enregistrement dans l’onglet Interfaces, les données sont affichés dans l’entête. Il s’agit des informations du premier enregistrement du lot de la ligne, les autres informations sont alors visualisables dans l’onglet Lots.

Interface

Site, déposant, code interface, nom du paramétrage sont affichées. L’état de l’interface est également affiché.

Lot

La référence est l’identifiant du lot ; en règle générale il s’agit de la référence entrée attendue, référence entrée directe, référence ordre de livraison, code article.

Le numéro fait référence à l’identification de chaque enregistrement. Le fichier fait référence au comptera de fichier généré ou intégré. Si il est non renseignée, cela signifie que l’enregistrement est dans un état « A traiter ».

La date et heure de création sont relatifs à la date d’intégration des données dans la table interface. La date et heure de modification sont relatifs à l’intégration des données dans les tables vives.

Autres conditions

Le processus de fonctionnement des interfaces est assuré par un traitement unique appelé : INTERFACE. Ceci est valable depuis le patch n°5 correspondant à la refonte des interfaces. Auparavant, 2 traitements séparés : INTERFIC et INTIMPORT remplissaient respectivement les fonctions de chargement ou génération de fichier et intégration des données.

L'algorithme qui suit donne les principales actions et contrôles effectués en fonction du mode de l'interface :

Import

Pour chaque fichier détecté dans le répertoire d'import (IMPREP) :

  •  Vérification si le nom du fichier se trouve parmi les paramétrages interface actives

Si Oui,

  • Déplacement du fichier dans le répertoire de traitement d'import (IMPTRTREP)
  • Contrôle technique de chaque zone d'enregistrement en fonction du modèle d'import associé à l'interface

Si contrôle OK,

  • Chargement des zones de chaque enregistrement du fichier dans des tables interfaces (WI*).
  • Déplacement du fichier dans le répertoire de destination final (IMBAKPREP)
  • Ajout d'une ligne horodatée dans le fichier trace ("Import du fichier…") ainsi qu'une ligne pour chaque lot identifié ("Traitement du lot…").
  • Ajout d'une ligne dans la table des lots interface (INTERLOT) avec le statut "A traiter".

Sinon

  • Ajout d'une ligne dans la table des lots interface (INTERLOT) avec le statut "Erreur import".

Sinon

  • Déplacement du fichier dans le répertoire d'erreur (IMPERRREP)

Pour chaque lot interface en import avec comme statut "A traiter" :

Pour chaque ligne du lot dans les tables interfaces :

  • Chargement des valeurs dans des masques de saisie pour chaque zone renseignée
  • Si mode création, affectation des valeurs par défaut
  • Contrôle fonctionnel de chaque zone saisie

Si contrôle OK,

  • Validation de la transaction
  • Maj du lot interface avec le statut "Traitée"
  • Ajout d'une ligne horodatée dans le fichier trace ("Traitement du lot…")

Sinon,

  • Abandon de la transaction
  • Maj du lot interface avec le statut "Erreur intégration".
  • Ajout d'une ligne dans la table des messages d'erreur (INTERMES) pour chaque zone en anomalie.

Export

Suite à chaque validation de traitement (Entrée, Sortie, Transfert, Réappro,…) :

  • Ajout d'une ligne dans les tables interfaces (WI*) pour chaque enregistrement concerné.
  • Ajout d'une ligne dans la table des lots interface (INTERLOT) avec le statut "A traiter".

Si gestion par fichiers plats,

  • Incrémentation du compteur défini au niveau de l'interface ou à défaut celui rattaché à la table des lots interface

Pour chaque lot interface en export avec comme statut "A traiter" :

Pour chaque ligne du lot dans les tables interfaces :

  •  Ajout d'un enregistrement dans un fichier interface situé dans le répertoire de traitement export (EXPREP2)

Si écriture OK,

  • Déplacement du fichier interface dans le répertoire export (EXPREP)
  • Copie du fichier dans le répertoire de sauvegarde (EXPBAKREP)
  • Maj du lot interface avec le statut "Traitée".
  • Sinon, Maj du lot interface avec le statut " Erreur export ".
