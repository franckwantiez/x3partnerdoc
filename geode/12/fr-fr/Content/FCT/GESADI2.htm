Tables diverses - Données

Les tables diverses sont des tables annexes utilisables dans tout le contexte du progiciel pour y stocker des codes et un nombre restreint d'informations rattachées. Par exemple, on y stockera les régimes de taxe, les modes de règlement, les motifs d'impayés…

Ces tables banalisées, identifiées par un numéro, permettent de stocker les éléments suivants :

    des codes qui servent d'identifiants, et qui seront stockés dans les champs utilisant le type de données ADI. Le nombre de carctères peut être rendu paramétrable par l'utilisateur sur certaines tables diverses, dans la définition de la table. L'utilisateur définira alors la longueur de ces codes par le biais d'une fonction dédiée.

    les intitulés longs et courts correspondants. Ces intitulés sont traduisibles. Les textes sont stockés dans la table ATEXTRA.

    un lien de dépendance éventuel vers une clé d'une autre table diverse (pour gérer, par exemple, le fait qu'une sous-famille ne puisse être rattachée qu'à une famille donnée).

    de 1 à 4 zones libres.

La signification d'un certain nombre de ces tables est prédéfinie dans le progiciel, mais il est possible d'ajouter des tables dans le cadre d'un vertical ou d'un spécifique. Les tranches réservées sont définies en annexe de cette documentation.

Ces tables diverses sont, pour des raisons techniques, stockées dans une seule table de la base de données. La saisie de valeurs dans ces tables diverses est assurée par cette fonction.

Les tables 900 à 999 sont relivrées systématiquement par le standard. De ce fait, s'il est nécessaire d'ajouter des enregistrements dans ces tables, ils doivent être avec un identifiant commençant par X, Y ou Z pour être conservés. Sans cela, ces enregistrements seront supprimés lors de la prochaine validation de dossier.

Pré-requis

Reportez-vous à la documentation de Mise en oeuvre

Gestion de l'écran

Ecran de saisie

Une fois le numéro de la table saisi, le tableau des codes existants est chargé, et on peut saisir ligne à ligne les codes, les intitulés long et court correspondants, les valeurs complémentaires associées si nécessaire, et le code de la famille liée (ces colonnes n'apparaissant que si des champs complémentaires et / ou un lien de dépendance ont été définis). Ces valeurs complémentaires peuvent être sélectionnées et contrôlées.

Lorsqu'on appelle cette fonction directement en tunnel depuis la saisie d'une valeur de code, le numéro de la table est imposé. On ne peut alors pas le changer.

Descriptions des champs

Bloc numéro 1

Saisissez la description de la fiche concernée.

Cet intitulé long est utilisé en titre dans les écrans et les états.

Table (champ NUMTAB)

Ce numéro identifie la table diverse. Un certain nombre de numéros sont prédéfinis dans le progiciel, mais il est possible d'en ajouter, par développement spécifique, dans des plages de numéros réservées à cet effet.
Intitulé (champ ZINTIT)
Table dépendance (champ DEPNUM)

On indique ici le numéro de table diverse qui sert de référence pour saisir la colonne Dépendance. Lors de l'utilisation de la table diverse, ceci impose une dépendance entre deux champs saisis par les tables diverses liées.

Code (champ CODE)

Liste des codes propres à chaque numéro de table. Ces codes servent d'identifiant au sein de chaque table.
Intitulé (champ LNGDES)

Cet intitulé, associé au code de la table, est saisi par défaut dans la langue de connexion, mais il est traduisible dans d'autres langues. En effet, par un clic droit sur le champ, on peut ouvrir une fenêtre permettant de saisir une liste de codes langues et d'intitulés associés. Sinon, la fonction de traduction des textes permet, en choisissant le chapitre Tables diverses, de réaliser une traduction dans toutes les langues des textes ainsi choisis.

Valeur par défaut (champ VALDEF)

Cette valeur, choisie parmi les codes disponibles de la table, est proposée par défaut lors de la saisie d'un champ obligatoire, contrôlé par cette table diverse.
