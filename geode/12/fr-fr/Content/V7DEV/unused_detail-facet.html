Detail facet

How to organize tabs and blocks?

The information in version 6 can be reorganized if you need to make it clearer and easier to understand.
Titles are mandatory in version 7 while they were not in version 6. Be aware of the rules for titles to avoid redundancy of texts. For example:

  • Adherence to a single vocabulary.
  • Usage of accurate and clear text.

What are the contents?

Property list

Same list of properties as you have in version6 are available for version 7.
Check the Property values displayed.
If a property needs a script to be displayed, refer to the PBA.

Key

This is the key value for a detail facet.

Reference

The underlined value allows the navigation to the Detail facet of the corresponding menu entry.

In the menu of this field, “List” allows for the navigation to the Query facet of corresponding menu entry.

The title of the reference is displayed if it exists.
Information will be provided soon for references composed of several fields.

One item grid

The grid is a horizontal display of the list of values for a property.
A reference is managed as described above (detail, list, and title).

Several items grid

Display the columns you have in version 6.
A reference is managed as described above (detail, list, and title).
Imminent information will be provided for reference composed by several fields.

What are the links on the page?

  • List allows the navigation to the Query facet.
  • Modify allows the navigation to the Classic function in modification mode.

Functional links

If links are needed, they should be defined by the PBA, for:

  • Fields.
  • Pages.

Regarding links, the naming standard is the following:
* use a verb for an action;
* otherwise, use a noun.
