Special cases

Workflow signature

When you click on a link in an email to approve or deny, an action that performs a signature after authentication is executed.

For this operation, the required badge can be any badge. ERPTRAN is enough for workflow signatures.

The connection to Sage X3 ends after the standard timeout set for web sessions. The corresponding badge consumption remains until the timeout has elapsed. This might create an over-consumption of badges in concurrent mode if many end-users sign in during a short period of time. This has been fixed in release 2019 R2. This has also been fixed in version 11.15 and update 9.15. If you click on a link that opens a function protected by a badge, the badge is consumed.

Batch server

The batch server is controlled by the Syracuse server. It must be run by a user with administration rights.

In the administrator security profile (Administration > Administration > Users > Security profiles), the administrator must have technicalSettings or StatusAndUsage authorizations.

In this example the user is BatchAdmin and the role is BatchRole:

  • BatchAdmin is not used at execution time for the batch tasks that have been requested by users.
  • The user is given at task submission and the corresponding rights are used.
  • The transferred badge rights are the ones granted to BatchRole.
  • BatchRole must therefore have a badge that allows executing all existing batch tasks.

If you list all these tasks, you notice that none require a dedicated badge to run. This means that a dedicated user and role consuming only an inexpensive badge can be used to run the batch server. ERPDEV is not necessary to run the batch server.

Multiple devices, browsers and tabs in concurrent licensing mode

Single browser and multiple tabs

Regardless of the fact that the tabs are in the same browser window or not:

  • Badges associated to a given user or role are only consumed once regardless of the number of opened tabs.
  • Each tab opens one session with its associated resources such as the memory and CPU.

Multiple browsers

If you access Sage X3 through multiple browsers such as Chrome and Internet Explorer in parallel:

  • A set of badges is consumed with a limit of five browsers.
  • An additional badge consumption is added for every five additional browsers.
  • The office client is counted as a browser. It uses a browser similar to Edge.

Multiple devices

Every connection consumes a dedicated set of badges when several devices (mobile, tablet and desktop) connect simultaneously with the same user.
