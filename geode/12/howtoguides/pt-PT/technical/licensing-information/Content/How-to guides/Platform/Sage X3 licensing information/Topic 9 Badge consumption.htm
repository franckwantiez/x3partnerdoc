Badge consumption

Presentation

Groups and roles define the relationship between users and badges:

  • A role is associated to one or more badges
  • A group is associated to a unique role
  • A user belongs to one or more groups
Standard roles are usually mono-badge. Some specific badges extend the solution scope such as ERPPSPLAN and ERPPSVIEW providing access to Production Scheduler.
If a user connects to a function controlled by a badge he is not associated to, a XXXX: access restricted by license warning message is displayed. In this case, he can display data in read-only mode but not edit it.

Rules

Rules for concurrent users

The consumption is triggered when the user connects.

For a multi-role context:

  • The user chooses a specific role.
  • Only the corresponding badges are consumed.

If all badges are in use, the user can't connect to Sage X3 functions. However, he can run administration functions.

Rules for named users

Badges are counted on the number of active users and their associated groups, roles and badges.

When the maximum number of badges is reached, a new user can't activate until an active user deactivates.

Concurrent users example

In this example, the user belongs to two groups:

  • Sales
  • Purchases

Each group is associated to a unique role:

  • Sales: Sales Manager
  • Purchases: Material Buyer

Each role is assigned to one or more badges (two in our case):

  • Sales Manager: ERPDIS and ERPFIN
  • Material Buyer: ERPTRAN and ERPDIS

When the user connects and selects the Material Buyer role, he consumes the two following badges:

  • ERPTRAN
  • ERPDIS

If the user switches to the Sales Manager role, he still consumes two badges, but not the same ones:

  • ERPDIS
  • ERPFIN
    • ERPTRAN is released.

Named users example

In some countries the licensing is by named users. When the user is set up as an active user, all badges assigned to him via his groups and roles are consumed. He also can't be assigned to another user.

The badges assigned to all roles belonging to groups that an active user is a member of are consumed.

When the licensed number of users is reached, additional users can't be activated until other users are deactivated.

In this example, the user consumes three badges as soon as he is declared in the system:

  • ERPDIS
  • ERPFIN
  • ERPTRAN

If ERPFULL is less expensive than ERPDIS and ERPFIN, it is better to purchase and assign an ERPFULL badge. This is not necessary if:
  • Many ERPDIS and ERPFIN badges are required.
  • This double consumption rarely happens.
As most of the functions associated to ERPTRANS are all included in ERPDIS, the ERPDIS badge might be enough. You can purchase the following badges:
  • ERPTRANS is the less expensive badge. You can use it to access all non-critical functions.
  • ERPDEV is the most expensive badge and gives access to any function. There is at least one granted for every license.
  • ERPPSPLAN and ERPPSVIEW are used to manage scheduling operations and are usually associated to an additional badge.
