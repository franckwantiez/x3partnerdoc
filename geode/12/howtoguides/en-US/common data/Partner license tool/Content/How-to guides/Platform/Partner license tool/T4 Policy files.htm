Policy files

Policy file management

A policy file cannot be edited or created until the partner file has been uploaded.

When you have uploaded the file and click Save, the + Create button will display:

You can either create a new policy file, or use an existing policy file as a template. Click the Actions icon next to Product code, then click Duplicate. For example, when you duplicate you can have several policy versions for the same product code/version.

A policy cannot be edited or deleted when at least one license file in the file system of the tool refers to it. When a license refers to a policy, the Referred check box will be selected.

Understanding how badges work

There are useful information in the Sage X3 online help center explaining how badges work, and giving information on the methods available to retrieve license information with Sage X3 code. You need to understand the concept of badges before creating a policy file. More information can be found here:

  • License information

Policy file content

Product code

An internal code will be given to your add-on. To ensure that the name is unique you can include your local identifier in the name. Alphanumeric characters only, without blank spaces, are authorized.

Add-on name

You have to provide the name of your add-on in the Title field in a free format. You can manage different translations of this name while adding new locales.

Product version

This field displays the Product version of the add-on to which the policy applies. This can be a numeric or an alphanumeric value.

Policy code

You can specify different parts of the same product in the Policy code field, and allocate different policy names to them.

Licenses for the same product with different policies, which have identical modules, are currently not catered for.

Policy version

There can only be one policy for each combination of four values. These values can only contain letters A-Z, a-z, digits 0-9, and an underscore (_). The product version and policy version may also contain dots.

Base product

You must specify a Base product. This is the product code (not product title) of the corresponding Sage X3 product. The license system of the application will only accept licenses which correspond to an installed Sage X3 product. The product code for Sage X3 = 1 and for Sage X3 HR & Payroll = 3.

Activity codes

You can specify any number of Activity codes with the following:

  • A code that starts with X, Y, or Z, and contains A-Z and 0-9.

  • A title that is user-friendly.

  • A condition with the following options:

    • Always valid

    • Never valid

    • Only valid when it contains the license

Badges

You can specify any number of Badges with the following:

  • A code that starts with one of the dictionary prefixes of the partner file, or with a Y.

  • A title that is user-friendly.

  • A list of functions which the badge should control, they are called critical functions.

You can separate the function names with a space and/or comma. Functions should be defined in the Functions function (GESAFC).

Only the list of functions should be defined in this function. The function name should start by your identifier, but it some cases it can start with SUBxxxx.
The Sage X3 license is loaded first, and any Sage X3 critical function that you may reference will be ignored.

Parameters

You can specify any number of Parameters with the following:

  • A code that starts with X, Y, or Z and contains A-Z and 0-9.
  • A title that is user-friendly.

Only numerical parameters are supported.

Languages

You can specify Languages with the following:

  • A code that consists of A-Z and 0-9.

  • A title that is user-friendly.

  • A condition with the following options:

    • Always valid

    • Never valid

    • Only valid when it contains the license

Legislations

You can specify Legislations with the following:

  • A code that consists of A-Z and 0-9.

  • A title that is user-friendly.

  • A condition with the following options:

    • Always valid

    • Never valid

    • Only valid when it contains the license

The tool does not check for name clashes with Sage X3 policies or other partner policies. If you want to control a parameter value, you will have to use the toolkit and control it in the code. The system will guarantee you that the value you checked is unchanged.

How to protect a Syracuse based function

When creating your policy, you can define key functions which are the only ones that are controlled. You will be able to only control the functions where the name starts with the identifier, which is defined in the partner file. To protect a Syracuse function, you must define a function in the Authorization field of the Representations function (GESASW).

You will be able to define this function (do not forget that the name should start by your identifier) in the policy file as a key function associated to one of your badges.
