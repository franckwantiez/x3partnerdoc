Managing documentation patches

How to edit elements to create the patch list

Use the Automatic patch creation function. This function is used to create a documentation patch list containing only the documentations modified or created at a given date and for a given activity code.

To access this function, go to Development > Utilities > Patches > Automatic patch creation, and select only the ADO, ADZ, and ADF codes.

See the Automatic patch creation (APATCHA) documentation.

On the first screen:

  1. Select the ADO, ADF, and ADZ codes.

  2. Enter the dates, activity code, and username (if needed).

  3. Validate.

On the second screen (the Number column displays the number of documentations to be patched):

  1. Select the Vertical check box in Type of patch.

  2. In the Patch field, enter the patch file name.

  3. If you want to create the file directly, select the Direct generation check box.

If this check box is not selected, a file containing only the structure of what is to be patched is created. The Patch creation function must then be used to create the patch list (see below).

In order to apply patches to vertical documentation, refer to the documentation of the following functions.

  • Automatic patch creation (APATCHA)

  • Patch creation (APATCH)

How to create a patch list

The procedure for a patch list creation is identical to the Automatic patch creation procedure detailed above, except for the list of documentations to patch which is not generated automatically (unless the Direct generation check box has not been selected in the Automatic patch creation function). You will have to enter the individual documentations to be integrated into the patch list.

If the Direct generation check box is selected, you only need to enter the name of the file containing the structure of the documentation. The list of the documentation to be patched is then displayed. The documentations that do not need to be delivered in the patch list can be removed.

This function is detailed in the Automatic patch creation (APATCHA) documentation.

  1. Go to Development > Utilities > Patches > Patch creation.

  2. Select the Specific or Vertical check box in Type of patch.

  3. Enter the path of the patch.

  4. Name the patch.

How to delete existing documentation in the destination folder

The integration of a documentation patch list overwrites existing documentation. Therefore, it is not necessary to delete the documentation before integrating the new documentation (except for vertical patch lists).

  1. Go to Development > Processing dictionary > Update transactions.

  2. For each transaction: ADF, ADO, ADZ.

How to incorporate the patch list

  1. Copy the patch list.

  2. Go to Development > Utilities > Patches > Patch integration.

  3. Go to Server.

  4. Enter the path to retrieve the patch list.

  5. Incorporate it to the directory.

The patch list integration is detailed in the Patch integration (PATCH) documentation.
