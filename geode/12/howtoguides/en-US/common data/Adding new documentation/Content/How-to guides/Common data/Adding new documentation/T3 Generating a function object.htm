Generating a function/object documentation

How to generate the structure of the documentation

The structure of the documentation is automatically created in the ADOCUMENT table, based on the information in the dictionary.

To generate the documentation structure:

  1. Go to Development > Data and parameters > Documentation > Documentation.

  2. Populate the Language field, the Type of documentation field and the Code field with the language code for the documentation, the type of documentation and the code of the documentation respectively.

    Example

    The code of the documentation is the code of the function or parameter to be documented.

  3. Click Generation in the Action panel. The Doc generation window (ADOCUMENT) opens.

  4. Selection block: Populate the Language field, the Type field and the Documentation codes fields (function or parameter to be documented, for example).

  5. Generate ADOCUMENT block: Select the All check box to automatically generate the documentation structure with all the paragraphs.

  6. Generate final document block: Select the Complete check box and the Field helps linked to documentations check box.

  7. Click OK. The generated paragraphs are displayed automatically in the Selection panel in the Current documentation list in the Documentation function (GESADO).

How to generate the documentation in HTM format

Documentation in HTM format can be generated from two locations:

  1. From the function dictionary:

    1. Go to Development > Script dictionary > Functions.

    2. In the Action panel, go to Documentation > Paragraphs.

  1. From the documentation menu:

    1. Go to Development > Data and parameters > Documentation > Documentation.

    2. Enter the Language field, the Type of documentation field and the Code field (function or parameter to be documented, for example).

Generate the final documentation in HTM format as follows:

  1. Click Generation in the Action panel. The Doc generation window (ADOCUMENT) opens.

  2. Selection block: Populate the Language field, the Type field and the Documentation codes fields (function or parameter to be documented, for example).

  3. Generate ADOCUMENT block: Select the All check box to automatically generate the documentation structure with all the paragraphs.

  4. Generate final document block: Select the Complete check box and the Field helps linked to documentations check box.

  5. Click OK. The HTM files are generated in the following folders:

    • FCT: HTM pages for functions.

    • OBJ: HTM pages for other objects such as parameters.

Table of documentation links

You can access the Documentation links function (GESADF) from the following locations:

  1. Whilst entering a documentation, go to Option > Links > Documentation links.

  1. From the function dictionary:

    1. Go to Development > Script dictionary > Functions.

    2. Enter the function code.

    3. In the Action panel, go to Documentation > Links.

  1. From your dashboard, go to Development > Data and parameters > Documentation > Documentation links.

How to create documentation links

You create or amend documentation links as follows:

  1. Enter the type of documentation (ADP, ADI, AML, etc.), the module and the function code in the header.

  2. Populate the table.

To navigate through the table, tabulate. To exit the table, press the ESC key.


The documentation in the links table will display in the correct paragraph of the help.

Example

In the documentation structure related to a function the paragraph PRD - Miscellaneous prerequisites (level 30/90) is mandatory. The documentation links will display in this paragraph in the final HTM file.

Entering documentation

You can access the entry function of the Documentation function (GESADO) from two locations:

  1. From the function dictionary:

    1. Go to Development > Script dictionary > Functions.

    2. In the Action panel, go to Documentation > Paragraphs.

  2. From your dashboard:

    1. Go to Development > Data and parameters > Documentation > Documentation.

    2. Enter the Language code, the Type of documentation (AFC, ADP, etc.) and the Documentation code fields.

      Example

      Function or parameter to be documented.

    3. Go to the Current documentation list in the Selection panel.

The Current documentation list displays all paragraphs for the current documentation. The paragraphs are organized by level and sublevel.

Example

The introduction paragraphs (Title and Presentation paragraphs) are level 10. The Title paragraph is sublevel 10, and the Presentation paragraph is sublevel 20, and so on.

You can add paragraphs as long as they comply with the given structure of the dedicated documentation.

Enter content as follows:

  1. Enter the help text in the free-format Text area.

  2. Save your modifications (to activate the Save action, tabulate or click a different part of the screen).

  3. Generate the documentation (the final HTM page) to view the final result.

Example

ATB

Table dictionary

100/10

Paragraph generated automatically.

INC

Include paragraph

 

This paragraph is used to insert a link to a paragraph of another function, whose content is displayed directly in the final documentation. An INC paragraph, therefore, only contains a link to the paragraph of another function.

The syntax of the link is type of documentation of the function being referenced/code of that function/level of the paragraph which is being referenced in that function/sublevel of that paragraph.

Example

AFC/GESACL/70/20. This link displays the content of the paragraph of level 70 and sublevel 20 of the Control tables function (GESACL) in the final HTM page of the function in which it has been inserted.

MIS

Miscellaneous

 

Paragraph added manually.

The paragraph title displays in the table of contents at the top left of the final HTM page.

MIN

Non listed miscellaneous

 

Paragraph added manually.

The paragraph title does not display in the table of contents of the final HTM page.

 

The levels and sublevels predefined for the paragraphs must be complied with when adding a new paragraph.
