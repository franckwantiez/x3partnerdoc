Prerequisites

Generation folder

Set up the path of the folder in which you want to generate your documentation:

  1. Go to Setup > Users > Users.

  2. Go to the Parameter section.

  3. Go to the DOC group, click the Actions icon, then click Detail.

  4. In the Value field of the DIRDOC parameter, enter your directory path. Use #@ to specify a local path.

    Example

    #@D:\EntManag\Doc\%.

  5. In the Value field of the DOCDOSS parameter, enter your server name@folder name.

Temporary documentation directory

Documentation can be generated in a temporary directory but it must be copied to the application directory to test the function help and field help. Initially the documentation is produced directly in the application in local mode.

To display the generated documentation correctly, create the tree structure and copy the files (images, CSS, etc.) to each directory.

Field helps

Each documentation record describes either a function, a parameter, an import template, etc. which is documented.

Each record consists of a set of paragraphs which, when generated, are combined to produce the help.

Types of generation

There are two types of generation: generation of the documentation structure and generation of an HTM file.

  • The generation of the documentation structure is based on the data dictionary. This type generates the various paragraphs that make up the structure of the documentation for the individual records, that is, all the paragraphs to be completed for each function, parameter, etc. After the generation of the basic structure, subrecords can be created as required.

  • Generation of an HTM file displays the paragraph contents. HTM file generation is useful for reviewing the automatically generated content from the dictionary in order to update it if necessary.
