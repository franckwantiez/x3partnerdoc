Minor side effect of changing the JRE or JDK and workaround

There is a minor side effect when the JRE or JDK path is changed. This was already the case when updating Oracle JRE or JDK 8 where the installation path was tagged with the minor version number.

If you try to uninstall from Programs and Features a Sage X3 component that was installed through the standard IZpack packaging, you will get this error if the JRE/JDK path has been changed since the installation:

Don't click Yes!
Click No to exit without changing anything.

The cause of this error is that, at installation time, the IZpack installer registered the uninstallation command in Windows registry with the full path of the java binary used for installation.

In the SAFE X3 Console uninstallation example beneath, we have the following data in the registry:

The UninstallString was recorded for performing the uninstallation procedure references with a Java path that is no longer relevant.

If you need to perform a clean uninstallation of a component in this situation, follow this procedure:

  1. Open Windows Explorer and locate the installation path for the Sage X3 component you want to uninstall.
  2. Go to the Uninstaller sub-directory and double-click the uninstaller.jar script file.

  3. This script is then launched with your current JRE/JDK version.
  4. Click Yes.
  5. Check the Force the deletion of <Safe X3 Component root directory> check box if you want to do a total cleanup. If you don't do so, there might be some unwanted files remaining under this directory after uninstallation and you might then need to clean them up manually.

    Click the Uninstall button and wait for the message indicating the completion of the uninstallation procedure.

  6. When uninstallation is finished and you are notified in the progression bar, click the Quit button.
