Run Sage X3 customization audit

You can run the audit either through:

  • The interactive function

  • The dedicated batch-capable function

When running the audit from an application folder, you must ensure the application folder has Read access rights on the X3 root folder. To do this:

  1. Go to the Folders function (GESADS) (Setup > General parameters > Folders).

  2. Select the X3 root folder record and go to the Links section.

You do not need to set any access rights if you run the audit while connected to the X3 root folder.

Launch from the interactive function

Open: Development > Utilities > Verifications > Customization > Customization audit management

The Customization audit management function (GESASPE) is a complete interface from which you can run the audit and view the results.

You can then select the Execution button to run the audit.

Launch from the dedicated batch-capable function

Open: Development > Utilities > Verifications > Customization > Customization audit submission

The Customization audit submission function (ACTLSPE) is dedicated to an audit task's submission and can also be submitted as a batch job.

Use the execution dialog box

Folder field Select or enter the folder you want to apply the audit to. If necessary, Sage X3 will prompt you for the Sage X3 user code that applies to the folder you have selected. You do not need to enter any password.
Impacted elements checkbox Select this checkbox to list the Sage X3 dictionary elements that are impacted when an issue is detected on a low-level element. If an issue is detected on a Sage X3 data type, the utility also lists all tables and screens that contain fields with that data type.
OK button Select OK to execute the function. You need to wait up to one or two minutes for the results if you are using the interactive mode. You can resume your activity if you are launching this as a batch job.
