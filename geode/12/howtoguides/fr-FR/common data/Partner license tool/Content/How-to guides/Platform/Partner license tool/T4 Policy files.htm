Fichiers de règles

Gestion du fichier de règles

Pour pouvoir créer ou modifier un fichier de règles, un fichier partenaire doit d'abord être importé.

Une fois que vous avez téléchargé ce fichier, cliquez sur Enregistrer et le bouton Créer s'affiche :

Vous pouvez soit créer un nouveau fichiers de règles, soit utiliser un fichier existant comme modèle. Cliquez sur l'icône Actions au niveau du Code produit et sélectionnez Dupliquer. La copie vous permet, par exemple, d'avoir plusieurs versions des règles pour un même code produit ou version de produit.

Les règles ne peuvent pas être modifiées ou supprimées si au moins un fichier de licence s'y réfère dans le système de fichiers de l'outil. Lorsqu'une licence se réfère aux règles, la case à cocher Référencé est sélectionnée.

Comprendre le fonctionnement des badges

Vous trouverez des informations utiles sur le centre d’aide en ligne Sage X3 concernant le fonctionnement des badges et les méthodes d’obtention d’informations de licence avec le code Sage X3. Il est important de comprendre le concept des badges et des licences avant de créer votre fichier de règles. Vous trouverez également plus d'informations sur ces pages (en anglais) :

  • Main principles of badge management

  • License information

Contenu du fichier de règles

Code produit

Il s'agit d'un code interne que vous attribuez à votre add-on. Afin de garantir que le nom soit unique, vous pouvez inclure votre identifiant local au nom. Les caractères alphanumériques sont autorisés, espaces exclus.

Nom de l'add-on

Vous devez indiquer le nom de votre add-on dans le champ Titre (format libre). Au moment de définir vos paramètres régionaux, vous pouvez également saisir les traductions de ce nom.

Version produit

Le champ Version produit affiche la version de produit de l'add-on à laquelle les règles s'appliquent. Cette valeur peut-être numérique ou alphanumérique.

Code des règles

Vous pouvez indiquer différentes parties du même produit dans le champ Code des règles, et leur associer différents noms de règles.

Il n'est pas possible actuellement de gérer, pour un même produit, des licences avec des règles différentes et des modules identiques.

Version des règles

Il ne peut y avoir qu'un seul ensemble de règles pour chaque combinaison de quatre valeurs. Ces valeurs peuvent uniquement contenir des lettres de A-Z et a-z, des chiffres de 0-9 et le tiret bas (_). La version de produit et la version des règles peuvent contenir des points (.).

Produit de base

Vous devez indiquer un Produit de base, soit le code (et pas le titre) du produit Sage X3 correspondant. Le système de licence de l'application acceptera uniquement les licences correspondant à un produit Sage X3 installé. Code produit pour Sage X3 = 1 ; code produit pour Sage X3 HR & Payroll = 3.

Codes activité

Vous pouvez indiquer autant de Codes activité que vous le souhaitez, en précisant :

  • un code commençant par X, Y, ou Z et contenant des caractères compris entre A-Z et 0-9,

  • un titre facile à comprendre,

  • une condition, dont les valeurs peuvent être :

    • Toujours valide

    • Jamais valide

    • Valide seulement si elle contient une licence

Badges

Vous pouvez indiquer autant de Badges que vous le souhaitez, en précisant :

  • un code commençant par un Y, ou un des préfixes du dictionnaire associé au fichier partenaire,

  • un titre facile à comprendre,

  • une liste de fonctions devant être contrôlées par le badge (appelées "fonctions critiques").

Vous pouvez séparer les noms de fonctions par un espace et/ou une virgule. Les fonctions doivent être définies dans le Dictionnaire des fonctions (GESAFC) :

Seules les fonctions incluses dans la liste doivent être définies dans cette fonction. Le nom de fonction doit commencer par votre identifiant, mais dans certains cas il peut commencer par SUBxxxx.
La licence Sage X3 est chargée en premier, et toutes les fonctions critiques Sage X3 que vous avez référencées sont ignorées.

Paramètres

Vous pouvez indiquer autant de Paramètres que vous le souhaitez, en précisant :

  • un code commençant par X, Y, ou Z et contenant des caractères compris entre A-Z et 0-9,
  • un titre facile à comprendre,

Seuls les paramètres numériques sont pris en charge.

Langues

Vous pouvez indiquer des Langues, en précisant :

  • un code composé de caractères compris entre A-Z et 0-9,

  • un titre facile à comprendre,

  • une condition, dont les valeurs peuvent être :

    • Toujours valide

    • Jamais valide

    • Valide seulement si elle contient une licence

Législations

Vous pouvez indiquer des Législations, en précisant :

  • un code composé de caractères compris entre A-Z et 0-9,

  • un titre facile à comprendre,

  • une condition, dont les valeurs peuvent être :

    • Toujours valide

    • Jamais valide

    • Valide seulement si elle contient une licence

L'outil ne vérifie pas les conflits de noms avec les règles de Sage X3 et les règles d'autres partenaires. Pour contrôler une valeur de paramètre, vous devez utiliser la boite à outils et la contrôler dans votre code. Le système garantit que la valeur que vous avez contrôlée est inchangée.

Comment protéger une fonction Syracuse

Lors de la création de vos règles, vous pouvez définir des fonctions clés qui seront les seules à être contrôlées. Vous serez en mesure de contrôler uniquement les fonctions dont le nom commence par l'identifiant qui est défini dans le fichier partenaire. Pour protéger une fonction Syracuse, vous devez définir une fonction dans le champ Autorisation de la fonction Représentations (GESASW) :

Vous pouvez ainsi définir cette fonction dans le fichier de règles en tant que fonction clé associée à l’un de vos badges. N’oubliez pas que le nom de fonction doit commencer par votre identifiant.
