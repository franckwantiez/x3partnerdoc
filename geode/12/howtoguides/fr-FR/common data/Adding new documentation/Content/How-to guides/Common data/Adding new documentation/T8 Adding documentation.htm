Gérer des patchs de documentation

Éditer des éléments pour créer un patch

Utilisez la fonction Création automatique de patch. Cette fonction vous permet de créer un patch (correctif) de documentation, contenant uniquement les documentations modifiées ou créées depuis une date donnée pour un code activité donné.

Pour accéder à cette fonction, allez à Développement > Utilitaires > Patchs > Création automatique de patch, et sélectionnez uniquement les codes ADO, ADZ et ADF.

Voir la documentation sur la Création automatique de patch (APATCHA).

Dans le premier écran :

  1. Sélectionnez les codes ADO, ADF, ADZ.

  2. Renseignez les dates, le code activité et l'utilisateur (si nécessaire),

  3. Validez.

Dans le second écran (dans la colonne Nombre figure le nombre de documentations qui vont être patchées) :

  1. Sélectionnez la case à cocher Vertical dans Type de patch.

  2. Dans le champ Patch, saisissez le nom du fichier patch.

  3. Si vous souhaitez créer le fichier directement, sélectionnez la case à cocher Génération directe.

Si cette case n'est pas cochée, un fichier contenant uniquement la structure de ce qui doit être patché est créé. La fonction Création de patchs doit alors être utilisée pour créer la liste de patchs (voir ci-dessous).

Pour patcher les documentations verticales, se reporter aux documentations des fonctions suivantes :

  • Création automatique de patch (APATCHA)

  • Création de patchs (APATCH)

Créer un patch

Cette procédure est identique à celle indiquée ci-dessus pour la Création automatique de patchs, à l’exception de la liste de documentations à patcher qui n’est pas générée automatiquement (sauf si la case Génération directe n’est pas sélectionnée dans la fonction Création automatique de patch). Vous devez alors indiquer, documentation par documentation, les documentations à intégrer dans le patch.

Si la case Génération directe est sélectionnée, indiquez le nom du fichier contenant la structure des documentations. La liste des documentations à patcher s'affiche. Les documentations qui ne doivent pas être livrées dans le patch peuvent être retirées.

Cette fonction est documentée dans l'aide sur la Création de patchs (APATCH).

  1. Allez dans Développement > Utilitaires > Patchs > Création de patchs.

  2. Sélectionnez la case à cocher Spécifique ou Vertical dans Type de patch.

  3. Indiquez le chemin du patch.

  4. Nommez le patch.

Supprimer des documentations existantes du dossier de destination

L’intégration d’un patch de documentation écrase les documentations existantes. Il n’est donc pas nécessaire de supprimer les documentations avant d’intégrer les nouvelles (excepté lorsqu’il s’agit de verticaux).

  1. Allez dans Développement > Dictionnaire scripts > Transactions de mise à jour.

  2. Pour chaque transaction : ADF, ADO, ADZ.

Intégrer le patch

  1. Copiez le patch.

  2. Allez dans Développement > Utilitaires > Patchs > Intégration de patchs.

  3. Sélectionnez Serveur.

  4. Saisissez le chemin pour récupérer le patch.

  5. Intégrez-le au dossier.

L’intégration des patchs est décrite dans la documentation Intégration d'un patch dans un dossier (PATCH).
