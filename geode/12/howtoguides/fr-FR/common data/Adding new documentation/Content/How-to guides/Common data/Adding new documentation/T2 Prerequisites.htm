Prérequis

Dossier de génération

Vous devez définir le chemin vers le dossier dans lequel vous souhaitez générer votre documentation :

  1. Allez dans Paramétrage > Utilisateurs > Utilisateurs.

  2. Allez sur la section Paramètres.

  3. Allez sur le groupe DOC, cliquez sur l'icône Actions, puis cliquez sur Détail.

  4. Dans le champ Valeur du paramètre DIRDOC, saisissez le chemin du répertoire. Utilisez #@ pour indiquer un chemin local.

    Exemple

    #@D:\EntManag\Doc\%

  5. Dans le champ Valeur du paramètre DOCDOSS, saisissez Nom du serveur@nom du dossier.

Répertoire de documentation provisoire

Il est possible de générer la documentation dans un répertoire provisoire, mais elle doit être ensuite copiée dans le répertoire de l’application afin de tester les aides sur fonctions et sur champs. Dans un premier temps, la documentation est créée directement dans l’application en local.

Créez l’arborescence et copiez les fichiers (images, CSS, etc.) dans chaque répertoire pour que la documentation générée s'affiche correctement.

Aides sur champs

Chaque documentation décrit une fonction, un paramètre, un modèle d’import (etc.) individuellement.

Chaque fiche de documentation contient un ensemble de paragraphes qui, une fois générés, constituent une "aide" complète.

Types de génération

Il y existe deux types générations : La génération de la structure de la documentation et la génération d'un fichier HTM.

  • La génération de la structure de la documentation s'appuie sur le dictionnaire des données. Cette génération crée les différents paragraphes d'une documentation, c’est-à-dire l’ensemble des fiches/paragraphes à compléter pour chaque fonction, paramètre, etc. Une fois la structure générée, il est possible de créer des sous-fiches.

  • La génération d'un fichier HTM affiche le contenu des paragraphes. Elle est donc utile pour vérifier le contenu généré automatiquement à partir du dictionnaire afin de le compléter si nécessaire.
