Procédure d’installation

Fonctions associées
  • Intégration de patchs (PATCH)
  • Processus (GESAPR)
  • Génération transactions (GENMSKTRT)

Procédure standard

L'installation d'un patch relatif aux données associées à l'élément clé LEG (législation), tels que les processus, se compose de plusieurs tâches :

  • Étape 1 : Installer le patch de données sur le point de connexion de l'Application (point de connexion parent).

  • Étape 2 : Installer le patch de données sur un point de connexion fonctionnel ou test (multi-législations).

  • Étape 3 : Installer le patch de données sur un point de connexion mono-législation

  • Étape 4 : Valider les transactions de processus sur chaque point de connexion via la fonction de Génération de transactions (GENMSKTRT).


La procédure d'installation est différente selon le mode de législation (multi ou mono-législation) du point de connexion fonctionnel ou test.

Pour un point de connexion mono-législation (code activité LEG défini à Non), le champ LEG ne s'affiche pas même s'il est renseigné avec le LEG du point de connexion (GDOSLEG).

Pour un point de connexion multi-législation (code activité LEG défini à Oui), le champ LEG est affiché. Lors de la création d'un enregistrement dans une table multi-législations, ce champ doit être initialisé avec la valeur LEGFIL.

Règles générales

Cette procédure permet de se conformer aux règles définies pour les données associées à l'élément clé LEG sur un point de connexion multi-législation et mono-législation :

  • Sur un point de connexion mono-législation, il est possible de gérer uniquement les données ayant leur champ LEG renseigné avec la législation du point de connexion.

  • Sur un point de connexion multi-législation, il est possible de gérer des données ayant le champ LEG vide (l'enregistrement est valable pour l'ensemble des législations) ou le champ LEG renseigné (l'enregistrement est alors valable uniquement pour la législation renseignée).

Étape 1 : Installer le patch de données sur un point de connexion de l'application

Cette étape consiste à installer le patch de données sur le point de connexion d'Application multi-législation (le point de connexion parent).

L'installation utilise la fonction standard Intégration de patchs (PATCH). C'est un processus progressif.

Sage X3 installera uniquement les données sans législation et les données relatives à une législation installée sur le point de connexion.
La fonction Intégration de patchs (PATCH) vous propose une mise à jour pour tous les points de connexion, soit le point de connexion parent et les points de connexion mono- et multi-législation(s).
N'oubliez pas de les retirer de la liste.
Exemple

Un patch de données contient les couples suivants :

  • Processus1, vide (Processus, Législation)

  • Processus1, FRA (Processus, Législation)


Sage X3 crée un couple de ce type sur un point de connexion multi-législation GER (allemand) et BRI (anglais britannique) : Processus1, vide.

Sage X3 crée 2 couples sur un point de connexion multi-législation FRA (français) et BRI : Processus1, vide et Processus1, FRA.

Étape 2 : Installer le patch de données sur un point de connexion multi-législations

Comme pour l'étape 1, l'installation utilise la fonction standard Intégration de patchs (PATCH).

Sage X3 installera uniquement les données sans législation et les données relatives à une législation installée sur le point de connexion.
La fonction Intégration de patchs (PATCH) vous propose une mise à jour pour tous les points de connexion, soit le point de connexion 'parent' et les points de connexion mono et multi-législation(s).
N'oubliez pas de les retirer de la liste.
Vous pouvez utiliser la fonction de Copie décrite à l'Étape 3 : Installation du patch de données sur un point de connexion mono-législation. Veuillez noter que chaque processus visuel doit être copié individuellement. Vous ne pouvez pas les copier en masse.
Exemple

Un patch de données contient les couples suivants :

  • Processus1, vide (Processus, Législation)

  • Processus1, FRA (Processus, Législation)


Sage X3 crée un couple de ce type sur un point de connexion multi-législation GER (allemand) et BRI (anglais britannique) : Processus1, vide.

Sage X3 crée 2 couples sur un point de connexion multi-législation FRA (français) et BRI : Processus1, vide et Processus1, FRA.

Étape 3 : Installer le patch de données sur un point de connexion mono-législation

Le patch de données ne peut pas être installé directement sur un point de connexion mono-législation.

Vous devez copier les processus mis à jour depuis le point de connexion de l'Application vers un point de connexion mono-législation.

Utilisez la fonction Processus (GESAPR). Cliquez sur Copie pour copier chaque processus manuellement vers un point de connexion mono-législation.

L'action Copie applique les règles suivantes :

  • Si les données sont liées à une législation spécifique, seul le processus lié à la législation du point de connexion mono-législation est copié.

    Si les mêmes données existent déjà sur le point mono-législation, le processus est mis à jour.

  • Si les données ne sont pas liées à une législation, le processus est copié sur le point de connexion mono-législation et affiche alors la législation correspondante.

    Si les mêmes données existent déjà sur le point mono-législation, le processus est mis à jour.

Lorsque des données identiques ne sont pas liées à une législation et, dans le même temps, sont liées à une législation spécifique, il faut copier un processus unique.

Le processus lié à une législation, s'il s'agit de la même législation que le point de connexion mono-législation. Sinon, copiez le processus sans législation.

Exemple

Un point de connexion Application, contient les deux couples suivants :

  • Processus1, vide (Processus, Législation)

  • Processus1, FRA (Processus, Législation)


Cliquez sur Copie pour copier le couple Processus1, FRA sur un point de connexion mono-législation FRA.

Cliquez sur Copie pour copier le couple Processus1, vide sur un point de connexion mono-législation BRI. Cela crée (ou met à jour) le couple Processus1, BRI sur le point de connexion mono-législation BRI.

Étape 4 : Valider les transactions de processus

Une fois les processus installés ou mis à jour, ils ne sont pas encore totalement opérationnels sur le serveur. Les liens définis sur ces processus ne fonctionnent pas. Il est nécessaire de les recoder afin qu'ils soient opérationnels.

Pour que les processus fonctionnent avec Sage X3, vous devez les valider via la fonction Génération transactions (GENMSKTRT) sur chacun des points de connexion où ils ont été installés ou copiés :

  1. sélectionnez le dossier (point de connexion),

  2. sélectionnez Oui dans la colonne Validation (code APR),

  3. cliquez sur OK pour lancer le processus de validation sur le point de connexion sélectionné.

Répétez ces étapes pour chaque point de connexion où un processus a été installé.
