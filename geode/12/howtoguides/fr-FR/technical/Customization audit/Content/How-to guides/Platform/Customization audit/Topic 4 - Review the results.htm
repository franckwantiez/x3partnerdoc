Review the results

Open: Development > Utilities > Verifications > Customization > Customization audit management

The results of your audit are listed in the Customization audit management function (GESASPE).

You can access this function from any of the available child folders. You can view the same results across all child folders. The table that contains the results (ACTLSPE) is a supervisor table that is unique across the whole solution.

This function shows all results, whether they were launched interactively or in batch mode, for any folders in the solution.

The left list displays all historical audits you can also browse through.

Header section

The header section displays the following information:

  • A unique sequence number that identifies the audit run

  • The script version of the audit script

    It is important to note the versions when comparing different audit runs, since the audit script will evolve and refine continuously through the different Sage X3 releases.

  • The folder that was audited for that run

  • Risks, errors, and fatal errors that were identified during this run

Risk, errors and fatal errors

The audit logic identifies three types of issues in the results:

  • Risks are elements that can cause additional work or a possible loss of data or metadata when updating or upgrading. There is also the risk of conflicting with standard elements when applying an update.

    Risk example

    Protecting a standard screen with an activity code prevents that screen from being updated when applying a standard update where it is present.

  • Errors are development elements that do not comply with Sage X3 development guidelines and can cause the same issues as risks.

    Error example

    Deleting or replacing a standard activity code.

  • Fatal errors are development elements that do not comply with Sage X3 development guidelines and can cause issues and malfunctions when operating or updating Sage X3.

Details section

The details section lists all risks, errors, and fatal errors in a paginated grid showing the following details:

  • The Object type field indicates the type of metadata element (table, screen, window, and so on). It is indicated through the standard abbreviation code for metadata types like in Sage X3 patches (ATB for tables, AMK for screens, AWI for windows, AML for local menus, and so on).

  • The identifier points to the actual object. For example, AMK/SOH4 points to the SOH4 screen.

  • The messages (two columns) display more details about the detected issue.

Select the line’s Detail action to drill into the details on each line. It takes you to the relevant item in the dictionary. You can then examine the item and fix the issue.
