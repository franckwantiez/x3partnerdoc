Ficheros de reglas

Gestión de ficheros de reglas

Para crear o modificar un fichero de reglas, primero hay que importar un fichero de partner.

Una vez importado el fichero, haz clic en Registrar, y se muestra el botón Crear:

Puedes crear un fichero de reglas nuevo o utilizar uno que ya tengas como modelo. Haz clic en el icono Acciones a nivel del Código de producto y selecciona Duplicar. Esta acción te permite, por ejemplo, tener varias versiones de las reglas para un mismo código o versión de producto.

Una regla no se puede modificar ni suprimir si hay, al menos, un fichero de licencia vinculado a esta en el sistema de ficheros de la herramienta. Cuando una licencia está relacionada con las reglas, la casilla Referenciado aparece marcada.

Funcionamiento de pases

En el centro de ayuda en línea de Sage X3, tienes información de utilidad sobre el funcionamiento de los pases y los métodos de obtención de información de licencias mediante el código Sage X3. Es necesario dominar los conceptos 'pase' y 'licencia' para poder crear un fichero de reglas. En estas páginas (disponibles en inglés) tienes más información:

  • Main principles of badge management (Principios básicos sobre la gestión de pases)

  • License information (Información de licencia)

Contenido de ficheros de reglas

Código de producto

Se trata de un código interno que se asigna a tu add-on. Con el fin de garantizar que el nombre es único, puedes incorporar tu identificador local al nombre. Puedes utilizar caracteres alfanuméricos sin espacios en blanco.

Nombre de add-on

Indica el nombre de tu add-on en el campo Título (campo de texto libre). A la hora de definir los parámetros regionales, también puedes introducir el nombre traducido.

Versión de producto

El campo Versión de producto muestra la versión de producto del add-on a la que se aplican las reglas. Este valor puede ser numérico o alfanumérico.

Código de reglas

Puedes indicar diferentes partes del mismo producto en el campo Código de reglas y asociarlos a diferentes nombres de reglas.

Por el momento, no es posible gestionar licencias para un mismo producto con reglas diferentes y módulos idénticos.

Versión de reglas

Solo puede haber un conjunto de reglas para cada combinación de 4 valores. Estos valores solo pueden contener letras (A-Z y a-z), números (0-9) y el carácter barra baja (_). La versión del producto y la versión de las reglas pueden contener puntos (.).

Producto de base

Para indicar el producto de base, introduce el código (no el título) del producto Sage X3 correspondiente. El sistema de licencias de la aplicación solo acepta licencias de un producto Sage X3 instalado. Código de producto de Sage X3 = 1; código de producto de Sage X3 HR & Payroll = 3.

Códigos de actividad

Puedes indicar tantos Códigos de actividad como quieras. Para ello:

  • Define un código que comience por X, Y o Z y que incluya letras (A-Z) y números (0-9).

  • Introduce un título que se entienda bien.

  • Añade una condición con algunos de estos valores:

    • siempre válido

    • nunca válido

    • válido solo si tiene licencia

Pases

Puedes indicar tantos Pases como quieras. Para ello:

  • Define un código que comience por Y o por uno de los prefijos del diccionario asociado al fichero del partner.

  • Introduce un título que se entienda bien.

  • Añade una lista de funciones que deba controlar el pase (conocidas como "funciones críticas").

Puedes separar los nombres de las funciones con espacios o comas. Las funciones se deben definir en Diccionario funciones (GESAFC).

Solo las funciones incluidas en la lista se deben definir en esta función. Debes incluir tu identificador al comienzo del nombre de la función, aunque en algunos casos puede empezar por SUBxxxx.
La licencia de Sage X3 se carga primero y se ignoran todas las funciones críticas de Sage X3 referenciadas.

Parámetros

Puedes indicar tantos Parámetros como quieras. Para ello:

  • Define un código que comience por X, Y o Z y que incluya letras (A-Z) y números (0-9).
  • Introduce un título que se entienda bien.

Recuerda que solo se tienen en cuenta los parámetros numéricos.

Idiomas

Puedes indicar los Idiomas. Para ello:

  • Introduce un código compuesto de letras (A-Z) y números (0-9).

  • Introduce un título que se entienda bien.

  • Añade una condición con algunos de estos valores:

    • siempre válido

    • nunca válido

    • válido solo si tiene licencia

Legislaciones

Puedes indicar las Legislaciones. Para ello:

  • Introduce un código compuesto de letras (A-Z) y números (0-9).

  • Introduce un título que se entienda bien.

  • Añade una condición con algunos de estos valores:

    • siempre válido

    • nunca válido

    • válido solo si tiene licencia

La herramienta no verifica los conflictos que pueda haber entre los nombres y las reglas de Sage X3 y de otros partners. Para controlar el valor de un parámetro en el código, utiliza las herramientas a tu disposición. El sistema garantiza que el valor que has verificado no cambie.

Protección de funciones Syracuse

Cuando creas las reglas, puedes definir funciones clave, que serán las únicas controladas. Solo podrás controlar las funciones cuyo nombre comience por el identificador definido en el fichero del partner.
