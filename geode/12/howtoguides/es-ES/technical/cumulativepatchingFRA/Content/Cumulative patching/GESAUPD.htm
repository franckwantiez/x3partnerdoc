Mise à jour (GESAUPD)

Ouvrir : Développement > Utilitaires > Patchs

Présentation

Utilisez la fonction Mise à jour (GESAUPD) pour visualiser toutes les mises à jour de dictionnaires qui ont été effectuées :

  • par installation d'un patch standard,
  • par correctif (hotfix),
  • par validation de dossier.

Cette fonction ne vous permet pas de créer d’enregistrement de mise à jour. En effet, seul l’easy patching et la validation de dossier peuvent créer des enregistrements dans ces tables système.

Enregistrements

Les lignes de mises à jour sont numérotées dans l'ordre chronologique inversé. Un enregistrement contient les éléments suivants :

  • Numéro de compteur par ordre chronologique inversé,
  • Dossier concerné,
  • Numéro de version interne,
  • Type de mise à jour (menu local 54) :
    • Standard : installation d’une mise à jour standard permettant de passer à une version différente de la version courante,
    • Hotfix : correctif qui ne remet pas à jour le numéro de version.
  • Type de process (menu local 61) :
    • Intégration correspond à la mise à jour d’un patch,
    • Validation correspond à une validation de dossier.
  • Statut de la mise à jour (menu local 64) : indique le statut courant de la mise à jour. Le dernier enregistrement correspond à la dernière mise à jour appliquée.
    • Les statuts possibles sont :
      • Contrôle OK,
      • En cours,
      • Terminé,
      • En erreur,
      • Terminé avec des avertissements.
    • Si le statut est En erreur, vous pouvez consulter les différentes traces et recommencer la mise à jour.
    • Si le statut est En cours, soit :
      • vous êtes connecté à un dossier sur lequel la mise à jour est en cours d'exécution,
      • la mise à jour a été interrompue à la suite d'un arrêt du serveur.
        Pour relancer une mise à jour, cliquez sur Mise à jour statut ; cette action passe le statut de la mise à jour à En erreur et vous pouvez alors relancer l’installation.
        Lorsque le statut d'intégration est En cours, seule l'action Mise à jour statut est disponible.
  • Le statut de génération peut être :
    • Terminé,
    • Non applicable,
    • En erreur,
    • Terminé avec des avertissements.
  • Date et heure UTC à la fin de l'installation du patch,
  • Fichiers traces globaux. Deux fichiers peuvent s'afficher :
    • La trace des contrôles préliminaires,
      Vous trouverez, par exemple, dans ce fichier certains contrôles qui ont empêché le passage de la mise à jour.
    • La trace qui s'affiche lorsque la validation est effectuée. Par exemple, la génération des fenêtres.

    Ces fichiers peuvent être visualisés via le menu Actions, option Lecture de traces.

Tables

Cette fonction contient deux tableaux :

  • Catalogue,
  • Détail.

Tableau Catalogue

Ce tableau contient les catalogues qui ont été installés lors de la mise à jour. Ces catalogues sont des archives contenant l’ensemble des éléments à patcher.

Dans ce tableau, vous avez accès à deux fichiers de trace via le menu Actions :

  • La trace des contrôles préliminaires,
  • La trace de la mise à jour (intégration).

Les colonnes du tableau sont :

  • Le nom du fichier trace,
  • Le statut,
  • La langue,
  • La législation,
  • Le code article,
  • La phase de contrôle de connexion,
  • La phase d'intégration de connexion.

Tableau Détail

Ce tableau contient le détail de tous les éléments mis à jour. Cela inclut :

  • Le catalogue,
  • Le type d’objet,
  • Les identifiants correspondants,
  • Le nom du fichier,
  • L’action de création s’il s’agit d’un nouvel élément, ou l'action de modification,
  • Le chemin d’accès au fichier lors de l’intégration,
  • Le numéro de ligne de l'enregistrement dans le catalogue,
  • La date et heure UTC de la mise à jour.
Cliquez sur le bouton Filtre détail pour réduire le nombre de résultats dans ce tableau. Ce bouton vous permet de filtrer les résultats par n'importe quel champ de détail du tableau Détail. Vous pouvez également :
  • utiliser une expression pour filtrer les résultats,
  • filtrer par différentes opérations (égal, non égal, plus grand que, comme, etc.).
Lorsqu'un filtre est appliqué au tableau Détail, le message Filtre de données activé est visible.

Annexe technique

Les tables stockant les informations de mise à jour (AUPDATE, AUPDATELINE, AUPDATEPATH) sont, au même titre que la table ADOSSIER, des tables système stockées uniquement dans leur dossier racine. En l’absence d’enregistrement dans un dossier donné, Sage X3 n’est pas en mesure de déterminer la version courante du dossier et refusera la connexion.

Vérifiez que ces tables sont à jour. La mise à jour s'effectue dans les deux cas suivants :
  • Lors de l’import dossier via la console. Un fichier AUPDATEPARAM.ini situé à la racine de l’arborescence du dossier stocke les valeurs correspondantes.
  • Lors d’une validation de dossier.
En cas de transfert direct de dossier par d’autres méthodes, vous devrez revalider le dossier pour mettre à jour ces tables.
