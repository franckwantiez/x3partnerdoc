Patch cumulatif

Introduction

À partir de la version 2019 R4 (12.0.19), un nouveau système de gestion des mises à jour a été mis en place dans Sage X3. Ce système s’applique à partir de la version 2019 R5 (12.0.20) et supérieures.

Vous devez utiliser au minimum la version 2019 R4 (12.0.19) pour bénéficier de la fonctionnalité de patch cumulatif. Si votre instance est sur une version inférieure, vous devez d'abord la mettre à jour en version 2019 R4 (12.0.19). Utilisez l'easy update pour effectuer cette mise à jour.

Principes

Sage livre désormais un seul fichier d'archive, contenant exclusivement la dernière version ou le dernier niveau de hotfix.

Cette livraison intègre uniquement la dernière version de chaque élément corrigé, ce qui optimise considérablement le nombre de patchs à effectuer. La fonction d’easy patching détermine quels sont les patchs à effectuer en fonction de la version d’origine.

Réalisez une sauvegarde de votre application avant de faire votre mise à jour.

La mise à jour s'effectue de façon optimisée, en prenant en compte uniquement les langues et législations utilisées dans le dossier. Vous devez avoir préalablement mis à jour Syracuse et le moteur d'exécution X3 (si nécessaire), en suivant les recommandations de la note d'installation. Les opérations s’enchaînent dans cet ordre :

  1. Le dossier X3 est mis à jour.

  2. Les dossiers applicatifs sont mis à jour en parallèle. Si des dossiers filles sont présents, ils sont traités après mise à jour du dossier parent correspondant.

  3. Les dossiers historisés ne sont pas mis à jour et doivent être validés après exécution du patch. Le message suivant dans la trace du dossier lié au dossier historisé le signale. Attention : Le dossier de référence SEED a été modifié. Relancez la validation du dossier pour le dossier historisé HSEED.

    Cette mise à jour peut être différée, si nécessaire, pour tenir compte des contraintes d’exploitation.

Une nouvelle fonction de consultation des mises à jour est disponible (GESAUPD). Vous pouvez y accéder depuis le menu des patchs (Développement > Utilitaires > Patchs) pour consulter le détail des mises à jour effectuées. Cette fonction est la référence pour les patchs, et non plus la table des patchs (qui reste disponible pour les patchs spécifiques). Pour les patchs spécifiques, utilisez la fonction Intégration de patchs (PATCH) dans Développement > Utilitaires > Patchs.

Les mises à jour officielles doivent impérativement être effectuées depuis la fonction d’easy patching. Le patcheur traditionnel reste disponible, mais uniquement pour les patchs faits de façon traditionnelle par un partenaire.
Si une erreur ou une interruption inattendue se produisent, la fonction GESAUPD vous indique ces erreurs. Après avoir résolu les incidents, redémarrez l'easy update en conservant tous les points d'entrée (dossiers). L'easy update détecte les éléments qui ont déjà été mis à jour, et traite uniquement le différentiel entre les éléments qui n'ont pas encore été intégrés. Cela est possible grâce au catalogue des patchs, qui détaille tous les éléments intégrés à chacun des dossiers de la solution.
L’easy patching ne prend plus en charge plus les dossiers de test. Vous devez créer un environnement dédié pour gérer vos tests. Il n'y a plus de testeur de patch à partir de la version 2019 R5 (12.0.20).

Principes de fonctionnement

Le patch cumulatif intègre uniquement la dernière version de chaque composant. Cela permet un gain de temps considérable lors de la mise à jour.

Exemple

Les versions consécutives Ra, Rb, Rc, Rd, Re et Rf sont toutes postérieures à la version 2019 R4 (12.0.19). Elles introduisent les évolutions suivantes :

  • L’écran X a été modifié et livré plusieurs fois (en Rb, Re, et Rf).
  • Le traitement Y a été modifié en Rc et Rd.
  • La table Z a été modifiée en Rb.
  • L’état R a été modifié en Rd.

Ce qui donne le schéma suivant :

Avec des patchs traditionnels, passer de R4 à Rf nécessitait de passer tout en séquence : l’écran X aurait été intégré et validé trois fois, le traitement Y deux fois.

Avec le nouveau système, la mise à jour livrée lors de la sortie de la Rf intègre une seule fois l’écran X, le traitement Y, la table Z et l’état R. Cette mise à jour permet uniquement de passer à la dernière version Rf, mais son application est intelligente et peut être faite sur toute version comprise entre R4 et Re :

  • Si vous êtes en Rb, l’écran X est patché une seule fois (avec la version Rf), le traitement Y est patché une seule fois (avec la version Rd), l’état R est également patché, mais pas la table Z :

  • Si vous êtes en Rd, seul l’écran X est patché :

Ce système ne fonctionne qu’à partir de la version 2019 R4 (12.0.19). Si vous êtes en version 2019 R2 (12.0.17) et souhaitez passer en version Rf, postérieure à la 2019 R4 (12.0.19), vous devez appliquer les patchs suivants :

  • Le patch vers la version 2019 R3 (12.0.18),
  • Le patch vers la version 2019 R4 (12.0.19),
  • Le patch qui est livré pour la Rf, et qui s’applique puisque l'on est en 2019 R4 (12.0.19).

Si vous êtes dans une version postérieure à 2019 R4 (12.0.19), appliquez uniquement le patch fourni en Rf.
