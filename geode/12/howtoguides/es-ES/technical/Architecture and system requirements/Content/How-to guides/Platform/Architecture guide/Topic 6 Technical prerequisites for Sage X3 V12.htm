Requisitos previos técnicos de Sage X3 V12

Lista de componentes necesarios para crear un entorno de Sage X3 V12

Nombre componente Nivel Descripción/Función Obligatorio Proporcionado en soporte X3 Notas
Sistema RDBMS Base de datos Oracle RDBMS o Microsoft SQL Server para almacenar los datos de aplicación. Sí No

Ver más adelante la matriz de compatibilidad.

Los clústers Oracle RAC y SQL Server están permitidos.
Conector de base de datos Base de datos Permite crear la base de datos a partir de la consola Sage X3. Opcional Sí

Es opcional si la base de datos se crea manualmente por un gestor de bases de datos o con las herramientas de gestión del proveedor de RDBMS.

No se puede instalar en un clúster RDBMS.
Motor de ejecución de Sage X3 Aplicación Entorno de ejecución 4GL Sí Sí El clúster puede tener varias instancias.
Aplicación de Sage X3 Aplicación

Núcleo de la aplicación

Contiene todos los programas 4GL, los diccionarios de datos y la definición de la interfaz de usuario correspondiente a la lógica empresarial de Sage X3.

 Sí Sí

El clúster puede tener varias instancias.

En este caso, hay que instalar el componente de la aplicación de Sage X3 en un servidor de ficheros redundante compartido en la red.
Herramientas de cliente SQL Server Aplicación Herramientas que permiten acceder al RDBMS de Microsoft SQL Server desde el motor de ejecución de Sage X3. Solo con SQL Server No

Instalada con RDBMS cuando SQL Server se encuentra en el mismo host que el cliente de Sage X3.

Se debe instalar cuando SQL Server no está ni en el mismo cliente que el motor de ejecución de Sage X3 ni en los motores de ejecución adicionales de Sage X3.
Servidor Apache HTTP Aplicación

Da acceso a los recursos de aplicación de Sage X3 independientes de la plataforma para el servidor de presentación web.

Suele estar instalado en el mismo host que el componente de aplicación de Sage X3.

 Sí No

Puede haber varias instancias.

Con el equilibrador de carga para una alta disponibilidad cuando la aplicación está en un clúster.

Es solo para uso interno.

Ver más adelante la matriz de compatibilidad.
Servidor web Presentación

Servidor de presentación web de la familia de software Sage X3

Basado en Node.js.

Este servidor proporciona:

  • acceso interactivo a la aplicación desde un navegador en un ordenador, una tablet o un smartphone
  • servicios web SOAP y REST entrantes
  • servicios web REST salientes y una estructura para los servicios web SOAP salientes

Gestiona la seguridad de acceso a la aplicación mediante distintos métodos de autentificación (básico, LDAP, LDAPS, oAuth2 o SAML2).

Tiene en cuenta:

  • tareas batch
  • workflows
 Sí Sí El clúster puede tener varias instancias.
MongoDB Presentación Registra toda la información técnica del servidor de presentación web. Sí Sí

Sage proporciona un paquete de instalación rápida para MongoDB Community Edition.

También se puede instalar desde MongoDB Open-Source o distribución estándar de MongoDB Enterprise.

MongoDB permite gestionar clústers.
Elasticsearch Aplicación Permite indexar los datos de aplicación de forma rápida, flexible y optimizada. Sí No

Se debe instalar a partir de la distribución estándar de Elasticsearch.
AdxAdmin de Sage X3 Gestión Motor de gestión Sí Sí Es obligatorio para poder gestionar componentes en Sage X3.
Consola de Sage X3 Gestión

Consola de gestión que se utiliza para configurar o actualizar componentes en los entornos de Sage X3.

Utiliza el motor de gestión AdxAdmin de Sage X3.

 Sí Sí

Se debe instalar en un servidor o estación de trabajo Windows.

Se puede instalar varias veces.
Servidor de impresión de Sage X3 Reporting

Servidor de reporting basado en el motor de impresión de Crystal Reports.

Genera informes en distintos dispositivos de impresión o formatos de fichero.

 Sí Sí

Se debe instalar en un servidor o estación de trabajo Windows.

Puede haber varias instancias.
Servidores ADC de Sage X3 Presentación Permite acceder a los subconjuntos de las funciones (Producción, Stock) a partir de un terminal móvil (ADC). Opcional Sí

Puede haber varias instancias.
Conector BO de Sage X3 Reporting Conector para Business Objects (BO) Opcional Sí, solo para clientes de V11 que actualizan a V12. Se dejará de utilizar en una versión posterior.
