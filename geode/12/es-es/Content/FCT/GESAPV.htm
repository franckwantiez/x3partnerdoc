Vistas del portal

La vista es el elemento unitario que constituye una página de portal. Está formada por dos elementos, capturados cada uno en una de las pestañas de la función:

  • Una fuente de datos que define un proceso de extracción de información a partir del referencial ADONIX X3. Una fuente de datos devuelve valores organizados en líneas, cada una conteniendo columnas (el número de columnas depende de la fuente de datos, el número de líneas depende de los datos realmente extraídos cuando se consulta la fuente y se lee la base)
  • Un componente visual que describe la forma en la que se mostrarán los datos extraídos.

La asociación de estos dos elementos viene acompañada de la definición de una serie de parámetros que se capturan en los cuadros de cada pestaña.

Cabecera

En la cabecera, se captura el código, que identifica una vista del portal, y un título.

  • Código (campo APVCOD)

Este código identifica de forma unívoca la ficha en curso.
  • campo APVINTIT

Pestaña Fuente de datos

Esta pestaña permite definir la fuente de datos, es decir el origen de la información que se va a mostrar.

Bloque Número 1

  • Tipo (campo SRCCODTYP)

Permite definir un tipo de fuente de datos. En función del tipo, se introducirá, si es necesario, un código complementario que identifica la propia fuente y una serie de parámetros complementarios definidos en el cuadro de la parte inferior de la pestaña.
  • Código (campo SRCCOD)

Identifica la fuente de datos. En función del tipo introducido anteriormente, este campo puede ser:

  • un código de multi-lista (tipo de fuente MLT).
  • el código de una consulta o consulta SQL (tipo de fuente REQ).
  • El código de una estadística (fuente de datos STA).
  • El código de un tratamiento (fuente de datos TRT).
  • vacío (es el caso de los tipos de fuentes URL y AGD, que sólo necesitan una lista de parámetros en el cuadro que aparece a continuación)
  • campo WDES

Tabla Fuente de datos

  • Parámetro (campo SRCPARCOD)

Define el código del parámetro necesario para la fuente de datos.
  • Título parámetro (campo SRCPARDES)

Título asociado al código anterior.

  • Fórmula (campo SRCPARFOR)

Permite introducir, en forma de una fórmula evaluada, el valor del parámetro.

Hay que tener en cuenta que:

  • esta fórmula puede ser alfanumérica, numérica o de tipo fecha según los casos, pero se convertirá el tipo de la expresión si no corresponde a lo esperado.
  • en ciertos casos, se pueden dar valores constantes y seleccionarlos mediante un clic derecho.
  • cuando se pueden dar varios valores (por ejemplo números de columnas), se expresan en forma de una cadena de caracteres que contiene una lista de valores numéricos separados por puntos y comas. Por lo general, la selección sucesiva de estas columnas puede realizarse mediante un clic derecho, con lo cual se completa la lista de valores.
  • Descripción (campo SRCPARLIB)

Título asociado al código anterior.

Pestaña Componente visual

Esta pestaña permite definir el formato de visualización de la información (en forma de cuadro, gráfico, agenda, etc.). Este formato, identificado por un componente visual y los parámetros que le están asociados, depende de la fuente de datos (ciertos componentes sólo funcionan con determinadas fuentes de datos).

Los componentes entregados y las restricciones correspondientes se definen en el anexo más abajo.

Bloque Número 1

  • Componente visual (campo CMPCODTYP)

Tabla Parámetros

  • Descripción (campo CMPPARDES)
  • Fórmula (campo CMPPARFOR)

No hay ninguna ayuda disponible para este campo

  • Descripción (campo CMPPARLIB)

Lista de las fuentes de datos estándar

  • AGD (Agenda)
  • MLT (Multi-lista)
  • REQ (Consulta)
  • STA (Estadística)
  • TRT (Tratamiento)
  • WEB (Página Web)

Los tipos de fuente disponibles en modo estándar se describen más abajo.

AGD (Agenda)

Este tipo de fuente corresponde a la agenda CRM.

No se le asocia ningún código.

Los parámetros que se deben capturar permiten definir una ventana de fechas de consulta:

  • el Número de días antes (de la fecha actual).
  • el Número de días después (de la fecha actual).

Ambos son expresiones numéricas.

MLT (Multi-lista)

Este tipo de fuente de datos corresponde a un conjunto de listas de selección (tal y como se visualizan en la lista izquierda de un objeto).

Se le asocia un código de multi-lista.

No hay otro parámetro asociado.

REQ (Consulta)

Este tipo de fuente de datos corresponde a una consulta o consulta SQL, identificada mediante su código. A continuación se definen dos parámetros adicionales:

  • el primero, denominado FRQ, define la frecuencia de extracción. El valor, que puede seleccionarse en una lista, va de 1 (en cada utilización) a 6 (la primera vez), pasando por niveles intermedios como 3 (todos los días). Al utilizar la fuente de datos, se lee la fecha del último cálculo de la consulta para saber si es necesario volver a calcularla antes de la visualización, en función del valor del parámetro.
  • el segundo, denominado INITLEVEL, es útil para las consultas no definidas en SQL. Permite definir el nivel de detalle en el que se sitúa para la visualización de los resultados de la consulta, cuando ésta utiliza rupturas (columna Grupo del generador de consultas). En el nivel 1, sólo se muestran las rúbricas cuyo nivel (indicado en la columna nivel) es igual a 1 y las columnas acumuladas correspondientes (las demás se envían vacías). En el nivel 2, se da el detalle de las rúbricas de nivel 1 y 2, y así sucesivamente.

STA (Estadística)

Este tipo de fuente corresponde a la visualización de una estadística, cuyo código se indica. Se deben indicar los siguientes parámetros :

  • la sociedad (CPY) y la planta (FCY) en las que se hace la consulta (si se indica una cadena vacía, todas se tomarán). Se puede realizar una selección en estos parámetros.
  • las fechas de inicio y fin para los dos intervalos de tiempo posibles (DATDEB1 y DATDEB2, DATFIN1 y DATFIN2)
  • el valor de los criterios (de 1 a 8 como máximo, denominados CRIT1 a CRIT8). El número de criterios para los que se indica un valor diferente de la cadena vacía define la profundidad en la que se muestra la estadística.

TRT (Tratamiento)

Este tipo de fuente corresponde a la ejecución de un tratamiento normalizado, cuyo nombre se indica. Se pueden solicitar parámetros que dependen del tratamiento (por defecto, se trata de 4 parámetros banalizados denominados P1 a P4). El tratamiento SUBAPSTRT y el tratamiento SUBAPSTRTI documentan la forma de crear una fuente de datos de este tipo.

WEB (Página Web)

Este tipo de fuente corresponde a la visualización de una página Web cuya dirección se evalúa a partir del parámetro URL que se debe definir.

Lista de los componentes visuales disponibles en modo estándar

  • FAG (Agenda flash)
  • FCA (Calendario flash)
  • FAG (Grafos flash)
  • ML1 (Multi-lista estándar)
  • RE1 (Consulta estándar)
  • ST1 (Consulta estándar)
  • TB1 (Cuadro estándar)

Los componentes disponibles en modo estándar se describen más abajo.

FAG (Agenda flash)

Este componente visual sólo se puede asociar a la agenda CRM (fuente AGD). Muestra en forma de un calendario los campos procedentes de la agenda, con interacciones vinculadas a las especificidades de la agenda CRM. Los parámetros asociados son:

  • el tipo de agenda (FCCTYPE), cuyos valores se pueden obtener mediante selección. Los valores posibles son 'XMONTH' (agenda mensual), 'XWEEK' (agenda semanal), 'XDAY' (agenda diario).
  • el número de columna de los campos Tipo de acción, Crono acción, Fecha de inicio, Hora de inicio, Fecha de fin, Hora de fin, Tercero, Interlocutor, Categoría, Objeto, Flag realizado. Estos campos se muestran en el orden exacto de extracción de la fuente de datos AGD; entonces, por lo general, no es necesario modificar las columnas por defecto (de 1 a N) mostradas frente a los parámetros, salvo si se desea no mencionar ciertos datos, en cuyo caso se pondrá el valor 0 frente a la columna. Puede ser el caso, por ejemplo, si la fecha de inicio es igual a la fecha de fin o si la noción de hora no se utiliza.
  • el indicador que define si el selector de calendario utilizado para pasar de una a otra de estas representaciones es presente (2=Sí, 1=No).

FCA (Calendario flash)

Este componente visual permite mostrar eventos fechados en forma de un calendario visual. Sólo se puede utilizar con las siguientes fuentes de datos: tratamiento, consulta.

Los parámetros asociados son:

  • el tipo de calendario (FCCTYPE), cuyos valores se pueden obtener mediante selección. Los valores posibles son 'XMONTH' (calendario mensual), 'XWEEK' (calendario semanal), 'XDAY' (calendario diario).
  • el indicador que define si el selector de calendario utilizado para pasar de una a otra de estas representaciones es presente (2=Sí, 1=No).
  • el número de las columnas de la fuente de datos que indica respectivamente la fecha y hora de inicio, y a continuación la fecha y hora de fin. A excepción de la fecha de inicio, que permite colocar el dato en el calendario, estos datos se pueden omitir. La selección directa de los campos mediante un clic derecho es posible si la fuente de datos asociada es una consulta.
  • el número de columna de la fuente de datos que indica el icono y el color utilizados para simbolizar el evento (campos opcionales).

FAG (Grafos flash)

Este componente visual se puede asociar a las estadísticas (fuente de datos STA), las consultas (fuente de datos REQ) o los tratamientos (fuente de datos TRT). Muestra uno de los importes procedentes de la fuente de datos en forma de una representación gráfica, con un título formado por uno de los elementos caracteres procedentes de la fuente de datos. Los parámetros asociados son:

  • el tipo de grafo (FGGTYPE) que puede seleccionarse mediante un clic derecho en la siguiente lista: 'XLINE' (grafo de líneas), 'XBAR' (grafo de barras), 'XAREA' (grafo de áreas), 'XPIE' (grafo de sectores).
  • los números de las columnas del campo Valor (FGCVAL) visualizables. Si hay una sola columna, se indica directamente su número; si hay varias, sus números deben estar dentro de una cadena de caracteres, separados por un punto y coma. Por ejemplo, se obtendrá una sintaxis de tipo "4;7;3;8"
  • el número de columna que corresponde al valor mostrado por defecto (FGCDEF). Si este campo no se indica, se utiliza el primer valor.
  • el número de columna del campo Título (FGCLABEL) mostrado. Hay que tener en cuenta que con una fuente STA, el primer importe es la columna 3, dado que la columna número 1 es el código almacenado en la tabla de agregación estadística y la columna número 2 es el título extraído de la tabla vinculada a este código (sólo se pueden utilizar estas dos columnas para indicar FGCLABEL, ya que todas las demás columnas son numéricas en este caso).
  • los indicadores que definen si el selector de grafo FGCSELG) utilizado para pasar de una a otra de las representaciones y el selector de valor (FGCSELC) para mostrar una de las columnas numéricas presentes en la lista de las columnas posibles son presentes (2=Sí, 1=No).

ML1 (Multi-lista estándar)

Este componente visual permite mostrar datos en forma de listas de valores (puesto que cada una de las listas se identifica mediante una cabecera, el hecho de hacer clic en la cabecera activa la visualización del contenido de la lista). Sólo se puede utilizar con la fuente de datos de tipo MLT (multi-lista) y no tiene ningún parámetro.

Al utilizar este componente, los campos subrayados, que corresponden a la clave principal de las fichas listadas, permiten, mediante un clic, entrar en la ficha correspondiente por medio de un túnel.

RE1 (Consulta estándar)

Este componente visual permite mostrar datos en forma de un cuadro de valores. Sólo se puede utilizar con la fuente de datos de tipo REQ (consulta) y no tiene ningún parámetro.

A diferencia del componente TB1 que es más genérico, autoriza un despliegue de los niveles sucesivos de la consulta a partir del nivel en el que se muestra (parámetro INITLEVEL), mediante un clic en un icono presente en la cabecera de título del cuadro.

Al utilizar este componente, ciertos campos pueden estar subrayados. En este caso, se trata de claves principales vinculadas a las tablas en línea en la consulta. Un clic en el campo permite entrar en la ficha correspondiente por medio de un túnel.

ST1 (Consulta estándar)

Este componente visual permite mostrar datos en forma de un cuadro de valores. Sólo se puede utilizar con la fuente de datos de tipo STA (estadística) y no tiene ningún parámetro.

A diferencia del componente TB1 que es más genérico, autoriza un zoom en una de las líneas de la consulta a partir del nivel en el que se muestra (parámetro INITLEVEL), mediante un clic en un icono presente en la cabecera de título del cuadro.

Al utilizar este componente, ciertos campos pueden estar subrayados. En este caso, se trata de claves principales vinculadas a las tablas en línea en la estadística. Un clic en el campo permite entrar en la ficha correspondiente por medio de un túnel.

TB1 (Cuadro estándar)

Este componente visual permite mostrar datos en forma de cuadros. Se puede utilizar con las fuentes de datos de tipo AGD (agenda), REQ (consulta), STA (estadística) o TRT (tratamiento) y no tiene ningún parámetro.

A diferencia de los componentes ST1 y RE1, no permite ningún zoom o despliegue. Por lo tanto se recomienda su utilización si se desea mostrar datos en una lista en un nivel dado sin permitir la visualización de un sub-detalle, aunque éste existe.
