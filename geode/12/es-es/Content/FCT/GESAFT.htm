Perfil función

Los códigos de perfil función permiten definir las familias de usuarios desde el punto de vista de la habilitación de las funciones (en la ficha usuarios se muestra un código de perfil función asociado). Este código de perfil se define en esta función.

En cabecera

La definición se realiza en una única pestaña indicando el código de perfil que se va a crear o modificar. Hay que tener en cuenta que, en la versión 140, la ficha del perfil muestra algunos datos que antes contenía la ficha del cliente, en particular estas dos pestañas:

  • Código perfil (campo PRFCOD)

Este código identifica de forma unívoca la ficha en curso.
  • Descripción (campo INTPRF)

Destinado a figurar en las pantallas y los informes en los que el código de la ficha puede introducirse o seleccionarse. Este texto permite aportar una descripción más clara de la ficha afectada.

Pestaña Acceso

Esta pestaña permite definir una casilla que da acceso a todas las funciones del código de perfil sin ningún tipo de restricción (por lo general, solo un código de perfil asociado al administrador tendrá estos privilegios).

A continuación, se definen los valores por defecto que solo se utilizan cuando se crea un nuevo usuario asociado al perfil. Estos valores por defecto forman la lista de códigos de acceso autorizados y la casilla Todos los códigos de acceso permite indicarlos directamente.

Estos valores por defecto existen porque un usuario que no sea administrador puede estar autorizado para crear un usuario sin tener el derecho de modificación en algunos códigos de acceso. Se supone que se pueden proponer valores por defecto satisfactorios asociados al perfil función (los perfiles función que puede utilizar el usuario se limitan con el parámetro AUZFCT).

Si la casilla Todas funciones autorizadas no está marcada, se puede definir de manera detallada la lista de funciones autorizadas con los permisos adicionales asociados, si es necesario. Esta definición se realiza en la función Habilitación funcional, a la que se accede desde el menú o directamente mediante un botón.

General

  • Todas funciones autorizadas (campo ALLFCT)
  • Portal fijado (campo FLGPOR)

Ficha usuario por def.

  • Todos los códigos de acceso (campo ALLACS)

Si esta casilla está marcada, el usuario tendrá todos los permisos sobre los informes, funciones, pantallas, campos, etc... protegidos por un código de acceso.

Tabla Ficha usuario por def.

  • Cód. de acceso (campo CODACC)

Los códigos de acceso son códigos que permiten proteger los datos o funciones en lectura, escritura y ejecución (cuando el contexto lo permite). En este caso los códigos actúan a modo de cerradura ubicada en un campo de una pantalla, transacción, informe o en un registro.

A continuación se define, para cada usuario, si existe la posibilidad de acceder a los datos protegidos de esta manera (por decirlo de alguna forma, si tiene la llave que permite abrir esta cerradura).

El número de códigos de acceso y la codificación es totalmente libre. Únicamente cuenta la asignación llevada a cabo en las diferentes fichas, que de esta manera podrán ser controladas.

Una casilla Códigos de acceso permite acceder a todos los códigos de acceso. Si esta casilla no está seleccionada, la captura podrá ser efectuada en la tabla de la pestaña.
  • Consulta (campo CONSUL)
  • Modificación (campo MODIF)
  • Ejecución (campo EXEC)
Actualización códigos de acceso

Pestaña Plantas

Esta pestaña permite definir, por módulo funcional, una planta por defecto a la que está asociado el usuario con el perfil. Esta planta se utiliza siempre que se necesita introducir una planta. La tabla solo mostrará los módulos que se utilizan en el dossier.

Tabla

  • N.º (campo NUMLIG)
  • Módulo (campo MODULE)
  • Planta por defecto (campo FCYDEF)

Esta tabla permite definir las plantas por defecto, por módulo.

Esta parametrización se utiliza cada vez que se introduce una planta.

Pestaña Informes

Esta pestaña contiene los siguientes campos:

  Un indicador Impresión diferida, que permite prohibir la posibilidad de lanzar informes en batch para los usuarios asociados al perfil.

 Un filtro de acceso a los informes, definido en forma de modelo (utilizando los metacaracteres "?" o "*"). De esta forma, se puede bloquear el acceso al usuario a ciertos informes.

  Un filtro de acceso por grupo de informes, que permite bloquear el acceso a ciertos informes. Los grupos de informes se definen en el menú local 97. Los 15 primeros están definidos en el estándar.

Los datos de la pestaña Informes no se pueden modificar para el perfil menú del administrador, definido con el parámetro ADMUSR (este usuario no tiene restricciones de acceso a los informes).

Autorizaciones

  • Impresión diferida (campo DIFIMP)

Este campo autoriza la impresión diferida.
  • Filtro informes (campo FILTRE)

Este campo permite limitar el acceso a los informes para determinados códigos

que se corresponden con el filtro indicado.

Ejemplo: AB??*

El usuario solamente tendrá derechos íntegros en los informes

que empiezan por AB y tienen al menos 4 caracteres.

Tabla Grupos

  • N.º (campo NUMLIG)
  • Grupo de informes (campo GRPETX)
  • Acceso (campo AUTETA)

Informes

Por defecto, los informes siguientes están asociados a la función :

Pero esto se puede modificar por parametrización.
