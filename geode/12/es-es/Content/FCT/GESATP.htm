Familias viñetas

Esta función permite definir :

  • el enlace entre las viñetas que aparecerán en el portal y la función encargada de su correcta visualización. Este enlace se establece por medio del módulo técnico.
  • Los parámetros que permiten caracterizar las viñetas utilizan familias, lo que ofrece la posibilidad de definir valores por defecto que se propondrán en la creación de viñetas.

Las diferentes familias posibles son las siguientes:

  • Favoritos (FAV): esta familia sirve exclusivamente para mostrar la lista de las funciones "favoritas" elegidas por el usuario. El único parámetro asociado es la altura de la viñeta inicializada por defecto con el valor 450
  • Lector de flujo vídeo (FLV): esta familia permite mostrar vídeos definidos en formato flv. Los parámetros asociados a esta viñeta son:
      • La altura (inicializada por defecto con el valor 300)
      • La fuente de datos: indica la ubicación del fichero flv a leer (en forma de enlace http)
      • El mensaje de espera: que se mostrará en la viñeta cuando el lector vídeo no está activado
  • Lector componente flash (FSH): esta familia permite mostrar cualquier tipo de componente flash disponible en la aplicación en forma de vista. Los parámetros asociados:
      • La altura (inicializada por defecto con el valor 300)
      • El acceso a los datos: da el nombre del programa encargado de proporcionar el flujo XML de datos que será interpretado por el componente flash (por defecto se indica el valor SUBAPSMON:CLOBEXP que corresponde al subprograma CLOBEXP del programa SUBAPSMON)
      • La vista del portal: el código de la vista del portal a mostrar (se debe parametrizar la vista del portal con un componente visual de tipo flash - opción B del objeto APV)
  • Menú (MENU): permite definir un menú al que se asociarán viñetas. Este menú puede ser plegado o desplegado por el usuario y puede contener submenús.
      • Menú desplegado: el menú aparece desplegado o plegado según el valor (inicializado por defecto con el valor true=desplegado)
  • Navegador (NAVIG): esta familia sirve exclusivamente para mostrar la lista de las funciones definidas en el perfil de menú del usuario conectado. El único parámetro asociado es la altura de la viñeta inicializada por defecto con el valor 450
  • Proceso (PROC): se utiliza esta familia para mostrar los procesos diseñados con el editor de procesos. El módulo de visualización (lector) muestra estos procesos en un modo interactivo que permite activar acciones y abrir funciones de la aplicación. Los parámetros a definir son los siguientes:
      • Altura: define la altura de la viñeta (por defecto 300, es posible ir hasta 450 sin usar la barra de desplazamiento vertical)
      • Acceso a los datos: Nombre del programa de acceso que proporciona los datos del proceso al lector (por defecto SUBAPR:CLOBEXP)
      • Código de proceso: El proceso debe estar presente en la tabla APROCESSUS
      • Menú proceso: Se puede asociar a cada viñeta de proceso un menú que aparecerá arriba en la parte izquierda en forma de treeview y permitirá navegar entre procesos de un mismo sector de actividad. Si no se indica este parámetro, la navegación entre los procesos no se propondrá en esta viñeta. Este código de menú proceso debe hacer referencia a un elemento de la tabla APROMEN.
  • Visualizador de texto (READ): Esta familia permite mostrar cualquier tipo de flujo texto, html o xml en una viñeta. Los parámetros a indicar son los siguientes
      • Altura: define la altura de la viñeta (por defecto 300)
      • Acceso a los datos: define el programa de acceso a los datos que proporcionará el flujo a mostrar. Este programa es libre, pero es obligatorio que esté definido con dos parámetros
          • Un campo parámetro de tipo Char pasado en value que devolverá el contenido de "parámetros"
          • Un clob pasado en variable que contendrá a la vuelta el flujo a mostrar en la viñeta
      • Parámetros: define una cadena alfanumérica que permitirá pasar parámetros al programa a ejecutar.
  • Lector de flujo RSS (RSS): Esta familia permite visualizar en una viñeta flujos rss publicados en formato rss 2.0. Los parámetros a indicar son los siguientes:
      • Fuente de datos: URL del hilo rss a mostrar (por ejemplo ésta es la dirección del hilo rss del periódico Le Monde http://www.lemonde.fr/rss/sequence/0,2-3208,1-0,0.xm)
      • Visualización detallada: muestra el texto asociado a la información del flujo rss (por defecto true = detalle mostrado)
      • Número de elementos: a mostrar en el hilo rss de la viñeta (por defecto 3)
  • Separador (SEP): Esta familia sólo se utiliza para crear una línea de separación en la visualización de las viñetas a seleccionar. Es de tipo "separador" y tiene un parámetro único que da el carácter a utilizar como separador (por defecto -)
  • Visualización de tablas (TAB):
      • Número de líneas: Número de líneas a mostrar (por defecto 20)
      • Fuente de datos: Programa a llamar para obtener el flujo de datos (por defecto SUBAPSMON:CLOBEXP)
      • Parámetros: Código de la vista del portal a mostrar (se debe parametrizar la vista del portal con un componente visual de tipo tabla - opción C del objeto APV)
  • Página Web (WEB): Esta familia permite mostrar páginas web
      • Altura: define la altura de la viñeta (por defecto 300)
      • Scrolling: Autoriza o no el scrolling en la viñeta (por defecto true=scrolling autorizado)
      • Fuente de datos: URL a mostrar
  • Notas (WNOTE): Esta familia permite disponer de viñetas de tipo bloc de notas o recordatorio
      • Texto: indicar aquí el texto inicial que aparecerá en la viñeta. Será sustituido por el texto introducido por el usuario una vez utilizada la viñeta
