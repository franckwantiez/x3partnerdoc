X3 Warehousing

Tablas

Esta función permite consultar líneas de auditoría por modificaciones de grabación en las tablas de la base.

Cada vez que se define un elemento auditable en una tabla (puede ser una creación, modificación o eliminación), se crea una línea por cada registro. Sólo se conoce la clave de grabación modificada, pero si se desea conocer con detalle ciertos campos modificados, estos campos tendrán que estar definidos en la tabla auditoría. Se pasa a continuación a la función de consulta Auditoría en campos.

Gestión de Pantalla

  • Cabecera
  • Pestaña Auditar

Cabecera

En el encabezamiento se encuentran criterios que permiten filtrar las líneas que apareceran en la consulta.

Bloque Número 1

  • Periodo (campo STRDAT)

Si se indica esta fecha, sólo se muestran en la consulta los datos cuya fecha de última modificación o de creación es posterior o igual.

A excepción de que haya un memo que fije si valor, el valor por defecto de esta fecha viene determinado por el parámetro usuario DATEDEBDFT.
  • campo ENDDAT

Si se indica esta fecha, sólo se muestran en la consulta los datos cuya fecha de última modificación o de creación es previa o igual.

A excepción de que haya un memo que fije si valor, el valor por defecto de esta fecha viene determinado por el parámetro usuario DATEFINDFT.
  • Tabla (campo TABLE)

Bloque Número 2

  • Usuario (campo USER)
  • Login (campo LOGIN)
  • Evento (campo EVENEMENT)

Permite filtrar las líneas a mostrar del hecho en el inicio de la auditoría. A falta de valor, se muestran todas las líneas, pero si se desea realizar un filtro, sólo se permiten 3 valores:

  • INSERT (para ver las creaciones)
  • UPDATE (para ver las modificaciones)
  • DELETE (para ver las supresiones)

Basta con introducir el primer caracter (I, U, D) para que la palabra se complete en ese campo.

  • Auditoría BI (campo AUDBI)

Pestaña Auditar

El cuadro situado en la pestaña permite visualizar con detalle los datos de la auditoría.

Tabla

  • Contador (campo SEQ)

Número de orden cronológico único que se atribuye en el momento de desencadenamiento del hecho auditado. Sólo se atribuye un número de orden cronológico por registro en el origen de la auditoría, y si se activa la auditoría en ciertos campos, las líneas correspondientes compartirán este número.

  • Tabla (campo TBL)
  • Fecha (campo DAT)

Define la fecha del sistema en el servidor de datos en el momento en el que se ha realizado el hecho auditado.

  • Hora (campo HOU)

Define la hora del sistema en el servidor de datos en el momento en el que se ha realizado el hecho auditado.

  • Evento (campo EVT)

Define el tipo de hecho auditado. Hay tres opciones posibles:

  • INSERT define una creación de línea.
  • DELETE define una supresión de línea.
  • UPDATE define una modificación de línea.
  • Clave (campo ID1)

Define el valor de la clave de registro al inicio de la auditoría, si la clave está compuesta por un sólo campo. Cuando está compuesta por varios campos hay que tener en cuenta que aquí sólo se almacena el primero.

La clave que identifica el registro se define en la pestaña Auditoría de la gestion de las tablas. Al código estadístico se vinculan un conjunto de totales basados sobre una serie de entre uno y ocho criterios cruzados.

  • Clave secundaria (campo ID2)

Cuando la clave de registro en el inicio de la auditiría se compone de varios campos, aquí aparece el valor de los campos en orden a partir del segundo; estos campos están separados por el caracter~.

La clave que identifica el registro se define en la pestaña Auditoría de la gestion de las tablas. Al código estadístico se vinculan un conjunto de totales basados sobre una serie de entre uno y ocho criterios cruzados.

  • Usuario (campo ADOUSR)

Define el código de usuario (en el sentido de la aplicación) que produce el hecho auditado.

Si este hecho ha sido desencadenado sin intervención de la aplicación, el campo aparecerá vacío.

  • Login (campo SYSUSR)
  • Cliente (campo ADRCLI)

Define la dirección (red) del usuario en el origen del hecho auditado. Si esta dirección no puede ser determinada, el campo aparecerá con la leyenda Unknown. Este es el caso para las modificaciones realizadas de forma externa a la aplicación.

  • Estado workflow (campo STA)
  • Estado BI (campo STABI)
Túnel

Esta función permite, si la ficha modificada está gestionada mediante un objeto de la aplicación, visualizar la ficha en cuestión entrando en la función personalizada.

Detalle de los campos

Tabla

  • Columna (campo COL)

Define el campo auditado bajo la forma CHAMP si se trata de un campo no dimensionado, y CHAMP(INDICE), si el campo está dimensionado.

  • Antiguo valor (campo OVAL)
  • Nuevo valor (campo NVAL)

Permite, una vez trazado el valor de ciertos campos modificados en la auditoría, abrir una ventana para mostrar el detalle de las modificaciones.
