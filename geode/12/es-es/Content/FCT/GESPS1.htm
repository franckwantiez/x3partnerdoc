Desencadenantes estadísticas

Se propone en el sistema una herramienta muy general de parametrización de estadísticas. Esta herramienta permite crear una base estadística con las siguientes características:

  • Lista de eventos activadores gestionados por el sistema y que permiten lanzar la actualización de la base estadística.
  • Definición de códigos estadísticos que permiten acumular los importes cifrados apoyándose en el evento activador y los criterios (de 1 a 8, más la sociedad y la planta).
  • Generación, para cada código de registro de acumulados según las combinaciones de criterios definidos.

Estos registros de acumulados pueden ser utilizados por estados o interrogaciones accesibles directamente desde una ficha de base, para obtener tablas estadísticas.

La parametrización de eventos activadores permite precisar para cada evento susceptible de ser ejecutado por el sistema, los datos en línea y los criterios de selección correspondientes.

Gestión de Pantalla

  • Pestaña General

Cabecera

Permite identificar el código activador.

  • Código (campo COD)

Permite identificar el elemento que activa la estadística.
  • Descripción (campo INTIT)

Permite definir un título asociado a cada ficha.

Pestaña General

En la primera pestaña se definen los datos generales ligados al desencadenante estadístico: la tabla principal sobre la que se gestiona la agregación, y las zonas que permiten determinar el registro del acumulativo a anotar.

Estos campos son importantes en la medida en que la tabla STAT que contiene los datos se estructura por sociedad, planta y fecha (según la periodicidad) y el sub-programa de acumulación debe saber en qué zonas de la tabla activadora se encuentran los valores a utilizar.

Características

  • Descrip. corta (campo INTSHO)

Encabezamiento corto de 10 caracteres alfanuméricos.

  • Módulo (campo MODULE)

Módulo al que pertenece la parametrización
  • Código actividad (campo CODACT)

Un código de actividad permite:

  • hacer opcional un elemento del diccionario si el valor asociado al código de actividad es nulo.
  • firmar los elementos específicos si están marcados por un código que empieza por X, Y o Z.
  • dimensionar un número máximo de líneas cuando el código de actividad marca los elementos de un cuadro.

De este modo, si el código de actividad no está activo, el elemento marcado no podrá utilizarse y el código asociado, si lo hay, no se generará ni podrá ser activado.

Bloque Número 2

  • Tabla desencadenante (campo RLGTBL)

Define la tabla principal a partir de la cual se definen los campos que se utilizarán para activar la actualización de las estadísticas. Esta tabla debe estar en línea en el momento de la ejecución con un registro actual de los datos a acumular.

Opciones

  • Campo fecha (campo DATFLD)

Define el campo de la tabla activadora donde se encuentra la fecha asociada al registro. Esta fecha permite determinar en qué periodo se acumulará el valor estadístico relativo al registro, teniendo en cuenta la periodicidad de cálculo escogida.

  • Campo sociedad (campo CPYFLD)

Define el campo de la tabla desencadenante donde se encuentra el código de la sociedad al que está asociado el registro. Si este campo, o si el campo planta están indicado, las estadísticas que se apoyan en este código podrán discernirse por sociedad.

  • Campo planta (campo FCYFLD)

Define el campo de la tabla activadora donde se encuentra el código de la planta asociado al registro. Si este campo está indicado, las estadísticas que se apoyan en este código podrán discernirse por planta.

Bloque Número 4

Pestaña Adelantado

La segunda pestaña permite completar la descripción del contexto de activación, si este es más complejo:

  • describiendo en un primer cuadro, las bisagras que permiten que se produzca la lectura de otras tablas vinculadas durante el proceso de acumulación. La orden de captura de las tablas en el cuadro es importante, puesto que una tabla puede estar vinculada en cascada a la tabla principal y al resto de tablas que la preceden.
  • dando las condiciones lógicas para que se realice la acumulación.
  • definiendo las variables (a través de las fórmulas de cálculo) utilizables directamente en la definición de estadísticas.

Tabla Número 1

  • N.º (campo NUMLIG3)
  • Tablas vinculadas (campo TBL)

Define una tabla que aparecerá en conexión en el proceso de acumulativo. Su registro se determina por un valor de clave de vínculo. Esto permite también basar los criterios de las estadísticas y los valores acumulados en los campos de esta tabla.

  • Abreviatura (campo ABRLNK)
  • Clave de vínculo (campo CLELNK)

Define el código de la clave de tabla que se emplea para realizar la lectura de las líneas vinculadas. Por defecto, la que se utiliza es la primera clave de la tabla.

  • Expresión de vínculo (campo EXPLNK)

Este campo se define como una o varias expresiones calculadas separadas por un punto y coma. Cada expresión se evalúa, y el resultado permite conocer el valor de la clave utilizada para realizar la vinculación. En caso de que se autoricen múltiples vinculaciones, no se puede dar más que los primeros elementos de la clave.

En las expresiones, se pueden utilizar constantes y campos procedentes de las tablas previamente definidas en la lista de vínculos.

Tabla Número 2

  • N.º (campo NUMLIG1)
  • Criterios de desencadenamiento (campo CRI)

Tabla Número 3

  • N.º (campo NUMLIG2)
  • Variable (campo VARNAM)

Define el nombre de una variable que puede utilizarse de forma directa en los acumulativos estadísticos basados en el evento activador, para evitar tener que definir expresiones calculadas sobre los campos de las tablas en línea.

  • Descripción (campo VARINTIT)

Título asociado al código anterior.

  • Expresión (campo VARFOR)

Define la fórmula de cálculo correspondiente a la variable.

Esta fórmula es una expresión calculada que puede tener constantes, operadores, funciones y todas las variables en línea en el contexto de agregación.

  • Divisa (campo VARCUR)

Con el fin de crear acumulativos sobre los importes en divisas diferentes convertidas automáticamente, se puede introducir aquí una constante o un campo que defina un código de divisa. A continuación se capturará, en la definición del acumulativo estadístico, el código de la divisa en la que debe gestionarse cada acumulativo.

Lista de los eventos activadores estándar

Los eventos activadores estándar aparecen en la documentación, organizados por módulo y accesibles a través del vínculo siguiente.

Hay que tener en cuenta que, en el caso de documentos complejos (de tipo cabecera y línea), pueden existir dos eventos activadores, uno producido una vez por la cabecera, y el otro por la línea. Según el nivel de detalle deseado la la agregación, se escogerá el uno o el otro.
