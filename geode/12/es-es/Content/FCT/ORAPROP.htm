Propiedades de Oracle

Esta función permite visualizar las principales propiedades de la base de datos cuando la base de datos utilizada es Oracle™.

Los datos que aquí se presentan no pueden modificarse.
Su modificación debe realizarse mediante las herramientas de administración propias de la base de datos utilizada.

Advertencia

Esta función permite, mediante peticiones en las tablas y vistas del sistema, ofrecer datos técnicos sobre la base de datos. De la interpretación de los resultados ofrecidos por esta función se encarga un administrador de base de datos, y puede depender de la versión de la base de datos utilizada.

Gestión de pantalla

  • Cabecera
  • Pestaña General
  • Pestaña Parámetros
  • Pestaña Memoria

Esta función permite visualizar las propiedades de la base de datos.
Estos elementos se presentan en tres pestañas: la primera ofrece las características generales, la segunda proporciona la lista de los parámetros y opciones seleccionados al iniciarse la base de datos, y la tercera ofrece información sobre la memoria utilizada por la base de datos.

Pestaña General

La primera pestaña presenta las propiedades generales de la base de datos.

Esta pantalla se compone de dos bloques.

Informaciones sobre la base de datos

Este primer bloque contiene las principales características de la base de datos.

Parámetros NLS

Este bloque permite visualizar las informaciones sobre los juegos de caracteres actuales para la base de datos, la instancia y la sesión actuales.

Bloque Número 1

  • Máquina (campo SRVMAC)

Nombre del servidor sobre el que arranca la base de datos.

  • Instancia (campo SRVINS)

Nombre de la instancia utilizada para identificar de forma unívoca una instancia específica si hay varias de ellas que comparten nombres de servicio comunes.

  • Fecha creación (campo CREDAT)

Fecha y hora de creación de la base de datos
  • campo CREHOU
  • Fecha de inicio (campo STRDAT)

Fecha y hora de inicio de la base de datos
  • campo STRHOU
  • Modo ARCHIVELOG (campo LOGMOD)

Indica si el modoARCHIVELOG está activado o desactivado.

  • NOARCHIVELOG :
    En modo, NOARCHIVELOG, los fichiros de redo log online se reescriben cada vez qeu un fichero de redo log online está lleno, y que se lanza un log switch. LGWR no reescribe sobre un grupo de fichero de redo log si el checkpoint para ese grupo no está terminado.
  • ARCHIVELOG:
    Si la base de datos está configurada en el modo ARCHIVELOG, los grupos de redo llenos deben ser archivados. Como todas las modificaciones realizadas sobre la base de datos se registran en los ficheros de redo log online, el DBA puede utilizar el salvado del disco duro y los ficheros de redo log archivados para restaurar la base de datos sin perder ningún dato.

    Se pueden archivar los ficheros de redo log :
    - Manualmente.
    - Automáticamente: Método recomendado.

    El parámetro de inicio LOG_ARCHIVE_START, si está en TRUE, indica que el archivado se realiza de forma automática.. En FALSE, indica que el DBA lo hace de forma manual. En modo automático, el archivado se realiza gracias al proceso ARCn, y de forma manual, con peticiones SQL.

 

  • N.º usuarios (campo NBRUSR)

Número de usuarios definidos en la base de datos.
  • N.º conexiones activas (campo NBRPRO)

Número de conexiones activadas en la base de datos.

Tabla Parámetros NLS

  • Parámetro (campo PAR)

Muestra los parámetros sobre los juegos de caracteres actuales y sus valores para la base de datos, la instancia y la sesión.

Los parámetros presentados dependen de la versión de Oracle. Estos parámetros no pueden modificarse desde esta función.

  • Base datos (campo BDDVAL)
  • Instancia (campo INSVAL)
  • Sesión (campo SESVAL)

Pestaña Parámetros

En la segunda pestaña se muestra la lista de parámetros de inicialización y sus valores. También se indica si se trata del valor por defecto o de un valor especificado.

Tabla

  • Parámetro (campo PAR)

Muestra los parámetros de inicio de la base de datos e indica si se trata del valor por defecto.

Los parámetros presentados dependen de la versión de Oracle. Estos parámetros no pueden modificarse desde esta función.

  • Valor (campo PARVAL)
  • Valor por defecto (campo DEFVAL)
  • Descripción (campo DES)

Pestaña Memoria

En la tercera pestaña se presentan las informaciones relativas a la utilización de memoria de la base de datos.

Esta pantalla ofrece información sobre las dos zonas de memoria de una base de datos Oracle.

Memoria SGA (Área Global del Sistema)

Es la zona de memoria que contiene las estructuras de datos a las que pueden acceder todos los procesos. Gestiona los datos compartidos de los diferentes usuarios.

Esta zona de memoria se compone de varios elementos:

  • Pool compartido: este parámetro precisa el tamaño del pool compartido. El pool compartido contiene principalmente cursores compartidos, procedimientos almacenados, estructuras de control y buffers de mensajes de ejecución paralela. Los valores altos mejoran el rendimiento en los sistemas multiusuarios. Los valores bajos utilizan menos memoria. Este valor se atribuye al parámetro de fichero de inicialización SHARED_POOL_SIZE. Observación: el pool compartido es un parámetro dinámico.
  • Caché de buffer: indica el número de buffers que contiene la caché de buffer de la base de datos. El tamaño de cada buffer situado en la caché corresponde al de un bloque de datos Oracle (que es especificado por el parámetro de inicialización DB_BLOCK_SIZE); en consecuencia, cada buffer de la base de datos situado en la caché puede contener un solo bloque de datos leído desde un fichero de datos. El tamaño de la caché es limitado, por lo que no se pueden tener todos los datos del disco en la caché. Este valor se atribuye al parámetro de fichero de inicialización DB_CACHE_SIZE(10g) o DB_BLOCK_BUFFERS.
  • Zona de memoria LARGE POOL: permite precisar la porción de memoria asignada a la zona de memoria LARGE POOL. Esta porción de memoria es utilizada por el servidor compartido como memoria de sesión, por la ejecución paralela para los buffers de mensajes y por la copia de seguridad para los buffers de E/S en el disco. Este valor se atribuye al parámetro de fichero de inicialización LARGE_POOL_SIZE.
  • Pool Java: precisa el tamaño del pool Java. Este valor se atribuye al parámetro de fichero de inicialización JAVA_POOL_SIZE.

Un gráfico presenta la asignación de memoria que utiliza cada componente SGA.

Memoria PGA (Área Global del Programa)

Es la zona de memoria que contiene todos los datos que no necesitan ser compartidos. Gestiona el funcionamiento de los diferentes procesos.


A continuación se incluye un esquema que muestra la utilización de cada una de estas zonas de memoria:

Memoria SGA

  • campo TXTAUT

Gestión automática de la memoria compartida (10g)

Este valor se atribuye al parámetro de fichero de inicio SGA_TARGET.

Si se indica SGA_TARGET, las zonas siguientes de memoria se recortan utomaticamente:

  • Buffer cache (DB_CACHE_SIZE)

  • Shared pool (SHARED_POOL_SIZE)

  • Large pool (LARGE_POOL_SIZE)

  • Java pool (JAVA_POOL_SIZE)

  • Streams pool (STREAMS_POOL_SIZE)
  • campo FLGAUT
  • campo TXTTOT

Tamaño total de la memoria SGA.

El tamaño fijo de la SGA corresponde a la memoria dedicada a la zona SGA que contiene datos generales sobre el estado de la base de datos y sobre la instancia.

Tamaño total de la SGA = pool compartido + caché sello + zona de memoria LARGE POOL + pool java + sello diario.

  • campo VALTOT
  • campo UNITOT
  • campo TXTMAX

Tamaño máximo alcanzado por la memoria SGA desde el inicio de la base de datos.

  • campo VALMAX
  • campo UNIMAX

  • Descripción (campo INFPAR)

Gráfico que presenta cada componente SGA y el tamaño de la memoria que emplea.

  • Memoria SGA (campo INFVAL)

Tabla Número 1

  • campo PARAM

Cuadro que presenta cada componente SGA y el tamaño de la memoria que emplea.

  • campo VALEUR
  • campo UNITE

Memoria PGA

  • campo TXTPGACUM

Acumulación de objetivos PGA: número de objetivos para el conjunto de las memorias PGA.

  • campo VALPGACUM
  • campo UNIPGACUM
  • campo TXTPGA

Cantidad memoria PGA en la actualidad: memoria PGA en curso.

  • campo VALPGA
  • campo UNIPGA
  • campo TXTPGAMAX

Cantidad máxima cedida a memoria PGA (desde el arranque): memoria PGA máxima concedida.

  • campo VALPGAMAX
  • campo UNIPGAMAX
  • campo TXTPGARAT

Porcentaje de éxitos en memoria caché: frecuencia con la que se encuentran los datos en la memoria caché, evitando el acceso al disco.

  • campo PGARAT
