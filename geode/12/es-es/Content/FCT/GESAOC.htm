Objetos

Algunas de las características de los objetos pueden ser modificadas por parametrización. Estas características almacenadas en la tabla 'objetos' no son actualizadas cuando se cambia de versión (no sufren ningún cambio pues en caso de evolución de versión).

La función de personalización de objeto permite definir estas características. Son las de las ventanas de selección y las listas izquierda. También se define la presencia de funciones asociadas (impresión, lista, estadísticas) y los vínculos automáticos del explorador de enlaces.

Gestión de Pantalla

Cabecera

La modificación de las características de un objeto se lleva a cabo introduciendo la siguiente información en dos pestañas:

  • Código objeto (campo ABREV)

El código de un objeto está constituido de 1 a 3 caracteres alfanuméricos. Las palabras reservadas de Adonix están prohibidas.

  • Descripción (campo ZLIBEL)

Título asociado al código anterior.

Pestaña Selección

En la primera pestaña, se define:

  • el orden del barrido de los datos en la lista izquierda principal y en las ventanas de selección.
  • los datos que se indican en estas selecciones (puede haber hasta 15 columnas en las listas de izquierda y selecciones).
  • funciones a las que se accede desde el menú Fichero.

Los datos que aparecen en la lista izquierda proceden de la tabla principal del objeto o de una tabla vinculada, o incluso de las expresiones que utilicen campos procedentes de estas tablas.

Pantalla de selección

  • Índice (campo SELCLE)

Este campo permite definir un indice de barrido del fichero principal para su visualización en la lista izquierda y la de las ventanas de selección. Si el campo está vacío, se utiliza el primer índice de la tabla.
  • Sentido (campo SELORD)

Este campo permite definir un sentido de barrido de la tabla principal para su visualización en la lista izquierda y la de las ventanas de selección.

  • Lista jerarquizada (campo SELTREE)

Cuando la clave de barrido escogida para la lista izquierda está compuesta de varias partes, la lista se realiza presentando únicamente la primera parte de la clave. Un doble clic, en uno de los valores de las claves, permite explorar los subvalores de la clave en el número de niveles de las subclaves existentes. Una vez exporados todos los niveles de las claves se puede seleccionar un registro de la tabla.

Un ejemplo de este funcionamiento puede observarse en el objeto de gestión de plantas: cada planta dependiendo de una sociedad, se visualiza, primero, la lista de sociedades y a continuación puede ser visualizada una sociedad con un doble clic: aparece la lista de ubicaciones de la sociedad y se selecciona una de ellas.

si la casilla no está marcada, se trata de una lista simple.

  • N.º caracteres (campo SELCAR)

Todo campo de una pantalla cuyo tipo está vinculado a un objeto, dispone de una ventana de selección a la que se puede acceder haciendo click en el botón derecho o mediante .

Este valor permite definir el número de caracteres utilizados para preceisar dónde debe comenzar la selección, en función de lo que ha sido introducido en el campo previo a la selección. Si este valor es N, la selección se inicia a partir de la primera clave cuyos N primeros caracteres sean superiores o iguales a la cadena introducida. Así:

  • Si se parametriza el valor 2, y si se abre la cadena DEF antes de abrir la selección, la ventana de selección comenzará la búsqueda a partir del primer valor de la clave superior o igual a DE (si existen claves que comiencen por DEA, son éstas las que aparecerán al principio de la ventana). Si se abre H, la selección comenzaría por la primera clave superior o igual a H.
  • Si se da a esta zona valor 0, y se abre la cadena UVW antes de abrir la selección, la selección comenzará a partir de A y no está limitada.

Tabla Campos de selección

  • Tabla (campo SELFIC)

Aquí se introduce la tabla en la que se encuentra el campo a mostrar. Esta tabla es, bien la tabla principal, bien una de las tablas definidas como vinculadas a la tabla principal en la definición del objeto.

Esta zona puede quedar vacía si se muestra una expresión calculada.

  • Campo (campo SELZON)

Aquí se capturan los campos presentes en la lista izquierda. Proceden de la tabla definida anteriormente.

  • Expresión (campo SELEXP)
  • Descripción (campo ZSELINT)

Este título se utiliza como títular de la columna y también para la lista izquierda y las ventanas de selección del objeto.

Por defecti, el título del campo está relleno. Este titular puede traducirse. Así, si se captura un título no conocido, la ventana de elección que permite crear un nuevo texto en el diccionario, se abrirá de forma automática.

  • Tipo (campo SELTYP)

Este tipo de datos se utiliza para formatear la expresión precedente en la lista izquierda.

  • Opciones (campo SELSAI)
  • Longitud (campo SELLNG)
  • Nivel (campo SELPCK)

Este título se utiliza como títular de la columna y también para la lista izquierda y las ventanas de selección del objeto.

Por defecti, el título del campo está relleno. Este titular puede traducirse. Así, si se captura un título no conocido, la ventana de elección que permite crear un nuevo texto en el diccionario, se abrirá de forma automática.

Informes

  • Impresión (campo RPT1)

Este código de impresión define el o los informes lanzados haciendo una elección en el menú Fichero / Imprimir. Hay que tener en cuenta que los rangos del informe pueden incarse a partir de la ficha actual: sólo se puede acceder a la opción Fichero / Imprimir si hay una ficha en la pantalla.

Por defecto, cuando se utilicen estos botones, el aplicativo intentará editar los informes cuyos nombres se indican aquí. Pero existe una tabla de indirección que permite definir, a partir del código interno aquí indicado, uno o varios informes (por ejemplo, la impresión de los albaranes de preparación y albarán de entrega en el objeto de las entregas)
  • Lista (campo RPT2)

Varios

  • Estadísticas (campo STA)

Esta casilla a marcar permite activar el botón 'Estadísticas' en el objeto. Este botón permite traer las cuestionarios estadísticas cuyo primero o primeros criterios se definen por los datos vinculados a la tabla principal o a las tablas vinculadas al objeto.

  • Bloqueo en modificación (campo MLOCK)

Permite desencadenar el cierre lógico cuando se entra en modificación de un registro, mientras que en estándar, está colocado desde la visualización del registro. Para entrar en modificación, hay que hacer click sobre la clave.

Pestaña Browser

En esta pestaña se define el tamaño de las listas izquierda y la presencia de listas izquierda complementarias en el objeto:

  • la lista de los últimos registros leídos (que presenta los últimos registros a los que ha accedido el usuario, al igual que los sistemas de ofimática, que permiten ver los útlimos documentos que se han abierto).
  • el explorador de vínculos, una persiana que permite acceder a otras fichas vinculadas a la ficha actual mediante hipervínculos parametrizables.

La tabla siguiente permite, si está presente el explorador de vínculos, definir cómo crear automáticamente un vínculo entre la ficha actual (mientras se crea o modifica) y otras fichas. Estos vínculos automáticos se crean en el grupo de vínculo definido por el parámetro LIAISAUTO que puede ser diferente del grupo de vínculo que ve el usuario (definido por el parámetro GRPLIAISON).

Explorador de vínculos

  • Activo (campo EXPLORE)

Si esta casilla está marcada, el explorador de enlaces está presente en las listas izquierdas del objeto.

  • Descrip. corta (campo ZLIBSHO)

Este campo permite una descripción corta que será sustituída por la descripción del objeto en la designación del enlace presente en el explorador de enlaces.   Para obtener enlaces con un objeto de descripción larga (Curso de divisasCurso, por ejemplo), es conveniente ofrecer un nombre corto (, por ejemplo), para poder visualizar una descripción tipo Utiliza curso XXX en vez de Utiliza curso de divisas XXX, ya que sería demasiado largo para ser visualizado en su totalidad en el explorador de enlaces.

Últimos leídos

  • Activo (campo DERLU)

Este campo permite activar/desactivar la lista izquierda Últimos leídos.
  • En primera posición (campo FIRLIS)

Este campo permite colocar en primera posición la lista izquierda de 'Últimos leídos'.

Tabla Vínculos automáticos

  • Objeto (campo DSTOBJ)

Este campo permite introducir el objeto de origen del enlace (se considera que los enlaces automáticos de un objeto se hacen hacia la ficha actual). Así pues, por ejemplo, si se desea realizar un enlace entre una sociedad (objeto CYP en curso de parametrización) y un usuario (campo CREUSR en la ficha de sociedad), será necesario definir un enlace automático en el objeto de pedido, ofreciendo AUS como objeto de enlace y [CYP]CREUSR como clave de enlace. El resultado será un enlace con la sociedad en el explorador de la ficha del usuario.
  • Expresión de vínculo (campo EXPLIEN)

Este campo permite identificar el objeto de enlace mediante una expresión (o una serie de expresiones separadas por un punto y coma si la clave de enlaces se presenta en varias partes). Esto permite identificar la ficha (del objeto de enlace) del enlace.

Tener en cuenta los límites de definición de estos enlaces automáticos. De hecho, sólo pueden utilizarse en las expresiones de vínculos de los campos de la tabla principal. Teniendo en cuenta la estructura de la tabla de enlaces, será posible completar esta tabla con una personalización.
  • Código vínculo (campo LNK)

El código de vínculo permite dar un significado al vínculo automático definido. Este código está definido en la tabla diversa número 61.
  • Tabla (campo BASPAG)

Este campo opcional permite definir, cuando se dimensiona, la variable de "base de cuadro" que define el número de citas de la tabla. Esto es lo que permite hacer que el índice actual varíe para obtener varios vínculos. Esto significa que se va a utilizar la variable índice en la expresión de enlace.

Notas

  • Campos obligatorios en ela lista izquierda
  • Informes lanzados desde los objetos
  • Valores por defecto

Campos obligatorios en ela lista izquierda

Es importante (para evitar errores en la ejecución en gestión de objeto) que los campos de la clave de barrido de la lista izquierda sean definidos en esta última.

Informes lanzados desde los objetos

Los códigos de informes definidos en gestión de objeto son códigos internos asociados por defecto a un informe con el mismo código. Sin embargo, esta asignación puede ser modificada por medio de una tabla indirecta. Esta tabla, a la que se puede acceder con la función Códigos de impresión, sirve para asociar a un código interno el código de otro u otros informes según la lengua del usuario. En caso de que existan varios informes, se puede seleccionar uno en la ejecución. La función Valores por defecto  permite definir, bajo la forma de expresiónes calculadas según el contexto del objeto, los valores por defecto para los parámetros de lanzamiento del informe.

Valores por defecto

Otra función permite personalizar los objetos autorizando:

  • la definición de valores por defecto en el momento de creación de la ficha.
  • colocar filtros por defecto.

Pero esta función, llamada Navegación, es una función de desarrollo. Por tanto es más delicada de manejar, y hay que tomar precauciones particulares para que el resultado quede perenne en caso de cambio de versión.
