Objetos

La gestión de objetos es el origen de la mayoría de las funciones del sistema. Un objeto está relacionado con la gestión completa de las fichas de una tabla o de un grupo de tablas (creación, consulta, modificación, anulación, etc.). Por ejemplo gestión de clientes, gestión de facturas, introducción de asientos... se implementan en la aplicación en forma de objetos.

Un objeto se presenta en una de las dos formas principales siguientes:

  una ventana compuesta de una cabecera y de una o varias pestañas situadas en la parte derecha, y de listas de selección situadas a la izquierda ("browser"). Al mismo tiempo se realiza el registro.

  una ventana compuesta de una pantalla única que contiene la lista de registros en una tabla.

Este objeto permite gestionar un registro en una tabla (por ejemplo, una ficha terceros) o en varias (la cabecera de pedido y las líneas correspondientes).

Además de la lista izquierda principal del objeto, es posible definirlo de antemano. Esto se hace mediante la declaración de un objeto de tipo browser para cada lista izquierda suplementaria.

Cabecera

El encabezado permite identificar el objeto.

  • Código objeto (campo ABREV)

El código de un objeto está constituido de 1 a 3 caracteres alfanuméricos. Las palabras reservadas de Adonix están prohibidas.

  • Descripción (campo ZLIBEL)

Destinado a figurar en las pantallas y los informes en los que el código de la ficha puede introducirse o seleccionarse. Este texto permite aportar una descripción más clara de la ficha afectada.
  • Descrip. corta (campo ZLIBSHO)
  • Tabla vinculada (campo NOMFIC)

Código de la tabla principal gestionada por el objeto

  • Modelo de datos (campo MODELE)

Pestaña General

General

  • Módulo (campo MODULE)

Indica el módulo de aplicación al que está vinculada la ficha (y el código, si lo tiene). Las fichas utilizadas por varios módulos se clasifican en uno de ellos tronco común ou supervisor.

Un desarrollo asociado al módulo (y el código correspondiente) sólo puede utilizarse si el módulo está activo en el dossier actual.

  • Título parámetro (campo ZLIBPAR)

En el caso de que el objeto gestionado no se identifica con una clave simple sino con una clave de 2 componentes (por ejemplo una tabla diversa donde el primer componente representa el número de tabla ), el primer componente de la clave se convierte en un parámetro para la gestión del objeto. Se debe indicar la descripción de este parámetro (Número de tabla, en el ejemplo).

 
  • Código actividad (campo CODACT)

Si está definido, un código de actividad asociado a un objeto permitirá convertirlo en opcional

  • Campo planta (campo ZSITE)

Aquí se define el campo de la ficha principal donde se encuentra el código de la planta al que se asocia el registro.
El hecho de definir esta planta permite generar automáticamente las restricciones de acceso a los datos de una planta.
  • Campo cód.acceso (campo ZACC)

Aquí se define el campo de la ficha principal donde se encuentra el código de acceso asociada al registro.

El hecho de definir este código permite generar automáticamente las restricciones de acceso a los datos de un código de acceso.

Bloque Número 2

  • Tipo de gestión (campo TYPGES)

Vínculos

  • Menú (campo MENU)

Esta zona define el código del menú en el que se presentará la función de gestión del objeto para el usuario ADMIN.

  • Fila en menú (campo RANG)

Indicar el rango donde aparecerá la función en el menú.
En un menú, las funciones se ordenarán de forma creciente de los rangos.
  • Impresión (campo RPT1)

Este código de impresión define el o los informes lanzados haciendo una elección en el menú Fichero / Imprimir. Hay que tener en cuenta que los rangos del informe pueden incarse a partir de la ficha actual: sólo se puede acceder a la opción Fichero / Imprimir si hay una ficha en la pantalla.

Por defecto, cuando se utilicen estos botones, el aplicativo intentará editar los informes cuyos nombres se indican aquí. Pero existe una tabla de indirección que permite definir, a partir del código interno aquí indicado, uno o varios informes (por ejemplo, la impresión de los albaranes de preparación y albarán de entrega en el objeto de las entregas)
  • Lista (campo RPT2)
  • Petición (campo RQT)

Permite proponer peticiones en el menún Fichero/lista de la función cuando no hay ninguna lista prevista. Si este campo está indicado, el supervisor propondrá todas las peticiones y peticiones sql del campo GROUPE igual la valor introducido. Si no está indicado, el valor que se tendrá en cuenta es el del código objeto. Si no se ha parametrizado ninguna petición, el supervisor propondrá la creación automática de una petición, estructurada por las columnas de la lista izquierda.

Opciones

  • Estadísticas (campo STA)

Esta casilla a marcar permite activar el botón 'Estadísticas' en el objeto. Este botón permite traer las cuestionarios estadísticas cuyo primero o primeros criterios se definen por los datos vinculados a la tabla principal o a las tablas vinculadas al objeto.

  • Supresión diferida (campo DELDEF)

El supervisor autorizará o denegará las supresiones diferidas en la tabla principal y las tablas secundarias gestionadas por el objeto. Esta posibilidad existe para los usuarios que estén parametrizados en « confirmación de supresión ». Cuando este usuario suprima una ficha del objeto, se mostrará una ventana de confirmación. En esta ventana, el usuario puede marcar la casilla «supresión diferida». La ficha existirá mientras no se lance el utilitario de supresión diferida

  • Import. (campo IMPORT)

Esta zona permite indicar que el objeto es importable mediante la función import-export.

  • Bloqueo en modificación (campo MLOCK)

Permite desencadenar el cierre lógico cuando se entra en modificación de un registro, mientras que en estándar, está colocado desde la visualización del registro. Para entrar en modificación, hay que hacer click sobre la clave.

 
  • Parámetro de consulta (campo DSYPAR)

Sobre un objeto que gestiona dos elementos de clave más allá de la planta y del depositario, se indica que estos elementos aparecen en caso de error en un registro.

 
  • Gestión planta/depositario (campo FCYDEP)

Se indica si hay una noción de planta o de depositante en esta función. El concepto de depositante se determina por la presencia del campo DEP en la clave primaria de la tabla vinculada al objeto.

Los automatismos del supervisor, para una función, en el sujeto de la planta o del depositante, son los siguientes:

  • entrada en la función no autorizada, si la planta y depositante actual no están posicionados.
  • filtro automático de la lista izquierda y la lista de los últimos leídos en función de la planta y el depositante actual.
  • alimentación de la planta y del depositante creación del registro
  • cambio de ambiente posible para el menú: Visualización / Cambio de entorno

Pestaña Selección

Se recuperan todos los datos que permiten construir la lista izquierda principal del objeto, así como la ventana de selección. Además, existe la posibilidad de definir opciones de selección.

Las opciones de selección

Este cuadro permite definir los filtros de selección de los registros de la tabla gestionada. Estos filtros, expresadas en forma de condiciones lógicas en los campos de la tabla y posibles constantes, están identificadas por una letra (mayúscula para el estándar y minúscula para el específico). Después se posicionan o no sobre lo campos con un tipo de dato ligado a ese objeto.

Ejemplo:

Al introducir pedidos de venta, sólo se quiere autorizar a los clientes activos. Para ello, se define un filtro de selección en el objeto Cliente, con la condición BPCSTA=2, y con el título «Cliente activo». En la pantalla de pedidos SOH0, el campo cliente BPCORD es de tipo dato BPC; este tipo está vinculado al objeto BPC. En el menú contextual del tipo de dato BPC, el menú "opciones" permite colocar o no los filtros declarados en los objetos de los clientes.

Definición de las pantallas de selección
(lista izquierda, pantalla de selección y de selección avanzada)

La lista de columnas que debe aparecer en la lista izquierda es introducida en el cuadro inferior de la segunda pestaña. Estas columnas pueden ser campos de la tabla principal o de una tabla secundaria que se identifica en la tercera pestaña. Éstas pueden ser una expresión; en este caso, se precisará el tipo de datos para almacenar el resultado de la expresión.

Tabla Opciones selección

  • Título opción (campo ZOPTLIB)

Este cuadro permite definir las opciones de gestión del objeto. Una opción permite caracterizar sub-conjuntos de registros identificados por una condición (clientes activos, cuentas marcables, etc.).

Una opción se define por:

  • Una letra clave
  • Un título
  • Una condición lógica
  • Un mensaje de error que aparece si el objeto no responde a la opción.
  • Código (campo OPTCOD)

Este cuadro permite definir las opciones de gestión del objeto. Una opción permite caracterizar sub-conjuntos de registros identificados por una condición (clientes activos, cuentas marcables, etc.).

Una opción se define por:

  • Una letra clave
  • Un título
  • Una condición lógica
  • Un mensaje de error que aparece si el objeto no responde a la opción.
  • Condición opción (campo OPTCND)
  • Mensaje error (campo ZOPTERR)

Pantalla de selección

  • Índice (campo SELCLE)

Este campo permite definir un indice de barrido del fichero principal para su visualización en la lista izquierda y la de las ventanas de selección. Si el campo está vacío, se utiliza el primer índice de la tabla.
  • Sentido (campo SELORD)

Este campo permite definir un sentido de barrido de la tabla principal para su visualización en la lista izquierda y la de las ventanas de selección.

  • Opciones selección (campo SELOPT)

Esta zona permite precisar una o varias opciones para los registros presentes en la ventana de selección.
  • Lista jerarquizada (campo SELTREE)

Cuando la clave de barrido escogida para la lista izquierda está compuesta de varias partes, la lista se realiza presentando únicamente la primera parte de la clave. Un doble clic, en uno de los valores de las claves, permite explorar los subvalores de la clave en el número de niveles de las subclaves existentes. Una vez exporados todos los niveles de las claves se puede seleccionar un registro de la tabla.

Un ejemplo de este funcionamiento puede observarse en el objeto de gestión de plantas: cada planta dependiendo de una sociedad, se visualiza, primero, la lista de sociedades y a continuación puede ser visualizada una sociedad con un doble clic: aparece la lista de ubicaciones de la sociedad y se selecciona una de ellas.

si la casilla no está marcada, se trata de una lista simple.

  • N.º caracteres (campo SELCAR)

Todo campo de una pantalla cuyo tipo está vinculado a un objeto, dispone de una ventana de selección a la que se puede acceder haciendo click en el botón derecho o mediante .

Este valor permite definir el número de caracteres utilizados para preceisar dónde debe comenzar la selección, en función de lo que ha sido introducido en el campo previo a la selección. Si este valor es N, la selección se inicia a partir de la primera clave cuyos N primeros caracteres sean superiores o iguales a la cadena introducida. Así:

  • Si se parametriza el valor 2, y si se abre la cadena DEF antes de abrir la selección, la ventana de selección comenzará la búsqueda a partir del primer valor de la clave superior o igual a DE (si existen claves que comiencen por DEA, son éstas las que aparecerán al principio de la ventana). Si se abre H, la selección comenzaría por la primera clave superior o igual a H.
  • Si se da a esta zona valor 0, y se abre la cadena UVW antes de abrir la selección, la selección comenzará a partir de A y no está limitada.

Tabla Número 2

  • Tabla (campo SELFIC)

Aquí se introduce la tabla en la que se encuentra el campo a mostrar. Esta tabla es, bien la tabla principal, bien una de las tablas definidas como vinculadas a la tabla principal en la definición del objeto.

Esta zona puede quedar vacía si se muestra una expresión calculada.

  • Campo (campo SELZON)

Aquí se capturan los campos presentes en la lista izquierda. Proceden de la tabla definida anteriormente.

  • Expresión (campo SELEXP)
  • Descripción (campo ZSELINT)

Este título se utiliza como títular de la columna y también para la lista izquierda y las ventanas de selección del objeto.

Por defecti, el título del campo está relleno. Este titular puede traducirse. Así, si se captura un título no conocido, la ventana de elección que permite crear un nuevo texto en el diccionario, se abrirá de forma automática.

  • Tipo (campo SELTYP)

Este tipo de datos se utiliza para formatear la expresión precedente en la lista izquierda.

  • Opciones (campo SELSAI)

Esta información permite formatear los datos eligiendo una opción. Las opciones dependen del tipo de datos (utilizar la ventana de búsqueda).

  • Longitud (campo SELLNG)

Es posible indicar un tamaño de columna alfanumérica construida por una expresión. Si no está indicado, el tamaño por defecto es de 30 caracteres.

  • Nivel (campo SELPCK)

se indica el nivel de jerarquía. Esto permite disociar los niveles de jerarquía de la lista izquierda, de la estructura de la clave. Este parámetro no basta como tal; necesita ser desarrollado.

Pestaña Entorno

Permite definir un conjunto de tablas necesarias para la ejecución de la función, el vínculo tablas pantallas para la gestión del importe, las pantallas complementarias accesibles a través de un botón o menú por las webs de servicios.

Tabla Número 1

  • Tablas a abrir (campo TABFIC)

Tablas suplementarias a la tabla de base gestionada por el objeto. Esta tablas se abren y se cierran de forma automática a la ejecución del objeto, bajo la abreviatura indicada, o bajo la abreviatura del diccionario. Los campos de estas tablas son accesibles en el objeto.

Además, los campos de estas tablas pueden utilizarse para definir una columna de lista izquierda y pantallas de selección desde que se indica el vínculo. Es posible definir vinculos jerarquizados, a condición de indicar estas tablas en el cuadro, en el orden que establece la jerarquía.

La clave de vínculo no debe tener homónimos. Si no se indica, el supervisor toma la clave primaria.

  • Índice (campo TABCLE)
  • Expresión de vínculo (campo TABLIEN)
  • Abrev (campo TABABR)
  • Código actividad (campo TABACT)

Tabla Pantallas complementarias

  • Pantallas (campo SCRNAM)

Para las WEB de servicios, se indican las pantallas suplementarias utilizadas en la gestión del objeto, desde un botón en la parte inferior de la pantalla o un menú.

 
  • Abreviaturas (campo SCRABR)
Añadido enlaces ATEXTRA

Este botón permite crear automáticamente los enlaces necesarios hacía la tabla de los textos traducibles ATEXTRA.

Pestaña Vistas

Pemite definir, un conjunto de vistas necesarias en la ejecución de la función. Estas vistas serán abiertas y cerradas en el proceso generado asociado al objeto WOobjeto.

 

Tabla

Esta vistas se abren y se cierran de forma automática a la ejecución del objeto, bajo la abreviatura indicada, o bajo la abreviatura del diccionario. Los campos de estas vistas son accesibles en el objeto.

  • Abrev (campo VUEABR)
  • Código actividad (campo VUEACT)
